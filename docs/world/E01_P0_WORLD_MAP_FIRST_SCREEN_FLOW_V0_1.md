## 0. 판정 요약

- P0 첫 화면은 E01 전체를 보여주는 화면이 아니다.
- 처음 선명하게 보이는 것은 `R08+R01`의 5개 노드뿐이다.
- P0의 목적은 세계 설명이 아니라 `보급소 -> 인양 장비 -> 첫 출격 -> 기본 런 -> 첫 보스` 흐름을 몸으로 이해시키는 것이다.
- P1/P2 지역은 첫 화면에서 자세히 보이지 않는다. 남/남서, 동/도심, 남부에 흐린 신호만 남긴다.
- R08은 안전 왕국이 아니라 작은 공백이다.
- R01은 첫 전투 지역이며, 집과 광고가 사람을 가족/입주자로 등록하려는 생활형 공포를 보여준다.
- 첫 보스는 괴물이 아니라 `가족 적합성 심사` 절차의 얼굴이다.

## 1. 첫 화면 목표

첫 화면에서 유저가 이해해야 하는 것:

| 목표 | 화면으로 보여주는 방법 |
|---|---|
| 여기는 지구 기반 월드맵이다 | 해안선/도시 노이즈는 희미하게 두되 실명 지명은 숨긴다. |
| 보급소는 완전 안전지대가 아니다 | 보급소 주변만 신호가 낮고, 바깥은 캠페인 노이즈가 흐른다. |
| 출격은 장비 기반이다 | R08-L02 역송출 주유소를 인양/출격 기준점으로 보여준다. |
| 광고는 환경이다 | R01 주택가 노드가 문, 우편함, 모델하우스, 전단 신호로 보인다. |
| 첫 목표는 R01-L03이다 | 모델하우스 결절은 처음부터 약한 보스 신호로 보이지만 바로 진입하지 못한다. |

첫 화면에서 아직 설명하지 않는 것:

- 전 지구 96개 캠페인.
- E02 이후 광역권.
- R05 방송 중추의 진실.
- R07 탈출 가능성.
- 윤서의 반품 담당자 핵심 과거.
- 라이프라인의 정체.

## 2. 화면 구성

모바일 세로 화면을 1차 기준으로 둔다. PC/가로 화면은 같은 정보 구조를 넓게 배치한다.

| 영역 | 내용 | 역할 |
|---|---|---|
| 상단 상태줄 | 보급소명, 보급태그 요약, 현재 신호 상태 | 현재 위치와 자원 압박 |
| 중앙 월드맵 | R08/R01 P0 노드 5개, 흐린 P1 신호 | 실제 선택과 이동 |
| 하단 노드 패널 | 선택 노드 이름, 위험, 인양 상태, 목표, 예상 회수물 | 출격 판단 |
| 하단 주요 버튼 | 출격/귀환/정비/잠김 상태 버튼 | 행동 결정 |
| 배경 노이즈 | 남/남서 물류, 동/도심 의료, 남부 산업의 흐린 신호 | 세계가 더 크다는 암시 |

주의:

- 화면 안에 긴 튜토리얼 설명문을 넣지 않는다.
- 처음부터 지역 설명을 장문으로 보여주지 않는다.
- 노드 패널의 문장은 짧고, 행동 판단에 필요한 정보만 둔다.

## 3. 첫 화면 노드 상태

첫 화면에 표시되는 P0 노드는 5개다. 다만 첫 순간부터 모두 출격 가능한 것은 아니다.

| 노드 | 화면 상태 | 클릭 가능 | 출격 가능 | 첫 화면 표현 |
|---|---|---|---|---|
| R08-L01 침묵 보급소 | 현재 위치 | 가능 | 불필요 | 선택된 허브, 낮은 신호 공백 |
| R08-L02 역송출 주유소 | 해금됨 | 가능 | 불필요 | 인양 기준점, 장비실 아이콘 |
| R01-L01 침묵 가장자리 | 출격 가능 | 가능 | 가능 | 첫 출격 노드, 낮은 위험 |
| R01-L02 분양 주택 루프 | 잠김 | 가능 | 불가 | 흐린 주택 신호, `경로 미확인` |
| R01-L03 모델하우스 결절 | 약한 보스 신호 | 가능 | 불가 | 멀리 깜빡이는 결절, `심사 신호 감지` |

P1/P2 첫 화면 처리:

| 구역 | 첫 화면 표현 | 이유 |
|---|---|---|
| R03 물류 회수 벨트 | 남/남서에 흐린 라벨 노이즈 | 윤서 과거를 너무 빨리 공개하지 않는다. |
| R02 의료 격리 구역 | 동/도심에 약한 백색 신호 | 등록 인간 상태를 첫 화면에서 노출하지 않는다. |
| R04 산업 오염 지대 | 남부에 회색 얼룩 | 해방 로봇 서사는 P1 이후로 미룬다. |
| R05 방송 중추 지구 | 보이지 않거나 원격 잡음만 | 세계 진실 후반용. |
| R06 상업/멤버십 지구 | 보이지 않음 | 인간 브로커 정보는 첫 화면에서 숨긴다. |
| R07 항만/이송 봉쇄 구역 | 흐린 해안선만 | 탈출 미스터리를 보류한다. |

## 4. P0 해금 흐름

```text
S0 월드맵 최초 진입
-> S1 R01-L01 첫 출격 완료
-> S2 R01-L02 기본 런 해금
-> S3 R01-L02 첫 정산 완료
-> S4 R01-L03 모델하우스 결절 해금
-> S5 R01-L03 첫 보스 처리
-> S6 P1 약한 신호 개방
```

| 상태 | 조건 | 월드맵 변화 |
|---|---|---|
| S0 | 월드맵 첫 진입 | R08-L01 선택됨. R08-L02와 R01-L01만 명확히 사용 가능. |
| S1 | R01-L01 최초 귀환 또는 인양 | R01-L02 경로가 선명해진다. R01-L03 결절 신호가 한 번 깜빡인다. |
| S2 | R01-L02 최초 출격 가능 | 분양 주택 루프가 기본 런 노드로 열린다. |
| S3 | R01-L02 첫 정산 완료 | R01-L03 모델하우스 결절의 잠금 문구가 바뀐다. |
| S4 | R01-L03 진입 조건 충족 | 첫 보스 노드가 출격 가능해진다. |
| S5 | R01-L03 처리 완료 | P0 완료. R01 주변 노이즈가 변하고 P1 약한 신호가 열린다. |
| S6 | P1 프리뷰 개방 | R01-L04/R01-L05, R03/R02/R04 약한 신호 표시. |

## 6. 버튼 문구 기준

자원명과 인양 등급은 `docs/world/WORLD_TERMINOLOGY_LOCK_V0_1.md`를 따른다. 아래 표는 버튼 톤을 고르기 위한 v0.1 후보이며, 핵심 용어 잠금과 충돌하면 안 된다.

| 상황 | 1차 후보 | 대체 후보 | 비고 |
|---|---|---|---|
| 출격 가능 | 출격 | 진입 | 모바일 액션 RPG 톤에는 `출격`이 가장 명확하다. |
| 허브 선택 | 정비 | 정산 | 보급소 기능이 정산 중심이면 `정산`도 가능하다. |
| 장비 노드 | 장비 확인 | 인양 장비 | R08-L02가 단순 스테이지가 아님을 보여준다. |
| 잠긴 노드 | 경로 미확인 | 신호 부족 | R01-L02에 적합하다. |
| 보스 약한 신호 | 심사 신호 감지 | 결절 신호 감지 | R01-L03 첫 노출에 적합하다. |
| 원격 지역 | 신호만 감지 | 수신 불안정 | P1/P2 프리뷰용. |

## 7. 출격/인양 표시

P0에서는 인양 등급을 복잡하게 설명하지 않는다. 유저용 문구는 짧게 둔다.

| 제작용 내부 등급 | P0 표시 | 사용 노드 | 의미 |
|---|---|---|---|
| A0 | 귀환 안정 | R08-L01, R08-L02 | 보급소 내부. 출격보다 정비/정산 영역. |
| A1 | 인양 가능 | R01-L01~R01-L03 | HP 0 시 긴급 인양 가능. 손실은 있을 수 있음. |
| A2 이상 | 신호 흐림 | P1/P2 프리뷰 | 첫 화면에서는 자세한 위험을 숨긴다. |

주의:

- `A0`, `A1`, `A2`는 제작용 내부 등급이다.
- 유저에게는 `귀환 안정`, `인양 가능`, `신호 흐림`만 보여준다.

HP 0 후 P0 기본 문구:

```text
비콘 신호 수신. 긴급 인양을 시작합니다.
```

인양 후 정산 화면 짧은 문구:

```text
인양 완료. 회수품을 정산합니다.
```

주의:

- P0에서는 인양을 죽음으로 보이게 하지 않는다.
- 인양을 무료 순간이동처럼 보이게 하지 않는다.
- 비콘, 회수선, 역송출 주유소가 연결되어 있다는 느낌을 남긴다.

## 8. 잠긴 노드와 흐린 신호 표현

| 표현 | 사용하는 곳 | 의미 |
|---|---|---|
| 얇은 점선 연결 | R01-L02 잠금 상태 | 길은 보이지만 아직 안전하게 고정되지 않음 |
| 희미한 결절 점멸 | R01-L03 초기 상태 | 보스 신호는 감지되지만 진입 조건 부족 |
| 낮은 노이즈 공백 | R08 주변 | 침묵권 |
| 흐린 라벨 조각 | R03 방향 | 물류/반품 신호가 멀리 있음 |
| 백색 잡음 | R02 방향 | 의료권 신호가 아직 해석되지 않음 |
| 회색 얼룩 | R04 방향 | 산업 오염권이 있다는 암시 |

금지:

- P1/P2 노드명을 첫 화면에서 모두 텍스트로 보여주지 않는다.
- R05/R07을 첫 화면에서 선택 가능한 것처럼 보이게 하지 않는다.
- 잠긴 노드를 단순 레벨 제한처럼 보이게 하지 않는다. 신호/인양/경로 고정 문제로 보이게 한다.

## 9. P0 완료 후 P1 개방

R01-L03 처리 후 바로 모든 P1을 열지 않는다. 먼저 약한 신호 상태로 열고, 이후 보급소 정산/대화/회수물 해석을 통해 갈래를 확정한다.

| P1 축 | 최초 표시 | 실제 해금 조건 후보 |
|---|---|---|
| R01 재방문 | 주택가 주변의 새 점선 | R01-L03 처리 후 자동 |
| R03 물류 회수 | 남/남서 라벨 노이즈 선명화 | R01-L03 정산품에서 반품 라벨 해석 |
| R02 의료 격리 | 동/도심 백색 신호 선명화 | R01에서 환자 팔찌/진료태그 흔적 회수 |
| R04 산업 오염 | 남부 회색 얼룩 선명화 | R03 또는 R01에서 고장난 로봇 부품 회수 |
| R06 상업/멤버십 | 아직 숨김 | R03/R02에서 브로커 단서 획득 후 |
| R08 확장 | 폐역/저장고 신호 표시 | 첫 보스 후 보급소 유지비 이벤트 |
