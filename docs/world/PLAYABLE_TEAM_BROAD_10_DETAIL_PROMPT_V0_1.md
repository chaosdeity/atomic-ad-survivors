이 문서는 플레이어블 제작팀에게 넘길 `10번 Nora Weiss / BROAD-10 / 노라, 공영 안내 수신자` 상세 결재 작업용 복붙 프롬프트다.

## 3. 핵심 잠금

BROAD-10의 핵심 질문은 유지한다.

```text
공식 안내가 계속 반복되면,
그것은 도움인가 명령인가?
```

핵심 문장 후보:

```text
분명하게 들린다고 해서,
동의한 것은 아니다.
```

최종 변화 문장 후보:

```text
정확한 문장도 사람을 대신 살아주지는 못한다.
```

BROAD-10은 신호를 끊는 캐릭터가 아니다.

```text
노라는 침묵으로 숨는 사람이 아니라,
너무 선명한 공식 신호가 사람을 안내 수신자/대피 승인자/반복 문구 대상으로 고정하기 전의 판정을 늦추는 캐릭터다.
```

### 4.3 QUIET-07과 분리

반드시 비교한다.

```text
QUIET-07 백여울:
낮은 수신, 약한 인양, 덜 들키지만 덜 구해지는 침묵권 리스크.

BROAD-10 노라:
너무 선명한 공식 수신, 청취 확인, 대피 문장 승인, 반복 송출에 붙잡히는 리스크.
```

금지:

```text
노라를 침묵권 피난자 2호로 만들지 않는다.
노라의 능력을 단순 전파 차단으로 만들지 않는다.
노라의 HP 0을 수신 공백/저소음 비콘 지연으로 쓰지 않는다.
```

### 4.4 C05 계열 다른 캠페인과 분리

반드시 비교한다.

```text
E01_C05:
마지막 수신탑.
방송 중추, 반복 뉴스, 세계 진실 미스터리.

E02_C05:
재난안내 생방송권.
도시 방재 스튜디오와 반복 안내 전광판.

E07_C05:
공영안내 수신권.
공영 스튜디오, 안내 라디오탑, 청취 확인, 대피 문장 승인.

E09_C05:
케이블뉴스 수신권.
구독, 뉴스 스튜디오, 케이블 중계소.

E10_C05:
알고리즘 추천권.
개인화 추천, 모델 학습, 광고홀.
```

BROAD-10은 `방송/추천 일반` 캐릭터가 아니다.
BROAD-10은 `공영 안내 수신` 캐릭터다.

### 4.5 HP 0 차별화

HP 0은 사망/부활이 아니다.

BROAD-10의 HP 0 후보:

```text
청취 확인 완료
대피 문장 자동 승인
공영 안내 문구에 프로필 고정
반복 송출 동기화
안전 문장 수신자 처리
```

보급소 인양 방식:

```text
정정 방송 조각
미승인 청취 로그

```

차별화:

```text
윤서:
현장 회수/잘못 붙은 태그에서 인양.

백여울:
수신 공백/저소음 비콘 지연/인양 지연.

리오:
객실 초기화/체크아웃 처리 직전 객실 카드 비콘.

미카:
개찰 미통과/막차 탑승 실패/플랫폼 잔류 처리 직전 인양.

노라:
청취 확인 완료/대피 문장 승인/공식 안내 반복에 프로필이 고정되기 직전 인양.
```

### 6.1 통과

```text
공영 안내 수신권이 플레이어블 관점으로 분명하다.
QUIET-07과 수신 리스크가 반대로 분리된다.

HP 0이 공식 안내 반복에 프로필 고정되기 직전 긴급 인양으로 차별화된다.
E07_C05 범위를 넘지 않는다.
```

### 6.2 조건부 통과

```text
캐릭터 축은 좋지만 방송인/해커/서포터화 위험이 남아 있다.
수신태그 정산 루트가 약하다.
HP 0 차별화가 아직 백여울과 가까운 편이다.
E07_C05 로컬 범위가 조금 넓다.
이름/언어권 검수가 필요하다.
```

### 6.3 보류

```text
노라가 플레이어블보다 NPC, 안내 방송 기록, 지역 장치로 더 강하다.

기존 캐릭터와 감정 질문이 겹친다.
```

