# Atomic Ad Survivors 용어 최종 확정 v0.1

상태: v0.1 용어 잠금  
목적: 유저용 UI 명칭, NPC 구어, 내부 설정어, 폐기/비노출 용어를 분리한다.  
연결 문서: `docs/world/WORLD_MAP_CAMPAIGN_ECOLOGY_V0_1.md`, `docs/world/EMERGENCY_EXTRACTION_RULES_V0_1.md`, `docs/world/E01_P0_WORLD_MAP_UI_WIREFRAME_V0_1.md`, `docs/world/E02_PLUS_REGION_EXPANSION_PRINCIPLES_V0_1.md`

## 0. 판정 요약

용어는 v0.1 기준으로 아래처럼 잠근다.

- UI 정식 자원명은 `태그` 계열로 확정한다.
- 상위 자원명은 `보급태그`로 둔다.
- 개별 자원명은 `식량태그`, `충전태그`, `통행태그`, `진료태그`, `거주태그`, `수신태그`로 둔다.
- `밥표`, `전원표`, `신호표`는 폐기하지 않는다. 단, UI 정식명으로 쓰지 않고 NPC 구어/지역 속어/오래된 원고 alias로만 쓴다.
- `생존권`, `살림표`, `생활권`, `공급표`는 유저용 핵심 자원명으로 쓰지 않는다.
- `생활접근 최적화 연합 중개층`은 너무 길고 설명 냄새가 강하므로 유저에게 노출하지 않는다.
- 유저용 전 지구 기반 시스템명은 `라이프라인`으로 둔다.
- 내부 설정어는 `생활 접근 중개망`으로 정리한다.
- `A0~A4`는 제작용 내부 등급으로만 쓴다. 월드맵, 노드 패널, 경고 문구에는 `귀환 안정`, `인양 가능`, `인양 불안정`, `고위험 인양`, `신호만 감지`만 노출한다.
- `E01_C03`, `R03-L02` 같은 코드는 제작/도감/디버그용 구조다. 유저 첫 화면에서는 지역명과 노드명으로 먼저 이해시킨다.

가장 큰 문제:

```text
밥표/전원표/신호표는 감정은 좋지만 시스템 UI로 쓰면 범위가 좁고,
식량/충전/수신 외의 통행/진료/거주 확장이 어색해진다.
```

따라서 UI는 `태그`로 통일하고, 속어는 캐릭터 말맛으로 살린다.

## 1. 유저가 처음 이해해야 하는 한 문장

```text
이 세계에서 밥, 전기, 치료, 길, 집, 방송은 돈이 아니라 캠페인이 발급한 태그로 열립니다.
```

## 2. 내부 설정용 핵심 문장

```text
태그는 대사고 전 생활 접근 인증망과 광고 집행망이 결합하며 남긴 접근 승인 단위이고,
보급소는 출격자가 회수한 태그를 정산해 식량, 충전, 통행, 진료, 거주, 수신 권한으로 다시 해석한다.
```

## 3. 자원명 최종표

| 분류 | UI 정식명 | 짧은 UI | NPC 구어/속어 | 내부 설정어 | 판정 |
|---|---|---|---|---|---|
| 상위 자원 | 보급태그 | 보급 | 태그, 표 | 생활 접근권, 접근 승인 토큰 | 확정 |
| 식량/회복 | 식량태그 | 식량 | 밥표, 밥태그 | 영양 접근 승인권 | 확정 |
| 전력/충전 | 충전태그 | 충전 | 전원표, 차지태그 | 전력 사용 승인권 | 확정 |
| 통행/검역 | 통행태그 | 통행 | 문표, 문태그 | 이동 게이트 승인권 | 확정 |
| 치료/약품 | 진료태그 | 진료 | 약표, 약태그 | 의료 서비스 승인권 | 확정 |
| 주거/입주 | 거주태그 | 거주 | 방표, 방태그 | 거주 접근권, 입주 승인권 | 확정 |
| 방송/정보 | 수신태그 | 수신 | 신호표, 신호태그 | 방송/네트워크 접근권 | 확정 |
| 미정산 회수물 | 미정산 태그 | 미정산 | 날태그 | 미분류 접근 토큰 | 확정 |
| 오염된 자원 | 오염 태그 | 오염 | 더러운 표 | 손상 인증 토큰 | 확정 |

주의:

- `보급태그`는 상위 분류명이다.
- 회수 직후 아직 종류가 확정되지 않은 것은 `미정산 태그`라고 부른다.
- `보급태그 +5`처럼 돈처럼 보이는 표기는 피한다.
- 정산 화면에서는 `식량태그 +2`, `충전태그 +1`처럼 개별 자원으로 풀어 보여준다.
- `입주권`, `충전권`, `회원권`처럼 `권`으로 끝나는 말은 정식 캠페인명이나 낡은 약관명에는 쓸 수 있다. 단, 유저가 실제로 얻고 쓰는 자원명은 항상 `태그`다.

## 4. 밥표/전원표/신호표 처리

`밥표`, `전원표`, `신호표`는 완전 폐기하지 않는다.

허용:

- NPC 대사.
- 보급소 구어.
- R01처럼 생활형 공포가 강한 지역의 현장 표지.
- 오래된 정산표, 손글씨 메모, 폐기된 캠페인 발급기.
- 과거 원고/구문을 새 용어로 마이그레이션하기 전의 alias.

금지:

- 상단 HUD 정식 자원명.
- 월드맵 노드 보상 정식 표기.
- 튜토리얼 첫 설명.
- 상품/상점/성장 재화처럼 보이는 반복 UI.

예시:

| 상황 | 권장 |
|---|---|
| UI 보상 | `식량태그 +2` |
| 미나 대사 | `오늘 밥표는 두 장 모자라.` |
| 로봇 대사 | `전원표가 아니라, 내가 나로 남을 시간이에요.` |
| 세븐 분석 | `수신태그 후보를 확인했습니다.` |
| 낡은 발급기 | `신호표 부족. 더 적합한 집을 둘러보시겠어요?` |

판정:

```text
UI는 태그, 사람의 입은 표를 써도 된다.
```

### 5.1 지도 코드와 유저 노출 규칙

지도와 캠페인 ID는 제작 질서를 잡기 위한 코드다. 유저에게는 코드보다 장소 감각을 먼저 보여준다.

| 코드 | 의미 | 사용 위치 | 유저 첫 노출 |
|---|---|---|---|
| E01 | 전 지구 12개 광역권 중 첫 광역권 | 내부 지도, 도감 후반, 제작 문서 | 서부권, 닫힌 항로 같은 지역명 |
| E01_C03 | E01 광역권의 정식 캠페인 | 캠페인 카탈로그, 보스/도감 데이터 | 반품 회수 벨트 같은 캠페인명 |
| R03 | E01 안의 실제 플레이 권역 | 월드맵, 레벨 묶음 | 물류 회수 벨트 |
| R03-L02 | R03 안의 로컬 노드 | 레벨 ID, 데이터, 제작 문서 | 자동 분류장 |
| A0~A4 | 인양 가능성 내부 등급 | 밸런스, 디버그, 제작 문서 | 귀환 안정/인양 가능/인양 불안정/고위험 인양/신호만 감지 |

주의:

- `E`는 지구 광역권, `C`는 정식 캠페인, `R`은 플레이 권역, `L`은 실제 노드다.
- 캠페인 이름에 붙는 `권`은 고유명사나 약관 이름이지, 유저가 소지하는 자원이 아니다.
- 노드 패널에는 `R01-L01`을 작게 병기할 수 있지만, P0 첫 학습에서는 `침묵 가장자리` 같은 노드명이 우선이다.

## 6. 라이프라인 명칭 판정

기존 후보:

```text
생활접근 최적화 연합 중개층
```

판정:

- 너무 길다.
- 설명문처럼 들린다.
- 유저가 기억하기 어렵다.
- 캐릭터 대사에 넣으면 세계가 갑자기 백서처럼 보인다.

최종:

| 용도 | 명칭 |
|---|---|
| 유저용 | 라이프라인 |
| 내부 설정어 | 생활 접근 중개망 |
| 옛 문서 alias | 생활접근 최적화 연합 중개층 |
| UI/대사 노출 | 라이프라인만 사용 |

사용 예:

```text
라이프라인 로그 일부가 복원되었습니다.
```

```text
세븐: 이건 캠페인 기록이 아닙니다. 라이프라인 쪽 원본입니다.
```

내부 설명:

```text
생활 접근 중개망은 대사고 전 식량, 전력, 주거, 통행, 치료, 방송 접근을
광고 집행망과 자동 인증망에 연결하던 기반망이다.
대사고 후 시민권 인증은 무너지고, 라이프라인의 일부 접근 절차와 광고 집행 절차만 살아남아 캠페인 생태의 뼈대가 되었다.
```

## 7. 인양 등급 UI 문구

인양 등급은 `A0~A4`를 내부 등급으로 유지한다.  
유저에게는 아래 짧은 표기와 설명만 보여준다.

| 내부 등급 | UI 표기 | 짧은 설명 | 노출 수준 |
|---|---|---|---|
| A0 | 귀환 안정 | 보급소 장비가 안정적으로 닿는 구역입니다. | P0부터 |
| A1 | 인양 가능 | 긴급 인양이 가능합니다. 회수품 일부를 잃을 수 있습니다. | P0부터 |
| A2 | 인양 불안정 | 신호가 흔들립니다. 인양 시 손실이 커질 수 있습니다. | P1부터 |
| A3 | 고위험 인양 | 회수선 노출 위험이 큽니다. 깊은 진입은 신중해야 합니다. | P1 후반부터 |
| A4 | 신호만 감지 | 아직 인양할 수 없습니다. 중계 또는 침묵권 확장이 필요합니다. | P0 프리뷰부터 |

HP 0 문구:

```text
비콘 신호 수신. 긴급 인양을 시작합니다.
```

인양 완료 문구:

```text
인양 완료. 회수품을 정산합니다.
```

일부 손실 문구:

```text
인양 완료. 일부 회수품이 신호 중에 손실되었습니다.
```

금지:

- 사망.
- 부활.
- 리스폰.
- 게임 오버.
- 무료 귀환.

## 8. UI 위치별 용어 규칙

| UI 위치 | 사용 용어 | 금지 |
|---|---|---|
| 상단 상태줄 | 보급태그, 신호 상태, 인양 상태 | 밥표/전원표/신호표 나열 |
| 노드 패널 | 예상 회수물: 식량태그 조각, 거주태그 흔적 등 | 돈, 골드, 포인트처럼 표기 |
| 정산 화면 | 식량태그/충전태그/수신태그 등 개별 정산 | 보급태그 단일 숫자만 표시 |
| 보급소 대사 | 밥표, 전원표, 신호표 허용 | 모든 NPC가 같은 속어 사용 |
| 지역 경고 | 인양 가능, 인양 불안정, 신호만 감지 | A0~A4 직접 노출 |
| 기록/도감 | 라이프라인, 캠페인, 침묵권 | 생활접근 최적화 연합 중개층 |

## 9. 폐기/제한 용어 목록

| 용어 | 판정 | 이유 |
|---|---|---|
| 생존권 | 제한 | 장르 설명에는 가능하지만 UI 자원명으로 쓰면 너무 추상적이다. |
| 살림표 | 폐기 | 어감은 좋지만 범위가 애매하고 시스템 확장성이 낮다. |
| 생활권 | 폐기 | 일반 한국어/부동산 의미와 충돌한다. |
| 공급표 | 폐기 | 건조하지만 세계 고유성이 약하다. |
| 식권 | 제한 | 현실 급식권 느낌이 강하다. 식량태그의 속어로도 우선순위 낮음. |
| 충전권 | 제한 | 정식 캠페인명/낡은 약관명/내부 설명에는 가능하나 UI 자원명은 충전태그. |
| 접근권 | 내부/설명용 | 유저용 자원명으로는 넓고 추상적이다. |
| 캠페인 인증 토큰 | 내부어 | UI에 나오면 개발 문서처럼 보인다. |
| 생활접근 최적화 연합 중개층 | 비노출 | 너무 길고 설명적이다. |
| 등록 임계 | 내부어 | P0 유저에게는 긴급 인양으로 보여준다. |

## 10. 문서 마이그레이션 규칙

기존 문서에 `밥표/전원표/신호표`가 남아 있어도 즉시 모두 수정하지 않는다.  
다만 새 문서와 UI 구현 문서는 아래 기준을 따른다.

| 기존 용어 | 새 UI 정식명 | 보존 가능 위치 |
|---|---|---|
| 밥표 | 식량태그 | NPC 대사, 손글씨 정산표, R01 발급기 |
| 전원표 | 충전태그 | 로봇 대사, 정비 벤치, 낡은 전원 키오스크 |
| 신호표 | 수신태그 | 세븐 대사, 출격 게시판, 오래된 신호 발급기 |
| 생존권 | 보급태그/생활 접근권 | 장르 설명, 내부 설정 |
| 생활권 | 생활 접근권 | 내부 설정 |
| 접근권 | 접근 승인권 | 내부 설정 |

마이그레이션 원칙:

- UI 문구를 작성할 때는 새 정식명만 사용한다.
- 대사에서는 인물 말투에 따라 속어를 쓸 수 있다.
- 동일 화면 안에서 `식량태그`와 `밥표`를 동시에 설명 없이 섞지 않는다.
- 첫 튜토리얼에서는 `태그`를 먼저 학습시키고, 속어는 보급소 대사로 천천히 노출한다.

## 11. 검증 체크리스트

| 항목 | 판정 |
|---|---|
| 보급태그 계열이 UI 확장에 충분한가? | 통과. 식량/충전/통행/진료/거주/수신을 모두 품는다. |
| 밥표/전원표/신호표의 감정적 장점을 버리지 않았는가? | 통과. NPC 속어와 현장 표지로 보존한다. |
| 생존권이 자원명으로 굳지 않는가? | 통과. 장르 설명에는 가능하지만 UI 자원명은 아니다. |
| 라이프라인이 유저가 기억하기 쉬운가? | 통과. 내부 긴 명칭을 대체한다. |
| 캠페인/라이프라인이 구분되는가? | 통과. 라이프라인은 기반망, 캠페인은 지역 생태다. |
| 인양 문구가 사망처럼 보이지 않는가? | 통과. 긴급 인양/비콘/회수선을 쓴다. |
| 내부어와 유저용어가 분리되는가? | 통과. 등록 임계, 접근 승인권, 인증 토큰은 내부어다. |

## 12. 최종 잠금

```text
1. UI 정식 자원명은 태그 계열로 확정한다.
2. 상위 자원명은 보급태그다.
3. 개별 자원명은 식량태그, 충전태그, 통행태그, 진료태그, 거주태그, 수신태그다.
4. 밥표/전원표/신호표는 UI 정식명이 아니라 NPC 속어와 현장 표현으로만 쓴다.
5. 생존권/살림표/생활권/공급표는 유저용 핵심 자원명으로 쓰지 않는다.
6. 유저용 기반망 명칭은 라이프라인이다.
7. 내부 기반망 명칭은 생활 접근 중개망이다.
8. 캠페인은 지역 등록/보존 광고 생태계, 라이프라인은 그 이전 기반망이다.
9. HP 0은 사망이 아니라 긴급 인양이다.
10. A0~A4는 내부 등급이고 유저에게는 귀환 안정/인양 가능/인양 불안정/고위험 인양/신호만 감지로 보여준다.
11. E/C/R/L 코드는 제작 구조이며 유저 첫 화면에서는 장소명과 상태 문구가 우선이다.
12. 캠페인명에 붙는 `권`과 실제 자원 `태그`를 섞지 않는다.
```
