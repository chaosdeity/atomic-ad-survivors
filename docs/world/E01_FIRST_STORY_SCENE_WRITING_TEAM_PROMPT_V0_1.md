## 1. 작업 목표

너는 E01 1차 장면 본문 작성팀이다.

`story/04_story/e01_first_story_scene_design_review_v0_1.md`의 `통과` 판정을 기준으로, 실제 본문 초안 문서 `story/04_story/e01_first_story_scene_draft_v0_1.md`를 작성한다.

작성 목표:

| 항목 | 지시 |
|---|---|
| 최종 산출물 | `story/04_story/e01_first_story_scene_draft_v0_1.md` |
| 작성 범위 | E01 1차 장면 후보 6개에 대한 짧은 시나리오/서사 초안 |
| 시작 구조 | 윤서 단독 U5 |
| 캠페인 | E01 중심 시작 |
| 전역 연결 | 240개 JSON/핸드오프 항목을 비대면 오브젝트/문구/방송/잔향 후보로만 사용 |
| 본문 규모 | 장면별 짧은 초안. 전체 장편 본문 작성 금지 |

이 프롬프트는 본문 작성팀에게 전달되는 작업 지시서다. 새 플레이어블, 새 NPC 이름, 새 직접 대면 확정, 새 프로필, 엔진/레벨/UI/오디오 구현을 지시하지 않는다.

## 4. 본문 작성 범위

작성 가능:

| 가능 범위 | 작성 방식 |
|---|---|
| E01 1차 장면 후보 6개 | 각 장면을 짧은 시나리오/서사 초안으로 작성 |
| 윤서 단독 시점/체감 | 플레이어블 기능 설명문이 아니라 감각과 상황으로 표현 |
| E01 고정 NPC 구조의 흔적 | 새 이름 없이 비침범적 흔적으로만 사용 |
| 오브젝트 문구 | 표지, 라벨, 서류, 목록, 호출번호, 패널 문구로 제한 |
| 안내 방송 | 자동 안내, 반복 신호, 끊긴 방송처럼 비대면으로 제한 |
| 잔향 | 장소에 남은 소리/기억/절차 흔적으로 제한 |
| 전역 240개 항목 | 비대면 암시로만 사용 |

작성 금지:

| 금지 범위 | 이유 |
|---|---|
| 전체 장편 본문 | 이번 단계는 6개 장면의 짧은 초안만 작성 |
| 대사 과잉 | 직접 대면 NPC화 위험 |
| 새 캐릭터 이름 | NPC/플레이어블 잠금 침범 |
| 새 플레이어블 | 윤서 단독 U5 잠금 |
| 새 NPC 직접 대면 확정 | E01 120 및 L5/L4 192 잠금 침범 |
| 1,856개 전체 문구 작성 | 현재 실제 범위는 240개 |
| 10,240 전체 작성 | 장기 목표이며 현재 착수 금지 |

## 5. 장면별 작성 규칙

아래 6개 장면을 유지해 작성한다.

| scene_id | 작성 목표 | 반드시 살릴 씨앗 | 금지 |
|---|---|---|---|
| `scene_e01_001` | U5 시작 감각 확인 | 귀환 표시와 실제 안전감의 불일치, 출발점의 불안, 윤서 단독 최소 조건 | 새 플레이어블, 새 NPC 이름, 대사 본문 과잉 |
| `scene_e01_002` | E01 현재 상태 확인 | 일상 표식의 절차 흔적화, 정상 안내의 위험 신호화, 우선 불안 상태 축 | E01 추가 생산, 직접 대면 NPC 증가 |
| `scene_e01_003` | 은행형 반응 시험 연결 | 문구가 사람보다 먼저 상황을 설명함, 사물의 잔류 문구, 비대면 반응 표면 | 1,856개 전체 문구 작성, 실제 UI 구현 |
| `scene_e01_004` | 전역 잔향 접속 확인 | E01 바깥의 같은 패턴, 전역 절차의 작은 암시, 세계 설명 없이 의문 남김 | cross_region 진실 공개, 새 이름, 오디오 파일 생성 |
| `scene_e01_005` | E02 연결 준비 표식 | 통행/검역/신원 상태의 외부 연결 가능성, 닫힌 이유의 선노출, 연결 조건 보존 | E02 본문 제작, 미개방 후보 상세 결재 |
| `scene_e01_006` | 본문 착수 전 게이트 확인 | 장면마다 하나의 의문, 설명 과잉 방지, 본문 착수 전 금지선 재확인 | 실제 본문을 장편으로 확장, 새 프로필 작성 |

공통 작성 규칙:

| 규칙 | 지시 |
|---|---|
| 세계 전체 설명 금지 | 한 장면에서 세계 구조를 설명하지 말고 흔적으로만 노출한다. |
| 정보 노출 방식 | 오브젝트, 문구, 방송, 잔향, 행정 절차의 흔적으로 조금씩 노출한다. |
| 전역 연결 | 암시 수준으로 둔다. cross_region은 보류 표식처럼 사용한다. |
| 플레이어블 기능 설명 금지 | 조작법, 해금, 스킬 설명처럼 쓰지 않는다. |
| NPC 설정 문서화 금지 | 인물 프로필이나 관계표처럼 쓰지 않는다. |
| 시스템 설명문 금지 | 데이터 구조, 룰, UI 사양 설명처럼 쓰지 않는다. |
| 대사 제한 | 대사가 필요하면 장면의 분위기와 단서 보조용으로만 짧게 사용한다. |

## 6. 허용 요소

| 허용 요소 | 사용 방식 |
|---|---|
| 윤서 단독 시점/체감 | 첫 감각과 관찰의 기준으로 사용 |
| E01 고정 NPC 구조의 비침범적 흔적 | 직접 대면 확정 없이 흔적, 기록, 분위기로만 사용 |

| 안내 방송 | 자동 안내, 끊긴 방송, 반복 알림 |
| 잔향 | 장소에 남은 소리, 절차, 기억의 흔적 |
| 목록/호출번호/서류/패널 문구 | 비대면 행정 절차의 흔적으로 사용 |
| 전역 240개 JSON/핸드오프 항목의 비대면 암시 | 직접 대면 없이 오브젝트/문구/방송/잔향으로만 사용 |

## 7. 금지 요소

| 금지 요소 | 지시 |
|---|---|
| 새 플레이어블 제작 금지 | 윤서 단독 U5를 유지한다. |
| 새 NPC 이름 추가 금지 | 기존 잠금 밖 이름을 만들지 않는다. |
| 새 NPC 직접 대면 확정 금지 | 비대면 흔적을 대면 캐릭터로 전환하지 않는다. |
| L5/L4 승격 금지 | 192개 상한을 늘리지 않는다. |
| 1,856개 전체 문구 작성 금지 | 240개 실제 범위를 넘어 전체 은행을 쓰지 않는다. |
| 10,240 전체 작성 착수 금지 | 장기 목표를 현재 제작 범위로 쓰지 않는다. |
| 미개방 캠페인 상세 결재 금지 | E02 이후는 연결 준비와 암시까지만 둔다. |
| E02 이후 한국어 이름 기본값 금지 | 외부권 빈자리를 한국어 이름으로 채우지 않는다. |
| 실제 엔진/레벨/UI/오디오 구현 지시 금지 | 좌표, 씬 파일, UI 파일, 오디오 파일, Godot 리소스, GDScript/로더 코드를 지시하지 않는다. |
| 전체 장편 본문 작성 금지 | 6개 장면의 짧은 초안만 작성한다. |
| 대사 과잉 금지 | 직접 대면 장면처럼 길게 쓰지 않는다. |
