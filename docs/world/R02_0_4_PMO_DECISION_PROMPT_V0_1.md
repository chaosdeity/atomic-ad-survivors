# R02 0.4 PMO Decision Prompt v0.1

## 목적

이 문서는 R03 0.3 제작 이관 이후, R02 0.4를 언제 PMO 판정에 올릴지 결정받기 위한 복붙용 프롬프트다.

---

## 0. 현재 상태

```text
R03 0.3은 PMO 최종 승인 완료.
R03는 구현/레벨/QA 팀에 제작 이관 가능.
R02 0.4 초안은 작성되어 있으나 제작 이관 단계는 아니다.
```

현재 추천:

```text
R03 구현 피드백 1회 후 R02 0.4 PMO 판정.
```

이유:

```text
R03는 목적지 판정이 사람에게 붙는 지역.
R02는 환자 등록이 사람을 계속 붙드는 지역.
둘 다 판정형 경험이므로, R03 손맛 통과 전 R02를 바로 열면 체감 피로가 생길 수 있다.
```

## 2. 복붙용 마스터 프롬프트

```text
당신은 Atomic Ad Survivors의 스토리 PMO다.

목표:
R03 0.3 제작 이관 이후 R02 0.4를 바로 PMO 판정에 올릴지,
R03 구현/플레이테스트 피드백 1회를 받은 뒤 올릴지 결정하라.

현재 상태:
- R03 0.3은 제작 이관 가능 판정이 났다.
- R03의 남은 구현 리스크는 라벨/레일/드론이 방해물이 아니라 시스템을 속이는 재미로 작동하는지다.
- R03 구현 플레이테스트 수락 기준은 R03-PT-01~08로 정리되어 있다.
- R02 0.4 초안은 작성되어 있다.
- R02 0.4 후보는 R02-L01 응급 접수 홀, R02-L02 장기 입원동, R02-L03 보험 심사 약국 조건부 확장이다.
- R02-L04/R02-L05와 HOLD-04 정식 해금은 보류다.

핵심 판단:
R02는 R03 다음 후보로 강하다.
하지만 R03의 목적지 판정 직후 R02의 환자 등록 판정을 바로 열면 판정형 지역 피로가 생길 수 있다.

판정 선택지:
1. R03 구현 피드백 1회 후 R02 0.4 PMO 판정
2. R02 0.4 PMO 사전 판정만 먼저 진행하고 제작 착수는 보류
3. R02 0.4 상세화 즉시 착수
4. R02 보류, R03 구현 수정 집중

검토할 기준:
- R03-PT-01~03 중 2개 이상이 통과해야 R02를 열어도 되는가?
- R03-PT-07에서 라벨/태그 혼동이 낮아야 하는가?
- R03 안의 처방/팔찌/진료태그 기록이 R02 궁금증을 충분히 만드는가?
- HOLD-04가 정식 해금처럼 보이지 않는가?
- R02-L01/L02가 0.4 주 슬라이스로 적절한가?
- R02-L03은 조건부 확장인가, 정산 씨앗인가?

출력 형식:

## 1. 판정
선택지 1~4 중 하나.

## 2. 이유
5개 이하 bullet.

## 3. R03 피드백 요구치
R03-PT 중 어떤 항목을 보고 R02 0.4를 열지 적어라.

## 4. R02 0.4 범위
R02-L01/L02/L03 각각 열기/조건부/보류 판정.

## 5. HOLD-04 노출 한계
U0~U2 허용 범위와 정식 U5 보류선을 적어라.

## 6. 다음 산출물
R02-L01 경험 상세, R02-L02 반복 반응, R02 오브젝트 후보, R02 UI 문구 중 우선순위를 정하라.
```
