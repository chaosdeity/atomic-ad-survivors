## 0. 판정 요약

- 이 문서의 좌표는 실제 GPS가 아니다.
- 좌표는 E01 광역권을 `0~100`으로 정규화한 내부 블록아웃 좌표다.
- 유저용 월드맵은 실명 지명을 숨기고, 해안선/도로축/도시 밀도/산지/항만/방송탑 실루엣만 남긴다.
- 목표 인식 밀도는 기존 기준대로 100명 중 10명 정도가 실제 장소를 알아채는 수준이다.
- E01 첫 시즌 노드는 총 40개이며, P0 5개, P1 16개, P2 19개다.
- P0는 화면 중앙 외곽의 침묵권에서 동/남동 주택가로 이동하는 짧은 동선으로 둔다.
- R03/R02/R04는 P1에서 약한 신호로 열고, R05/R07 깊은 구역은 P2 원격 감지권으로 둔다.

## 1. 좌표계

```text
좌표 범위: X 0~100, Y 0~100
X 0 = 서쪽 해안/항만 방향
X 100 = 동쪽 내륙/외곽 방향
Y 0 = 북쪽/고지대/방송 축
Y 100 = 남쪽/산업/항만 하부 축
```

주의:

- 이 좌표는 지도 아트, UI, 레벨 선택, 인양 반경 계산의 기준점이다.
- 실제 장소명을 유저에게 직접 노출하기 위한 좌표가 아니다.
- 추후 지도 이미지가 만들어지면, 이 좌표를 기준으로 노드 위치를 미세 조정한다.

## 4. 인양 반경 블록아웃

기준점은 `R08-L02 역송출 주유소`다. 실제 인양은 순간이동이 아니라 회수선, 비콘, 역송출 장비, 침묵권 노이즈 차폐가 맞물린 제한 구조다.

| 반경 등급 | 기준 거리 | 인양 상태 | 대표 노드 | 설계 의미 |
|---|---:|---|---|---|
| A0 내부 침묵권 | 0~6 | 안정 | R08-L01, R08-L02 | 정산, 수리, 귀환 가능 |
| A1 근거리 출격권 | 7~20 | 안정~불안정 | R01-L01~R01-L04, R08-L03, R08-L04 | 초반 반복 출격권 |
| A2 중거리 회수권 | 21~34 | 불안정 | R03-L01~R03-L03, R02-L01~R02-L03, R06-L01~R06-L03 | 비콘 소모, 회수선 노출 증가 |
| A3 고위험 외곽권 | 35~48 | 매우 불안정 | R04 대부분, R07 일부, R03/R02/R06 보스권 | 실패 시 태그/전리품 손실 큼 |
| A4 원격 감지권 | 49 이상 또는 강신호권 | 인양 불가/조건부 | R05 깊은 구역, R07-L05 | 중계기/침묵권 확장 전에는 진입 제한 |

주의:

- 실제 거리는 좌표상의 직선거리만으로 결정하지 않는다.
- 방송 신호, 검역 봉쇄, 회색 오염, 지하 구조, 브로커 방해에 따라 인양 상태가 한 단계 악화될 수 있다.
- R05는 좌표상 멀지 않아도 강한 송출 신호 때문에 원격 감지권으로 취급한다.

## 6. 월드맵 노출 순서

### 6.1 초기 화면

초기 화면에서 선명하게 보이는 노드는 5개뿐이다.

```text
R08-L01 침묵 보급소
R08-L02 역송출 주유소
R01-L01 침묵 가장자리
R01-L02 분양 주택 루프
R01-L03 모델하우스 결절
```

초기 화면의 목적:

- 유저가 첫 지역을 주택가로 즉시 이해한다.
- 보급소가 왕국이 아니라 작은 공백임을 이해한다.
- 인양이 보급소 장비와 거리 제한에 묶인다는 감각을 만든다.
- R03/R02/R04는 흐린 신호로만 보여서 세계가 더 크다는 느낌을 남긴다.

### 6.2 P1 확장

P1은 첫 보스 후 열리는 가지다.

P1의 실제 해금은 모든 노드 즉시 개방이 아니라 `신호 감지 -> 해석 가능 -> 출격 가능` 순서로 처리한다. 세부 기준은 `docs/world/E01_P1_EXPANSION_DESIGN_V0_1.md`를 따른다.

| 해금 축 | 열리는 노드 | 지도 효과 |
|---|---|---|
| 침묵권 확장 | R08-L03, R08-L04 | 보급소 주변 공백이 조금 넓어진다. |
| R01 재방문 | R01-L04, R01-L05 | 주택가가 플레이어를 학습했다는 변형을 보인다. |
| 윤서 회수축 | R03-L01, R03-L02, R03-L03 | 남서쪽 물류 신호가 선명해진다. |
| 의료 구조축 | R02-L01, R02-L02, R02-L03 | 도심 병원권의 백색 신호가 열린다. |
| 로봇 충전축 | R04-L01, R04-L02, R04-L03 | 남부 산업 오염과 충전태그 문제가 열린다. |
| 인간 브로커축 | R06-L01, R06-L02, R06-L03 | 도심 상업권의 인간 거래가 보인다. |

### 6.3 P2 확장

P2는 후반 또는 다음 시즌 연결용이다.

| 해금 축 | 열리는 노드 | 지도 효과 |
|---|---|---|
| 윤서 보스 | R03-L04, R03-L05 | 반품 회수 벨트가 적대적으로 닫힌다. |
| 의료 보스 | R02-L04, R02-L05 | 치료와 감금의 경계가 드러난다. |
| 로봇 보스 | R04-L04, R04-L05 | 해방 로봇의 재동기화 분기가 열린다. |
| 인간 악역 보스 | R06-L04, R06-L05 | 브로커와 캠페인 약관의 결탁이 보인다. |
| 방송 미스터리 | R05-L01~R05-L05 | 대사고 기록과 라이프라인의 의도가 복원된다. |
| 항만 탈출 | R07-L01~R07-L05 | 바깥 세계와 봉쇄의 진위가 열린다. |
| 보급소 내부 위기 | R08-L05 | 침묵권이 추적될 수 있음을 보여준다. |

## 7. 연결선 규칙

월드맵에서 모든 노드를 선으로 촘촘히 연결하지 않는다. 연결선은 캠페인의 이동 절차와 인양 가능성을 보여주는 정보다.

### 7.1 P0 필수 연결

```text
R08-L01 -> R08-L02 -> R01-L01 -> R01-L02 -> R01-L03
```

의미:

- R08-L01은 정산/휴식이다.
- R08-L02는 출격과 긴급 인양의 장비실이다.
- R01-L01은 침묵권 바깥의 첫 위험이다.
- R01-L02는 기본 반복 런이다.
- R01-L03은 첫 보스 결절이다.

### 7.2 P1 가지 연결

```text
R01-L02 -> R01-L04
R01-L03 -> R01-L05
R08-L02 -> R08-L03 -> R08-L04
R01-L05 -> R03-L01 -> R03-L02 -> R03-L03
R01-L03 -> R02-L01 -> R02-L02 -> R02-L03
R03-L03 -> R04-L01 -> R04-L02 -> R04-L03
R06-L01 -> R06-L02 -> R06-L03
```

주의:

- R06은 처음부터 지도 위에 선명한 상업 지구로 보이지 않는다.
- R06-L01은 R02 또는 R03에서 브로커 단서를 얻은 뒤 열리는 편이 자연스럽다.

### 7.3 P2 가지 연결

```text
R03-L03 -> R03-L04 -> R03-L05
R02-L03 -> R02-L04 -> R02-L05
R04-L03 -> R04-L04 -> R04-L05
R06-L03 -> R06-L04 -> R06-L05
R08-L03 -> R05-L01 -> R05-L02 -> R05-L03 -> R05-L04 -> R05-L05
R07-L01 -> R07-L02 -> R07-L03 -> R07-L04 -> R07-L05
R08-L04 -> R08-L05
```

주의:

- R05는 좌표상 북쪽 도심과 가깝지만, 신호가 너무 강해서 초반 인양이 불가능하다.
- R07은 항만/공항/검역이라는 물리적 거리와 봉쇄 절차 때문에 후반으로 둔다.

## 8. 월드맵 화면 구성 기준

### 8.1 첫 화면 구도

첫 화면은 `R08+R01`만 화면 중앙에 읽히게 한다.

| 화면 영역 | 보이는 것 | 숨기는 것 |
|---|---|---|
| 중앙 | 침묵 보급소, 역송출 주유소, 주택가 첫 출격권 | 정확한 실제 지명 |
| 동/남동 | 분양 주택 루프, 모델하우스 약한 결절 | 도시 전체 규모 |
| 남/남서 | 물류 벨트의 흐린 라벨 신호 | R03 보스와 윤서 과거 |
| 동/도심 | 병원권의 백색 노이즈 | 등록 인간의 상태 |
| 남부 | 산업 오염의 회색 얼룩 | 해방 로봇 핵심 정보 |
| 서부 | 닫힌 항로의 흐릿한 해안선 | 실제 탈출 가능성 |
| 북부 | 수신탑 신호 잡음 | 대사고 진실 |

### 8.2 UI 표기 기준

| 내부 상태 | 유저용 표기 |
|---|---|
| A0 안정 | 귀환 안정 |
| A1 안정~불안정 | 인양 가능 |
| A2 불안정 | 인양 불안정 |
| A3 매우 불안정 | 고위험 인양 |
| A4 원격 감지 | 신호만 감지 |

주의: 인양 등급 표기는 `docs/world/WORLD_TERMINOLOGY_LOCK_V0_1.md` 기준으로 v0.1 잠금한다.

## 9. 논리 검증

| 검증 항목 | 판정 |
|---|---|
| 실제 지구 기반인가? | 통과. 서쪽 해안/항만, 도심 고밀도, 외곽 주거, 남부 산업, 북부 방송축의 관계를 유지했다. |
| 실명 노출이 과한가? | 통과. 문서 내부에는 방향과 기능만 있고 유저용 실명은 없다. |
| P0가 너무 넓은가? | 통과. P0는 R08과 R01만 사용해 첫 플레이를 좁게 유지한다. |
| R05가 좌표상 가까운데 왜 후반인가? | 설명 필요. 강한 송출 신호와 캠페인 간섭 때문에 A4 원격 감지권으로 둔다. |
| R08이 캠페인이 되어버렸는가? | 통과. R08은 침묵권 예외 공간이며 E01_C08 비상보급/대피 잔여 절차에 연결된다. |
| 인양이 무료 순간이동처럼 보이는가? | 통과. R08-L02 기준 반경, 비콘, 회수선, 신호 간섭으로 제한한다. |
| 40개 노드가 모두 첫 시즌 구현 대상으로 오해되는가? | 방지 필요. P0만 첫 빌드, P1/P2는 확장/후반 후보로 명시했다. |

## 11. 데이터 파일 스키마

`data/world/e01_world_map_nodes_v0_1.json`은 Godot 또는 별도 월드맵 툴에서 직접 읽을 수 있는 기준 데이터다.

| 필드 | 의미 |
|---|---|
| `schemaVersion` | 데이터 구조 버전 |
| `region` | E01 광역권 정보와 좌표계가 실제 GPS가 아님을 표시 |
| `coordinateSystem` | 0~100 정규화 좌표의 방향 의미 |
| `visibilityPriorities` | P0/P1/P2 노출 단계 설명 |
| `extractionTiers` | A0~A4 인양 등급과 UI 표기 |
| `extractionOriginNodeId` | 인양 기준 노드. 현재는 R08-L02 |
| `nodes` | 40개 로컬 노드의 좌표, 캠페인, 우선순위, 인양 등급 |
| `links` | 월드맵 노드 간 연결선 37개 |

제작자가 각 노드의 위치감, 시각 특징, 플레이/서사 기능을 확인할 때는 `docs/world/E01_WORLD_MAP_PRODUCTION_TABLE_V0_1.md`를 기준으로 삼는다.

## 12. 러프 이미지 사용 원칙

`assets/world/e01_world_map_blockout_v0_1.svg`는 최종 아트가 아니라 제작용 시각 자료다.

| 항목 | 원칙 |
|---|---|
| 목적 | 좌표, 연결선, 인양 반경, P0/P1/P2 노출감을 한눈에 확인한다. |
| 색상 | 최종 아트 팔레트가 아니라 제작용 구분색이다. |
| 지리 | 실제 GPS가 아니라 0~100 정규화 좌표다. |
| 실명 노출 | 유저용 지명을 직접 드러내지 않는다. |
| 재생성 | JSON 좌표가 바뀌면 `tools/generate_e01_world_map_blockout.ps1`을 다시 실행한다. |
