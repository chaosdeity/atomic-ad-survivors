## 3. 반드시 유지할 핵심

핵심 질문:

```text
공식 안내가 계속 반복되면,
그것은 도움인가 명령인가?
```

핵심 문장:

```text
분명하게 들린다고 해서,
동의한 것은 아니다.
```

최종 변화 문장:

```text
정확한 문장도 사람을 대신 살아주지는 못한다.
```

한 줄 정의:

```text
노라는 방송인이 아니라,
공식 안내가 사람을 안전한 수신자로 확정하기 전
청취 확인과 대피 문장 승인을 늦추는 생존자다.
```

잘못 믿는 것:

```text
노라는 공식 안내가 선명할수록 더 안전하다고 믿는다.
```

이 믿음은 일부 맞다.
공영 안내, 재난 문구, 정정 방송, 다국어 안내는 사람을 살리기 위해 만들어진 절차다.

하지만 위험하다.
캠페인은 노라를 죽이려는 것이 아니라, 노라를 이미 안내를 들은 사람, 이미 확인한 사람, 이미 대피 문장에 동의한 사람으로 보존하려 한다.

## 9. 태그 / 정산 / 자원 잠금

정산 루트:

| 정산 루트 | 고정 의미 | 위험 |
|---|---|---|
| 수신태그 | 방송/안내/기록 접근권 | 방송 공격력이나 정보 마나가 되면 실패 |
| 통행태그 | 대피 문장과 이동 안내의 보조 접근권 | 무료 탈출권이나 빠른 이동권이 되면 실패 |
| 안내 수신 로그 | BROAD-10의 핵심 기록 | 단순 퀘스트 로그가 되면 실패 |
| 정정 방송 조각 | 공식 문장의 불일치 | 해킹 키가 되면 실패 |
| 청취 확인표 | 들었다가 동의했다로 닫히는 표 | 파티 버프 승인서가 되면 실패 |
| 대피 문장 승인표 | 안전 문장 자동 승인 위험 | 무료 대피권이 되면 실패 |
| 반복 송출 잔향 | 공식 안내 과잉의 후유증 | 광역 음파 공격 자원이 되면 실패 |
| 미승인 청취 로그 | 아직 판단 중인 사람의 단서 | 범용 분석 포인트가 되면 실패 |
