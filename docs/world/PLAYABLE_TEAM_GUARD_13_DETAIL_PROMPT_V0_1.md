# GUARD-13 상세 결재 플레이어블팀 프롬프트 v0.1

이 문서는 플레이어블 제작 AI팀에게 `13번 전태주 / GUARD-13 / 보호자 등록 거부자 상세 결재`를 맡기기 위한 복붙용 프롬프트다.

목표는 20인 로스터를 다시 만드는 것이 아니다.
목표는 상세 결재 전 재검토에서 `조건부 유지` 판정을 받은 13번 후보가 플레이어블로 성립하는지 판정하고, 조건부 통과 또는 보류 사유를 결재 가능한 문서로 고정하는 것이다.

핵심:

```text

GUARD-13은 타인의 치료 접근권이
보호자/보증인 서명으로 담보 잡히는 절차를
지연시키는 13번 플레이어블 후보이다.
```

상세 결재는 아래를 판단해야 한다.

1. GUARD-13이 플레이어블이어야 하는가, NPC/사건 기록/병원 오브젝트로 충분한가.

3. E12_C02 이동진료 승인권을 우선 원점으로 삼아도 되는가.
4. E04_C02 열대격리 진료권은 보조 후보로 남기는 것이 맞는가.
5. 전태주 이름을 임시 유지할지, 이름/국적/현실권 재검토를 조건부로 걸지 판단한다.
6. 전투 리듬이 힐러/탱커/보디가드가 아니라 보호자 서명, 보증인 칸, 호출벨, 임시 서약서의 판정 지연에서 나오는가.
7. HP 0과 긴급 인양이 보호자 서명 확정 / 치료 접근권 담보 고정으로 차별화되는가.
8. 상세 결재 이후 조건부 해소 문서가 필요한지, 아니면 프로필 v1.0으로 갈 수 있는지 판정한다.

반드시 이어받을 조건:

```text
전태주 이름은 조건부 유지한다.
상세 결재에서 전태주 이름을 최종 확정하지 않는다.
E12_C02 이동진료 승인권을 우선 원점으로 검토한다.
E04_C02 열대격리 진료권은 보조 후보로 남긴다.
E01/R02 의료 후속 직접 원점은 보류한다.
HOLD-04는 환자/퇴원 보류, GUARD-13은 보호자/보증인/치료 접근권 담보 절차로 분리한다.
R02 의료 NPC의 병원 행정 얼굴을 빼앗지 않는다.
```

상세 결재 첫 잠금 문장:

```text
하람은 자기 몸이 환자로 고정되는 절차를 늦춘다.

```

이 문장이 흔들리면 GUARD-13은 상세 결재에서 보류한다.

## 4. 핵심 잠금

핵심 질문:

```text
내가 서명하면,
누구의 치료권이 담보가 되는가?
```

핵심 문장 후보:

```text
보호자란 이름이 붙어도,
사람을 맡길 수 있다는 뜻은 아니다.
```

최종 변화 문장 후보:

```text
보호는 대신 서명하는 일이 아니라,
닫히는 접근권을 다시 사람 쪽에 남기는 일이다.
```

한 줄 정의 후보:

```text
GUARD-13은 의사나 보호자 영웅이 아니라,
타인의 치료 접근권이 보호자/보증인 서명으로 담보 잡히기 전
그 책임 칸을 비워 두는 플레이어블 후보이다.
```

잘못 믿는 것 후보:

```text
태주는 보호자 등록을 거부하면
책임에서 벗어날 수 있다고 믿는다.
```

이 믿음은 일부 맞다.
보호자/보증인 등록은 치료 접근권을 사람 사이의 담보와 책임 전가로 바꿀 수 있다. 아무 서명이나 하지 않는 것은 사람을 빚이나 대리 책임으로 묶지 않기 위한 방어일 수 있다.

하지만 위험하다.
등록을 거부하는 것만으로는 누군가의 치료 접근권이 열리지 않는다. 캠페인은 GUARD-13을 죽이려는 것이 아니라, 보호자 등록 거부자, 보증인 미제출자, 호출벨 미응답자, 임시 서약 미완료자로 보존하려 한다. 거부가 닫힘으로 굳어지면, 보호받아야 할 사람의 접근권도 같이 닫힌다.

## 5. 원점 캠페인 잠금

```text
E12_C02 이동진료 승인권
```

E12_C02 기준:

```text
관리자용 현실 권역:
E12 아프리카 해안-자원 도시 벨트

관리자용 현실 위치:
이동 진료소, 도시 병원 연결망, 약품 천막

인프라 앵커:
이동진료 트럭, 승인 환자, 임시 약품, 승인 데스크

유저 노출 지역명:
E12_C02 이동진료 승인권

등록하려는 역할:
진료 대상, 이동 환자, 승인 환자, 보호자/보증인 대기자
```

보조 후보:


E04_C02는 격리 동의/보호자 확인을 만들 수 있지만, 격리 환자와 진료 대상이 중심이라 HOLD-04와 가까워질 위험이 있다. 따라서 상세 결재에서는 E04_C02를 보조 후보로만 두고, 원점은 E12_C02 우선을 검토한다.


## 6. 이름/표기 검수

현재 임시 유지:

```text
임시 본명:
전태주

GUARD-13

유저 첫 표기 임시안:
태주, 보호자 등록 거부자
```

상세 결재에서의 원칙:

```text
전태주는 임시명으로 유지할 수 있다.
그러나 E12_C02를 원점으로 잡는다면 이름/국적/언어권 재검토를 조건부 해소 항목으로 남긴다.
상세 결재 문서에서 전태주를 최종 확정하지 않는다.
상세 결재 문서에서 전태주를 임의 폐기하지 않는다.
GUARD-13 코드명은 유지 가능하다.
```

대체 유저 첫 표기 후보:

| 후보 | 용도 | 주의 |
|---|---|---|
| 태주, 보호자 등록 거부자 | 레지스트리 연속성 | 한국어권 이름 확정처럼 보이지 않게 임시 표기라고 명시 |
| GUARD-13, 보증인 칸을 비운 사람 | 이름 보류 시 안전 | 유저 첫 표기로는 차가우므로 상세 결재에서 보조 후보 |
| 보호자 등록 거부자 | 기능 선명 | 캐릭터성이 약함 |
| 호출벨을 끊지 않은 사람 | HP 0 이미지 강함 | 역할명으로는 문학적임 |

상세 결재 판정 조건:

```text
이름/국적/언어권이 해결되지 않으면 통과가 아니라 조건부 통과로 둔다.
프로필 v1.0 전에는 이름/표기 검수를 별도 해소해야 한다.
```

## 7. HOLD-04와 반드시 분리

반드시 비교한다.

| 항목 | HOLD-04 강하람 | GUARD-13 |
|---|---|---|
| 슬롯 | 4번 프로필 v1.0 고정 | 13번 상세 결재 대상 |
| 중심 역할 | 퇴원 보류자 / 환자 | 보호자 등록 거부자 / 보증인 절차 거부자 |
| 중심 절차 | 퇴원 보류, 보험 심사, 병상 재배정, 환자 팔찌 | 보호자 등록, 보증인 서명, 호출벨 응답, 임시 서약 |
| 캠페인이 붙이는 이름 | 환자, 보험 대상자, 퇴원 보류자 | 보호자, 보증인, 대리 서명자, 치료 접근권 책임자 |
| 전투 감각 | 퇴원 취소와 병상 재배정 완료를 늦춘다 | 보호자 서명과 치료 접근권 담보 고정을 늦춘다 |
| HP 0 감각 | 환자 팔찌 잠김, 퇴원 보류 확정, 보험 심사 고정 | 보호자 서명 확정, 보증인 등록 완료, 호출벨 신호 단절 |
| 금지 중복 | 보호자/보증인 축 선점 금지 | 퇴원 보류 환자/힐러/보험 심사 축 반복 금지 |

금지:
