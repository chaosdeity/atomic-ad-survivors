# Atomic Ad Survivors 구용어 마이그레이션 v0.1

상태: 1차 적용 완료 / 잔여 원고 검수 전  
목적: 세계 기반 v0.3에서 잠근 태그/인양 용어가 실제 제작 문서와 스크립트에 다시 섞이지 않도록 교체 원칙과 적용 범위를 기록한다.  
기준 문서: `WORLD_FOUNDATION_LOCK_V0_3.md`, `WORLD_TERMINOLOGY_LOCK_V0_1.md`, `EMERGENCY_EXTRACTION_RULES_V0_1.md`

## 0. 판정 요약

1차 마이그레이션은 완료로 본다.

적용 완료 범위:

- `scripts/`의 유저 노출 문자열.
- `story/90_implementation/`의 구현 지시 문서.
- HP 0, 보상, 정산, 태그 후보, 런 결과 문구의 핵심 표현.

아직 남은 범위:

- `story/01_bible/`, `story/03_character/` 등 서사 원고 전반의 대사/문체 검수.
- NPC 속어로 남길 `밥표/전원표/신호표`의 화자별 사용 규칙.
- 내부 변수명 `ticket`, `ration` 계열의 실제 코드명 교체 여부.

## 1. 교체 기준

| 구용어 | 기준 용어 | 허용 예외 |
|---|---|---|
| 사망 보상 | 인양 정산 | 금지 예시를 설명할 때만 제한적으로 허용 |
| HP 0 사망 | HP 0 긴급 인양 | 금지 예시를 설명할 때만 제한적으로 허용 |
| 부활/리스폰 | 귀환/인양/재출격 | 장르 비교 설명에서만 허용 |
| 배급권 | 보급태그 또는 개별 태그 | 오래된 내부 문서의 설명 문맥 |
| 밥표 | 식량태그 | NPC 속어, 손글씨 정산표, 낡은 표지 |
| 전원표 | 충전태그 | 로봇/NPC 속어, 낡은 충전 표지 |
| 신호표 | 수신태그 | 세븐 속어, 낡은 수신 표지 |
| 생존권 | 보급태그, 접근 태그, 생활 접근 | 장르 한 줄 설명에서만 조심스럽게 사용 |

## 2. 스크립트 적용 결과

유저에게 직접 보일 수 있는 문자열은 태그 계열로 1차 교체했다.

| 파일 | 적용 내용 |
|---|---|
| `scripts/hud_controller.gd` | 배급권 표시를 보급태그/태그 표시로 변경 |
| `scripts/level_up_cards.gd` | 밥표/전원표/신호표 후보를 식량태그/충전태그/수신태그 후보로 변경 |
| `scripts/local_response_state.gd` | 지역 응답 부품명을 식량태그/충전태그/수신태그로 변경 |
| `scripts/meta_progression.gd` | 정산 라벨과 메타 진행 문구를 태그 계열로 변경 |
| `scripts/run_result_evaluator.gd` | 결과 사유와 확정/후보 문구를 태그 계열로 변경 |
| `scripts/main.gd` | HUD, 튜토리얼, 패널 문구의 배급권/밥표/전원표/신호표 표현을 태그 계열로 변경 |
| `scripts/route_phrase_resolver.gd` | 경로 설명의 구용어를 기준 용어로 변경 |

검증:

```text
rg "배급권|밥표|전원표|신호표" scripts
=> 결과 없음
```

## 3. 구현 문서 적용 결과

`story/90_implementation/`은 실제 제작자가 보는 지시 문서이므로 UI/시스템 기준 용어를 우선 적용했다.

적용 기준:

- 플레이어 HP 0은 `사망`이 아니라 `긴급 인양`으로 쓴다.
- 보상은 `사망 보상`이 아니라 `성과 기반 정산` 또는 `인양 정산`으로 쓴다.
- 정식 자원명은 `식량태그`, `충전태그`, `수신태그`, `보급태그`로 쓴다.
- `밥표/전원표/신호표`는 NPC 속어 또는 낡은 표지라는 설명에서만 남긴다.

검증:

```text
rg "밥표|전원표|신호표|배급권|사망 보상|HP 0 사망|생존권|회수/사망" story/90_implementation scripts
=> story_deepening_0_2_asset_brief.md의 NPC 속어 설명 1건만 남음
```

## 4. 남겨도 되는 표현

아래 표현은 세계관 기준과 충돌하지 않는다.

- `HP 0은 사망이 아니다.`
- `첫 회수가 죽음처럼 보이는가?`
- `사망/부활/리스폰 같은 단어를 쓰지 않는다.`
- `밥표/전원표/신호표는 NPC 속어로만 쓴다.`

이 표현들은 구용어 사용이 아니라 금지 기준 또는 플레이테스트 확인 문구다.
