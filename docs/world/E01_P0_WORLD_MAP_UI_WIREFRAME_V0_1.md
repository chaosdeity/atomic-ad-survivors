# Atomic Ad Survivors E01 P0 월드맵 UI 와이어프레임 v0.1

상태: v0.1 제작용 와이어프레임 기준  
목적: E01 첫 화면에서 `R08 보급소 -> R01 첫 출격 -> R01 기본 런 -> R01 첫 보스 -> P1 약한 신호` 흐름을 UI 단위로 고정한다.  
연결 문서: `docs/world/E01_P0_WORLD_MAP_FIRST_SCREEN_FLOW_V0_1.md`, `docs/world/E01_WORLD_MAP_BLOCKOUT_V0_1.md`, `docs/world/R01_FIRST_SORTIE_LEVEL_CONNECTION_V0_1.md`, `docs/world/R01_L03_FIRST_BOSS_DETAIL_V0_1.md`, `docs/world/EMERGENCY_EXTRACTION_RULES_V0_1.md`, `docs/world/WORLD_TERMINOLOGY_LOCK_V0_1.md`  
색채 기준: `docs/world/E01_WORLD_MAP_ART_COLOR_DIRECTION_V0_1.md`  
시각화 파일: `assets/world/e01_p0_world_map_ui_wireframe_v0_1.svg`

## 0. 판정 요약

- P0 월드맵은 전 지구 설명 화면이 아니라 첫 출격 조작 화면이다.
- 첫 화면에서 선명하게 보이는 노드는 `R08-L01`, `R08-L02`, `R01-L01`, `R01-L02`, `R01-L03` 다섯 개뿐이다.
- 처음부터 출격 가능한 노드는 `R01-L01 침묵 가장자리` 하나다.
- `R01-L02`는 경로가 보이지만 잠겨 있고, `R01-L03`은 약한 보스 신호로 보이지만 바로 출격할 수 없다.
- 모바일 세로 화면을 1차 기준으로 한다. 데스크톱은 같은 정보 구조를 넓게 배치한다.
- UI는 대시보드가 아니라 `침묵 보급소의 출격 게시판/수신 지도`처럼 보여야 한다.
- 최종 색채가 아니다. 와이어프레임 색은 기능 구분용이며, 노란색/황혼 톤을 기본 팔레트로 삼지 않는다.

가장 중요한 잠금:

```text
첫 월드맵 화면은 "어디를 갈 수 있는지"보다
"왜 지금 이 한 곳만 출격 가능한지"를 직관적으로 보여준다.
```

## 1. 화면 판타지

유저용 느낌:

```text
보급소 벽에 붙은 출격 지도.
지도는 살아 있는 GPS가 아니라, 끊긴 회수선과 캠페인 신호를 수동으로 맞춘 수신판이다.
```

제작 기준:

| 항목 | 기준 |
|---|---|
| 재료감 | 낡은 주유소 정비판, e-ink 지도, 종이 태그, 신호 테이프, 수동 핀 |
| 정보 톤 | 짧은 라벨, 신호 상태, 인양 가능 여부, 행동 버튼 |
| 색 방향 | 침묵권은 청록/민트, 캠페인 위험은 코랄/적색, 의료는 백색 잡음, 산업은 회색/녹색 |
| 노란색 사용 | 보스 신호/주의 신호의 작은 강조에만 사용 |
| 금지 톤 | 황혼 포스트아포칼립스, 화려한 위성 지도, SaaS 관리자 화면, 장문 튜토리얼 카드 |

## 2. 화면 크기 기준

| 플랫폼 | 기준 해상도 | 1차 배치 | 이유 |
|---|---:|---|---|
| 모바일 | 360x640 | 상단 상태줄, 중앙 지도, 하단 노드 패널 | 모바일 액션 RPG의 실제 첫 사용 환경 |
| 데스크톱 | 1280x720 | 상단 상태줄, 좌측 지도, 우측 노드 패널 | 테스트/개발/PC 빌드용 |

공통 원칙:

- 지도 조작 영역이 화면의 중심이어야 한다.
- 노드 패널은 선택한 노드의 판단 정보만 보여준다.
- 긴 설명, 세계관 기록, 96개 캠페인 정보는 이 화면에 올리지 않는다.
- 잠긴 노드는 클릭 가능하되 출격 버튼은 비활성이다.

## 3. 제작용 와이어프레임

파일:

```text
assets/world/e01_p0_world_map_ui_wireframe_v0_1.svg
```

구성:

- 우측: 데스크톱 16:9 기준.

## 4. 모바일 첫 화면 배치

```text
0~48     상단 상태줄
48~448   월드맵/노드 영역
448~640  선택 노드 패널 + 주요 버튼
```

| 영역 | 위치 | 내용 | 규칙 |
|---|---|---|---|
| 상단 상태줄 | 최상단 48px | 보급소명, 보급태그 요약, 신호 상태 | 한 줄만 사용 |
| 월드맵 | 중앙 약 400px | R08/R01 노드 5개, 흐린 P1 신호 | 지도가 화면의 주인공 |
| 선택 노드 패널 | 하단 약 132px | 노드명, 위험/인양, 목표, 회수물 | 4줄 이하 |
| 주요 버튼 | 하단 48~56px | 출격/정비/장비 확인/잠김 | 버튼 하나를 가장 크게 |

모바일에서 피해야 할 것:

- 하단 패널을 화면 절반 이상으로 키우지 않는다.
- 긴 지역 설명을 넣지 않는다.
- P1/P2 노드명을 모두 보이게 하지 않는다.
- 보급태그 종류를 한꺼번에 많이 늘어놓지 않는다.

## 5. 데스크톱 배치

```text
0~56       상단 상태줄
56~720     좌측 지도 70%, 우측 선택 패널 30%
```

| 영역 | 내용 | 규칙 |
|---|---|---|
| 좌측 지도 | E01 P0 확대 지도, 흐린 P1 신호 | 지도는 큰 단일 표면으로 둔다. |
| 우측 패널 | 선택 노드 정보, 예상 회수물, 행동 버튼, 짧은 로그 | 카드 여러 개를 쌓지 않는다. |
| 상단 상태줄 | 보급소/태그/회수선/현재 단계 | 데스크톱에서도 높이를 낮게 유지한다. |

우측 패널은 카드 묶음이 아니라 하나의 장비 패널처럼 보이게 한다. 내부 구획선은 써도 되지만, 독립 카드 여러 장이 떠 있는 레이아웃은 피한다.

## 9. 보스 신호 UI

R01-L03은 처음부터 완전히 숨기지 않는다. 목표가 있다는 감각은 주되, 세부 내용은 잠근다.

| 보스 신호 단계 | 조건 | 지도 표현 | 패널 문구 |
|---|---|---|---|
| 미약함 | 최초 화면 또는 신호 1개 | 아주 얇은 링 | 심사 신호 감지 |
| 감지됨 | 신호 2개 | 링이 조금 두꺼워짐 | 결절 신호 분석 중 |
| 근접 | 신호 3개 | 라벨 표시, 출격 가능 | 결절 접근 가능 |
| 침묵 | 첫 보스 처리 | 링이 꺼지고 잔류 점만 남음 | 스마일 홈 결절 침묵 |

주의:

- `스마일 홈 가족심사 관리자`라는 정식 역할명은 보스 상세나 조우 후에 노출한다.
- 첫 화면에서는 `심사 신호 감지` 정도가 적합하다.
- `스마일 홈 시어머니` 별칭은 보스전 조우 배너/HP 바에서 쓰는 편이 낫다.

## 10. 인양 표시

P0에서는 등급명을 과하게 설명하지 않는다.

| 제작용 내부 등급 | 지도 배지 | 패널 문구 | 사용 |
|---|---|---|---|
| A0 | 안정 | 귀환 안정 | R08-L01, R08-L02 |
| A1 | 인양 | 인양 가능 | R01-L01~R01-L03 |
| A2 이상 | 흐림 | 신호만 감지 | P1/P2 프리뷰 |

주의:

- `A0`, `A1`, `A2`는 제작 데이터와 디버그 표기다.
- 실제 노드 패널, 지도 배지, 경고 문구에는 내부 등급 코드를 직접 출력하지 않는다.

지도 표현:

- A0: 얇은 청록 안전 링.
- A1: 청록 링 안쪽에 작은 회수선 표시.
- A2 이상: 라벨 없는 흐린 잡음.

금지:

- `사망`, `부활`, `리스폰` 같은 단어를 쓰지 않는다.
- 인양을 무료 순간이동처럼 보이게 하지 않는다.
- R08-L02와 비콘/회수선의 관계를 화면에서 분리하지 않는다.

## 11. 보급태그 표시

자원명은 `docs/world/WORLD_TERMINOLOGY_LOCK_V0_1.md`의 태그 계열 잠금을 따른다.

첫 화면 상단에는 모든 태그를 펼치지 않는다.

| 위치 | 표시 후보 | 이유 |
|---|---|---|
| 상단 상태줄 | 보급태그 0 | 상위 자원 존재만 알려준다. |
| 노드 패널 회수물 | 식량태그 조각, 거주태그 흔적, 수신태그 후보 | 지역별 맥락에서만 보여준다. |
| 정산/보급소 화면 | 식량태그/충전태그/수신태그 등 상세 | 월드맵 첫 화면보다 정산 화면이 적합하다. |

판정:

```text
P0 월드맵 첫 화면에서는 보급태그 상위명만 선명하게,
개별 태그는 선택 노드의 예상 회수물에서만 제한적으로 보여준다.
```

## 12. P1 흐린 신호 표현

R01-L03 처리 후 표시되는 P1 프리뷰:

| 신호 | 위치 감각 | 표현 | 첫 문구 |
|---|---|---|---|
| R01-L04 | R01 아래쪽 | 짧은 점선 | 신호 해석 필요 |
| R01-L05 | R01 위/서쪽 | 불안정 화살표 | 귀환 신호 이상 |
| R03 | 남/남서 | 흐린 라벨 조각 | 라벨 신호 감지 |
| R02 | 동/도심 | 백색 잡음 | 백색 신호 감지 |
| R04 | 남부 | 회색 얼룩 | 충전 신호 감지 |
| R08 확장 | 보급소 주변 | 작은 공백 확장 | 침묵권 신호 |

처리:

- P1 프리뷰는 첫 보스 직후 바로 출격 가능 상태로 열지 않는다.
- 클릭하면 짧은 문구와 `신호 해석 필요` 버튼 상태만 보인다.
- 보급소 정산/대화/회수물 해석 후 실제 노드가 열린다.

## 13. 구현 데이터 후보

월드맵 UI는 아래 필드를 기준으로 만들면 된다.

| 필드 | 예시 | 설명 |
|---|---|---|
| `id` | `R01-L01` | 로컬 노드 ID |
| `label` | `침묵 가장자리` | 유저용 노드명 |
| `visibility` | `clear`, `faint`, `hidden` | 노출 정도 |
| `state` | `current`, `available`, `locked`, `boss_signal`, `cleared` | 노드 상태 |
| `extractionTier` | `A0`, `A1` | 제작용 인양 등급. 유저에게 직접 출력하지 않음 |
| `extractionLabel` | `귀환 안정`, `인양 가능` | 유저용 인양 표시 |
| `signalState` | `none`, `faint`, `detected`, `near`, `silent` | 보스/원격 신호 |
| `panelGoal` | `주택가 입구의 캠페인 신호를 확인한다` | 패널 목표 문구 |
| `rewardPreview` | `식량태그 조각, 낡은 보급품` | 예상 회수물 |
| `primaryAction` | `출격` | 주요 버튼 |
| `lockReason` | `경로 미확인` | 잠김 이유 |
