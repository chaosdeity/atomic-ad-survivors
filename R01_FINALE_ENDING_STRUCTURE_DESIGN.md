# R01 Finale Ending Structure Design

상태: P0-6 설계 기준서
범위: R01 외곽 주택가의 270~300초 피날레와 300초 종료 의미 정의
주의: 이 문서는 구현 지시 전 단계다. 보스 조건, 보상 수치, 성장 수치, 카드/업글, 맵 구조를 변경하지 않는다.

## 1. 현재 300초 종료 문제 요약

현재 300초 종료는 전투 루프상 생존 성공이지만, 플레이어에게는 "왜 끝났는지"와 "R01에서 무엇을 해낸 것인지"가 충분히 전달되지 않는다.

문제는 세 가지다.

1. 270초 이후 피날레 압박은 시작되었지만, 300초 도달이 세계관 사건으로 정리되지 않는다.
2. 첫 강제 회수와 일반 300초 종료가 감정적으로 충분히 분리되지 않는다.
3. 송출 기록 확보 상태와 보스 조우 가능 상태에 따라 다른 종료 의미가 필요하지만, 현재는 모두 비슷하게 느껴진다.

300초 종료는 단순 승리 화면이 아니라 "외곽 주택가를 한 바퀴 통과했고, 보급소가 회수 가능한 기록을 확보했다"는 사건이어야 한다.

## 2. 270~300초 피날레 목표

270~300초는 마지막 전투 보너스 시간이 아니라, R01이 플레이어를 놓치지 않으려는 짧은 압박 구간이다.

목표:

- 플레이어가 마지막 30초를 위험 구간으로 인식한다.
- 압박 링, 위험 전단 낙하, speaker/charger/signal/elite 전조가 "피날레"로 묶여 보인다.
- 300초 도달은 그냥 타임아웃이 아니라 보급소 회수선 또는 보스 진입선이 닫히는 순간으로 읽힌다.
- 보스 조건이 열려 있지 않은 상태에서도 300초 생존은 의미 있는 정찰 성과로 정리된다.
- 보스 조건이 열려 있는 상태에서는 300초 도달이 보스 진입 실패가 아니라 "진입 가능 상태 확인" 또는 "다음 조우 확정 신호"처럼 읽힌다.

피날레 감정:

```text
외곽 주택가는 끝나지 않았다.
하지만 이번 출격은 충분히 깊숙이 들어갔다.
```

## 3. 300초 도달 시 가능한 종료 타입

### 3.1 회수

기본 종료 타입이다. 보스전 중이 아니고 300초에 도달하면 침묵 보급소가 윤서를 회수한다.

용도:

- 송출 기록이 부족한 상태
- 보스 조건이 아직 열리지 않은 상태
- 첫 강제 회수 이후 일반 출격의 기본 성공 종료

표현 원칙:

- "승리"보다 "정상 회수"에 가깝게 표현한다.
- HP 0 사망과 다르게, 플레이어가 지역 통과에 성공했다는 감각을 준다.

### 3.2 보스 진입

보스 조건이 충족된 상태에서 240초 이후에는 기존 규칙대로 보스 조우가 우선한다. 300초 도달이 보스 조우 조건을 새로 열거나 변경해서는 안 된다.

용도:

- 송출 기록 3개 확보
- 보스 클리어 전
- 240초 이후 보스 루트 ready

처리 원칙:

- 보스 조우 가능 상태라면 300초 종료보다 보스 진입이 우선이다.
- 300초까지 보스가 시작되지 않았다는 상태는 디버그/버그 검토 대상이어야 한다.
- 구현 시 300초 판정 전에 기존 `_try_start_boss_encounter()` 우선순위를 유지한다.

### 3.3 신호 붕괴

첫 출격의 강제 회수는 신호 붕괴다. 일반 300초 종료와 혼동하면 안 된다.

용도:

- 첫 출격
- 70초 경고, 88초 surge, 108초 강제 회수

처리 원칙:

- 첫 회수는 실패/사망이 아니라 보급소가 윤서를 끌어낸 사건이다.
- 일반 300초 종료는 첫 회수 이후의 정상 회수 또는 정찰 완료다.
- 일반 300초 종료에 "신호 붕괴"를 남발하지 않는다.

### 3.4 탈출/정산

장기적으로는 300초 도달을 "탈출"로 부를 수도 있지만, 0.2에서는 외곽 주택가가 폐쇄형 탈출 맵이 아니다. 따라서 P0 구현에서는 "정상 회수 / 정산"이 더 안전하다.

용도:

- 300초 생존 완료
- 결과 화면과 보급소 정산으로 자연스럽게 연결

표현 원칙:

- 지역을 완전히 클리어했다는 표현은 피한다.
- "이번 출격에서 회수 가능한 기록을 충분히 확보했다"는 방향으로 정리한다.

## 4. 첫 회수와 일반 300초 종료의 차이

| 항목 | 첫 강제 회수 | 일반 300초 종료 |
| --- | --- | --- |
| 발생 시점 | 첫 출격 약 108초 | 첫 회수 이후 300초 |
| 원인 | 캠페인 신호 붕괴, 생체 한계 초과 | 보급소 회수선 도달, 정찰 완료 |
| 감정 | 위험한 세계에 끌려 나옴 | 한 구간을 버텨 통과함 |
| 결과 문구 톤 | 긴급, 생존, 회수 | 정상 회수, 기록 확보, 다음 목표 |
| 보상 의미 | 첫 회수 루프 개방 | 생존 성과와 송출 기록/흔적 정산 |
| UI 금지 | 사망 보상처럼 보이면 안 됨 | 완전 클리어처럼 보이면 안 됨 |

첫 강제 회수 문구 예:

```text
캠페인 신호가 윤서의 이름을 끝까지 읽기 전에, 침묵 보급소가 회수선을 당겼다.
```

일반 300초 종료 문구 예:

```text
침묵 보급소가 외곽 주택가의 회수선을 고정했다. 이번 출격의 기록은 끊기지 않았다.
```

## 5. 송출 기록 0~3개 상태별 종료 문구

문구는 내부 수치 대신 상태를 암시해야 한다. `records=2` 같은 표기는 일반 UI에 노출하지 않는다.

| 송출 기록 상태 | 300초 종료 문구 후보 | 다음 목표 톤 |
| --- | --- | --- |
| 0개 | 침묵 보급소가 회수선을 당겼다. 주택가의 광고음은 아직 윤서의 이름을 정확히 발음하지 못한다. | 첫 신호 이벤트 추적 |
| 0개 | 같은 울타리를 세 번 지나친 뒤에야 보급소 신호가 돌아왔다. 기록판은 아직 빈칸을 남겨 둔다. | 더 깊이 생존 |
| 1개 | 첫 잡음 영수증이 끊기기 전에 회수됐다. 다음 광고는 더 긴 문장 뒤에 숨어 있다. | 두 번째 신호 추적 |
| 1개 | 보급소 기록판에 얇은 영수증 잡음이 붙었다. 주택가는 다음 루프를 이미 준비하고 있다. | 180초 신호 접근 |
| 2개 | 길 잃은 카탈로그가 모델하우스 쪽으로 접혔다. 아직 마지막 절차가 빠져 있다. | 240초 신호 접근 |
| 2개 | 같은 집들의 순서가 어긋나기 시작했다. 보급소가 세 번째 신호 자리를 비워 둔다. | 모델하우스 결절 확인 |
| 3개 | 개장 전 점검표가 완성됐다. 외곽 주택가 중앙의 모델하우스 결절이 다음 출격을 기다린다. | 보스 조우 준비 |
| 3개 | 송출관 접근 절차가 끊기지 않고 보급소까지 넘어왔다. 스마일 홈 시어머니의 검증 신호가 가까워졌다. | 보스 진입 |

## 6. 보스 미조우 상태의 300초 종료 처리

보스 미조우 상태란 다음 중 하나다.

- 송출 기록이 3개 미만이다.
- 보스가 이미 처리되어 직접 조우 대상이 아니다.
- 보스 루트 ready가 아니다.

처리 원칙:

1. 300초 도달 시 정상 회수로 처리한다.
2. 결과 화면 제목은 `정상 회수` 또는 `외곽 주택가 정찰 완료` 계열이 적합하다.
3. 결과 설명은 송출 기록 상태에 맞춰 다음 목표를 안내한다.
4. 보스가 안 나왔다는 사실을 실패로 말하지 않는다.
5. 보급소로 돌아가 성장/정산 루프가 자연스럽게 이어져야 한다.

문구 예:

```text
외곽 주택가의 루프를 끝까지 버텼습니다.
침묵 보급소가 끊기지 않은 기록을 회수했습니다.
```

## 7. 보스 조우 가능 상태의 300초 종료 처리

보스 조우 가능 상태란 다음 조건을 말한다.

```text
송출 기록 3개 확보
보스 미처리
240초 이후
```

기존 기준상 이 상태에서는 스마일 홈 시어머니 조우가 가능해야 한다.

처리 원칙:

1. 보스 조우 가능 상태에서는 300초 일반 종료보다 보스 진입이 우선한다.
2. 만약 300초에 도달했는데 보스가 시작되지 않았다면, 결과 화면에서 보스 미등장을 자연문으로 숨기기보다 구현 진단 대상에 올린다.
3. 일반 UI는 "조건 충족 수치"를 보여주지 않고, "모델하우스 결절이 열렸다"는 문장으로 안내한다.
4. 300초 도달만으로 보스 보상이나 캠페인 코어 파편을 지급하지 않는다.

문구 예:

```text
모델하우스 결절의 현관등이 꺼지지 않는다.
보급소 기록판은 다음 출격을 보스 조우 절차로 고정했다.
```

## 8. 피날레 압박과 보상 정산 연결 원칙

피날레 압박은 보상 수치를 직접 늘리는 장치가 아니다. 보상은 기존 생존 시간, 처치, 엘리트 처치, 신호 이벤트, 흔적, 보스 성과 기준을 따른다.

연결 원칙:

- 270초 이후 압박은 플레이 감정과 종료 의미를 강화한다.
- 압박 링/위험 전단 낙하를 맞았다고 별도 재화를 지급하지 않는다.
- 압박을 피했다고 별도 업글 재료를 주지 않는다.
- 300초 도달은 기존 정산 루프를 여는 사건이지, 새 재화 지급 조건이 아니다.
- 송출 기록은 소비/차감되지 않는다.
- 캠페인 송출관은 후속 목표/상위 신호로만 언급한다.

결과 화면에서 가능한 표현:

```text
피날레 압박을 버티고 회수선에 도달했습니다.
```

결과 화면에서 피해야 할 표현:

```text
피날레 압박 보상 +1
송출 기록 소비
좌표 조각 획득
```

## 9. 금지사항

- 보스 조건 변경 금지.
- 300초 도달만으로 보스 보상 지급 금지.
- 보스 조우만으로 캠페인 코어 파편 지급 금지.
- 보상 수치 변경 금지.
- 업글/카드/성장 수치 변경 금지.
- 새 재화 추가 금지.
- `좌표 조각` 추가 금지.
- `송출 기록` 소비/차감 금지.
- 내부 상태 수치 일반 UI 직접 노출 금지.
- 외곽 주택가를 완전 클리어 후 버리는 맵처럼 표현 금지.
- 0.2에서 캠페인 송출관 직접 보스전 구현 금지.

## 11. 구현 우선순위 제안

1. TASK A: 결과 타입 라벨 분리
2. TASK B: 송출 기록 상태별 종료 문구
