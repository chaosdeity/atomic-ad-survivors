# 3. 적 디자인 문법

## A — 어두운: "기계의 부패"

- **공통 시각 요소**: 노출된 척추형 파이프, 발광하는 녹색 코어/안구, 가죽처럼 벗겨진 살, 황동 부속
- **위협도-디자인 상관**:
  - Tier 1 (잡몹 24–32px): 작은 변종 짐승. 그린 코어 1개
  - Tier 2 (엘리트 40px): 인간형 시체 + 기계 부속. 그린 코어 2–3개
  - Tier 3 (보스 80px+): 거대 산업 로봇 / 융합체. 코어 5개+ 와 거대 파이프
- **귀여움 vs 위협감**: 100% 위협. 귀여움 0%
- **변종 확장**: 색조 시프트로는 부족 → **부속 추가**(파이프 수, 코어 수, 입의 균열)로 변종을 만든다

## B — 밝은: "광고 마스코트의 부패"

- **공통 시각 요소**: **활짝 웃는 둥근 얼굴** + 광고 카피("BUY NOW!") + 흘러나오는 그린 슬라임. 표면은 광고지만 갈라진 균열에서 누출
- **위협도-디자인 상관**:
  - Tier 1 (32px): 빌보드 종양, 가전 좀비. 균열 0–1개
  - Tier 2 (40px): 1950s 주부 좀비, 마스코트 햄스터. 균열 2–3개 + 슬라임
  - Tier 3 (보스): "Cheerful Corporate Mascot Robot". 강제 미소 + 한쪽 눈 깨짐
- **귀여움 vs 위협감**: **60:40 (귀여움 우세지만 위협이 침범)**. 균열·슬라임이 위협 시그널
- **변종 확장**: 같은 마스코트 베이스에 **광고 카피만 교체**해서 변종 양산 가능 ("BUY", "SALE", "NEW!" 등)

## C — 귀여운: "슬픈 마스코트"

- **공통 시각 요소**: 둥근 큰 눈 + **눈물 자국** + 단추 봉합선 + 분홍/베이지 봉제. 갈라진 곳에서 별이나 솜이 새어나옴
- **위협도-디자인 상관**:
  - Tier 1 (16×16): TV 광고 유령, 호박 머리 좀비
  - Tier 2 (36–48px): 곰인형, 햄스터. 망치/막대사탕 무기
  - Tier 3 (보스): "Please Stay" 마스코트. 거대 눈물 + 말풍선 + 페로몬 슬라임
- **귀여움 vs 위협감**: **80:20**. 위협은 *체념과 끈적임*으로 표현 (잡으려 손 뻗는 자세, 페로몬 장판)
- **변종 확장**: **같은 봉제 메시에 색만 바꾸기**가 충분히 작동. 의상·악세서리(리본, 단추 수)로 변종

**모든 톤 공통 적군 규칙**
- 적의 정면 픽업 가능 영역(hitbox)은 스프라이트 중심 **±10px**
- 죽음 모션: 1프레임 발광 → 분해 파편 8개 (파편 색은 톤별 강조색)
- 적군 외곽선 색은 캐릭터와 동일 라이너 색 통일

# 7. Godot 4 구현 시 주의점

## 7.1 권장 프로젝트 셋업 (공통)

```ini
# project.godot 핵심 설정
[display]
window/size/viewport_width = 480
window/size/viewport_height = 270         # 16:9, 1px = 1tile×0.5 기준
window/size/window_width_override = 1920
window/size/window_height_override = 1080
window/stretch/mode = "viewport"          # 픽셀 퍼펙트
window/stretch/aspect = "keep"
window/stretch/scale_mode = "integer"     # ★ 정수 배율만 허용

[rendering]
textures/canvas_textures/default_texture_filter = 0   # ★ Nearest
2d/snap/snap_2d_transforms_to_pixel = true
2d/snap/snap_2d_vertices_to_pixel = true
anti_aliasing/quality/msaa_2d = 0
```

**카메라 (Camera2D)**
- `position_smoothing_enabled = false` (또는 매우 낮게)
- 카메라 위치를 `round()` 처리해서 sub-pixel 흔들림 차단
- 화면 흔들림은 `offset`을 정수 픽셀로만 변경

**TileMap**
- 모든 톤 16px 베이스 → `TileSet > tile_size = (16, 16)`
- `physics_layers`는 벽/장애물/파괴가능 3개 분리
- 파괴 가능 타일은 `TileMapLayer` 별도 분리 (16×16 묶음 단위 파괴)

## 7.2 톤별 추가 주의점

### A 어두운
- **Light2D 다수**(가스등, 코어). `CanvasModulate` 배경 `#1A1612` 후 점광원 색 `#7FFF3F`/`#FFB020`
- 라이트 텍스처는 **64×64 그라데이션 PNG** 1장 + `color` 속성으로 톤 변형
- `OccluderPolygon2D`를 벽 타일에 자동 부여 (스크립트로 일괄)

### B 밝은
- Light2D 거의 불필요 → 평면 컬러
- `CanvasModulate` 사용하지 않음
- 데미지 플래시는 `Material > shader`로 흰색 합성 — 광고 폭죽 색감 유지를 위해 `lighten` 모드

### C 귀여운
- 모든 스프라이트에 **드롭섀도우 셰이더** 1패스
  ```gdshader
  shader_type canvas_item;
  uniform vec4 shadow_color : source_color = vec4(0.55, 0.42, 0.36, 0.30);
  uniform vec2 shadow_offset = vec2(1.0, 1.0);
  // ...sample at -offset, blend behind
  ```
- 입자(별·하트)는 `GPUParticles2D` + 커스텀 메시 (사각형 대신 별 폴리곤)

## 7.5 톤 전환 가능성을 남기는 방법

발주 전 톤이 흔들릴 수 있다면, 다음 조치로 후속 전환 비용을 줄일 수 있음:
1. **모든 색을 `Color` 변수 또는 `Theme`을 통해 참조** — 스프라이트 외 모든 UI/이펙트 색은 코드 상수 금지
2. **VFX는 셰이더 uniform 색으로** 정의 (텍스처는 그레이스케일)
3. 단, **캐릭터·적 픽셀아트는 톤 전환 불가** — 한 톤 확정 후 발주

| 항목 | A 어두운 | B 밝은 | C 귀여운 |

**권장 (편향 없는 관찰)**:
- **차별화 + 정서적 강도**를 원하면 **C 귀여운**

- **장르 정통성 + 코어 유저 어필**을 원하면 **A 어두운**

3개 모두 동일 세계관(원자력 시대 + 방사능)을 공유하므로 **시스템 설계는 톤 무관**하게 진행 가능하며, 톤 확정은 캐릭터·타일 발주 직전까지 미룰 수 있습니다.
