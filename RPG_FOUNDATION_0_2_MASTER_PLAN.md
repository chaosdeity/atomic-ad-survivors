# RPG Foundation 0.2 Master Plan

## 0. 목적

이 문서는 기존 4개 전략 문서의 방향을 폐기하지 않는다. 다만 `10분 검증 슬라이스`를 바로 구현 대상으로 삼기 전에, Atomic Ad Survivors 0.2가 먼저 고정해야 하는 RPG 기반 구조를 새 기준으로 둔다.

핵심 결론은 다음과 같다.

```text
검증 슬라이스는 목표가 아니라 게이트다.
0.2는 먼저 출격형 광고 정산 액션 RPG의 기반을 갖춰야 한다.
```

## 1. 현재 0.2의 정확한 장르 정의

### 판정

0.2의 중심 장르는 `출격형 광고 정산 액션 RPG`다.

| 후보 | 판정 | 이유 |
|---|---|---|
| 뱀서류 | 부분 요소 | 탑다운 이동, 자동공격, 다수 적, 런 중 카드 성장은 뱀서류 문법을 쓴다. 그러나 이것이 장르 중심은 아니다. |
| 로그라이트 | 구조 요소 | 반복 출격, 실패 후 정산, 보급소 복귀, 영구 강화가 있다. 다만 죽음 파밍이나 랜덤 빌드 중심으로 읽히면 실패다. |
| 액션 RPG | 강한 하위축 | 윤서 조작, 차징 반품 도장, 적 절차 대응, 지역 위험 선택이 순간 재미를 담당한다. |
| 캠페인 RPG | 상위축 | R01 지역 상태, NPC 반응, 태그 권한, 보스 결절, 보급소 귀환이 장기 의미를 만든다. |
| 출격형 RPG | 핵심 형식 | 한 번 나가고, 위험을 안고, 회수되어 돌아오고, 정산과 장소 반응을 거쳐 다시 들어간다. |
| 생존 RPG | 감정 축 | HP 0은 사망이 아니라 긴급 인양이며, 태그는 돈이 아니라 생존권/접근권이다. |
| 혼합 장르 | 보조 설명 | 섞인 것은 맞지만 중심축 없이 혼합이라고 부르면 설계 판단이 흐려진다. |

따라서 0.2의 설계 질문은 “뱀서류보다 재미있는가?”가 아니라 다음이어야 한다.

```text
나간 행동이 정산되고, 정산이 보급소와 R01을 바꾸며,
그 변화 때문에 다시 위험한 지역 안쪽으로 들어가고 싶은가?
```

## 2. 축약 검증 전에 먼저 만들어야 할 기반

### 2.1 E01/R01 맵 계층

`Viewport`, `Local Combat Map`, `R01 Campaign Region`, `E01 World Campaign Graph`, `Global Campaign Network`를 분리한다. 480x270은 화면이고, 1920x810은 초기 블록아웃 또는 짧은 노드이며, R01 전체의 기준이 아니다.

### 2.2 R01 대형 지역 기준

R01은 작은 아레나가 아니라 큰 캠페인 지역이다. 첫 실전 기준은 최소 64 screens급으로 잡고, 121 screens 후보는 큰 R01 블록아웃 검증 대상으로 둔다. 단순히 맵을 크게 만드는 것이 아니라 `침묵 가장자리 -> 반복 주택가 -> 오픈하우스 거리 -> 모델하우스 접근로`가 이동 거리와 위험으로 읽혀야 한다.

### 2.3 위험 구역 선택 구조

유저는 더 안전한 경계에 머물지, 오픈하우스 거리로 들어가 태그 후보와 감사 리스크를 얻을지, 배수로 침묵 주머니나 가짜 귀환로를 선택할지 판단해야 한다. 위험 구역은 적 밀도만 올리는 공간이 아니라 정산 결과와 지역 변화를 바꾸는 선택지다.

### 2.4 광고 감사 / 카드 엔진

카드 선택은 단순 능력치 상승이 아니라 다음 감사 기준을 준비하는 행위여야 한다. 차징, 표식, 처치, 체류, 저체력 생존, 오브젝트 접촉이 정산 근거로 남고, 감사 실패는 맵 반응으로 돌아와야 한다.

### 2.5 NPC 노출 제한

NPC를 많이 보여주는 것이 RPG가 아니다. 0.2는 보급소 핵심 NPC 5명의 역할을 고정하되 직접 반응은 2~3명부터 시작하고, R01 NPC는 직접 대면/음성 잔향/오브젝트 흔적/P1 예고로 단계화한다.

### 2.6 작은 보급소 / 밀도형 허브 원칙

침묵 보급소는 큰 마을이 아니다. 0.2에서는 작은 공간 안에 회수 플랫폼, 정산 카운터, 정비대, 이름 보관함, 출격 게시판, 차징 조율대가 밀도 있게 읽혀야 한다.

### 2.7 회수 실패 비용

HP 0은 죽음이 아니지만 공짜 귀환도 아니다. 빠른 실패가 최적 파밍이 되지 않도록 후보 손실, 보류, 오염, 지역 악화, NPC 반응, 가짜 귀환로 강화 같은 비용이 있어야 한다.

### 2.8 태그 경제 계약

식량태그, 충전태그, 수신태그는 돈이 아니라 생존권/정비권/접근권이다. 후보, 승인, 보류, 오염을 구분하고, 보급소의 물리 시설과 NPC 반응에 연결한다.

### 2.9 보스/죽음/승리 용어 마이그레이션

`GAME OVER`, `Victory`, `boss_victory`, `스마일 홈 시어머니` 기본명, `캠페인 송출관` 첫 보스화는 유저-facing 기준에서 정리되어야 한다. 내부 상태명은 남더라도 화면 문구는 `긴급 인양`, `정상 회수`, `정산 실패`, `결절 처리`, `심사 중단`으로 변환해야 한다.

### 2.10 몹/오브젝트 배치 논리

몹과 trace는 아무 곳에나 있으면 안 된다. 쿠폰 전단은 광고물/우편함/표지판 근처, 미소 우편함은 집 앞 등록선, 청소기/홈케어 장치는 빈 집과 청소 경로, 상담원/키오스크는 오픈하우스 거리, 행복 보증 상담원은 모델하우스 접근로에서 나와야 한다.

### 2.11 저장 경계

0.2에서 장기 세이브 전체를 완성하지 않는다. 다만 세션 결과, 첫 회수 여부, R01 방문 상태, 태그 후보/승인/오염, 보급소 반응, 보스 결절 처리 여부처럼 테스트 판단에 필요한 최소 상태는 경계가 있어야 한다.

### 2.12 행동 기반 플레이테스트 지표

설문만으로 판정하지 않는다. 다음 행동을 본다.

| 지표 | 의미 |
|---|---|
| 재출격 선택까지 걸린 시간 | 보급소와 정산이 다음 행동을 만들었는가 |
| 위험 구역 재진입률 | R01이 단순 전투장이 아니라 탐색 대상인가 |
| 태그 후보/승인 이해도 | 태그가 돈이 아니라 권한으로 읽히는가 |
| 회수 원인 설명 가능 여부 | HP 0/정상 회수/정산 실패가 구분되는가 |
| NPC 반응 기억률 | NPC가 기능 UI 얼굴이 아니라 사건의 증인인가 |
| 맵 방향 언급 | L01/L02/L03가 같은 지역 안쪽으로 느껴지는가 |

## 3. 지금 바로 10분 테스트하면 안 되는 이유

현재 상태에서 바로 10분 테스트를 하면, 전략 자체가 아니라 미완성 기반의 결함을 테스트하게 된다.

| 영역 | 잘못된 판정 위험 |
|---|---|
| 맵 | R01이 지역이 아니라 넓은 도형으로 보이면 “캠페인 RPG가 재미없다”는 잘못된 결론이 난다. |
| 보급소 | 보급소가 장소가 아니라 텍스트 UI면 “귀환 감정이 없다”는 결과만 나온다. |
| NPC | NPC가 지역 압력의 결과가 아니라 설명문이면 세계관이 장황하다는 인상을 준다. |
| 전투 | 적 공격이 광고/주거/가족 심사 절차로 읽히지 않으면 그냥 약한 뱀서류로 보인다. |
| 태그 | 식량태그/충전태그/수신태그가 권한이 아니라 보상 재화처럼 보이면 RPG 경제가 무너진다. |
| 회수 | HP 0과 정상 회수, 보스 회수의 비용 차이가 없으면 긴장감이 없다. |
| 용어 | 죽음/승리/처치 표현이 섞이면 새 정체성이 화면에서 배신된다. |
| 카드 | 카드가 감사 준비가 아니라 스탯 카드면 광고 정산 RPG의 핵심이 사라진다. |

## 4. 먼저 만들어야 할 최소 기반

이 목록은 완성 범위가 아니라, 검증이 의미 있으려면 최소한 있어야 하는 기반이다.

| 기반 | 최소 기준 |
|---|---|
| R01 지역감 | 64 screens 이상으로 읽히는 큰 지역 구조. 이동 거리, 시야, 위험 구역, 회수선이 구분되어야 한다. |
| L01/L02/L03 구분 | L01은 외곽 진입, L02는 반복 주택가 안쪽, L03는 모델하우스 접근으로 거리와 위험이 다르게 느껴져야 한다. |
| 보조 루트 | L04 배수로 침묵 주머니와 L05 가짜 귀환로는 가지/위험/재방문 루트로 남아야 한다. |
| 보급소 | 작은 장소로 읽히고 최소 5개 시설의 위치와 역할이 분리되어야 한다. |
| 첫 회수 비용 | 첫 강제 회수는 세계관 입문이면서 이후 일반 실패와 다른 예외임이 명확해야 한다. |
| 태그 3종 | 식량태그, 충전태그, 수신태그가 돈이 아니라 접근권으로 보이며 후보/승인/보류/오염 상태를 가져야 한다. |
| NPC 직접 노출 | 보급소 2~3명, R01 직접 대면 1~2명 이하부터 시작한다. 나머지는 잔향/오브젝트/예고로 숨긴다. |
| 배치 이유 | 적과 오브젝트는 광고 인프라, 주소 등록선, 청소 경로, 상담 구역 같은 출처를 가져야 한다. |
| 용어 | 유저-facing 화면에서 죽음/부활/승리/처치/시어머니 기본명 혼선을 제거한다. |
| 지표 | 플레이테스트 전에 행동 로그와 질문지를 함께 정의한다. |

## 5. 기반 통과 판정

0.2는 다음 조건을 통과한 뒤 10분 검증 슬라이스로 넘어간다.

1. R01이 작은 스테이지 버튼 묶음이 아니라 하나의 큰 주택가 지역으로 읽힌다.
2. 보급소가 업글 메뉴가 아니라 돌아온 장소로 읽힌다.
3. 태그가 골드가 아니라 생존권/정비권/접근권으로 읽힌다.
4. 회수 실패가 죽음보다 덜하지만, 공짜 보상보다 훨씬 불편하다.
5. NPC가 기능 UI 얼굴이 아니라 출격 결과를 받아내는 사람/로봇으로 보인다.
6. 적과 오브젝트가 광고 절차의 일부로 보인다.
7. 보스는 강한 적이 아니라 R01 지역 약관의 얼굴로 보인다.

## 6. 기존 4개 전략 문서의 대체 기준

기존 4개 문서는 0.2의 감정 목표와 검증 질문을 잡는 데 유효하다. 그러나 구현 순서의 최상위 기준은 아래로 대체한다.

```text
Foundation first.
R01 지역감, 보급소 장소성, 태그 계약, 회수 비용, 용어 정리가 통과된 뒤
10분 플레이 검증을 한다.
```
