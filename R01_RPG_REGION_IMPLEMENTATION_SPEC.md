# R01 RPG Region Implementation Spec

## 0. R01 핵심 재미

R01의 핵심 재미는 다음 한 문장으로 고정한다.

```text
위험하지만 들어가야 다음 판이 바뀌는 지역
```

R01은 넓은 전투장이 아니다. 윤서가 더 안쪽으로 들어갈수록 광고 감사, 가족 심사, 주소 등록, 회수선 불안, 태그 후보/오염, NPC 반응이 바뀌는 캠페인 지역이다.

## 1. 위험 구역 설계

위험 구역은 단순 적 밀도 증가가 아니다. 각 구역은 `위험`, `얻는 근거`, `잃을 수 있는 것`, `다음 방문 변화`를 가져야 한다.

| 구역 | 위험 | 얻는 근거 | 잃을 수 있는 것 | 다음 방문 변화 |
|---|---|---|---|---|
| 침묵 가장자리 | 낮은 압력, 낮은 보상 | 첫 흔적, 기본 정산, 회수선 안정 | 깊은 태그 후보 부족 | 첫 회수 후 안내 문구와 회수선 표시 변화 |
| 반복 주택가 | 주소 등록, 우편함 압력 | 식량태그/충전태그 후보, 기본 trace | 등록 압력 누적, 후보 보류 | 우편함/문패/전단의 문구 변화 |
| 오픈하우스 거리 | 상담원/키오스크, 광고 감사 | 수신태그 후보, 송출 기록, 고위험 정산 근거 | 캠페인 등록 압력 상승, 수신 오염 | 오픈하우스 간판, 방송, 적 출현 위치 변화 |
| 모델하우스 접근로 | 가족 심사, 보스 전조 | 결절 접근 근거, 보스 신호 | 인양 지연, 태그 오염, 심사 중단 실패 | 스마일 홈 심사관 접근 문구와 가족사진 잔향 |
| 배수로 침묵 주머니 | 낮은 신호, 인양 불안 | 조용한 우회, 숨겨진 trace | 회수선 안정도 하락, 수신 보류 | 구조 신호 약화/회복 여부 표시 |
| 가짜 귀환로 | 빠른 귀환처럼 보이나 등록 절차 강화 | 짧은 루트, 위험한 식량태그 후보 | 식량태그 오염, 반복 실패 압력 | 가짜 귀환 표지 강화, NPC 경고 변화 |

## 2. 광고 감사 / 카드 엔진 연결

### 원칙

카드는 단순 능력치 선택이 아니다. 유저가 다음 감사 기준을 준비하는 장부 조각이어야 한다.

| 연결점 | 구현 의미 |
|---|---|
| 카드 선택 | 이번 런의 행동이 어떤 정산 근거로 기록될지 결정한다. |
| 위험 구역 체류 | 태그 후보, 오염, 승인 가중치, 캠페인 압력에 영향을 준다. |
| 감사 실패 | 숫자 손실만이 아니라 다음 방문의 맵 반응으로 남는다. |
| 오픈하우스 거리 체류 | 수신태그 후보와 송출 기록 후보를 만든다. |
| 가짜 귀환로 | 식량태그 후보를 빠르게 주는 것처럼 보이나 오염 또는 보류 위험을 건다. |
| 배수로 | 수신 강도를 낮추는 대신 인양 안정도를 흔든다. |
| 반품 도장 차징 | 표식/정산 근거를 남기는 윤서의 고유 대응이다. |

### 감사 결과가 맵으로 돌아오는 방식

| 결과 | 맵 반응 |
|---|---|
| 식량태그 후보 승인 | 보급소 인간 구역 반응, 반복 주택가의 일부 압박 완화 |
| 식량태그 오염 | 가짜 귀환로 표식 강화, 집 앞 배급 광고 왜곡 |
| 충전태그 후보 승인 | 정비대/로봇 반응, 홈케어 장치 일부 지연 |
| 충전태그 보류 | 청소기/홈케어 장치 재동기화, 정비대 불안 |
| 수신태그 후보 승인 | 출격 게시판 정보 증가, 오픈하우스 거리 방향성 강화 |
| 수신태그 오염 | 잘못된 귀환 표지, 송출관 잔향 과노출 |

## 3. 적/오브젝트 배치 규칙

```text
몹과 오브젝트는 아무 데나 있으면 안 된다.
어디서 왜 나왔는지 암시해야 한다.
```

| 대상 | 배치 출처 | 금지 |
|---|---|---|
| 쿠폰 전단 떼 | 광고물, 우편함, 표지판, 오픈하우스 현수막 근처 | 빈 도로 한가운데 무작위 스폰 |
| 미소 우편함 | 집 앞, 주소 등록선, 문패가 반복되는 골목 | 열린 광장에 의미 없이 배치 |
| 홈케어 청소기 | 빈 집 입구, 청소 경로, 모델하우스 관리선 | 쿠폰 전단과 같은 원형 무리로만 사용 |
| 상담원/키오스크 | 오픈하우스 거리, 안내 부스, 분양 표지 근처 | 침묵 가장자리 초반에 과노출 |
| 행복 보증 상담원 | 모델하우스 접근로, 가족사진 프레임, 계약 테이블 근처 | 일반 적처럼 초반부터 반복 |
| 위험 전단/압박 링 | 캠페인 등록 압력 상승 지점, 감사 실패 지점 | 플레이어 주변 랜덤 장판처럼 사용 |
| trace | 위험 동선, 이야기 이유, 오브젝트 출처가 있는 지점 | 반짝이는 재화처럼 길바닥에 흩뿌림 |

## 4. 재방문 변화

R01은 같은 맵을 반복 플레이하는 곳이 아니라, 회수 결과를 들고 다시 들어가는 지역이어야 한다.

| 조건 | 변화 |
|---|---|
| 첫 강제 회수 후 | 외곽 광고 문구가 윤서의 회수 기록을 반영한다. 보급소 NPC가 회수 원인을 다르게 말한다. |
| 송출 기록 1 후 | 오픈하우스 거리 방향의 신호가 더 또렷해진다. |
| 송출 기록 2 후 | 모델하우스 접근로의 가족 심사 문구가 섞인다. |
| 송출 기록 3 후 | 스마일 홈 심사관 접근 조건이 보인다. 송출관은 잔향으로만 남긴다. |
| 태그 오염 후 | 관련 구역의 광고/오브젝트/적 출현이 오염된 태그 방향으로 변한다. |
| 보스 outcome 후 | R01은 완전히 정화되지 않는다. 심사 중단, 기억 추출, 결절 처리 결과에 따라 잔향이 달라진다. |
| 반복 실패 후 | 가짜 귀환로가 더 그럴듯해지고, 빠른 실패 파밍을 유도하는 대신 비용을 키운다. |

## 5. R01 구현 우선순위

### P0. 지역으로 읽히기 위한 최소 기반

| 항목 | 기준 |
|---|---|
| 64 screens급 R01 실전 구조 | 넓다는 느낌보다 경계/주택가/오픈하우스/모델하우스 접근의 거리감이 우선이다. |
| 주 경로 4구역 | 침묵 가장자리, 반복 주택가, 오픈하우스 거리, 모델하우스 접근로를 실제 이동과 위험으로 구분한다. |
| 보조 루트 씨앗 | 배수로 침묵 주머니와 가짜 귀환로는 완성하지 않더라도 위치/의미/위험을 남긴다. |
| 배치 출처 | P0 적/오브젝트는 모두 source가 있어야 한다. |
| 광고 감사 hook | 체류, 차징, 표식, 적 처리, trace 회수가 정산 근거로 기록될 구조를 둔다. |
| 태그 후보/보류/오염 hook | 실제 수치 밸런스보다 상태 구분이 먼저다. |
| 첫 재방문 변화 | 첫 회수 후 맵 문구/위험/보급소 반응 중 최소 1개가 변한다. |
| 회수선 감각 | 안전한 외곽과 불안한 안쪽의 차이를 표시한다. |

### P1. 캠페인 반응 강화

| 항목 | 기준 |
|---|---|
| L04/L05 실사용 | 배수로와 가짜 귀환로가 선택 가능한 작전권으로 작동한다. |
| R01 NPC 잔향 | 직접 대면 1~2명, 음성 잔향 1명, 오브젝트 흔적 1~2개를 배치한다. |
| 송출 기록 단계 | 1/2/3 단계가 지도, 보급소, R01 문구에 반영된다. |
| 보스 전조 | 스마일 홈 심사관이 적 하나가 아니라 지역 약관의 얼굴로 예고된다. |

### P2. 확장

| 항목 | 기준 |
|---|---|
| 보스 outcome 세분화 | 기억 추출, 심사 중단, 결절 처리 등 결과에 따른 R01 변화 확장 |
| 고밀도 배치 변형 | 30/100/300 밀도 실험을 source 기반 배치와 함께 검증 |
| NPC 후폭풍 | R01 NPC 매트릭스의 P1 예고 인물들을 단계적으로 노출 |
| E01 연결 | R01 이후 지역 슬롯과 송출관 잔향을 광역 작전도로 연결 |

## 6. P0 통과 기준

1. 플레이어가 “다음 스테이지”가 아니라 “같은 동네 안쪽”으로 들어간다고 느낀다.
2. 적이 아무 데서나 나오는 몬스터가 아니라 광고/주거 절차에서 발생한다고 읽힌다.
3. 위험 구역에 머무는 이유와 비용이 동시에 보인다.
4. 회수 후 R01이 최소 하나 이상 변한다.
5. 스마일 홈 심사관은 갑툭튀 보스가 아니라 모델하우스 접근로가 낳은 얼굴로 예고된다.
