# Supply Upgrade Playtest Checklist

작성일: 2026-05-17
대상: Meta Upgrade Tree 1차 적용 후 보급소 UI, 16개 업글, 신호 단서 기반 보스 루트

## 1. 목적

이 문서는 새 기능 구현 문서가 아니라 플레이테스트 기준 문서다.

확인 목표:

- 첫 회수 후 보급소 진입이 자연스러운지 확인한다.
- 16개 업글이 270px 기준 UI에서 읽히는지 확인한다.
- 구매 상태, 잠김 조건, 재화 부족 상태가 헷갈리지 않는지 확인한다.
- 구매 후 다음 출격에 실제 스탯이 반영되는지 확인한다.
- 신호 단서 기반 보스 루트가 플레이어에게 납득되는지 확인한다.
- 보스 조우까지 평균 런 수와 피로도를 기록한다.

## 2. 현재 자동 검증 기준

필수 명령:

```powershell
powershell -ExecutionPolicy Bypass -File tools\qa_check.ps1
```

통과 기준:

- Godot headless에서 `scenes/main.tscn`이 오류 없이 로드된다.
- `tools/balance_sim.py`가 Meta Upgrade Tree 16개와 효과 프리뷰를 출력한다.
- `tools/playtest_metrics_probe.gd`가 첫 5분 계측 연결을 확인한다.
- `tools/ui_compaction_probe.gd`가 정산/보급소 긴 문구가 모바일 UI를 넘치게 만들지 않는지 확인한다.
- `git diff --check`에서 whitespace error가 없어야 한다.
- 기존 dirty 문서가 있어도 코드 검수와 분리해서 판단한다.

## 12. 현재 자동 확인된 사실

최근 코드 기준으로 자동 확인할 수 있는 사항:

- `scripts/meta_progression.gd`의 업글 수는 16개다.
- 기존 4개 업글 ID는 유지된다.
- 보급소 버튼은 16개 이상 동적 생성된다.
- ScrollContainer 콘텐츠 높이가 표시 영역보다 커져 스크롤 대상이 된다.
- `선택 가능`, `흔적 부족`, `잠김` 문구가 코드상 분리되어 있다.
- 보급소 로그/힌트/액션 버튼은 자동 프로브 기준 1줄 compact 표시를 유지한다.
- 정산 화면은 자동 프로브 기준 12줄 이하 compact 표시를 유지한다.
- 다음 출격 초기화 시 `meta_progression.bonuses()`가 `player_stats`에 반영된다.
- XP, 차징, 사거리, 저체력 방어, 엘리트 피해, 보스 방패 피해 보정 연결점이 존재한다.
- 일반 보상 예상치는 `후보(미지급)`으로 표시되고, 실제 전단 보정은 `전단 회수 동선 보정`으로 표시된다.

자동 확인만으로 부족한 사항:

- 실제 손플레이 기준 보급소 복잡도.
- 스크롤 발견성.
- 보스 첫 조우까지 피로도.
- 업글 선택이 다양하게 느껴지는지.
- 보스 대응 업글이 필수처럼 느껴지는지.

## 13. PMO 판단 필요 항목

1. 보스 첫 조우까지 목표 평균 런 수를 몇 회로 잡을지.
2. 16개 업글을 첫 보급소부터 모두 보여줄지, 카테고리/단계별로 접을지.
3. 중복 단서 전환 전단 보상이 장기적으로 과한지.
4. `전단 회수 동선`이 빠른 파밍을 유도하는지.
5. 보스 대응 업글을 선택 보상으로 둘지, 보스 학습 필수 루트로 둘지.
6. 장기적으로 저장/불러오기 없이 세션 성장만 유지할지.
