## 0. Core Position

Atomic Ad Survivors의 0.2 성장은 다음 문장으로 고정한다.

```text
런은 매번 새롭고,
세계는 매번 누적된다.
```

단, 이 철학을 UI로 설명하지 않는다. 유저는 먼저 윤서의 전투 손맛, 카드 선택, 보급소 귀환, 개조 체감, 보스 재도전 욕구를 느껴야 한다. 깊이 파는 유저만 외곽 주택가가 자기 행동을 기억하고 있다는 구조를 발견하면 된다.

0.2 고정 사항:

| 항목 | 결정 |
| --- | --- |
| 플레이어블 | 윤서 1명 |
| 자동공격 | 반품 스캐너 |
| 차징 스킬 | 반품 도장 |
| 거점 | 침묵 보급소 |
| 필수 시설 | 정산 카운터, 정비대, 이름 보관함, 출격 게시판, 차징 조율대 |
| 지역 | `r01_suburb / 외곽 주택가` |
| 첫 보스 | 스마일 홈 시어머니 |
| 보스 선택 | 결절 파괴, 기억 추출 |
| 반복 재화 | 광고 전단 조각 |
| 보스 재화 | 캠페인 코어 파편 |
| 스토리 단서 | 송출 기록 3개, 송출관 접근 절차 확정 |
| P0 흔적 | 지워지지 않은 문패, 반쯤 탄 가족사진, 이름 없는 영수증 |
| P1 흔적 | 고장 난 마스코트 배지, 납 차폐 배터리, 무음 라디오 |

금지:

- `좌표 조각` 재화 추가
- `송출 기록`을 소비 재화로 사용
- 사망 자체 보상
- 보스 조우만으로 코어 파편 지급
- 내부 상태 수치를 UI에 직접 노출

## 1. 0.2 RPG Growth Loop

### Core Loop

```text
출격
-> 런 중 카드 선택
-> 자동공격/차징/회피로 성과 획득
-> 광고 전단 조각, 흔적, 송출 기록, 코어 파편 조건 달성
-> 회수/사망/보스 처리
-> 보급소 정산
-> 윤서/장비/보급소/지역 상태 변화
-> 재출격
```

회수와 사망은 보상 원인이 아니라 정산 지점이다. 정산 원인은 생존 시간, 처치, 엘리트 처치, 신호 이벤트 성공, 흔적 회수, 보스 HP 성과, 보스 처리다.

### First 10 Minutes

목표:

- 이동하면 자동공격이 나간다는 것을 즉시 이해한다.
- 반품 도장이 단일 버튼 차징 스킬로 시원하게 느껴진다.
- 첫 전투는 HP 0 사망이 아니라 `signal_overload` 강제 회수로 끝난다.
- 첫 회수 후 반드시 개조 1개를 한다.

권장 흐름:

```text
0:00~0:30  쿠폰 전단 떼로 이동/자동공격 학습
0:30~1:10  차징 윈도우와 반품 도장 학습
1:10~1:40  캠페인이 윤서 이름을 읽으려 함
~1:40      신호 과부하 강제 회수
보급소     광고 전단 조각 정산, 첫 개조
재출격     같은 지역이지만 더 잘 버틸 수 있음
```

성공 감정:

```text
졌는데 끝난 게 아니라, 다음 출격이 바로 기대된다.
```

### First 30 Minutes

목표:

- 120초 전후 신호 이벤트를 본다.
- P0 흔적 1개 이상을 회수한다.
- 보급소 시설이 오브젝트/대사/기능으로 반응한다.
- 광고 전단 조각으로 2~3개 개조를 산다.

권장 흐름:

```text
2~3번째 런
-> 120초 이벤트 성공
-> 송출 기록 1 확보
-> 흔적 1개 회수
-> 보급소에서 보관/소모/캠페인 활용 중 선택
-> 출격 게시판 목표 갱신
```

성공 감정:

```text
방금 중요한 걸 찾았고, 보급소가 그걸 알아봤다.
```

### First 60 Minutes

목표:

- 송출 기록 3개를 확보하고 보급소에서 해석한다.
- 스마일 홈 시어머니 조우 조건을 연다.
- 보스에게 의미 있는 피해를 넣으면 코어 파편을 받는다.
- 보스를 처리하면 외곽 주택가가 끝나는 것이 아니라 달라진다.

권장 흐름:

```text
120초 이벤트 -> 송출 기록 1
180초 이벤트 -> 송출 기록 2
240초 이벤트 -> 송출 기록 3
보급소 해석 -> 송출관 접근 절차 확정 / 중심 결절 노출
보스 조우 -> HP 65%/25% 성과 또는 처리
보스 선택 -> 결절 파괴 또는 기억 추출
재방문 -> 지역 문구/보상/이벤트 변화
```

성공 감정:

```text
못 깼어도 다음엔 깰 수 있겠다.
깼어도 이 지역이 끝난 건 아니다.
```

### Boss Failure Retry Flow

보스 조우만으로는 코어 파편을 주지 않는다. 대신 HP 임계값을 넘긴 실패는 다음 시도에 도움을 준다.

```text
보스 HP 65% 이하
-> 코어 파편 +1
-> 보스 대응 개조 1개 열림
-> 세븐/도윤의 패턴 분석 문구
-> 다음 조우에서 읽을 수 있는 피드백 증가
```

핵심은 "보스에게 졌다"가 아니라 "보스 내부 구조를 건드렸다"로 읽히게 하는 것이다.

### Post-Boss Revisit Flow

보스 처리 후 외곽 주택가는 폐쇄되지 않는다.

| 선택 | 다음 재방문 감각 |
| --- | --- |
| 결절 파괴 | 광고 농도 일부 감소, 일부 문패/집이 조용해짐, 안정 보상 증가 |
| 기억 추출 | 가족사진/문패/이름 없는 영수증 관련 미스터리 증가, 송출관 연구 단서 강화 |

보스 처리 후 재방문의 목적:

- 내가 선택한 방식이 지역에 남았다는 감각
- 새 흔적/잔향/게시판 문구 발견
- 상위 송출관 접근 목표의 존재감 강화

## 2. Ad Flyer Fragment Balance

`광고 전단 조각`은 0.2의 반복 성장 재화다. 평범한 골드가 아니라 캠페인 환경에서 떨어져 나온 판촉 물질이며, 보급소가 정화해 장비 절연재, 필터, 출력 보정판으로 쓴다.

### 지급 기준 초안

| 조건 | 지급량 | 비고 |
| --- | ---: | --- |
| 45초 미만 | 0 | 빠른 죽음 파밍 방지 |
| 45~59초 | 0~1 | 첫 전투 예외 외에는 낮게 |
| 60초 생존 | +1 | 첫 의미 있는 생존 |
| 90초 생존 | +1 | 초반 안정 보상 |
| 120초 신호 이벤트 성공 | +1~2 | 시간 도달만으로 지급하지 않음 |
| 180초 신호 이벤트 성공 | +2 | 중반 성과 |
| 240초 신호 이벤트 성공 | +2~3 | 보스 접근 전 성과 |
| 엘리트 처치 | +1 | 60초 이후부터 유효 |
| 처치 수 보너스 | +1씩, 런당 최대 +3 | 60초 이후부터 적용 |
| 흔적 캠페인 활용 | +1~2 또는 고위험 카드 해금 | 위험 반응 동반 |

### 첫 강제 회수 보정

첫 런은 `signal_overload`로 반드시 회수된다. 첫 회수 후 개조 1개는 보장해야 하므로 기본 지급을 둔다.

권장:

```text
첫 강제 회수 기본 지급: 광고 전단 조각 3
성과 추가 지급: 0~3
첫 회수 기대 지급: 3~5
```

표현:

```text
첫 현장 데이터 정산
광고 전단 조각 +3
```

### 소비 기준 초안

0.2 초반 개조는 작고 즉각적이어야 한다.

| 개조 | 비용 | 위치 | 목적 |
| --- | ---: | --- | --- |
| 스캐너 출력 보정 I | 전단 2 | 정비대 | 첫 개조 후보, 자동공격 체감 |
| 반품 도장 압력 보정 I | 전단 2 | 차징 조율대 | 첫 개조 후보, 차징 손맛 |
| 회수선 절연 보강 I | 전단 2 | 차징 조율대 | 첫 개조 후보, 초보자 안정 |
| 기록판 보강 | 전단 2 | 출격 게시판 | 다음 신호 이벤트 힌트 |
| 보급소 필터 교체 | 전단 3 | 정산 카운터 | 송출 기록/정산 문구 확장 |
| 이름 보관함 라벨 정리 | 전단 2 | 이름 보관함 | 흔적 보관 반응 강화 |

### 3런 기대 개조 수

| 플레이어 | 1런 후 | 3런 후 | 의도 |
| --- | ---: | ---: | --- |
| 초보 | 1개 | 2개 | 막혀도 성장감 유지 |
| 평균 | 1개 | 3개 | 120초 이벤트 접근 |
| 숙련 | 1~2개 | 4개 | 빠른 신호 진행과 빌드 시도 |

전단 경제의 핵심:

- 1개 개조는 가볍다.
- 3개 개조부터 빌드 방향이 보인다.
- 6개 이상 누적해도 전투가 붕괴하지 않는다.

## 3. Campaign Core Fragment Balance

`캠페인 코어 파편`은 보스/고농도 결절 성과 재화다. 보스 조우가 아니라 내부 구조에 유의미한 피해를 줬을 때만 지급한다.

### 지급 기준 초안

| 조건 | 지급량 | 반복 제한 |
| --- | ---: | --- |
| 보스 조우만 함 | 0 | 없음 |
| HP 65% 이하 첫 도달 | +1 | 보스별 최초 1회 확정 |
| HP 65% 이하 반복 도달 | 낮은 확률 또는 누적 2회당 +1 | 파밍 속도 제한 |
| HP 25% 이하 첫 도달 | +1 | 보스별 최초 1회 확정 |
| HP 25% 이하 반복 도달 | 누적 2회당 +1 | 숙련 반복 보상 |
| 보스 처리 성공 | +2 | 처리 선택당 기본 |
| 동일 보스 반복 처치 | +1 또는 전단/흔적 중심 | 코어 인플레 방지 |

0.2에서는 복잡한 확률보다 `first_clear_flags`와 `repeat_progress_counter`를 권장한다.

### 사용처

| 개조 | 비용 | 효과 | 의도 |
| --- | ---: | --- | --- |
| 결절 파편 차폐 I | 코어 1 | 보스 특정 패턴 피격 후 회복/무적창 소폭 개선 | 보스 재도전 보정 |
| 코어 잔향 분석 I | 코어 1 | 보스 HP 65% 이하 패턴 힌트 표시 | 학습 보조 |
| 외피 균열 기록 I | 코어 1 | 반품 도장이 보스 외피에 주는 피드백/효율 소폭 증가 | 차징 손맛 강화 |
| 송출관 연구 샘플 | 코어 2 | 상위 송출관 연구 문구/후속 목표 해금 | 상위 성장 |

금지:

- 코어 파편으로 일반 DPS를 크게 올리는 것
- 코어 파편을 전단처럼 반복 소비시키는 것
- 보스 실패를 자동 승리 보정으로 바꾸는 것

## 4. Yunseo 0.2 Growth Tree

윤서의 0.2 성장은 3계열만 사용한다.

```text
반품 스캐너 출력
반품 도장 개조
회수선 안정화
```

각 계열은 내부적으로 트리 구조를 가질 수 있지만, 0.2 UI에서는 복잡한 스킬트리처럼 보이지 않는다. 보급소의 정비대/차징 조율대에서 작은 개조를 고르는 느낌이 우선이다.

### 반품 스캐너 출력

역할: 기본 자동공격 안정화. 윤서가 약한 기본캐처럼 느껴지지 않게 한다.

| 개조 | 비용 | 효과 | 체감 |
| --- | ---: | --- | --- |
| 스캐너 출력 보정 I | 전단 2 | 자동공격 피해 +1 또는 안정성 증가 | 초반 잡몹 정리 개선 |
| 스캔선 초점 조정 | 전단 3 | 사거리 소폭 증가 | 이동 중 안정 |
| 가까운 반품 우선 | 전단 3 | 타깃 전환 낭비 감소 | 전투가 덜 답답함 |
| 판촉 외피 긁기 | 전단 4 | 엘리트 외피에 자동공격 누적 보정 소폭 | 보스 전 단계 대비 |

상한:

- 0.2에서 자동공격은 편해지되, 차징/카드 선택을 대체하면 안 된다.

### 반품 도장 개조

역할: 윤서의 손맛 핵심. 모바일 단일 버튼의 만족도를 책임진다.

| 개조 | 비용 | 효과 | 체감 |
| --- | ---: | --- | --- |
| 도장 압력 보정 I | 전단 2 | 차징 피해 +2 | 첫 손맛 강화 |
| 넓은 접수면 | 전단 3 | 전방 부채꼴 범위 소폭 증가 | 빗나감 스트레스 완화 |
| 반품 사유: 과장광고 | 전단 4 | 신호 오브젝트/엘리트 외피 추가 피해 | 목표 대응력 |
| 접수 후 둔화 | 전단 4 | 명중 적 짧은 둔화 | 생존/컨트롤 보상 |
| 외피 균열 기록 | 코어 1 | 보스 외피 명중 피드백/효율 소폭 증가 | 보스전 차징 의미 강화 |

모바일 조작 연결:

- 입력은 단일 탭 유지.
- 개조는 버튼 수를 늘리지 않고 범위, 피해, 둔화, 목표 대응력을 바꾼다.
- 차징 성공 VFX/사운드가 수치보다 먼저 체감되어야 한다.

### 회수선 안정화

역할: 초보자 안전망과 첫 강제 회수의 장비적 개연성.

| 개조 | 비용 | 효과 | 체감 |
| --- | ---: | --- | --- |
| 회수선 절연 보강 I | 전단 2 | 신호 과부하 경고 시간 증가 또는 회수 직전 데이터 보존 | 실패 납득 |
| 릴 장력 조절 | 전단 3 | 치명 피해 후 짧은 무적/경고, 런당 1회 후보 | 초보자 안정 |
| 데이터 묶음 보존 | 전단 4 | 회수 시 일부 목표 진행 정보 보존 | 긴 런 실패 완화 |
| 침묵권 닻 | 전단 4 + 송출 기록 해석 조건 | 신호 이벤트 목표 표시 안정화 | 180/240초 도전 보조 |

주의:

- 회수선 안정화는 사망 보상이 아니다.
- 더 잘 버티고, 성과를 보존하는 개조여야 한다.

## 5. Run Card Selection, Stacking, And Unlocks

0.2 카드 풀은 12장으로 고정한다.

```text
기본 8장
해금 4장
```

### 카드 선택 빈도

권장:

- 첫 런 60초 안에 카드 선택 1회.
- 3~5분 런에서 카드 3~5회.
- 보스 접근 런에서 카드 5~7회.

목표:

- 전투 중 뿌듯함을 자주 준다.
- 너무 자주 멈춰 모바일 템포를 깨지 않는다.

### 희귀도 구조

| 희귀도 | 비율 | 역할 |
| --- | ---: | --- |
| common | 65% | 기본 빌드 형성 |
| uncommon | 25% | 방향성/해금 카드 |
| rare | 10% | 강한 생존/특수 효과 |

초반 0.2에서는 너무 많은 희귀도 단계가 필요 없다.

### 중첩 규칙

| 카드 유형 | 중첩 원칙 |
| --- | --- |
| 피해 증가 | 선형 증가, 상한 필요 |
| 생존 1회 효과 | 횟수는 유지, 회복량/보호량만 증가 |
| 위험 보상 | 보상과 위험이 함께 증가 |
| 투사체 추가 | 1회 추가 후 이후는 피해/확률 증가로 전환 가능 |
| 제어 효과 | 완전 무력화 금지, 범위/감소율만 증가 |

### 윤서 차징 카드 부족 해결

현재 기본 카드 중 명확한 차징 카드는 `달콤한 역송출` 1장뿐이다. 0.2에서 차징 손맛을 살리려면 카드 추가보다 기존 카드의 보조 태그/개조 연동을 활용한다.

권장 조정:

- `상쾌한 반품`: 피격 반사 외에 반품 도장 명중 후 짧은 반사 확률 증가 후보.
- `프리미엄 침묵`: 반품 도장 명중 지점 주변 발사 빈도 감소 연동.
- `유효기간 지난 사랑`: 저체력 시 차징 쿨다운/차징 피해 보조 후보.
- 반품 도장 개조에서 차징 방향성을 보완한다.

0.2에서 새 차징 카드가 꼭 필요하면 P1 후보 1장을 둔다.

```text
카드 후보: 반품 사유 추가
효과: 반품 도장이 적중한 적에게 짧은 취약 효과를 남긴다.
주의: 12장 카드 풀을 넘기면 P1로 미룬다.
```

### 위험 보상 카드

위험 카드는 나쁜 선택이 아니라 강한 유혹이어야 한다.

예:

```text
빛나는 고객님
효과: 경험치 획득량 증가
위험: 적 이동속도 증가
```

설계 기준:

- 유저가 위험을 읽고도 고르고 싶어야 한다.
- 위험은 즉시 불쾌한 벌이 아니라, 다음 전투 압박과 지역 반응으로 돌아와야 한다.
- 내부 `campaign_exploitation`이나 `sync_risk` 수치는 보여주지 않는다.

## 6. Trace Growth Structure

흔적은 재화가 아니다. 회수한 물건을 어떻게 처리하느냐가 전투, 보급소, 지역 반응에 연결된다.

선택:

```text
보관
소모
캠페인 활용
```

0.2에서는 모든 흔적에 세 선택을 전부 구현하지 않아도 된다. 하지만 데이터 구조는 세 선택을 받을 수 있어야 한다.

### 선택별 보상 차이

| 선택 | 전투/기능 보상 | 거점/스토리 보상 | 감정 |
| --- | --- | --- | --- |
| 보관 | 즉시 전투 이득은 작음 | 이름 보관함, NPC 대사, 지역 반응 | "남겼다" |
| 소모 | 장비/시설 개조에 직접 도움 | 해당 흔적은 보관되지 않음 | "살기 위해 썼다" |
| 캠페인 활용 | 강한 보상/위험 카드/정산 효율 | 캠페인이 유저를 더 정확히 부름 | "유혹을 탔다" |

### P0 흔적 적용

| 흔적 | 보관 | 소모 | 캠페인 활용 |
| --- | --- | --- | --- |
| 지워지지 않은 문패 | 이름 보관함 라벨, 일부 문패 변화 둔화 | 회수선/침상 안정 재료 | 0.2에서는 비권장 또는 잠금 |
| 반쯤 탄 가족사진 | 사진 잔향, 복희 대사, 기억 추출 반응 | 1회 보호/방어막 계열 재료 | 기억 변조 위험 후보, 0.3 권장 |
| 이름 없는 영수증 | 빈 고객란 경고, 윤서 서사 씨앗 | 정산 효율/고위험 카드 재료 | XP/전단 보너스, 캠페인 농도 상승 |

보관이 손해처럼 보이지 않게 하는 방법:

- 보관 즉시 NPC/보급소 오브젝트 반응을 준다.
- 보관 흔적이 특정 카드/지역 문구/보스 후 반응을 연다.
- 보관은 전투력 대신 애착과 세계 변화를 준다.

소모가 악한 선택처럼 보이지 않게 하는 방법:

- "살기 위해 쓴다"는 문구를 준다.
- 윤서/도윤이 실용적 선택으로 인정한다.
- 소모 보상이 확실히 체감되어야 한다.

캠페인 활용의 고위험 고보상:

- 보상은 즉각 매력적이어야 한다.
- 다음 런에서 광고 문구가 유저를 더 정확히 부른다.
- 위험 수치는 숨기고 현상으로 보여준다.

## 7. Minimum Outpost Growth

보급소는 메뉴가 아니라 돌아오는 장소다. 0.2에서는 보급소 레벨 UI를 앞세우지 않고, 오브젝트 변화, NPC 반응, 기능 해금으로 성장감을 준다.

### 시설별 최소 구조

| 시설 | 기본 기능 | 성장 방식 | 0.2 체감 |
| --- | --- | --- | --- |
| 정산 카운터 | 런 결과/재화/흔적 표시 | 보급소 필터 교체 | 정산 문구와 송출 기록 해석 확장 |
| 정비대 | 반품 스캐너 개조 | 전단/코어 사용 | 자동공격 안정 |
| 이름 보관함 | 흔적 보관 | 흔적 보관/라벨 정리 | 물건이 장소에 남음 |
| 출격 게시판 | 다음 목표 표시 | 기록판 보강/송출 기록 | 신호 이벤트 힌트 |
| 차징 조율대 | 반품 도장/회수선 개조 | 전단, 배터리 흔적, 코어 | 차징 손맛과 회수 안정 |

### 성장 단계 표현

| 조건 | 변화 |
| --- | --- |
| 첫 개조 완료 | 정비대 조명 하나 켜짐 |
| 송출 기록 1 | 출격 게시판에 찢긴 기록 붙음 |
| 흔적 보관 | 이름 보관함에 라벨/사진 추가 |
| 보스 HP 65% | 코어 파편 유리병 등장 |
| 보스 처리 | 카탈로그 벽에 찢긴 광고 추가 |

금지:

- "보급소 레벨 3" 같은 노골적 성장 UI
- 시설이 거대한 마을처럼 확장되는 0.2 과욕
- 모든 시설을 별도 복잡 UI로 분리

## 8. R01 Suburb Ecology And Growth Link

외곽 주택가는 클리어 후 버리는 맵이 아니다. 0.2에서는 작게 구현하되, 지역이 플레이어 행동을 기억한다는 감각을 반드시 남긴다.

### 대표 상태값

```text
campaign_pressure
boss_outcome
revisit_count
adaptation_charge
discovered_traces
broadcast_records_found
```

### 성장과 지역 상태 연결

| 성장/선택 | 지역 반응 | 보상 변화 |
| --- | --- | --- |
| 송출 기록 확보 | 게시판 목표 갱신, 모델하우스 결절 전조 | 보스 접근 절차 진행 |
| 흔적 보관 | 관련 오브젝트/문구가 조용해짐 또는 선명해짐 | 특정 카드/대사 해금 |
| 흔적 소모 | 보급소 기능 강화, 지역에는 해당 흔적 잔향 약화 | 즉시 전투 성장 |
| 캠페인 활용 | 광고 문구가 유저를 직접 호명 | 전단/XP/위험 카드 보상 |
| 차징 과사용 | 캠페인이 반품 도장을 언급 | 차징 대응형 이벤트 씨앗 |
| 결절 파괴 | 캠페인 농도 일부 감소 | 안정 보상, 전단 정산 안정 |
| 기억 추출 | 미스터리/가족사진/문패 반응 증가 | 기록/흔적/송출관 연구 보상 |

### 반복 방문 보상

| 조건 | 보상/반응 |
| --- | --- |
| 재방문 1회 | "같은 집이 또 있다" 문구 |
| 재방문 3회 | 단골 고객 배너, 외곽 주택가가 윤서를 알아보는 느낌 |
| 송출 기록 2개 | 모델하우스 방향성 강화 |
| 보스 처리 후 | 선택에 따른 게시판 문구와 지역 분위기 변화 |

보상은 수량 변화만으로 만들지 않는다. 문구, 오브젝트, 이벤트, 적 조합의 작은 변화가 함께 있어야 한다.

## 9. Hidden State UI Principle

아래 값들은 내부 설계용이다.

```text
human_stability
robot_autonomy
trace_preservation
campaign_exploitation
sync_risk
local_memory_profile
```

유저에게 직접 보여주지 않는다.

금지 문구:

```text
trace_preservation +10
human_stability +5
캠페인 활용 점수 증가
성향: 인간 우선
```

대신 이렇게 보여준다.

| 내부 변화 | 유저 표현 |
| --- | --- |
| `human_stability` 상승 | 침상 쪽 조명이 따뜻해짐, 미나/복희 대사 |
| `robot_autonomy` 상승 | 충전 스탠드 주변 로봇 대사, 잡음 감소 |
| `trace_preservation` 상승 | 이름 보관함 오브젝트 증가 |
| `campaign_exploitation` 상승 | 카운터 위 원치 않는 쿠폰, 광고 문구가 윤서를 부름 |
| `sync_risk` 상승 | 출격 게시판/차징 조율대에 신호 노이즈 |
| `local_memory_profile` 변화 | 외곽 주택가 배너/오브젝트/적 조합 변화 |

원칙:

- 시스템은 계산한다.
- 세계는 현상으로 반응한다.
- 유저는 해석한다.

```text
유저가 좋아할 것인가?
```

8점 미만 항목:

### 차징 손맛 7점

문제:

- 카드 12장 중 명확한 차징 카드는 `달콤한 역송출` 중심이다.
- 윤서의 얼굴인 반품 도장이 카드 풀에서 충분히 밀리지 않을 수 있다.

개선:

1. 반품 도장 개조를 초반 3개 중 하나로 반드시 노출한다.
2. 차징 명중 VFX, 사운드, 적 광고 문구 파열을 먼저 polish한다.
3. `프리미엄 침묵`, `유효기간 지난 사랑`에 차징 보조 연동을 검토한다.
4. 테스트에서 여전히 약하면 P1 카드 `반품 사유 추가`를 13번째 후보로 준비한다.

## 11. Balance And Implementation Notes

0.2 밸런스 테스트 기준:

- 첫 전투는 숙련자가 오래 버틸 수 있지만 반드시 강제 회수되는가?
- 첫 회수 후 전단 3개 이상으로 개조 1개가 가능한가?
- 3런 안에 평균 2~3개 개조가 가능한가?
- 120/180/240초 송출 기록이 시간 보상이 아니라 이벤트 성과로 느껴지는가?
- 광고 전단 조각이 빠른 죽음 파밍으로 모이지 않는가?
- 보스 조우만으로 코어 파편이 지급되지 않는가?
- 보스 HP 65/25 성과가 못 깬 유저에게도 "다음엔 가능"을 주는가?
- 흔적 보관/소모/캠페인 활용 중 하나만 정답처럼 보이지 않는가?
- 내부 상태가 UI 점수표로 새지 않는가?

0.2에서 절대 넣지 말 것:

- 로봇 플레이어블 실제 구현
- 코어 해방/신호 우회 보스 선택
- 캠페인 송출관 직접 보스전
- 복잡한 호감도
- 정착지 운영
- 다지역 연쇄 시나리오
- 가챠/유료 성장 재화
- 12장을 넘는 카드 풀 확장
- 보급소 시설 전체 3단계 구현

### Implementation Order Recommendation

1. 첫 런과 강제 회수, 첫 정산을 먼저 만든다.
2. 광고 전단 조각 지급/소비와 윤서 첫 개조 3종을 연결한다.
3. 기본 카드 8장으로 3~5분 런 재미를 만든다.
4. P0 흔적 3개와 이름 보관함 반응을 붙인다.
5. 송출 기록 3개와 출격 게시판 목표를 붙인다.
6. 스마일 홈 시어머니, HP 성과, 코어 파편 정산을 붙인다.
7. 보스 선택 후 외곽 주택가 재방문 반응을 붙인다.
8. 플레이테스트 후 차징 손맛, 전단 지급량, 보스 재도전 보정을 조정한다.
