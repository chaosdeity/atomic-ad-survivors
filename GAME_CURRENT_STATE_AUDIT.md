## 1. 현재 게임 상태 요약

현재 프로젝트는 Godot 4 계열의 단일 메인 씬 게임이다. `project.godot`의 main scene은 `res://scenes/main.tscn`이고, 실제 런타임 대부분은 `scripts/main.gd`에서 조립된다.

구현된 플레이 루프:

```text
Main scene 실행
-> 이동/자동공격/차징 전투
-> XP 획득 및 레벨업 카드 3택
-> 첫 출격 강제 회수
-> 침묵 보급소
-> 세션 내 영구 강화
-> 재출격
-> 보스 신호 단계 UI
-> 스마일 홈 시어머니 조우
-> 보스 회수 또는 보스 처치
-> 보급소 복귀
```

구현된 전투 요소:

| 영역 | 현재 상태 |
| --- | --- |
| 플레이어 이동 | 키보드/입력 기반 이동, 카메라 추적 |
| 자동공격 | 사거리 내 가장 가까운 적 또는 보스 자동 타격 |
| 차징 | 8초 주기, 1초 윈도우, 일반/방향집중 판정 |
| 적 | basic, fast, tank, signal, elite 역할 |
| 방어 타입 | normal, anti_auto, anti_charge, plated, exposed_core |
| 웨이브 | 0~300초 시간 구간 기반 웨이브 |
| 첫 회수 | 70초 경고, 88초 붕괴, 108초 회수 |
| 보스 | 스마일 홈 시어머니, 3페이즈, 패턴 4종 |
| 결과 화면 | 게임오버, 생존, 긴급 회수, 보스 회수, 보스 처치 |
| 보급소 | 흔적 표시, 4개 세션 영구 강화, 다음 목표 표시 |
| 디버그 | F1~F12, Shift/Ctrl 조합으로 주요 경로 테스트 |

## 2. 전투와 밸런스 현황

핵심 수치:

| 항목 | 값 |
| --- | ---: |
| 플레이어 HP | 100 |
| 자동공격 1틱 | 15 피해 |
| 자동공격 주기 | 0.5초 |
| 일반 차징 | 20 피해 |
| 방향집중 차징 | 30 피해 |
| 기본몹 HP | 45 |
| 엘리트 HP | 180 |
| 보스 HP | 1750 |
| 매치 시간 | 300초 |

현재 전투의 장점:

- 자동공격, 차징, 방향집중 차징의 역할이 분리되어 있다.
- 적 역할과 방어 타입이 생겨 단순 HP 덩어리만 상대하지 않는다.
- 보스는 자동저항, 차징저항, 코어 노출을 통해 공격 타입 전환을 요구한다.
- 데미지 숫자, 피격 플래시, 보스 상태 HUD가 있어 전투 피드백 기반은 있다.

현재 전투의 리스크:

- 자동공격/차징/카드/보스 피해 흐름이 `main.gd`에 밀집되어 있다.
- 런타임 카드와 보급소 영구 강화가 같은 `player_stats` 적용 경로에 섞인다.
- 보스 조우 전까지의 성과 기반 보상 구조가 아직 코드에 없다.
- 웨이브 진행은 시간표형이고, RPG 성장과 결합할 때 재조정이 필요하다.

## 3. 카드 시스템 현황

`scripts/level_up_cards.gd`에는 17개 카드가 있다.

카드 분류:

- 자동: 광고 문구 강화, 방송 범위 확장, 분열 광고탄
- 차징: 원자력 충전, 군중 설득, 고속 예열 코일, 황금 송출각, 재동기화 오류, 리콜 강제반품
- 광역: 쿠폰 연쇄 폭죽, 잔류 광고잉크
- 생존: 가벼운 안전화, 튼튼한 도시락, 즉석 영양제, 긴급 역송출, 회복성 광고
- 보조: 학습용 전단지 등

선택 로직:

- 매 레벨업마다 3장 선택.
- 최소 1개 체감형 카드가 포함되도록 보정.
- 선택한 빌드 태그가 2개 이상 쌓이면 해당 방향 카드 가중치가 소폭 증가.
- 3장 전부 같은 빌드로 굳지 않도록 억제.

카드 시스템 리스크:

- 카드 데이터가 코드 상수 배열에 박혀 있다.
- 희귀도, 해금 조건, 보급소 업글과의 연계가 없다.
- 카드가 런 중 성장인지 장기 RPG 성장인지 문서/코드 경계가 약하다.

## 4. 보스와 루트 현황

보스: `스마일 홈 시어머니`

캠페인 송출관은 현재 직접 보스가 아니라 송출 기록 3개 이후 보이는 상위 접근 목표/후속 신호로 남긴다.

구현된 패턴:

| 패턴 | 역할 |
| --- | --- |
| 황금 시간대 스윕 | 선 예고 후 가로/세로 빔 |
| 보증서 방패 | anti_auto 상태, 방패 파괴 유도 |
| 신호 왜곡 세일 | anti_charge 상태, 차징 오판 억제 |
| 가정용 안전 시연 | 레일 예고 후 돌진, 회피 후 코어 노출 |

페이즈:

- Phase 1: HP 100~65%, 스윕/방패 중심.
- Phase 2: HP 65~25%, 왜곡/돌진 추가.
- Phase 3: HP 25% 이하, 기존 패턴을 빠른 간격으로 반복.

보스 결과:

- 패배: `신호 과부하 회수`, 보스 분석/코어 파편 지급.
- 처치: `송출 침묵`, 캠페인 코어 파편 지급, 보급소 복귀.

가장 큰 미완성:

- 보스 조우 조건이 아직 `sortie_index`, `boss_signal_unlocked`, 240초 조건에 의존한다.
- 신호추적 UI는 존재하지만 실제 성과 기반 단서 시스템은 코드에 없다.
- 보스 클리어 이후 다음 지역/외곽 신호 시스템은 안내만 있고 구현은 없다.

## 5. 메타 성장 현황

`scripts/meta_progression.gd`는 세션 내 성장 상태를 가진다. 저장/불러오기는 없다.

현재 재화/흔적:

- `torn_ad_flyer`
- `campaign_core_fragment`

현재 강화:

| 강화 | 비용 | 효과 |
| --- | --- | --- |
| 광고 잔상 해독 | 전단 1 | 일반공격 +1 |
| 수동 송출 안정화 | 전단 1 | 차징 +2 |
| 보급소 응급처치 | 전단 1 | 최대 HP +5 |
| 코어 신호 판독 | 코어 파편 1 | 보스 코어 노출 +0.2초 |

성장 구조 리스크:

- 빠른 죽음 파밍 방지용 성과 계산이 없다.
- 런 종료 보상 공식이 없다.
- 신호 단서/송출 기록 진행도가 없다.
- 업글 개수와 비용/해금 구조가 장기 RPG에 부족하다.

## 6. HUD/UI 현황

`scripts/hud_controller.gd`는 코드 UI로 다음을 만든다.

- HP/차징 게이지
- 상단 통계
- 라우트/목표 라벨
- 보스 HP/상태 패널
- 레벨업 카드 패널
- 결과 화면
- 보급소 화면
- 디버그 패널

UI 리스크:

- 모든 UI가 하나의 컨트롤러에 수동 좌표로 들어가 있어 화면 높이 270px에 빡빡하다.
- 보급소 UI가 성장/목표/강화/결과 역할을 동시에 한다.
- 텍스트가 늘어날수록 줄바꿈과 겹침 위험이 높다.
- Web export 한글 폰트 로딩/가독성은 별도 수동 검수가 필요하다.
