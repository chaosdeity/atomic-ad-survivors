## 4. 추가 질문 Q7~Q11 분석

### Q7. 모델 5의 코어 vs 캐주얼 DPS 격차?

**1.76x — 5개 모델 중 최고.**

| 플레이어 | M5 DPS |
|---|---:|
| 캐주얼 | 150 |
| 미드코어 | 219 |
| 코어 | 263 |

- 격차: **263 / 150 = 1.76x**
- 임계값 1.5x **통과** (0.26 여유)
- 5개 모델 중 가장 높은 스킬 천장 (M4 1.63x, M2 1.52x 보다 큼)

**이유 분석**:

- 캐주얼: `charge_use=0.30 × move_acc=0.50 × aim_acc=0.60` = 자주 윈도우를 놓치고, 조준 보너스도 잘 못 얻음 → 차징 효과 미미
- 코어: `0.95 × 0.95 × 0.95` = 거의 매 윈도우 성공 + 거의 매번 조준 보너스(+60% 데미지, +50% AoE)
- 차이 3중 곱셈으로 격차가 확대됨 (M2는 2중 곱셈으로 1.52x)

### Q8. 모델 5의 분당 입력은 M2 대비 몇 배?

**거의 동일 (1.05~1.09x).**

| 비교 | M2 IPM | M5 IPM | 배수 |
|---|---:|---:|---:|
| 캐주얼 | 127 | 128 | 1.01x |
| 미드코어 | 114 | 120 | 1.05x |
| 코어 | 103 | 112 | **1.09x** (+9 IPM) |

**해석**: 코어 기준 분당 +9 입력은 5초마다 약 1회 방향 입력 추가에 해당 (12 windows/min × 0.9 success ≈ 11). 사용자가 예상한 "분당 ~12회 추가"와 일치. M3·M4의 *2배* 증가에 비해 **무시할 수준**.

### Q9. 모델 5의 입력 피크 등급은?

**🟡 보통 (두 손 필요) — 6.9/s.**

- 코어 기준 IPM: **112** (등급 임계값: <60 양호 / 60–120 보통 / ≥120 부담)
- 코어 기준 피크: **6.9/s**
- M2와 거의 동일한 등급(M2 코어: 103 IPM, 6.3/s)
- M3·M4의 🔴 부담(227+ IPM, 10+/s)과 명확히 구분됨

**모바일 적합도**: 평소엔 한 손(이동만) + 5초마다 1초간 두 손(탭+방향) — *interrupt-driven* 입력 모델. 지속 두 손 조작이 아니라 *짧은 펄스*라는 점에서 M3·M4와 본질적으로 다름.

### Q10. 보정된 매개변수에서 보스 처치율이 5~30% 범위인가?

**아니오. 15개 조합 모두 여전히 0%.**

매개변수 보정 효과:
- `ENEMY_RAMP 0.012→0.006` → 생존 시간 v1 대비 평균 +50% 증가
- `BOSS_SPAWN_SEC 480→300` → 보스 등장 시점 180s 당김
- 최강 조합(M4 코어) 생존: v1 244s → v2 **318s**

그런데도 보스 처치율 0% — 왜?

| 모델 | 코어 생존 | 보스 등장 후 시간 | 보스 데미지 가능량 |
|---|---:|---:|---:|
| M4 코어 | 318s | 18s | 258 × 0.6 × 18 = 2,786 (4.6%) |
| M5 코어 | 245s | 0s (도달 못함) | 0 |
| M3 코어 | 302s | 2s | 162 (0.3%) |
| M2 코어 | 244s | 0s (도달 못함) | 0 |
| M1 코어 | 182s | 0s (도달 못함) | 0 |

**원인**: 5개 모델 중 4개가 보스 등장(300s) 전 사망. 도달한 M3·M4도 보스 HP 60,000 대비 5% 미만만 깎음. **보스 메커니즘은 v2에서도 평가 불가**.

**해결 매개변수 조합** (v3 권고):
- `BOSS_HP 60000 → 20000` (모든 조합이 일부라도 보스 진입 시 수치적 영향)
- 또는 `ENEMY_RAMP 0.006 → 0.003` (생존을 400s+ 끌어올림)
- 또는 `BOSS_SPAWN_SEC 300 → 180` (모든 조합이 보스 도달)
- 셋 중 하나만 적용해도 보스 처치율 5~30% 범위 진입 예상

### Q11. 1차 시뮬과 비교했을 때 직관 위배가 해소됐는가?

**부분 해소 — 4개 위배 중 2개만 해소.**

| # | 직관 위배 (v1) | v2 결과 | 해소? |
|---|---|---|:---:|
| 1 | 모든 모델 스킬 천장 1.5x 미만 | M2(1.52), M4(1.63), M5(1.76) 통과 | ✅ **해소** |
| 2 | M3가 M2보다 생존 김 | M2 코어 244s vs M3 코어 302s (M3가 +58s) | ❌ **악화** |
| 3 | M3 미드가 M2 코어보다 생존 김 | M3 미드 275s vs M2 코어 244s | ❌ **여전** |
| 4 | M1 DPS가 모든 프로파일 동일 | 여전히 100/100/100 | ⚠️ 의도된 결과 (해소 불필요) |
| 5 | 보스 처치율 0% (신규 의심) | 0% (Q10) | ❌ **여전** |

**위배 #2 악화 분석**:

v1에서 M2 코어(171s) vs M3 코어(200s) 격차 29s
v2에서 M2 코어(244s) vs M3 코어(302s) 격차 **58s** (악화)

원인은 명확함:
- M2의 처치 속도: `100/30 (베이스) + 15/5 × 0.9 (AoE) = 3.33 + 2.70 = 6.03 마리/s`
- M3 코어의 처치 속도: `100 × 1.336 / 30 = 4.45 마리/s`
- 수치상 M2가 더 빠르게 적을 죽여야 하지만, 시뮬레이션 모델링이 **차징의 군집 처치를 시간적으로 평탄화**시켜버림
- 차징은 5초마다 *몰아서* 15마리 청소 → 윈도우 사이 5초간 적 누적 → 접촉 데미지 누적
- M3는 *지속적*으로 4.45마리/s 처치 → 적 누적 곡선이 더 평탄 → 접촉 데미지 안정적

**이건 모델링의 한계인가, 실제 게임의 진실인가?**

부분적으로 게임의 진실:
- *Vampire Survivors* 실제 플레이에서도 "지속 DPS 무기 > 폭딜 무기"가 군집 통제에 우세한 패턴이 관찰됨
- 폭딜 무기는 보스에 강하지만 잡몹 통제는 약함

부분적으로 모델링 한계:
- 실제 게임에서 광역 폭딜은 *위치 선정*과 결합 → 위험할 때 사용 가능 → 컨디션 기반 가치
- 시뮬은 차징을 *기계적으로 매 윈도우마다 사용* → 위치/타이밍 판단의 가치를 흡수 못함

## 5. 새로운 핵심 발견 — 모델 5의 위치

| 지표 | M2 코어 | M4 코어 | **M5 코어** |
|---|---:|---:|---:|
| 평균 DPS | 210 | 258 | **263** (M4 초과) |
| 스킬 천장 | 1.52x | 1.63x | **1.76x** (최고) |
| 분당 입력 | 103 | 237 | **112** (M2 근접) |
| 입력 피크/s | 6.3 | 10.8 | **6.9** (M2 근접) |
| 모바일 등급 | 🟡 보통 | 🔴 부담 | **🟡 보통** |
| 생존 | 244s | 318s | **245s** |

**M5의 시그니처**: M4 수준의 DPS·스킬 천장을 M2 수준의 입력 부담으로 달성.

**단, M5의 약점**:

- **생존이 M2 수준**(245s)에 머무름 (M4는 318s)
- 평소 자동 발사가 베이스라인(`dps_mult=1.0`)이라 적 누적 통제력이 M4 대비 부족
- 평균 DPS는 차징 시점에 누적된 결과 — 적 누적이 누설되면 결국 사망
- *DPS 높음 ≠ 생존 김* — 모델링 디테일에서 비롯된 직관 위배

**이게 시뮬 결함인가**:

- 시뮬은 정확. 자동 발사가 베이스라인이면 적군 누적 곡선이 더 가파름
- 실제 게임에서도 "강한 폭딜 + 약한 지속 DPS" 빌드는 생존이 어려운 패턴 (VS의 폭탄 빌드 등)
- M5가 *실전*에서 잘 작동하려면 차징 외에도 베이스 DPS 보강 (영구 강화·아이템 시너지) 필요

### M5 검증을 위한 다음 액션

1. **보스 메커니즘 평가** — v3에서 `BOSS_HP=20000` 또는 `BOSS_SPAWN_SEC=180`으로 보스 처치율을 5~30% 범위로 끌어올린 뒤 M5의 보스전 성능 검증

3. **실제 프로토타입 + 플레이테스트**:
   - 차징 윈도우 동안 *방향 입력 UI*가 직관적인가
