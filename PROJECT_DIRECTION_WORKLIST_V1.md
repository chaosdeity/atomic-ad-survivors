# Atomic Ad Survivors Project Direction Worklist V1

## 0. 문서 목적

이 문서는 `PROJECT_DIRECTION_LOCK_V1.md`를 실제 작업으로 쪼갠 목록이다.

목표는 세 가지다.

1. 기존 문서와 새 기준선의 충돌을 정리한다.
2. 0.2 구현 작업을 첫 재미 검증 중심으로 재배열한다.
3. 장기 모바일 F2P/BM 구조는 0.2에 구현하지 않되, 지금부터 설계 충돌을 막는다.

이 문서는 최종 일정표가 아니다. 우선순위와 작업 순서를 잡기 위한 실행 목록이다.

---

## 1. 작업 원칙

### 1.1 기준선

모든 작업은 다음 파일을 기준으로 한다.

- `PROJECT_DIRECTION_LOCK_V1.md`

기준선과 충돌하는 기존 문서는 즉시 폐기하지 않는다. 먼저 충돌 지점을 표시하고, 대체 문서를 만든 뒤 병합한다.

### 1.2 0.2의 목적

0.2의 목적은 BM 검증이 아니다.

0.2의 목적:

```text
윤서 차징 손맛
카드 선택 고민
위험 구역 진입 욕심
밥표/전원표/신호표 후보 생성
행동 기반 정산
보급소 배분
다음 런 변화
재출격 욕구
```

### 1.3 지금 구현하지 않는 것

0.2에서 구현하지 않는다.

- 실제 과금
- 캐릭터 가챠
- 장비 가챠
- 장비 8슬롯 전체
- 캠페인 카탈로그 실제 판매
- 다중 노선 기록 전체 시스템
- 실제 광고/콜라보 수익화

다만 데이터 구조와 문서 방향은 장기 BM과 충돌하지 않게 설계한다.

## 3. Phase B: 0.2 구현 작업 목록

### B0. 현재 구현과 기준선 차이 감사

목표:

- 현재 Godot 구현이 `PROJECT_DIRECTION_LOCK_V1.md`와 얼마나 다른지 확인한다.

확인 대상:

- `scripts/main.gd`
- `scripts/game_config.gd`
- `scripts/wave_director.gd`
- `scripts/enemy_controller.gd`
- `scripts/level_up_cards.gd`
- `scripts/run_result_evaluator.gd`
- `scripts/meta_progression.gd`
- `scripts/r01_layout_blockout.gd`

주요 확인:

- 적 속도/압박이 충분한가
- 카드가 도구/밥벌이/약관으로 나뉘는가
- 밥표/전원표/신호표 후보가 있는가
- 오픈하우스 위험 구역이 있는가
- 정산이 행동 기반인가
- 보급소 배분이 있는가

완료 기준:

- 구현 차이 표가 만들어진다.

우선순위: P0

### B1. 런 행동 로그 추가

목표:

- 정산이 랜덤 보상표가 아니라 행동 결과로 보이게 한다.

필요 로그:

- 생존 시간
- 머문 구역
- 오픈하우스 체류 시간
- 차징 성공 횟수
- 반품 표식 처치 수
- 로봇 적 처리 수
- 낮은 체력 생존 시간
- 약관 카드 사용량
- 검문 도장 상태
- 캠페인 압력 변화

완료 기준:

- 정산 화면에서 최소 3개 이상의 행동 사유를 표시할 수 있다.

우선순위: P0

### B2. 배급권 후보 시스템 구현

목표:

- 밥표/전원표/신호표 후보를 런 중 행동으로 만든다.

최소 조건:

- 90초 이상 생존 또는 특정 조건으로 밥표 후보
- 로봇 적 처리 또는 명령 오류 처리로 전원표 후보
- 오픈하우스 체류 또는 신호 행동으로 신호표 후보

완료 기준:

- 런 중 UI 또는 정산에서 후보 수가 보인다.
- 정산에서 승인/보류가 발생한다.

우선순위: P0

### B3. 정산 결과 개편

목표:

- 실패/귀환 후 결과가 `회수` 한 단어로 끝나지 않게 한다.

필요 결과:

- 귀환 성공
- 긴급 인양
- 일부 인양
- 정산 실패
- 신호 확보
- 미완 귀환

완료 기준:

- 결과 화면이 확정 배급권, 후보 정산, 승인/보류 사유를 표시한다.

우선순위: P0

### B4. 12장 카드 풀 재구성

목표:

- 기존 카드 풀을 도구/밥벌이/약관 12장 구조로 재정렬한다.

카드 분류:

- 도구 카드 4장
- 밥벌이 카드 4장
- 약관 카드 4장

필수 구현:

- 첫 카드 선택에서 3종 분류가 하나씩 보장
- 두 번째 카드에서 시너지 1장 보장
- 약관 카드 대가 표시
- 카드가 정산 후보와 연결

완료 기준:

- 카드 선택이 단순 스탯 상승이 아니라 이번 판 방향을 바꾼다.

우선순위: P0

### B5. 광고 감사/압력 문턱 구현

목표:

- 발라트로식 문턱 압박을 액션으로 번역한다.

구조:

```text
시간 경과
-> 광고 감사 단계 상승
-> 적 밀도/속도/검수 이벤트 증가
-> 카드/차징/정산 엔진이 부족하면 공간 상실
```

주의:

- 모든 적을 무작정 빠르게 만들지 않는다.
- 감사/검수 계열, 위험 구역, 실패 문턱에서 압박을 준다.

완료 기준:

- 플레이어가 다음 카드 전까지 성장 문턱을 넘겨야 한다고 느낀다.

우선순위: P0

### B6. R01 오픈하우스 위험 구역 구현

목표:

- 맵이 좁고 평평하게 느껴지는 문제를 해결한다.

필수 요소:

- 회수선 외곽
- 반복 주택가
- 오픈하우스 거리
- 모델하우스 접근 암시

오픈하우스 규칙:

- 체류 시 신호표 후보
- 적 밀도/압력 증가
- 귀환 거리와 위험 증가

완료 기준:

- 플레이어가 "저 안쪽 위험한데 가보고 싶다"를 느낀다.

우선순위: P0

### B7. 보급소 배분 최소 구현

목표:

- 정산 후 배급권을 어디에 쓸지 결정하게 한다.

최소 선택:

- 밥표 -> 인간 구역
- 전원표 -> 정비대/해방 로봇
- 신호표 -> 출격 게시판/지역 해석

완료 기준:

- 배분 선택이 다음 런 변화 1개 이상으로 이어진다.

우선순위: P0

### B8. 다음 런 변화 구현

목표:

- 반복 출격이 같은 판 반복으로 느껴지지 않게 한다.

최소 변화:

- 인간 구역 배분 시 시작 회복/지원 변화
- 전원표 배분 시 차징/정비 카드 변화
- 신호표 배분 시 오픈하우스 신호 위치 표시
- 캠페인 압력 상승 시 적 밀도/위험 변화

완료 기준:

- 재출격 화면에서 직전 선택의 영향이 보인다.

우선순위: P0

### B9. 0.2 보스 범위 재결정

목표:

- 첫 5분 검증과 첫 1시간 구조에서 보스를 어디까지 구현할지 확정한다.

결정 필요:

- 첫 5분에는 보스 암시만
- 첫 1시간에는 스마일 홈 시어머니 구현
- 보스 결과는 전체 5종이 아니라 2종 또는 3종만

완료 기준:

- 보스가 0.2 검증 범위를 망치지 않으면서 장기 구조를 암시한다.

우선순위: P1

### B10. QA/계측 체크리스트 작성

목표:

- 재미 검증을 감이 아니라 질문과 지표로 본다.

체크:

- 첫 30초 차징 사용률
- 첫 카드 선택 고민 시간
- 오픈하우스 진입률
- 신호표 후보 획득률
- 정산 납득도
- 보급소 배분 선택률
- 재출격률

완료 기준:

- 플레이테스트 후 무엇을 고쳐야 하는지 판단 가능하다.

우선순위: P0

## 4. Phase C: 장기 BM/라이브 설계 작업

### C0. 모바일 F2P 경제 헌법 작성

목표:

- 장기 BM이 게임 철학과 충돌하지 않게 한다.

포함:

- 생존권 판매 금지
- 윤서 기본 지급
- 캐릭터 조각 성장
- 장비 8슬롯
- 장비/기업 세트
- 캠페인 카탈로그
- 선택형 광고와 금지선

완료 기준:

- 0.2 구현 데이터가 장기 BM과 충돌하지 않는다.

우선순위: P1

### C1. 캐릭터 가챠/조각 성장 설계

목표:

- 캐릭터 애정 기반 매출 구조를 설계한다.

필수:

- 완제 드롭
- 조각 드롭
- 중복 전환
- 천장
- 복각
- 윤서 조각 성장
- 윤서 변주 폼 후기 검토

완료 기준:

- 캐릭터 판매가 성능 압박만으로 보이지 않는다.

우선순위: P1

### C2. 장비 8슬롯/세트템 설계

목표:

- RPG다운 장비 성장과 장기 BM 천장을 만든다.

슬롯:

- 출력 코어
- 회수복
- 작업장갑
- 정산벨트
- 기동화
- 식별목걸이
- 약관반지
- 보조장치

세트:

- 스마일 홈
- 오픈하우스
- 해피파워
- 정상고객
- 반품 담당
- 캠페인 활용
- 환경 동조

완료 기준:

- 장비가 캐릭터 고유 차징 무기를 대체하지 않고 강화/변조한다.

우선순위: P1

### C3. 캠페인 카탈로그 설계

목표:

- 일반 배틀패스보다 세계관에 맞는 시즌형 상품을 만든다.

구조:

- 무료 라인
- 유료 라인
- 주기적 복각
- 후발 유저 핵심 보상 접근
- 시즌 한정 칭호/프로필/장식

완료 기준:

- 가챠보다 "사도 괜찮다"는 느낌의 상품 구조가 된다.

우선순위: P1

### C4. 노선 기록/명성 설계

목표:

- 단일 선악 점수가 아닌 다중 노선 평판을 만든다.

노선 후보:

- 인간 구역
- 해방 로봇
- 신호 해석
- 보급소 생존
- 캠페인 활용
- 환경 동조
- 결절 파열
- 기억 보존

완료 기준:

- 모든 노선 1등이 불가능하고, 유저 선택이 다른 결과로 이어진다.

우선순위: P2

### C5. 선택형 광고/광고 제거 설계

목표:

- 모바일 광고 수익을 넣되 세계관을 배신하지 않는다.

가능:

- 선택형 보상 광고
- 광고 제거
- 메타 보상
- 일반 성장 재료 소량
- 카탈로그 포인트 소량
- 복귀 보상 보정

금지:

- 밥표/전원표/신호표 지급
- 보스 부활
- 정산 실패 무효
- 보급소 생존

완료 기준:

- 실제 광고 보상은 생존권 판정과 분리된다.

우선순위: P2
