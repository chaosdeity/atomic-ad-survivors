# First 5 Minute Fun And RPG Loop 0.2

## 0. 문서 목적

이 문서는 `Atomic Ad Survivors 0.2`가 첫 5분 안에 재미있을 수 있는지 검증하기 위한 최소 플레이 루프 설계서다.

이 문서는 전체 게임 완성 문서가 아니다. 이 문서는 `PROJECT_DIRECTION_LOCK_V1.md`의 기준을 첫 플레이 경험으로 압축한다.

검증할 핵심 질문:

> 첫 5분 안에 플레이어가 윤서의 차징 손맛, 카드 선택 고민, 위험 구역 욕심, 배급권 후보 생성, 행동 기반 정산, 보급소 배분, 다음 런 변화를 하나의 루프로 이해하는가?

0.2의 최종 질문:

> 플레이어가 한 판 더 누르는가?

---

## 1. 기준선

### 1.1 장르 기준

0.2 첫 5분은 다음 장르 정체성을 증명해야 한다.

> 출격형 광고 정산 액션 RPG

따라서 첫 5분은 단순 뱀서류 생존장으로 보이면 안 된다.

첫 5분에 보여야 할 것:

- 윤서가 출격하는 이유
- 광고 환경 안에서의 전투
- 반품 도장/차징 손맛
- 카드가 전투와 정산을 바꾸는 감각
- 밥표/전원표/신호표 후보
- 오픈하우스 위험 구역
- 실패/귀환 후 행동 기반 정산
- 보급소 배분
- 다음 런 변화

### 1.2 0.2 첫 5분에서 하지 않는 것

첫 5분 검증에서는 다음을 하지 않는다.

- 실제 과금
- 캐릭터 가챠
- 장비 가챠
- 장비 8슬롯
- 전체 보스전
- 모든 보스 결과 구현
- 다중 노선 기록 전체 시스템
- 자동 원정
- 장기 캠페인 결과 전체 구현

보스는 첫 5분에 직접 넣지 않는다. 스마일 홈 시어머니와 모델하우스 결절은 목표판, 광고 문구, 잠긴 접근로로만 암시한다.

### 1.3 첫 5분은 튜토리얼이 아니다

첫 5분은 조작 설명만 하는 구간이 아니다.

첫 5분은 이 게임의 최소 재미 구조를 보여주는 작은 한 판이어야 한다.

```text
출격 이유
-> 전투 손맛
-> 카드 선택
-> 위험 구역 욕심
-> 배급권 후보
-> 정산
-> 보급소 배분
-> 다음 런 변화
```

## 2. 첫 5분 성공 조건

### 2.1 플레이어가 반드시 경험해야 하는 것

| 경험 | 목표 |
| --- | --- |
| 광고 세계의 이상함 인지 | 100% |
| 반품 도장/차징 사용 | 100% |
| 카드 선택 2회 이상 | 90% 이상 |
| 오픈하우스 위험 구역 인지 | 100% |
| 밥표/전원표/신호표 중 2종 이상 후보 생성 | 80% 이상 |
| 행동 기반 정산 확인 | 100% |
| 보급소 배분 선택 | 100% |
| 다음 런 변화 확인 | 100% |

### 2.2 플레이어 감정 목표

첫 5분 안에 플레이어가 느껴야 할 감정 순서:

1. 여기는 이상하다.
2. 반품 도장/차징이 손에 맞는다.
3. 카드가 단순 스탯 상승이 아니다.
4. 저 안쪽은 위험하지만 가보고 싶다.
5. 밥표/전원표/신호표가 그냥 골드가 아니라 생존권이다.
6. 내가 한 행동 때문에 정산이 달라졌다.
7. 보급소가 하루 더 버티는 느낌이 있다.
8. 다음 판은 다르게 해보고 싶다.

### 2.3 실패 조건

다음 중 하나라도 강하게 나오면 첫 5분 검증은 실패다.

- 그냥 평범한 뱀서류처럼 느껴진다.
- 맵이 평지 생존장처럼 느껴진다.
- 카드가 단순 스탯 상승처럼 느껴진다.
- 밥표/전원표/신호표가 골드처럼 느껴진다.
- 정산이 랜덤 보상표처럼 느껴진다.
- 보급소가 상점처럼 느껴진다.
- 위험 구역에 들어갈 이유가 없다.
- 적 압박이 약해 도망만 치면 된다.
- 실패가 사망/게임오버처럼 읽힌다.

## 3. 전체 흐름

```text
보급소 출격 게시판
-> R01 회수선 외곽 진입
-> 첫 전투와 반품 도장 손맛
-> 첫 카드 선택
-> 반복 주택가 이상 현상
-> 오픈하우스 거리 노출
-> 첫 배급권 후보 생성
-> 광고 감사/압력 문턱
-> 두 번째 카드 선택
-> 더 깊이 들어갈지 귀환할지 고민
-> 실패 또는 귀환
-> 행동 기반 정산
-> 보급소 배급권 배분
-> 다음 런 변화 표시
-> 재출격
```

첫 5분의 핵심은 "버틴 시간"이 아니라 "이번 판이 어떤 정산으로 끝났는가"다.

## 4. 출격 전: 보급소 출격 게시판

### 4.1 목표

플레이어가 왜 나가는지 5초 안에 이해해야 한다.

출격 이유:

> 보급소가 오늘 버티려면 밥표/전원표/신호표 중 일부가 필요하다.

### 4.2 화면 정보

출격 게시판 최소 정보:

```text
오늘의 출격
R01 외곽 주택가

필요 배급권
밥표: 인간 구역 유지
전원표: 정비대 유지
신호표: 모델하우스 신호 해석

권고
오픈하우스 거리에서 신호표 후보가 감지됩니다.
```

### 4.3 주의

첫 출격 전부터 긴 설명을 넣지 않는다.

플레이어가 기억해야 할 단어는 3개다.

```text
밥표
전원표
신호표
```

## 5. 0~30초: 진입과 손맛

### 5.1 R01 진입

목표:

- 광고가 배경이 아니라 시스템이라는 감각을 준다.
- 윤서가 정상 고객이 아니라 검수 대상이라는 인상을 준다.

문구 후보:

```text
R01 외곽 주택가
정상 고객 확인 중...
반품 담당자 감지. 검문 보류.
```

또는:

```text
스마일 홈 방문권이 없습니다.
그래도 입장은 가능합니다. 책임은 고객님께 있습니다.
```

### 5.2 첫 전투

목표:

- 조작보다 먼저 손맛을 준다.
- 반품 도장이 윤서의 정체성임을 보여준다.

초반 적:

- 배송 로봇
- 검문 로봇 소수
- 전단 드론 소수

차징 성공 피드백:

```text
반품 처리!
```

또는:

```text
고객님, 이 상품은 되돌아갑니다.
```

### 5.3 손맛 기준

반품 도장/차징은 다음을 만족해야 한다.

- 맞혔다는 느낌이 즉시 난다.
- 적이 밀리거나 표식이 남는다.
- 차징이 자동공격보다 더 의도적인 행동으로 느껴진다.
- 첫 30초 안에 최소 1회 이상 쓰게 된다.

## 6. 30~45초: 첫 카드 선택

### 6.1 목표

첫 카드 선택은 "공격력 +10%" 같은 단순 강화로 보이면 안 된다.

플레이어는 첫 카드에서 이번 판의 방향을 느껴야 한다.

### 6.2 첫 카드 출현 규칙

첫 카드 선택은 반드시 3종이 하나씩 나온다.

| 카드 종류 | 역할 | 예시 |
| --- | --- | --- |
| 도구 카드 | 전투 손맛 변경 | 반품 라벨 |
| 밥벌이 카드 | 정산/배급권 변경 | 야근 정산표 |
| 약관 카드 | 위험한 대가와 강한 효과 | 정상 고객 스티커 |

### 6.3 첫 카드 예시

#### 반품 라벨

```text
[도구] 반품 라벨
차징에 맞은 적에게 반품 표식을 붙입니다.
표식 적 처치 시 주변 적을 짧게 밀어냅니다.
```

#### 야근 정산표

```text
[밥벌이] 야근 정산표
90초 이후 귀환하면 밥표 후보 +1.
살아 돌아오면 야근입니다. 못 돌아오면 결근입니다.
```

#### 정상 고객 스티커

```text
[약관] 정상 고객 스티커
잠시 일부 검문 로봇의 공격이 지연됩니다.
대가: 만료 후 검수 로봇이 추가 등장합니다.
```

### 6.4 UI 원칙

카드 선택 화면에서 우선 보여줄 정보:

- 카드 종류
- 한 줄 효과
- 조합 힌트
- 대가

상세 효과는 길게 누르기 또는 상세 보기로 보낸다.

## 7. 45~90초: 반복 주택가와 첫 욕심

### 7.1 목표

맵이 평지 생존장이 아니라 지역이라는 느낌을 준다.

플레이어가 "저 안쪽에 뭔가 있다"를 느껴야 한다.

### 7.2 반복 주택가 이벤트

반복 주택가에서 발생할 수 있는 요소:

- 같은 우편함이 반복된다.
- 문패가 상품명으로 덮여 있다.
- 가족사진 광고판이 윤서를 가족 구성원으로 분류하려 한다.
- 오픈하우스 거리 방향 표식이 보인다.

문구 후보:

```text
같은 우편함이 두 번째로 열렸습니다.
```

```text
가족 구성 정보가 비어 있습니다.
현장에서 보정합니다.
```

```text
오픈하우스 방문 확인 시 신호표 후보가 발생합니다.
```

### 7.3 첫 배급권 후보

이 구간에서는 최소 1종의 후보가 생겨야 한다.

예:

- 60초 생존: 밥표 후보 진행도 표시
- 로봇 적 처리: 전원표 후보 진행도 표시
- 오픈하우스 방향 진입: 신호표 후보 조건 표시

UI 예시:

```text
정산 후보
밥표 후보: 감지 중
전원표 후보: 0
신호표 후보: 오픈하우스 체류 필요
```

## 8. 90~150초: 오픈하우스 위험 구역

### 8.1 목표

첫 5분의 핵심 공간은 오픈하우스 거리다.

플레이어가 다음 질문을 해야 한다.

> 지금 들어가면 얻을 수 있다. 그런데 빠져나올 수 있나?

### 8.2 오픈하우스 거리 규칙

오픈하우스 거리 기본 규칙:

| 요소 | 효과 |
| --- | --- |
| 적 밀도 증가 | 생존 압박 증가 |
| 광고음/문구 증가 | 캠페인 압력 체감 |
| 체류 시간 | 신호표 후보 생성 |
| 로봇 적 증가 | 전원표 후보 기회 |
| 낮은 체력 생존 | 밥표 후보 추가 판정 |

구역 진입 문구:

```text
오픈하우스 거리
방문만 해도 사은품. 살아 나가면 추가 사은품.
```

### 8.3 후보 조건 예시

0.2 검증용 임시 조건:

| 조건 | 결과 |
| --- | --- |
| 오픈하우스 거리 25초 체류 | 신호표 후보 +1 |
| 오픈하우스 거리 50초 체류 | 신호표 승인 확률 상승 |
| 로봇 적 15기 처리 | 전원표 후보 +1 |
| 체력 35% 이하 12초 생존 | 밥표 또는 전원표 후보 판정 |
| 반품 표식 적 20기 처치 | 밥표 후보 승인 확률 상승 |

### 8.4 귀환 고민

오픈하우스 구간에서는 귀환 고민이 필요하다.

문구 후보:

```text
신호표 후보가 고정되기까지 20초.
회수선 안정도 낮음.
```

```text
보급소 권고: 지금 돌아오면 오늘 밥은 있습니다.
더 들어가면 내일을 바꿀 수 있습니다.
```

## 9. 광고 감사/압력 문턱

### 9.1 목표

이 게임은 도망만 치면 되는 뱀서류가 되면 안 된다.

광고 감사/압력 문턱은 발라트로식 문턱을 액션으로 번역하는 장치다.

핵심:

> 다음 감사가 오기 전까지 내 카드/차징/정산 엔진이 문턱을 넘을 수 있는가?

### 9.2 첫 5분 감사 단계

0.2 검증용 단계:

| 시간 | 단계 | 효과 |
| --- | --- | --- |
| 0:00~0:30 | 진입 | 낮은 압력, 조작/차징 손맛 |
| 0:30~1:30 | 1차 광고 확인 | 첫 카드 선택, 기본 적 증가 |
| 1:30~2:30 | 1차 감사 | 오픈하우스 위험 구역 유도 |
| 2:30~4:00 | 2차 감사 | 검수 로봇/빠른 적/압력 증가 |
| 4:00~5:00 | 귀환 압박 | 후보 고정 또는 손실 고민 |

### 9.3 압박 방식

모든 적을 무작정 빠르게 만들지 않는다.

압박은 다음 방식으로 준다.

- 검수 로봇 등장
- 빠른 적 비율 증가
- 오픈하우스 거리 내 적 밀도 증가
- 캠페인 압력 상승 시 후보 보류 위험 증가
- 회수선 안정도 하락
- 특정 시간 이후 안전한 외곽으로 돌아가는 길에 적 배치

### 9.4 실패 원인 표시

압박을 넘지 못했을 때 플레이어가 이유를 이해해야 한다.

나쁜 표현:

```text
실패
```

좋은 표현:

```text
정산 실패
사유: 검수 로봇 압박을 돌파하지 못했습니다.
보류: 신호표 후보 1
```

## 10. 두 번째 카드 선택

### 10.1 목표

두 번째 카드에서는 첫 카드와 이어지는 느낌이 있어야 한다.

### 10.2 출현 규칙

두 번째 카드 선택:

- 첫 선택 카드와 시너지 있는 카드 1장 보장
- 다른 정산 방향 카드 1장
- 위험한 약관 카드 1장

### 10.3 예시

첫 카드가 `반품 라벨`이면:

```text
[도구] 전단 폭죽
반품 표식 적이 죽을 때 전단 폭발을 일으킵니다.
```

첫 카드가 `야근 정산표`이면:

```text
[밥벌이] 저체력 근무수당
체력이 낮은 상태로 버티면 추가 배급권 후보가 생깁니다.
```

첫 카드가 `정상 고객 스티커`이면:

```text
[약관] 약관 자동 갱신
약관 효과 시간이 늘어납니다.
대가: 캠페인 압력 +1.
```

## 11. 종료/귀환/실패

### 11.1 목표

런 종료는 단순 사망이나 게임오버가 아니다.

윤서는 죽지 않는다.

결과는 이번 판 행동의 정산으로 읽혀야 한다.

### 11.2 결과명 후보

| 상황 | 결과명 |
| --- | --- |
| 안정적으로 종료 | 귀환 성공 |
| 낮은 체력으로 종료 | 긴급 인양 |
| 체력 0 | 일부 인양 |
| 신호표 후보 확보 후 종료 | 신호 확보 |
| 너무 깊이 들어갔다 실패 | 검수 실패 |
| 후보는 많지만 승인 실패 | 정산 실패 |

### 11.3 결과 처리 원칙

결과는 다음을 보여준다.

- 확정 배급권
- 후보 배급권
- 승인/보류/손실
- 캠페인 압력 변화
- 행동 사유

## 12. 정산 화면

### 12.1 정산 화면 목표

정산 화면은 보상표가 아니다.

정산 화면은 "내가 한 행동 때문에 이렇게 됐다"를 보여주는 화면이다.

### 12.2 정산 화면 예시

```text
귀환 결과
긴급 인양

확정 배급권
밥표 1장

후보 정산
신호표 후보 1장 승인
전원표 후보 1장 보류

위험
캠페인 압력 +1

정산 사유
오픈하우스 거리에서 33초 체류
차징 성공 7회
검문 로봇 처리 2기
약관 카드 1장 사용
```

### 12.3 정산 문구 톤

문구는 광고 시스템을 조롱해야 한다. 인간이나 로봇을 조롱하지 않는다.

예:

```text
축하합니다. 오늘도 굶지 않을 자격을 얻으셨습니다.
```

```text
신호표가 승인되었습니다. 광고 시스템은 아직 이 실수를 모릅니다.
```

```text
전원표 후보는 보류되었습니다. 로봇 고객님의 정상성이 부족합니다.
```

## 13. 보급소 배분

### 13.1 목표

보급소는 상점이 아니다.

보급소는 살아 돌아와서 가져온 것을 나누는 장소다.

### 13.2 첫 5분 배분 선택

첫 5분 후 정산에서는 너무 많은 선택지를 주지 않는다.

최소 선택:

| 배분 | 효과 | 다음 런 변화 |
| --- | --- | --- |
| 밥표 -> 인간 구역 | 인간 안정도 +1 | 시작 회복/지원 후보 |
| 전원표 -> 정비대 | 정비대 기능 +1 | 차징/도구 카드 출현 보정 |
| 신호표 -> 출격 게시판 | 모델하우스 노출도 +1 | 오픈하우스 신호 위치 표시 |

### 13.3 보급소 문구 예시

밥표 배분:

```text
인간 구역의 조명이 조금 늦게 꺼집니다.
```

전원표 배분:

```text
정비대의 한쪽 팔이 다시 움직입니다.
```

신호표 배분:

```text
출격 게시판에 새 핀이 꽂혔습니다.
```

## 14. 다음 런 변화

### 14.1 목표

첫 5분 프로토타입에서 다음 런은 완전히 새로울 필요 없다.

하지만 플레이어가 다음을 느껴야 한다.

> 방금 선택 때문에 다음 출격이 조금 달라졌다.

### 14.2 변화 예시

| 이전 결과 | 다음 런 변화 |
| --- | --- |
| 밥표를 인간 구역에 배분 | 시작 체력/회복 지원 후보 증가 |
| 전원표를 정비대에 배분 | 차징 관련 카드 또는 도구 카드 출현률 증가 |
| 신호표를 게시판에 고정 | 오픈하우스 거리 신호 위치 표시 |
| 캠페인 압력 상승 | 적 밀도 증가, 대신 신호표 후보 증가 |
| 약관 카드 많이 사용 | 검수 로봇 등장 가능성 증가 |

### 14.3 재출격 버튼 예시

나쁜 문구:

```text
재출격
```

좋은 문구:

```text
재출격
목표: 오픈하우스 거리에서 신호표 후보 고정
변화: 정비대 전원 복구로 도구 카드 출현률 증가
위험: 캠페인 압력 +1
```

## 15. 첫 5분에서 보스 처리

### 15.1 보스는 직접 넣지 않는다

첫 5분에는 보스를 직접 넣지 않는다.

이유:

- 차징/카드/위험 구역/정산/배분을 먼저 검증해야 한다.
- 보스를 넣으면 검증 범위가 커져 실패 원인을 알기 어렵다.

### 15.2 보스 암시

스마일 홈 시어머니와 모델하우스 결절은 다음 방식으로 암시한다.

- 모델하우스 방향 표지
- 가족 적합성 문구
- 오픈하우스 배너
- 출격 게시판의 잠긴 목표
- 신호표 요구 문구

문구 예시:

```text
모델하우스 결절
신호표 3장 필요
```

또는:

```text
가족 적합성 심사 대기 중.
방문권이 부족합니다.
```

## 19. 기존 첫 10분 구조와의 관계

기존 문서의 첫 10분 강제 회수 구조는 폐기하지 않는다.

다만 새 기준에서는 다음처럼 재배치한다.

```text
첫 5분:
차징/카드/위험 구역/배급권/정산/배분 검증

첫 10분:
강제 인양 또는 첫 큰 정산 경험

첫 30~60분:
스마일 홈 시어머니 접근, 보급소 성장, R01 지역 결과
```

즉 첫 5분은 전체 0.2의 앞부분이다.

첫 5분이 재미없으면 첫 보스와 장기 RPG는 붙여도 힘을 받지 못한다.

## 20. 최종 요약

첫 5분 재미 증명의 핵심은 다음이다.

> 플레이어는 R01에 들어가 광고 로봇과 싸운다.
> 반품 도장으로 위기를 뚫는다.
> 카드 선택으로 이번 판 방향을 만든다.
> 오픈하우스 거리라는 위험 구역에 들어갈지 고민한다.
> 밥표/전원표/신호표 후보를 만든다.
> 실패하거나 귀환한다.
> 정산에서 행동 기반 결과를 본다.
> 보급소에서 누가 하루 더 버틸지 배분한다.
> 다음 런이 조금 달라진다.
> 그래서 한 판 더 누른다.

이 루프가 재미있으면 `Atomic Ad Survivors 0.2`는 계속 밀 가치가 있다.

이 루프가 재미없으면 보스, 장기 스토리, 장비, BM을 붙여도 해결되지 않는다.

## 21. 보급소 배분 반응 1차 구현 기록

첫 5분 루프의 마지막 감정인 “보급소가 하루 더 버틴다”를 강화하기 위해 배분 반응을 구현했다.

반영:

- 밥표를 인간 구역에 주면 인간 구역/미나 계열 반응이 표시된다.
- 전원표를 정비대에 주면 정비대/팝시 계열 반응이 표시된다.
- 신호표를 출격 게시판에 주면 세븐/게시판 반응이 표시된다.
- 선택지에는 반응 예고가 붙고, 적용 후에는 즉시 반응이 한 줄 표시된다.

예:

```text
적용 완료: 전원표 -> 정비대 - 정비대의 한쪽 팔이 낮은 소리로 돌아갑니다.
```

이 구현의 목적은 배분을 강화 구매가 아니라 생존자와 시설에 대한 선택으로 느끼게 하는 것이다.

남은 과제:

1. 실제 NPC 대사 로그로 확장할지 결정한다.
2. 신호표 배분을 지도/오픈하우스 방향 표시와 더 직접 연결한다.
3. 배분 반응을 플레이테스트에서 5초 안에 읽히는지 확인한다.

---

## 22. 신호표 배분 지도 반영 1차 구현 기록

첫 5분 루프에서 “다음 런이 조금 달라졌다”를 더 강하게 보여주기 위해 신호표 배분을 월드 표시와 연결했다.

반영:

- 신호표를 출격 게시판에 배분하면 보급소 게시판 문구가 바뀐다.
- 다음 출격 시작 시 신호표 게시판 안내가 뜬다.
- 런 중 플레이어 주변에 오픈하우스/모델하우스 방향 나침반이 표시된다.
- 월드 지도에는 신호표 후보 핀과 경로선이 표시된다.
- 신호표 배분이 2회 이상 누적되면 오픈하우스 이후 모델하우스 방향이 더 직접 표시된다.

이 구현의 목적은 정답 경로를 강제로 찍어주는 것이 아니다.

목적은 다음 감정이다.

> 신호표를 게시판에 꽂았더니, 다음 출격에서 위험 구역의 방향이 실제로 보인다.

검증할 질문:

1. 신호표 배분 후 플레이어가 오픈하우스 방향을 더 빨리 이해하는가?
2. 방향 표시가 탐험감을 죽이지 않고 욕심을 자극하는가?
3. 신호표 배분이 단순 진행 재화가 아니라 “지도 해석권”처럼 느껴지는가?

## 23. 지역 약관 예고 1차 구현 기록

첫 5분 루프에서 위험 구역 욕심이 제대로 작동하려면, 플레이어가 위험을 모르고 들어가는 것이 아니라 “저 조건을 채우면 보상이 생기지만 뒤끝이 있다”를 알아야 한다.

반영:

- 보급소 출격 게시판에 대표 지역 약관을 표시한다.
- 보급소 하단 힌트에 이번 출격 약관 예고를 표시한다.
- 재출격 시작 시 지역 약관 요약 공지가 뜬다.
- 오픈하우스/배수로/끊긴 산책로/모델하우스 구역 진입 시 약관 배너가 뜬다.
- 런 중 정산 HUD에 현재 구역의 짧은 약관 태그가 붙는다.

예:

```text
지역 약관: 오픈하우스 25초=신호표 / 산책로 20초=밥표+오염
```

이 구현의 목적은 플레이어가 정산 화면에서 처음 규칙을 배우는 것이 아니라, 출격 전부터 선택의 대가를 알고 들어가게 하는 것이다.

검증할 질문:

1. 플레이어가 오픈하우스 거리 진입 전에 보상 조건을 이해하는가?
2. 약관 예고가 너무 길어서 전투 흐름을 방해하지 않는가?
3. 정산 결과가 “예고된 약관의 결과”로 납득되는가?

## 24. 오염 결과 다음 런 반영 1차 구현 기록

첫 5분 루프에서 “다음 런이 조금 달라졌다”는 감각을 강화하기 위해 오염 정산 결과를 R01 상태에 누적하고, 다음 출격의 문구/약관/적 압력/맵 흔적으로 연결했다.

반영:

- 오염 밥표/전원표/신호표 누적값을 R01 지역 상태에 저장한다.
- 정산 후 “지역 오염 기록”과 다음 출격 변화 문구가 표시된다.
- 오염 종류에 따라 보급소 출격 게시판 목표 문구가 바뀐다.
- 오염 종류에 따라 런 시작 약관과 현재 구역 HUD 약관 태그가 바뀐다.
- 다음 런의 적 스폰 압력, 이동 속도, 엘리트 판정, 적 역할 가중치가 소폭 변한다.
- 맵 위에 오염 밥표/전원/신호 표식이 해당 구역 앵커에 표시된다.

예:

```text
지역 오염 기록: 오염 밥표 1 / 오염 신호표 1
다음 출격 변화: 오픈하우스/모델하우스 신호 감사가 더 민감해집니다.
```

이 구현의 목적은 오염을 벌점으로만 만드는 것이 아니다.

목적은 다음 감정이다.

> 정산에서 만든 결과가 다음 출격의 지역 성격을 바꿨다.

검증할 질문:

1. 플레이어가 오염 결과를 “내가 만든 다음 런 변화”로 받아들이는가?
2. 오염 표식이 위험 구역 욕심을 더 만들거나, 최소한 회피 판단을 만들고 있는가?
3. 오염 누적 압력이 첫 5분 플레이를 과하게 벌하지 않는가?

## 25. 보급소 NPC 로그 1차 구현 기록

첫 5분 루프에서 보급소가 상점처럼 보이는 문제를 줄이기 위해, 정산/오염/배분/강화/신호/보스 결과를 NPC 한 줄 대사와 보급소 이벤트 로그로 남기기 시작했다.

반영:

- 미나, 도윤, 팝시, 세븐, 복희를 보급소 미니맵 표식으로 표시한다.
- 최근 반응 NPC는 보급소 미니맵에서 더 밝게 표시된다.
- 정산 결과가 있으면 미나가 정산 로그를 남긴다.
- 오염 결과가 있으면 미나/팝시/세븐 중 관련 NPC가 반응한다.
- 신호 단서 획득/중복은 세븐 로그로 남는다.
- 첫 회수, 보스 분석, 보스 처리 결과도 NPC 로그로 남는다.
- 보급소 UI에는 `보급소 로그:` 라인으로 최근 반응 1~2개가 보인다.

예:

```text
보급소 로그: 미나: 정산표 접었어. 밥표1은 보급소 이름으로 보관한다.
```

목적은 장문 대화창을 여는 것이 아니다.

목적은 다음 감정이다.

> 내가 가져온 것과 배분한 것이, 보급소 사람들의 말과 위치에 남았다.

검증할 질문:

1. 보급소가 단순 강화 메뉴가 아니라 돌아온 장소처럼 보이는가?
2. NPC 로그가 짧고 조건에 맞게 느껴지는가?
3. 로그가 UI를 가리지 않고도 감정적 보상을 주는가?

## 26. 플레이테스트 계측 1차 구현 기록

첫 5분 루프가 “느낌상 괜찮다”에서 끝나지 않도록, 런 결과와 디버그 HUD에 핵심 체감 지표를 남기기 시작했다.

반영:

- 차징 사용 횟수, 명중 횟수, 빗나감, 다중 명중, 완벽 차징을 기록한다.
- 첫 차징, 첫 카드 선택, 첫 위험 구역, 첫 배급권 후보, 첫 감사 통과, 첫 저체력 시간을 기록한다.
- 오픈하우스 체류 시간, 배급권 후보 최대치, 광고 감사 최대 진행률을 기록한다.
- 런 결과 정산에 `플레이테스트 계측:` 한 줄 요약을 표시한다.
- F12 디버그 HUD에 실시간 플레이테스트 요약을 표시한다.

요약 예:

```text
플레이테스트 계측: 핵심 5/7 | 차징 67%(4/6) | 카드 38s | 후보 91s | 위험 27s
```

이 계측은 유저에게 점수를 보여주기 위한 기능이 아니다.

목적은 다음 판단을 빠르게 하기 위함이다.

> 재미가 부족한 이유가 차징 미사용인지, 카드 도달 지연인지, 위험 구역 유도 실패인지, 정산 후보 체감 실패인지 분리해서 본다.

검증할 질문:

1. 첫 5분 안에 핵심 지표가 최소 4/7 이상 나오는가?
2. 차징 명중률이 낮다면 조작/타이밍/적 속도 중 무엇이 문제인가?
3. 카드 2회 선택 전까지 몹 압박이 충분히 상승하는가?
4. 배급권 후보가 너무 늦거나 너무 자동으로 생기지 않는가?
5. 오픈하우스 체류 시간이 실제 위험 욕심을 보여주는가?

자동 확인:

- `tools/playtest_metrics_probe.gd`를 추가했다.
- `tools/qa_check.ps1`에서 Godot headless로 계측 프로브를 실행한다.
- 프로브는 스크립트 런으로 `핵심 4/7 이상`, 결과 화면 계측 라인, 차징/카드/후보 기록이 모두 연결되는지 확인한다.

주의:

이 프로브는 사람 손맛 검증이 아니다.
계측 파이프라인이 끊기지 않았는지 확인하는 안전장치다.

## 27. 정산/보급소 모바일 UI 압축 1차 구현 기록

첫 5분 루프가 정산과 보급소까지 이어지면서, 결과 문구와 보급소 로그가 길어져 480x270 모바일 기준 화면을 가릴 위험이 생겼다.

반영:

- 정산 화면의 기본 통계를 2줄로 압축했다.
- 정산 진행 기록은 우선순위가 높은 6줄 이하로 줄이고, 나머지는 `정산 기록 N개 더 있음`으로 접는다.
- 보급소 게시판/목표/약관/반응 문구를 짧게 잘라 표시한다.
- 보급소 로그는 최근 핵심 반응 1개를 한 줄로 보여준다.
- 보급소 배분/정비 버튼은 한 줄 compact 문구로 표시한다.
- `tools/ui_compaction_probe.gd`를 추가해 긴 문구가 들어와도 정산/보급소 UI가 기준선을 넘지 않는지 확인한다.
- `tools/qa_check.ps1`에서 UI 압축 프로브를 자동 실행한다.

자동 확인 기준:

- 정산 패널 12줄 이하.
- 보급소 요약 9줄 이하.
- 보급소 로그, 힌트, 액션 버튼 1줄 유지.
- 보급소 액션 버튼 최대 86자 유지.

주의:

이 작업은 문구를 없애는 것이 아니라, 첫 화면에서는 핵심만 보여주고 감정/세부 사유는 다음 런 변화와 보급소 반응으로 분산하기 위한 것이다.

## 28. 오염 표식 전투 가독성 1차 구현 기록

오염 정산 결과가 다음 런의 R01에 남기 시작하면서, 오염 표식이 전투 위험 장판이나 차징 표식처럼 보일 위험이 생겼다.

반영:

- 오염 표식 색을 빨강/노랑 계열 활성 위험색에서 낮은 채도의 지역 흔적 색으로 낮췄다.
- 오염 표식 반경과 선 두께를 줄였다.
- 회전 점과 라벨 알파를 낮춰 전투 VFX보다 아래 계층으로 보이게 했다.
- 보스전 중에는 오염 표식을 숨긴다.
- 적이 많이 몰린 위치에서는 오염 라벨을 숨긴다.
- 가까이 접근했을 때만 오염 라벨을 제한적으로 보여준다.
- `tools/contamination_readability_probe.gd`를 추가해 오염 표식 시각 강도 기준을 자동 확인한다.
- `tools/qa_check.ps1`에서 오염 표식 가독성 프로브를 자동 실행한다.

자동 확인 기준:

- 오염 표식 최대 4개.
- 최대 반경 28.5 이하.
- 채움 알파 0.08 이하.
- 외곽선 알파 0.34 이하.
- 일반 전투 라벨 1개 이하.
- 적 밀집 전투에서는 라벨 0개.
- 보스전에서는 오염 표식 숨김.

목적:

> 오염은 “다음 런의 지역 상태가 남았다”는 흔적이어야지, 플레이어가 피해야 할 즉시 위험 장판처럼 보여서는 안 된다.
