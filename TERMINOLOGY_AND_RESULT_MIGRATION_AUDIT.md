# Terminology and Result Migration Audit

## 0. 목적

이 문서는 코드나 기존 문서를 수정하지 않는다. 0.2 RPG 기반을 위해 화면/문서/구현에 남아 있는 낡은 표현과 충돌 표현을 정리할 목록만 만든다.

## 1. 기준 용어

### 보스명

| 용도 | 기준 |
|---|---|
| UI 짧은 이름 | 스마일 홈 심사관 |
| 문서/에셋 기준명 | 스마일 홈 가족심사 관리자 |
| 광고식 별칭 | 스마일 홈 시어머니 |
| 상위 결절 | 캠페인 송출관 |

`스마일 홈 시어머니`는 주민 속칭/광고식 별칭으로만 사용한다. `캠페인 송출관`은 0.2 첫 보스가 아니라 상위 잔향/후속 접근 목표다.

### 결과/회수 용어

| 피해야 할 표현 | 권장 표현 |
|---|---|
| 사망 | 긴급 인양, 등록 임계, 전투 불능 |
| 부활 | 회수 후 복귀, 미완 귀환 |
| 죽음 보상 | 정산 근거, 후보 승인, 보류 해제 |
| GAME OVER | 정산 실패, 긴급 인양 |
| SURVIVED | 정상 회수 |
| Victory | 정상 회수, 결절 처리 |
| 보스 처치 | 결절 처리, 심사 중단 |
| 보스 사망 | 결절 침묵, 관리자 무력화 |

### 태그 용어

| 기준명 | 속어 |
|---|---|
| 식량태그 | 밥표, 식권, 저녁딱지 |
| 충전태그 | 전원표, 충전권, 정품전원 |
| 수신태그 | 신호표, 수신권, 안내표 |

속어는 NPC 대사, 낡은 안내판, 지역 풍미로만 사용한다. 시스템 UI와 정산 기준명은 식량태그/충전태그/수신태그다.

## 3. 결과 화면/보급소 문구 위험

| 위험 | 설명 | 우선순위 |
|---|---|---|
| 지급과 후보 혼선 | 결과 화면에서 실제 승인 태그와 후보 태그가 같은 보상처럼 보일 수 있다. | P0 |
| 승리/패배 이분법 | 정상 회수, 긴급 인양, 보스 회수, 결절 처리의 차이가 사라진다. | P0 |
| 죽음 파밍 | 실패가 자동 보상으로 읽히면 빠른 실패가 최적화된다. | P0 |
| 보급소 상점화 | 정산 카운터와 정비대가 구매/업글 메뉴처럼 보이면 귀환 감정이 사라진다. | P1 |
| 태그 재화화 | 식량태그/충전태그/수신태그가 골드처럼 보이면 세계관 계약이 무너진다. | P0 |
| 보스 정체 혼선 | 시어머니, 심사관, 가족심사 관리자, 송출관이 한 층으로 섞이면 첫 캠페인 목표가 흐려진다. | P0 |

## 4. 마이그레이션 순서

1. 결과 화면의 `GAME OVER`, `Victory`, `boss_victory` 표시명부터 정리한다.
2. 보스명 기준을 UI/문서/별칭/상위 결절로 분리한다.
3. `밥표/전원표/신호표`를 시스템 기준명에서 내리고, 식량태그/충전태그/수신태그를 기준명으로 올린다.
4. 보급소 문구에서 상점/업글/재화 느낌을 줄이고 정산/배분/귀환 언어로 바꾼다.
5. 문서의 `처치/사망/부활` 표현은 전투 내부 이벤트와 유저-facing 결과를 분리해 정리한다.
