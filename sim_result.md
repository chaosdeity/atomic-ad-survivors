## 2. 6개 핵심 질문 분석

### Q1. M1(순수 자동)의 코어/캐주얼 DPS 격차가 1.5배 미만인가?

**예. 정확히 1.00x.**

| 플레이어 | DPS |
|---|---:|
| 캐주얼 | 100 |
| 미드코어 | 100 |
| 코어 | 100 |

**해석**: 모델 1은 입력이 이동만이고 무기는 100% 자동·자동 조준이므로 *시간당 DPS는 플레이어 유형과 완전히 독립*. 시뮬레이션 코드상 `dps_mult=1.0`이 모든 프로파일에 동일 적용되어 평균 DPS가 정확히 BASE_DPS(100)으로 수렴함. 베이스라인으로서 의도된 결과이며, 스킬 천장은 0(없음).

단, **총 데미지**는 차이가 남: 코어 12,115 > 캐주얼 10,530. 이는 코어가 더 오래 생존(121s vs 105s)하기 때문. 즉 "DPS 격차"가 아니라 "생존 격차"로 코어가 보상받음.

---

### Q2. M2(차징 윈도우)의 입력 피크가 모바일 양호 등급인가?

**아니오. 🟡 보통 등급 (두 손 필요).**

- 코어 기준 입력 피크: **6.0/s**
- 코어 기준 분당 입력: **103 IPM**
- 등급 임계값: <60 IPM 🟢 양호 / 60–120 IPM 🟡 보통 / ≥120 IPM 🔴 부담
- 결과: 103 IPM → 🟡 **보통**

**해석**: 피크 자체(6/s)는 낮으나 분당 입력 100 초과로 양호 등급은 아님. 다만 모델 1(93 IPM)과의 차이는 +10/min에 불과 — 차징 탭 1회 추가뿐이라 입력 부담 증가는 미미. 모바일 한 손 플레이에 가장 근접한 "표현력 추가" 옵션.

---

### Q3. M3·M4(방향 집중)의 분당 입력이 M2 대비 몇 배인가?

**M3: 2.20배, M4: 2.29배 (코어 기준).**

| 비교 | M2 IPM | 대상 IPM | 배수 |
|---|---:|---:|---:|
| M3 / M2 (코어) | 103 | 227 | **2.20x** |
| M4 / M2 (코어) | 103 | 236 | **2.29x** |
| M3 / M2 (미드) | 114 | 224 | 1.96x |
| M4 / M2 (미드) | 114 | 234 | 2.05x |
| M3 / M2 (캐주얼) | 126 | 222 | 1.76x |
| M4 / M2 (캐주얼) | 126 | 228 | 1.81x |

**해석**: 우측 가상 스틱을 잡는 순간 분당 입력이 약 **2배 이상** 폭증. 차징 추가(M2→M4: 103→236)보다 방향 추가(M2→M3: 103→227)의 입력 부담 증가가 훨씬 큼. 모바일 한 손 플레이 즉시 탈락.

---

### Q4. M4가 M2 단독 대비 코어 DPS가 얼마나 증가하는가?

**+49 DPS, +31.2% 증가.**

| 지표 | M2 코어 | M4 코어 | 차이 |
|---|---:|---:|---:|
| 평균 DPS | 157 | 206 | **+49 (+31.2%)** |
| 총 데미지 | 26,961 | 50,336 | +23,375 (+86.7%) |
| 분당 입력 | 103 | 236 | +133 (+129%) |
| 입력 피크/s | 6.0 | 10.5 | +4.5 |
| 생존 | 171s | 244s | +73s |

**의미 있는가?** — **ROI 관점에서는 의미 없음.**

- DPS 증가율: **+31%**
- 입력 부담 증가율: **+129%**
- **입력 부담이 4배 더 빠르게 늘어남.** 코어 플레이어 입장에서도 한 손 → 두 손 + 지속 조준이라는 큰 부담을 +31% DPS로 정당화하기 어려움.
- 단, **총 데미지는 +87% 증가** — 생존 시간 자체가 늘어나 누적 데미지는 크게 벌어짐. "긴 매치의 누적 보상"으로는 가치 있음.

### Q5. 보스 처치율 분포가 합리적인가?

**아니오. 12개 조합 모두 0%. 보스 메커니즘이 평가 불가능한 상태.**

| 모델 | 캐주얼 | 미드코어 | 코어 |
|---|:---:|:---:|:---:|
| M1_자동 | 0% | 0% | 0% |
| M2_차징 | 0% | 0% | 0% |
| M3_방향 | 0% | 0% | 0% |
| M4_차징+방향 | 0% | 0% | 0% |
| **평균** | **0%** | **0%** | **0%** |

**원인**: 보스 등장은 480s인데 **최강 조합(M4 코어)이 244s에서 사망**. 즉 모든 12,000회 매치 중 단 한 번도 보스가 등장하지 못함. 보스 HP(60,000)·보스 데미지 분배(60%) 매개변수가 평가되지 못한 상태.

**무엇이 결정 변수인가**:
- `ENEMY_RAMP=0.012` — 시간당 적 스폰 증가 계수가 너무 강함
- t=200s에서 스폰율 4.4/s vs 자동 처치율 ~3.3/s → 적 누적 가속화 → 접촉 데미지로 사망

이 지표는 **현재 매개변수에서는 무의미**. 보스 메커니즘을 평가하려면 `BOSS_SPAWN_SEC`을 240s로 당기거나 `ENEMY_RAMP`를 0.005~0.008로 낮춰야 함.

### Q6. 의외의 결과 또는 코드 결함이 의심되는 부분

코드 자체의 결함은 발견되지 않았으나, **결과 패턴에서 4가지 직관 위배**가 관찰됨:

#### ⚠️ 1. M3(방향)이 M2(차징)보다 생존이 더 긺

| 모델 | 코어 생존 | 코어 DPS |
|---|---:|---:|
| M2_차징 | 171s | 157 |
| M3_방향 | 200s | 135 |

- **직관**: 차징의 광역 즉살(4마리/윈도우)이 적 누적을 잘 억제할 것
- **결과**: 차징보다 지속 DPS +35%가 적 누적 억제에 더 효과적
- **원인 추정**: `CHARGE_AOE_TARGETS=4`가 너무 작음. Vampire Survivors·Brotato의 광역기는 보통 화면 절반(20~50마리)을 휩쓰는데, 시뮬은 매 5초마다 4마리만 즉살. 적 풀(최대 150) 대비 미미.
- **권고**: AoE 타겟 수를 12~20으로 상향하면 차징 가치가 적절히 반영될 가능성.

#### ⚠️ 2. M3 미드코어 생존(175s)이 M2 코어 생존(171s)보다 김

- 차징 모델 최상위(M2 코어)가 방향 모델 *중급*(M3 미드코어)보다 짧게 산다는 결과
- 1번과 동일한 원인. 차징 효과 과소 모델링의 부산물.

#### ⚠️ 3. 모든 모델이 스킬 천장 1.5x 미만 — 단 한 모델도 통과 못 함

- 최대값 M4가 **1.47x로 임계값에 0.03 부족**
- M4가 통과하지 못한 것은 *간발의 차이*. 매개변수 미세 조정으로 통과 가능.
- 결정 매트릭스로 사용 시: **M4가 사실상 유일한 통과 후보**로 봐야 하며, "전부 탈락"으로 해석하면 안 됨.

#### ⚠️ 4. M1 캐주얼·미드·코어가 모두 정확히 DPS=100

- 코드 일관성: 모델 1은 `dps_mult=1.0` + 차징 없음 → 시간당 데미지가 100으로 고정
- *통계적 우연이 아니라 결정론적 결과* — 실제 게임에서는 적이 적을 때 발사가 빗나가거나 적이 멀리 있을 때 효율이 떨어지는데, 시뮬은 이를 무시
- **수치적으로는 정확**하지만 **현실성은 부족**. 모델 1의 "스킬 천장 0"이라는 결론은 시뮬레이션 단순화의 산물일 수 있음.

#### 결함 판정

코드 버그는 없음. 매개변수 보정 필요한 지점:
1. `CHARGE_AOE_TARGETS: 4 → 12~20` (차징 가치 정상화)
2. `BOSS_SPAWN_SEC: 480 → 240~300` 또는 `ENEMY_RAMP: 0.012 → 0.006` (보스 메커니즘 평가 가능화)
3. M1의 효율을 누락된 측면(조준 빗나감 페널티) 추가 모델링 검토

## 3. 결론 — 1차 결과의 의사결정 가치

### 신호로 받아들일 부분 ✅
- **모델 3(방향 단독)은 비추** — 입력 부담 2배 증가에 비해 스킬 천장 1.14x로 가장 낮음. M4의 일부일 뿐 단독 가치 없음.
- **모델 4가 사실상 유일한 "의미 있는 입력" 후보** — 1.47x로 임계값 근접, 매개변수 보정 시 통과 가능. 다만 모바일 적합도는 🔴.
- **모델 2(차징 단독)는 모바일 친화 + 표현력 추가의 절충점** — 1.31x로 임계값 미달이나 입력 부담 거의 0(+10 IPM). 캐주얼/모바일이 주 타깃이면 합리적 선택.

### 신호로 받아들이지 말 부분 ❌
- 모든 모델 "탈락"이라는 결론 (M4는 0.03 차이로 사실상 통과)
- 보스 처치율 0% (지표가 측정되지 못한 상태일 뿐)
- 차징(M2)이 방향(M3)보다 생존 약하다는 결과 (AoE 모델링 한계)

### 다음 액션 권고
1. **매개변수 보정 후 재시뮬레이션**:
   - `CHARGE_AOE_TARGETS=15`, `ENEMY_RAMP=0.006`, `BOSS_SPAWN_SEC=300`
   - 보스 처치율과 차징 가치가 정상 분포에 들어오는지 확인
2. **모델 4 변종 탐색**:

3. **실제 프로토타입 + 플레이테스트**: 시뮬레이션이 측정 못한 게임필·학습 곡선·터치 피로감을 최종 검증
