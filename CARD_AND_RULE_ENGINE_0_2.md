# Card And Rule Engine 0.2

## 0. 문서 목적

이 문서는 `Atomic Ad Survivors 0.2`의 카드 룰 엔진과 첫 카드 풀을 정의한다.

목표는 카드 수를 늘리는 것이 아니다.

목표는 `발라트로식 재미`를 액션 RPG 구조 안에 맞게 번역하는 것이다.

목표는 첫 5분 안에 플레이어가 다음을 느끼게 하는 것이다.

> 카드는 단순 스탯 상승이 아니라, 점점 올라가는 광고 감사 기준을 넘기 위해 전투 규칙과 정산 규칙을 비트는 선택이다.

이 문서는 `PROJECT_DIRECTION_LOCK_V1.md`, `FIRST_5_MIN_FUN_AND_RPG_LOOP_0_2.md`, `RATION_TICKET_SETTLEMENT_0_2.md`를 기준으로 한다.

## 1. 카드 시스템 핵심 원칙

### 1.1 카드 시스템의 정체성

Atomic Ad Survivors의 카드 시스템은 다음 구조를 목표로 한다.

> 발라트로식 문턱 돌파 재미
> +
> 모바일 액션 RPG의 전투 압박
> +
> 광고 세계의 행동 기반 정산

즉, 플레이어는 단순히 적을 많이 죽이는 것이 아니다.

플레이어는 매 순간 광고 시스템이 요구하는 감사 기준을 넘겨야 한다.

그 기준을 넘기는 방법이 카드, 차징, 장비, 위험 구역, 정산 조작이다.

### 1.2 고정 진화표를 피한다

일반 뱀서류식 카드/스킬 구조는 공격 스킬 + 보조 스킬 조합을 알면 정답 진화표가 된다.

Atomic Ad Survivors는 고정 진화표가 아니라 상황 반응형 카드 구조로 간다.

```text
같은 카드라도
맵 상태
배급권 부족
캠페인 압력
보급소 배분
현재 캐릭터
사용한 차징 무기
위험 구역 체류
에 따라 가치가 달라져야 한다.
```

### 1.3 카드의 역할

카드는 네 가지를 바꾼다.

1. 싸움 방식
2. 정산 방식
3. 위험 대가
4. 광고 감사 기준을 넘기는 방식

카드는 단순 공격력 보너스가 아니라, 이번 판의 규칙을 작게 바꾸는 장치다.

### 1.4 반품 도장은 윤서의 기본 정체성이다

윤서의 반품 도장은 카드로 처음 얻는 스킬이 아니다.

반품 도장은 윤서의 기본 차징 무기이며, 세계를 해석하는 방식이다.

따라서 `반품 라벨` 같은 카드는 반품 도장을 해금하는 카드가 아니라, 반품 도장을 강화/변조하는 카드다.

## 2. 발라트로식 엔진 번역

### 2.1 그대로 베끼지 않는다

발라트로의 재미는 포커 자체가 아니다.

진짜 재미는 다음 구조다.

```text
점점 높아지는 목표값
-> 현재 덱/조합으로 목표값을 넘길 수 있을지 계산
-> 작은 보정이 곱연산으로 커짐
-> 이번 판만의 이상한 규칙이 생김
-> 다음 선택이 완전히 달라짐
```

Atomic Ad Survivors는 이 구조를 액션 RPG로 옮긴다.

```text
점점 높아지는 광고 감사 기준
-> 현재 카드/차징/장비로 압박을 뚫을 수 있을지 판단
-> 차징, 표식, 전단, 송출, 약관이 곱연산으로 커짐
-> 이번 출격만의 광고 약관이 생김
-> 다음 카드 선택과 위험 구역 진입 판단이 달라짐
```

### 2.2 핵심 매핑

| 발라트로 구조 | Atomic Ad Survivors 구조 | 설명 |
| --- | --- | --- |
| 블라인드 목표 점수 | 광고 감사 기준 | 일정 시간/구역마다 넘어야 하는 처리량 문턱 |
| 칩 | 기본 처리값 | 차징 적중, 처치, 표식, 장판, 구역 체류가 만드는 기본값 |
| 멀티 | 처리 승수 | 카드, 약관, 위험 구역, 장비가 곱으로 키우는 값 |
| 조커 | 지속 카드/약관/장비 효과 | 이번 판 규칙을 계속 비트는 핵심 엔진 |
| 보스 블라인드 | 지역 약관/감사 조건 | 이번 출격에서 특정 행동을 방해하거나 다른 가치를 만듦 |
| 상점 | 카드 선택/보급소 카탈로그 | 다음 조합 방향을 고르는 장소 |
| 안티 상승 | 캠페인 압력 단계 | 시간이 지날수록 기준과 적 압박이 같이 증가 |

### 2.3 점수 대신 처리량을 쓴다

플레이어에게 "점수 10,000점을 내라"라고 말하면 이 게임의 세계와 맞지 않는다.

대신 내부적으로는 점수와 같은 역할을 하는 값을 둔다.

공식 명칭 후보:

```text
반품 처리량
광고 처리량
감사 대응값
정산 처리값
```

0.2 내부 구현명은 `처리량`으로 둔다.

플레이어 표시명은 상황에 따라 다르게 보여준다.

예:

```text
광고 감사 대응 중
처리량 820 / 기준 1,000
```

또는

```text
검수 기준 미달
오픈하우스 압력 상승
```

### 2.4 이 구조가 필요한 이유

단순 뱀서류는 플레이어가 도망칠 수 있으면 긴장이 빠진다.

Atomic Ad Survivors는 다음 구조가 필요하다.

```text
현재 카드 조합이 기준 처리량을 넘김
-> 공간이 열린다
-> 위험 구역에 들어갈 욕심이 생김

현재 카드 조합이 기준 처리량을 못 넘김
-> 적 이동속도/밀도/검수 압력이 오른다
-> 도망만으로는 해결되지 않는다
-> 다음 카드 선택이 절실해진다
```

따라서 적은 단순히 많이 나오는 것이 아니라, 감사 기준 미달 시 플레이어의 공간을 실제로 닫아야 한다.

## 3. 광고 감사 문턱 엔진

### 3.1 핵심 정의

광고 감사 문턱은 이 게임의 발라트로식 목표값이다.

일정 시간 또는 구역 진입마다 광고 시스템이 플레이어에게 기준을 요구한다.

```text
감사 구간 시작
-> 기준 처리량 제시
-> 플레이어가 전투/차징/카드로 처리량 생성
-> 기준 달성 또는 미달
-> 보상/압력/정산 후보 변화
```

### 3.2 첫 5분 감사 곡선 초안

아래 수치는 최종 밸런스가 아니다.

목표는 발라트로처럼 기준이 점점 무섭게 올라간다는 감각을 만드는 것이다.

| 시간/구간 | 감사 이름 | 기준 처리량 | 달성 시 | 미달 시 |
| --- | --- | ---: | --- | --- |
| 0:30 | 정상 고객 예비 감사 | 120 | 첫 카드 선택 안정 | 검문 로봇 소폭 증가 |
| 1:00 | 전단 확산 감사 | 260 | 밥표/전원표 후보 판정 | 적 밀도 증가 |
| 1:30 | 오픈하우스 방문 감사 | 520 | 신호표 후보 노출 | 오픈하우스 압력 증가 |
| 2:00 | 가족 적합성 감사 | 1,000 | 두 번째 카드 시너지 강화 | 빠른 검수 로봇 등장 |
| 2:30 | 정산 예비 감사 | 1,800 | 후보 승인률 보정 | 후보 보류율 증가 |
| 3:00 | 모델하우스 신호 감사 | 3,200 | 신호 위치 표시 | 회수선 안정도 감소 |
| 4:00 | 재심사 절차 | 6,000 | 심층 귀환 보정 | 포위형 웨이브 |
| 5:00 | 결절 예비 감사 | 10,000 | 모델하우스 접근 암시 | 강제 인양 위험 |

첫 5분 프로토타입에서 5:00 이후는 완성하지 않아도 된다.

하지만 곡선은 반드시 보여야 한다.

플레이어가 느껴야 할 감각:

> 방금까지는 됐는데, 다음 감사 기준은 지금 조합으로는 못 넘길 수도 있다.

### 3.3 적 압박과 연결

기준 처리량 미달은 숫자 손해로만 끝나면 안 된다.

전투 공간이 바로 바뀌어야 한다.

| 상태 | 적 압박 기준 |
| --- | --- |
| 기준 여유 달성 | 적 이동속도는 플레이어보다 느림. 위험 구역 진입 여유가 생김 |
| 기준 근소 달성 | 적 밀도 증가. 카드 선택 고민이 생김 |
| 기준 미달 1회 | 빠른 검수 로봇 투입. 일부 적이 플레이어 속도 80~90%까지 접근 |
| 기준 미달 2회 | 포위 스폰. 일부 적이 플레이어 속도 95~105%까지 접근 |
| 기준 미달 3회 | 도망만으로 해결 불가. 차징/카드/귀환 판단 필요 |

이 구조 때문에 맵은 넓기만 해서는 안 된다.

넓은 맵 안에서도 광고 감사가 플레이어의 동선을 조여야 한다.

### 3.4 위험 구역의 역할

오픈하우스 거리는 단순 고난도 구역이 아니다.

오픈하우스 거리는 감사 기준을 넘기기 위한 고위험 증폭 구역이다.

```text
안전 구역
처리량은 안정적이지만 성장 속도가 부족함

오픈하우스 거리
적 압박과 캠페인 압력이 증가하지만
신호표 후보와 처리량 증폭 기회가 생김
```

즉, 플레이어는 위험 구역에 들어가야 다음 기준을 넘길 힘을 얻는다.

위험 구역을 피하면 당장은 안전하지만, 다음 감사 기준에서 밀린다.

## 4. 처리량 공식

### 4.1 기본 공식

0.2의 카드 엔진은 다음 공식을 따른다.

```text
처리량 =
(기본 처리값 + 추가 처리값)
* 처리 승수
* 구역 계수
* 약관 계수
- 압력 손실
```

정확한 수치는 바뀔 수 있다.

하지만 구조는 고정한다.

### 4.2 기본 처리값

기본 처리값은 플레이어 행동에서 나온다.

| 행동 | 기본 처리값 역할 |
| --- | --- |
| 차징 적중 | 가장 중요한 순간 처리값 |
| 반품 표식 부여 | 후속 처리량 씨앗 |
| 반품 표식 적 처치 | 처리량 보너스와 조합 트리거 |
| 적 처치 | 기본 처리값 |
| 전단 폭발 | 연쇄 처리값 |
| 송출 잔류 장판 피해 | 구역 장악 처리값 |
| 로봇 명령 오류 | 전원표/제어 처리값 |
| 저체력 생존 | 위험 감수 처리값 |
| 오픈하우스 체류 | 신호/위험 구역 처리값 |

### 4.3 추가 처리값

추가 처리값은 카드가 만드는 플러스 보정이다.

예:

```text
반품 라벨:
차징 적중 시 반품 표식 처리값 +40

전단 폭죽:
전단 폭발 1회당 처리값 +25

오픈하우스 방문 확인:
오픈하우스 체류 5초마다 신호 처리값 +30
```

### 4.4 처리 승수

처리 승수는 발라트로식 재미의 핵심이다.

단순 `+10% 피해`보다 다음 형태가 좋다.

```text
반품 표식 적 처치 처리량 x1.5
오픈하우스 거리 안에서 전단 폭발 처리량 x2
저체력 상태에서 차징 처리량 x2.5
캠페인 압력 3 이상이면 약관 카드 처리량 x1.7
```

플레이어가 느껴야 하는 것은 다음이다.

> 이 카드 하나 때문에 숫자가 갑자기 튄다.

단, 화면에 거대한 점수놀이처럼 보이면 안 된다.

UI는 짧게 보여준다.

```text
반품 처리 x2.0
신호 정산 x1.5
약관 증폭 x1.7
```

### 4.5 정산값과 처리량은 연결되지만 같지 않다

처리량은 현재 감사 기준을 넘기는 값이다.

정산값은 런 종료 후 밥표/전원표/신호표 후보를 승인/보류하는 근거값이다.

같은 행동이 두 값에 다르게 작용할 수 있다.

예:

```text
전단 폭발
처리량: 주변 적을 정리해 감사 기준을 넘기는 데 도움
정산값: 오픈하우스 방문 확인 카드가 있을 때 신호표 승인 근거가 됨
```

카드는 이 둘 중 하나만 바꿀 수도 있고, 둘 다 바꿀 수도 있다.

## 5. 지역 약관과 런 변주

### 5.1 지역 약관은 보스 블라인드 역할이다

각 출격에는 지역 약관이 하나 이상 붙을 수 있다.

지역 약관은 이번 런의 보스 블라인드다.

예:

| 지역 약관 | 효과 | 가치가 올라가는 카드 |
| --- | --- | --- |
| 가족 구성 정보 누락 | 가족 광고 적이 빨라짐. 가족 광고 처리량 증가 | 가족 할인 약관, 송출 잔류 |
| 정상 고객만 통과 | 검문 로봇 공격이 빨라짐. 검문 우회 보상 증가 | 정상 고객 스티커, 로봇 명령 뒤집기 |
| 반품 불가 구역 | 차징 실패 시 압력 증가. 차징 성공 처리량 증가 | 반품 라벨, 반품 불가 동의서 |
| 오픈하우스 사은품 | 위험 구역 보상 증가. 체류 압력 증가 | 오픈하우스 방문 확인, 전단 폭죽 |
| 전력 감사 주간 | 로봇 적 증가. 전원표 정산값 증가 | 배터리 절약 영수증, 로봇 명령 뒤집기 |

같은 카드라도 지역 약관에 따라 가치가 달라져야 한다.

### 5.2 0.2 최소 적용

0.2에서는 지역 약관을 많이 만들 필요 없다.

최소 3개만 있어도 된다.

1. 정상 고객만 통과
2. 오픈하우스 사은품
3. 반품 불가 구역

이 3개만으로도 첫 카드 선택의 가치가 매판 달라진다.

### 5.3 지역 약관은 스토리 문구가 아니다

지역 약관은 실제 규칙이어야 한다.

나쁜 예:

```text
오늘은 오픈하우스 행사가 열립니다.
```

좋은 예:

```text
오픈하우스 사은품
오픈하우스 거리 처리량 x1.6
오픈하우스 체류 중 캠페인 압력 상승 +25%
```

## 6. 카드 선택 엔진

### 6.1 완전 랜덤 선택은 피한다

카드 선택은 무작위처럼 보여도 내부적으로는 판세를 읽어야 한다.

카드 선택 후보는 다음 요소를 본다.

- 현재 보유 카드
- 현재 감사 기준 달성률
- 부족한 배급권 후보
- 현재 지역 약관
- 캠페인 압력
- 플레이어가 많이 쓴 행동
- 위험 구역 체류 여부

### 6.2 카드 제안 슬롯

첫 5분에서는 카드 선택 3장을 다음 구조로 제안한다.

첫 카드:

```text
도구 1장
밥벌이 1장
약관 1장
```

두 번째 카드:

```text
현재 카드와 곱연산 시너지 1장
현재 부족한 감사 기준을 보완하는 카드 1장
위험하지만 큰 승수를 주는 약관 카드 1장
```

세 번째 카드:

```text
현재 정산 후보를 강화하는 카드 1장
다음 감사 기준을 넘기 위한 처리량 카드 1장
판을 뒤집는 약관 카드 1장
```

### 6.3 카드가 반드시 가져야 할 정보

모든 카드는 다음 필드를 가져야 한다.

```text
id
card_type
tags
trigger
base_value_change
multiplier_change
settlement_conversion
risk_cost
synergy_tags
ui_hint
flavor_line
```

단순히 `damage_percent`만 있는 카드는 실패다.

### 6.4 카드 등급

0.2에서는 카드 등급을 복잡하게 넣지 않는다.

대신 카드의 역할을 확실히 나눈다.

```text
일반 카드:
기본 처리값 또는 정산값을 만든다.

증폭 카드:
특정 조건의 처리량을 곱으로 키운다.

룰 변경 카드:
정산 방식, 압력 대가, 후보 변환 규칙을 바꾼다.
```

장기적으로는 희귀도/등급을 추가할 수 있다.

하지만 0.2에서는 `룰 변경 카드가 실제로 작동하는가`가 먼저다.

## 7. 카드 3분류

| 카드 종류 | 역할 | 플레이어 감각 |
| --- | --- | --- |
| 도구 카드 | 전투 행동을 바꿈 | 이번 판 손맛이 바뀐다 |
| 밥벌이 카드 | 배급권/정산을 바꿈 | 뭘 챙겨 올지가 바뀐다 |
| 약관 카드 | 위험한 대가를 받고 강해짐 | 광고와 더러운 계약을 한다 |

### 2.1 도구 카드

도구 카드는 전투 손맛을 바꾼다.

핵심 연결:

- 반품 도장
- 차징
- 표식
- 전단 폭발
- 송출 잔류
- 로봇 명령 오류

### 2.2 밥벌이 카드

밥벌이 카드는 이번 판 정산 방향을 바꾼다.

핵심 연결:

- 밥표 후보
- 전원표 후보
- 신호표 후보
- 위험 구역 체류
- 저체력 생존
- 로봇 처리

### 2.3 약관 카드

약관 카드는 강하지만 대가가 있다.

핵심 연결:

- 캠페인 압력
- 검수 로봇
- 후보 손실
- 정산 보류
- 임시 정상 고객 판정
- 강한 차징/피해 보정

## 8. 카드 설명 형식

카드는 유저 친화적으로 설명한다.

기본 형식:

```text
카드 이름
종류: 도구 / 밥벌이 / 약관

한 줄 효과:
플레이어가 바로 이해해야 하는 효과.

상세 효과:
실제 동작.

켜지는 조합:
다른 카드와 연결될 때 생기는 효과.

대가:
있다면 위험 또는 손실.

짧은 농담:
광고 세계 특유의 아이러니한 문구.
```

카드 선택 화면에서는 전체 상세를 모두 보여주지 않는다.

우선 표시:

- 종류
- 한 줄 효과
- 조합 힌트
- 대가

상세는 포커스/길게 누르기/상세 보기에서 보여준다.

## 9. 카드 출현 규칙

### 9.1 첫 카드 선택

첫 카드 선택은 반드시 3종이 하나씩 나온다.

```text
도구 카드 1장
밥벌이 카드 1장
약관 카드 1장
```

목적:

- 카드 종류를 바로 이해시킨다.
- 전투/정산/위험 대가 중 하나를 고르게 한다.

### 9.2 두 번째 카드 선택

두 번째 카드 선택은 첫 선택과 연결되어야 한다.

```text
첫 선택 카드와 시너지 있는 카드 1장
다른 정산 방향 카드 1장
위험한 약관 카드 1장
```

목적:

- 이번 판 빌드가 생겼다는 느낌을 준다.
- 단순 랜덤 선택이 아니라 이어지는 선택으로 보이게 한다.

### 9.3 세 번째 카드 선택

세 번째 카드 선택은 현재 판세에 반응한다.

```text
현재 정산 후보를 강화하는 카드 1장
전투 생존/돌파 카드 1장
위험한 약관 카드 1장
```

목적:

- 플레이어가 이미 만든 정산 방향을 밀지, 생존을 보정할지, 위험한 계약을 할지 고민하게 한다.

## 10. 첫 5분 카드 풀 12장

0.2 첫 검증용 카드 풀은 12장으로 둔다.

- 도구 카드 4장
- 밥벌이 카드 4장
- 약관 카드 4장

이 카드들은 최종 전체 카드 풀이 아니라 첫 5분 재미 검증용이다.

### 10.1 12장 풀의 역할

12장은 카드 목록이 아니라 엔진 검증 세트다.

이 12장으로 반드시 확인해야 할 것은 다음이다.

1. 처리량을 만드는 카드가 있는가?
2. 처리량을 곱으로 키우는 카드가 있는가?
3. 처리량을 배급권 후보로 바꾸는 카드가 있는가?
4. 위험 대가를 받고 감사 기준을 넘기는 카드가 있는가?
5. 같은 카드가 지역 약관에 따라 다르게 보이는가?

### 10.2 12장 엔진 역할표

| 카드 | 발동 조건 | 처리량 역할 | 정산 역할 | 위험/대가 | 발라트로식 역할 |
| --- | --- | --- | --- | --- | --- |
| 반품 라벨 | 차징 적중 | 표식 기본 처리값 생성 | 밥표/차징 사유 생성 | 없음 | 기본 칩 생성 카드 |

| 송출 잔류 | 차징 사용/장판 처치 | 구역 장악 처리값 | 신호표 승인 근거 | 위치 의존 | 특정 핸드 강화 카드 |
| 로봇 명령 뒤집기 | 로봇에게 차징 적중 | 적 압박을 제어값으로 변환 | 전원표 후보 근거 | 로봇 구간 의존 | 카드 해석 전환 카드 |

| 배터리 절약 영수증 | 로봇 처리 | 로봇 처리값 보정 | 전원표 후보/승인 | 로봇 스폰 의존 | 특정 슈트 보상 카드 |

| 저체력 근무수당 | 낮은 체력 생존 | 위기 처리량 증폭 | 밥표/전원표 후보 | 사망이 아닌 인양 위험 | 고위험 배수 카드 |
| 정상 고객 스티커 | 카드 선택/검문 구간 | 검문 압박 지연 | 신호표 승인 보정 | 만료 후 검수 로봇 | 임시 면제 카드 |
| 가족 할인 약관 | 가족 광고 적 상대 | 가족 적 처리량 증폭 | 신호/기억 사유 | 캠페인 압력 증가 | 조건부 멀티 카드 |

### 10.3 0.2에서 반드시 들어가야 하는 곱연산

0.2 카드가 단순 효과 카드에 머물지 않으려면 최소 4개의 곱연산 축이 필요하다.

```text
차징 축:
반품 라벨 -> 반품 불가 동의서 -> 저체력 근무수당

연쇄 축:
반품 라벨 -> 전단 폭죽 -> 오픈하우스 방문 확인

신호 축:
송출 잔류 -> 오픈하우스 방문 확인 -> 가족 할인 약관

로봇 축:
로봇 명령 뒤집기 -> 배터리 절약 영수증 -> 정상 고객 스티커
```

각 축은 최종 진화 무기가 아니라, 이번 판의 처리량/정산 방식을 바꾸는 경로다.

같은 축도 지역 약관에 따라 강해지거나 약해져야 한다.

### 10.4 첫 5분 최소 승수 목표

첫 5분 안에 플레이어는 최소 한 번 이상 다음 감각을 봐야 한다.

```text
기본 처리값
-> 카드 보정
-> 조건부 승수
-> 감사 기준 돌파
```

예:

```text
차징 적중 80
+ 반품 표식 40
x 반품 불가 동의서 1.8
x 오픈하우스 구역 1.4
= 처리량 302
```

이 숫자를 그대로 크게 띄울 필요는 없다.

하지만 플레이어는 `카드 때문에 기준을 넘겼다`고 느껴야 한다.

## 11. 도구 카드 4장

## 11.1 반품 라벨

기본 정보:

- ID: `tool_return_label`
- 종류: 도구 카드
- 태그: 반품, 차징, 표식
- 추천 등장: 첫 카드 선택 가능

한 줄 효과:

> 차징에 맞은 적에게 반품 표식을 강화해 붙인다.

상세 효과:

- 윤서의 반품 도장에 맞은 적에게 `반품 표식`을 4초간 부여 또는 강화한다.
- 반품 표식이 붙은 적은 받는 차징 피해가 증가한다.
- 반품 표식 적이 죽으면 짧게 밀려나 주변 적에게 충돌 피해를 준다.

켜지는 조합:

- `전단 폭죽` 보유 시: 반품 표식 적 처치 시 전단 폭발 발생.
- `야근 정산표` 보유 시: 반품 표식 적 처치가 밥표 승인 확률에 영향.
- `반품 불가 동의서` 보유 시: 표식 적에게 차징 피해 추가 증가.

대가:

- 없음.

짧은 농담:

> 고객님, 이 상품은 다시 광고판으로 돌아갑니다.

기대 감각:

- 차징이 그냥 강한 공격이 아니라 표식을 남기는 핵심 행동으로 느껴진다.

## 11.2 전단 폭죽

기본 정보:

- ID: `tool_flyer_pop`
- 종류: 도구 카드
- 태그: 전단, 폭발, 연쇄
- 추천 등장: 반품 라벨과 시너지 카드

한 줄 효과:

> 적이 죽을 때 일정 확률로 전단 폭발을 일으킨다.

상세 효과:

- 적 처치 시 일정 확률로 작은 전단 폭발이 발생한다.
- 전단 폭발은 주변 적에게 피해를 주고 짧게 감속시킨다.
- 차징으로 처치한 적은 전단 폭발 확률이 증가한다.

켜지는 조합:

- `반품 라벨` 보유 시: 반품 표식 적은 전단 폭발 확률이 크게 증가한다.
- `오픈하우스 방문 확인` 보유 시: 오픈하우스 거리의 전단 폭발이 신호표 후보 판정에 영향을 준다.

대가:

- 없음.

짧은 농담:

> 홍보물은 폐기되지 않습니다. 더 넓게 배포될 뿐입니다.

기대 감각:

- 처치가 연쇄로 번지는 화면 청소 쾌감이 생긴다.

## 11.3 송출 잔류

기본 정보:

- ID: `tool_broadcast_residue`
- 종류: 도구 카드
- 태그: 송출, 잔류, 구역 장악
- 추천 등장: 첫 카드 또는 두 번째 카드 선택 가능

한 줄 효과:

> 차징이 지나간 자리에 짧은 신호 자국을 남긴다.

상세 효과:

- 차징 사용 시 이동 방향 또는 공격 방향으로 짧은 `송출 잔류` 장판을 남긴다.
- 장판 위의 적은 지속 피해를 받는다.
- 장판 위에서 적이 죽으면 신호표 승인 확률에 영향을 줄 수 있다.

켜지는 조합:

- `오픈하우스 방문 확인` 보유 시: 오픈하우스 거리에서 송출 잔류 처치가 신호표 후보 승인에 영향.
- `가족 할인 약관` 보유 시: 가족 광고 적에게 송출 잔류 피해 증가, 캠페인 압력 위험 증가.

대가:

- 없음.

짧은 농담:

> 방송은 끝났지만, 바닥은 아직 듣고 있습니다.

기대 감각:

- 플레이어가 맵에 자기 흔적을 남기며 위험 구역을 장악하는 느낌이 생긴다.

## 11.4 로봇 명령 뒤집기

기본 정보:

- ID: `tool_robot_command_flip`
- 종류: 도구 카드
- 태그: 로봇, 해방, 명령
- 추천 등장: 두 번째 카드 이후

한 줄 효과:

> 차징으로 맞힌 로봇 적이 잠깐 주변 적을 막는다.

## 15. 룰 변경 카드 방향

12장 카드 풀은 첫 검증용이다.

하지만 첫 검증용이라고 해서 단순 효과 카드만 넣으면 안 된다.

0.2부터 최소한의 룰 변경이 들어가야 한다.

발라트로식 재미의 핵심은 `강한 카드`가 아니라 `규칙이 바뀌는 카드`다.

따라서 12장 중 일부는 반드시 다음 중 하나를 해야 한다.

1. 특정 행동의 처리량 계산 방식을 바꾼다.
2. 특정 행동의 정산 후보 변환 방식을 바꾼다.
3. 위험 구역의 가치를 바꾼다.
4. 캠페인 압력을 대가이자 증폭 자원으로 바꾼다.
5. 실패/인양 시 손실 규칙을 바꾼다.

### 15.1 좋은 룰 변경 예

예:

- 오픈하우스 거리에서는 전단 폭발이 신호표 정산 근거로 계산된다.
- 반품 표식 적 처치가 한 번 더 처리량을 계산한다.
- 명령 오류 로봇은 처치 1기가 아니라 전원표 정산 2기로 계산된다.
- 캠페인 압력이 높을수록 약관 카드 처리량이 증가한다.
- 밥표 후보가 승인될 때마다 다음 오픈하우스 체류 보상이 증가한다.
- 전원표 후보가 보류되면 로봇 명령 오류 지속 시간이 증가한다.
- 신호표 후보가 오염되면 다음 런에서 신호 위치가 더 정확히 표시된다.
- 반품 표식 적 처치가 정산 사유를 복사한다.

### 15.2 0.2 필수 룰 변경 4개

0.2에는 최소 다음 4개를 넣는다.

| 룰 변경 | 관련 카드 | 의미 |
| --- | --- | --- |
| 표식 처치 재계산 | 반품 라벨, 전단 폭죽 | 차징 표식이 처치 연쇄와 처리량을 다시 만든다 |
| 위험 구역 정산 변환 | 오픈하우스 방문 확인, 송출 잔류 | 위험 구역 행동이 신호표 후보로 변환된다 |
| 로봇 처치값 변환 | 로봇 명령 뒤집기, 배터리 절약 영수증 | 명령 오류 로봇이 전원표 정산에서 더 크게 계산된다 |
| 약관 압력 증폭 | 반품 불가 동의서, 약관 자동 갱신 | 캠페인 압력이 위험이자 곱연산 자원이 된다 |

### 15.3 나쁜 룰 변경 예

피해야 할 것:

- 설명 없이 즉사
- 이미 얻은 보상을 무작위 삭제
- 특정 카드 없으면 보스 불가
- 정산을 완전 랜덤으로 만드는 효과
- 너무 긴 텍스트로만 이해되는 효과

## 16. 캐릭터/장비와의 장기 관계

0.2에서는 윤서 1명만 플레이어블이다.

장기적으로는 캐릭터마다 고유 차징 무기가 있고, 같은 공통 카드도 다르게 해석될 수 있다.

예:

```text
반품 라벨
윤서: 반품 도장 표식 강화
다른 캐릭터: 해당 캐릭터의 고유 차징 표식으로 변환
```

장비는 고유 차징 무기를 교체하지 않는다.

장비는 다음을 보정한다.

- 차징 출력
- 카드 해석
- 정산 후보
- 위험 구역 보정
- 캠페인 압력 대가

## 17. 데이터 구조 초안

### 17.1 AuditSegment

광고 감사 구간 데이터.

```text
AuditSegment
- id
- display_name
- start_time_sec
- end_time_sec
- zone_id
- target_processing_value
- success_effects
- fail_effects
- enemy_pressure_profile
- next_segment_id
```

역할:

- 이번 구간의 기준 처리량을 정한다.
- 기준 달성/미달 결과를 정한다.
- 적 압박 프로필을 바꾼다.

### 17.2 RunScoreEvent

런 중 행동 로그.

```text
RunScoreEvent
- event_type
- timestamp_sec
- zone_id
- source_actor
- source_card_id
- target_tags
- action_tags
- base_processing_value
- settlement_tags
```

예:

```text
event_type: charge_hit
action_tags: charge, return_stamp
target_tags: robot, inspector
base_processing_value: 80
settlement_tags: robot, charge_success
```

### 17.3 CardRuntimeRule

카드가 런 중 규칙에 개입하는 방식.

```text
CardRuntimeRule
- card_id
- card_type
- trigger_conditions
- add_processing_rules
- multiplier_rules
- settlement_conversion_rules
- pressure_cost_rules
- ui_feedback_rules
```

카드는 이 구조 중 최소 하나 이상을 가져야 한다.

단순 설명 텍스트만 있는 카드는 카드가 아니다.

### 17.4 SettlementConversionRule

런 행동을 배급권 후보로 바꾸는 규칙.

```text
SettlementConversionRule
- required_event_tags
- required_zone_id
- count_threshold
- ticket_type
- candidate_delta
- approval_weight_delta
- hold_risk_delta
```

예:

```text
오픈하우스 거리에서 송출 잔류 처치 8회
-> 신호표 후보 승인 가중치 증가
```

### 17.5 RegionalClause

지역 약관.

```text
RegionalClause
- id
- display_name
- rule_text
- processing_modifiers
- enemy_modifiers
- settlement_modifiers
- pressure_modifiers
- favored_card_tags
- punished_card_tags
```

지역 약관은 카드와 별개로 이번 런의 판세를 바꾼다.

### 17.6 계산 순서

처리량 계산은 반드시 같은 순서로 실행한다.

```text
1. 행동 이벤트 기록
2. 기본 처리값 계산
3. 카드의 추가 처리값 적용
4. 지역/약관/카드 승수 적용
5. 압력 손실 적용
6. 감사 기준과 비교
7. 성공/미달 결과 적용
8. 정산 후보/사유 로그 저장
```

이 순서가 흔들리면 카드 설명과 실제 결과가 어긋난다.

### 17.7 UI 최소 표시

첫 구현에서 UI는 복잡하면 안 된다.

최소 표시:

```text
감사 기준
820 / 1,000

활성 보정
반품 처리 x1.8
오픈하우스 x1.4

위험
캠페인 압력 +1
```

정산 화면에서는 다음만 보여도 된다.

```text
정산 사유
오픈하우스 체류 32초
차징 성공 9회
반품 표식 처치 14기
송출 잔류 처치 6기
```

## 18. 구현 매핑 초안

현재 구현에 있는 일부 카드/효과는 새 카드로 흡수 가능하다.

| 기존 방향 | 새 카드 후보 |
| --- | --- |
| 반품 도장 표식/차징 강화 | 반품 라벨 |
| 처치 폭발/쿠폰 연쇄 | 전단 폭죽 |
| 장판/잔류 피해 | 송출 잔류 |
| 적 감속/교란 | 로봇 명령 뒤집기 또는 정상 고객 스티커 |
| 저체력 생존 보상 | 저체력 근무수당 |
| 강한 차징 피해 증가 | 반품 불가 동의서 |

주의:

- 기존 카드를 이름만 바꾸지 않는다.
- 새 기준에서는 카드가 정산 후보와 연결되어야 한다.

## 22. 최종 요약

0.2 카드 시스템의 목표는 12장을 많이 보여주는 것이 아니다.

목표는 다음이다.

> 광고 감사 기준은 계속 올라간다.
> 도망만으로는 언젠가 밀린다.
> 도구 카드는 처리량을 만들고, 밥벌이 카드는 처리량을 배급권 후보로 바꾸며, 약관 카드는 위험한 대가로 승수를 준다.
> 지역 약관은 매판 카드 가치를 바꾸고, 카드 조합은 기준 문턱을 넘기 위한 엔진이 된다.

이 구조가 작동하면 카드는 단순 강화표가 아니라 `Atomic Ad Survivors`의 반복 재미를 만드는 핵심 엔진이 된다.

최종 검증 문장:

> 플레이어가 다음 카드 선택 전까지 버티는 것이 아니라, 다음 감사 기준을 넘기기 위해 카드를 고르고 위험 구역에 들어가는가?

## 23. 1차 구현 반영 기록

### 23.1 구현 반영 파일

이번 구현으로 반영된 파일:

- `scripts/level_up_cards.gd`
- `scripts/main.gd`
- `scripts/run_result_evaluator.gd`
- `scripts/debug_tools.gd`
- `tools/balance_sim.py`

### 23.2 카드 풀 교체

기존 런타임 카드 풀은 자동/차징/생존 강화 중심이었다.

이번 구현에서 0.2 기준 카드 12장 구조로 교체했다.

```text
도구 카드 4장:
- 반품 라벨
- 전단 폭죽
- 송출 잔류
- 로봇 명령 뒤집기

밥벌이 카드 4장:
- 야근 정산표
- 배터리 절약 영수증
- 오픈하우스 방문 확인
- 저체력 근무수당

약관 카드 4장:
- 정상 고객 스티커
- 가족 할인 약관
- 반품 불가 동의서
- 약관 자동 갱신
```

첫 카드 선택은 도구/밥벌이/약관이 하나씩 나오도록 보장한다.

### 23.3 광고 감사 기준 구현

런 중 다음 감사 구간이 실제 코드에 들어갔다.

```text
0:30  정상 고객 예비 감사      120
1:00  전단 확산 감사           260
1:30  오픈하우스 방문 감사      520
2:00  가족 적합성 감사        1,000
2:30  정산 예비 감사          1,800
3:00  모델하우스 신호 감사     3,200
4:00  재심사 절차             6,000
5:00  결절 예비 감사          10,000
```

처리량이 기준을 넘으면 감사 통과로 기록된다.

기준 미달 시:

- 감사 압력 단계 증가
- 즉시 압박 엘리트 투입
- 이후 `spawn_pressure`, `speed_mult`, `elite_chance`, fast/speaker/charger/signal 비중 증가

### 23.4 처리량 이벤트 구현

현재 처리량은 다음 행동에서 발생한다.

```text
자동 사격 적중
반품 도장 적중
반품 표식 적 처치
일반 처치
로봇형 적 처치
전단 폭발 피해
송출 잔류 장판 피해
보스/결절 적중
저체력 근무수당 상태 유지
오픈하우스 체류 보정
```

즉, 감사 기준은 단순 시간 제한이 아니라 실제 전투 행동과 카드 조합이 만드는 내부 점수다.

### 23.5 카드별 실제 연결

현재 코드 기준:

| 카드 | 실제 연결 |
| --- | --- |
| 반품 라벨 | 반품 표식 지속 증가, 차징 피해 소폭 증가, 표식 처치 처리량 증가 |
| 전단 폭죽 | 처치 폭발 해금, 폭발 처리량 증가 |
| 송출 잔류 | 차징 장판 해금, 오픈하우스 처리량 증가 |
| 로봇 명령 뒤집기 | 차징 오류/감속 강화, 로봇 처치 처리량 증가 |
| 야근 정산표 | 감사 기준 감소 |
| 배터리 절약 영수증 | 로봇 처치 처리량 증가, 전체 처리량 소폭 증가 |
| 오픈하우스 방문 확인 | 오픈하우스 처리량 승수 증가 |
| 저체력 근무수당 | 저체력 처리량 증가, 저체력 피해 감소 |
| 정상 고객 스티커 | 감사 기준 감소, 대신 감사 압력 증가 |
| 가족 할인 약관 | 오픈하우스 처리량 증가, 대신 감사 압력 증가 |
| 반품 불가 동의서 | 차징 피해 대폭 증가, 차징 처리량 증가, 실패 리스크 기록 |
| 약관 자동 갱신 | 전체 처리량 승수 증가, 대신 감사 압력 증가 |

### 23.6 정산 연결


### 24.3 두 번째 카드 시너지 보장

카드 데이터에 `synergy_ids`를 추가했다.

첫 카드 선택 후 다음 카드 선택부터는 이미 고른 카드의 `synergy_ids`를 기준으로 연결 카드가 최소 1장 나오도록 보정한다.

예:

```text
반품 라벨 선택

```

이 보정은 고정 진화표가 아니다.

역할은 “이번 판에서 내가 이미 고른 카드가 다음 선택에 영향을 준다”는 체감을 만드는 것이다.

## 26. 4차 구현 반영 기록

### 26.1 런 중 정산 후보 HUD

카드/감사/지역 선택이 만드는 배급권 후보가 런 중 HUD에 표시된다.

표시 예:

```text
정산 밥1+1 전1 신2  약관
```

이 HUD는 최종 정산표가 아니라 현재 판세 전망이다.

목표:

> 카드 선택과 구역 진입이 단순 전투력이 아니라 정산 결과를 바꾸고 있다는 것을 전투 중에 보여준다.

### 26.2 지역 약관 차별화

지역 약관이 오픈하우스 하나에만 묶이지 않고 R01 보조 루트까지 확장됐다.

| 지역 | 카드/룰 엔진 역할 |
| --- | --- |
| 오픈하우스 거리 | 처리량 승수, 신호표 후보, 캠페인 압력 |
| 배수로 침묵 주머니 | 신호표 후보, 신호 승인 보정, 조용한 우회 |
| 끊긴 광고 산책로 | 밥표 후보, 오염 위험, 가짜 안정 루트 |

### 26.3 오염 판정

정산 결과에 `오염`이 실제로 분리됐다.

오염은 실패가 아니다.

오염은 “승인은 됐지만 광고 시스템이 그 표를 읽었다”는 상태다.

카드 엔진 관점에서 오염은 장기적으로 다음 역할을 맡는다.

- 약관 카드의 위험한 대가
- 깊은 신호표 추적의 꼬리표
- 가짜 귀환로/환경 동조 루트의 후속 사건 씨앗
- 다음 런 지역 압력 또는 노선 기록의 근거

### 26.4 이번 구현으로 닫힌 과제

닫힌 과제:

- 런 중 배급권 후보 HUD 표시.
- 오픈하우스/배수로/끊긴 광고 산책로의 지역 약관 차별화 1차 적용.
- 오염 판정을 결과 화면에 별도 표시.

아직 남은 과제:

1. 카드별 처리량 기여도를 정산 화면에 표시한다.
2. 지역 약관을 런 시작/출격 게시판에서 미리 예고한다.
3. 오염이 다음 런 지역 변화나 노선 기록으로 이어지게 한다.
4. 실제 플레이테스트로 감사 기준/승인률/오염률 수치를 재조정한다.

## 27. 5차 구현 반영 기록

### 27.1 카드별 처리량 기여 로그

카드가 런 중 만든 처리량 또는 기준 완화분이 별도 로그로 기록된다.

기록 대상:

- 카드 ID
- 카드 이름
- 누적 기여 처리량
- 발동 횟수
- 짧은 기여 사유

예:

```text
반품 라벨
처리량 +420
사유: 반품 표식, 차징 표식
```

### 27.2 기여로 잡는 항목

현재 1차 구현은 다음을 카드 기여로 계산한다.

| 카드 | 기여로 잡는 것 |
| --- | --- |
| 반품 라벨 | 차징/표식 처치 처리량 |
| 전단 폭죽 | 전단 폭발/표식 연쇄 처리량 |
| 송출 잔류 | 장판/송출 잔류 처리량 |
| 로봇 명령 뒤집기 | 로봇 처치/명령 오류 처리량 |
| 야근 정산표 | 감사 기준 완화분 |
| 배터리 절약 영수증 | 로봇 처리량/전원표 처리 보정 |
| 오픈하우스 방문 확인 | 오픈하우스 체류 처리량 |
| 저체력 근무수당 | 저체력 처리량/기준 완화분 |
| 정상 고객 스티커 | 감사 기준 완화분 |
| 가족 할인 약관 | 오픈하우스/모델하우스 처리량 |
| 반품 불가 동의서 | 차징 약관 처리량 |
| 약관 자동 갱신 | 약관 증폭 처리량 |

이 수치는 정확한 회계표가 아니라 플레이어가 이해할 수 있는 기여 추정치다.

목표는 “이 카드가 이번 판에 실제로 일했다”를 보여주는 것이다.

### 27.3 결과 화면 표시

결과 화면에 상위 카드 기여가 표시된다.

예:

```text
카드 기여: 반품 라벨 처리량 +380 (반품 표식)
카드 기여: 오픈하우스 방문 확인 처리량 +210 (오픈하우스 체류)
카드 기여: 야근 정산표 - 밥표 후보와 감사 기준 완화
```

표시 원칙:

- 처리량 기여 상위 2개를 우선 표시한다.
- 남는 자리가 있으면 정산 전용 카드 기여를 1개 표시한다.
- 결과 화면이 회계표처럼 과밀해지지 않도록 최대 3줄만 표시한다.

### 27.4 닫힌 과제와 남은 과제

닫힌 과제:

- 카드별 처리량 기여도 정산 표시 1차 구현.

남은 과제:

1. 지역 약관을 런 시작/출격 게시판에서 미리 예고한다.
2. 오염이 다음 런 지역 변화나 노선 기록으로 이어지게 한다.
3. 보급소 배분에 NPC/시설 반응 문구를 연결한다.
4. 실제 플레이테스트로 카드 기여 산식이 납득되는지 검증한다.
