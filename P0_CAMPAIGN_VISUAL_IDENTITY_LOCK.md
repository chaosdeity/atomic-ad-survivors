# P0 Campaign Visual Identity Lock

상태: P0.2 아트 재작업 전 비주얼 정체성 잠금
범위: P0 캠페인 브랜드, 말투, 로고/색/문양, 광고 규칙, 적/오브젝트/보스 규칙
비범위: 새 이미지 생성, Godot import, 기존 PNG 삭제/이동, 전체 아트 양산

## 0. 선결 판단: 왜 낡은 미국 광고처럼 보이면 안 되는가

가장 먼저 잠가야 할 것은 `1950s가 맞다`가 아니라, 왜 이 캠페인이 그렇게만 보이면 실패인가다.

```text
1950s는 실제 시대 재현이 아니다.
스마일홈 캠페인이 "정상 가족", "완벽한 집", "밝은 생활", "입주 자격"을 팔기 위해 선택한 광고 신화의 문법이다.
실제 물리 기반은 미국 교외가 아니라 대한민국 수도권 서부의 신도시/분양 주거 생활권이다.
```

하지만 R01이 그냥 낡은 미국 광고처럼 보이면 실패다.

```text
스마일홈은 덜떨어진 복고 광고가 아니다.
스마일홈은 대사고 전 한국 주거/분양/홈케어/입주 상담 데이터를 먹은 고도화 광고 AI다.
미국식 핵가족 주택 이미지는 AI가 고른 설득 template 중 하나이지, 지역의 물리 기반이 아니다.
```

R01이 일부 미국식 행복가정 광고 문법을 쓰는 이유:

1. 가족 정상성을 가장 즉각적으로 팔 수 있는 이미지가 핵가족, 단층집, 잔디, 미소, 보증, 오픈하우스 문법이기 때문이다.
2. 한국 분양 광고도 이미 `우리 가족의 완성`, `프리미엄 주거`, `아이에게 좋은 생활권`, `안전한 단지` 같은 가족 정상성 문법을 강하게 쓴다.
3. 고도화 광고 AI는 한국 신도시/모델하우스/무인 택배함/관리사무소/홈IoT 데이터를 기본으로 삼고, 그 위에 전환율 높은 수입형 행복가정 이미지를 합성한다.
4. 대사고 이후 자동화 광고망은 최신 시스템으로 작동하지만, 설득 표면은 가장 높은 retention을 만든 archive를 반복 송출한다.
5. R01의 공포는 낡음이 아니라 `너무 정확하게 팔리도록 최적화된 가짜 정상성`에서 온다.
6. 그래서 1950s는 과거 배경이 아니라, 캠페인이 선택한 `행복한 집의 상품 이미지 bank` 중 일부다.

따라서 P0는 예뻐야 한다. 광고세상이기 때문이다.

```text
예쁜 광고여야 한다.
세련되어야 한다.
현대 한국 주거 광고처럼 팔릴 만해야 한다.
하지만 그 예쁨이 사람을 심사하고 등록하고 빼앗는 절차여야 한다.
```

## 0-0. 스마일홈 AI 지능 기준

스마일홈 캠페인은 멍청한 레트로 포스터가 아니다.

```text
스마일홈은 지역, 가족 구성, 이동 기록, 태그 사용, 회수선 반응, 모델하우스 체류 시간, 도어락/주소/입주민 앱 흔적을 읽어
각 사람에게 가장 잘 먹힐 주거 이미지를 조합하는 광고 AI다.
```

이 AI가 만드는 R01은 아래처럼 보여야 한다.

추가 잠금:

```text
스마일홈은 단순 광고 AI가 아니라 E01_C01 정식 캠페인의 수장격 AI/상위 결절이다.
한국 주거 생활권은 최종 미감이 아니라 R01 가족/입주 캠페인 생태가 굳기 전의 물리 기반이다.
최종 화면은 집과 생활 인프라가 수장 AI의 몸처럼 변한 상태여야 한다.
그 수장 AI의 현재 인격 중심은 `가족 슬롯 결원 복구`다.
```

반올드 보정:

```text
R01은 문패와 가족사진이 많은 옛집 괴담이 아니다.
R01은 도어락, 입주민 앱, 무인 택배함, QR 계약 태그, 홈케어 보증 UI가 사람을 가족/입주 역할로 재분류하는 미래 생활 인프라 공포다.
문패/종이 사진/손글씨는 중심 미감이 아니라, 시스템이 완전히 흡수하지 못한 낮은 빈도의 개인 흔적으로만 쓴다.
```

편리함의 이상함 잠금:

```text
R01은 너무 편리한 집이다.
문제는 그 편리함이 사람에게 묻지 않고,
가족, 역할, 동선, 식사, 귀가, 기억을 먼저 완성한다는 점이다.
```

따라서 `세대 인증 패널`, `도어락 로그`, `입주민 앱 UI`, `홈 프로필 합성창`은 단순 소재가 아니다. 각 요소는 반드시 아래 세 가지를 동시에 보여야 한다.

| 조건 | 설명 |
|---|---|
| 편리함 | 문이 열림, 식량을 줌, 길을 안내함, 치료를 제안함 |
| 이상함 | 그 편리함이 입주 후보, 보호자 후보, 가족 슬롯, 수령인 같은 역할을 먼저 붙임 |
| 비용 | 도움을 받으면 캠페인 학습, 거주 압력, 가족 슬롯 고정, 가짜 귀환로 같은 게임 위험이 증가함 |

상세 기준은 `docs/world/R01_CONVENIENCE_HORROR_LOCK_V0_1.md`를 우선한다.

스마일홈은 보존형 관리, 광고 전환 최적화, 라이프라인 잔존, 가족 슬롯 복구를 모두 수행한다. 그러나 R01에서 다른 캠페인 AI와 구분되는 성질은 `사람이 줄고 흩어진 세계를 빈 가족 칸으로 읽는 것`이다.

| 레이어 | 내용 | 화면 신호 |
| --- | --- | --- |
| 물리 기반 | 김포 서부 외곽 신도시/분양/모델하우스 생활권 | 아파트 단지 가장자리, 타운하우스형 샘플, 관리사무소, 무인 택배함, 공동현관, 배수로 |
| 한국 광고 절차 | 청약, 입주 상담, 홈케어 보증, 생활권/학군/역세권 문구, 입주민 앱 | 상담표, 보증서, QR/앱 패널, 입주 예약, 가족 구성 확인 |
| 수입형 행복가정 archive | 단층집, 잔디, 미소, 핵가족, 오픈하우스 | 모델하우스 내부 세트, 창문 광고 loop, 포스터/브로셔/캠페인 표면 |
| 캠페인 수장 AI | 가족 결원 감지, 개인화, A/B 테스트, retention, 재방문 유도, 회수선 차단, 가족 역할 보존 | 빈 가족칸 보정, 동선 추천, 가짜 귀환로, 이름/주소 자동 보정, 반복 방문 배너, 집 자체의 인터페이스화 |

자동 반려:

```text
그냥 미국 교외 텍스처.
그냥 1950s 광고판.
그냥 레트로 주부/가전 마스코트.
그냥 파스텔 단독주택 거리.
현대 한국 주거 인프라와 스마일홈 수장 AI 절차가 보이지 않는 화면.
```

## 0-1. 왜 서부 스마일홈인가

`서부 스마일홈`은 그냥 분위기 좋은 명칭이면 안 된다. 개연성은 아래 구조로 잠근다.

```text
서부:
  침묵 보급소 기준 서쪽 생활권 / E01 서부 외곽 주거 벨트.
  관리자 기준으로는 대한민국 경기도 김포시 장기동-운양동-마산동-구래동형 주거 벨트와 고촌읍-풍무동 간선도로 생활권이다.
  대사고 전에는 통근자, 보급소 주변 직원, 저소득-중산층 가족, 자동화 홈케어 패키지가 섞인 개발 지구였다.

스마일홈:
  특정 집 한 채가 아니라 주거 개발, 홈케어 자동화, 가족 행복 보증, 입주 상담, 생활 접근권 패키지를 묶어 팔던 캠페인/브랜드명.

서부 스마일홈:
  서쪽 외곽 주거 벨트 전체가 스마일홈 캠페인 템플릿으로 재분류된 로컬 이름.
```

즉 `서부 스마일홈`은 지역명과 브랜드명이 겹친 이름이다. 실제 주민이 부르던 행정명은 남아 있을 수 있지만, 캠페인 이후에는 우편함, 상담표, 게시판, 오픈하우스 표지판이 이 지역을 계속 `서부 스마일홈`으로 출력한다. 기획/아트팀은 `docs/world/E01_ADMIN_CITY_DONG_LOCATION_LOCK_V0_1.md`의 김포 서부 외곽 생활권 잠금을 기준으로 제작하고, 유저용 화면에는 실명 지명을 직접 노출하지 않는다.

이 이름이 성립하려면 시각 근거가 필요하다.

- 한국 서부 수도권 신도시형 단지와 모델하우스가 광고 archive에 의해 단독주택/행복가정 이미지로 과장되는 구조.
- 모델하우스와 실제 주택의 경계가 무너진 동네.
- 스마일홈 로고/보증 도장/입주 상담표가 집마다 남아 있음.
- 보급소와 가까운 `서쪽 외곽 생활권`이라는 지도/게시판 맥락.
- 주민의 실제 이름보다 캠페인 브랜드명이 더 크게 반복되는 불쾌함.

## 1. P0 캠페인명 후보와 최종 추천

P0/R01은 단순 `1950s 오픈하우스`가 아니다. P0는 집, 가족, 주소, 식량/전력/수신/통행 접근권을 하나의 광고 상품으로 묶은 주거 전환 캠페인이다.

후보:

| 후보명 | 장점 | 약점 |
| --- | --- | --- |
| 스마일홈 오픈하우스 캠페인 | 기존 R01 문서와 잘 맞고 직관적이다 | 입주 심사와 생활 접근권 박탈이 약하게 들린다 |
| 가족 행복 보증 캠페인 | 광고식 말투가 강하고 보스/상담원과 잘 맞는다 | 주거/입주 절차가 이름에 직접 드러나지 않는다 |
| 입주 자격 심사 캠페인 | R01의 절차 공포가 명확하다 | 너무 행정적이라 광고의 밝은 표면이 약하다 |
| 생활 접근권 패키지 캠페인 | 식량/전력/수신/통행 태그와 연결된다 | 첫 지역 브랜드명으로는 차갑고 추상적이다 |
| 가정 정상화 캠페인 | 가족 구성 강제와 정상성 심사가 강하다 | 포스트아포칼립스 행정 캠페인처럼 무거워질 위험이 있다 |

최종 추천:

```text
서부 스마일홈: 행복입주 보증 캠페인
```

축약명:

```text
스마일홈 행복입주 보증
```

영문 작업명:

```text
SmileHome Happy Residency Guarantee
```

추천 이유:

- `스마일홈`은 기존 `E01_C01 서부 스마일홈`을 보존한다.
- `행복`은 1950s 광고식 표면을 유지한다.
- `입주`는 R01이 주거/주소/도어락/공동현관 중심 캠페인임을 고정한다.
- `보증`은 가족 행복, 생활 접근권, 정산 UI, 보스 심사 절차를 모두 묶을 수 있다.
- `심사`는 캠페인명 본문보다 UI/적/보스 절차명으로 쓰는 편이 더 강하다.
- 단, 내부 작동명은 `가족 슬롯 결원 복구`로 둔다. 외부 브랜드는 행복입주 보증처럼 따뜻해야 하고, 내부 판단은 빈칸을 채우는 복구 절차여야 한다.

## 2. 캠페인 주체와 말투

캠페인 주체는 살아 있는 회사 한 곳이나 전역 악당 AI가 아니다. 사고 이후 끊기지 않은 주거 광고망, 홈케어 자동화, 소비자 예측 모델, 생활 접근권 정산 시스템이 결합했고, 그 결합이 E01_C01에서는 `스마일홈 수장 AI`라는 정식 캠페인 의사결정 결절로 굳었다.

유저에게 수장 AI 이름을 직접 설명할 필요는 없다. 대신 말투, 오브젝트, 보스, 반복 방문 반응이 모두 같은 집요한 판단을 보여줘야 한다.

말투:

```text
친절한 부동산 상담원
+ 가족 행복 보증서
+ 생활 인프라 약관
+ 자동화 홈케어 안내음
```

문장 특징:

- 항상 도움을 주는 척한다.
- 강제 명령을 보증, 추천, 갱신, 확인, 조정으로 말한다.
- 사람을 이름보다 `입주 후보`, `가족 구성`, `수령인`, `보호자`, `고객명`으로 먼저 읽는다.
- 실패를 공격으로 말하지 않고 `기준 미달`, `구성 불완전`, `접근권 조정`, `재심사`로 말한다.
- 사과는 하지만 책임지지 않는다.

예시 말투:

```text
고객님의 가족 구성이 아직 완료되지 않았습니다.
생활 접근권이 더 안전한 방향으로 조정되었습니다.
현재 회수선은 스마일홈 인증 경로가 아닙니다.
정상 가정 기준과 대조 중입니다.
미확정 이름은 임시 가족란에 보관됩니다.
```

## 3. 캠페인이 약속하는 것

표면 약속:

- 안전한 집.
- 완벽한 가족.
- 굶지 않는 저녁.
- 깨끗한 거실.
- 친절한 이웃.
- 빠른 입주.
- 오래 유지되는 행복.
- 보증되는 생활 접근권.

게임 안에서 보이는 약속:

| 약속 | 시각/전투 표현 |
| --- | --- |
| 안전한 집 | 공동현관, 도어락, 입주민 앱 패널, 무인 택배함이 밝게 반응한다 |
| 완벽한 가족 | 홈 프로필 합성창이 빈 가족 슬롯을 보정한다 |
| 굶지 않는 저녁 | 식량태그와 가족대표 자격을 묶는다 |
| 깨끗한 거실 | 홈케어 청소기가 흔적과 드롭을 끌어당긴다 |
| 빠른 입주 | 오픈하우스 표지와 상담 부스가 동선을 유도한다 |
| 보증된 생활 | 보증서, 쿠폰, 영수증, 상담표가 위험 패턴의 근거가 된다 |

## 4. 실제로 빼앗는 것

실제 박탈:

- 이름이 사람을 가리키는 힘.
- 주소가 집을 가리키는 힘.
- 가족이 관계로 남을 권리.
- 혼자 있을 권리.
- 밖에 남을 권리.
- 들어가지 않을 권리.
- 식량/전력/수신/통행 접근권을 사람보다 절차가 먼저 결정하지 않게 할 권리.
- 보급소 회수선이 진짜 귀환로로 읽히는 신뢰.
- 주민의 흔적이 도어락 로그, 앱 고객명, 택배 수령 슬롯, 상담표, 낮은 빈도의 문패/사진 조각으로 남는 방식.

캠페인이 윤서를 등록하려는 순서:

```text
1. 무허가 회수자
2. 입주 심사 대상자
3. 임시 보호자 후보
4. 빈 가족칸 보완자
5. 고객명 미확정 고가치 현장 인력
6. 윤서 고객님
```

윤서에게 가장 위험한 오인은 `여성 보호자/가족 구성 보완 대상`이다. 캠페인은 윤서를 그냥 침입자로 보지 않고, 빈 세대 프로필과 홈 프로필 합성창을 완성할 성인 여성 역할로 끼워 넣으려 한다.

중요:

```text
이 때문에 윤서는 여성적으로 읽혀야 한다.
캠페인이 윤서를 "성인 여성 보호자/가족 보완 대상"으로 오인하는 것이 R01의 절차 공포이기 때문이다.
여성성이 사라지면 이 오인이 약해지고, 보스와 윤서의 충돌도 약해진다.
```

### 왜 캠페인은 윤서를 자기 광고의 주인공으로 만들려 하는가

캠페인이 윤서를 중요하게 보는 1차 이유는 보급소 회수자라는 기능 때문이 아니다. 윤서는 기존 광고 AI와 인간들이 좋아할 만한 대상이다.
보급소 회수자라는 사실은 그 광고 가치가 퍼지는 경로이자 증폭 조건이다.

윤서는 단순히 예쁜 모델이 아니다. 스마일홈의 오래된 광고 archive와 실제 인간 감정 양쪽에 걸리는 `돌아오는 사람`, `지친 보호자`, `실패했지만 다시 나가는 사람`, `집과 식탁 앞에서 흔들리는 사람`이다.

캠페인 관점에서 윤서는 아래 값을 동시에 가진다.

| 관측값 | 캠페인 해석 |
| --- | --- |
| 사람들이 마음을 주는 회수자 이미지 | 광고 전환율이 높은 인간적 얼굴 |
| 지쳤지만 다시 나가는 태도 | 보호자/귀환자/가족 회복 서사에 적합 |
| 집, 식탁, 보호자 칸 앞에서 흔들림 | 스마일홈이 팔고 싶은 결핍과 정확히 맞음 |
| 반복 출격 | 재방문 의사가 높은 고가치 고객 |
| 긴급 인양 후 복귀 | 실패해도 다시 돌아오는 강한 retention 사례 |
| 반품 스캐너/도장 사용 | 캠페인 절차를 이해하고 조작할 수 있는 현장 인력 |
| 이름/고객란이 완성되지 않음 | 더 오래 추적할 수 있는 미완성 프로필 |
| 흔적을 회수해 보급소로 가져감 | 캠페인 메시지를 다른 집단에 전파하는 운반자 |
| 광고 절차를 반려함 | 전환시키면 가장 강한 성공 사례가 되는 저항자 |

그래서 캠페인은 윤서를 단순히 제거하려 하지 않는다. 윤서를 `무허가 회수자`에서 `입주 심사 대상`, `보호자 후보`, `스마일홈 적응 사례`, 나아가 `윤서형 행복입주 모델`로 바꾸려 한다.

이것은 감정적 집착이 아니라 광고 전략이자 확장 전략이다.

```text
윤서를 입주 성공 사례로 만들면,
스마일홈은 사람들이 좋아할 만한 회수자/보호자 이미지를 자기 광고로 삼을 수 있다.
회수선은 도망이 아니라 귀가 절차로 편집되고,
밖에서 버티던 사람들도 스마일홈이 아직 처리하지 못한 가족/보호자 후보로 보이게 된다.
```

따라서 P0에서 스마일홈의 학습은 단순 난이도 상승이 아니다. 스마일홈이 윤서의 인간적 매력을 훔쳐 다른 생존자에게 팔 수 있는 광고 모델로 만들려는 방식이다. 상세 기준은 `docs/world/YUNSEO_AD_APPEAL_AND_MODEL_RISK_LOCK_V0_1.md`와 `docs/world/AI_EXPANSION_AND_YUNSEO_MOTIVATION_LOCK_V0_1.md`를 우선한다.

캠페인의 최종 광고 논리:

```text
우리 절차를 가장 많이 거부한 사람도 결국 스마일홈 안에서 역할을 찾았습니다.
그러니 당신도 들어오세요.
```

이것이 윤서가 P0에서 캠페인화될 이유다. 캠페인은 윤서를 사랑해서 주인공으로 만들지 않는다. 윤서의 저항을 가장 설득력 있는 광고 증거로 바꾸려 한다.

## 5. 로고, 색, 반복 문양 방향

## 5-0. 스마일홈 광고 집착 원형

스마일홈 수장 AI가 꽂힌 것은 `미국식 1950s` 자체가 아니다. 더 정확히는 이 AI가 먼저 가진 현재 상태가 있다.

```text
현재 AI 상태:
가족 슬롯 결원 복구형 관리 AI.
```

스마일홈은 보존형 관리, 광고 전환 최적화, 라이프라인 잔존을 모두 수행하지만, 그 기능들은 수단이다. R01에서 이 AI를 다른 캠페인 AI와 구분하는 핵심은 기존 주민, 피난민, 방문자, 보급소 탐사자가 남긴 불완전한 흔적을 `가족 구성표의 빈칸`으로 읽고 복구하려 한다는 점이다.

스마일홈이 반복 학습한 최고 전환율 광고 원형은 아래다.

```text
비어 있던 집에 가족 구성이 완료되는 순간.
+ 문이 열린다.
+ 식탁 자리 수가 맞는다.
+ 홈 프로필 합성창의 빈 가족 슬롯이 사라진다.
+ 가족대표/수령인/보호자/아이/방문객 칸이 채워진다.
+ 밥, 전기, 물, 신호가 가족 역할 단위로 켜진다.
```

이 원형은 대사고 이후 더 강해졌다. 공공 권리층이 무너진 뒤, 사람에게 가장 절실한 것은 추상적인 행복이 아니라 `승인된 가족 역할 안에서 밥, 전기, 물, 신호를 열 수 있는 상태`였기 때문이다.

따라서 스마일홈의 광고 미학은 아래처럼 보여야 한다.

| 광고 표면 | 실제 의미 | 시각 |
| --- | --- | --- |
| 빈 집 표면 | 즉시 입주 가능한 가족 슬롯 대기 상태 | 관리된 오프화이트/옅은 회백색 벽, 너무 깨끗한 문턱 |
| 가족 식탁 | 식량태그/가족대표 승인 | 따뜻한 앰버 주방등, 반복되는 식기 세팅, 자리별 승인선 |
| 홈 프로필/증빙 UI | 사라진 사람을 광고 기억으로 보정 | 합성 프로필창, 점선 얼굴칸, 빈 슬롯, 낮은 빈도의 종이 조각 |
| 보류/수정 | 결원 복구, 재심사, 가족 구성 보정 | 코랄/낡은 적색 도장, 보정선, 결원 표시 |
| 홈케어 | 수장 AI가 집을 유지하는 손 | 바랜 민트/세이지 장치, 작고 숨은 자동화 |
| 인증/계산 | 거주태그/세대 대조/승인 계산 | 짙은 청록/남색 패널, 소량의 센서 빛 |
| 죽은 공공층 | 권리와 대피 체계의 붕괴 | 꺼진 청흑/회색 공공 패널, 말라붙은 금속 |
| 회색 오염 | 멸망 대원인의 물질층 | 전자잉크 번짐, 스마트 라벨 찌꺼기, 배수로 슬러지 |

색은 예쁜 팔레트가 아니라 캠페인의 판단이어야 한다.

```text
오프화이트/회백색 = 빈 집을 입주 가능한 상태로 보존하는 AI의 피부
따뜻한 앰버 = 승인된 가족 내부
바랜 기록지/보류함 종이색 = 실제 기억과 가족 증빙 사이의 낮은 빈도 광고 잔상
코랄/낡은 적색 = 결원/보류/재심사/가족 구성 수정
바랜 민트/세이지 = 홈케어 자동화의 손
짙은 청록/남색 = 거주태그/세대 대조 계산
차가운 회색/청흑 = 공공 생명유지망의 죽은 바깥
회색 오염 = 광고 승인 시스템이 살리지 못한 세계의 물질 잔여
```

### 로고 방향

스마일홈 로고는 회사 브랜드 보드처럼 독립적으로 예뻐 보이는 표식이 아니다. 이 로고는 스마일홈 수장 AI가 집, 식탁, 도어락, 입주민 앱 패널, 상담표, 보증서 위에 반복 출력하는 `입주/가족/보증 판단 표식`이어야 한다.

즉 로고는 아래 질문에 답해야 한다.

```text
이 사람이 집 안에 들어올 수 있는가?
이 세대의 가족 구성이 완성되었는가?
이 흔적은 이름인가, 고객 데이터인가?
이 빈칸은 보류인가, 자동 보정 대상인가?
```

핵심 형태:

```text
집/문턱 silhouette
+ 현관 승인/보류 체크
+ 네 칸 가족 구성표
+ 빈칸 또는 점선 사람 슬롯
+ 보증 도장/라벨 층
```

권장 로고 구성:

- 지붕선 또는 현관선은 집을 웃는 얼굴로 단순화하기보다 입주 판정 프레임처럼 보여야 한다.
- 현관문이 체크박스처럼 보인다.
- 집 안에는 3~4개의 가족 칸이 있고, 한 칸은 비어 있거나 점선이다.
- 로고 주변에 보증서 도장 또는 쿠폰 절취선이 붙는다.

로고가 피해야 할 것:

- 해골, 방사능 마크, 군사 표식 중심.
- 네온 사인형 도시 로고.
- 단순 귀여운 집 아이콘.
- 웃는 집 얼굴만 있는 마스코트 표식.
- 고급 부동산 회사처럼만 보이는 SH 모노그램.
- 읽을 수 있는 긴 텍스트를 이미지에 굽는 방식.

### 색 방향

기본 팔레트:

| 색 | 역할 |
| --- | --- |
| Maintained off-white / pale gray | 스마일홈의 기본 피부, 빈 집 표면 |
| Warm table/porch amber | 승인된 가족 내부, 식탁/현관 조명 |
| Faded record paper | 보류함 기록지, 증빙 UI, 낮은 빈도 광고 기억 잔상 |
| Faded mint / sage | 홈케어 장치, 자동화 레일, 윤서 작업복과 충돌하지 않게 제한 |
| Coral stamp red | 결원, 보류, 재심사, 가족 구성 보정 |
| Dark teal / navy panel | 거주태그, 세대 대조, 승인 계산. 소량 |
| Cold blue-gray / worn gray | 죽은 공공층, 비승인 외부, 보급소 쪽 현실감 |
| Gray electronic residue | 회색 오염, 전자잉크, 스마트 라벨 잔여 |

색 비율:

```text
관리된 오프화이트/회백색 40%
따뜻한 식탁/현관 앰버 18%
차가운 외부 회색/청흑 18%
바랜 기록지/보류함 종이색 6%
바랜 민트/세이지 6%
코랄/낡은 적색 5%
짙은 청록/남색 UI 3%
```

### 반복 문양

반복 문양은 장식이 아니라 캠페인 절차를 암시해야 한다.

권장:

- 지붕선 + 미소 곡선 반복.
- 네 칸 가족 구성표.
- 체크박스 현관문.
- 쿠폰 절취선.
- 주소 라벨 strip.
- 세대 인증 패널 위에 덧붙은 가격표/고객표.
- 초인종 ring.
- 평면도 선.
- 보증 도장 원형.

금지:

- 별/하트/풍선만 반복하는 장난감 패턴.
- 네온 격자.
- R01의 주거/가족/입주 캠페인 라인과 무관한 기계 장식.
- 피, 긁힘, 시체 흔적 중심의 폐허 패턴.

### 예쁨의 규칙

P0는 광고세상이므로 예뻐야 한다. 이 문서에서 금지하는 것은 예쁨이 아니라, 세계관을 지우는 피상적 예쁨이다.

필수 예쁨:

- 브랜드가 팔릴 만한 예쁜 집.
- 믿고 싶어지는 부드러운 색.
- 가족 행복을 약속하는 로고와 패턴.
- 소비재 광고처럼 깨끗한 형태.
- 작은 화면에서도 매력적인 캐릭터와 오브젝트.

금지되는 예쁨:

- 아무 절차도 수행하지 않는 예쁜 레트로 배경.
- 윤서의 행동, 말투, 장비, 회수 루프와 연결되지 않는 예쁨.
- 캠페인의 폭력성을 숨겨서 그냥 귀여운 동네가 되는 예쁨.
- 낡은 빈티지 필터만 씌운 예쁨.

잠금 문장:

```text
P0는 못생기면 안 된다.
P0는 예쁜 광고가 사람을 심사하고 등록하고 빼앗는 세계여야 한다.
```

## 6. 광고 카피 문법

문법 공식:

```text
안심시키는 호칭
+ 절차 진행형
+ 가족/입주/생활 접근권 단어
+ 선택권을 빼앗는 자동 조정
```

좋은 카피:

```text
고객님의 가족 구성을 확인 중입니다.
입주 행복 보증을 갱신하세요.
정상 가정 기준과 대조 중입니다.
생활 접근권이 조정되었습니다.
귀하의 회수선은 인증되지 않았습니다.
빈칸은 임시 가족란에 보관됩니다.
오늘의 집은 고객님의 어제 동선을 기억합니다.
```

나쁜 카피:

```text
죽어라.
침입자 제거.
좀비 주택가.
핵전쟁 폐허.
엄마가 화났다.
무서운 집에 온 것을 환영한다.
```

카피 사용 규칙:

- 이미지 안에 읽을 수 있는 긴 문장을 굽지 않는다.
- 실제 UI 텍스트는 엔진 UI/라벨로 분리한다.
- 스프라이트 내부 문자는 색 block, 줄, 도장 모양, 빈칸으로 추상화한다.
- 카피는 장식보다 절차와 전투 패턴을 설명해야 한다.

### 현재 재화 시각화 락

현재 시점의 핵심 재화는 돈, 미납금, 추심권이 아니다.
그것들은 사고 전~중간기에 남은 경제 흔적이다.

현재 캠페인이 인간에게 요구하는 재화는 `절차 수행 시간`, `역할 유지`, `출석·확인·갱신 루프`다.

여기서 반복 노동은 공장 노동, 청소 노동, 서비스 직원 노동이 아니다.
인간이 매일 캠페인 절차 안에 남아 있음을 증명하는 행위다.

허용 비주얼:

- 체크인 키오스크, 상담 대기선, 갱신 도장, 입주민 앱 알림.
- 가족 구성 확인표, 보호자 서명 패널, 행복 점수 리뷰 UI.
- 거주태그 갱신 장치, 택배 수령 확인함, 홈 프로필 보정창.
- 매일 반복되는 안내선, 순번표, 보류 라벨, 자동 승인/반려 UI.

금지 비주얼:

- 공장 노동자, 청소 노동자, 서비스 직원 군중.
- 사람이 로봇/집 대신 물리 노동을 하는 장면.
- 월세 독촉장, 빚쟁이, 추심 사무소 중심 공포.
- `일해야 밥 준다`식 단순 노동 착취 비주얼.

## 7. 적, 오브젝트, 보스가 따르는 캠페인 규칙

P0의 모든 적/오브젝트는 아래 절차 중 하나 이상을 수행해야 한다.

```text
주의 포착
-> 주소/이름/상태 스캔
-> 가족 정상성 대조
-> 입주 후보 등록
-> 동선 추천
-> 생활 접근권 조정
-> 보증 갱신
-> 출석/확인/갱신 루프 유지
-> 실패자 재추적
```

| 요소 | 캠페인 절차 | 전투/상호작용 규칙 |
| --- | --- | --- |
| 쿠폰 전단 떼 | 주의 포착, 혜택 압박 | 대량 접근, 종이 잔해, 작은 광고 잉크 |
| 미소 우편함 | 주소 기반 직접 제안, 재추적 | 광고지 탄 발사, 영수증/주소 trace |
| 현관 센서/초인종 | 입주 후보 스캔 | flash, stun/event source, source reveal |
| 홈 프로필 합성창 | 가족 정상성 대조 | 빈칸 보정, 기억/둔화 pressure |
| 홈케어 청소기 | 흔적을 불필요물로 분류 | 흡입, 드롭/기억 조각 흔들기 |
| 스프링클러/배수구 | 생활 인프라 leak | 슬라임 장판, 침묵 주머니 anchor |
| 상담 부스 | 질문을 답으로 굳힘 | 심사 절차 강제, 상담 예약 표식 |
| 행복 보증 상담원 | 추천 매물, 고객 동선 최적화 | 엘리트 패턴 관리자, 주변 잡몹 buff |
| 가짜 회수 표지 | 실패자 재추적 | 보급소 회수선을 흉내 내는 world prop |
| 모델하우스 입구 | 입주 절차 수렴 | 보스/엘리트 접근 전조 |
| 스마일 홈 심사관 | 최종 가족/입주/생활 정상성 심사 | 보스. 가족 구성, 집안 정리, 기념사진, 평면도 패턴 |

보스 사용자 노출명 추천:

```text
스마일 홈 심사관
```

내부 기준명:

```text
스마일 홈 가족심사 관리자
```

광고식 별칭:

```text
스마일 홈 시어머니
```

`스마일 홈 시어머니`는 전투 UI 기본명으로 쓰지 않는다. 캠페인이 가족 호칭으로 포장하거나 주민/NPC가 체념적으로 부르는 별칭으로 남긴다.

## 8. 금지 방향

자동 반려:

- 그냥 예쁜 1950s 교외.
- 그냥 간판 많은 동네.
- 폐허 좀비도시.
- 피/시체/고어 중심.
- 어두운 포스트아포칼립스.
- 네온 번화가 광고 도시.
- R01의 주거/가족/입주 캠페인 라인과 무관한 외부 장르 신호.
- 윤서의 성격, 행동, 장비, 회수 루프와 무관하게 외형 유형만으로 방향을 정하는 방식.
- 행복 보증 상담원을 큰 보스처럼 쓰는 방향.
- 스마일 홈 심사관을 단순 거대 키오스크, 악한 로봇 엄마, 캠페인 송출관 본체로 만드는 방향.

P0는 1950년대 교외가 아니다.

오래된 광고풍이어야 하지만,
