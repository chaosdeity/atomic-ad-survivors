```text

R03의 핵심 오브젝트는 방해물이 아니다.
```

## 3. R03-PT-01 라벨 전조

목표:

```text
라벨은 맞는 장판이 아니라, 붙일 대상을 바꿀 수 있는 판정 상태다.
```

테스트 절차:

| 단계 | 관찰 |
|---|---|
| 1 | 송장 프린터 전조를 처음 본 유저가 피할 수 있는가 |
| 2 | 두 번째 전조에서 적을 장판에 넣는 시도가 나오는가 |
| 3 | 라벨이 붙은 뒤 경로/드론/정산 변화가 연결되는가 |

통과 기준:

```text
5분 플레이 안에 라벨 회피 1회 이상.
적 또는 오브젝트 라벨 전이 1회 이상.
플레이어가 "저걸 저쪽에 붙이면 되네"라고 이해할 수 있음.
```

실패 시 조정:

| 문제 | 조정 |
|---|---|
| 전조가 안 보임 | 바닥 윤곽/인쇄음/색 대비 강화 |
| 피하기 어려움 | 전조 시간 증가, 첫 구간 밀도 감소 |
| 역이용이 안 나옴 | 적 이동 경로를 프린터 전조와 겹치게 배치 |

---

## 4. R03-PT-02 레일 조작

목표:

```text
레일은 조작권을 빼앗는 바닥이 아니라, 전투 위치를 바꾸는 도구다.
```

테스트 절차:

| 단계 | 관찰 |
|---|---|
| 1 | 유저가 레일 방향을 읽는가 |
| 2 | 레일 밖 안전 지대를 인식하는가 |
| 3 | 적/드론을 레일에 태워 유리한 위치를 만드는가 |

통과 기준:

```text
R03-L02 첫 3분 안에 레일 회피 1회, 레일 이용 1회가 모두 나온다.
유저 피드백에서 "밀려서 짜증난다"보다 "이걸 이용할 수 있다"가 먼저 나온다.
```

실패 시 조정:

| 문제 | 조정 |
|---|---|
| 강제 이동감이 큼 | push_strength 감소, 레일 폭 축소 |
| 방향을 못 읽음 | 화살표/조명/모터음 강화 |
| 이용할 대상이 없음 | 느린 적 또는 드론 스캔 구간을 레일 옆에 배치 |

## 5. R03-PT-03 드론 오유도

목표:

```text
반송 드론은 추적 탄환이 아니라, 누가 반송 대상인지 잘못 읽는 장치다.
```

테스트 절차:

| 단계 | 관찰 |
|---|---|
| 1 | 드론 스캔선이 보이는가 |
| 2 | 유저가 라벨 붙은 적을 드론 쪽으로 유도하는가 |
| 3 | 오유도 성공 후 정산/다음 런 변화가 보이는가 |

통과 기준:

```text
R03-L02 플레이 1회 안에 드론 오유도 성공 1회 이상.
성공 시 유저가 피해 회피보다 "시스템이 실수했다"는 감각을 얻음.
```

실패 시 조정:

| 문제 | 조정 |
|---|---|
| 탄환처럼 보임 | 스캔/확인/혼란 상태를 시각적으로 분리 |
| 오유도 난이도 높음 | 라벨 적 우선순위 강화 |
| 성공 보상이 약함 | 정산 사유와 다음 런 변화 문구 강화 |

## 6. R03-PT-04 보관함 선택

목표:

```text
미수령 보관함은 상자가 아니라 선택이다.
열면 물자가 오고, 남기면 기록이 남는다.
```

테스트 절차:

| 단계 | 관찰 |
|---|---|
| 1 | 첫 보관함을 안전하게 조사할 수 있는가 |
| 2 | 열기와 남기기의 차이가 짧게 전달되는가 |
| 3 | 다음 런에서 남긴 보관함 잔향이 보이는가 |

통과 기준:

```text
첫 보관함은 유저가 손해를 크게 걱정하지 않고 실험 가능.
두 번째부터 위험/보상 판단이 생김.
보관함 남기기 후 다음 런 잔향이 1회 이상 표시됨.
```

## 7. R03-PT-05 보류 스탬프

목표:

```text
보류 스탬프는 해제가 아니다.
판정을 잠깐 늦추고, 늦춘 흔적을 남긴다.
```

통과 기준:

| 항목 | 기준 |
|---|---|
| 손맛 | 위험한 라벨/드론 판정을 늦추는 즉시 피드백 |
| 제한 | 모든 위험을 지우지 않음 |
| 후폭풍 | 남발 시 보류 기록 재조회 |
| 정산 | 흔적 보존 또는 후보 보류로 연결 |

실패 신호:

```text
유저가 보류 스탬프를 "무적 버튼"으로 이해한다.
또는 눌러도 무슨 일이 일어났는지 모른다.
```

## 8. R03-PT-06 반복 반응

목표:

```text
R03은 유저의 악용 패턴을 기억해야 한다.
하지만 벌주는 느낌보다 다시 속일 수 있는 느낌이 먼저다.
```

반복 반응 후보:

| 이전 행동 | 다음 런 변화 |
|---|---|
| 드론 오유도 다수 | 반송 드론 재검사 |
| 보관함 남김 | 야드 안내판에 보관함 번호 잔향 |
| 레일 우회 다수 | 레일 방향 전환 빨라짐 |
| 보류 스탬프 남발 | 보류 기록 재조회 |

통과 기준:

```text
다음 런 변화 1개 이상이 결과 화면과 보급소 중 하나에 표시된다.
플레이어가 변화 원인을 직전 런 행동과 연결할 수 있다.
```

## 9. R03-PT-07 정산 이해

목표:

```text
유저는 보상은 알아야 하지만, 설정 강의는 읽지 않아야 한다.
```

정산 체크:

| 항목 | 기준 |
|---|---|
| 확정 태그 | 보급태그/통행태그가 명확히 보임 |
| 후보 태그 | 후보와 확정이 분리됨 |
| 진료태그 기록 | R02 씨앗으로 보류됨 |
| 라벨 | 태그 자원처럼 보이지 않음 |
| RETURN-05 | 정식 해금처럼 보이지 않음 |

테스트 질문:

```text
이번 런에서 무엇을 얻었나?
무엇이 보류됐나?
다음 런에서 무엇이 달라질 것 같나?
라벨과 태그가 같은 것처럼 보였나?
```

## 10. R03-PT-08 톤 검산

목표:

```text
조롱 대상은 광고 시스템, 약관, 고객센터 말투다.
사람과 로봇은 조롱하지 않는다.
```

통과 기준:

| 항목 | 기준 |
|---|---|
| 윤서 | 현장 판단자. 긴 해설자 아님 |
| RETURN-05 | 미확인 수취인 씨앗. 동료 아님 |
| R02 씨앗 | 의료 접근 문제. 병원 악역화 아님 |
| 드론/레일 | 우스운 성실함은 있어도 생존자 조롱 없음 |

## 11. 계측 후보

구현 후 플레이테스트 계측에 넣을 값:

| metric_key | 의미 |
|---|---|
| `r03_label_dodges` | 라벨 회피 횟수 |
| `r03_label_transfers` | 라벨 전이 횟수 |
| `r03_route_bypasses` | 레일/게이트 우회 횟수 |
| `r03_drone_misdirects` | 드론 오유도 횟수 |
| `r03_locker_opened` | 보관함 열기 횟수 |
| `r03_locker_left` | 보관함 남기기 횟수 |
| `r03_hold_uses` | 보류 스탬프 사용 횟수 |
| `r03_repeat_reactions_seen` | 다음 런 변화 표시 횟수 |

최소 통과 예시:

```text
R03 라벨 6회 이상
라벨 전이 1회 이상
레일 우회 2회 이상
드론 오유도 1회 이상
보관함 선택 1회 이상
다음 런 변화 1회 이상
```

## 12. 실패 분류

| 실패 유형 | 의미 | 우선 조치 |
|---|---|---|
| nuisance_fail | 방해물처럼만 느껴짐 | 전조/역이용/보상 피드백 조정 |
| readability_fail | 화면에서 읽히지 않음 | 색/소리/간격 조정 |
| agency_fail | 조작권을 빼앗김 | 강제 이동/추적 강도 감소 |
| reward_fail | 속인 보람이 없음 | 정산 사유/후보 태그 강화 |
| lore_overload_fail | 설정이 과다함 | UI 문구 압축, 로그로 이동 |
| unlock_confusion_fail | 캐릭터 해금처럼 보임 | RETURN-05/R02 노출 축소 |

## 13. 최종 결론

R03 0.3 구현 플레이테스트의 핵심 질문은 하나다.

```text
유저가 "왜 자꾸 귀찮게 해"가 아니라,
"이 멍청하게 성실한 시스템을 또 속일 수 있겠는데"라고 느끼는가?
```

그 느낌이 나오면 R03은 제작 이관 성공이다.
안 나오면 수치보다 먼저 전조, 역이용, 정산 피드백을 고친다.
