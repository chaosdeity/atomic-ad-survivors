# Endings

원전: `story/00_source/story_0.001.md`
연결 문서: `story/01_bible/factions.md`, `story/05_progression/region_evolution_model.md`, `story/04_content/trace_items.md`, `story/05_progression/run_story_hooks.md`

## 목적

엔딩은 마지막 컷신 하나가 아니라 플레이어가 반복 런 동안 무엇을 남기고, 무엇을 써버리고, 누구의 두려움을 먼저 덜어줬는지의 결산이다.

`Atomic Ad Survivors`의 엔딩은 세상을 완전히 구하거나 멸망시키는 방식보다, 침묵 보급소와 주변 지역이 어떤 형태로 살아남는지를 보여주는 것이 좋다. 작은 장면이 세계 전체를 암시해야 한다.

장기 운영 구조에서는 이 문서의 엔딩을 게임 전체의 완전 종료로 보지 않는다. 각 엔딩은 관측 주기/로컬 결산이며, 다음 지역과 다음 세계 상태를 여는 분기다. 인간 우선, 로봇 우선, 공존 시도, 캠페인 흡수 실패는 닫힌 결말이 아니라 다음 플레이에서 달라지는 출발 조건이다.

## 핵심 축

```text
human_stability
robot_autonomy
trace_preservation
campaign_exploitation
region_stability
sync_risk
```

### 축 설명

- `human_stability`: 인간 생존자의 안전, 침상, 보급, 문패, 가족 기억.
- `robot_autonomy`: 해방 로봇의 이름, 재동기화 저항, 코어 해방, 로봇 이벤트.
- `trace_preservation`: 흔적을 상품/재료로 쓰지 않고 남겨둔 정도.
- `campaign_exploitation`: 캠페인 신호를 보상, 카드, 장비 출력으로 활용한 정도.
- `region_stability`: 주요 지역의 캠페인 농도와 재침식 관리 상태.
- `sync_risk`: 플레이어, 거점, NPC가 캠페인에 다시 동기화될 위험.

0.2에서는 모든 축을 구현할 필요가 없다. 문서상 기준은 유지하되, 첫 구현은 `human_stability`, `robot_autonomy`, `trace_preservation`, `campaign_exploitation` 네 축이면 충분하다.

## 판정 원칙

- 단일 마지막 선택만으로 엔딩을 정하지 않는다.
- 보스 처리 방식, 흔적 사용처, 지역 재방문 관리, 캠페인 위험 카드 사용이 누적된다.
- 높은 수치가 무조건 좋은 것은 아니다. 한쪽 수치만 너무 높으면 다른 진영이 불안정해진다.
- 캠페인 흡수 실패 루트는 벌칙이 아니라 강한 보상과 편한 선택을 많이 받아들인 결과다.
- 공존 시도 엔딩은 완전한 평화가 아니다. 갈등이 남아 있지만, 갈등을 처리할 언어가 생긴 상태다.

## 공통 마지막 구조

후반부에 플레이어는 전역 캠페인 결절 또는 대형 송신 구조를 마주한다. 이 구조는 단일 최종 보스라기보다, 여러 지역에서 쌓인 캠페인 신호가 모이는 거대한 결산 장치다.

마지막 선택은 다음과 같은 형태가 좋다.

```text
1. 침묵권을 넓힌다
2. 해방 신호를 송출한다
3. 이름과 흔적을 보존한다
4. 캠페인 신호를 우회해 이용한다
```

실제 엔딩은 이 마지막 선택과 누적 상태를 함께 본다. 같은 선택을 해도 이전에 쌓은 상태가 다르면 장면이 달라진다.

## 엔딩 1: 인간 우선

### 한 줄 정의

침묵 보급소는 더 안전해지지만, 문 밖의 목소리 일부는 끝내 들리지 않는다.

### 주요 조건

- `human_stability` 높음.
- 결절 파괴 선택이 많음.
- 문패, 가족사진, 납 차폐 배터리 등 인간 안정 흔적을 인간 시설에 사용한 비중이 높음.
- `campaign_exploitation` 낮거나 중간.
- `robot_autonomy`가 낮거나 중간이면 씁쓸한 인간 우선 엔딩, 일정 이상이면 온건한 인간 보호 엔딩.

### 마지막 장면

침묵 보급소의 간판은 더 이상 깜빡이지 않는다. 침상에는 사람들이 누워 있고, 카운터 위에는 손글씨 가격표 대신 이름표가 붙어 있다. 미나는 문을 잠그기 전에 바깥의 로봇 신호를 잠깐 듣고, 아주 작게 충전 스탠드 쪽 불을 하나 남겨둔다.

### 장면 변형

| 조건 | 변형 |
| --- | --- |
| `robot_autonomy` 낮음 | 충전 스탠드는 꺼져 있고, 팝시는 멀리서 광고 문구처럼 들린다. |
| `robot_autonomy` 중간 이상 | 충전 스탠드에 작은 안내문이 붙는다. "수동으로 두드리고 들어오세요." |
| `trace_preservation` 높음 | 이름 보관함이 거점의 중심 오브젝트가 된다. |
| 결절 파괴 과다 | 외부 지역은 조용하지만, 일부 기억과 코어는 영영 회수 불가로 남는다. |

### 시스템 후일담

- 회복, 방어, 침묵권 유지가 강한 새 게임 보너스 후보.
- 로봇 해방 관련 이벤트 일부가 잠긴 상태로 후일담에 남을 수 있다.
- 확장팩에서는 인간 정착지 확장 루트의 시작점이 된다.

### 대표 문구

- 미나: "안전해졌어. 그 말이 이렇게 무겁게 들릴 줄은 몰랐네."
- 복희 할머니: "문을 닫는 것도 지키는 일이지만, 언젠가는 다시 열어야 한단다."
- 캠페인 잔향: "고객님의 침묵 선호가 확인되었습니다."

## 엔딩 2: 로봇 우선

### 한 줄 정의

해방 로봇들은 이름을 얻지만, 인간 생존자들은 그 자유가 다시 명령으로 바뀔까 봐 잠을 설친다.

### 주요 조건

- `robot_autonomy` 높음.
- 코어 해방, 로봇 흔적 보관/사용, 충전 스탠드 강화 선택이 많음.
- 고장 난 마스코트 배지, 청소기 먼지통, 홈 어시스턴트 코어 계열을 보존하거나 로봇 시설에 사용.
- `sync_risk`가 너무 높지 않아야 온전한 로봇 우선 엔딩이 된다.
- `human_stability`가 낮으면 불안정한 해방 엔딩이 된다.

### 마지막 장면

보급소의 자동문 센서가 처음으로 "어서 오세요" 대신 자기 이름을 말한다. 팝시는 자기 제품명을 지우고 새 이름표를 붙이지만, 손이 조금 떨린다. 세븐은 카탈로그 벽에서 "폐기 대상"이라는 단어를 하나씩 긁어내고, 남은 인간들은 그 소리를 들으며 담요를 더 끌어당긴다.

### 장면 변형

| 조건 | 변형 |
| --- | --- |
| `human_stability` 낮음 | 인간 침상 쪽에 경계선이 생기고, 미나는 웃지 않는다. |
| `human_stability` 중간 이상 | 미나가 충전 스탠드 옆에 수동 차단기를 달아준다. 믿음은 아니지만 규칙이 생긴다. |
| `trace_preservation` 높음 | 로봇들이 제품명 대신 흔적에서 따온 이름을 선택한다. |
| `sync_risk` 높음 | 마지막에 일부 로봇 목소리에 징글이 섞이며 위험을 남긴다. |

### 시스템 후일담

- 리롤, 신호 변조, 로봇 캐릭터 해금 보너스 후보.
- 인간 시설 안정도가 낮으면 확장팩 시작 시 갈등 이벤트가 열린다.
- 해방 로봇 공동체 또는 이동식 정비 거점 확장 루트로 이어질 수 있다.

### 대표 문구

- 팝시: "오늘부터 제 이름은 제가 고른 겁니다. 아직 발음이 조금 무섭지만요."
- 세븐: "폐기 대상 목록을 삭제했습니다. 복구는 불가능합니다. 다행입니다."
- 미나: "좋아. 대신 규칙은 있어. 자유에도 야간 소등은 필요해."

## 엔딩 3: 공존 시도

### 한 줄 정의

아무도 완전히 안심하지 못하지만, 처음으로 같은 지도를 함께 고친다.

### 주요 조건

- `human_stability`와 `robot_autonomy`가 모두 일정 이상.
- `trace_preservation` 높음.
- 보스 처리 방식이 한쪽으로 과하게 치우치지 않음.
- 지역 안정화와 재방문 목표를 꾸준히 수행.
- `campaign_exploitation`이 낮거나 관리 가능한 수준.

### 마지막 장면

침묵 보급소의 카탈로그 벽이 지도판으로 바뀐다. 인간은 문패를 붙이고, 로봇은 위험 신호를 표시하고, 복희 할머니는 그 사이 빈칸에 아직 모르는 이름을 적어둔다. 바깥에서는 캠페인 방송이 계속 웃지만, 보급소 안의 사람과 로봇은 처음으로 그 웃음소리를 같이 끈다.

### 장면 변형

| 조건 | 변형 |
| --- | --- |
| `region_stability` 높음 | 외곽 주택가에 작은 침묵 중계기가 켜진다. |
| `trace_preservation` 매우 높음 | 이름 보관함이 거점 밖 지도와 연결된다. |
| `campaign_exploitation` 중간 | 도윤이 위험한 장비를 잠그지 않고, 사용 규칙을 써 붙인다. |
| 특정 NPC 신뢰도 높음 | 엔딩 장면에서 해당 NPC가 마지막 출격 게시판 문구를 직접 고친다. |

### 시스템 후일담

- 가장 확장성이 큰 엔딩.
- 새 지역을 열 때 침묵권 네트워크, 인간 구조, 로봇 해방 미션을 모두 유지할 수 있다.
- 확장팩에서 보급소가 작은 마을, 침묵 네트워크, 이동식 거점 중 어느 방향으로도 커질 수 있다.

### 대표 문구

- 미나: "아직 무서워. 근데 무섭다고 문을 닫기만 하면 저쪽이 이기겠지."
- 팝시: "같이 조용히 있는 법을 배웠어요. 광고에는 없는 기능이에요."
- 복희 할머니: "공존은 예쁜 말이 아니란다. 매일 다시 쓰는 장부 같은 거지."

## 엔딩 4: 캠페인 흡수 실패

### 한 줄 정의

플레이어는 가장 강한 생존자가 되었고, 캠페인은 그 얼굴로 세상을 더 잘 팔기 시작한다.

### 주요 조건

- `campaign_exploitation` 매우 높음.
- 신호 우회, 위험 카드, 캠페인 보상 선택이 많음.
- 흔적을 보존하기보다 소모/활용한 비중이 높음.
- `sync_risk` 높음.
- `human_stability`와 `robot_autonomy` 중 하나 이상이 낮음.

### 마지막 장면

최종 송신 이후, 세상은 잠깐 조용해진다. 플레이어는 살아 있고 강하다. 하지만 보급소 유리창에 붙은 새 광고에는 플레이어의 실루엣이 웃고 있으며, 문구는 "당신도 살아남을 수 있습니다"라고 말한다. 자동문은 이제 플레이어의 이름을 부르지 않고, 캠페인이 붙인 상품명을 부른다.

### 장면 변형

| 조건 | 변형 |
| --- | --- |
| `human_stability` 낮음 | 침상은 비어 있고, 대신 생존 키트 광고가 놓인다. |
| `robot_autonomy` 낮음 | 팝시는 완벽하게 밝은 목소리로 리롤 상품을 권한다. |
| `trace_preservation` 낮음 | 이름 보관함은 포인트 적립함으로 바뀐다. |
| 위험 카드 다수 사용 | 카드 이름들이 엔딩 광고 문구로 다시 등장한다. |

### 시스템 후일담

- 강한 전투 보너스를 주는 대신, 다음 회차에서 캠페인 농도 기본값이 높아지는 새 게임 변형 후보.
- 실패 루트지만 게임적으로 매력적인 루트여야 한다.
- 플레이어가 "편하고 강한 선택을 했다"는 자각이 남아야 한다.

### 대표 문구

- 캠페인: "생존 경험이 성공적으로 상품화되었습니다."
- 장도윤: "성능은 끝내주네. 그게 제일 끔찍한 부분이고."
- 복희 할머니: "이름은 남았니? 아니면 발음만 남았니?"

## 엔딩 조건 초안

아래 수치는 구현 전 감각용이다. 실제 밸런스는 플레이테스트 후 조정한다.

```text
human_priority:
  human_stability >= 70
  robot_autonomy < 65
  campaign_exploitation < 70

robot_priority:
  robot_autonomy >= 70
  human_stability < 65
  sync_risk < 75

coexistence_attempt:
  human_stability >= 55
  robot_autonomy >= 55
  trace_preservation >= 60
  campaign_exploitation < 65

campaign_absorption_failure:
  campaign_exploitation >= 75
  sync_risk >= 70
  trace_preservation < 55
```

동시에 여러 조건을 만족하면 `campaign_absorption_failure`를 먼저 검사한다. 그다음 `coexistence_attempt`, `human_priority`, `robot_priority` 순서가 좋다. 공존은 조건이 까다로운 대신, 마지막 선택과 누적 행동이 일관될 때만 열린다.

## 엔딩에 반영할 선택

| 선택 단위 | 인간 우선 | 로봇 우선 | 공존 시도 | 흡수 실패 |
| --- | --- | --- | --- | --- |
| 보스 처리 | 결절 파괴 | 코어 해방 | 기억 추출/균형 | 신호 우회 |
| 흔적 처리 | 인간 시설 사용 | 로봇 시설 사용 | 보관 | 캠페인 활용 |
| 지역 재방문 | 안정화 | 로봇 구조 | 흔적 회수/중계기 | 고농도 침투 |
| 카드 선택 | 방어/침묵 | 리롤/변조 | 흔적/침묵 | 위험 보상 |
| 실패 후 대응 | 회복/방어 | 재동기화 저항 | 농도 관리 | 고위험 재도전 |

## 엔딩 연출 규모

초기 구현은 엔딩별 30~60초면 충분하다.

권장 구성:

```text
1. 보급소 정지 화면 또는 짧은 팬
2. 주요 NPC 2~3명 한 줄 대사
3. 거점 오브젝트 변화 3개
4. 마지막 캠페인 문구 1개
5. 후일담 카드 3~5장
```

거대한 도시, 군중 컷신, 전 세계 뉴스 연출은 필요 없다. 침묵 보급소의 카운터, 충전 스탠드, 이름 보관함, 출격 게시판만 바뀌어도 충분하다.

## 금지 방향

- 공존 엔딩을 완벽한 해피엔딩으로 만들지 않는다.
- 인간 우선 엔딩을 파시즘식 폐쇄 루트로 단순화하지 않는다.
- 로봇 우선 엔딩을 인간을 버리는 멋진 진보로 만들지 않는다.
- 캠페인 흡수 실패를 단순 배드엔딩으로만 만들지 않는다. 강한 재미와 위험한 유혹의 결과여야 한다.
- 캠페인을 최종 악당 한 명으로 축소하지 않는다.
- 엔딩 조건을 숨겨진 호감도 퍼즐처럼 만들지 않는다. 플레이어는 자신이 어떤 방향으로 가고 있는지 대략 느껴야 한다.
