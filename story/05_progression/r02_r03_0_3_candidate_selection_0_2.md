## 0. 결론

0.3의 주 후보는 `R03 물류 회수 벨트 / E01_C03 반품 회수 벨트`로 잡는다.

이유:

| 기준 | R03 판정 |
|---|---|
| R01 이후 즉시 신선도 | 주택/가족 다음에 컨테이너, 레일, 라벨, 보관함이 나오면 화면과 조작 감각이 바로 달라진다. |
| 윤서 결합 | 윤서의 회수/정산/보류 정체성을 설명이 아니라 런 기믹으로 검산한다. |
| 런 재미 | 송장 장판, 분류 레일, 보관함, 반송 드론이 액션 RPG 오브젝트로 바로 바뀐다. |
| 반복 출격 | 같은 루트 반복 시 배송 경로가 학습되고, 지름길 차단/보관 기한/압축 라인이 자연스럽게 생긴다. |
| 0.3 범위 관리 | R03-L01~L02만으로도 새 재미를 만들 수 있다. R03-L05 최종 보스는 P2로 남겨도 된다. |

R02는 버리지 않는다.

R02는 `진료태그`, `퇴원 불가`, `환자 등록`, `구출하면 위험한 인간`이라는 강한 아이러니가 있다. 다만 0.3 주 후보로 전면화하면 회복/보험/환자 윤리 UI를 동시에 설계해야 해서 톤과 구현 부담이 커진다. R02는 0.3에서 원격 신호, 보급소 대화, 진료태그 압박, HOLD-04 U1~U2 씨앗으로 먼저 넣는 편이 안전하다.

## 2. 0.3 주 후보: R03 물류 회수 벨트

### 2.1 한 줄 목표

```text
R03은 윤서가 쓰는 회수/보류 기술이 어디서 왔는지 설명하지 않고, 유저가 직접 라벨과 경로를 비틀며 느끼게 하는 지역이다.
```

R03은 "빠른 배송 농담 지역"이 아니다.

R03의 핵심은 이것이다.

```text
목적지가 없으면 사람도 보관 상태가 된다.
완료 판정은 구조일 수도 있고, 너무 정확한 폐기일 수도 있다.
```

### 2.2 0.3에서 열 범위

0.3에서는 R03 전체를 열지 않는다.

| 범위 | 포함 여부 | 이유 |
|---|---|---|
| R03-L01 반품 접수 야드 | 포함 | 윤서가 반품 라벨/오배송/수취인 오류를 읽는 첫 지역. |
| R03-L02 자동 분류장 | 포함 | 0.3의 핵심 런 기믹. 벨트, 색상 라벨, 분류 게이트를 테스트한다. |
| R03-L03 보증 심사 창고 | 조건부 포함 | PATCH/R04 가교와 보증 부품 씨앗. 0.3 범위가 커지면 포함한다. |
| R03-L04 파쇄 전 대기라인 | 제외 | 시간 제한/압축 대기/인간 약탈 흔적은 P2 압박으로 남긴다. |
| R03-L05 최종 반품 심사실 | 제외 | 최종 반품 심사 보류관 직접 대면과 윤서 핵심 보스는 P2로 남긴다. |

0.3의 보스는 R03-L05 최종 보스가 아니다.

0.3에는 아래 중 하나만 둔다.

| 타입 | 설명 | 권장 |
|---|---|---|
| 미니 절차 | 자동 분류장 경로 심사. 보스라기보다 중간 약관 테스트. | 1순위 |
| 엘리트 오브젝트 | 반송 드론/분류 게이트/라벨 프린터 복합전. | 2순위 |
| 최종 보스 예고 | 최종 반품 심사 보류관의 서명 없는 화면, 보류 도장, 음성 잔향. | 필수 씨앗 |

### 2.3 0.3 핵심 재미

| 재미 | 구현 감각 | 유저가 느낄 것 |
|---|---|---|
| 라벨 부착 | 특정 장판이나 적 공격이 윤서/적/오브젝트에 배송 상태를 붙인다. | "아, 상태가 붙으면 경로가 바뀐다." |
| 분류 레일 | 이동 방향이 바뀌는 벨트, 느려지는 보관 구역, 빠르게 밀리는 출고 라인. | "맵이 적처럼 일한다." |
| 보관함 선택 | 열면 보상 후보, 안 열면 다음 런 보관 상태 유지. | "지금 먹을까, 남길까." |
| 반송 유도 | 반송 드론을 적에게 유도하거나, 반송 판정을 카드/태그로 비튼다. | "시스템을 때리는 게 아니라 시스템에 접수시킨다." |
| 회수선 압박 | 오래 머물면 보관 기한이 줄고, 빠른 회수는 압축 라인을 자극한다. | "빨리 끝내는 것도 정답이 아니네." |

## 4. 0.3 제작 범위 제안

### 4.1 0.3 핵심 슬라이스

```text
R08 침묵 보급소
-> R01 재방문 반응
-> R03-L01 반품 접수 야드
-> R03-L02 자동 분류장
-> R03 중간 절차/미니 이벤트

```

R01 이후 유저가 "이 게임은 지역마다 조작과 정산 감각이 달라진다"고 느끼게 한다.

## 5. R03 0.3 세부 기획 기준

### 5.1 지역 구조

| 구역 | 역할 | 핵심 오브젝트 |
|---|---|---|
| 접수 야드 | 첫 진입, 라벨/보관함 학습 | 컨테이너, 접수 키오스크, 수동 보류 스탬프 |
| 반품 라벨 통로 | 상태 이상/경로 보정 학습 | 송장 프린터, 라벨 장판, 반송 안내판 |
| 자동 분류장 | 메인 전투 기믹 | 분류 레일, 색상 게이트, 반송 드론 |
| 미수령 보관 구역 | 보상 선택/다음 런 잔향 | 보관함, 이름 조각, 보관 기한 표시 |
| 보증 창고 입구 | R04/PATCH 씨앗 | 보증 스티커, 부품 케이지, 정품 확인 로그 |

### 5.2 정산 항목

R03은 보급태그와 통행태그를 주 태그로 쓴다.

| 정산 항목 | 의미 | UI 문구 방향 |
|---|---|---|
| 보급태그 후보 | 보관함, 미수령 물자, 반품 회수 보상 | "보관함 회수로 보급태그 후보가 올라왔습니다." |
| 통행태그 후보 | 분류 레일 우회, 출고 게이트 통과 | "분류 경로 우회로 통행태그 후보가 생겼습니다." |
| 수신태그 후보 | 송장 오류, 안내 방송, 목적지 로그 | R03 핵심은 아니지만 후속 신호로 제한 |
| 흔적 | 이름 조각, 목적지 누락 로그, 보류 도장 | 메타/대사/캐릭터 반응에 사용 |

금지:

```text
R03 전용 신규 태그를 늘리지 않는다.
반품 라벨은 태그가 아니라 상태/오브젝트/문구로 둔다.
```

### 5.3 카드/빌드 연결

| 카드 방향 | R03 효과 |
|---|---|
| 보류 카드 | 라벨 상태 지속을 멈추거나 보관 기한을 늦춘다. |
| 경로 카드 | 분류 레일 방향을 짧게 비튼다. |
| 반송 카드 | 표식 적에게 반송 드론을 유도한다. |
| 정산 카드 | 보급태그/통행태그 후보 승인률을 조정한다. |
| 위험 보상 카드 | 압축 경고선 안에서 버티면 흔적/후보 보상이 오른다. |

카드는 R03을 쉽게 만드는 것이 아니라, 약관을 비틀 방법을 늘린다.

### 5.4 보급소 후폭풍

| R03 행동 | 보급소 변화 |
|---|---|
| 보관함을 많이 열었다 | 보급소에 미수령 물품 더미가 생기고, 미나가 보관 기한을 걱정한다. |
| 보관함을 남겼다 | 다음 출격 게시판에 남겨둔 이름/목적지 조각이 뜬다. |
| 반송 드론을 악용했다 | 세븐이 "시스템이 그 실수를 기억한다"고 경고한다. |
| 보증 부품을 회수했다 | PATCH/R04 관련 대사가 1단계 열린다. |
| 압축 라인을 오래 건드렸다 | R03-L04 파쇄 전 대기라인 원격 경고가 열린다. |

0.3 주 후보:

1. R01 이후 유저가 가장 빨리 "다른 지역은 다른 게임 감각"이라고 느낀다.

3. RETURN-05, PATCH, R02, R04로 이어지는 씨앗을 한 번에 깔 수 있다.
4. R02의 강한 아이러니를 소모하지 않고 다음 확장 카드로 남길 수 있다.
