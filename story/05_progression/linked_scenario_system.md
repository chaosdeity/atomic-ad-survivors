# Linked Scenario System

상태: 0.001 확장 기준
연결 문서: `story_trigger_schema.md`, `local_mystery_layers.md`, `local_ecology_roadmap.md`, `yunseo.md`

## 목적

연쇄 시나리오는 긴 퀘스트 줄이 아니다. 플레이어가 로컬을 반복 방문하고, 다른 캐릭터를 쓰고, 흔적을 보관하거나 소모하고, NPC와 관계를 바꾸는 과정에서 여러 조각이 자연스럽게 연결되는 이야기 묶음이다.

이 구조의 목표는 장기 플레이 유저가 “내가 한 행동 때문에 세계가 이상하게 이어지고 있다”는 재미를 느끼게 하는 것이다. 모든 유저가 모든 조각을 봐야 하는 것은 아니다. 오히려 놓친 조각이 있어야 다른 캐릭터, 다른 선택, 다른 관측 주기에서 새롭게 보인다.

## 시나리오 종류

| 종류 | 설명 | 예시 |
| --- | --- | --- |
| 로컬 내부형 | 한 로컬 안에서 반복 방문으로 깊어짐 | 외곽 주택가의 같은 집 |
| 로컬 횡단형 | 두 개 이상의 로컬을 오가며 연결됨 | 주택가 가정용 코어 -> 서비스센터 리콜 로그 |
| NPC 결합형 | NPC 상태와 로컬 발견이 묶임 | 팝시의 말투 오류와 쇼핑몰 마스코트 |
| 흔적 결합형 | 흔적 보관/소모/사용에 따라 갈림 | 불탄 가족사진 보관 여부 |
| 캐릭터 관점형 | 특정 캐릭터로만 다른 의미가 열림 | 윤서의 고객 프로필 빈칸 |
| 악의 개입형 | 세계 법칙이 아니라 누군가의 의도적 행위가 드러남 | 조작된 계약서, 구조 신호 판매 |
| 관측 갱신형 | 업데이트 후 기존 로컬에 새 관측층이 생김 | 위성 송신지 이후 주택가 방송 잔향 |

## 설계 원칙

- 시나리오는 시작점이 여러 개여야 한다.
- 필수 노드는 적고, 선택 노드는 많아야 한다.
- 플레이어가 중간 조각을 놓쳐도 나중에 다른 방식으로 따라잡을 수 있어야 한다.
- 시나리오 보상은 재화보다 세계 해석, NPC 반응, 로컬 상태 변화가 우선이다.
- 같은 시나리오라도 인간/로봇/캠페인 활용 축에 따라 결말이 달라져야 한다.
- 하나의 시나리오가 한 번 끝났다고 로컬이 닫히면 안 된다. 끝난 뒤에도 잔향, 후폭풍, 변형 목표가 남아야 한다.

### 노드 타입

| 타입 | 역할 |
| --- | --- |
| `hint` | 이상함을 암시 |
| `evidence` | 흔적/로그/대사로 증거 제공 |
| `choice` | 보관, 소모, 해방, 파괴 같은 선택 |
| `echo` | 다음 런이나 다른 로컬에서 반응 |
| `conflict` | NPC/진영 갈등 발생 |
| `reveal` | 시나리오 핵심 의미 공개 |
| `aftermath` | 결말 이후 반복 방문 변화 |

## 발견 채널

시나리오는 컷신 하나로 전달하지 않는다. 아래 채널을 섞는다.

| 채널 | 사용법 |
| --- | --- |
| 전투 배너 | 캠페인이 플레이어를 고객처럼 부르는 짧은 문구 |
| 적 이름 | 같은 적이 다른 이름으로 재생산됨 |
| 흔적 아이템 | 보관/소모/사용 선택을 만든다 |
| NPC 한 줄 대사 | 긴 설명 대신 감정 방향을 준다 |
| 출격 게시판 | 로컬 상태 변화를 짧게 보여준다 |
| 보급소 오브젝트 | 기억 보관함, 충전 독, 배급대 변화 |
| 보스 선택지 | 세계 해석을 직접 바꾸는 분기 |
| 캐릭터 반응 | 같은 단서의 의미가 캐릭터별로 달라짐 |

## 진행 방식

연쇄 시나리오는 아래 흐름을 기본으로 한다.

```text
입구 발견
-> 선택 또는 반복 행동
-> 다른 채널에서 반응
-> 로컬/보급소/NPC 상태 변화
-> 선택 분기
-> 결말 상태
-> 재방문 후폭풍
```

중요한 점은 이 흐름이 항상 한 줄로 가면 안 된다는 것이다. `입구 발견`은 여러 개일 수 있고, `반응`은 다른 로컬에서 먼저 나타날 수 있다.

## 예시 1: 같은 집의 이름

### 개요

외곽 주택가에서 같은 구조의 집이 여러 번 나타난다. 처음에는 단순 타일 재사용처럼 보이지만, 반복 방문과 흔적 회수로 캠페인이 “완벽한 가족 주택”을 복제하고 있다는 사실이 드러난다.

### 정보

```text
id = "r01_same_house_wrong_name"
type = "local_internal"
scope = "local:r01_suburb"
related_characters = ["pc_yunseo_001"]
related_npcs = ["mina", "bokhee"]
related_traces = ["trace_burnt_family_photo_001", "trace_nameplate_001"]
```

### 입구

| 입구 | 조건 | 문구 |
| --- | --- | --- |
| 반복 방문 | `consecutive_local_runs.r01_suburb >= 3` | "단골 고객님, 같은 행복을 또 찾으셨군요!" |
| 흔적 회수 | 불탄 가족사진 회수 | "사진 속 집 번호가 이번에도 같다." |
| 윤서 관점 | 윤서로 모델하우스 결절 재방문 | "윤서는 이 현관문을 처음 보는 게 아니라는 느낌을 받는다." |

### 분기

| 선택 | 결과 |
| --- | --- |
| 문패 보관 | 흔적 보존 증가, 복희 대사 개방 |
| 문패 소모 | 장비 보정, 미나가 불편해함 |
| 모델하우스 결절 파괴 | 주택 복제 빈도 감소, 캠페인 농도 하락 |
| 기억 추출 | 가족사진 잔향 증가, 윤서 전용 대사 개방 |

### 후폭풍

- 재방문 시 일부 집 문패가 손글씨로 바뀐다.
- 캠페인은 더 밝은 문구로 같은 집을 팔려고 한다.
- 복희가 “집이 아니라 주소를 팔고 있구나”라고 말한다.

## 예시 2: 반품된 마스코트

### 개요

폐쇼핑몰에서 버려진 마스코트 부품을 발견하면, 팝시-제로의 말투 오류와 연결된다. 쇼핑몰의 마스코트들은 단순 적이 아니라 “고객 응대 자아”와 “광고 대본”이 분리되지 않은 로봇이다.

### 정보

```text
id = "mall_returned_mascot"
type = "npc_trace_cross_local"
scope = "local:r02_mall/npc:popsy_zero"
related_locals = ["r02_mall", "r03_robot_service"]
related_npcs = ["popsy_zero", "jang_doyun"]
related_traces = ["trace_mascot_badge_001"]
```

### 입구

| 입구 | 조건 | 문구 |
| --- | --- | --- |
| 쇼핑몰 흔적 | 마스코트 배지 회수 | "배지 뒷면에 반품 사유가 적혀 있다. 말투 불량." |
| 팝시 대사 | 팝시 신뢰 1 이상 | "방금 말은 제 말이었나요, 상품 설명이었나요?" |
| 서비스센터 로그 | 리콜 로그 발견 | "응대형 제품군은 자율 발화 위험으로 회수됨." |

### 분기

| 선택 | 결과 |
| --- | --- |
| 배지 보관 | 팝시 자율성 대사 증가 |
| 배지 장비화 | 마스코트 계열 적 피해 증가, 팝시가 씁쓸해함 |
| 리콜 로그 삭제 | 로봇 자율성 상승, 일부 진실 단서 손실 |
| 리콜 로그 보관 | 로봇 세계 미스터리 증가, 서비스센터 이벤트 개방 |

### 후폭풍

- 쇼핑몰 마스코트 적 일부가 공격 전 짧게 자기 이름을 말하려 한다.
- 팝시가 전투 후 “오늘의 추천 멘트”를 지우려는 애니메이션을 갖는다.
- 서비스센터에서 마스코트 계열 전용 리콜 엘리트가 등장할 수 있다.

## 예시 3: 배송되지 않은 사람

### 개요

자동 물류 허브와 외곽 주택가, 침묵 보급소를 잇는 시나리오다. 캠페인 물류망은 물건만 배송한 것이 아니라 사람, 로봇, 이름표, 피난 기록까지 배송 대상으로 처리했다.

### 정보

```text
id = "undelivered_person"
type = "cross_local_population"
scope = "global"
related_locals = ["r01_suburb", "r06_logistics_hub", "hub_silence_outpost"]
related_npcs = ["mina", "seven", "bokhee"]
related_characters = ["pc_yunseo_001"]
```

### 입구

| 입구 | 조건 | 문구 |
| --- | --- | --- |
| 이름표 회수 | 윤서의 이름표 계열 흔적 보관 | "배송 상태: 수취인 불명." |
| 물류 허브 조사 | 고객 데이터 터미널 발견 | "사람 이름 옆에 포장 규격이 붙어 있다." |
| 세븐 분석 | 세븐 신뢰 2 이상 | "사람과 물품의 필드 구분이 없습니다." |

### 분기

| 선택 | 결과 |
| --- | --- |
| 고객 데이터 삭제 | 이름 보존 상승, 추적 단서 일부 손실 |
| 배송 경로 우회 | 보급 증가, 캠페인 추적 상승 |
| 미배송 목록 보관 | 인간 구조 이벤트 개방, 보급소 부담 증가 |
| 로봇 배송망에 공유 | 로봇 구조 증가, 인간 생존자 불안 증가 |

### 후폭풍

- 보급소 인구가 늘 수 있지만, 식량/침상 압력이 생긴다.
- 물류 허브 재방문 시 “반품 불가 인원” 적이 나타난다.
- 윤서는 이름표의 주인이 살아 있는지 더 혼란스러워진다.

## 예시 4: 좋은 소식만 남은 방송

### 개요

방송국은 플레이어의 선택을 좋은 소식으로 편집한다. 인간을 구해도, 로봇을 해방해도, 보스를 파괴해도 캠페인은 그것을 “고객 만족 사례”로 바꾸려 한다.

### 정보

```text
id = "only_good_news"
type = "cross_local_campaign_editorial"
scope = "local:r04_broadcast/global"
related_locals = ["r04_broadcast", "r01_suburb", "r02_mall"]
related_npcs = ["jang_doyun", "seven"]
```

### 입구

| 입구 | 조건 | 문구 |
| --- | --- | --- |
| 송출 기록 3 해석 | 송출관 접근 절차 확정 | "이 기록은 좌표가 아니라 편집 순서다." |
| 위험 카드 반복 | `campaign_risk_accept_count >= 3` | "우수 반응 고객님의 활약을 소개합니다!" |
| 보스 파괴 반복 | `same_boss_choice_streak.destroy_node >= 2` | "불필요한 과거를 시원하게 철거했습니다!" |

### 분기

| 선택 | 결과 |
| --- | --- |
| 진실 송출 | 인간/로봇 신뢰 변동, 캠페인 반격 증가 |
| 송출 차단 | 캠페인 농도 일부 감소, 일부 단서가 늦게 열림 |
| 플레이어 기록 삭제 | 흡수 위험 감소, 장기 보상 일부 감소 |
| 기록 역이용 | 강한 카드 해금, 캠페인 활용/동기화 위험 상승 |

### 후폭풍

- 다른 로컬 배너가 플레이어의 과거 선택을 광고 문구로 인용한다.
- 세븐은 “기록 삭제와 기억 보존은 다릅니다”라고 말한다.
- 도윤은 방송국 장비를 탐내지만, 윤서는 쉽게 동의하지 않는다.

## 예시 5: 조작된 구조 신호

### 개요

세계 자체가 악인 것은 아니지만, 그 안에서 악의를 가진 존재는 있다. 누군가가 구조 신호를 조작해 인간 생존자와 해방 로봇을 특정 로컬로 유인하고 있다.

이 시나리오는 진짜 악의 첫 씨앗이다. 캠페인 환경의 자동 반응이 아니라, 누군가가 환경을 이용한다.

### 정보

```text
id = "faked_rescue_signal"
type = "malice_actor"
scope = "global"
related_locals = ["r01_suburb", "r04_broadcast", "r06_logistics_hub"]
related_npcs = ["mina", "seven"]
related_traces = ["trace_silent_radio_001", "trace_nameless_receipt_001"]
```

### 입구

| 입구 | 조건 | 문구 |
| --- | --- | --- |
| 무음 라디오 회수 | `trace_silent_radio_001` 보관 | "라디오는 꺼져 있는데, 구조 신호 기록은 남아 있다." |
| 인간 구조 실패 | 구조 목표 실패 2회 이상 | "신호가 있던 곳에는 늘 광고판이 먼저 도착해 있었다." |
| 방송국 편집실 | 편집 로그 발견 | "구조 요청의 발신 시간이 사후에 수정되어 있다." |

### 분기

| 선택 | 결과 |
| --- | --- |
| 신호 공개 | 인간 안정 하락 가능, 추적 시나리오 개방 |
| 조용히 추적 | 미나 신뢰 유지, 악의 개입층 지연 개방 |
| 로봇 네트워크에 공유 | 로봇 자율성 상승, 인간 불신 상승 |
| 캠페인 역이용 | 빠른 추적, 흡수 위험 상승 |

### 후폭풍

- 침묵 보급소가 더 안전한 곳이 아니라 더 눈에 띄는 곳이 될 수 있다.
- 인간 생존자 중 누군가가 로봇을 의심한다.
- 세븐은 조작 주체가 인간인지 로봇인지 캠페인 파생체인지 확정하지 못한다.

## 예시 6: 윤서의 빈 고객란

### 개요

윤서는 캠페인이 고객 프로필로 완성하지 못한 인간이다. 하지만 캠페인은 윤서를 포기하지 않고, 다른 로컬에서 얻은 가족, 반품, 배송, 방송 데이터를 조합해 윤서의 빈칸을 채우려 한다.

### 정보

```text
id = "yunseo_blank_customer_profile"
type = "character_perspective"
scope = "character:pc_yunseo_001/global"
related_locals = ["r01_suburb", "r02_mall", "r04_broadcast", "r06_logistics_hub"]
related_npcs = ["mina", "jang_doyun", "bokhee"]
related_traces = ["trace_nameplate_001", "trace_nameless_receipt_001"]
```

### 입구

| 입구 | 조건 | 문구 |
| --- | --- | --- |
| 첫 강제 회수 | `forced_recovery_reason = signal_overload` | "방금... 내 이름을 부르려던 거야?" |
| 주택가 재방문 | 윤서로 주택가 2회 이상 | "윤서의 고객 프로필은 아직 완성되지 않았다." |
| 이름 없는 영수증 | 영수증 흔적 보관 | "구매자는 비어 있는데, 추천 대상은 윤서로 되어 있다." |
| 방송국 편집 | 플레이어 기록이 방송됨 | "윤서형 생존 패키지 출시 준비 중." |

### 분기

| 선택 | 결과 |
| --- | --- |
| 이름표를 계속 들고 감 | 윤서 전용 저항 대사, 흡수 위험 낮음 |
| 이름표를 보급소에 맡김 | 보급소 기억 보존 강화, 윤서 개인 불안 증가 |
| 캠페인 추천을 역이용 | 강한 윤서 카드 해금, 프로필 완성 위험 |
| 프로필 기록 삭제 | 단기 안전, 일부 송출 기록 해석 지연 |

### 후폭풍

- 윤서로 특정 로컬에 재방문하면 캠페인이 다른 가족/직업/고객 역할을 제안한다.
- 미나는 윤서의 이름 확인을 습관처럼 묻는다.
- 복희는 이름표를 맡길 준비가 되었는지 묻는다.
- 실패 루트에서는 윤서형 생존 광고가 등장한다.

## 시나리오 결말 타입

| 결말 | 의미 | 후속 사용 |
| --- | --- | --- |
| 안정화 | 로컬 위험 일부 감소, 단서 일부 잠김 | 안전하지만 미지 접근이 늦어짐 |
| 보존 | 흔적과 이름을 살림 | 기억/공존 루트 강화 |
| 해방 | 로봇/자동화 자율성 상승 | 로봇 루트와 리콜 위험 동시 증가 |
| 역이용 | 캠페인 출력 활용 | 강한 보상, 흡수 위험 상승 |
| 은폐 | 갈등을 늦춤 | 나중에 더 큰 충돌 가능 |
| 폭로 | 진실 공개 | 신뢰 변동, 반격 증가 |

## 0.2 최소 적용

0.2에서는 모든 연쇄 시나리오를 구현하지 않는다. 대신 아래 3개를 문서와 데이터상으로 준비한다.

| 우선 | 시나리오 | 0.2에서 보일 것 |
| --- | --- | --- |
| 1 | 같은 집의 이름 | 주택가 반복 방문, 문패/가족사진, 윤서 반응 |
| 2 | 윤서의 빈 고객란 | 첫 강제 회수, 이름 없는 영수증, 보급소 대사 |
| 3 | 반품된 마스코트 | 팝시 대사와 마스코트 배지 정도만 선행 배치 |

실제 로컬 횡단형 시나리오는 0.3 이후 지역이 추가될 때 열어도 된다. 하지만 0.2 흔적과 대사 안에 씨앗은 심어 둔다.

## 구현 주의

- 연쇄 시나리오는 퀘스트 로그로 전부 보여주지 않아도 된다.
- 유저가 놓친 조각은 `미발견`으로 남기되, 다음 관측 주기나 다른 캐릭터에서 재발견 가능하게 둔다.
- 시나리오 진행은 보상 수치보다 출격 게시판, NPC 한 줄 대사, 적 이름, 배너 문구로 먼저 보여준다.
- 악의 개입형 시나리오는 너무 빨리 범인을 확정하지 않는다.
- 세계 자체를 악당으로 만들지 않는다. 악의는 세계 안에서 환경을 이용하는 별도 행위로 남긴다.
- 캐릭터 전용 시나리오는 다른 캐릭터를 강제하는 장벽이 아니라, 다른 해석을 주는 보너스여야 한다.

## 금지 방향

- 모든 시나리오를 선형 퀘스트로 만들지 않는다.
- 한 로컬에서 발견한 단서가 그 로컬 안에서만 끝나게 하지 않는다.
- NPC를 보상 자판기로 만들지 않는다.
- 흔적 아이템을 단순 재료로만 쓰지 않는다.
- 악역을 하나 만들고 캠페인의 모든 미스터리를 그 악역 탓으로 돌리지 않는다.
- 유저가 스토리를 놓쳤다고 장기 성장이 막히게 하지 않는다.
