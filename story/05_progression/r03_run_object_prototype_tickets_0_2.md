## 0. 공통 원칙

R03 오브젝트는 모두 같은 질문으로 검산한다.

```text
이 오브젝트는 사람을 죽이는가?
아니면 사람과 적과 물자를 배송 상태로 잘못 정리하려 하는가?
```

0.3 기준 답:

```text
R03 오브젝트는 살해 장치가 아니라 판정 장치다.
공격은 HP를 깎을 수 있지만, 표현은 라벨 부착, 위치 보정, 반송 확인, 보관 기한 단축, 경로 재계산이다.
```

공통 금지:

```text
라벨을 신규 태그 자원으로 만들기.
보관함을 단순 보상 상자로 만들기.
보류 스탬프를 만능 해제로 만들기.
반송 드론을 단순 추적 미사일로 만들기.
레일을 답답한 길막으로 만들기.
압축 경고선을 즉사 함정으로 만들기.
R03-L05 보스권을 0.3에서 소비하기.
```

## 2. 공통 상태와 데이터 초안

### 2.1 라벨 상태

라벨은 태그가 아니다.

| 상태 | 내부 키 후보 | 의미 | 일반 UI 표현 |
|---|---|---|---|
| 없음 | `none` | R03 판정 상태 없음 | 표시 없음 |
| 미수령 | `unclaimed` | 아직 확인되지 않음 | 미수령 |
| 반송 대기 | `return_pending` | 되돌려야 할 대상으로 분류 | 반송 대기 |
| 경로 보정 | `route_adjusted` | 레일/게이트 경로 영향 증가 | 경로 보정 |
| 보류 | `held` | 판정 변화 지연 | 보류 |
| 완료 임박 | `closing` | 판정이 곧 닫힘 | 완료 임박 |

| 이벤트 | 발생 조건 | 사용처 |

## 3. R03-OBJ-01 송장 프린터

### 3.1 티켓

```text
ticket_id:
R03-OBJ-01

name:
송장 프린터

priority:
P0

prototype_goal:
전조가 보이는 라벨 장판을 만들고, 플레이어/적/오브젝트에 라벨 상태를 붙인다.
```

### 3.2 최소 기능

| 항목 | 기준 |
|---|---|
| 배치 | R03-L01 반품 접수 야드, 라벨 통로 |
| 전조 | 바닥 사각 윤곽 0.7~1.0초, 짧은 인쇄음 |
| 발동 | 전조 후 라벨 장판 생성 |
| 대상 | 플레이어, 적, 일부 보관함/드론 |
| 결과 | `미수령`, `반송 대기`, `경로 보정` 중 하나 부착 |
| 회피 | 이동으로 피할 수 있어야 함 |
| 역이용 | 적을 장판에 넣으면 적에게 라벨 부착 |

### 3.3 상태

| 상태 | 설명 | 다음 상태 |
|---|---|---|
| idle | 대기 | telegraph |
| telegraph | 송장 윤곽 표시 | active |
| active | 라벨 장판 적용 | cooldown |
| cooldown | 재출력 대기 | idle |
| jammed | 보류 스탬프나 카드로 잠시 멈춤 | cooldown |

### 3.4 정산 연결

| 행동 | 정산 후보 | 문구 |
|---|---|---|
| 적에게 라벨 부착 | 보급태그 후보 보정 | 송장 프린터가 적 웨이브를 잘못 접수했습니다. |
| 라벨을 보류 | 흔적 보존 | 라벨 판정이 보류되어 목적지 누락 로그가 남았습니다. |
| 라벨 상태로 레일 우회 | 통행태그 후보 | 경로 보정 라벨을 역이용해 통행태그 후보가 생겼습니다. |

### 3.5 QA

| 체크 | 통과 기준 |
|---|---|
| 전조 | 480x270 기준 바닥 윤곽이 보인다 |
| 회피 | 첫 방문 기본 이동속도로 피할 수 있다 |
| 역이용 | 적에게 붙이는 성공 사례가 1회 이상 나온다 |
| 가독성 | 라벨 상태명은 2~4글자 수준으로 짧다 |
| 톤 | 문구가 사람을 조롱하지 않는다 |

금지:

```text
프린터를 단순 피해 장판으로 만들기.
라벨 상태를 신규 태그 자원처럼 표시하기.
피할 수 없는 광역 라벨 부착.
```

## 4. R03-OBJ-02 자동 분류 레일

### 4.1 티켓

```text
ticket_id:
R03-OBJ-02

name:
자동 분류 레일

priority:
P0

prototype_goal:
이동 방향 또는 속도 변화를 통해 R03의 공간이 전투 리듬을 바꾸는 감각을 만든다.
```

### 4.2 최소 기능

| 항목 | 기준 |
|---|---|
| 배치 | R03-L02 자동 분류장 중심 |
| 전조 | 레일 방향 화살표, 낮은 모터음, 게이트 색상 |
| 효과 A | 레일 위 이동속도 보정 |
| 효과 B | 지정 방향으로 약하게 밀기 |
| 대상 | 플레이어, 적, 반송 드론 |
| 역이용 | 적/드론을 유리한 게이트로 밀어 넣기 |
| 안전선 | 플레이어 조작을 완전히 빼앗지 않는다 |

### 4.3 상태

| 상태 | 설명 | 비고 |
|---|---|---|
| off | 멈춤 | 튜토리얼/안전 구간 |
| forward | 기본 방향 | 파란 화살표 |
| reverse | 역방향 | 노란 화살표 |
| fast | 빠른 출고 라인 | 위험 보상 |
| slow | 보관 대기 라인 | 압박 완화/드론 유도 |
| locked | 캠페인 학습으로 일부 방향 고정 | 재방문 반응 |

### 4.4 정산 연결

| 행동 | 정산 후보 | 문구 |
|---|---|---|
| 레일 우회 | 통행태그 후보 | 분류 경로 우회로 통행태그 후보가 생겼습니다. |
| 적을 게이트로 유도 | 보급태그 후보 보정 | 자동 분류장이 적 웨이브를 임시 보관 처리했습니다. |
| 역방향 레일에서 흔적 회수 | 흔적 | 역방향 레일 아래 목적지 누락 로그를 회수했습니다. |

### 4.5 QA

| 체크 | 통과 기준 |
|---|---|
| 조작감 | 레일 위에서도 방향 입력이 의미 있다 |
| 가독성 | 모바일 화면에서 화살표 방향을 읽을 수 있다 |
| 리듬 | 레일이 지루한 이동 방해가 아니라 적 유도 도구가 된다 |
| 반복 | 같은 경로 반복 시 locked/reverse 변형이 가능하다 |

금지:

```text
장시간 이동 불능.
즉사 구덩이로 밀어 넣기.
레일 방향이 보이지 않는 상태에서 강제 이동.
```

## 5. R03-OBJ-03 미수령 보관함

### 5.1 티켓

```text
ticket_id:
R03-OBJ-03

name:
미수령 보관함

priority:
P0

prototype_goal:
열면 즉시 후보를 얻고, 남기면 다음 런에 잔향이 남는 선택 오브젝트를 만든다.
```

### 5.2 최소 기능

| 항목 | 기준 |
|---|---|
| 배치 | R03-L01 후반, R03-L02 보관 구역 |
| 상호작용 | 열기 또는 남기기 |

| 남기기 결과 | 다음 런에 같은 번호/이름 조각/목적지 누락 로그 등장 |
| 위험 | 열기 누적 시 미수령 알림/드론 압박 증가 |

### 5.3 상태

| 상태 | 설명 | 다음 런 반응 |
|---|---|---|
| closed | 닫힌 보관함 | 없음 |
| opened | 회수 완료 | 보급소 물자 반응 |
| left | 남김 | 같은 번호 잔향 |
| held | 보류 스탬프로 기한 지연 | 이름 조각 보존 |
| contaminated | 잘못된 라벨과 연결 | 드론/압축 압력 상승 |
| expired_warning | 보관 기한 경고 | R03-L04 원격 경고 |

### 5.4 정산 연결

| 행동 | 정산 후보 | 문구 |
|---|---|---|
| 보관함 열기 | 보급태그 후보 | 미수령 보관함 회수로 보급태그 후보가 올라왔습니다. |
| 출고문 근처 보관함 열기 | 통행태그 후보 | 출고 경로 기록에서 통행태그 후보가 생겼습니다. |
| 보관함 남기기 | 다음 런 잔향 | 미수령 상태가 다음 출격까지 남습니다. |
| 보관함 보류 | 흔적 보존 | 보관 기한이 보류되어 이름 조각이 남았습니다. |

### 5.5 QA

| 체크 | 통과 기준 |
|---|---|
| 선택성 | 열기와 남기기가 둘 다 의미 있다 |
| 보상감 | 열면 좋지만 찝찝한 기록이 따라온다 |
| 잔향 | 남긴 보관함이 다음 런에서 확인된다 |
| UI | 보관함 상태가 너무 긴 문장 없이 읽힌다 |
| 톤 | 사람을 물자 취급하는 시스템을 조롱하고, 사람은 조롱하지 않는다 |

금지:

```text
보관함을 그냥 랜덤 상자로 만들기.
열기가 항상 정답.
남기기가 항상 손해.
보관함 안의 사람/피해자를 가볍게 소비.
```

## 9. 오브젝트 간 조합

### 9.1 1차 조합

| 조합 | 유저가 배우는 것 | 예시 |
