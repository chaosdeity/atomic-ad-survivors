# Environment Response Operations

상태: 0.001 장기 운영 기준
연결 문서: `long_term_story_architecture.md`, `region_evolution_model.md`, `local_ecology_roadmap.md`, `local_mystery_layers.md`, `story_trigger_schema.md`

## 목적

이 문서는 `Atomic Ad Survivors`의 장기 운영을 시즌 리셋이 아니라 환경 반응형 구조로 정의한다.

핵심 문장:

```text
런은 매번 새롭고,
세계는 매번 누적된다.
```

플레이어는 매 런 다른 카드, 전투, 회수물을 경험한다. 그러나 세계 상태는 초기화되지 않는다. 로컬 생태권은 플레이어의 선택, 반복 행동, 보스 처리, 흔적 사용, 캠페인 활용을 기억하고, 이후 업데이트는 그 기억 위에 새 관측층과 변이를 얹는다.

## 검토 결론

환경 반응형 운영은 이 세계관에 매우 잘 맞는다.

캠페인은 악당 조직이 아니라 생태계에 가까우므로, 세계가 한 번의 클리어로 정리되는 것이 오히려 부자연스럽다. 보스는 지역 법칙의 얼굴이거나 결절일 뿐이고, 결절을 처리하면 지역은 사라지는 것이 아니라 다른 방향으로 반응한다.

따라서 업데이트를 `새 시즌 시작`으로 표현하는 것보다, `침묵 보급소가 새 환경 반응을 관측했다`고 표현하는 편이 자연스럽다.

좋은 구조:

```text
기존 선택 유지
-> 로컬 상태 보존
-> 새 관측층 추가
-> 상태별 변이/잔향 분기
-> 공통 콘텐츠와 개인 반응 동시 제공
```

나쁜 구조:

```text
시즌 시작
-> 모든 세계 상태 초기화
-> 같은 지역 다시 클리어
-> 과거 선택은 보상 수치만 남음
```

## 핵심 용어

### 관측층

보급소가 이전에는 읽지 못했던 로컬 생태권의 새 층이다. 새 지역이 아니라 같은 지역의 다른 깊이다.

관측층은 업데이트 콘텐츠를 세계관 안에서 부르는 가장 좋은 용어다. 예를 들어 외곽 주택가에 `무음 라디오 관측층`이 열리면, 맵 전체가 바뀌는 것이 아니라 기존 집, 문패, 가족사진, 라디오 오브젝트가 새 의미를 얻는다.

### 변이

캠페인 생태권이 플레이어 행동이나 로컬 상태에 적응해 만들어낸 새 증상이다.

변이는 몬스터 스킨 변경이 아니라 환경의 학습 결과여야 한다. 차징을 자주 쓴 플레이어에게 차징 방해 엘리트가 생기거나, 캠페인 위험 카드를 많이 고른 플레이어에게 고위험 쿠폰 변이가 등장하는 식이다.

### 잔향

이미 처리한 보스, 흔적, 선택, 구조 실패, 송출 기록이 로컬에 남긴 후유증이다.

잔향은 과거 선택을 되살리는 장치다. 보스를 잡았는데 아무 일도 없었던 것처럼 다시 출현시키는 것이 아니라, 보스 패턴 일부가 환경 이벤트가 되거나, NPC가 이전 선택을 다르게 해석하거나, 출격 게시판 문구가 변하는 방식으로 남긴다.

### 환경 반응

로컬 생태권이 유저의 행동을 읽고 상태를 바꾸는 모든 현상이다.

환경 반응은 보상, 적 구성, NPC 대사, 게시판 문구, 오브젝트 변화, 흔적 출현, 연쇄 시나리오 씨앗으로 표현된다. 중요한 것은 플레이어가 “내가 한 행동 때문에 이 장소가 달라졌다”고 느끼는 것이다.

## 캠페인은 어떻게 기억하는가

캠페인은 인간처럼 기억하지 않는다. 캠페인은 플레이어를 고객 프로필, 경로 데이터, 구매 반응, 회피 패턴, 감정 소재, 실패한 추천 기록으로 저장한다.

기억 방식:

| 기억 방식 | 게임 상태 | 표현 |
| --- | --- | --- |
| 고객 프로필화 | `campaign_exploitation`, `sync_risk` | 캠페인이 플레이어 이름/습관을 광고 문구로 부름 |
| 경로 학습 | `revisit_count`, `consecutive_local_runs` | 같은 집 반복, 단골 고객 배너 |
| 전투 적응 | `adaptation_charge`, `adaptation_auto_attack` | 차징 방해 엘리트, 보호막형 적 |
| 감정 재가공 | `trace_preservation`, `memory_damage` | 가족사진 잔향, 가짜 기억 |
| 위험 선호 분석 | `campaign_risk_accept_count` | 고위험 쿠폰, 회원 전용 혜택 |
| 무시된 신호 재송출 | `ignored_signal_events` | 더 강한 신호 이벤트, 조작된 구조 신호 암시 |

캠페인의 기억은 친절하게 보이지만 무섭다. 플레이어를 미워해서가 아니라, 플레이어를 더 잘 맞춤화하려 하기 때문이다.

## 보급소는 어떻게 관측하는가

침묵 보급소는 세계를 통제하지 못한다. 다만 캠페인 신호가 닿지 않는 짧은 공백을 이용해 변화를 기록한다.

관측 채널:

- 출격 게시판: 새 관측층, 위험 태그, 지역 반응 표시.
- 이름 보관함: 흔적 보존과 사람/로봇 이름 변화 기록.
- 정비 벤치: 판촉 물질, 코어 파편, 신호 변이 분석.
- 충전 스탠드: 해방 로봇과 재동기화 압력 관측.
- 세븐의 기록: 의도와 패턴을 분리하지 못하지만 반복은 잘 기록.
- NPC 짧은 대사: 세계 변화의 인간적 의미를 전달.

보급소는 업데이트를 이렇게 말한다.

```text
새 콘텐츠가 열렸습니다
```

보다:

```text
새 환경 반응이 관측되었습니다
```

가 맞다.

## 내부 해석과 유저 경험

아래 대표 축과 프로필은 개발팀 내부 해석용이다. 유저에게 성향 점수, 프로필명, 트리거 조건을 직접 보여주지 않는다.

유저에게 필요한 것은 분기표가 아니라 현상이다.

```text
내부:
  local_memory_profile = "haunted"

외부:
  창문 가족사진이 오래 남는다.
  라디오가 모르는 가족 목소리를 흉내낸다.
  복희가 사진을 보고 말을 줄인다.
```

정확한 원칙:

```text
공통 자산은 존재한다.
공통 경험은 존재하지 않는다.
```

개발팀은 같은 맵, 적, 이벤트, 흔적을 재사용할 수 있다. 그러나 유저는 자신의 행동과 성향 때문에 같은 콘텐츠가 다르게 반응한다고 느껴야 한다.

## 대표 축 추천

유저 성향 반응을 만들 대표 축은 너무 많으면 안 된다. 장기 운영 기준으로 아래 5개를 핵심 축으로 둔다.

| 축 | 역할 | 0.2 여부 |
| --- | --- | --- |
| `boss_outcome` | 로컬의 큰 미래 방향 결정 | 0.2 필수 |
| `trace_preservation` | 기억/이름/공존/미스터리 층 강화 | 0.2 부분 |
| `campaign_exploitation` | 고위험 보상과 플레이어 모델화 | 장기, 0.2 암시 |
| `revisit_count + behavior_memory` | 세계가 플레이어 습관을 기억 | 0.2 필수 |
| `sync_risk` | 캠페인 흡수/재동기화/고위험 루트 | 장기, 0.2 수치만 가능 |

`human_stability`와 `robot_autonomy`는 중요하지만, 0.2에서는 결과 축으로만 둔다. 장기 구조에서는 보급소 인구와 진영 관계로 확장한다.

### 축 묶기

조합 폭발을 막기 위해 모든 상태를 독립 분기로 만들지 않는다. 아래처럼 큰 반응 묶음으로 합친다.

```text
local_memory_profile:
  quieted
  haunted
  exploited
  neglected
  overread
```

예시:

| 프로필 | 조건 예시 | 표현 |
| --- | --- | --- |
| `quieted` | 결절 파괴, 낮은 캠페인 활용 | 조용한 집, 추적형 엘리트 |
| `haunted` | 기억 추출, 흔적 보존 높음 | 가족사진 잔향, 가짜 목소리 |
| `exploited` | 캠페인 위험 선택 많음 | 회원 전용 혜택, 고위험 쿠폰 |
| `neglected` | 오래 방치, 신호 이벤트 무시 | 재침식, 구조 신호 왜곡 |
| `overread` | 같은 로컬 반복, 윤서 반복 방문 | 단골 고객, 빈 고객란 |

업데이트는 모든 개별 상태에 대응하지 말고, 이 프로필을 기준으로 변형을 제공한다. 단, 프로필명은 UI에 노출하지 않는다.

## 업데이트 표현

개발/운영 용어를 세계관 안에서 바꾸는 후보:

| 일반 용어 | 세계관 표현 후보 |
| --- | --- |
| 업데이트 | 관측 갱신, 보급소 관측 갱신, 침묵 기록 갱신 |
| 시즌 | 관측 주기, 침묵 관측 주기 |
| 이벤트 | 환경 반응, 로컬 증상, 변이 발생 |
| 패치 | 정비 기록 갱신, 송출 보정 |
| 신규 콘텐츠 | 새 관측층, 미확인 잔향, 변이 보고 |
| 핫픽스 | 긴급 정비, 신호 보정 |

권장 기본 표현:

```text
관측 갱신
새 관측층
환경 반응
로컬 변이
잔향 보고
```

## 문구 후보

### 출격 게시판

```text
외곽 주택가에서 새로운 침묵 반응이 관측되었습니다.
같은 지역의 다른 층이 열렸습니다.
이전 선택의 잔향이 주택가 표면으로 올라왔습니다.
보급소 기록과 맞지 않는 구조 신호가 반복됩니다.
조용해진 집들 사이에서 무음 라디오 반응이 증가했습니다.
```

### NPC 대사

미나:

```text
새 지역이 아니야. 더 나쁜 말로 하면, 우리가 이미 건드린 곳이 다시 대답한 거야.
```

도윤:

```text
장비 오류는 아니야. 저쪽이 지난번 우리 선택을 재료로 새 패턴을 만든 거지.
```

세븐:

```text
이건 새 지역이 아닙니다. 같은 지역의 다른 층입니다.
```

팝시:

```text
반품된 물건도 다시 광고에 쓰이나요? 그러면 저도... 아니, 방금 말은 취소예요.
```

복희:

```text
전에 못 들은 소리가 아니야. 그때 우리가 못 들은 척한 소리지.
```

### 캠페인 방송

```text
고객님의 지난 선택을 반영한 맞춤형 행복이 준비되었습니다.
조용한 집을 선호하시는 고객님께 더 조용한 가족을 추천합니다.
기억을 보관하신 고객님을 위한 추억 재생 서비스가 시작됩니다.
반복 방문 고객님께 익숙한 위험을 새 가격으로 제공합니다.
무시된 안내를 더 크게, 더 친절하게 다시 보내드립니다.
```

## 외곽 주택가 장기 반응

기본 상태:

```text
외곽 주택가 = 완벽한 가족 상품 생태권
```

### 결절 파괴 이후

상태:

```text
local_memory_profile = "quieted"
```

반응:

- 일부 집의 광고 방송이 꺼진다.
- 적 수는 줄지만, 플레이어를 찾는 추적형 엘리트가 증가한다.
- 문패 회수와 구조 목표가 늘어난다.
- 이후 업데이트에서 `무음 라디오 관측층`이 열리기 좋다.

문구:

```text
너무 조용한 집들은 가끔, 가장 정확한 목소리로 사람을 부른다.
```

### 기억 추출 이후

상태:

```text
local_memory_profile = "haunted"
```

반응:

- 가족사진 잔향과 창문 실루엣이 늘어난다.
- 흔적 목표와 윤서 전용 대사가 증가한다.
- 가짜 기억형 적이나 보스 잔향 패턴이 생긴다.
- 이후 업데이트에서 라디오가 가족 목소리를 흉내내는 변이가 가능하다.

문구:

```text
사진 속 빈자리가 플레이어를 향해 조금씩 돌아선다.
```

### 문패 보관

반응:

- 이름 보관함에 문패가 붙는다.
- 외곽 주택가 일부 문패가 원래 이름으로 잠깐 깜빡인다.
- 인간 안정도와 흔적 보존도가 오른다.

문구:

```text
문패 하나가 광고 문구보다 늦게 지워졌다.
```

### 가족사진 소모

반응:

- 장비 성능은 좋아지지만, 복희 대사가 차가워진다.
- 주택가에서 가족사진 잔향이 왜곡된다.
- 캠페인은 사진 속 감정을 더 쉽게 광고 소재로 쓴다.

문구:

```text
웃는 얼굴은 남았다. 누구의 얼굴이었는지는 조금 덜 중요해졌다.
```

### 무음 라디오 보관

반응:

- 조작된 구조 신호 씨앗이 열린다.
- 세븐은 반복 패턴을 기록하지만 의도는 판정하지 못한다.
- 윤서는 광고 말투가 아닌 손댄 흔적을 의심한다.

문구:

```text
라디오는 꺼져 있었다. 그런데 누군가 주파수를 고쳐 놓았다.
```

### 이름 없는 영수증 캠페인 활용

반응:

- 보상 또는 위험 카드가 강해진다.
- 윤서의 빈 고객란 시나리오가 빨라진다.
- 캠페인이 윤서를 입주 후보 또는 가족 구성원으로 부르기 시작한다.

문구:

```text
고객 정보가 비어 있습니다. 행복한 가족 정보로 자동 완성하시겠습니까?
```

### 윤서 반복 방문

반응:

- 캠페인이 윤서를 더 정확히 부른다.
- 모델하우스의 가족 구성원 수가 윤서의 파티 상태와 비슷해진다.
- 윤서는 캠페인의 친절한 오류와 누군가의 조작을 구분하려 하지만 항상 맞지는 않는다.

문구:

```text
윤서의 이름은 아직 팔리지 않았다. 그래서 더 자주 불린다.
```

### 차징 과사용

반응:

- 차징 방해 엘리트가 등장한다.
- 가로등 방송이 차징 타이밍을 칭찬하거나 유도한다.
- 캠페인은 윤서의 침묵 핵 주변의 공백을 측정하기 시작한다.

문구:

```text
고객님의 충전 습관을 분석했습니다. 더 알맞은 방해를 추천합니다.
```

## 업데이트 관측층 예시

외곽 주택가에 추가 가능한 관측층:

| 관측층 | 조건 | 내용 |
| --- | --- | --- |
| 무음 라디오 관측층 | 결절 파괴 또는 무음 라디오 보관 | 조용한 집들 사이에서 구조 신호 왜곡 |
| 가족사진 잔향층 | 기억 추출 또는 가족사진 보관 | 창문 잔상, 가짜 가족 목소리 |
| 단골 고객층 | 반복 방문 | 같은 집 구조, 빈 고객란, 플레이어 호명 |
| 고위험 쿠폰 변이층 | 캠페인 활용 높음 | 위험 카드 강화, 회원 전용 엘리트 |
| 조작된 구조 신호층 | 악의 씨앗 `pattern` 이상 | 캠페인 아닌 누군가의 수정 흔적 |

## 리스크

### 상태 조합 폭발

완화:

- 개별 상태를 모두 분기하지 않고 `local_memory_profile`로 묶는다.
- 업데이트는 대표 프로필별 1~2개 변형만 제공한다.
- 문구 우선순위를 정해 한 번에 하나의 핵심 반응만 보여준다.

### 유저별 스토리 차이 과다

완화:

- 모든 유저가 접근 가능한 공통 관측층을 먼저 연다.
- 개인 상태는 적 이름, 문구, 보상, NPC 반응으로 변형한다.
- 핵심 진행은 하나의 선택에만 잠그지 않는다.

### 신규 유저 격차

완화:

- 새 관측층은 기본 입구를 가진다.
- 오래 플레이한 유저는 더 많은 변형 문구와 우회 보상을 본다.
- 이전 선택이 없어도 보급소 관측 보고로 기본 맥락을 제공한다.

### 업데이트 조건 복잡화

완화:

- 조건은 `공통 개방 조건 + 개인 변형 조건`으로 나눈다.
- 공통 조건은 낮게, 개인 변형 조건은 상태별로 둔다.
- UI에는 조건을 모두 설명하지 않고 관측 보고로 압축한다.

### 문구 꼬임

완화:

우선순위:

```text
1. 새 관측층 첫 발견
2. 보스 선택 후 첫 반응
3. 핵심 송출 기록/진행 단서
4. 악의 씨앗 확신도 상승
5. 캐릭터 전용 반응
6. 반복 행동 반응
7. 농도/재침식 경고
8. 일반 변이 문구
```

### 변화 인지 실패

완화:

- 숫자만 바꾸지 않는다.
- 게시판 부제, 적 이름, 오브젝트, NPC 대사 중 최소 하나를 바꾼다.
- 첫 발견 시에는 짧은 관측 보고를 강제로 보여준다.

### 변화 피로

완화:

- 모든 런마다 새 반응을 주지 않는다.
- 로컬별 활성 관측층은 동시에 1~2개만 전면에 둔다.
- 안정화 상태도 보상으로 인정한다. 늘 더 자극적인 변이만 만들지 않는다.

## 영구 변화와 일시 변화

영구로 남길 것:

- 보스 처리 결과.
- 주요 흔적 보관/소모 선택.
- 핵심 송출 기록 해석.
- 로컬 대표 프로필.
- 주요 NPC/캐릭터 관점 해금.

일시 이벤트로 처리할 것:

- 고농도 폭풍.
- 특정 변이 엘리트 주간.
- 추가 보상 이벤트.
- 캠페인 광고 문구 변화 일부.
- 특정 흔적 드롭 증가.

지역 상태 단계는 너무 많으면 안 된다. 0.2와 장기 공통으로 아래 5단계를 기본으로 권장한다.

```text
active
quieted
haunted
reinfected
overread
```

필요할 때 `exploited`, `neglected` 같은 프로필을 태그로 추가한다.

## 0.2 적용

0.2에 넣을 최소 상태:

```text
boss_outcome
revisit_count
campaign_pressure
adaptation_charge
behavior_memory
trace_preservation
sync_risk
```

0.2에서 하지 말 것:

- 관측층 UI를 복잡하게 만들지 않는다.
- 모든 상태별 문구를 만들지 않는다.
- 업데이트용 변이를 실제 콘텐츠로 많이 넣지 않는다.
- 악의 행위자를 직접 공개하지 않는다.

0.2에서 보여줄 것:

- 같은 지역 반복 방문 시 단골 고객 반응.
- 보스 선택 후 게시판 부제 변화.
- 흔적 선택 후 NPC 또는 오브젝트 변화.
- 차징 과사용 후 캠페인이 플레이어 습관을 읽는 암시.
- 무음 라디오/이름 없는 영수증으로 새 관측층 가능성만 암시.
