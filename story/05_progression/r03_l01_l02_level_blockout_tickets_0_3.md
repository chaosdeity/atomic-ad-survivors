# R03-L01/L02 레벨 블록아웃 티켓 v0.3

## 문서 상태

```text
상태:
초안 v0.3

판정:
R03 0.3의 직접 출격 구역을 R03-L01 반품 접수 야드와 R03-L02 자동 분류장으로 제한하고, 레벨 블록아웃 제작 티켓으로 분해한다.

용도:
레벨 디자이너가 R03을 "창고 배경"이 아니라 목적지 판정이 사람에게 붙는 플레이 공간으로 만들기 위한 블록아웃 기준.
```

---

## 0. 블록아웃 잠금

열 것:

```text
R03-L01 반품 접수 야드
R03-L02 자동 분류장
```

열지 않을 것:

```text
R03-L04, R03-L05는 0.3에서 열지 않는다.
R03-L03은 보증 심사 창고 입구 신호만 둔다.
RETURN-05 정식 U5는 열지 않는다.
```

핵심 기준:

```text
라벨은 태그가 아니다.
보급태그, 통행태그, 진료태그는 정산/보급소에서만 생활 접근 자원으로 읽힌다.
```

---

## 1. 전체 동선

권장 동선:

```text
보급소 출격
-> R03-L01 야드 진입로
-> 반품 접수 야드
-> 라벨 통로
-> 미수령 보관 구역
-> R03-L02 자동 분류장
-> 분류 레일 중심부
-> 반송 드론 재검사 구역
-> 보급소 귀환
```

동선 목표:

| 구간 | 유저 체감 | 핵심 오브젝트 |
|---|---|---|
| L01 진입 | 공간이 R01과 다르다 | 컨테이너, 접수 번호판 |
| L01 접수 | 상태가 붙는다 | 송장 프린터, 보류 스탬프 |
| L01 보관 | 보상이 선택이다 | 미수령 보관함 |
| L02 초입 | 길이 판정한다 | 자동 분류 레일 |
| L02 중심 | 시스템을 역이용한다 | 반송 드론, 레일 게이트 |
| L02 끝 | 후속 지역이 보이지만 열리지 않는다 | 압축 경고선, 보증 창고 입구 신호 |

## 3. R03-L01-BLOCKOUT-01 야드 진입로

```text
ticket_id:
R03-L01-BLOCKOUT-01

goal:
유저가 20초 안에 R01 주택가가 아니라 물류/반품 권역에 들어왔음을 알게 한다.
```

### 배치 기준

| 항목 | 기준 |
|---|---|
| 시작 공간 | 넓고 낮은 위험도 |
| 시야 신호 | 컨테이너 줄, 접수 번호판, 닫힌 출고문 |
| 이동 | 좌우 회피 여유가 있는 2.5~3 화면 폭 |
| 첫 위험 | 피해보다 라벨 전조가 먼저 |
| 첫 보상 | 작은 흔적 또는 보급태그 후보 암시 |

### 금지

```text
시작 10초 안에 레일로 플레이어를 강제로 밀기.
배경을 일반 창고처럼만 만들기.
물류 농담만으로 지역 정체성을 대체하기.
```

```text
ticket_id:
```

### 배치 기준

## 6. R03-L02-BLOCKOUT-01 자동 분류장 초입

```text
ticket_id:
R03-L02-BLOCKOUT-01

goal:
레일이 이동 리듬을 바꾸지만 조작을 빼앗지는 않는다는 감각을 준다.
```

### 배치 기준

| 항목 | 기준 |
|---|---|
| 핵심 오브젝트 | R03-OBJ-02 자동 분류 레일 |

### 실패 기준

```text
레일 때문에 플레이어가 죽음보다 답답함을 먼저 느끼면 실패다.
레일은 길막이 아니라 리듬 변화여야 한다.
```

## 7. R03-L02-BLOCKOUT-02 분류 레일 중심부

```text
ticket_id:
R03-L02-BLOCKOUT-02

goal:
레일, 게이트, 반송 드론이 함께 작동해 시스템 실수를 유도하는 재미를 만든다.
```

### 배치 기준

| 항목 | 기준 |
|---|---|
| 핵심 오브젝트 | 자동 분류 레일, 반송 드론 |
| 전투 밀도 | 중간 이상, 단 라벨 전조는 읽혀야 함 |
| 역이용 | 드론을 적에게 붙이거나 레일로 밀 수 있음 |
| 보상 | 오유도 성공 시 보급태그 후보/흔적 |
| 반복 반응 | 드론이 다음 런에 윤서 표식을 먼저 검사 |

### 전투 루프

```text
라벨 전조 확인
-> 라벨 회피 또는 적에게 전이
-> 레일 방향 이용
-> 반송 드론 오유도
-> 보류 스탬프로 판정 지연
```

## 8. R03-L02-BLOCKOUT-03 반송 재검사 구역

```text
ticket_id:
R03-L02-BLOCKOUT-03

goal:
반복 출격 시 캠페인이 유저의 악용 패턴을 기억한다는 감각을 만든다.
```

### 배치 기준

| 상태 | 변화 |
|---|---|
| 첫 방문 | 드론 추적이 느리고 명확 |
| 오유도 2회 이상 | 드론이 라벨 대상 확인 시간을 줄임 |
| 보류 스탬프 남발 | 스탬프 재충전 포인트 위치가 불리해짐 |
| 보관함 남김 많음 | 재검사 안내판에 번호 잔향 표시 |

### 금지

```text
드론을 단순 추적 탄환으로 만들지 않는다.
드론은 "누가 반송 대상인가"를 잘못 읽는 장치다.
```

## 11. 최종 결론

R03-L01은 규칙 학습 구역이고 R03-L02는 규칙 역이용 구역이다.

좋은 블록아웃은 유저가 아래 순서로 말하게 만든다.

```text
저 바닥에 뭐가 찍힌다.
저 레일은 짜증나는 길막이 아니라 이용할 수 있다.
드론을 저쪽에 붙이면 된다.
보관함은 그냥 상자가 아니네.
다음 런에 얘들이 내가 한 짓을 기억하네.
```

추천 다음 작업:

```text
1. R03-L01-BLOCKOUT-01~03을 회색 박스로 배치
2. R03-L02-BLOCKOUT-01~04를 레일/드론 테스트 구역으로 배치
3. r03_engine_data_schema_handoff_0_3.md의 event_key를 블록아웃 오브젝트에 연결
```
