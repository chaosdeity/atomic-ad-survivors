## 0. 사용 원칙

윤서는 R03에서 해설자가 아니다.

윤서의 말투:

| 기준 | 방향 |
|---|---|
| 길이 | 1줄, 짧게 |
| 감정 | 건조함, 현장 판단, 감정 절제 |
| 농담 | 광고 시스템과 약관 말투만 비꼰다 |
| 깊이 | 과거 설명 대신 멈칫함, 말수 감소, 선택 지연으로 표현 |
| 전투 | "처치"보다 "접수 실수", "보류", "경로 변경" 쪽 |
| 금지 | 구원 선언, 전문가 자랑, 정의 연설, 과거 독백 |

R03 문구 기준:

- `보급태그`, `통행태그`, `수신태그`, `진료태그` 등 기준 태그명을 쓴다.
- 라벨, 송장, 보관함, 보류 도장은 태그가 아니라 상태/오브젝트/흔적이다.
- R03은 사람을 죽이는 지역이 아니라 목적지/수취/반송 판정을 닫으려는 지역이다.
- 문구의 조롱 대상은 광고 시스템, 자동 분류, 고객센터식 안내다.
- 사람, 생존자, 배송 노동자, 환자, 로봇을 조롱하지 않는다.

## 1. 윤서 톤 가이드

### 1.1 좋은 방향

| 상황 | 톤 |
|---|---|
| 라벨이 붙음 | 질색하지만 당황을 길게 말하지 않는다 |
| 보관함 앞 | 물자보다 기록을 먼저 걱정한다 |
| 레일에 밀림 | 짧게 상황을 받아들이고 역이용을 찾는다 |
| 드론을 속임 | 건조한 농담 가능 |
| 보류 스탬프 | 익숙하지만 자랑하지 않는다 |
| 목적지 누락 로그 | 말수가 줄고 문장이 짧아진다 |

### 1.2 피해야 할 방향

| 금지 방향 | 이유 |
|---|---|
| 윤서가 R03 규칙을 모두 설명 | 유저가 손으로 배워야 한다 |
| 윤서가 반품 업무 전문가처럼 말함 | 구문명 직업 확정으로 읽힐 위험 |
| 윤서가 모두를 구하겠다고 선언 | 캐릭터 핵심과 충돌 |
| 윤서가 배송/반품 밈으로만 농담 | R03의 인간 문제를 가볍게 만든다 |
| 윤서가 RETURN-05 관점을 대체 | 이루의 수취/반송/미수령 기능을 훔친다 |

## 12. 금지 문구 은행

아래 방향은 쓰지 않는다. 실제 UI/대사에 넣지 않는다.

| 금지 방향 | 대체 방향 |
|---|---|
| 윤서가 반품 전문가처럼 자랑 | 현장 판단과 보류 감각으로 표현 |
| 윤서가 모두를 구한다고 선언 | 오늘 데려올 수 있는 것부터 처리 |
| 라벨을 자원처럼 말함 | 라벨은 상태/판정/흔적으로 표현 |
| 보관함을 보상 상자처럼 소비 | 물자와 목적지 기록이 같이 온다는 부담 |
| 반송을 귀환과 동일시 | 반송은 보관 좌표 재배정일 수 있음 |
| R03 전체 진실 설명 | R03-L01~L02의 체감만 제공 |
| RETURN-05 정식 해금 암시 | U0~U2 씨앗, 흔적, 기록으로 제한 |
| R03-L05 보스권 암시 과다 | 최종 심사실은 P2 예고로만 둔다 |
