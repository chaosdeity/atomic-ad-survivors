## 0. 결론

추천 경로:

```text
R03 구현 피드백 1회
-> R03-PT-01~08 결과 확인
-> R02 0.4 PMO 판정
-> R02-L01/L02 상세화 착수
```

이유:

| 이유 | 설명 |
|---|---|
| 판정형 지역 피로 방지 | R03은 목적지 판정, R02는 환자 등록 판정이다. 연속으로 열면 체감이 비슷해질 수 있다. |
| R03 씨앗 효율 확인 | R03 안의 처방/팔찌/진료태그 기록이 실제로 궁금증을 만들었는지 확인해야 한다. |
| UI 자원 이해 확인 | 보급태그/통행태그/진료태그가 한 화면에서 자연스럽게 읽히는지 먼저 본다. |
| HOLD-04 과노출 방지 | R02를 너무 빨리 밀면 HOLD-04가 정식 해금처럼 보일 수 있다. |
| 제작 리듬 유지 | R03 손맛을 먼저 잡아야 R02가 윤리/정산 중심으로 차별화된다. |

## 1. 게이트 선택지

| 선택 | 판정 | 사용 조건 |
|---|---|---|
| R02-GATE-01 | R03 피드백 후 PMO 판정 | 기본 추천. R03 구현/플레이테스트 1회 후 판단 |
| R02-GATE-02 | R02 PMO 사전 판정만 진행 | R03 구현 중 문서팀이 놀지 않게 하되, 제작 착수는 보류 |
| R02-GATE-03 | R02 상세화 즉시 착수 | R03-PT 주요 항목이 이미 빠르게 통과하거나 일정상 병렬화가 필요할 때만 |
| R02-GATE-04 | R02 보류, R03 수정 집중 | R03이 방해물처럼 느껴지거나 태그/라벨 혼동이 클 때 |

기본값:

```text
R02-GATE-01
```

## 2. R03 피드백에서 봐야 할 조건

R02 0.4 PMO 판정 전에 최소 아래를 확인한다.

| 조건 ID | 조건 | 통과 기준 |
|---|---|---|
| R02-GATE-CHECK-01 | R03 손맛 | R03-PT-01~03 중 2개 이상 통과 |
| R02-GATE-CHECK-02 | R03 반복 반응 | R03-PT-06 통과 또는 조건부 통과 |
| R02-GATE-CHECK-03 | 태그 이해 | R03-PT-07에서 라벨/태그 혼동이 낮음 |
| R02-GATE-CHECK-04 | R02 씨앗 반응 | 처방/팔찌/진료태그 기록이 후속 궁금증을 만듦 |
| R02-GATE-CHECK-05 | 캐릭터 과노출 | RETURN-05/HOLD-04가 정식 해금처럼 보이지 않음 |

실패 시:

| 실패 | 처리 |
|---|---|
| R03 손맛 실패 | R02를 열지 말고 R03 전조/역이용/정산 피드백 수정 |
| 태그 혼동 | R02 문서 진입 전에 UI 자원명/상태명 분리 보강 |
| R02 씨앗 무반응 | R02 0.4보다 R03 씨앗 문구/정산 보강 우선 |
| 캐릭터 과노출 | HOLD-04/RETURN-05 노출 단계 축소 |

## 3. R02 0.4를 올릴 준비가 되는 상태

R02 0.4 PMO 판정으로 넘어가는 조건:

```text
R03가 시스템을 속이는 손맛을 최소 1회 증명한다.
R03 정산에서 진료태그 기록이 "후속 의료권 신호"로 읽힌다.
유저가 R02를 새 지역으로 궁금해하지만, HOLD-04 해금을 기대하지는 않는다.
```

R02 0.4 문서팀 착수 범위:

| 우선 | 산출물 |
|---:|---|
| 1 | R02-L01 응급 접수 홀 경험 상세 |
| 2 | R02-L02 장기 입원동 반복 반응 |
| 3 | R02 오브젝트 6종 후보 |
| 4 | R02 정산/보급소 UI 문구 샘플 |
| 5 | HOLD-04 U0~U2 씨앗 티켓 |
| 6 | R02-L03 조건부 확장 판정 |

## 6. 최종 권고

권고:

```text
R02-GATE-01.
R03 구현 피드백 1회 후 R02 0.4 PMO 판정.
```

단, 문서팀 병렬 작업이 필요하면 `R02-GATE-02`까지는 허용한다.

```text
사전 판정은 가능.
제작 착수는 R03-PT 결과 확인 뒤.
```

이렇게 하면 R03의 손맛을 먼저 지키고, R02는 그 뒤에 윤리/정산 중심의 다른 맛으로 열린다.
