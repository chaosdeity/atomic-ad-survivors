# R01-C02 Conflict Ticket 0.2

상태: 0.2 충돌 티켓 초안
목적: `C01 가족/주거`와 `C02 의료/보험`의 첫 충돌인 "가족 구성원인가, 장기 환자인가?"를 R01 후속 씨앗으로 심고, R02 의료/보험 후보로 자연스럽게 넘긴다.
연결 문서: `story/05_progression/campaign_experience_atlas_0_2.md`, `story/05_progression/r01_campaign_experience_audit_0_2.md`, `story/03_regions/r01_suburb.md`, `story/03_regions/r01_boss.md`

## 0. 핵심 질문

```text
같은 사람을 R01은 가족 구성원/세대원으로 읽고,
R02는 장기 환자/진료 대상으로 읽는다.

그 사람을 데려오는 것은 구원인가,
아니면 진료태그와 식량태그가 동시에 흔들리는 일인가?
```

이 충돌은 R01 안에서 완전히 해결하지 않는다. R01에서는 작은 흔적과 정산 부담으로만 보여주고, R02 의료/보험 진입 시 본격화한다.

## 1. 충돌 티켓

```text
CampaignConflictTicket
  id: r01_c02_family_or_patient_0_2
  primary_campaign: C01_family_housing
  secondary_campaign: C02_medical_insurance
  contested_subject: long_stay_household_member
  conflicting_classification: family_member_vs_long_term_patient
  affected_tags: food_tag, residence_tag, medical_tag
```

| 필드 | 내용 |
|---|---|
| contested_subject | 장기 입원 기록이 남은 세대원, 또는 병원식 배급 스티커가 붙은 R01 가족 구성원 |
| conflicting_classification | R01은 세대원/가족대표/식탁 자리로 읽고, C02는 환자/장기 관리 대상/진료 보류자로 읽는다. |
| affected_tags | 식량태그, 거주태그, 진료태그 |
| run_expression | R01 식탁/문패 근처에 병원식 배급 라벨, 처방 스티커, 흰 팔찌 조각이 낮은 확률로 등장한다. |
| npc_expression | ROOM-12 또는 DINING-4의 "자리를 비우면 배급이 흔들린다"가 R02에서는 "퇴원하면 진료가 끊긴다"로 뒤집힌다. |
| boss_or_elite_expression | R01 보스의 가족 구성표와 R02 퇴원 심사표가 같은 사람을 서로 다른 칸에 넣으려 한다. |
| settlement_expression | `식량태그 접근 일부 유지`, `진료태그 조건 미확정`, `세대원 등록 보류` |
| player_choice | 가족칸 보류, 우회 배급, 진료태그 조건 보류, 미완 귀환 |
| short_term_reward | 식량태그 안정 또는 진료태그 힌트 |
| long_term_aftereffect | 보급소 침상/식량/진료 접근 부담 증가, R02 진입 시 환자 분류 충돌 오픈 |
| forbidden_resolution | 가족에게 데려가면 치료 문제도 자동 해결 |

## 2. R01에서 심는 최소 씨앗

R01에서는 C02를 설명하지 않는다. 아래 정도의 흔적만 둔다.

| 씨앗 | 위치 | 문구/표현 | 조건 |
|---|---|---|---|
| 흰 팔찌 조각 | ROOM-12 근처 또는 식탁 아래 | `이름 대신 병동 번호가 남아 있다.` | 흔적 보존 중간 이상 |
| 병원식 배급 라벨 | 식탁 배급기/네 번째 자리 근처 | `식량태그 라벨 위에 진료 식이 스티커가 덧붙어 있다.` | 식량태그 불안정 |
| 처방 보류 스티커 | 문패/우편함 근처 | `퇴원 처리 전 주소 확인 필요.` | 수신태그 중간 이상 |
| 미나 정산 반응 | 보급소 정산 후 | `먹는 문제랑 치료 문제가 같은 이름에 붙었어.` | 해당 흔적 회수 |
| 세븐 분석 반응 | 보급소 분석 | `세대원 기록과 진료 대상 기록이 같은 사람을 가리킵니다.` | 수신태그 배분 |

금지:

- R01에서 병원 구역을 미리 열지 않는다.
- 의료 캠페인을 감염/공포 스킨으로 예고하지 않는다.
- "환자를 가족에게 돌려보내자"를 정답처럼 만들지 않는다.

## 3. R02로 넘기는 방식

R02 첫 진입 전 게시판 후보:

| 조건 | 게시판 문구 |
|---|---|
| R01 흰 팔찌 조각 보관 | `진료태그 조건이 세대 기록과 충돌합니다.` |
| 병원식 배급 라벨 보관 | `식량태그와 진료 식이 기록이 같은 이름에 붙었습니다.` |
| ROOM-12 우회 배급 후 | `우회 배급 기록이 퇴원 심사에서 조회될 수 있습니다.` |
| DINING-4 자리 보류 후 | `빈 가족 칸이 장기 환자 기록과 겹칩니다.` |

R02 첫 오브젝트 후보:

| 오브젝트 | R01 연결 |
|---|---|
| 백색 팔찌 접수기 | R01 흰 팔찌 조각의 원본 |
| 병원식 배급 키오스크 | R01 식탁 배급기와 같은 이름을 호출 |
| 퇴원 보류 문 | R01 거주태그/식량태그를 진료태그 조건으로 재검토 |
| 보호자 호출 스피커 | 가족 구성원과 보호자 역할을 뒤섞음 |

## 4. 플레이어 선택 구조

이 충돌에서 유저가 고르는 것은 선악이 아니라 어느 조건을 먼저 보류할지다.

| 선택 | 즉시 효과 | 비용 |
|---|---|---|
| 가족칸 보류 | C01 가족 심사 압력 감소 | 식량태그 배급 안정이 늦어진다. |
| 우회 배급 유지 | ROOM-12/DINING-4의 식량태그 접근 유지 | R02에서 비공식 배급 기록으로 조회될 수 있다. |
| 진료태그 조건 보류 | 치료 접근이 바로 끊기지 않음 | 퇴원/거주 상태가 미확정으로 남는다. |
| 미완 귀환 | 확정 분류를 피함 | 보급소 부담과 재방문 필요가 늘어난다. |

## 5. 정산 문구 후보

| 상황 | 문구 |
|---|---|
| 씨앗 첫 회수 | `세대 기록과 진료 기록이 같은 이름을 가리킵니다.` |
| 식량태그 우선 | `식량태그 접근은 유지됩니다. 진료태그 조건은 미확정입니다.` |
| 진료태그 우선 | `진료태그 조건은 보류됩니다. 가족칸 심사는 다시 열립니다.` |
| 둘 다 보류 | `세대원 등록과 진료 대상 등록이 모두 닫히지 않았습니다.` |
| 강제 확정 위험 | `확정 처리는 빠릅니다. 빠른 만큼 다른 접근이 끊깁니다.` |

## 8. QA 기준

| 질문 | 통과 기준 |
|---|---|
| R01에서 R02를 과하게 설명하는가? | 설명하지 않는다. 흰 팔찌/라벨/짧은 정산만 남긴다. |
| 가족 복귀가 정답처럼 보이는가? | 아니어야 한다. 식량태그와 진료태그 비용이 같이 보인다. |
| 의료 캠페인이 악당처럼 보이는가? | 아니어야 한다. 치료 접근 유지가 실제로 중요해야 한다. |
| 유저가 다음 지역을 궁금해하는가? | "퇴원이 왜 위험하지?" 정도의 질문이 남으면 통과. |
