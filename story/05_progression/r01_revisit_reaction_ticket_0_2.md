# R01 Revisit Reaction Ticket 0.2

상태: 0.2 구현 티켓 초안
목적: `R01 서부 스마일홈`의 반복 출격 반응 중 0.2에서 바로 구현할 L1 단골 인식과 L2 행동 학습을 제작/구현 단위로 쪼갠다.
연결 문서: `story/05_progression/r01_campaign_experience_audit_0_2.md`, `story/05_progression/r01_settlement_phrase_bank_0_2.md`, `story/05_progression/environment_response_operations.md`, `R01_MONSTER_OBJECT_PLACEMENT_LOGIC.md`, `R01_REGION_REBUILD_0_2.md`

## 0. 범위

이번 티켓은 반복 출격 전체가 아니라 0.2에서 체감 가능한 최소 반응만 다룬다.

포함:

- L1 단골 인식.
- L2 행동 학습.
- 출격 게시판/가로등 방송/오브젝트 상태/적 배치 변화.
- 보스 전조와 연결되는 낮은 강도의 재방문 변화.

비포함:

- 보스 결과별 전체 지역 재구성.
- P1 NPC 본격 재방문 분기.
- C02/R02 이후 충돌 이벤트.
- 96개 캠페인 전역 반복 반응.

핵심 원칙:

```text
반복 출격은 패널티가 아니다.
캠페인이 유저를 더 잘 맞춤화하고 있다는 불쾌한 친절이다.
```

## 1. 상태값 제안

기존 문서와 충돌하지 않게 아래 상태값을 최소 후보로 둔다.

| 상태값 | 타입 | 의미 |
|---|---|---|
| `r01_revisit_count` | int | R01 출격 완료 또는 긴급 인양 후 재진입 횟수 |
| `r01_same_route_count` | int | 유저가 같은 주요 경로를 반복한 정도 |
| `r01_failed_run_count` | int | R01에서 등록 임계/긴급 인양이 발생한 횟수 |
| `r01_charge_overuse_count` | int | 차징/고출력 행동을 반복한 정도 |
| `r01_trace_preserved_count` | int | 문패/사진/주소/영수증 등 흔적을 보관한 횟수 |
| `r01_boss_exposure_level` | int | 모델하우스 결절/보스 전조 노출도 |
| `r01_reaction_profile` | enum | 현재 재방문 반응 묶음 |

`r01_reaction_profile` 후보:

| 값 | 조건 | 의미 |
|---|---|---|
| `fresh` | 첫 방문 또는 낮은 재방문 | 아직 단골 인식 없음 |
| `regular_customer` | 재방문 2회 이상 | 단골 고객 말투 시작 |
| `route_learned` | 같은 경로 반복 | 추천 동선/route bait 증가 |
| `failed_path` | 실패/긴급 인양 반복 | 이전 경로에 coupon trail 증가 |
| `charge_scheduled` | 차징 과사용 | 가족 플래너/차징 방해 후보 |
| `haunted_house` | 흔적 보존 높음 | 가족사진/문패/창문 잔향 증가 |

## 2. L1 단골 인식

### 2.1 발동 조건

| 조건 | 설명 |
|---|---|
| `r01_revisit_count >= 2` | 가장 기본 조건 |
| 같은 캐릭터로 R01 2회 이상 | 문패/방송 호명이 조금 더 정확해짐 |
| 같은 로컬 구역 반복 진입 | 해당 구역 오브젝트가 유저 동선을 기억 |

### 2.2 표현 채널

| 채널 | 구현 |
|---|---|
| 출격 게시판 | `단골 고객 문구가 늘어남` |
| 가로등 방송 | 캠페인이 유저를 단골/입주 후보로 부름 |
| 문패 | 이름 또는 역할 호명이 더 정확하지만 틀림 |
| 우편함 | 집 front보다 player path를 바라보는 빈도 증가 |
| 가족사진/창문 | 창문 가족 잔향이 플레이어 쪽을 보는 프레임 증가 |
| 오픈하우스 표지판 | 이전에 많이 간 방향을 더 친절하게 추천 |

### 2.3 문구

| 위치 | 문구 |
|---|---|
| 출격 게시판 | `단골 고객 문구가 늘어남` |
| 가로등 방송 | `단골 고객님, 같은 행복을 또 찾으셨군요.` |
| 문패 조사 | `문패가 이전 이름을 거의 맞혔다. 거의.` |
| 우편함 조사 | `우편함이 집보다 길을 더 잘 기억한다.` |
| 가족사진 조사 | `사진 속 시선이 이번에는 먼저 따라왔다.` |
| 정산 | `재방문 경로가 캠페인 프로필에 일부 반영되었습니다.` |

### 2.4 구현 티켓

| 티켓 | 내용 | 우선순위 |
|---|---|---|
| L1-01 | `r01_revisit_count >= 2`일 때 출격 게시판 문구 추가 | P0 |
| L1-02 | 가로등 방송 단골 문구 3개 랜덤 후보 추가 | P0 |
| L1-03 | 문패 조사 문구를 첫 방문/재방문으로 분기 | P0 |
| L1-04 | 우편함 방향 변화 또는 조사 문구 변화 | P1 |
| L1-05 | 가족사진/창문 잔향 빈도 증가 | P1 |

## 3. L2 행동 학습

### 3.1 같은 경로 반복

조건:

```text
r01_same_route_count >= 2
```

표현:

| 채널 | 구현 |
|---|---|
| 표지판 | 이전 경로 방향을 더 친절하게 가리킴 |
| route sign robot | fake return route 또는 loop 외곽에 제한 등장 |
| 바닥 흔적 | coupon trail이 이전 player path 근처에 증가 |
| 정산 | `추천 동선이 이전 경로를 학습했습니다.` |

금지:

- 길막만 늘려서 답답하게 만들지 않는다.
- 실제 회수 UI와 fake route를 혼동시키지 않는다.

### 3.2 반복 실패/긴급 인양

조건:

```text
r01_failed_run_count >= 1
```

표현:

| 채널 | 구현 |
|---|---|
| 이전 경로 | 쿠폰 흔적, route bait 증가 |
| 적 배치 | 빠른 검수 로봇 또는 route sign robot 낮은 확률 등장 |
| 출격 게시판 | `이전 경로에 쿠폰 흔적 증가` |
| 정산 | `등록 임계 직전 경로가 광고 동선으로 저장되었습니다.` |

금지:

- 실패한 유저를 조롱하지 않는다.
- 실패 직후 난이도를 크게 올리지 않는다.

### 3.3 차징 과사용

조건:

```text
r01_charge_overuse_count >= 2
```

표현:

| 채널 | 구현 |
|---|---|
| 엘리트 후보 | 가족 플래너 |
| 패턴 | 차징 윈도우 직전 작은 경고 구역 생성 |
| 문구 | `5초 뒤 가족사진 촬영이 예약되어 있습니다.` |
| 정산 | `출력 기록이 가족 일정에 임시 등록되었습니다.` |

금지:

- 차징 버튼을 직접 잠그지 않는다.
- 차징을 쓴 유저가 처벌받는 느낌만 받게 하지 않는다.

### 3.4 흔적 보존 과다

조건:

```text
r01_trace_preserved_count >= 2
```

표현:

| 채널 | 구현 |
|---|---|
| 가족사진 | 창문 잔상과 사진 프레임 반응 증가 |
| 문패 | 이름 조각이 더 선명해짐 |
| 캠페인 위험 | 캠페인도 이름/주소를 더 정확히 읽음 |
| 정산 | `이름은 선명해졌고, 캠페인도 더 잘 읽습니다.` |

금지:

- 흔적 보존을 단순 선한 수집으로 만들지 않는다.

## 4. 반응 우선순위

같은 출격에서 여러 반응 조건이 동시에 켜지면 아래 순서로 대표 문구를 고른다.

| 우선순위 | 반응 | 이유 |
|---:|---|---|
| 1 | 보스 전조 | 모델하우스 접근/보스 흐름이 최우선 |
| 2 | 반복 실패 | 유저가 직전 경험을 알아차려야 함 |
| 3 | 같은 경로 반복 | 맵 동선 학습 체감 |
| 4 | 차징 과사용 | 전투 습관 학습 체감 |
| 5 | 흔적 보존 과다 | 감정/미스터리 체감 |
| 6 | 단골 인식 | 기본 재방문 풍미 |

## 5. 최소 구현 묶음

0.2 최소 묶음:

| 묶음 | 포함 티켓 |
|---|---|
| 출격 게시판 | L1-01, 반복 실패 문구, 같은 경로 문구 |
| 가로등 방송 | 단골 문구 3개, 차징 과사용 문구 1개 |
| 오브젝트 조사 | 문패 재방문 문구, 우편함 재방문 문구 |
| 적 배치 | 반복 실패 후 route bait 또는 route sign robot 낮은 확률 |
| 정산 | 태그 기준명으로 반복 반응 4개 문구 출력 |

P0에서 하지 않는 것:

- 가족 플래너 완성형 엘리트.
- 창문 가족 잔향 고급 연출.
- 보스 잔향 패턴의 상시 환경 이벤트.
- 캠페인 간 충돌 L5.

## 7. QA 기준

| 질문 | 통과 기준 |
|---|---|
| 반복 출격이 패널티처럼 느껴지는가? | 아니어야 한다. 친절한 맞춤화처럼 보이다가 찝찝해야 한다. |
| 내부 수치가 보이는가? | 보이면 실패. 문구/오브젝트/적 배치로만 보인다. |
| 첫 방문 유저가 손해 보는가? | 아니어야 한다. L1/L2는 재방문 풍미와 압박만 준다. |
| 실패한 유저를 조롱하는가? | 절대 금지. 실패 경로는 캠페인이 학습한 흔적으로만 남긴다. |
| 차징 방해가 불공정한가? | 버튼 잠금 금지. 예고 후 위치 압박만 허용. |
| 문구가 태그 기준명을 지키는가? | 정산/게시판은 `식량태그`, `충전태그`, `수신태그` 기준. |
