## 0. 0.4 후보 판정

R02 0.4는 "병원 스테이지"가 아니다.

핵심 문장:

```text
R02는 살리기 위해 환자 상태를 끝내지 않는 지역이다.
```

0.4에서 열 후보:

| 노드 | 판정 |
|---|---|
| R02-L01 응급 접수 홀 | 0.4 주 후보 |
| R02-L02 장기 입원동 | 0.4 주 후보 |
| R02-L03 보험 심사 약국 | 조건부 확장 또는 정산 씨앗 |
| R02-L04 격리문 복도 | 0.4에서 열지 않는다 |
| R02-L05 퇴원 불가 병동 | 0.4에서 열지 않는다 |

보류:

```text
HOLD-04 정식 해금은 0.4에서 확정하지 않는다.
퇴원 불가 심사 대리 얼굴 직접 대면은 R02-L05 이후로 둔다.
진료태그를 회복 아이템이나 돈처럼 쓰지 않는다.
```

## 1. R03에서 넘어오는 씨앗

| R03 씨앗 | R02 확장 방향 |
|---|---|
| 처방 봉투 오배송 | R02-L01 접수번호/환자 번호 충돌 |
| 진료태그 접근 보류 기록 | R02-L03 약품/보험 심사 부담 |
| 백색팔찌 포장물 | R02-L01 환자 등록 규칙 |
| HOLD-04 원격 번호 | R02-L02 퇴원 보류자 씨앗 |
| 퇴원 보류 번호표 | R02-L03 또는 L05 후속 압력 |

R03에서 R02로 넘어갈 때 유저가 알아야 하는 것:

```text
배송 상태였던 기록이 R02에서는 환자 상태가 된다.
목적지를 잃은 물건보다, 퇴원 조건을 잃은 사람이 더 어렵다.
```

## 2. 유저 경험 목표

### 2.1 첫 느낌

| 시간 | 체감 | 구현 장치 |
|---:|---|---|
| 0~30초 | 조용하고 밝은데 답답하다 | 접수 번호판, 팔찌 발급기, 대기 의자 |
| 30~90초 | 등록되면 보호받지만 이동이 줄어든다 | 환자 팔찌, 침상 배정선 |
| 90~150초 | 누군가를 데려오면 빈자리가 생긴다 | 병상 대장, 보호자 호출 |
| 150~180초 | 진료태그가 생존 접근을 잠근다 | 처방 창구, 보험 심사 키오스크 |

### 2.2 R03과의 차이

| 기준 | R03 물류/반품 | R02 의료/보험 |
|---|---|---|
| 상태 | 수취/반송/보류 | 환자/보호자/퇴원 보류 |
| 이동 압박 | 레일과 목적지 | 병상 배정과 격리 동선 |
| 보상 | 보급태그/통행태그 후보 | 진료태그 접근, 약품 기록 |
| 아이러니 | 목적지가 사람에게 붙음 | 보호가 자유를 닫음 |
| 반복 반응 | 드론 재검사 | 재접수/재입원/병상 재배정 |

## 3. 0.4 노드 초안

### 3.1 R02-L01 응급 접수 홀

한 줄 정의:

```text
사람이 먼저 들어오는 곳이 아니라, 환자 번호가 먼저 붙는 곳.
```

핵심 오브젝트:

| ID 후보 | 오브젝트 | 역할 |
|---|---|---|
| R02-OBJ-01 | 접수 팔찌 발급기 | 환자 상태 부착 |
| R02-OBJ-02 | 보호자 미확인 창구 | 가족/보호자 판정 충돌 |
| R02-OBJ-03 | 대기 번호 전광판 | 순서가 곧 접근권이 됨 |
| R02-OBJ-04 | 응급 분류선 | 이동 경로 제한 |

유저 재미:

```text
팔찌 상태를 피하거나, 다른 판정과 충돌시키거나, 접수 순서를 흔든다.
```

금지:

```text
병원을 악한 조직처럼 만들기.
의료진을 조롱하기.
진료태그를 즉시 회복 보상처럼 쓰기.
```

### 3.2 R02-L02 장기 입원동

한 줄 정의:

```text
살아 있는 상태를 유지하기 위해 나가는 선택을 계속 미루는 공간.
```

핵심 오브젝트:

| ID 후보 | 오브젝트 | 역할 |
|---|---|---|
| R02-OBJ-05 | 병상 대장 | 빈 병상 재배정 |
| R02-OBJ-06 | 자동 침대 | 보호/고정 경계 |
| R02-OBJ-07 | 커튼 호출선 | 시야/소리 압박 |
| R02-OBJ-08 | 퇴원 안내문 | 나가는 조건의 부재 |

유저 재미:

```text
구출과 유지 사이에서 정답 없는 선택을 만든다.
한 사람을 움직이면 다른 병상과 진료태그 부담이 흔들린다.
```

### 3.3 R02-L03 보험 심사 약국

0.4 판정:

```text
조건부 확장.
R02-L01/L02가 충분히 작동한 뒤 열거나, 정산/보급소 씨앗으로 둔다.
```

핵심 오브젝트:

| ID 후보 | 오브젝트 | 역할 |
|---|---|---|
| R02-OBJ-09 | 처방 창구 | 약품 접근 조건 |
| R02-OBJ-10 | 보험 심사 키오스크 | 진료태그 부담 |
| R02-OBJ-11 | 봉인 약품함 | 열기/남기기 선택 |

## 6. HOLD-04 처리

0.4에서 가능한 것:

| 단계 | 허용 |
|---|---|
| U0 | 원격 번호, 퇴원 보류 기록 |
| U1 | 병상/팔찌/퇴원 안내문에 남은 반응 |
| U2 | 짧은 음성 또는 로그로 "나가도 되는가" 질문 |

0.4에서 금지:

```text
HOLD-04 정식 해금.
전투 조작 캐릭터화.
치료 담당 동료화.
퇴원 보류 문제의 정답 제시.
```

## 10. 최종 판정

R02 0.4는 R03 다음 후보로 충분히 강하다. 단, R03의 "목적지 판정" 바로 뒤에 R02의 "환자 등록 판정"을 열면 둘이 모두 판정형 지역이라 피로할 수 있다.

따라서 0.4 제안:

```text
R03 0.3이 손맛 중심으로 통과하면 R02 0.4는 윤리/정산 중심으로 연다.
R02-L01/L02를 먼저 만들고, R02-L03은 조건부 확장으로 둔다.
R02-L04/L05와 HOLD-04 정식 해금은 보류한다.
```

추천 다음 작업:

```text
1. R02-L01 응급 접수 홀 경험 상세 작성
2. R02-L02 장기 입원동 반복 반응 설계
3. R02 오브젝트 6종 후보 선정
```
