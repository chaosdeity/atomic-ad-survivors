## 0. 판정 요약

R01 서부 스마일홈은 아틀라스 기준으로 **0.2 기준 사례로 사용 가능**하다.

통과한 것:

- C01 가족/주거의 광고 약속, 생활권 통제, 대표 태그, 보스 약관이 선명하다.
- 집/문패/우편함/가족사진/오픈하우스/청소기 적이 런 재미와 세계관을 동시에 담당한다.
- NPC 12명은 이미 의뢰자 목록이 아니라 역할/이름/주소/자리/배급의 타협 구조로 정리되어 있다.
- 스마일 홈 가족심사 관리자는 "강한 적"이 아니라 가족 적합성 심사 절차의 얼굴로 작동한다.
- 보스 결과가 결절 파괴, 기억 추출, 코어 해방, 신호 우회로 갈라져 반복 방문 근거를 만든다.

보강해야 할 것:

- 일부 R01 구현/문서 문구에 `밥표`, `전원표`, `신호표`가 UI 정식 자원처럼 남아 있다. R01 현장 속어로는 가능하지만, 정산 UI/보상 기준명은 `식량태그`, `충전태그`, `수신태그`로 바꿔야 한다.
- 반복 출격 반응은 문서상 풍부하지만, L1 단골 인식과 L2 행동 학습을 구현 티켓으로 더 잘게 쪼개야 한다.
- C01 vs C02, C01 vs C05, C01 vs C06 충돌은 가능성이 적혀 있으나, 아직 충돌 티켓 형태로 잠겨 있지는 않다.
- 정산/출격 게시판 문구가 아틀라스 문체로 별도 뱅크화되어야 한다.

결론:

```text
R01은 새 NPC를 더 늘릴 단계가 아니다.
지금 필요한 것은 아틀라스 슬롯에 맞춘 구현 티켓화, 용어 정리, 반복 반응 데이터화다.
```

## 11. 바로 쓸 티켓 초안

### T01. R01 정산 문구 기준명 교체

```text
목표:
정산 UI와 출격 게시판에서 밥표/전원표/신호표를 UI 정식명처럼 쓰지 않는다.

교체:
밥표 후보 -> 식량태그 후보
전원표 후보 -> 충전태그 후보
신호표 후보 -> 수신태그 후보

허용:
NPC 대사, 낡은 표지, 손글씨, 캠페인화 인간 속어에서는 유지 가능.
```

### T02. R01 반복 출격 L1 단골 인식

```text
조건:
r01_revisit_count >= 2

표현:
가로등 방송이 "단골 고객님" 계열 문구 출력.
문패 일부가 이전 출격 캐릭터 이름 또는 역할을 틀리게 호명.
우편함이 집 front보다 player path를 더 자주 바라봄.

금지:
내부 점수나 반복 횟수를 UI에 직접 표시하지 않는다.
```

### T03. R01 반복 출격 L2 행동 학습

```text
조건:
반복 실패, 같은 경로 반복, 차징 과사용 중 하나.

표현:
이전 player path 근처 coupon trail 증가.
route sign robot 또는 route bait 증가.
차징 과사용 시 가족 플래너 엘리트 후보 등장.

금지:
차징 버튼 잠금 같은 직접 처벌.
```

### T04. R01 C01 vs C02 충돌 씨앗

```text
목표:
R02 의료/보험으로 넘어가기 전에 "가족 구성원인가, 장기 환자인가" 질문을 작은 흔적으로 심는다.

R01 표현:
ROOM-12 또는 DINING-4 주변에 병원식 배급/처방 스티커가 붙은 식량태그 흔적.

보급소 표현:
미나가 식량태그와 진료태그가 같은 사람에게 동시에 걸릴 때의 부담을 짧게 언급.
```
