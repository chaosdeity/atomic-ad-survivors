## 0. 판정 요약

침묵 보급소 초기 NPC는 5명 기준으로 잠근다.

```text
H01 미나
H02 장도윤
H03 팝시
H04 세븐
H05 복희 할머니
```

이 5명은 각각 하나의 시설을 대표한다.

```text
미나:
정산 카운터

장도윤:
정비 벤치

팝시:
충전 스탠드

세븐:
출격 게시판 / 카탈로그 벽

복희:
이름 보관함
```

핵심 원칙:

```text
보급소 NPC는 메뉴의 얼굴이 아니라,
런에서 돌아온 결과가 공동체에 어떤 압력을 남기는지 보여주는 사람들이다.
```

따라서 초기 구현은 대사량을 늘리는 것보다, 다음 상태값에 반응하는 기준을 먼저 고정한다.

```text
인간 안정도
로봇 자율성
캠페인 농도
흔적 보존
보급 압력
수신/인양 안정도
```

## 2. 공통 운용 규칙

### 2.1 초기 5명 고정

보급소 초기 상주 NPC는 5명으로 충분하다.

새 NPC를 추가하기 전에 다음을 먼저 확인한다.

```text
1. 기존 5명 중 누가 담당할 수 있는 기능인가?
2. 시설 확장으로 해결 가능한가?
3. 대사 조건 추가로 해결 가능한가?
4. 오브젝트 흔적이나 방송 잔향으로 처리 가능한가?
```

초기 보급소는 큰 마을이 아니다.

```text
작은 카운터
정비 벤치
이름 보관함
충전 스탠드
출격 게시판
침상과 뒷방
```

이 범위 안에서 5명의 반응이 먼저 살아야 한다.

### 2.2 플레이어블과 분리

PATCH, 윤서, 리오 같은 플레이어블 또는 플레이어블 후보는 보급소에 머물 수 있다.
하지만 보급소 상주 NPC 5명 수에 섞지 않는다.

```text
플레이어블:
전투/성장/해금/긴급 인양을 가진다.

보급소 NPC:
시설/상태값/흔적/선택 후폭풍을 담당한다.
```

### 2.3 대사보다 상태 반응 우선

초기 구현 대사량은 NPC당 10~15줄이면 충분하다.

대신 다음 조건별 반응이 있어야 한다.

```text
런 귀환
흔적 보관
흔적 소모
캠페인 농도 상승
인간 안정도 하락
로봇 자율성 상승
보스 결과
새 지역 해금
```

## 3. H01 미나

### 기본 프로필

```text
이름:
미나

구분:
인간

담당 시설:
정산 카운터
회수물 분류대
태그 배분표
작은 창고

유저 첫 표기:
미나, 정산 카운터

담당 기능:
런 귀환 정산
회수물 분류
보급태그 후보 변환
흔적 보관/소모 선택 안내
보급소 상태 경고

핵심 믿음:
정산표가 맞으면 보급소는 버틴다.
```

미나는 사람을 살리기 위해 계산한다.

하지만 계산만으로는 사람을 다 살릴 수 없다는 것을 배워야 한다.

### 핵심 문장

```text
“태그 하나가 밀리면,
사람 하나의 밤도 밀린다.”
```

### 상태값 반응

| 상태 | 반응 |
|---|---|
| 인간 안정도 낮음 | 식량태그와 침상 배분을 먼저 본다. 흔적 소모를 권할 수 있다. |
| 인간 안정도 높음 | 말투가 조금 부드러워지고, 보관 선택을 더 쉽게 허용한다. |
| 로봇 자율성 상승 | 로봇용 부품과 충전 순서를 카운터 아래에 따로 정리한다. |
| 캠페인 농도 상승 | 쿠폰, 보증서, 행복 문구를 찢어 카운터 옆에 쌓아둔다. |
| 흔적 보존 상승 | 정산표에 “소모 금지” 칸을 만든다. |

### R01 연결

미나는 R01에서 돌아온 문패, 주소, 식탁 흔적, 사진 조각을 “당장 쓸 수 있는 자원”과 “아직 쓰면 안 되는 흔적”으로 나눈다.

대표 반응:

```text
“이건 태그로 바꾸면 오늘은 버텨.
근데 그러면 내일 누가 있었는지는 모르게 돼.”
```

### 금지 방향

```text
상점 주인화
돈만 밝히는 장사꾼화
튜토리얼 도우미화
무조건 따뜻한 보호자화
냉혈한화
```

## 4. H02 장도윤

### 기본 프로필

```text
이름:
장도윤

구분:
인간

담당 시설:
정비 벤치
역송출 장비대
차폐 배터리 선반

유저 첫 표기:
도윤, 정비 벤치 기술자

담당 기능:
자동공격 강화
차징 스킬 개조
역송출 장비 강화
캠페인 신호를 이용한 고위험 출력 옵션

핵심 믿음:
위험한 신호도 충분히 이해하면 통제할 수 있다.
```

도윤은 캠페인 기술을 가장 잘 아는 인간이다.

그래서 모두에게 필요하지만, 모두에게 조금 의심받는다.

### 핵심 문장

```text
“광고 신호를 막는 것도 방법인데,
되쏘는 건 더 빠르지.”
```

### 상태값 반응

| 상태 | 반응 |
|---|---|
| 캠페인 농도 상승 | 위험 출력 개조를 제안한다. 효율을 먼저 보고 대가를 늦게 걱정한다. |
| 인간 안정도 낮음 | 강화보다 방어 장비와 회수선 안정화를 먼저 권한다. |
| 로봇 자율성 상승 | 로봇 신호 변조 장치를 연구하려 한다. PATCH와 충돌 가능. |
| 충전태그 안정 | 안정형 차징 개조가 열린다. |
| 인양 안정도 낮음 | 역송출보다 회수선 보강을 우선해야 한다고 인정한다. |

### R01 연결

도윤은 R01의 스마트 도어, 가족사진 프레임, 가짜 비콘음, 상담 단말기를 기술적 현상으로 먼저 본다.

그 시선은 유용하지만 위험하다.

대표 반응:

```text
“이건 자동 안내가 아니야.
사람이 손댄 흔적이 있어.
그래서 더 별로고.”
```

### PATCH와의 분리

```text
도윤:
장비 출력, 역송출, 위험 신호 이용.

PATCH:
자기 몸을 부품처럼 써온 수리 로봇.
부품 윤리와 로봇 자율성의 감정축.
```

도윤은 PATCH를 고치려 들면 안 된다.
PATCH를 도구처럼 분석하려는 순간 불편해져야 한다.

### 금지 방향

```text
대장장이 NPC화
전기 마법사화
천재 해커화
매드 사이언티스트화
안전한 강화만 주는 기능 NPC화
```

## 5. H03 팝시

### 기본 프로필

```text
이름:
팝시

구분:
해방 로봇

담당 시설:
충전 스탠드
카드 리롤 단말
작은 폐품 선반

유저 첫 표기:
팝시, 충전 스탠드 마스코트

담당 기능:
카드 리롤
광고풍 카드 해금
로봇 자율성 이벤트
재동기화 위험 알림

핵심 믿음:
인간이 자신을 귀여워하면 버리지 않을 것이다.
```

팝시는 귀여운 로봇이 아니라, 귀여움이 생존 전략이 되어버린 로봇이다.

### 핵심 문장

```text
“웃는 얼굴은 기본값이에요.
기분은 아직 설정 중이고요.”
```

### 상태값 반응

| 상태 | 반응 |
|---|---|
| 로봇 자율성 낮음 | 광고 멘트와 상품 문구를 반복한다. |
| 로봇 자율성 높음 | 제품 대본보다 자기 판단을 더 자주 말한다. |
| 인간 안정도 낮음 | 충전 스탠드 뒤에 숨고 리롤 기능만 조용히 제공한다. |
| 캠페인 농도 상승 | 목소리에 짧은 징글 노이즈가 섞인다. |
| 흔적 보존 상승 | 로봇 이름표와 폐품 선반 이벤트가 열린다. |

### R01 연결

팝시는 VAC-0와 직접 대비된다.

```text
VAC-0:
집을 안전하게 만들기 위해 흔적을 치우는 홈케어 장치.

팝시:
버려지지 않기 위해 광고 말투를 유지하는 마스코트 로봇.
```

R01에서 회수한 청소 제외 스티커나 마스코트 배지에 반응할 수 있다.

대표 반응:

```text
“작은 물건들이 많이 들어 있네요.
버려진 게 아니라 숨어 있던 걸 수도 있어요.”
```

### 금지 방향

```text
개그 마스코트화
펫 로봇화
리롤 상점 UI화
무조건 순수한 로봇화
귀여운 말투만 남기는 캐릭터화
```

## 6. H04 세븐

### 기본 프로필

```text
이름:
세븐

구분:
해방 로봇

담당 시설:
출격 게시판
카탈로그 벽
지역 위험도 패널

유저 첫 표기:
세븐, 카탈로그 벽 기록 로봇

담당 기능:
지역 정보
적 도감
회수 목표 추천
캠페인 농도 변화 예고
반복 패턴 보고

핵심 믿음:
충분한 기록이 모이면 의도도 판정할 수 있다.
```

세븐은 정보를 많이 아는 로봇이 아니다.

그는 기록과 의도 사이의 차이를 늦게 배우는 로봇이다.

### 핵심 문장

```text
“저는 치우는 법을 압니다.
남겨야 하는 것은 배우는 중입니다.”
```

### 상태값 반응

| 상태 | 반응 |
|---|---|
| 흔적 보존 상승 | 회수 목표 추천 정확도가 오른다. |
| 캠페인 농도 상승 | 보고서가 광고 문구에 오염된다. |
| 로봇 자율성 상승 | 보고 문장에 자기 판단이 섞인다. |
| 수신태그 안정 | 지역 로그와 잔향 분석이 선명해진다. |
| 인양 안정도 낮음 | 위험 경로를 추천 대상에서 제외하려 한다. |

### R01 연결

세븐은 MAIL-LOOP, FRAME-LEFT, SIGN-BACK의 데이터를 분석할 수 있다.

하지만 의도 판정은 실패해야 한다.

대표 반응:

```text
“프레임 내 구성원 기록이 남아 있습니다.
개인 식별 신뢰도는 낮습니다.”
```

### 복희와의 분리

```text
세븐:
기록, 패턴, 반복, 수신 데이터.

복희:
이름, 흔적, 남겨야 하는 감정의 무게.
```

세븐은 이름을 보관하지 않는다.
복희는 데이터를 분석하지 않는다.

### 금지 방향

```text
전지적 AI화
도감 UI 마스코트화
세계관 설명 NPC화
무조건 정답을 주는 분석관화
감정 없는 완벽 로봇화
```

## 7. H05 복희 할머니

### 기본 프로필

```text
이름:
복희 할머니

유저용 표기:
복희
또는 복희 할머니, 이름 보관함

구분:
인간

담당 시설:
이름 보관함
손글씨 기록대
흔적 감정대

담당 기능:
흔적 감정
이름 조각 보관
기억 조각 열람
공존 루트 힌트
엔딩 축 보존 점수 안내

핵심 믿음:
이름을 적어두면 사라지지 않는다고 믿고 싶어 한다.
```

복희는 캠페인이 크게 말하는 세계에서, 작게 적는 사람이다.

### 핵심 문장

```text
“광고는 크게 말하지.
기억은 작게 말한단다.”
```

### 상태값 반응

| 상태 | 반응 |
|---|---|
| 흔적 보존 상승 | 이름 보관함 주변 메모가 늘어난다. |
| 흔적 소모 증가 | 말수가 줄고, 버린 흔적의 이름을 물어본다. |
| 인간 안정도 상승 | 새 피난민 이름을 적는다. |
| 로봇 자율성 상승 | 로봇에게도 이름을 물어보고, 먼저 허락을 구한다. |
| 캠페인 농도 상승 | 가짜 이름표와 너무 반듯한 글씨를 경계한다. |

### R01 연결

복희는 NAME-PAINTER, FRAME-LEFT, MAIL-LOOP, KEY-RING의 흔적을 한데 묶는 감정축이다.

하지만 복희가 모든 것을 보존하라고 말하면 안 된다.

대표 반응:

```text
“이름을 적는 건 붙잡는 일이기도 하단다.
그래서 조심해야 해.”
```

### 세븐과의 분리

세븐은 기록을 분석한다.
복희는 이름을 묻는다.

FRAME-LEFT 같은 잔향 앞에서 복희는 “얼굴 말고 이름”을 먼저 찾는다.

### 금지 방향

```text
현자 할머니화
감성 보관소화
엔딩 정답 NPC화
무조건 보존만 권하는 캐릭터화
퀘스트 할머니화
```

## 8. 관계선 기준

```text
미나에게 인정받고 싶지만, 밝은 말투 때문에 오히려 의심받는다.
```

대표 대화:

```text
미나:
“그 멘트, 네가 고른 거야?”

팝시:
“아직 확인 중이에요.”
```

```text

신호를 되쏘고 싶어 한다.

신호의 반복 패턴을 기록한다.
```

대표 대화:

```text
도윤:
“그건 농담이야.”

세븐:
“위험 정보로 분류하지 않아도 됩니까?”
```

```text
로봇에게도 이름을 물어본다.

```

대표 대화:

```text
복희:

팝시:
“그 기능은... 아직 잠겨 있는 것 같아요.”
```

### 세븐과 복희

```text
복희:
기록 안에서 이름을 찾는다.
```

대표 대화:

```text
세븐:
“개인 식별 신뢰도는 낮습니다.”

복희:
“그래도 부를 때 쓸 조각은 있겠지.”
```
