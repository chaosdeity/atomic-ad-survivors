## 1. 샘플링 목적

샘플링의 목적은 전체 문구를 한 번에 쓰는 것이 아니라, 전역 캠페인별 밀도가 살아나는 최소 표본을 먼저 고르는 것이다.

우선 확인할 것:

1. 각 권역이 자기 현실권/언어권의 제도 흔적을 갖는가.
2. L3 이하 슬롯이 직접 대면 NPC처럼 읽히지 않는가.
3. 상태값 후폭풍이 문구 표본에 들어가는가.
4. 작은 결말이 이름/과거사 없이도 전달되는가.
5. 플레이어블 해금, 성장, 전투 기능을 대신 설명하지 않는가.

## 2. 샘플링 단위

샘플링 단위:

```text
캠페인 권역 -> 정식 캠페인 -> 계층 -> 표현 방식 -> 상태 키 -> 작은 결말 연결
```

단위 원칙:

1. 표본은 `NPC 한 명`이 아니라 `은행형 문구 슬롯`이다.
2. 표본은 본명, 얼굴, 반복 대면, 장문 과거사를 갖지 않는다.
3. 표본은 캠페인별 오브젝트, 잔향, 로그, 자동 음성, 목록, 호출번호로 먼저 드러난다.
4. L3 대사 은행도 직접 대면 대사가 아니라 조건 반응 묶음으로만 작성한다.
5. 같은 권역 안에서 표현 방식이 한쪽으로 쏠리면 다음 표본은 다른 표현 방식으로 돌린다.

표 1:

| 기준 | 값 | 이유 |
|---|---:|---|
| 전체 L3 이하 은행 슬롯 | 1,856 | 전체 문구 작성 대상이 아니라 잠금 수량 |
| 1차 샘플 총량 | 240 | 전체의 약 13%만 먼저 표본화 |
| 정식 캠페인 수 | 96 | E01~E12 각 8개 캠페인 기준 |
| 정식 캠페인당 최소 샘플 | 2 | 모든 캠페인이 최소 두 표현 방식을 갖게 함 |
| 권역 기본 샘플 | 16 | 8개 캠페인 x 최소 2개 |
| E02 1차 가중 샘플 | 32 | 첫 외부권이라 현실권/언어권 검산을 먼저 함 |

| L5/L4 샘플 허용 | 0 | 이번 문서는 L3 이하 전용 |
| 새 이름 허용 | 0 | L3 이하 본명 부여 금지 |
| 직접 대면 증가 허용 | 0 | 은행형 문구만 허용 |

## 3. 캠페인별 최소 샘플 수

원칙:

1. 모든 정식 캠페인은 최소 2개 표본을 가진다.
2. 최소 2개 중 하나는 상태 반응성 표본, 하나는 오브젝트/잔향성 표본으로 둔다.
3. E02는 첫 외부권이므로 캠페인당 평균 4개 표본까지 허용한다.
4. E03~E12는 캠페인당 평균 2~3개 표본만 허용한다.
5. cross_region은 캠페인별 표본이 아니라 전역 반복 문장/삭제 로그/자동 음성 기준으로 제한한다.

## 4. 표현 방식별 샘플 기준

표 3:

| 표현 방식 | 샘플 목적 | 금지 | 예시 방향 |
|---|---|---|---|
| 대사 은행 | 상태값 변화 뒤의 짧은 반응 어조 확인 | 대면 인사, 자기소개, 퀘스트 수락 문장 | 조건 반응, 믿음 흔들림, 짧은 오해 |
| 잔향 | 장소에 남은 후폭풍과 반복음을 확인 | 말하는 NPC처럼 호출, 본명 언급 | 끊긴 음성, 반복 안내, 남은 소리 |
| 오브젝트 | 사물 하나가 결말 단서를 갖는지 확인 | 상점 물품 설명, 강화 재료 설명 | 라벨, 단말, 봉인, 스티커 |
| 호출번호 | 사람 대신 번호가 남는 감각 확인 | 실명 대기자 호출, 직접 만남 유도 | 접수번호, 탑승번호, 회수번호 |

| 자동 음성 | 시스템이 사람보다 오래 말하는 감각 확인 | 안내 NPC처럼 친절한 튜토리얼 | 재시도 안내, 폐쇄 안내, 자동 거절 |
| 로그 | 실패 기록이 경로와 반전을 남기는지 확인 | 사건 전말 확정, 범인 확정 | 삭제 로그, 재시도 기록, 장부 누락 |
| 표지 | 짧은 문구가 공간 규칙을 바꾸는지 확인 | 실제 도시/기관/회사 실명 | 폐쇄 문구, 경고 라벨, 번역 차이 |
| 영수증 | 결제 실패와 권리 상실을 흔적으로 확인 | 상점 기능, 보상 지급 설명 | 실패 내역, 환불 불가, 권리 흔적 |
| 대기열 기록 | 순번이 구조를 보장하지 않는 감각 확인 | 대면 순서 예약, NPC 등장 예고 | 밀린 순번, 사라진 열, 진입 기록 |

## 5. 금지 문구

금지 문구 유형:

1. `나는 누구다`처럼 본명을 확정하는 자기소개.
2. `나를 찾아와라`처럼 직접 대면을 유도하는 문장.
3. `상점`, `강화`, `해금`, `퀘스트`, `튜토리얼` 기능을 드러내는 문장.
4. 플레이어블 캐릭터의 능력, 해금 순번, 성장 기능을 설명하는 문장.
5. 실제 도시, 실제 기관, 실제 회사 실명을 유저 노출명으로 쓰는 문장.
6. E02 이후 권역에서 한국어 이름을 기본값처럼 붙이는 문장.
7. 캠페인 미개방 권역의 최종 결말, 반전, 범인을 확정하는 문장.
8. L3 이하 슬롯을 장문 과거사나 대면 독백으로 확장하는 문장.

허용 방향:

```text
상태값 후폭풍.
잘못된 믿음.
제도 흔적.
오브젝트가 남긴 결말 조각.
반복되는 자동 문구.
이름 없는 누락과 실패 기록.
```

## 7. 플레이어블 침범 방지 기준

```text

```
