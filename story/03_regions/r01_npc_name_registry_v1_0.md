# R01 NPC 이름 레지스트리 v1.0

## 문서 상태

```text
상태:
스토리팀/시나리오팀 기준 v1.0 후보

용도:
R01 핵심 NPC 12명의 코드명, 역할명, 내부 본명/원본 이름, 유저 노출 시점, 해금 조건을 고정한다.

판정:
통과 추천

주의:
이 문서는 코드명을 본명으로 교체하기 위한 문서가 아니다.
R01 초반 유저 노출은 여전히 코드명/역할명 우선이다.
본명은 역할에서 사람으로 돌아오는 틈이며, 해금되어도 즉시 구원을 뜻하지 않는다.
```

---

## 0. 결론

R01 핵심 NPC는 이름을 가져야 한다.

다만 이름은 첫 노출 정보가 아니라 후반 해금 정보다.

R01은 이름, 문패, 주소, 상담표, 가족사진이 사람을 역할로 바꾸는 지역이다.
따라서 유저가 처음 만나는 이름은 `ROOM-12`, `PORCH-LINE`, `FRAME-LEFT` 같은 코드명/역할명이어야 한다.

하지만 끝까지 코드명만 남기면 안 된다.

```text
사람이 역할에서 조금 빠져나오려면,
언젠가는 이름 조각이 필요하다.
```

---

## 1. 이름 확정 기준

```text
1. R01 핵심 NPC 12명 사이에서 같은 본명 금지.
2. 첫 시즌 주요 인물과 같은 이름 금지.
3. 윤서, 미나, 복희, 도윤, 리오, 세븐, 팝시와 호명감이 강하게 겹치는 이름 회피.
4. 같은 성씨는 가능하지만, 같은 이름/비슷한 음가 반복은 피한다.
5. MAIL-LOOP, VAC-0처럼 사람이 아닌 NPC는 본명을 붙이지 않는다.
6. FRAME-LEFT는 완전 본명보다 사진 밖에 맡길 이름 조각을 우선한다.
7. 이름은 성별/나이를 과하게 고정하지 않아야 한다.
```

## 2. v0.1 후보 조정 결과

v0.1에서 다음 후보는 조정한다.

```text
서도영 -> 서이겸
이유:
장도윤의 "도윤"과 도영/도윤 호명감이 붙는다.
OPEN-HOST는 플레이어블 전환 후보라 이름이 더 선명해야 한다.

임하윤 -> 임하온
이유:
윤서의 "윤"과 직접 겹치는 느낌을 줄인다.
NAME-PAINTER는 이름 오염 축이므로 유저 기억에서 더 깨끗하게 분리한다.

노윤재 -> 노기준
이유:
윤서/연우/하온과 음가가 몰리는 것을 피한다.
KEY-RING은 낡은 관리실/열쇠판 이미지가 강하므로 더 단단한 이름이 어울린다.
```

나머지는 유지한다.

## 4. 노출 규칙

### 4.1 초기 노출

초기에는 본명을 노출하지 않는다.

```text
OPEN-HOST
ROOM-12
PORCH-LINE
NAME-PAINTER
MAIL-LOOP
VAC-0
DRAIN-MARK
SIGN-BACK
KEY-RING
DINING-4
WELCOME-DESK
FRAME-LEFT
```

이유:

```text
R01은 사람을 이름보다 역할로 읽는 지역이다.
유저도 처음에는 이들을 역할로 보아야 한다.
그 불편함이 지역 감정이다.
```

### 4.2 중간 노출

중간에는 본명 전체가 아니라 이름 조각을 먼저 보여준다.

예:

```text
연...
지호
하온
...채율
태오
기준
가람
해원
아린
```

이름 조각은 다음 오브젝트에서 나온다.

```text
문패 조각
상담표 조각
순번표
열쇠 이름표
사진 뒤쪽 손글씨
수신 로그
복희 이름 보관함
```

### 4.3 후반 노출

후반에는 코드명과 본명을 병기할 수 있다.

예:

```text
ROOM-12 / 강연우
PORCH-LINE / 백지호
WELCOME-DESK / 정해원
FRAME-LEFT / 아린
```

주의:

```text
본명 해금은 역할 해방의 틈이지, 즉시 구원이 아니다.
본명을 알게 된 뒤에도 캠페인 농도나 흔적 보존 상태가 낮으면 위험이 남는다.
```

## 5. NPC별 이름 해금 의미

### OPEN-HOST / 서이겸

```text
해금 의미:
길과 방을 안내한 역할 뒤에 사람이 있었다는 확인.

위험:
본명이 너무 빨리 나오면 OPEN-HOST의 불편한 거리감이 약해진다.
```

### ROOM-12 / 강연우

```text
해금 의미:
가족대표가 아니라 식탁에 앉아 있던 사람으로 돌아오는 첫 틈.

위험:
본명을 말한다고 식량태그/거주태그 문제가 자동 해결되면 안 된다.
```

### PORCH-LINE / 백지호

```text
해금 의미:
순번을 부르는 사람이 순번 밖에서 불릴 수 있게 되는 순간.

위험:
질서의 얼굴이 단순 선량한 피난민 대표가 되면 안 된다.
```

### NAME-PAINTER / 임하온

```text
해금 의미:
남의 이름을 덧칠하던 사람이 자기 이름도 문패 밖에 둘 수 있게 되는 순간.

위험:
이름 수집가나 감성 NPC로 흐르면 안 된다.
```

### DRAIN-MARK / 문채율

```text
해금 의미:
완전히 숨는 사람에서 찾을 수 있는 흔적을 남기는 사람으로 이동.

위험:
보급소 이주나 자유 찬양으로 끝나면 안 된다.
```

### SIGN-BACK / 권태오

```text
해금 의미:
표지판을 고치던 사람이 진짜 회수선 기준을 믿기 시작하는 순간.

위험:
OPEN-HOST 비난용 피해자로 축소되면 안 된다.
```

### KEY-RING / 노기준

```text
해금 의미:
열쇠가 주인을 증명한다는 믿음에서, 확인 대기라는 방식을 받아들이는 순간.

위험:
낡은 세계 찬양이나 열쇠 상인화 금지.
```

### DINING-4 / 류가람

```text
해금 의미:
네 번째 자리가 아니라 네 번째 자리에 앉아 있던 사람으로 분리되는 순간.

위험:
단순 구출 대상이나 ROOM-12 반복으로 보이면 안 된다.
```

### WELCOME-DESK / 정해원

```text
해금 의미:
담당자나 상담원이 아니라, 절차 안에서 보류하려던 사람으로 보이는 순간.

위험:
친절한 접수원이나 OPEN-HOST 복제 캐릭터가 되면 안 된다.
```

### FRAME-LEFT / 채아린

```text
해금 의미:
사진 속 역할이 아니라 사진 밖에 맡길 수 있는 이름 조각이 생기는 순간.

위험:
사진 속 유령이나 감성 회상 NPC가 되면 안 된다.
```

## 6. 본명이 없는 NPC

### MAIL-LOOP

```text
판정:
본명 없음.

이유:
MAIL-LOOP는 사람이 아니라 주소/배송/수신 기록 잔향이다.
사람 이름을 붙이면 주소 축이 흐려진다.
```

### VAC-0

```text
판정:
본명 없음.

이유:
VAC-0는 모델명/장치명으로 유지한다.
사람 이름이나 귀여운 별명은 마스코트화 위험이 있다.
```

## 7. 최종 잠금

```text
1. R01 핵심 NPC 12명 중 사람형/잔향형 NPC의 이름 후보는 중복 없음.
2. 첫 노출은 코드명/역할명 우선.
3. 본명은 흔적 보존/복희 이름 보관/보스 후 기억 추출 이후 해금.
4. MAIL-LOOP, VAC-0는 본명 없음.
5. FRAME-LEFT는 채아린 전체보다 "아린" 이름 조각 우선.
6. 본명 해금은 즉시 구원이 아니라, 역할에서 사람으로 돌아오는 첫 틈이다.
```
