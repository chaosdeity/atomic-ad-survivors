# 전역 NPC 수량 충족 매트릭스 v0.1

## 문서 상태

```text
상태:
작성 완료 v0.1

작업 성격:
전역 NPC/잔향/기록/배경 슬롯의 목표 수량을 96개 캠페인 전체에 재배치하는 수량 충족 문서.
새 캠페인 작성 아님.
모든 NPC 상세 프로필 작성 아님.
E01 한국권 NPC 추가 생산 아님.

판정:
전역 수량 충족
```

이 문서는 기존 `420개 전역 씨앗`을 폐기하지 않는다.
대신 420개를 `빈 캠페인 방지용 1차 게이트`로 두고, 실제 전역 규모감의 만족 기준을 `768개 제작 슬롯`으로 올린다.

핵심:

```text
768개는 전부 직접 대면 NPC가 아니다.
768개는 L5/L4/L3/L2/L1/L0를 합친 전역 제작 슬롯이다.
본명 후보는 일부 L5/L4에만 허용한다.
L3 이하는 역할명, 로그명, 잔향명, 오브젝트명, 목록명, 자동 문구로 처리한다.
```

## 1. 기존 420개와 목표 768개의 차이

기존 420개는 `96개 캠페인 중 빈칸을 없애는 최소 네트워크`다.
만족 기준 768개는 `실제 시즌이 순차 오픈될 때 각 캠페인이 대사, 잔향, 오브젝트, 작은 결말을 받을 수 있는 제작 슬롯`이다.

| 구분 | 의미 | 사용 |
|---|---|---|
| 420개 | 1차 씨앗 | 캠페인 빈칸 방지, 전역 연결 최소 보장 |
| 768개 | 만족 기준 슬롯 | 시즌 오픈 순서에 따른 실제 제작 대기열 |
| 1,152개 | 금지 상한 | 96개 캠페인 x 12명 직접 NPC 방식. 만들지 않음 |

768개가 필요한 이유:

```text
각 캠페인에는 직접 대면 NPC가 없어도 상태 반응, 로그, 명단, 자동 문구, 오브젝트 잔향이 필요하다.
420개는 전체 지도를 비우지 않는 최소치지만, 모든 시즌에서 반복 사용하기에는 얇다.
768개는 캠페인당 평균 8개 슬롯을 제공하되, 직접 대면과 본명은 제한한다.
```

## 7. 전역 수량 운영 원칙

1. E01은 120개에서 멈춘다.
2. 전역 증분은 E02~E12와 cross_region에만 붙인다.
3. 증분 슬롯은 이름이 아니라 캠페인 절차, 오브젝트, 로그, 잔향, 작은 결말을 먼저 가진다.
4. L5/L4 120개만 상세 후보로 관리하고, 그중 실제 v1.0은 시즌당 1~3명만 올린다.
5. L3 이하 648개가 세계 밀도를 담당한다.
6. 본명은 수량 충족의 수단이 아니다.
7. 캠페인 미개방 지역은 상세 프로필이 아니라 슬롯만 예약한다.
8. 시즌이 열리면 해당 권역의 현실권/언어권 검수를 다시 거친다.
