# 전역 NPC 이름/원점 검수 v0.1

## 문서 상태

```text
상태:
작성 완료 v0.1

작업 성격:
전역 NPC 제작 전에 캠페인 ID, 현실권/언어권, 계층, 이름 필요 여부를 검수하는 게이트 문서.
새 NPC 대량 생산 아님.
E01 한국권 NPC 추가 생산 아님.
새 플레이어블 프로필 작성 아님.

판정:
통과
```

이 문서는 `E01 한국권 NPC를 계속 늘리는 흐름`을 차단하고, E02 이후 NPC가 각 캠페인의 현실권/언어권/등록 절차를 먼저 통과한 뒤 만들어지도록 잠근다.

후속 적용:

```text
story/03_regions/global_npc_quantity_satisfaction_matrix_v0_1.md
story/03_regions/global_npc_expansion_slot_registry_v0_1.md
```

위 두 문서의 768개 제작 슬롯 기준에도 이 검수 원칙을 그대로 적용한다.

핵심 판정:

```text
E01은 초기 보급소, R01, 첫 시즌 이해에 필요한 최소 기준점으로 유지한다.
전역 NPC는 캠페인이 실제로 열리는 순서에 따라 E02~E12에서 만든다.
E02 이후 후보에 한국어 임시 본명을 붙여 빈 슬롯을 채우지 않는다.
L5/L4만 이름 후보를 검토한다.
L3 이하는 역할명, 직책명, 잔향명, 오브젝트명, 목록명으로 처리한다.
```

## 2. 캠페인 개방 게이트

NPC를 만들기 전에 아래 순서를 확인한다.

| 순서 | 확인 항목 | 실패 시 처리 |
---:|---|---|
| 1 | 캠페인 ID가 `GLOBAL_CAMPAIGN_CATALOG_V0_1.md`에 있는가 | 새 캠페인 생성 금지, 기존 96개 중 하나로 재배정 |
| 2 | 해당 캠페인이 실제 슬라이스/시즌 작업으로 열렸는가 | L5/L4 상세화 보류, G3 이하 씨앗/잔향만 유지 |
| 3 | 현실권/언어권/등록 절차가 확인되었는가 | 이름 재검토 또는 현지권 재검토 |
| 4 | NPC 계층이 L5/L4인가 | L3 이하는 본명 없이 처리 |
| 5 | 플레이어블 기능과 겹치지 않는가 | 기능 삭제, 계층 하향, 보류 중 하나 |
| 6 | 미스터리/반전/작은 결말 중 하나를 담당하는가 | 기능 없는 이름이면 폐기 |

현재 개방 판정:

| 범위 | 판정 | 허용 |
|---|---|---|
| E01 | 초기 기준점 완료 | 기존 문서 유지, 추가 과밀 생산 금지 |
| E02 | 첫 외부 슬라이스 진행 | C03/C05/C07 중심, 일부 C01/C04/C08은 잔향/오브젝트 수준 |
| E03~E12 | 전역 씨앗 단계 | G3/G2/G1/G0 유지, L5/L4 상세화 보류 |
| cross_region | 장기 미스터리 단계 | 본명보다 로그/명단/오브젝트 우선 |

## 5. E02 이후 필수 질문 답변

### 5.1 왜 한국어 이름이면 안 되는가

E02 이후 캠페인은 E01 한국권이 아니다.
한국어 본명을 임시로 붙이면 해당 NPC가 E01 출신처럼 읽히거나, 전역 캠페인의 현실권/언어권이 제작 편의상 지워진다.

허용되는 예외:

```text
이주, 피난, 국제물류, 가족 이산, 다국적 노동, 장기 체류 등 명확한 캠페인 이유가 있을 때.
그 경우에도 "현지권 재검토"를 통과하기 전 본명 확정 금지.
```

### 5.2 해당 현실권/언어권에 맞는가

맞아야 한다.
하지만 "전형적 현지 이름"을 붙이는 것이 답은 아니다.
관리자 필드에서 현실권, 언어권, 이주권, 인프라 앵커를 먼저 확인하고, 유저에게는 캠페인식 역할명으로 노출한다.

### 5.3 이름이 정말 필요한 계층인가

L5/L4만 검토한다.
L3 이하는 이름이 아니라 기능이 먼저다.

```text
L5/L4:
이름 후보 검토 가능.

L3:
대사 은행/상태 반응명으로 처리.

L2:
잔향/로그/안내/오브젝트명으로 처리.

L1/L0:
목록/번호/자동 문구로 처리.
```

### 5.4 이름 없이 오브젝트/로그/잔향으로 처리해도 되는가

대부분은 그렇다.
특히 E02 이후 첫 노출은 사람 이름보다 등록 절차, 안내 문구, 실패 로그, 명단 조각이 먼저다.

### 5.5 플레이어블 기능을 빼앗고 있지 않은가

NPC는 플레이어블의 해금, 전투, 성장, HP 0 조건, 고유 시설 기능을 설명하거나 대체하지 않는다.
침범 위험이 있으면 아래 중 하나로 처리한다.

```text
계층 하향
잔향 처리
오브젝트 처리
보류
폐기
```

### 5.6 작은 결말, 미스터리, 반전 중 무엇을 담당하는가

모든 후보는 아래 중 하나 이상을 가져야 한다.

| 역할 | 예 |
|---|---|
| 작은 결말 | 보관 연장 거부, 회수 기록 보존, 배급명단 밖의 생존 |
| 미스터리 | 늦은 안내 방송, 이름 삭제 명찰, 미수령품 기록 |
| 반전 | 통행 불가가 안전지대가 됨, 정품 인증 실패가 구조가 됨 |
| 상태 후폭풍 | 수신태그 하락, 캠페인 농도 상승, 인간 안정도 하락 |

기능이 없는 이름 후보는 폐기한다.
