# R01 NPC 이름 해금 매트릭스 v0.1

## 문서 상태

```text
상태:
스토리팀/시스템팀 연결용 v0.1

용도:
R01 핵심 NPC의 코드명, 중간 이름 조각, 본명/원본 이름 해금 조건, 연결 상태값, 해금 후 위험을 표로 관리한다.

판정:
필요

주의:
이름 해금은 수집 보상이 아니다.
이름은 역할에서 사람으로 돌아오는 첫 틈이며, 캠페인 농도와 흔적 보존 상태에 따라 위험도 함께 열린다.
```

---

## 0. 기본 원칙

```text
이름을 안다 = 해방 완료가 아니다.
이름을 안다 = 그 사람을 역할이 아니라 사람으로 부를 수 있는 조건 하나가 생긴다.
```

R01의 이름 해금은 다음 세 단계를 따른다.

```text
1. 코드명/역할명:
초기 인식.

2. 이름 조각:
흔적 보존, 문패 조각, 상담표 조각, 사진 뒤쪽 손글씨, 복희 이름 보관함에서 일부 노출.

3. 본명/원본 이름:
선택 분기와 상태값 조건을 만족했을 때 코드명과 병기.
```

## 2. 복희 연결 우선순위

복희 이름 보관함과 직접 연결할 NPC:

```text
1. ROOM-12 / 강연우
2. NAME-PAINTER / 임하온
3. FRAME-LEFT / 아린
4. KEY-RING / 노기준
5. DINING-4 / 류가람
```

간접 연결:

```text
PORCH-LINE / 백지호
WELCOME-DESK / 정해원
SIGN-BACK / 권태오
DRAIN-MARK / 문채율
```

연결 제외:

```text
MAIL-LOOP
VAC-0
```

## 3. 세븐 연결 우선순위

세븐 수신/기록 분석과 직접 연결할 NPC:

```text
1. MAIL-LOOP
2. FRAME-LEFT / 아린
3. WELCOME-DESK / 정해원
4. SIGN-BACK / 권태오
5. DRAIN-MARK / 문채율
```

세븐이 할 수 있는 것:

```text
기록 대조
수신 로그 분석
원본/보정 기록 분리
가짜 비콘음 비교
주소/상담표/사진 기록 충돌 표시
```

세븐이 하면 안 되는 것:

```text
의도 판정
본명 확정
사람 구원 판정
```

윤서의 보류 선택과 직접 연결할 이름 해금:

```text
DINING-4:
네 번째 자리 보류 후 류가람 이름 조각 해금.

WELCOME-DESK:
상담 질문 보류 후 정해원 이름 조각 해금.

FRAME-LEFT:
사진 빈칸 보류 후 아린 이름 조각 해금.
```

## 5. 캠페인 농도별 이름 위험

### 낮음

```text
이름 조각이 비교적 사람의 흔적으로 남는다.
복희 보관 성공률이 높다.
세븐 분석 잡음이 낮다.
```

### 중간

```text
이름과 역할이 함께 노출된다.
본명 해금과 역할 호출이 섞일 수 있다.
```

예:

```text
강연우 / 가족대표
류가람 / 네 번째 가족 구성원
정해원 / 상담 담당
```

### 높음

```text
본명이 캠페인 호출 자료로 오염될 수 있다.
이름을 바로 말하는 것보다 보관/보류가 우선된다.
```

위험 대사 후보:

```text
"본명 확인.
역할 재배정을 진행합니다."
```

## 6. UI 표기 권장

초기:

```text
ROOM-12
PORCH-LINE
NAME-PAINTER
```

중간:

```text
ROOM-12 / 이름 조각: 연...
PORCH-LINE / 이름 조각: 지호
NAME-PAINTER / 이름 조각: 하온
```

후반:

```text
ROOM-12 / 강연우
PORCH-LINE / 백지호
NAME-PAINTER / 임하온
```

FRAME-LEFT 예외:

```text
FRAME-LEFT / 아린
```

주의:

```text
FRAME-LEFT / 채아린
```

이 표기는 후반 최종 선택 이후에만 사용한다.
초기에는 사진 속 정답처럼 보일 수 있으므로 피한다.
