## 3. 플레이어블 상태 잠금

| 항목 | 잠금값 | 스토리 제작에서의 의미 | 금지되는 해석 |
|---|---|---|---|
| 출시 초기 플레이어블 | 윤서 단독 U5 시작 | 1차 스토리는 윤서 단독 시작 구조를 기준으로 작성 가능 | +1 플레이어블 추가, 후보 당김 |
| 플레이어블 신규 제작 | 없음 | 새 코드명/새 이름/새 원점 캠페인을 만들지 않음 | 새 후보 작성, 새 상세 결재 |

| 기존 프로필 v1.0 | 기능 산출물 보존 | 이미 있는 산출물은 참고 가능 | 신규 프로필 v1.0 작성, 출시 초기 해금 확정 |
| E02 이후 이름 원칙 | 한국어 이름 기본값 금지 | 외부 광역권은 현실권/언어권 재검토 전 빈 슬롯을 한국어 이름으로 채우지 않음 | E02 이후 새 한국어 기본명 추가 |

## 8. 스토리 제작에서 사용 가능한 범위

| 사용 가능 범위 | 허용 방식 | 금지되는 확장 |
|---|---|---|
| 윤서 단독 U5 시작 구조 | 1차 스토리의 플레이어블 기준으로 사용 | 신규 플레이어블 추가, +1 후보 확정 |
| 기존 통과 NPC/오브젝트/잔향/문구 | 이미 통과한 구조의 맥락 참조 | 새 NPC 이름, 새 직접 대면 이벤트 추가 |
| E01 120 고정 NPC 구조 | E01 내부 관계와 기존 기능축 참조 | E01 추가 생산 |
| 전역 L3 이하 240개 은행형 JSON/핸드오프 항목 | 배경 반응, 오브젝트, 잔향, 목록, 호출번호의 후보로 참조 | 1,856개 전체 작성, 런타임 구현 |
| L5/L4 192개 | 직접 대면 후보 최대치로만 참조 | 새 승격, 새 이름 추가, 상세 결재 재개 |
| 10,240 장기 목표 | 장기 확장 규모 감각으로만 참조 | 현재 전체 문구 작성 착수 |

판정: 스토리 제작은 기존 잠금 구조를 소재와 제약으로 사용할 수 있다. 새 제작 라인을 여는 근거로 사용할 수는 없다.

## 9. 재개/확장 조건
