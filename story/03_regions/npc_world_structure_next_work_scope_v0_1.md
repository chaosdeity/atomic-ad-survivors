## 1. 최종 판정

## 4. NPC/월드 선행 잠금

| 잠금 | 유지 기준 |
|---|---|
| E01 | 120 고정, 추가 생산 없음 |
| 전역 슬롯 | 2,048 슬롯 유지 |
| L5/L4 | 192개만 이름/직접 대면 후보 유지 |
| L3 이하 | 1,856개는 대사/잔향/오브젝트/호출번호/목록 등 은행형 표현 유지 |
| 현재 적용 대상 | 240개 샘플/티켓까지만 적용 검토 |
| 전체 작성 제한 | 1,856개 전체 문구 작성 금지 |
| 직접 대면 제한 | 새 NPC, 새 이름, 새 직접 대면 이벤트, L5/L4 증가 금지 |
| JSON 생성 제한 | 조건 해소 문서 완료 전 실제 data/ 디렉터리 또는 .json 파일 생성 금지 |
| 엔진 연결 제한 | 240개 JSON은 조건 해소 후에도 런타임 직결이 아니라 엔진/레벨 변환 전 중립 소스 데이터로만 취급 |
| 플레이어블 침범 제한 | NPC/월드 필드에 플레이어블 해금 조건처럼 읽히는 내용을 넣지 않음 |

## 5. 진행 가능 작업 목록

| 우선 | 작업 후보 | 진행 가능 범위 | 금지선 |
|---:|---|---|---|
| 1 | 전역 NPC L3 이하 240개 JSON 파일 생성 조건 해소 완료 여부 확인 | 생성 경로, 파일 분리 단위, 중립 소스 데이터 역할, 로더 비연결, 검증 절차, 변경 범위 승인 여부를 확인한다. | 실제 JSON 생성 금지 |
| 2 | 전역 NPC L3 이하 240개 실제 JSON 파일 생성 | 조건 해소 문서 완료 후에만 240개 샘플/티켓을 승인된 경로와 단위로 생성한다. | 240개 초과, 1,856개 전체 작성, 런타임 직결 금지 |
| 3 | 전역 NPC L3 이하 240개 JSON 생성 후 QA | parse/count/reference/sample_id 1:1/staged diff/금지 플래그를 검증한다. | 새 NPC, 새 이름, 직접 대면, L5/L4 증가 금지 |
| 4 | 전역 NPC L3 이하 240개 레벨/UI/오디오 핸드오프 적용 티켓 분배 | 기존 240개 적용 티켓을 레벨, UI, 오디오, 오브젝트, QA 소유 단위로 넘긴다. | 새 대사 확장이나 전체 은행 작성 금지 |
| 5 | 전역 NPC L5/L4 192개 직접 대면 후보 재검토 | 새 이름 추가가 아니라 기존 통과 슬롯의 현실권/언어권/직접 대면 위험만 검토한다. | L5/L4 증가, 신규 대면 캐릭터 추가 금지 |
| 6 | E03 이후 지역별 NPC 은행 구조 점검 | 지역별 L3 이하 은행이 이름/대면 없이 월드 반응으로 유지되는지 점검한다. | 새 NPC 양산 금지 |
| 7 | 전세계 캠페인 미스터리/반전/작은 결말 연결 슬롯 점검 | 플레이어블 없이 NPC/오브젝트/잔향 연결로만 캠페인 단서를 점검한다. | 플레이어블 해금, 미개방 후보 상세화 금지 |

## 7. 우선순위 추천

| 우선순위 | 추천 작업 | 판정 |
