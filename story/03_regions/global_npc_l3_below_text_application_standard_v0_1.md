## 1. 적용 목적

이 문서의 목적은 240개 샘플 문구를 바로 전체 콘텐츠로 확산시키는 것이 아니라, 어느 팀이 어떤 단위로 받아야 하는지 정하는 것이다.

적용 목적:

1. 은행형 문구가 직접 대면 NPC로 오해되지 않게 한다.
2. 레벨 오브젝트, 로그, 표지, 자동 음성, 상태 반응의 배치 기준을 나눈다.
3. 스토리팀이 작은 결말과 미스터리를 확정하지 않고 암시로만 쓰게 한다.
4. 레벨팀이 실제 도시/기관/회사 실명 없이 지역 절차와 물성으로 권역감을 살리게 한다.
5. 240개 표본 적용이 1,856개 전체 적용으로 번지지 않게 한다.

표 1:

| 항목 | 기준 | 금지 |
|---|---|---|
| 적용 총량 | 240개 1차 샘플만 적용 후보로 본다 | 240개 초과 적용, 1,856개 전체 적용 |
| 적용 대상 | L3/L2/L1/L0 은행형 문구 | L5/L4 직접 대면 후보 증가 |
| 적용 방식 | 상태 반응, 오브젝트, 잔향, 로그, 표지, 목록, 자동 음성 | 직접 대면 이벤트, 퀘스트 배포기화 |
| 이름 정책 | L3 이하 본명 없음, 역할/번호/오브젝트 중심 | 새 이름 추가, E02 이후 한국어 기본명 |
| 지역 표현 | 절차, 오브젝트, 안내 방식, 실패 기록으로 표현 | 실제 도시/기관/회사 실명 노출 |
| 스토리 처리 | 작은 결말과 미스터리 암시만 허용 | 캠페인 결말 확정 |
| 플레이어블 관계 | 플레이어블이 남긴 상태값 후폭풍만 표현 | 플레이어블 해금 조건/기능 설명 |
| 엔진 처리 | 데이터 적용 전 후보 필드와 위치만 지정 | JSON/CSV/Godot 리소스 즉시 생성 |

## 2. 적용 단위

적용 단위:

```text
sample_id -> source_slot_id -> 캠페인 -> 계층 -> 표현 방식 -> 상태 키 -> 적용 위치 -> 전달팀
```

단위 기준:

1. `sample_id`는 문구 본문을 식별하는 최소 단위다.
2. `source_slot_id`는 1,856개 은행 슬롯 중 어떤 묶음에서 왔는지 확인하는 출처다.
3. `캠페인`은 적용 가능한 지역/레벨 묶음이며, 미개방 캠페인은 배치 후보로만 남긴다.
4. `계층`은 적용 깊이를 정한다. L3는 조건 반응, L2는 오브젝트/잔향, L1은 번호/목록, L0는 자동 문구/표지 중심이다.
5. `표현 방식`은 실제 레벨 배치 형태를 정한다.
6. `상태 키`는 표시 조건과 숨김 조건을 정한다.

판정:

```text
적용 단위는 NPC 한 명이 아니라 은행형 문구 슬롯이다.
어떤 적용 단위도 새 NPC, 새 이름, 직접 대면 씬으로 승격하지 않는다.
```

## 3. 적용 금지선

금지선:

```text
240개 초과 적용 금지.
1,856개 전체 적용 금지.
새 NPC 추가 금지.
새 이름 추가 금지.
직접 대면 이벤트 추가 금지.
L5/L4 증가 금지.
플레이어블 해금 설명문으로 사용 금지.
실제 도시/기관/회사 실명 노출 금지.
캠페인 결말 확정 금지.
```

보류 조건:

1. 문구가 직접 플레이어에게 말 거는 문장처럼 읽히면 보류한다.
2. 오브젝트 없이 사람 대사처럼만 성립하면 보류한다.
3. 상태 키 없이 항상 노출되어 결말을 확정하면 보류한다.
4. 현실권/언어권이 이름이나 실명에 기대면 보류한다.
5. 플레이어블 해금/성장/전투 기능 설명처럼 읽히면 보류한다.

## 5. 계층별 적용 방식

| 계층 | 적용 방식 | 위치 예시 | 상태 키 필요 | 금지 |
|---|---|---|---|---|
| L3 | 조건 대사 은행, 상태 반응 텍스트, 비대면 방송 조각 | 귀환 후 UI 로그, 지역 상태 반응 패널, 조건부 단말 문구 | 필요 | 직접 대면 대사, 이름 있는 NPC 대사 |
| L2 | 잔향, 오브젝트, 자동 음성, 로그, 표지, 영수증 | 레벨 오브젝트, 벽면 안내, 단말 로그, 반복 음성 트리거 | 필요 | 상호작용 NPC화, 긴 설명문 |
| L1 | 호출번호, 목록, 대기열 기록 | 게시판, 명단 UI, 번호표, 대기열 단말 | 조건부 필요 | 본명 목록, 구출 대상 명단 확정 |
| L0 | 자동 문구, 표지, 빈칸, 시스템 흔적 | 배경 표지, 자동 안내, 비활성 단말, 라벨 | 선택 | 스토리 결말 확정, 특정 인물 암시 과다 |

판정:

```text
계층이 낮을수록 인물성은 줄이고 물성/절차/번호성을 높인다.
L3도 사람 대면이 아니라 조건 반응 은행으로만 적용한다.
```

## 6. 표현 방식별 적용 방식

표 2:

| 표현 방식 | 적용 위치 | 상태 키/플래그 필요 여부 | 레벨팀 전달 방식 | 금지 |

| 표지 | 경고문, 폐쇄 라벨, 급수 제한, 검역 색상 표식 | 선택 | 표지 위치, 시야 우선도, 언어권 힌트 전달 | 실제 도시/기관/회사 실명 |
| 영수증 | 결제 실패, 환불 불가, 급수권, 권리 흔적 | 필요 | 획득 여부, 줍기 금지 여부, UI 확대 여부 전달 | 재화/상점 기능 설명 |
| 대기열 기록 | 진입 기록, 피난 순서, 계산대/도로/진료 대기열 | 조건부 필요 | 큐 화면, 순번 범위, 갱신 여부 전달 | 생존자 명단 확정, 직접 구조 이벤트 |

판정:

```text
표현 방식은 모두 레벨팀 전달 가능한 배치 형태로 환산된다.
어떤 표현 방식도 직접 대면 NPC나 새 기능 제공자로 환산하지 않는다.
```

## 7. 상태 키/플래그 연결 기준

상태 키 연결 기준:

| 상태 키 축 | 연결 용도 | 표시 조건 | 숨김/보류 조건 |
|---|---|---|---|
| `campaign_density` | 캠페인 압력 상승, 지역 혼잡, 안내 불일치 | 캠페인 농도 상승 후 | 결말을 직접 말할 때 |
| `human_stability` | 인간 안정도 저하, 대기열 붕괴, 생존권 흔들림 | 안정도 낮음 | 특정 인물 사망 확정처럼 보일 때 |
| `robot_autonomy` | 자동 장치 판단, 로봇/시스템 우선 처리 | 자동성 상승 후 | 플레이어블 로봇 기능 설명으로 읽힐 때 |
| `identity_mismatch` | 등록 실패, 명단 불일치, 접근 거절 | 신원/등록 충돌 발생 후 | 본명 공개가 필요할 때 |
| `route_access` | 통행권, 환승, 피난로, 선석/게이트 | 경로가 열리거나 막힌 뒤 | 새 퀘스트 길 안내가 될 때 |
| `supply_pressure` | 배급, 급수, 냉방, 보급 순서 | 보급 압력 상승 후 | 보급소 NPC 기능 대체처럼 보일 때 |
| `payment_failure` | 결제 실패, 환불 불가, 송금 지연 | 결제/권리 충돌 후 | 상점/경제 기능 설명으로 보일 때 |
| `trace_retained` | 물건/기록/표식이 사람보다 오래 남음 | 흔적 보관 후 | 인물 정체를 확정할 때 |
| `trace_spent` | 흔적 소모, 권리 소진, 증거 상실 | 흔적 소모 후 | 보상 지급 설명으로 보일 때 |
| `broadcast_trust` | 방송 신뢰 붕괴, 안내 불일치 | 방송 신뢰 하락 후 | 진실 확정 방송처럼 보일 때 |
| `quarantine_delay` | 검역 지연, 봉쇄, 통과 대기 | 검역 플래그 활성 후 | 실제 기관 절차를 실명화할 때 |
| `return_loop` | 반품/회수/배송 루프 | 회수 경로 반복 후 | RETURN 플레이어블 기능 설명으로 보일 때 |

플래그 연결 기준:

```text
플래그는 문구 표시 여부만 제어한다.
플래그는 새 퀘스트, 새 NPC, 새 플레이어블 해금 조건을 만들지 않는다.
플래그 이름은 엔진 데이터 작성 전까지 문서용 후보로만 둔다.
```
