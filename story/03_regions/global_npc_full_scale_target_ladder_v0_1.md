# 전역 NPC 풀스케일 목표 사다리 v0.1

## 문서 상태

```text
상태:
작성 완료 v0.1

작업 성격:
전역 NPC/잔향/기록/배경 슬롯의 장기 목표 수량을 10배 스케일로 정의하는 문서.
새 캠페인 작성 아님.
새 플레이어블 작성 아님.
E01 한국권 NPC 추가 생산 아님.
직접 대면 NPC 10,240명 작성 아님.

판정:
장기 풀스케일 목표 설정
```

현재 1,024개 제작 슬롯은 전역 NPC 전체의 약 1/10로 본다.
이 문서는 장기 목표를 `10,240개 제작 슬롯`으로 정의하되, 본명/대면/장문 프로필을 대량 생산하지 않도록 계층과 파동을 잠근다.

핵심:

```text
10,240개는 직접 대면 NPC 수가 아니다.
10,240개는 L5/L4/L3/L2/L1/L0 전체 제작 슬롯이다.
L5/L4는 256개 이하로 묶고, 나머지 9,984개는 기능, 상태 반응, 잔향, 로그, 오브젝트, 목록, 자동 문구가 담당한다.
E01 한국권은 계속 120개 기준점을 유지한다.
```

## 0. 단계 요약

| 단계 | 목표 슬롯 | 현재 대비 | 의미 | 판정 |
|---|---:|---:|---|---|
| P0 | 420 | - | 1차 게이트. 빈 캠페인 방지 | 완료 |
| P1 | 768 | +348 | 전역 수량 충족 기준 | 완료 |
| P2 | 1,024 | +256 | 검수 전 최소 과충족 기준 | 완료 |
| P3 | 2,048 | +1,024 | 1차 검수 배치. 전역 NPC의 약 20% | 이번 확장 |
| P4 | 4,096 | +2,048 | 시즌 1~2 누적 제작 대기열 | 후속 |
| P5 | 10,240 | +6,144 | 장기 풀스케일 목표 | 장기 |

판정:

```text
사용자가 말한 "이제 1/10 생산된 것 같다"는 감각을 장기 목표로 반영한다.
1,024개를 10% 기준으로 두고, 풀스케일 목표를 10,240개 제작 슬롯으로 둔다.
다만 실제 다음 검수 배치는 2,048개까지만 올린다.
```

## 1. 풀스케일 계층 목표

| 계층 | 풀스케일 목표 | 비율 | 이름/대면 정책 |
|---|---:|---:|---|
| L5 핵심 | 64 | 0.6% | 이름 후보 가능. 직접 대면도 시즌별 제한 |
| L4 준핵심 | 192 | 1.9% | 이름 후보 조건부. 대다수는 역할명 우선 |
| L3 기능/상태 반응 | 768 | 7.5% | 본명 불필요. 조건 반응과 작은 결말 담당 |
| L2 잔향/로그/오브젝트 | 2,048 | 20.0% | 사람 이름 금지 또는 fragment_only |
| L1 배경/목록/호출 번호 | 3,584 | 35.0% | 본명 금지 |
| L0 제도 흔적/자동 문구 | 3,584 | 35.0% | 인물화 금지 |
| 합계 | 10,240 | 100% | 직접 대면 수로 해석 금지 |

핵심 제한:

```text
L5/L4 합계는 256개다.
풀스케일의 97.5%는 L3 이하가 담당한다.
따라서 10,240개는 프로필 10,240개가 아니라 월드 밀도 10,240 슬롯이다.
```


## 5. 다음 실제 생산 범위

다음 실제 확장은 풀스케일 10,240개 전체가 아니다.
검수팀에 넘기기 전 `2,048개 1차 검수 배치`까지만 올린다.

출력 문서:

```text
story/03_regions/global_npc_2048_pre_review_wave_v0_1.md
```

이 문서는 아래 조건을 만족해야 한다.

1. E01은 120개에서 더 늘리지 않는다.
2. 1,024에서 2,048까지 +1,024 슬롯만 추가한다.
3. 증분은 E02~E12와 cross_region에만 배정한다.
4. L5는 거의 늘리지 않는다.
5. L3 이하를 중심으로 상태 반응, 잔향, 로그, 오브젝트, 목록을 늘린다.
6. 검수팀은 2,048개 기준으로 하향/폐기/이름 제거를 수행한다.
