## 0. 판정 요약

퇴원 불가 심사 대리 얼굴은 R02 백색팔찌 격리권의 보스권 절차를 사람 얼굴로 보여주는 핵심 NPC다.

R02는 사람을 죽이려 하지 않는다.
R02는 사람을 계속 환자 상태로 살려 두려 한다.

이 NPC는 그 선의를 가장 위험한 형태로 말한다.

```text
살려 두면 붙잡는 것이 아니다.
```

판정:

```text
프로필 v1.0 유지.
직접 대면은 R02-L05 퇴원 불가 병동에서 제한적으로 허용.
하람의 퇴원 보류 전투와 해금 결절은 침범하지 않는다.
```

## 1. 한 줄 정의

```text
퇴원 불가 심사 대리 얼굴은 나가면 무너질 사람을 살려 둔다는 이유로, 사람을 끝나지 않는 환자 역할에 묶는 R02 절차의 얼굴이다.
```

## 3. 지역 질문

R02의 질문:

```text
치료가 끝나지 않았다는 말은 보호인가, 붙잡기인가?
```

이 NPC가 대표하는 세부 질문:

```text
나가면 무너질 사람을 계속 살려 두는 절차는 어디까지 보호이고, 어디서부터 감금인가?
```

보여줘야 하는 것:

1. R02는 악한 병원이 아니라, 사람을 환자 역할로 보존하는 캠페인이다.
2. 퇴원 승인은 자유의 문이 아니라 다른 등록 압력의 시작일 수 있다.
3. 하람의 "보류"는 이 절차를 부수는 것이 아니라 닫히는 순간을 늦추는 방식이다.

## 4. 핵심 오판

퇴원 불가 심사 대리 얼굴의 핵심 오판:

```text
퇴원 후 버틸 조건이 없으면, 퇴원시키지 않는 것이 사람을 살리는 일이다.
```

이 믿음은 일부 맞다.

```text
약품, 병상, 보호자 확인, 보험 심사가 끊기면 실제로 사람이 무너질 수 있다.
```

하지만 위험하다.

```text
살아 있음이 계속 환자 번호로만 유지되면, 사람은 나갈 준비를 할 수 없다.
```

대표 문장:

```text
"나가는 것보다 무너지는 쪽을 더 많이 봤습니다."
```

## 5. 보이는 역할과 실제 기능

### 보이는 역할

```text
퇴원 심사 대리.
병상/보험/보호자 확인을 검토하는 사람.
퇴원 불가 판정을 읽어 주는 얼굴.
```

### 실제 기능

```text
R02의 치료태그 압박과 퇴원 보류 후폭풍을 대표한다.
하람의 해금 전후로 "환자 상태에서 빠져나오는 일"의 비용을 보여준다.
R02-L05 보스권에서 선택이 닫히기 직전의 절차 언어를 제공한다.
```

이 NPC가 주면 안 되는 것:

```text
회복
치료
힐
안전한 퇴원 승인
하람 해금 조건
병원 보스 정답 설명
```


반응 단계:

| 조건 | 반응 | 구현 메모 |
|---|---|---|
| `treatment_tag` 부족 | 퇴원 심사가 자동으로 반려된다 | 진료태그는 돈이 아니라 치료 접근 조건임을 유지 |
| `medical_registration` 높음 | 이름보다 환자 번호와 병상 번호를 먼저 부른다 | 본명 공개 금지 |
| `human_stability` 낮음 | 퇴원보다 병상 재배정을 권한다 | 선의가 더 강해져 더 위험해짐 |
| `campaign_density` 높음 | 심사 문구가 친절하고 단정해진다 | 악당 말투 금지 |
| `supply_pressure` 높음 | 약품 보급 부족을 이유로 퇴원 순번을 닫는다 | 보급소/미나 기능 대체 금지 |
| `bed_reassign_pressure` 높음 | 빈 병상이 즉시 다른 환자 번호를 부른다 | 하람 HP 0 연출의 보조 |

## 7. 선택 후폭풍

### 선택 A. 퇴원 심사를 강제로 닫는다

결과:

```text
대상 하나는 R02 절차에서 빠져나올 수 있다.
하지만 빈 병상과 미처리 진료태그가 다른 사람에게 재배정된다.
human_stability가 흔들리고 bed_reassign_pressure가 상승한다.
```

NPC 대사:

```text
"나간 기록은 만들 수 있습니다. 버틸 기록은 누가 만듭니까?"
```

### 선택 B. 심사를 보류한다

결과:

```text
퇴원도 재입원도 확정되지 않는다.
하람의 방식과 연결되지만, 하람의 전투 기능을 대신하지 않는다.
trace_preservation과 선택지 유지가 오른다.
```

NPC 대사:

```text
"보류는 결정이 아닙니다. 그래서 제일 오래 사람을 붙잡습니다."
```

### 선택 C. 진료태그를 더 쓴다

결과:

```text
약품과 병상 접근이 잠시 열린다.
대신 R02의 환자 등록 신호가 선명해지고 campaign_density가 상승한다.
```

NPC 대사:

```text
"치료 접근이 승인되었습니다. 퇴원 심사는 그만큼 더 정확해집니다."
```

### 선택 D. 심사 기록만 보존한다

결과:

```text
즉시 구조 없음.
복희/세븐이 분석할 수 있는 퇴원 불가 판정 조각이 남는다.
```

## 8. 플레이어블 충돌 방지

| 대상 | 충돌 위험 | 방지 기준 |
|---|---|---|
| 윤서 | 현장 회수 판단을 R02 심사 NPC가 설명함 | 이 NPC는 사람을 데려오지 않는다. 윤서가 회수하고, 이 NPC는 퇴원 판정이 닫히는 압력만 보여준다 |
| PATCH | 의료 장비 수리나 병상 장치 수리 기능으로 이동 | 수리 기능 없음. 의료 장치와 병상은 오브젝트 반응으로만 사용 |
| 하람 | 퇴원 보류 전투/해금 기능 대체 | 핵심 금지. 하람은 판정을 비트는 플레이어블이고, 이 NPC는 닫히려는 판정의 얼굴이다 |
| 이루 | 반송/보관 기능과 혼선 | 사람을 물류 대상으로 보관하지 않는다. 병상 재배정만 다룬다 |
| 유건 | 의료 장비 전력/충전 문제로 이동 | 전력/충전 문제는 R04/유건 축으로 남긴다 |
| 여울 | 낮은 수신/침묵권 피난을 치료 퇴원 루트로 오해 | 침묵권이나 피난 루트 제안 금지 |
| 리오 | 정품 복구/객실 초기화 공포 선공개 | R02는 사람 환자 등록만 다룬다. 로봇 정품 복구 언급 금지 |
| 미카 / GATE-09 | 통과 판정/게이트 심사 반복 | 퇴원 심사는 의료/보험 판정이다. 출항/환승 심사와 분리 |
| 향후 10~12번 후보 | Nora의 방송 수신, Mara의 약관 회원, Rafaela의 태그 중개 축 침범 | 보험/진료태그는 R02 내부 압박으로 제한하고 글로벌 약관/태그 중개는 선점하지 않는다 |

## 9. 보급소/R01 NPC 충돌 방지

| 대상 | 충돌 위험 | 처리 |
|---|---|---|
| 미나 | 보급/약품 정산을 대신함 | 정산 금지. 미나는 귀환 후 진료태그와 보급 압력의 비용을 본다 |
| 장도윤 | 의료 장비 개조/강화 연결 | 강화 금지. 심사 스크린과 병동문은 절차 오브젝트 |
| 팝시 | 로봇 치료/충전과 혼선 | 충전/리롤 기능 없음 |
| 세븐 | R02 정답 분석 제공 | 세븐은 퇴원 심사 기록의 패턴을 분석할 수 있지만 해결책을 주지 않는다 |
| 복희 | 이름 보관 기능 대체 | 환자 이름이 아니라 환자 번호/퇴원 심사 조각만 남긴다 |
| ROOM-12 | "나오면 다른 사람이 무너짐" 구조 반복 | ROOM-12는 식량/거주태그, R02는 진료/병상/퇴원 심사로 분리 |
| WELCOME-DESK | 상담/접수 보류 반복 | R02는 의료 심사, R01은 거주 상담이다. 창구 언어를 분리 |

## 11. 비주얼/사운드 방향

비주얼:

```text
흰 가운보다 심사 조끼, 마스크, 투명 가림판, 팔찌 스캐너가 먼저 보인다.
얼굴은 선명하지만 명찰은 역할명으로만 보인다.
손에는 청진기가 아니라 퇴원 불가 판정 스탬프와 재심사 패널이 있다.
```

공간:

```text
퇴원 심사 데스크 뒤쪽에 잠긴 병동문.
퇴원 안내문은 밝지만, 문구가 너무 자주 갱신된다.
빈 병상 화면에는 "재배정 대기"가 떠 있다.
```

사운드:

```text
조용한 병동 안내음.
심사 스크린의 낮은 확인음.
팔찌 스캔이 실패할 때마다 심장박동처럼 반복되는 경고음.
```

금지 비주얼:

```text
미친 의사
악당 병원장
힐러 캐릭터
수술실 보스
```
