## 0. 판정 요약

최종 반품 심사 보류관은 R03 반품 회수 벨트의 보스권 절차를 사람 얼굴로 보여주는 핵심 NPC다.

R03은 사람을 물건처럼 보관하려는 지역이 아니다.
R03은 수취, 반송, 보관, 폐기 같은 물류 언어가 사람에게 붙어버린 지역이다.

이 NPC는 그 언어를 가장 차갑게 말한다.

```text
판정이 닫혀야 사람이 돌아간다.
```

판정:

```text
프로필 v1.0 유지.
직접 대면은 R03-L05 최종 반품 심사실에서 제한적으로 허용.
윤서 과거와 이루 전투 기능은 침범하지 않는다.
```

## 1. 한 줄 정의

```text
최종 반품 심사 보류관은 완료되지 않은 대상을 계속 다치게 둘 수 없다는 이유로, 사람의 목적지를 심사 번호로 닫으려는 R03 절차의 얼굴이다.
```

## 3. 지역 질문

R03의 질문:

```text
수취인이 없으면 사람은 어디에 보관되는가?
```

이 NPC가 대표하는 세부 질문:

```text
완료 판정이 나야만 돌아갈 수 있는 세계에서,
미완료로 남는 것은 보호인가 폐기 대기인가?
```

보여줘야 하는 것:

1. R03의 공포는 배송 농담이 아니라 목적지 판정의 폭력이다.
2. 반품 완료는 항상 구조가 아니다.
3. 윤서가 쓰는 보류와 반려의 기술은 R03의 차가운 절차에서 나온 언어를 다르게 쓰는 방식이다.

## 4. 핵심 오판

최종 반품 심사 보류관의 핵심 오판:

```text
판정이 닫혀야 사람이 돌아간다.
완료되지 않은 대상은 계속 다친다.
```

이 믿음은 일부 맞다.

```text
보관 기한이 길어질수록 사람과 물건은 더 위험한 라인으로 밀린다.
목적지가 없는 대상은 실제로 더 쉽게 잃어버린다.
```

하지만 위험하다.

```text
판정을 닫는 순간, 사람은 잘못된 목적지로 너무 정확하게 보내질 수 있다.
```

대표 문장:

```text
"완료되지 않은 물건은 계속 다칩니다."
```

## 5. 보이는 역할과 실제 기능

### 보이는 역할

```text
최종 반품 심사 담당자.
보류 상태를 끝내고 완료/반려/폐기 판정을 내리는 사람.
```

### 실제 기능

```text
R03의 배송상태 압력과 최종 판정 후폭풍을 대표한다.
윤서의 회수자 정체성을 설명하지 않고, 윤서가 마주해야 하는 절차의 원형을 보여준다.
이루의 반송 수취인 전투와 달리 "판정이 닫히는 순간"을 담당한다.
```

이 NPC가 주면 안 되는 것:

```text
배송 퀘스트
상자 보상
윤서 과거 회상 설명
이루의 반송 전투 기능
빠른 배송 우회
```

```text

```

반응 단계:

| 조건 | 반응 | 구현 메모 |
|---|---|---|
| `delivery_state` 오염 | 보류 대상이 더 정확한 오배송으로 밀린다 | MAIL-LOOP 인간판 금지 |
| `trace_preservation` 높음 | 라벨 아래 원래 목적지 조각이 남는다 | 복희 반응 가능 |
| `campaign_density` 높음 | 심사 스캐너가 더 빠르게 완료 판정을 내리려 한다 | 보스권 압박 |
| `supply_pressure` 높음 | 오래 보관된 대상부터 압축 대기라인으로 이동한다 | R03-L04 잔향 연결 |
| `passage_tag` 부족 | 출고 문이 열리지 않아 보류가 길어진다 | R07 통행 기능과 분리 |
| `robot_autonomy` 상승 | 보증 스티커와 정품 판정 언어가 섞인다 | PATCH/R04 선공개 금지 |

## 7. 선택 후폭풍

### 선택 A. 최종 반품 판정을 완료한다

결과:

```text
한 대상의 보류가 끝난다.
하지만 목적지가 틀렸다면 더 정확하게 잘못 보내진다.
delivery_state가 안정되는 대신 trace_preservation이 낮아질 수 있다.
```

NPC 대사:

```text
"완료되었습니다. 이제 더는 여기서 다치지 않습니다. 어디로 갔는지는 별도 기록입니다."
```

### 선택 B. 판정을 보류한다

결과:

```text
대상은 아직 배송 완료가 아니다.
보관 기한과 공급 압박이 늘지만, 원래 목적지 조각이 남을 수 있다.
```

NPC 대사:

```text
"보류는 친절하지 않습니다. 다만 잘못 끝내지 않는 방법일 수는 있겠군요."
```

### 선택 C. 라벨을 반려한다

결과:

```text
잘못된 목적지 판정을 막는다.
대신 R03의 캠페인 농도가 상승하고, 자동 분류장이 더 공격적으로 경로를 재계산한다.
```

NPC 대사:

```text
"반려된 대상은 돌아가지 않습니다. 돌아갈 곳을 다시 증명해야 합니다."
```

### 선택 D. 원래 흔적만 보존한다

결과:

```text
즉시 구조 없음.
윤서/복희/세븐이 읽을 수 있는 라벨 아래 이름 조각과 목적지 파편이 남는다.
```

## 8. 플레이어블 충돌 방지

| 대상 | 충돌 위험 | 방지 기준 |
|---|---|---|
| 윤서 | 윤서 과거 설명자 또는 반품 담당자 원본으로 고정 | 이 NPC는 윤서의 사고 전 직장 상사가 아니다. 윤서의 보류 감각이 마주하는 R03 현재 절차의 얼굴이다 |
| PATCH | 보증 스티커/정품 판정으로 로봇 부품화 서사 침범 | PATCH의 부품 윤리와 수리 기능은 침범하지 않는다. 보증 언어는 R03-R04 가교 흔적으로만 사용 |
| 하람 | R02 퇴원 심사와 판정 보류 구조 반복 | R03은 배송/수취/반송 판정, R02는 의료/퇴원 판정으로 분리 |
| 이루 | RETURN-05의 반송 수취인 전투 기능 반복 | 핵심 금지. 이루는 반송 수취인으로 전투/해금 기능을 가진다. 이 NPC는 최종 심사 절차의 압력만 담당 |
| 유건 | 정품/계량/충전 기능으로 확장 | R04 정품충전 공단 기능 선공개 금지 |
| 여울 | 침묵권 피난이나 인양 반경과 연결 | R03의 보관/배송 문제로 제한 |
| 리오 | 정품 복구/객실 정리 로봇 축 선공개 | 체류/숙박권과 객실 초기화 언급 금지 |
| 미카 / GATE-09 | 통과 심사/개찰 판정 반복 | R03의 심사는 물류 목적지 판정이다. 환승/막차/개찰 심사와 분리 |
| 향후 10~12번 후보 | Nora의 방송 수신, Mara의 약관 회원, Rafaela의 항만시장 태그 중개 축 침범 | R03은 E01 물류 회수권으로 제한하고 글로벌 항만시장/대형마트 약관 축을 선점하지 않는다 |

## 9. 보급소/R01 NPC 충돌 방지

| 대상 | 충돌 위험 | 처리 |
|---|---|---|
| 미나 | 반품 정산을 직접 처리 | 이 NPC는 정산하지 않는다. 미나는 귀환 후 라벨/태그/보관 기한의 비용을 본다 |
| 장도윤 | 스캐너/분류장 장비 개조 | 강화 기능 금지 |
| 팝시 | 로봇 보증/리롤 기능과 혼선 | 보증 스티커는 대사 은행 수준으로만 연결 |

| 복희 | 이름 보관 기능 대체 | 이 NPC는 이름을 보존하지 않는다. 라벨 아래 이름 조각만 남긴다 |
| MAIL-LOOP | 주소/배송 잔향 반복 | MAIL-LOOP는 주소 잔향, 이 NPC는 최종 반품 심사로 분리 |

## 11. 비주얼/사운드 방향

비주얼:

```text
물류 작업복보다 심사실용 방진 조끼와 얇은 장갑.
명찰에는 이름 대신 심사 권한 등급과 반품 번호가 보인다.
손에는 바코드 스캐너가 아니라 보류/완료/반려 도장이 있다.
```

공간:

```text
최종 반품 심사실은 창고라기보다 공항 심사대와 정산실이 섞인 느낌.
거대 스캐너, 심사 번호표, 반품 번호판, 닫힌 출고 문이 핵심이다.
```

사운드:

```text
컨베이어가 멈췄는데도 라벨 프린터가 계속 움직이는 소리.
완료 판정음은 밝지만 너무 짧다.
보류 도장은 둔하고 늦게 울린다.
```

금지 비주얼:

```text
택배 기사
반품 센터 점장
박스 상인
배송 개그 캐릭터
```
