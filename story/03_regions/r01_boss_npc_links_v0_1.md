## 0. 판정 요약

R01 첫 보스는 죽이는 적이 아니다.
보스는 R01의 가족 심사 절차와 지역 광고 약관이 얼굴을 얻은 결절이다.

이번 연결의 핵심은 아래처럼 고정한다.

| 축 | 담당 NPC/흔적 | 보스 연결 |
|---|---|---|
| 길 | OPEN-HOST | 보스 진입 동선, 추천 평면도, 안내 검증 |
| 질문 | WELCOME-DESK | 입주 상담, 가족 구성 질문, 질문 보류 |
| 자리 | DINING-4 | 보스 직전 빈 가족 칸, 기념사진 시간 |
| 배급/대표 | ROOM-12 | 가족대표 수령인, 식량태그 압박, DINING-4 압박 완화 |
| 줄/순번 | PORCH-LINE | 상담 순번표, 가족 심사 대기열, 보스 후 해산 압박 |
| 주소/호출 | MAIL-LOOP | 가족 수령 호출, 주소/사진 불일치 로그 |
| 이름/문패 | NAME-PAINTER | 이름이 역할로 붙는 속도, 문패/상담표 결합 |
| 사진/기억 | FRAME-LEFT | 보스 후 기억 추출, 가족사진 프레임 잔향 |

Vertical Slice 기준 판정:

```text
직접 구현:
OPEN-HOST
ROOM-12
PORCH-LINE
DINING-4

비직접 필수:
MAIL-LOOP
WELCOME-DESK 상담표/질문 흔적

보스 후 짧은 잔향:
FRAME-LEFT

흔적 선노출:
NAME-PAINTER
VAC-0

P1 직접 보류:
DRAIN-MARK
SIGN-BACK
KEY-RING
WELCOME-DESK 본격 대면
FRAME-LEFT 본격 분기
```

다음 작업 포인터는 **R01 보스 문서 본문 정리**가 먼저다.

```text
이유:
r01_boss.md는 보스 패턴과 선택 구조가 잡혀 있지만,
최신 NPC 플래그, Vertical Slice 범위, "가족심사 관리자" 기준이 본문에 아직 통합되지 않았다.
플레이어블 20인 로스터로 넘어가기 전에 보스 본문과 NPC 연결을 한 번 맞추는 편이 안전하다.
```

## 1. r01_boss.md 처리 판정

이번 작업에서 `story/03_regions/r01_boss.md`는 읽기만 했다.
수정하지 않았다.

읽기 결과:

| 항목 | 현재 초안 상태 | 이번 문서 판정 |
|---|---|---|
| 보스 정체 | 가족 상담형 홈 어시스턴트, 지역 결절 | 유지 가능 |
| 보스 기능 | 빈집/우편함/청소기/사진/문패를 모델하우스로 연결 | NPC 연결과 잘 맞음 |
| 전투 패턴 | 쿠폰, 청소기, 가족사진 플래시, 추천 평면도 | NPC 플래그 연결 필요 |
| 처치 후 선택 | 결절 파괴, 코어 해방, 기억 추출, 신호 우회 | NPC 반응/상태값 연결 필요 |
| 명칭 | 스마일 홈 시어머니 | 최신 문서 기준으로 재검토 필요 |
| 문구 | 보스 사망/처치 표현 일부 존재 | "결절 처리/무력화/심사 중단" 쪽으로 정리 필요 |

수정 필요성:

```text
필요함.
다만 이번 작업에서는 새 연결 문서를 먼저 만들고,
r01_boss.md 본문 정리는 다음 작업으로 넘긴다.
```

본문 정리 때 할 일:

```text
1. 보스 명칭을 "스마일 홈 가족심사 관리자" 기준으로 정리한다.
2. 기존 "스마일 홈 시어머니" 명칭을 유지할지, 광고식 별칭으로 낮출지 판단한다.
3. 보스 패턴마다 NPC 플래그 입력/출력을 붙인다.
4. 보스 선택 결과를 상태값과 NPC 후폭풍으로 연결한다.
5. "죽음"보다 "절차 중단/결절 처리/기억 추출" 언어를 우선한다.
```

## 2. 보스 핵심 기준

보스의 설계 기준:

```text
보스는 가족을 빼앗는 괴물이 아니다.
보스는 가족을 상품/절차/사진/주소/자리로 다시 읽는 관리자다.
```

보스가 하는 일:

| 절차 | 플레이 체감 | 연결 NPC |
|---|---|---|
| 입주 상담 | 질문에 답하면 역할이 생긴다 | WELCOME-DESK |
| 추천 동선 | 안전해 보이는 길이 보스 진입을 만든다 | OPEN-HOST |
| 가족 구성표 | 빈칸이 생기면 새 얼굴을 찾는다 | DINING-4, FRAME-LEFT |
| 대표 수령인 | 식량/침상/저녁이 사람을 묶는다 | ROOM-12 |
| 대기 순번 | 줄이 정확할수록 상담표가 사람을 읽는다 | PORCH-LINE |
| 주소 호출 | 주소가 사람보다 먼저 돌아온다 | MAIL-LOOP |
| 문패/이름 | 이름이 사람보다 역할에 붙는다 | NAME-PAINTER |

보스전 승리의 의미:

```text
보스를 죽여서 R01을 정화하는 것이 아니다.
가족 심사 절차를 멈추고,
이름/주소/자리/사진/질문을 서로 분리하기 시작하는 것이다.
```

## 3. 보스 전 진입 구조

Vertical Slice에서 보스 전 진입은 세 단계로 자른다.

| 단계 | 노출 | 핵심 체크 |
|---|---|---|
| P0 첫 30분 | PORCH-LINE, MAIL-LOOP, OPEN-HOST 흔적, NAME-PAINTER/VAC-0 흔적 | R01이 사람을 줄/주소/문패/청소 대상으로 읽기 시작한다 |
| 모델하우스 접근 | ROOM-12 직접 대면, OPEN-HOST 동선 검증, WELCOME-DESK 상담표 흔적 | 구출보다 조건 만들기가 먼저라는 압박을 준다 |
| 보스 직전 | DINING-4 네 번째 자리, 가족 구성 질문, 사진 프레임 흔들림 | 빈칸을 그대로 두는 일이 전투보다 먼저 어려운 선택이 된다 |

진입 플래그:

| 플래그 | 보스 진입 효과 |
|---|---|
| `open_host_route_verified` | 추천 평면도 패턴의 가짜 안전지대를 일부 줄인다 |
| `welcome_desk_question_hold` | 가족 구성 질문 자동 제출을 늦춘다 |
| `dining4_fourth_seat_hold` | 기념사진 시간의 빈 가족 칸 보정 압박을 낮춘다 |
| `room12_bypass_ration` | DINING-4가 잠깐 몸을 움직일 수 있는 배급 안정 근거가 생긴다 |
| `mail_loop_call_hold` | 가족 수령 호출이 보스 직전 대기열로 폭주하지 않는다 |
| `porch_line_flexible_ticket` | 정확한 순번표가 보스 심사 자료로 붙는 속도를 늦춘다 |
| `name_painter_names_to_bokhee` | 문패 이름이 상담표/주소/사진에 자동 결합되는 속도를 늦춘다 |

진입 실패 압박:

| 실패/보류 | 보스전 반영 |
|---|---|
| OPEN-HOST 동선을 무검증 수용 | 추천 평면도 패턴이 더 친절하게 보이지만 위험 구역으로 이어진다 |
| WELCOME-DESK 질문을 제출 | 가족 구성표가 플레이어를 더 빨리 빈칸 후보로 읽는다 |
| DINING-4를 강제로 일으킴 | 네 번째 자리가 비고 사진 보정 압박이 빨라진다 |
| ROOM-12를 강제 이동 | 배급/침상 공백이 보스 후 정산 위험으로 남는다 |
| MAIL-LOOP을 끔 | 주소 호출은 줄지만 증거 로그도 사라진다 |
| PORCH-LINE을 해산 | 사람은 흩어지지만 약한 사람이 먼저 밀린다 |
| 이름을 문패에 그대로 둠 | 보스가 이름을 사람보다 역할 태그로 읽는다 |

## 4. 보스 패턴과 NPC 연결

### 4.1 오늘의 가족 특가

기존 보스 패턴:

```text
쿠폰 탄막.
친절한 광고가 실제 위험으로 접혀 들어온다.
```

NPC 연결:

| 연결 | 반영 |
|---|---|
| PORCH-LINE | 정확한 순번표가 있으면 쿠폰 표식이 번호/차례처럼 읽힌다 |
| WELCOME-DESK | 상담표 질문이 제출되어 있으면 쿠폰 문구가 가족 구성 질문으로 바뀐다 |
| MAIL-LOOP | 가족 수령 호출이 활성화되어 있으면 쿠폰 탄막 일부가 주소 호출음과 동기화된다 |

구현 메모:

```text
Vertical Slice에서는 패턴 자체를 복잡하게 늘리지 않는다.
대신 보스 진입 전 선택에 따라 문구/예고음/안전지대 이름만 바꿔도 충분하다.
```

### 4.2 깨끗한 거실

기존 보스 패턴:

```text
홈케어 청소기와 흡입 장판.
바닥 드롭과 흔적도 같이 흔들린다.
```

NPC 연결:

| 연결 | 반영 |
|---|---|
| VAC-0 | 청소 제외 목록이 없으면 이름 조각/메모/식탁 흔적이 청소 대상으로 보인다 |
| NAME-PAINTER | 문패 칠 흔적이 지워지면 빈칸이 더 깨끗한 입주 후보처럼 보인다 |
| ROOM-12 | 식탁 흔적이 지워지면 우회 배급 근거가 약해진다 |

구현 메모:

```text
VAC-0는 이 구간에서 직접 NPC로 크게 열지 않는다.
청소 흔적과 제외 목록 후보만 있으면 보스 패턴과 충분히 연결된다.
```

### 4.3 기념사진 시간

기존 보스 패턴:

```text
큰 플래시.
둔화 또는 포즈 고정.
빠진 가족은 나중에 합성된다는 공포.
```

NPC 연결:

| 연결 | 반영 |
|---|---|
| DINING-4 | 네 번째 자리가 비었는지, 보류되었는지가 사진 보정 압박을 바꾼다 |
| FRAME-LEFT | 보스 후 가족사진 프레임 잔향의 직접 원인이 된다 |
| NAME-PAINTER | 이름이 문패에 붙어 있으면 사진 캡션처럼 자동 결합될 수 있다 |
| MAIL-LOOP | 주소 로그가 사진 촬영 기록과 결합하면 수령인/가족 기록으로 오염된다 |

구현 메모:

```text
DINING-4는 여기서 ROOM-12 반복이 아니다.
ROOM-12는 배급/대표/침상 접근의 문제이고,
DINING-4는 비어 있는 가족 칸이 자동으로 채워지는 문제다.
```

### 4.4 추천 평면도

기존 보스 패턴:

```text
모델하우스 평면도 선이 바닥에 그어지고,
동선이 "가족 친화적"으로 수정된다.
```

NPC 연결:

| 연결 | 반영 |
|---|---|
| OPEN-HOST | 검증된 동선이면 일부 가짜 안전길을 피한다 |
| WELCOME-DESK | A-창구 직행을 택했다면 평면도가 상담 창구로 몰아간다 |
| SIGN-BACK | 직접 구현은 보류하지만, 가짜 귀환로 예고 표식으로 P1 씨앗을 남긴다 |

구현 메모:

```text
OPEN-HOST의 안내를 정답 길로 만들지 않는다.
검증된 길도 덜 나쁜 길일 뿐이며, 보스 후 후폭풍 대사가 남아야 한다.
```

## 5. NPC별 보스 연결

| NPC | 보스 전 입력 | 보스전 연결 | 보스 후 출력 |
|---|---|---|---|
| OPEN-HOST | `open_host_route_verified` 또는 무검증 안내 | 추천 평면도, 보스 진입 동선 | 안내 후폭풍, 플레이어블 전환 후보성은 보류 |
| ROOM-12 | `room12_bypass_ration`, `room12_left_in_place`, `room12_forced_removed` | 배급/가족대표 압박이 DINING-4와 결합 | 구출 완료가 아니라 정산/자리 후폭풍 |
| PORCH-LINE | `porch_line_flexible_ticket`, `porch_line_exact_queue`, `porch_line_dispersed` | 순번표가 상담 심사 자료로 붙음 | 보스 후 줄이 바로 사라지지 않음 |
| NAME-PAINTER | `name_painter_names_to_bokhee`, `name_painter_hold_marks` | 문패 이름이 사진/주소/상담표에 붙는 속도 조절 | FRAME-LEFT 이름 조각 보관 방식과 연결 |
| MAIL-LOOP | `mail_loop_call_hold`, `mail_loop_active`, `mail_loop_shutdown` | 가족 수령 호출, 주소/사진 결합 위험 | 세븐 분석, SIGN-BACK/KEY-RING P1 검증 씨앗 |
| DINING-4 | `dining4_fourth_seat_hold`, `dining4_forced_stand` | 빈 가족 칸, 기념사진 시간, 가족 완성 페이즈 | FRAME-LEFT 빈칸 안정도에 반영 |
| WELCOME-DESK | `welcome_desk_question_hold`, `welcome_desk_fast_track` | 입주 상담, 가족 구성 질문, A-창구 직행 | 상담표 증거 보관 또는 질문 재개 위험 |
| FRAME-LEFT | `frame_left_hint_seen` | 보스 전 직접 등장 금지, 사진 프레임 흔들림만 | 기억 추출/프로필 잔향, `frame_left_arin_fragment` |

Vertical Slice에서 직접 대면하지 않는 P1 NPC:

| NPC | 보스 연결 방식 |
|---|---|
| DRAIN-MARK | 보스 후 회수선이 불안정할 때 침묵권 루트 예고 |
| SIGN-BACK | 추천 평면도/가짜 귀환로 표지판으로 P1 씨앗 유지 |
| KEY-RING | 보스 후 문패/주소/사진 대조 필요성으로 관리실 루트 예고 |
| VAC-0 | 깨끗한 거실 패턴과 청소 제외 목록 흔적으로만 연결 |

## 6. DINING-4 고정

DINING-4는 보스 전 감정 압박의 핵심이다.

고정 기준:

```text
DINING-4는 ROOM-12 반복이 아니다.
ROOM-12는 식량태그/거주태그/가족대표 접근권의 문제다.
DINING-4는 빈 가족 칸이 생기면 캠페인이 새 얼굴을 끼워 넣는 문제다.
```

보스 직전 장면:

| 상태 | 장면 |
|---|---|
| `dining4_fourth_seat_hold` 있음 | 네 번째 자리는 비어 있지만 보류 표시가 붙어 있다 |
| `dining4_left_seated` | DINING-4가 계속 앉아 있어 빈칸은 막지만 스스로 역할에 더 묶인다 |
| `dining4_forced_stand` | 자리가 비고 보스가 새 가족 구성원을 추천하기 시작한다 |

보스 패턴 반영:

| 패턴 | 반영 |
|---|---|
| 기념사진 시간 | 네 번째 자리 상태에 따라 사진 보정 문구가 달라진다 |

## 7. WELCOME-DESK 연결

WELCOME-DESK 본격 직접 대면은 보류한다.

## 8. OPEN-HOST 안내 후폭풍

OPEN-HOST는 착한 길 안내자가 아니다.
그는 덜 나쁜 길을 알고 있지만, 그 길도 과거에는 사람을 집 안으로 넣던 안내였다.

보스 연결:

| 시점 | 연결 |
|---|---|
| 보스 전 | 안내 흔적/목소리로 신뢰를 보류한다 |
| 보스 직전 | 추천 동선과 실제 동선을 검증하게 한다 |
| 보스전 | 추천 평면도 패턴이 OPEN-HOST 안내의 위험한 얼굴로 보인다 |
| 보스 후 | "그때는 살리는 길이라고 믿었다"는 후폭풍을 남긴다 |

필수 플래그:

```text
open_host_trace_seen
open_host_voice_warned
open_host_route_verified
open_host_route_rejected
open_host_route_published
```

구현 금지:

```text
OPEN-HOST의 검증 동선을 보스전 쉬운 길 정답으로 만들지 않는다.
검증된 길은 피해를 줄일 뿐, R01 절차의 일부였다는 후폭풍이 남아야 한다.
```

## 10. FRAME-LEFT 기억 추출/잔향

| P1 | 본격 선택은 후속 문서에서 처리한다 |

기억 추출 선택과 연결:

| 보스 선택 | FRAME-LEFT 반응 |
|---|---|
| 결절 파괴 | 사진 보정은 약해지지만 일부 원본 로그가 손실될 수 있다 |
| 코어 해방 | 프레임 장치가 멈추지 않아 리콜/재동기화 위험이 남는다 |

보스 직후 최소 대사:

| 조건 | 대사 | 용도 |
