## 0. 한 줄 정의

막차가 끊긴 뒤에도 열리지 않는 보관함 앞에서, 수령되지 않은 물건이 사람을 대신 기다린다고 믿는 수취인 잔류자.

## 2. 지역 질문

```text
수령되지 않은 물건은 언제부터 사람을 대신 기다리는가?
보관 기간 연장은 보호인가, 회수 지연인가?
```

---

## 3. 핵심 오판

```text
보관 기간이 연장되면 수취인도 아직 안전하다.
```

이 오판은 RETURN-05의 반송/미수령 축과 연결되지만, 이루의 전투 기능을 대체하지 않는다.
이 NPC는 "배송 방향"이 아니라 "보관 만료와 수취인 부재"를 담당한다.

## 4. 상태값/플래그

| 키 | 용도 |
|---|---|
| `delivery_state` | 보관함이 수취/미수령/만료 사이에서 흔들림 |
| `passage_tag` | 막차 이후 역전 통행 제한 |
| `signal_tag` | 보관함 알림과 보급소 수신의 어긋남 |
| `trace_preservation` | 물건이 사람의 흔적을 대신 보존 |

| 플래그 | 의미 |
|---|---|
| `e02_locker_expiry_seen` | 보관함 만료 알림 최초 확인 |
| `e02_last_train_missed` | 막차 이후 회수실 진입 |
| `e02_locker_recipient_spoken` | 제한 대면 1회 완료 |
| `e02_locker_extension_refused` | 보관 기간 연장 거부 선택 |

---

## 5. 선택 후폭풍

| 선택 | 후폭풍 |
|---|---|
| 보관 기간 연장 | 보관함은 열리지 않고 수신태그 소모가 증가한다 |
| 보관 기간 거부 | 물건 일부는 회수되지만 수취인 호출 로그가 끊긴다 |
| 수취인 이름 보류 | 복희/RETURN-05 계열 반응이 열리지만 즉시 구원으로 처리하지 않는다 |

## 6. 플레이어블 충돌 방지

| 대상 | 금지 |
|---|---|
| RETURN-05 | 반송 수취인 전투, 송장 방향 조작, 미수령 보관 HP 0 대체 금지 |
| GATE-09 | 막차 환승 심사, 개찰 미통과, 플랫폼 잔류 기능 대체 금지 |
