# R01 NPC 선택 분기별 후폭풍 매트릭스 v0.1

## 문서 상태

```text
상태:
스토리팀/시스템팀 연결용 v0.1

용도:
R01 핵심 NPC 12명의 핵심 선택지, 상태값 변화, 다른 NPC 반응, 이름 해금/보류 조건, 후속 작업 연결을 한 표로 관리한다.

판정:
필요

주의:
이 문서는 정답 선택지를 만들기 위한 문서가 아니다.
R01 선택은 사람을 완전히 구하는 버튼이 아니라, 어떤 위험을 줄이고 어떤 위험을 남길지 정하는 방식이어야 한다.
```

## 0. 판정 요약

R01 NPC 선택 분기별 후폭풍 매트릭스는 지금 단계에서 작성해야 한다.

이유는 명확하다.

R01 NPC 12명의 드라마, 본명/호칭, 이름 해금 조건은 한 바퀴 정리되었다.
이제 각 NPC의 선택이 다음에 무엇을 흔드는지 정리해야 한다.

핵심 질문:

```text
유저가 한 사람을 덜 다치게 만들었을 때,
R01 전체에서는 무엇이 더 위험해지는가?
```

R01의 선택은 단순히 좋음/나쁨이 아니다.

```text
사람을 데려오면 빈칸이 생긴다.
이름을 남기면 캠페인도 읽는다.
주소를 보관하면 호출도 남는다.
길을 공개하면 구조선도 오지만 추적도 온다.
표지판을 고치면 사람은 살 수 있지만 캠페인은 다시 고친다.
```

따라서 R01의 핵심 분기는 모두 후폭풍을 가져야 한다.

## 1. 상태값 표기

이 문서는 상태값 변화를 다음처럼 표기한다.

```text
인간 안정도:
사람들이 서로를 밀지 않고, 이름/순서/역할을 견디는 정도.

캠페인 농도:
R01 광고/가족/입주 절차가 얼마나 강하게 작동하는지.

흔적 보존:
이름, 주소, 문패, 사진, 상담표, 열쇠, 표식이 사람의 흔적으로 남는 정도.

인양 안정도:
보급소 회수선, 비콘, 역송출, 긴급 인양이 얼마나 안정적으로 닿는지.

로봇 자율성:
VAC-0/PATCH 계열 장치가 기능 반복을 넘어 질문할 수 있는 정도.

수신 안정도:
MAIL-LOOP, FRAME-LEFT, SIGN-BACK, 세븐 분석에 필요한 신호/기록 안정도.

통행/거주 안정도:
문, 배수로, 스마트 도어, 식탁, 상담표가 사람을 덜 잘못된 공간으로 묶는 정도.
```

증감 표기:

```text
++:
강하게 상승.

+:
상승.

0:
큰 변화 없음.

-:
하락.

--:
강하게 하락.

보류:
확정은 늦추지만 후속 조건이 필요.
```

중요:

```text
캠페인 농도 상승은 항상 나쁜 것만은 아니다.
일부 단서는 캠페인이 강하게 작동할 때 더 잘 보인다.
하지만 농도가 높아지면 사람은 역할로 더 빨리 고정된다.
```

## 2. 전역 원칙

### 2.1 좋은 선택 하나 금지

R01 선택에는 완전한 정답이 없다.

권장 분기는 있어도, 비용이 없어서는 안 된다.

예:

```text
ROOM-12를 바로 데려오면:
그 사람은 자리에서 벗어나지만 다른 사람의 식량태그/침상 접근이 흔들린다.

NAME-PAINTER의 이름을 모두 지우면:
잘못된 귀속은 줄지만, 복희가 보관할 이름 조각도 줄어든다.

DRAIN-MARK의 표식을 공유하면:
인양 안정도는 오르지만 캠페인 추적 위험도 오른다.
```

### 2.2 보류는 행동이다

윤서다운 선택은 많은 경우 "확정"이 아니라 "보류"다.

```text
이름 확정 보류
주소 호출 보류
자리 재배정 보류
상담 질문 보류
열쇠 사용 보류
사진 빈칸 보류
```

보류는 아무것도 하지 않는 것이 아니다.
보류에는 시간, 태그, 다른 NPC 신뢰, 흔적 보존이 필요하다.

### 2.3 이름 해금은 보상만이 아니다

본명/이름 조각은 선택 결과의 일부다.

하지만 이름이 열리면 캠페인도 그 이름을 읽을 수 있다.

```text
이름 해금:
사람으로 부를 수 있는 조건.

이름 오염:
역할 호출/문패/상담표/사진 프레임에 다시 붙을 위험.
```

### 2.4 NPC 반응은 거점의 날씨다

선택 결과는 단순 퀘스트 완료가 아니라 다른 NPC 대사로 돌아와야 한다.

```text
복희:
이름 조각을 말려 보관할 수 있는가?

세븐:
기록을 분석하되 의도 판정에 실패하는가?

PATCH:
로봇 자율성 씨앗을 감지하는가?

미나:
정산표와 태그 배분에서 감정이 아니라 비용을 보는가?

윤서:
데려오고 싶지만 강제로 데려올 수 없는 이유를 다시 확인하는가?
```

## 4. NPC별 선택 후폭풍

## 4.1 OPEN-HOST / 서이겸

핵심 이벤트:

```text
OPEN-HOST가 알려주는 우회 동선이 실제로 덜 위험한 길인지 검증한다.
```

선택지:

| 선택 | 결과 | 상태값 변화 | 다른 NPC 반응 | 대가 |
|---|---|---|---|---|
| A. OPEN-HOST의 길을 그대로 따른다 | 빠르게 모델하우스 접근 가능 | 통행 안정도 + / 캠페인 농도 + | SIGN-BACK이 경계, WELCOME-DESK가 "질문도 길"이라고 반응 | 상담 동선으로 깊이 들어갈 위험 |
| B. OPEN-HOST의 길을 전부 거부한다 | 캠페인 동선 노출 감소 | 캠페인 농도 - / 인간 안정도 - | DRAIN-MARK 호감 일부 상승, PORCH-LINE 대기 증가 | 진행 지연, 일부 NPC 접근 지연 |
| C. 비콘/상담표/표지판으로 검증 후 제한 수용 | 덜 위험한 길만 사용 | 흔적 보존 + / 통행 안정도 + | SIGN-BACK 신뢰 상승, WELCOME-DESK 상담표 연결 | 시간이 걸리고 캠페인 농도 중간 이상에서 재오염 |
| D. OPEN-HOST의 동선을 모두에게 공개한다 | 단기 구조 가능성 상승 | 인간 안정도 + / 캠페인 농도 ++ | DRAIN-MARK 반발, SIGN-BACK 가짜 귀환로 경고 | 길이 알려져 캠페인도 따라옴 |

권장:

```text
C.
OPEN-HOST는 믿거나 거부하는 대상이 아니라, 검증해야 하는 사람이다.
```

이름 영향:

```text
C 이후 OPEN-HOST가 "서이겸" 이름 조각을 일부 인정할 수 있다.
단, 본명 완전 해금은 플레이어블 전환 검토 또는 후반 신뢰도 조건 이후.
```

후속 플래그:

```text
open_host_route_verified
open_host_route_rejected
open_host_route_published
open_host_name_fragment
```

## 4.2 ROOM-12 / 강연우

핵심 이벤트:

```text
ROOM-12가 자리를 비워도 다른 사람들의 식량태그/거주태그 접근이 무너지지 않게 만든다.
```

선택지:

| 선택 | 결과 | 상태값 변화 | 다른 NPC 반응 | 대가 |
|---|---|---|---|---|
| A. ROOM-12를 바로 데려온다 | ROOM-12는 자리에서 벗어남 | 인간 안정도 - / 식량 안정도 -- | PORCH-LINE 줄 흔들림, 미나 정산 불안 | 다른 사람의 배급/침상 접근 흔들림 |
| B. ROOM-12를 그대로 둔다 | 즉시 붕괴는 없음 | 식량 안정도 0 / 흔적 보존 - | 복희가 이름 보관 실패 가능 | ROOM-12가 역할로 더 굳음 |
| C. 우회 배급/침상 접근을 만든다 | 자리와 생존 접근권 분리 | 인간 안정도 + / 식량 안정도 + / 흔적 보존 + | PORCH-LINE 재방문 분기 개방, 복희 이름 보관 가능 | 캠페인이 우회 구조를 감지할 수 있음 |
| D. 가족대표 역할을 다른 사람에게 넘긴다 | 시스템상 빈칸은 막힘 | 캠페인 농도 ++ / 인간 안정도 - | DINING-4가 강하게 반응 | 다른 사람을 새 역할에 묶음 |

권장:

```text
C.
ROOM-12는 구출보다 우회 조건이 먼저다.
```

이름 영향:

```text
C 이후 복희가 "연우" 이름 조각을 보관할 수 있다.
강연우 완전 표기는 재방문 후에만 쓴다.
```

후속 플래그:

```text
room12_forced_removed
room12_left_in_place
room12_bypass_ration
room12_role_transferred
room12_name_fragment
```

## 4.3 PORCH-LINE / 백지호

핵심 이벤트:

```text
현관 앞 줄을 무너뜨리지 않으면서, 캠페인이 줄을 순번/상담 자료로 읽지 못하게 한다.
```

선택지:

| 선택 | 결과 | 상태값 변화 | 다른 NPC 반응 | 대가 |
|---|---|---|---|---|
| A. 줄을 해산한다 | 즉시 상담 진입은 줄어듦 | 캠페인 농도 - / 인간 안정도 -- | ROOM-12 배급 대기 흔들림, 미나 부담 증가 | 사람들이 서로를 밀고 순서 기억이 무너짐 |
| B. 줄을 정확히 유지한다 | 질서는 안정됨 | 인간 안정도 + / 캠페인 농도 + | WELCOME-DESK가 순번표 위험 경고 | 정확한 줄이 상담 순번으로 전환됨 |
| C. 유연한 순번표를 만든다 | 사람은 기억하되 절차는 흐림 | 인간 안정도 + / 흔적 보존 + | WELCOME-DESK 질문 보류와 연결 | 줄이 느려지고 불만이 생김 |
| D. PORCH-LINE을 보급소로 데려온다 | 현관 줄 관리자가 사라짐 | 인간 안정도 - / 흔적 보존 + | 윤서가 강제 구출 문제를 느낌 | R01 현관 앞이 바로 무너질 수 있음 |

권장:

```text
C.
줄을 없애는 것이 아니라, 사람이 서로를 밀지 않게 하되 캠페인이 정확히 읽지 못하게 해야 한다.
```

이름 영향:

```text
C 이후 재방문에서 "지호" 이름 조각 가능.
백지호 완전 해금은 인간 안정도 중간 이상 필요.
```

후속 플래그:

```text
porch_line_dispersed
porch_line_exact_queue
porch_line_flexible_ticket
porch_line_removed
porch_line_name_fragment
```

## 4.4 NAME-PAINTER / 임하온

핵심 이벤트:

```text
문패 이름을 지우거나 덧칠하는 대신, 이름 조각과 문패 표식을 분리한다.
```

선택지:

| 선택 | 결과 | 상태값 변화 | 다른 NPC 반응 | 대가 |
|---|---|---|---|---|
| A. 모든 덧칠 이름을 그대로 둔다 | 흔적은 많이 남음 | 흔적 보존 + / 캠페인 농도 + | MAIL-LOOP 주소와 결합 위험 | 틀린 이름이 사람을 잘못된 집에 묶음 |
| B. 모든 이름을 지운다 | 잘못된 귀속 감소 | 캠페인 농도 - / 흔적 보존 -- | 복희가 이름 조각을 잃음 | 빈 문패가 새 가족 슬롯으로 읽힘 |
| C. 이름 조각을 복희에게 맡긴다 | 이름과 문패 표면 분리 | 흔적 보존 ++ / 캠페인 농도 보류 | 복희 신뢰 상승, 윤서 보류 표시 연결 | 문패에는 빈칸 공포가 남음 |
| D. 문패에는 임시 표식만 남긴다 | 캠페인이 읽기 어려움 | 캠페인 농도 - / 흔적 보존 + | DRAIN-MARK 표식과 연결 | 유저가 표식을 잘못 읽을 수 있음 |

권장:

```text
C + D.
이름은 복희에게, 문패에는 캠페인이 덜 읽는 보류 표식을 남긴다.
```

```text
이름, 주소, 열쇠, 사진, 상담표가 서로 빠르게 결합한다.
본명 해금이 오히려 역할 호출로 오염될 수 있다.
```

### 5.3 흔적 보존

상승 요인:

```text
NAME-PAINTER 이름 조각 복희 전달
MAIL-LOOP 로그 분리 보관
VAC-0 청소 제외 목록
KEY-RING 확인 대기 열쇠판
FRAME-LEFT 이름 조각 보관
```

하락 요인:

```text
이름 전체 삭제
MAIL-LOOP 종료
VAC-0 효율 청소 방치
사진 프레임 파괴
순번표/상담표 완전 폐기
```

후속 효과:

```text
흔적 보존 높음:
복희 이름 보관함과 세븐 기록 분석이 동시에 강해진다.

흔적 보존 낮음:
NPC는 역할명으로 더 굳고, 본명 해금 조건이 늦어진다.
```

### 5.4 인양 안정도

```text
DRAIN-MARK 회수선 기준점
```

```text
DRAIN-MARK 표식 완전 은폐
가짜 귀환로 표지판 신뢰
수신태그 과활성 방치
배수로 깊은 침묵 주머니 진입
```

후속 효과:

```text
인양 안정도 낮음:
숨은 루트는 자유처럼 보이지만, 쓰러졌을 때 아무도 늦게 온다.

인양 안정도 높음:
보급소가 모든 길을 점유하지 않고도, 중간 기준점까지만 닿을 수 있다.
```

### 5.5 로봇 자율성


## 9. 최종 요약

R01 NPC 선택은 사람을 완전히 구하는 버튼이 아니다.

R01의 선택은 다음 중 하나를 고르는 일이다.

```text
무엇을 남길 것인가.
무엇을 보류할 것인가.
무엇을 당장 덜 위험하게 만들 것인가.
무엇이 나중에 더 위험해질 수 있는가.
```

권장 방향은 분명하다.

```text
확정 대신 보류.
파괴 대신 분리.
구출 대신 조건 만들기.
안내 대신 검증.
흔적 대신 역할로 굳지 않게 하기.
```

이 매트릭스의 다음 단계는 선택 분기를 시간순으로 배치하는 것이다.

가장 먼저 이어질 작업:

```text
R01 보스 전후 NPC 상태 변화 정리.
```
