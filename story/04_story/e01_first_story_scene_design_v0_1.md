| 항목 | 값 |
|---|---|

## 1. 최종 판정

| 판정 항목 | 결과 | 근거 |
|---|---|---|
| 최종 판정 | 1차 장면 설계 가능 | `story_campaign_scene_priority_selection_v0_1.md`의 `E01 + 전역 잔향 연결` 판정 기준 |
| 장면 범위 | E01 중심 | 윤서 단독 U5 시작과 E01 120 고정 구조를 해치지 않음 |
| 전역 연결 | 비대면 잔향/오브젝트/문구 후보 | 240개 JSON/핸드오프 항목을 직접 대면으로 전환하지 않음 |
| 본문 작성 | 금지 유지 | 실제 스토리 본문, 대사 본문, 새 캐릭터 프로필 작성 없음 |

판정문: 1차 장면은 설계 가능하다. 단, 이번 문서는 장면 목표와 기능을 정리하는 설계안이며 실제 본문이나 대사를 쓰지 않는다.

## 2. 시작 캠페인

| 항목 | 잠금값 | 설계 적용 |
|---|---|---|
| 시작 캠페인 | E01 | 첫 장면군은 E01 안에서 시작 |
| 시작 플레이어블 | 윤서 단독 U5 | 새 플레이어블 없이 단독 감각 유지 |
| E01 NPC | 120 고정 | 기존 범위만 참조, 추가 생산 없음 |
| 전역 NPC 슬롯 | 2,048 유지 | 슬롯 구조 변경 없음 |
| L5/L4 | 192개 직접 대면 후보로만 참조 | 새 승격 금지 |
| L3 이하 | 1,856개 은행형 표현 유지 | 전체 문구 작성 금지 |
| 실제 사용 가능 데이터 | 240개 JSON/핸드오프 항목 | 비대면 오브젝트/잔향/문구 후보로만 사용 |
| 장기 목표 | 10,240 | 현재 착수 금지 |

판정: 시작 캠페인은 E01이며, E02 이후는 연결 준비 또는 암시 범위를 넘지 않는다.

## 3. 장면 제작 방식

| 구분 | 이번 문서에서 작성 | 이번 문서에서 금지 |
|---|---|---|
| 장면 목표 | 각 장면이 확인해야 할 플레이 감각/월드 기능 | 실제 사건 서술 본문 |
| 장면 기능 | 시작, 상태 확인, 잔향 연결, 다음 캠페인 준비 등 구조 역할 | 대사 본문, 연출 세부 문장 |
| 등장 가능 요소 | 윤서, 기존 E01 120 구조, 기존 통과 NPC/오브젝트/잔향/문구, 240개 비대면 후보 | 새 플레이어블, 새 NPC 이름, 새 프로필 |
| 금지 요소 | 장면별 forbidden_overlap에 명시 | L5/L4 승격, 1,856개 전체 작성, 10,240 착수 |

장면 설계 원칙:

| 원칙 | 적용 |
|---|---|
| 첫 장면은 윤서 단독 감각을 해치지 않는다. | 시점은 윤서 또는 시스템/환경 관찰로 제한한다. |
| 전역 잔향은 작은 암시로만 쓴다. | 세계 전체 설명이나 결말 확정으로 쓰지 않는다. |
| NPC는 기존 범위를 넘지 않는다. | E01 120 고정과 L5/L4 192 상한을 유지한다. |
| 은행형 문구는 후보로만 쓴다. | 240개 JSON/핸드오프 항목을 본문 대사로 확장하지 않는다. |

## 5. 전역 잔향 연결 방식

| 연결 표면 | 허용 방식 | 금지 |
|---|---|---|
| 오브젝트 | 240개 JSON/핸드오프 항목을 사물 표식, 잔류 기록, 소형 문구 후보로 연결 | 새 NPC 대면 이벤트, 실제 레벨 배치 |
| 문구 | 조건 반응, 표지, 호출번호, 목록, 로그 후보로만 연결 | 대사 본문 작성, 1,856개 전체 문구 작성 |
| 방송 | 자동 안내나 반복 신호 가능성으로만 연결 | 실제 오디오 파일 생성, 성우 대면 NPC화 |
| 잔향 | 장소에 남은 비대면 흔적으로만 연결 | 세계 전체 설명, 결말 확정 |
| cross_region | `hold_for_context` 성격의 보류 표식으로만 연결 | 조기 진실 공개, 직접 대면화 |

연결 원칙:

| 원칙 | 적용 |
|---|---|
| 240개 항목은 직접 대면이 아니다. | 오브젝트/문구/방송/잔향 후보로만 사용한다. |
| 전역 잔향은 한 장면의 주제가 아니다. | E01 장면 주변부의 작은 이상 신호로만 둔다. |
| E02 이후는 준비만 한다. | E02 본문, 이름, 후보 상세 결재는 열지 않는다. |
| 구현 산출물은 만들지 않는다. | 실제 좌표, UI 파일, 오디오 파일, Godot 리소스, GDScript/로더 코드 없음. |
