## 1. 1차 스토리 제작 착수 가능 범위 요약

`npc_world_pre_story_structure_final_lock_v0_1.md`의 최종 판정 `1차 스토리 제작 착수 가능`을 기준으로 한다.

| 잠금 항목 | 유지값 | 이번 문서 적용 |
|---|---|---|
| 출시 초기 플레이어블 | 윤서 단독 U5 시작 | 첫 장면 기준으로만 사용 |
| 플레이어블 신규 제작 | 없음 | 새 후보/새 코드명/새 해금 순번 없음 |
| 미개방 캠페인 후보 | 상세 결재 금지 | 후보 상세화 없음 |
| E01 NPC | 120 고정 | 기존 구조만 참조, 추가 생산 없음 |
| 전역 NPC 슬롯 | 2,048 유지 | 슬롯 구조 변경 없음 |
| L5/L4 | 192개 직접 대면 후보로만 참조 | 승격/추가 이름 없음 |
| L3 이하 | 1,856개 은행형 표현 유지 | 전체 문구 작성 없음 |
| 현재 실제 데이터 | 240개 JSON/핸드오프 항목 | 후보 배경 반응으로만 사용 |
| 장기 풀스케일 | 10,240 장기 목표 | 현재 전체 작성 금지 |

판정: 스토리 제작은 열되, 본문 작성보다 먼저 E01 중심 장면 순서와 전역 잔향 연결 범위를 확정한다.

## 2. 첫 제작 캠페인 후보 선정

| 후보 판정 | 장점 | 위험 | 이번 판정 |
|---|---|---|---|
| E01 중심 시작 | 윤서 단독 U5와 E01 120 고정 구조를 가장 안전하게 사용 | 전역 240개 핸드오프 성과가 장면 우선순위에 늦게 붙을 수 있음 | 보조 판정 |
| E01 + 전역 잔향 연결 | E01을 첫 제작 축으로 유지하면서 240개 은행형 JSON/핸드오프 항목을 배경 반응으로 연결 가능 | cross_region 조기 진실 공개로 오독될 수 있음 | 최종 판정 |
| E01 이후 E02 연결 준비 | 다음 캠페인 연결 방향을 일찍 확보 | E02 개방이나 미개방 후보 상세 결재로 번질 위험 | 후순위 준비 |

최종 우선순위:

| 순위 | 캠페인/축 | 제작 의미 | 제한 |
|---:|---|---|---|
| 1 | E01 | 첫 제작 캠페인. 윤서 단독 U5 시작과 기존 E01 구조 확인 | E01 NPC 120 추가 생산 금지 |
| 2 | E01 내부 전역 잔향 연결 | 240개 은행형 항목 중 E01과 충돌하지 않는 오브젝트/잔향/문구 후보를 장면 주변부에 연결 | 실제 레벨/UI/오디오 구현 금지 |
| 3 | E02 연결 준비 | E01 이후 연결점만 설계 | E02 본문 개방, 새 이름, 상세 결재 금지 |
| 4 | cross_region 보류 축 | 장기 미스터리 방향성 메모만 유지 | `hold_for_context` 유지, 조기 진실 공개 금지 |
| 5 | E03~E12 | 현재 제작 우선순위 밖 | 본문 작성/플레이어블 후보 상세화 금지 |

판정: 첫 제작 캠페인은 E01이며, 제작 방식은 `E01 + 전역 잔향 연결`로 잠근다.

## 3. 첫 장면 후보 선정

이번 문서는 실제 본문을 쓰지 않는다. 아래 항목은 장면 슬롯 후보와 제작 순서 후보일 뿐이다.

| 우선순위 | 장면 후보 | 목적 | 사용할 수 있는 범위 | 금지 |
|---:|---|---|---|---|
| 1 | U5 시작 기준 장면 | 윤서 단독 시작 구조와 플레이어블 범위 확인 | 윤서 단독 U5, 기존 시작 조건 | 새 플레이어블, 새 해금 조건 |
| 2 | E01 현재 상태 확인 장면 | E01 120 고정 구조와 기존 NPC/오브젝트 맥락 확인 | 기존 통과 NPC/오브젝트/잔향/문구 | E01 추가 생산, 새 이름 |
| 3 | E01 은행형 반응 장면 | L3 이하 240개 중 E01 후보를 배경 반응으로 시험 배치할 장면 범위 선정 | 조건 반응, 오브젝트, 호출번호, 표지, 잔향 후보 | 실제 배치/좌표/파일 생성 |
| 4 | 전역 잔향 접속 장면 | E01 바깥 세계가 있음을 암시하되 캠페인을 열지 않음 | 240개 JSON/핸드오프 항목의 비대면 후보 | cross_region 진실 공개, E02 이후 이름 기본값 |
| 5 | E02 연결 준비 장면 | 다음 캠페인으로 넘어갈 수 있는 구조적 접점만 정리 | route_access, quarantine_delay, identity_mismatch 등 상태 축 | E02 본문 개방, 새 NPC 상세 결재 |

첫 장면 판정:

| 항목 | 판정 |
|---|---|
| 첫 장면 축 | U5 시작 기준 장면 |
| 첫 캠페인 축 | E01 |
| 첫 확장 연결 | 전역 잔향 연결 |
| E02 상태 | 연결 준비만 허용 |
| cross_region 상태 | 보류 유지 |

## 4. 사용할 수 있는 NPC/오브젝트/잔향 범위

| 범위 | 사용 가능 | 사용 방식 | 금지되는 확장 |
|---|---|---|---|
| 윤서 | 가능 | 단독 U5 시작 기준 | 새 플레이어블 추가 |
| E01 기존 NPC 120 구조 | 가능 | 기존 관계/기능축/상태 반응 참조 | 추가 생산, 새 이름 |
| L5/L4 192개 | 제한 가능 | 직접 대면 후보 최대치로만 참조 | 승격, 새 직접 대면 이벤트 |
| L3 이하 1,856개 | 구조 참조 가능 | 은행형 표현 총량 경계로만 참조 | 전체 문구 작성 |
| 240개 JSON/핸드오프 항목 | 가능 | E01 장면 주변의 오브젝트/잔향/목록/호출번호/표지 후보로 참조 | 실제 구현, 좌표/파일 확정 |
| cross_region 12개 핸드오프 항목 | 제한 가능 | 보류 표식, 장기 미스터리 방향성 메모 | 조기 진실 공개, 본문 확정 |
| E02 이후 은행형 항목 | 제한 가능 | E01 이후 연결 준비용 압력/상태 축 참조 | 캠페인 본문 개방, 한국어 이름 기본값 |
| 10,240 장기 목표 | 참조만 가능 | 장기 규모 감각 | 현재 전체 작성 착수 |

판정: 사용할 수 있는 것은 기존 구조와 240개 핸드오프 항목의 검토 범위다. 실제 본문, 새 NPC, 새 이름, 실제 구현 산출물은 만들지 않는다.

## 5. 아직 열면 안 되는 캠페인/후보

| 대상 | 현재 상태 | 열면 안 되는 이유 |
|---|---|---|
| E02 본문 제작 | 연결 준비만 허용 | 첫 제작 캠페인은 E01이며, E02 이후 이름/현실권 검토 원칙 유지 필요 |
| E03~E12 본문 제작 | 보류 | 지역별 직접 대면/이름/플레이어블 후보로 번질 위험 |
| cross_region 본문 제작 | 보류 | 장기 미스터리와 진실 공개 시점이 아직 잠금 |
| 미개방 캠페인 플레이어블 후보 | 보류 | 상세 결재 금지 |
| 새 L5/L4 후보 | 보류 | 192개 상한 유지 |
| L3 이하 1,856개 전체 작성 | 보류 | 현재 실제 데이터 범위는 240개 |
| 10,240 전체 작성 | 보류 | 장기 목표이며 현재 제작 대상 아님 |
| 레벨/UI/오디오 실제 구현 | 보류 | 핸드오프 QA 통과는 구현 착수 승인 아님 |

판정: 지금 열 수 있는 것은 E01 중심 장면 순서와 전역 잔향 연결 설계뿐이다.
