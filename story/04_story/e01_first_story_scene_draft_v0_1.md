## 3. 장면별 본문 초안

### scene_e01_001

| 항목 | 내용 |
|---|---|
| scene_id | `scene_e01_001` |
| scene_title | U5 시작 감각 확인 |
| scene_purpose | 윤서 단독 U5 시작 감각과 출발점의 불안을 확인한다. |
| allowed_scope | 윤서 단독 시점, 시작 지점 주변의 표지/잔향/상태 반응 후보 |
| mystery_hook | 귀환 표시와 실제 안전감의 불일치 |
| reversal_seed | 안전지대처럼 보이는 출발점이 불안의 기준점이 됨 |
| small_resolution_seed | 윤서가 단독으로 움직일 수 있는 최소 조건 확인 |
| guard_note | 새 플레이어블, 새 NPC 이름, 대사 본문 과잉 금지 |

draft_text:

윤서는 켜진 패널 앞에서 혼자 깨어난다. 패널에는 귀환 확인을 뜻하는 짧은 문구가 남아 있지만, 화면 가장자리의 숫자 하나가 빈칸으로 밀려 있다. 방 안은 조용하고, 너무 깨끗해서 방금 정리된 것처럼 보인다.

바닥의 안내선은 출구를 가리키지만 중간에서 한 번 끊긴다. 윤서가 발을 옮길 때마다 천장 안쪽에서 오래된 호출음 같은 잔향이 따라온다. 이곳은 시작점처럼 보이지만, 안전하다는 증거는 어디에도 없다.

### scene_e01_002

| 항목 | 내용 |
|---|---|
| scene_id | `scene_e01_002` |
| scene_title | E01 현재 상태 확인 |
| scene_purpose | E01의 일상 표식이 절차 흔적으로 변한 상태를 확인한다. |
| allowed_scope | E01 120 고정 구조 안의 기존 기능축, 기존 오브젝트/표지/호출번호/잔향 후보 |
| mystery_hook | E01의 일상 표식이 이미 절차 흔적으로 변함 |
| reversal_seed | 정상 안내가 오히려 위험 신호가 됨 |
| small_resolution_seed | 현재 지역의 우선 불안을 하나의 상태 축으로 정리 |
| guard_note | E01 추가 생산, 직접 대면 NPC 증가 금지 |

draft_text:

복도에는 생활 안내처럼 보이는 표지가 줄지어 붙어 있다. 가까이 보면 대부분은 환영 문구가 아니라 반납 순서, 대기 번호, 폐쇄된 방의 관리 절차를 짧게 반복하는 표식이다. 누군가를 부르는 이름은 없고, 번호와 상태만 남아 있다.

윤서는 열린 문보다 닫힌 문이 더 많은 구역을 지난다. 정상 운영을 알리는 작은 램프가 켜질 때마다 반대편 통로의 잠금 표시도 함께 깜박인다. 이곳은 망가진 것이 아니라, 망가지지 않은 방식으로 사람을 밀어내고 있다.

### scene_e01_003

| 항목 | 내용 |
|---|---|
| scene_id | `scene_e01_003` |
| scene_title | 은행형 반응 시험 연결 |
| scene_purpose | 240개 JSON/핸드오프 항목을 비대면 반응 표면으로만 시험 연결한다. |
| allowed_scope | E01 후보의 조건 반응, 오브젝트, 표지, 호출번호, 잔향 후보 |
| mystery_hook | 문구가 사람보다 먼저 상황을 설명함 |
| reversal_seed | 대사가 아니라 사물의 잔류 문구가 사실에 가까움 |
| small_resolution_seed | 사용할 수 있는 비대면 반응 표면을 확인 |
| guard_note | 1,856개 전체 문구 작성, 실제 UI 구현 금지 |

draft_text:

윤서가 손잡이가 없는 접수 패널 앞에 멈추자, 패널은 사람을 찾지 않고 상태만 갱신한다. 남은 문구는 짧다. 접수 보류, 재시도, 호출 대기. 어느 줄에도 이름은 없다. 대신 같은 형식의 빈칸이 몇 번이고 반복된다.

옆의 서류함에는 꺼내지 못한 얇은 종이가 걸려 있다. 종이 끝에는 반쯤 지워진 번호와 사용되지 않은 확인란이 남아 있다. 윤서는 그 문구들이 누군가의 설명보다 더 먼저 이곳의 규칙을 말하고 있다는 것을 알아차린다.

### scene_e01_004

| 항목 | 내용 |
|---|---|
| scene_id | `scene_e01_004` |
| scene_title | 전역 잔향 접속 확인 |
| scene_purpose | E01 바깥의 같은 패턴을 작은 암시로만 접속한다. |
| allowed_scope | 방송 잡음, 반복 안내, 삭제 로그, 번호 흔적 후보 |
| mystery_hook | E01 바깥의 같은 패턴이 아주 작게 겹침 |
| reversal_seed | 지역 문제처럼 보인 현상이 전역 절차의 일부일 수 있음 |
| small_resolution_seed | 세계 설명 없이 다음 의문만 남김 |
| guard_note | cross_region 진실 공개, 새 이름, 오디오 파일 생성 금지 |

draft_text:

벽 안쪽 스피커가 한 번 낮게 울린다. 방송은 또렷한 안내가 아니라 여러 지역의 말투가 겹친 잡음처럼 지나가고, 마지막에는 같은 번호 형식만 남는다. 윤서가 올려다보는 순간 스피커의 표시등은 꺼지고, 패널에는 이전 기록 삭제라는 흔적만 남는다.

그 소리는 E01의 고장처럼 들리지 않는다. 너무 멀리서 왔고, 너무 익숙한 절차를 따라 끊겼다. 윤서는 세계를 이해하지 못한 채로, 이 구역만의 문제가 아닐지도 모른다는 작은 의문을 들고 다음 문 앞에 선다.

### scene_e01_005

| 항목 | 내용 |
|---|---|
| scene_id | `scene_e01_005` |
| scene_title | E02 연결 준비 표식 |
| scene_purpose | E02를 열지 않고 통행/검역/신원 상태의 연결 가능성만 남긴다. |
| allowed_scope | route_access, quarantine_delay, identity_mismatch 계열 비대면 후보 |
| mystery_hook | 통행/검역/신원 상태가 E01 밖으로 이어질 가능성 |
| reversal_seed | 다음 지역의 문이 열리는 것이 아니라 닫힌 이유가 먼저 보임 |
| small_resolution_seed | E02를 열지 않고 연결 조건만 보존 |
| guard_note | E02 본문 제작, 미개방 후보 상세 결재 금지 |

draft_text:

통로 끝의 게이트에는 열림 표시 대신 확인 대기 표시가 떠 있다. 문은 바깥을 보여주지 않는다. 대신 검역 지연, 통행권 재확인, 신원 불일치 같은 짧은 항목이 순서 없이 켜졌다가 사라진다. 모두 E01 안에서 시작되었지만, 끝은 이곳에 없는 것처럼 보인다.

윤서는 문을 열 방법을 얻지 못한다. 다만 닫힌 이유가 하나가 아니라는 사실만 남는다. 다음 지역은 아직 장면 밖에 있고, 지금 확인할 수 있는 것은 그곳으로 이어지는 절차의 그림자뿐이다.

### scene_e01_006

| 항목 | 내용 |
|---|---|
| scene_id | `scene_e01_006` |
| scene_title | 본문 착수 전 게이트 확인 |
| scene_purpose | 장면마다 하나의 의문을 남기고 설명 과잉을 막는다. |
| allowed_scope | 기존 통과 구조 점검, 240개 후보의 보류/사용/제외 분류 |
| mystery_hook | 장면마다 하나의 의문만 남기는지 확인 |
| reversal_seed | 많이 설명할수록 미스터리가 약해지는 구조를 확인 |
| small_resolution_seed | 본문 착수 전 금지선 재확인 |
| guard_note | 실제 본문 장편화, 대사 과잉, 새 프로필 작성 금지 |

draft_text:

윤서는 지나온 표식들을 하나씩 떠올린다. 안전하지 않은 시작점, 정상 안내처럼 보인 위험 신호, 사람보다 먼저 말한 사물, 먼 곳에서 겹친 방송, 열리지 않는 문. 어느 것도 충분한 설명은 아니지만, 모두 같은 방향으로 작은 흠을 남긴다.

초안은 여기서 멈춘다. 더 많은 이름을 붙이지 않고, 더 많은 세계를 설명하지 않고, 아직 열리지 않은 문을 억지로 열지 않는다. 다음 단계로 넘길 것은 사건의 결론이 아니라, 장면마다 남긴 하나의 의문과 그것을 지키는 금지선이다.
