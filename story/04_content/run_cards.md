## 설계 원칙

런 카드는 플레이어가 가장 자주 읽는 세계관 문장이다. 카드 이름과 광고풍 설명은 밝고, 짧고, 살짝 웃겨야 한다. 실제 효과 문장은 오해 없이 명확해야 한다.

카드는 설정 설명서가 아니다. 한 장의 종이 광고, 쿠폰, 제품 라벨처럼 보여야 한다. 표면은 친절하지만 효과를 읽으면 이상하고 위험한 세계라는 것이 드러나야 한다.

## 데이터 필드

```text
RunCard
  id
  name
  flavor_text
  effect_text
  tags
  rarity
  unlock_condition
  stack_rule
  narrative_axis
```

## 문체 규칙

- `name`: 2~6어절. 광고 상품명처럼 기억돼야 한다.
- `flavor_text`: 1문장, 가능하면 25자 내외.
- `effect_text`: 숫자/효과 중심으로 명확히 쓴다.
- 설명이 슬퍼도 표면은 너무 울지 않는다.
- 실제 효과와 광고풍 설명은 반드시 분리한다.

좋은 예:

- 광고풍: "마음에 안 드셨나요? 돌려드립니다. 힘껏."
- 실제 효과: "피격 시 일정 확률로 가장 가까운 적에게 피해를 반사한다."

나쁜 예:

- "이 카드는 캠페인 사회에서 소비주의의 비극을 보여준다."
- "대충 강해진다."

## 중첩 규칙 기준

- 피해 증가 카드: 같은 수치가 선형 증가.
- 확률 카드: 초반은 빠르게, 후반은 완만하게 증가.
- 위험 보상 카드: 이득과 위험이 함께 증가.
- 1회성 방어 카드: 발동 횟수보다 회복량/보호량 위주로 증가.
- 침묵/방해 카드: 적 행동을 완전히 끊지 말고 빈도나 강도를 낮춘다.

## 카드 UI 문장 구조

카드에는 세 줄을 권장한다.

```text
카드 이름
광고풍 설명
실제 효과
```

예시:

```text
상쾌한 반품
마음에 안 드셨나요? 돌려드립니다. 힘껏.
피격 시 일정 확률로 가장 가까운 적에게 피해를 반사합니다.
```

## 금지 방향

- 광고풍 설명이 실제 효과를 가리게 만들지 않는다.
- 모든 카드가 같은 농담 톤이 되지 않게 한다.
- 너무 긴 카드 설명을 쓰지 않는다.
- 위험 보상 카드의 위험을 숨기지 않는다.
- 카드가 세계관과 무관한 일반 RPG 버프처럼 보이지 않게 한다.
- 캠페인 위험 카드를 무조건 나쁜 선택으로 만들지 않는다. 강한 유혹이어야 한다.
