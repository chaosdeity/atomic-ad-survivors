# Trace Items

상태: 0.001 기준 작성
원전: `story/00_source/story_0.001.md`
연결 문서: `story/02_hub/npcs.md`, `story/03_regions/r01_suburb.md`, `story/03_regions/r01_boss.md`

## 설계 원칙

흔적은 캠페인이 광고로 바꾸기 전의 이름, 사용감, 손때가 남은 물건이다. 단순 수집품이나 장비 재료가 아니라, 전투 효과와 거점/스토리 효과를 동시에 가진다.

좋은 흔적은 플레이어가 효과만 보고 기억하는 것이 아니라, "이걸 소모해도 되나?" 하고 잠깐 망설이게 만들어야 한다. 보관하면 세계와 엔딩 축에 남고, 소모하면 즉시 성장에 도움이 된다.

## 데이터 필드

```text
TraceItem
  id
  name
  short_description
  source_region
  source_type
  related_npcs
  combat_effect
  hub_story_effect
  preserve_effect
  consume_effect
  ending_axis
  notes
```

## 운용 규칙

- `보관`: 이름 보관함, NPC 대사, 공존/기억 축에 유리하다.
- `소모`: 시설 강화, 전투 보너스, 즉시 진행에 유리하다.
- `진영 사용`: 인간 안정도, 로봇 자율성, 캠페인 활용도 중 하나를 올린다.
- 흔적은 한 번 얻고 끝나는 도감이 아니라, 회수 후 어디에 둘지 선택하는 물건이다.

## 금지 방향

- 흔적을 단순 재료 아이콘으로만 만들지 않는다.
- 모든 흔적에 장문 사연을 붙이지 않는다. 한 줄 설명과 NPC 반응으로 충분히 전달한다.
- 전투 효과가 너무 좋아서 보관 선택이 바보처럼 느껴지게 만들지 않는다.
- 보관만 정답이 되게 만들지 않는다. 소모도 플레이어의 생존 선택이어야 한다.
- 캠페인 활용 흔적을 무조건 악으로 만들지 않는다. 강한 유혹이어야 한다.
