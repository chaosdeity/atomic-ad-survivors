## 0. 판정 요약

판정:

```text
조건부 통과
```

Nora Weiss / BROAD-10 / 노라, 공영 안내 수신자는 10번 슬롯에 둘 가치가 있다. 9번 GATE-09가 통행/환승 절차를 통해 첫 외부 광역권 감각을 연다면, 10번 BROAD-10은 공영 안내와 반복 수신이 사람을 안전한 수신자, 대피 문장 승인자, 반복 문구 대상으로 고정하는 절차를 플레이어블 관점으로 읽게 한다.

다만 바로 프로필 v1.0으로 가지 않는다. 아래 조건이 먼저 잠겨야 한다.

1. 방송인, 아나운서, 언론인, 해커, 소리 마법사, 버프/디버프 서포터로 흐르면 안 된다.
2. 백여울 / QUIET-07의 낮은 수신 리스크와 반대로, 노라는 너무 선명한 공식 수신에 붙잡히는 리스크로 고정해야 한다.
3. E07_C05 공영안내 수신권만 사용하고 E07 전체, E09_C05 케이블뉴스 수신권, E10_C05 알고리즘 추천권을 선점하지 않는다.
4. HP 0은 수신 공백이 아니라 청취 확인 완료, 대피 문장 자동 승인, 반복 송출 동기화 직전 긴급 인양으로 차별화해야 한다.
5. Nora Weiss 이름은 유지하되, 독일어 원문 표기와 유저 노출 표기는 프로필 v1.0 전에 검수한다.

## 1. 한 줄 정의


이 후보의 힘은 신호를 끊거나 방송을 장악하는 데 있지 않다. 노라는 너무 정확하고 선명한 안내가 "당신은 들었습니다", "당신은 확인했습니다", "당신은 안전 문장에 동의했습니다"로 닫히기 전의 짧은 틈을 읽는다.

따라서 전투의 중심 문장은 아래로 둔다.

```text
분명하게 들린다고 해서,
동의한 것은 아니다.
```

## 3. 핵심 오판

결재안:

```text
노라는 공식 안내가 선명할수록 더 안전하다고 믿는다.
```

이 믿음은 일부 맞다. 공영 안내는 사람을 죽이려는 신호가 아니다. 반복되는 안내, 재난 문구, 정정 방송, 다국어 안내는 혼란 속에서 누군가를 살리기 위해 만들어진 절차다.

하지만 이 믿음은 위험하다. E07_C05 공영안내 수신권은 노라를 해치려는 것이 아니라, 노라를 "안내를 들은 사람", "안전 문장을 승인한 사람", "대피 지침을 확인한 사람"으로 보존하려 한다. 그 순간 노라는 아직 판단 중인 사람이 아니라 이미 동의한 수신자로 고정된다.

더 깊은 오류:

- 정확한 문장은 안전한 선택을 대신해 준다고 믿는다.
- 안내가 반복되면 책임이 분명해진다고 믿는다.
- "청취 확인"은 들었다는 표시일 뿐, 동의가 아니라고 스스로를 설득한다.
- 하지만 확인표가 닫히면 캠페인은 노라를 안전 문장 수신자로 읽는다.

통과 조건:

```text
노라는 안내를 만드는 사람이 아니다.
안내가 사람을 이미 동의한 수신자로 확정하기 전,
플레이어가 직접 청취 확인과 대피 문장 승인을 늦춰 보게 하는 관점이어야 한다.
```

## 4. 플레이어블이어야 하는 이유

노라는 NPC나 안내 방송 기록으로도 성립한다. 공영 안내 라디오탑에서 반복 문구를 남기거나, 수신소의 기록으로 등장하는 방식은 충분히 가능하다.

하지만 플레이어블로 검토할 가치가 있는 이유는 노라의 핵심이 "정보 제공"이 아니라 "수신 절차의 판정 조작"이기 때문이다. 플레이어가 직접 조작할 때 아래 감각이 생긴다.

1. 어떤 안내를 들은 것으로 둘지, 재청취 대기로 넘길지 선택한다.
2. 수신태그를 써서 정정 방송 조각을 남길지, 통행태그를 써서 대피 문장 승인을 늦출지 판단한다.
3. 공식 안내를 너무 오래 늦추면 캠페인이 노라의 프로필을 더 선명한 안전 수신자로 고정한다.
4. 보급소 회수선이 노라를 찾으려면 완전한 침묵이 아니라 미승인 청취 로그가 남아야 한다.
5. "정보를 아는 캐릭터"가 아니라 "정확한 문장을 아직 동의하지 않은 상태로 붙잡는 캐릭터"가 된다.

NPC로 남을 때의 한계:

```text
NPC 노라는 반복 안내의 위험을 말할 수 있다.
플레이어블 노라는 반복 안내가 동의로 바뀌는 순간을 직접 늦추게 한다.
```

기존 후보와의 차이:

| 대상 | 차이 |
|---|---|
| 윤서 | 윤서는 현장의 마지막 좌표와 회수물을 붙잡는다. 노라는 공식 문장이 사람을 확정하기 전의 청취 상태를 붙잡는다 |
| PATCH | PATCH는 보급소 수리 로봇과 자기부품화의 축이다. 노라는 공영 안내를 듣는 인간 후보로, 수리/부품/충전을 다루지 않는다 |
| 백여울 | 여울은 낮은 수신 때문에 덜 들키지만 덜 구해진다. 노라는 너무 선명한 공식 수신 때문에 더 잘 들리고 더 쉽게 등록된다 |
| 리오 | 리오는 객실을 비었다고 초기화하지 않는 로봇이다. 노라는 사람을 안전 문장 수신자로 확인 완료하지 않는 인간 후보다 |
| 미카 | 미카는 통과/환승/잔류 판정을 늦춘다. 노라는 청취 확인/정정 방송/대피 문장 승인 판정을 늦춘다 |

따라서 현 판정은 조건부 통과다.

## 5. 전투 리듬

### 5.1 기본 리듬

기본 리듬:

```text
청취 확인 / 정정 방송 / 재청취 / 대피 문장 승인 판정을 밀고 당긴다.
```

노라는 적을 소리로 공격하거나 방송을 장악하는 캐릭터가 아니다. 노라의 조작감은 너무 선명한 공식 안내가 닫히기 전에 "아직 미확인", "재청취 필요", "정정 대기", "승인 보류" 상태를 남기는 데서 나온다.

전투 흐름 예시:

1. 반복 안내가 적 호출로 굳기 전 청취 확인을 미완료로 둔다.
2. 대피 문장이 안전 명령으로 닫히기 전 정정 방송 조각을 끼워 넣는다.
3. 반복 전광판의 한 줄을 빈칸으로 만들어 캠페인 호출을 한 박자 늦춘다.
4. 안내 수신 로그를 보급소 비콘으로 되접어 HP 0 때 잡힐 좌표를 남긴다.
5. 과사용 시 공영 안내가 더 선명해지고, 노라의 프로필이 안전 수신자로 다시 고정된다.

### 5.2 위험한 강점

| 강점 | 효과 | 위험 |
|---|---|---|
| 청취 확인 보류 | 공식 안내가 즉시 동의/등록으로 닫히는 것을 늦춘다 | 보류가 길어지면 반복 송출이 더 강해진다 |
| 정정 방송 조각 | 틀린 문장을 잠시 정정 대기로 바꾼다 | 정정이 누적되면 수신태그 부담이 커진다 |
| 재청취 전환 | 적 호출이나 캠페인 명령을 재확인 상태로 돌린다 | 재청취 상태가 길면 노라가 반복 문구 대상이 된다 |
| 대피 문장 승인 지연 | "안전한 경로"라는 명령을 아직 동의 전 상태로 둔다 | 통행태그와 수신태그를 동시에 압박한다 |
| 수신태그 역송출 | 긴급 인양 단서가 될 로그를 남긴다 | 여울처럼 낮은 수신이 아니라 과수신 부담이 된다 |

### 5.3 실패 시 후폭풍

노라의 실패는 단순 쿨다운 실패가 아니다.

```text
청취 확인표가 닫힌다.
대피 문장 승인표가 자동 서명된다.
반복 송출이 노라의 이름을 포함한다.
안전 문장이 노라의 프로필을 대신 설명한다.
보급소 비콘은 너무 선명한 공식 신호에 묻힌다.
```

실패 후폭풍:

- 수신태그 손실.
- 안내 수신 로그 오염.
- 정정 방송 조각 소모.
- 반복 송출 잔향 증가.
- 긴급 인양 때 보급소 회수선이 공식 안내 좌표와 충돌함.

### 5.4 플레이어 조작감

노라의 조작감은 정보 해킹이 아니다.

```text
정확한 안내를 무시하지 않고,
그 안내가 동의와 등록으로 닫히기 전에
짧은 보류 상태를 만드는 조작감.
```

추천 조작 감각:

- 반복 문구가 닫히기 전 짧은 타이밍 입력.
- 청취 확인, 재청취, 정정 대기 중 하나를 선택.
- 한 문장을 늦추면 다른 문장이 더 선명해지는 부담.
- 회피보다 "아직 확인하지 않았다" 상태 유지.
- 과사용 시 노라가 공식 안내의 대표 수신자로 고정되는 위험.

### 5.5 파티/동행 시 역할

노라는 파티 버퍼나 정보 서포터가 아니다.

파티 역할:

```text
팀 전체를 강화하는 것이 아니라,
특정 캠페인 절차가 "안전한 공식 문장"으로 닫히는 순간을 늦추고,
보급소가 잡을 수 있는 미승인 수신 로그를 남긴다.
```

다른 캐릭터와의 조합:

- 윤서가 회수해야 할 흔적을 "안내 완료"로 덮지 않도록 정정 대기를 남긴다.
- 여울의 낮은 수신과 반대로, 노라는 너무 선명한 신호를 덜 닫히게 만든다.
- 미카가 게이트와 시간표를 늦추듯, 노라는 안내 문장과 청취 확인을 늦춘다.
- 세븐이 로그를 분석하더라도, 노라는 분석가가 아니라 로그에 고정될 위험을 몸으로 받는 사람이다.

## 6. 태그 / 정산 / 자원

| 항목 | 결재안 |
|---|---|
| 관련 태그 | 수신태그, 통행태그, 보급태그 |
| 주 정산 자원 | 안내 수신 로그, 정정 방송 조각, 청취 확인표 |
| 보조 정산 자원 | 대피 문장 승인표, 반복 송출 잔향, 라디오 잡음 좌표 |
| 상태값 반응 | 캠페인 농도, 수신 안정도, 인양 안정도, 보급소 노출도, 인간 안정도 |
| 과사용 위험 | 공식 안내가 너무 선명해져 노라의 프로필을 안전한 수신자로 다시 고정한다 |

정산 루트:

```text
주 루트:
수신태그
안내 수신 로그
정정 방송 조각
청취 확인표

보조 루트:
통행태그
보급태그
대피 문장 승인표
반복 송출 잔향
라디오 잡음 좌표
```

정산 감각:

```text
좋은 결과는 안내를 끊어 버리는 것이 아니라,
안내가 동의와 등록으로 닫히기 전에
사람이 아직 판단 중이라는 로그를 남기는 것이다.
```

상태값 반응:

| 상태값 | 반응 |
|---|---|
| 캠페인 농도 높음 | 공영 안내가 더 정확해져 보류 판정이 어려워진다 |
| 수신 안정도 높음 | 안내는 잘 들리지만, 노라가 안전 수신자로 고정될 위험도 커진다 |
| 수신 안정도 낮음 | 여울과 달리 이득이 아니라 정정 방송 조각이 불안정해지는 부담으로 처리한다 |
| 인양 안정도 낮음 | 미승인 청취 로그가 없으면 보급소가 노라를 공식 안내 좌표와 구분하기 어렵다 |
| 보급소 노출도 높음 | 보급소 비콘이 공영 안내 반복 송출에 섞여 노출될 수 있다 |
| 인간 안정도 낮음 | "안내를 들었는데 왜 따르지 않느냐"는 압박이 커진다 |

금지:

```text
수신태그를 방송 공격력으로 만들지 않는다.
정정 방송 조각을 해킹 키로 만들지 않는다.
청취 확인표를 파티 버프 승인서로 만들지 않는다.
대피 문장 승인표를 무료 탈출권으로 만들지 않는다.
반복 송출 잔향을 광역 음파 공격으로 만들지 않는다.
```

## 7. HP 0 / 긴급 인양 차별화

노라의 HP 0은 사망/부활이 아니다.

```text
청취 확인 완료 / 반복 송출 동기화
```

| 5 | 긴급 인양 | 보급소가 미승인 청취 로그와 정정 방송 조각의 불일치를 잡아 되당긴다 |

| 캐릭터 | HP 0 감각 | 노라와의 차이 |
|---|---|---|
| 윤서 | 마지막 현장 좌표와 회수선 | 노라는 현장 좌표가 아니라 미승인 청취 로그와 정정 방송 조각이 좌표가 된다 |

금지:

```text
HP 0을 방송 사고, 감전, 기절, 단순 음파 과부하로 처리하지 않는다.
여울처럼 수신 공백/저소음 비콘 지연으로 쓰지 않는다.
보급소 회수선이 즉시 안정적으로 잡는 연출로 만들지 않는다.
노라가 직접 방송을 끊어 부활하는 방식으로 만들지 않는다.
```

## 13. 성장 방향

초기:

```text
공식 안내가 선명할수록 더 안전하다고 믿는다.
```

중반:

```text
정확한 문장이 사람의 판단을 대신하기 시작하면,
그 문장은 도움과 명령 사이를 오간다는 것을 배운다.
```

후반:

```text
공식 안내를 부수지 않고도,
아직 동의하지 않은 사람의 상태를 남길 수 있게 된다.
```

최종 변화 문장:

```text
정확한 문장도 사람을 대신 살아주지는 못한다.
```

성장 방향의 핵심은 방송 장악이 아니다. 노라는 더 큰 목소리를 갖는 캐릭터가 아니라, 더 정확한 문장 앞에서도 사람의 판단을 보류할 수 있는 캐릭터가 되어야 한다.
