## 0. 판정 요약

판정:

```text
조건부 통과
```

Maya Chen / MODEL-15 / 마야, 추천 모델 이탈자는 15번 산출 슬롯에 둘 가치가 있다. 단, 이 후보는 `리뷰 거부자`, `방송인`, `해커`, `인플루언서`, `추천 알고리즘을 부수는 천재`가 아니다.

핵심은 E10_C05 알고리즘 추천권이 사람을 `추천 대상`, `시청자`, `개인화 광고 프로필`, `모델 학습 샘플`로 등록하려는 순간을 플레이어블 관점으로 늦추는 데 있다.

통과 이유:

1. 9~13번 이후 첫 E10 북미 태평양권 후보로, 외부 광역권 분산을 회복한다.
2. BROAD-10의 공식 수신, CLAUSE-11의 약관 동의, LEDGER-12의 장부 정산과 다른 `모델 학습/광고 타깃 확정` 절차를 연다.
3. HP 0을 전투 사망이 아니라 `모델 학습 완료`, `추천 프로필 고정`, `광고 타깃 확정`으로 차별화할 수 있다.
4. 기존 한국어권 임시명 박다미를 폐기하고 다국적/북미 태평양권 후보로 전환한다.

조건부 사유:

1. 리뷰어/인플루언서/해커 클리셰로 새기 쉽다.
2. E10_C05가 열리면 빅테크/데이터센터/알고리즘 세계가 너무 빨리 커질 수 있다.
3. BROAD-10과 수신태그, CLAUSE-11과 약관/동의, OBS-20과 관제/관측 축이 겹칠 수 있다.
4. 전투가 `추천 안 받기`라는 수동적 개그로 얕아질 위험이 있다.

핵심 질문:

```text
사람이 추천되지 않으면 자유로운가,
아니면 학습되지 않은 데이터로 다시 붙잡히는가?
```

## 3. 캠페인이 등록하려는 역할

E10_C05 알고리즘 추천권이 마야를 등록하려는 역할:

```text
추천 대상
시청자
개인화 광고 프로필
모델 학습 샘플
반응률 측정 대상
관심사 클러스터
광고 타깃 후보
피드 잔류자
```

마야가 비트는 절차:

```text
추천됨 -> 클릭함 -> 시청함 -> 반응함 -> 학습됨 -> 광고 타깃으로 고정됨
```

이 절차는 사람을 해치려는 악의가 아니라, 사람을 더 정확히 도와주고 붙잡기 위한 시스템으로 출발한다. 문제는 "정확함"이 사람을 설명하는 문장으로 닫히는 순간이다.

## 4. 핵심 오판

초안:

```text
추천을 거부하면 모델에서 빠져나갈 수 있다고 믿음.
```

결재안:

```text
아무것도 누르지 않으면 아직 학습되지 않았다고 믿는다.
```

이 믿음은 일부 맞다. 반응하지 않으면 모델은 확정값을 얻지 못한다. 하지만 캠페인은 무반응도 반응으로 읽는다. 머무른 시간, 스크롤 멈춤, 화면을 닫는 순간, 추천을 숨기는 행동까지 모두 `다음 추천`의 재료가 된다.

더 깊은 오류:

- 침묵하면 데이터가 남지 않는다고 믿는다.
- 추천을 숨기면 추천 모델에서 제외된다고 믿는다.
- 틀린 추천이 나오면 아직 자신을 모른다고 믿는다.
- 하지만 틀린 추천까지 모델은 "교정 데이터"로 삼는다.

통과 조건:

```text
마야는 추천 알고리즘을 해킹하거나 파괴하지 않는다.
마야는 자신이 학습 완료 샘플로 닫히는 순간을 늦추고,
보급소가 잡을 수 있는 미학습 로그를 남기는 관점이어야 한다.
```

## 6. 플레이어블이어야 하는 이유

마야는 NPC나 로그 기록으로도 작동할 수 있다. 추천 피드 잔향, 데이터센터 화면, 광고 서버의 깨진 프로필로만 남겨도 캠페인 설명은 가능하다.

하지만 플레이어블로 검토할 가치가 있는 이유는 마야의 핵심이 정보 제공이 아니라 `반응 판정 조작`이기 때문이다.

플레이어블일 때 생기는 조작감:

1. 어떤 추천을 숨길지, 어떤 추천을 미학습 상태로 남길지 선택한다.
2. 추천을 늦추면 적 호출도 늦지만, 미학습 로그가 쌓여 다음 웨이브가 이상하게 튄다.
3. 광고 타깃 확정을 막으려면 완전 침묵이 아니라 보급소가 읽을 수 있는 `불완전 반응`을 남겨야 한다.
4. 유저가 "클릭하지 않기"가 아니라 "아직 모델이 나를 닫지 못하게 만들기"를 조작하게 된다.

NPC로 남을 때의 한계:

```text
NPC 마야는 추천 피드가 위험하다고 말할 수 있다.
플레이어블 마야는 추천 모델이 사람을 닫는 순간을 직접 늦추게 한다.
```

## 7. 전투 리듬

기본 리듬:

```text
추천 / 노출 / 반응 / 미학습 / 타깃 확정 판정을 밀고 당긴다.
```

마야는 해커가 아니다. 적을 데이터로 삭제하거나 서버를 폭파하지 않는다. 마야의 리듬은 모델이 원하는 확정 반응을 주지 않되, 보급소가 인양할 만큼의 불완전 로그를 일부러 남기는 것이다.

전투 동사:

```text
숨긴다
미룬다
흐린다
반응을 끊는다
미학습으로 남긴다
타깃 확정을 늦춘다
추천 피드를 굶긴다
오답 추천을 교정하지 않는다
```

금지 동사:

```text
해킹한다
폭로한다
방송한다
리뷰한다
팔로워를 모은다
서버를 장악한다
광고를 조종한다
추천으로 적을 소환한다
```

위험한 강점:

| 강점 | 효과 | 위험 |
|---|---|---|
| 추천 숨김 | 특정 캠페인 호출이나 추적을 한 박자 늦춘다 | 숨김 행동 자체가 반응 데이터로 읽힌다 |
| 미학습 로그 | 보급소가 잡을 인양 좌표를 남긴다 | 로그가 누적되면 모델이 더 강하게 추론한다 |
| 오답 추천 유지 | 적의 다음 경로를 틀린 피드로 잠시 돌린다 | 너무 오래 두면 오답도 프로필 일부가 된다 |
| 타깃 확정 지연 | 광고 타깃/관심사 고정을 늦춘다 | 지연이 길면 캠페인 농도가 상승한다 |
| 반응률 흐림 | 웨이브 호출의 우선순위를 흔든다 | 흐림이 과하면 보급소도 마야를 놓친다 |

## 9. HP 0 / 긴급 인양 차별화

마야의 HP 0은 사망/부활이 아니다.

핵심 상태명:

```text
모델 학습 완료 / 광고 타깃 확정
```

HP 0 흐름:

| 단계 | 상태 | 연출 |
|---|---|---|
| 1 | 추천 프로필 고정 | 추천 프로필 조각이 하나의 완성된 문장으로 붙는다 |
| 2 | 반응률 확정 | 무반응과 숨김 행동까지 반응률로 계산된다 |
| 3 | 광고 타깃 확정 | 마야의 이름 대신 관심사 클러스터가 표시된다 |
| 4 | 모델 학습 완료 | "학습 완료 샘플" 표식이 닫힌다 |
| 5 | 긴급 인양 | 보급소가 미학습 로그와 광고 타깃 보류표의 불일치를 잡아 되당긴다 |

긴급 인양 문구 후보:

```text
[긴급 인양]
MODEL-15 인양 완료.
미학습 로그 회수.
추천 프로필 조각 일부 손실.
```

차별점:

| 캐릭터 | HP 0 감각 | MODEL-15와의 차이 |
|---|---|---|
| BROAD-10 | 청취 확인 완료, 반복 송출 동기화 | 마야는 공식 안내가 아니라 개인화 추천 모델에 학습된다 |
| CLAUSE-11 | 자동결제 승인, 약관 동의 고정 | 마야는 약관이 아니라 반응률과 광고 타깃으로 고정된다 |
| LEDGER-12 | 미정산 장부 확정 | 마야는 장부가 아니라 추천 프로필 조각에 묶인다 |
| OBS-20 | 얼굴인증/세대 인증 실패 | 마야는 관제 영상이 아니라 추천 피드와 반응 로그로 읽힌다 |

금지:

```text
HP 0을 사이버 공격 실패, 계정 정지, SNS 논란, 방송 사고로 처리하지 않는다.
추천을 끄면 즉시 안전해지는 구조로 만들지 않는다.
보급소가 데이터 삭제를 해 주는 안전한 관리자처럼 보이게 하지 않는다.
```

## 10. 해금 조건

```text

```

해금 단계:
