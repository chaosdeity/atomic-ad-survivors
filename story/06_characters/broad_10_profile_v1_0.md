## 0. 캐릭터 핵심

노라는 방송인이 아니다.
아나운서, 라디오 DJ, 언론인 영웅, 방송국 내부자, 정보 해커, 소리 마법사, 파티 서포터도 아니다.

노라는 E07_C05 공영안내 수신권에서 공식 안내가 사람을 "안전한 수신자", "청취 확인 완료자", "대피 문장 승인자"로 확정하기 전의 판정을 늦추는 플레이어블 후보다.

핵심 질문:

```text
공식 안내가 계속 반복되면,
그것은 도움인가 명령인가?
```

핵심 문장:

```text
분명하게 들린다고 해서,
동의한 것은 아니다.
```

최종 변화 문장:

```text
정확한 문장도 사람을 대신 살아주지는 못한다.
```

잘못 믿는 것:

```text
노라는 공식 안내가 선명할수록 더 안전하다고 믿는다.
```

이 믿음은 일부 맞다. 공영 안내, 재난 문구, 정정 방송, 다국어 안내는 사람을 살리기 위해 만들어진 절차다. 그러나 캠페인은 노라를 죽이려는 것이 아니라, 노라를 이미 들은 사람, 이미 확인한 사람, 이미 동의한 사람으로 보존하려 한다.

플레이어블로서 노라가 다루는 질문:

```text
공식 문장이 사람을 안전하다고 확정하기 전,
아직 판단 중인 상태를 남길 수 있는가?
```

## 4. 플레이어블 기능

노라의 기능은 방송, 해킹, 광역 버프가 아니다.
노라는 공식 안내가 사람을 등록하려는 절차를 잠시 미완료, 재청취, 정정 대기, 승인 보류 상태로 남긴다.

| 기능 | 의미 | 실패 위험 |
|---|---|---|
| 청취 확인 보류 | "들었다"가 "동의했다"로 닫히기 전 청취 확인표를 미완료로 둔다 | 보류가 길어지면 반복 송출이 더 선명해진다 |
| 정정 방송 조각 | 틀린 공식 문장을 잠시 정정 대기로 바꾸고 인양 단서를 남긴다 | 정정이 누적되면 수신태그 부담이 커진다 |
| 재청취 전환 | 적 호출이나 안전 명령을 즉시 확정하지 않고 재청취 상태로 넘긴다 | 재청취가 길면 노라가 반복 문구 대상이 된다 |
| 대피 문장 승인 지연 | 안전 경로라는 이름의 명령을 아직 동의 전 상태로 둔다 | 통행태그와 수신태그를 동시에 압박한다 |
| 수신태그 역송출 | 긴급 인양 때 잡힐 미승인 수신 로그를 보급소 회수선 쪽으로 접는다 | 공식 안내 좌표가 너무 선명하면 보급소 비콘이 섞인다 |

플레이어블이어야 하는 이유:

```text
NPC 노라는 반복 안내의 위험을 말할 수 있다.
플레이어블 노라는 반복 안내가 동의로 바뀌는 순간을 직접 늦추게 한다.
```

기존 후보와의 분리:

- 윤서는 현장에 남은 좌표와 회수물을 붙잡는다. 노라는 공식 문장이 사람을 확인 완료자로 고정하기 전의 청취 상태를 붙잡는다.
- PATCH는 자기 몸을 부품처럼 써 온 보급소 수리 로봇이다. 노라는 공영 안내를 듣는 인간 후보로, 수리/부품/충전을 다루지 않는다.
- 여울은 낮은 수신 때문에 덜 들키지만 덜 구해진다. 노라는 너무 선명한 공식 수신 때문에 더 잘 들리고 더 쉽게 등록된다.
- 리오는 객실을 비었다고 초기화하지 않는 로봇이다. 노라는 사람을 안전 문장 수신자로 확인 완료하지 않는 인간 후보다.
- 미카는 통과/환승/잔류 판정을 늦춘다. 노라는 청취 확인/정정 방송/대피 문장 승인 판정을 늦춘다.

## 5. 전투 리듬

기본 리듬:

```text
청취 확인 / 정정 방송 / 재청취 / 대피 문장 승인 판정 지연
```

고정 동사:

```text
되묻는다
보류한다
정정한다
재청취로 넘긴다
승인을 늦춘다
청취 확인을 미완료로 둔다
반복 문구의 빈칸을 만든다
안전 문장을 반송한다
미승인 청취 로그를 남긴다
수신태그 역송출 비콘을 접는다
```

금지 동사:

```text
방송한다
해킹한다
송출을 장악한다
여론을 조작한다
진실을 폭로한다
소리로 폭발시킨다
팀 전체를 강화한다
적을 세뇌한다
공식 문구를 제작한다
채널을 점령한다
```

전투 흐름 예시:

1. 반복 안내가 적 호출로 굳기 전 청취 확인을 미완료로 둔다.
2. 대피 문장이 안전 명령으로 닫히기 전 정정 방송 조각을 끼워 넣는다.
3. 반복 전광판의 한 줄을 빈칸으로 만들어 캠페인 호출을 한 박자 늦춘다.
4. 안내 수신 로그를 보급소 비콘으로 되접어 HP 0 때 잡힐 좌표를 남긴다.
5. 과사용 시 공영 안내가 더 선명해지고, 노라의 프로필이 안전 수신자로 다시 고정된다.

플레이어 조작감:

```text
정확한 안내를 무시하지 않고,
그 안내가 동의와 등록으로 닫히기 전에
짧은 보류 상태를 만드는 조작감.
```

파티/동행 시 역할:

```text
팀 전체를 강화하지 않는다.
특정 캠페인 절차가 "안전한 공식 문장"으로 닫히는 순간을 늦추고,
보급소가 잡을 수 있는 미승인 수신 로그를 남긴다.
```

실패 시 후폭풍:

```text
청취 확인표가 닫힌다.
대피 문장 승인표가 자동 서명된다.
반복 송출이 노라의 이름을 포함한다.
안전 문장이 노라의 프로필을 대신 설명한다.
보급소 비콘은 너무 선명한 공식 신호에 묻힌다.
```

## 6. 태그 / 정산 / 자원

관련 태그:

```text
주 태그:
수신태그

보조 태그:
통행태그

제한 사용:
보급태그는 대피 문장 승인표와 보급소 노출도 반응에만 제한적으로 사용한다.
```

정산 루트:

| 정산 루트 | 고정 의미 | 사용 방식 | 위험 |
|---|---|---|---|
| 수신태그 | 방송/안내/기록 접근권 | 청취 확인, 재청취, 정정 대기를 처리하는 주 태그 | 방송 공격력이나 정보 마나가 되면 실패 |
| 통행태그 | 대피 문장과 이동 안내의 보조 접근권 | 대피 문장 승인 지연과 안내 경로 보류에 제한적으로 사용 | 무료 탈출권이나 빠른 이동권이 되면 실패 |
| 안내 수신 로그 | BROAD-10의 핵심 기록 | 어떤 안내를 들었지만 아직 승인하지 않았는지 남긴다 | 단순 퀘스트 로그가 되면 실패 |
| 정정 방송 조각 | 공식 문장의 불일치 | 틀린 문장을 정정 대기로 돌리고 인양 단서가 된다 | 해킹 키가 되면 실패 |
| 청취 확인표 | "들었다"가 "동의했다"로 닫히는 표 | 청취 확인 완료를 미루거나 미완료로 남긴다 | 파티 버프 승인서가 되면 실패 |
| 대피 문장 승인표 | 안전 문장 자동 승인 위험 | 승인하지 않은 문장을 보류하고 HP 0 위험을 만든다 | 무료 대피권이 되면 실패 |
| 반복 송출 잔향 | 공식 안내 과잉의 후유증 | 과사용과 HP 0 손실에 남는 흔적 | 광역 음파 공격 자원이 되면 실패 |
| 미승인 청취 로그 | 아직 판단 중인 사람의 단서 | 보급소가 공식 안내 좌표와 노라를 분리해 잡는 인양 단서 | 범용 분석 포인트가 되면 실패 |

상태값 반응:

| 상태값 | 반응 |
|---|---|
| 캠페인 농도 높음 | 공영 안내가 더 정확해져 보류 판정이 어려워진다 |
| 수신 안정도 높음 | 안내는 잘 들리지만, 노라가 안전 수신자로 고정될 위험도 커진다 |
| 수신 안정도 낮음 | 여울과 달리 이득이 아니라 정정 방송 조각이 불안정해지는 부담으로 처리한다 |
| 인양 안정도 낮음 | 미승인 청취 로그가 없으면 보급소가 노라를 공식 안내 좌표와 구분하기 어렵다 |
| 보급소 노출도 높음 | 보급소 비콘이 공영 안내 반복 송출에 섞여 노출될 수 있다 |
| 인간 안정도 낮음 | "안내를 들었는데 왜 따르지 않느냐"는 압박이 커진다 |

## 7. HP 0 / 긴급 인양 차별화

## 13. 성장 방향

초기:

```text
공식 안내가 선명할수록 더 안전하다고 믿는다.
```

노라는 들리지 않는 혼란보다 잘 들리는 공식 안내를 더 믿는다. 누군가가 정확한 문장을 반복해 준다면, 적어도 어디로 가야 하는지는 알 수 있다고 생각한다.

중반:

```text
정확한 문장이 사람의 판단을 대신하기 시작하면,
그 문장은 도움과 명령 사이를 오간다는 것을 배운다.
```

노라는 "들은 사람"과 "동의한 사람"이 다르다는 사실을 몸으로 배운다. 정정 방송을 남기는 일은 방송을 장악하는 일이 아니라, 아직 판단 중인 사람의 상태를 잃지 않는 일이다.

후반:

```text
공식 안내를 부수지 않고도,
아직 동의하지 않은 사람의 상태를 남길 수 있게 된다.
```

노라는 더 큰 목소리를 얻는 캐릭터가 아니다. 더 정확한 문장 앞에서도 사람의 판단을 보류할 수 있는 캐릭터가 되어야 한다.

최종 변화 문장:

```text
정확한 문장도 사람을 대신 살아주지는 못한다.
```
