## 4. 플레이어블 핵심

리오의 핵심은 청소가 아니다.
리오의 핵심은 공실 처리 거부다.

```text
방이 비었다는 판정을 늦춘다.
체크아웃 처리와 객실 초기화를 거부한다.
사라질 체류 흔적을 잠시 보존한다.
정품 복구가 가까워져도 청소 완료 상태로 돌아가지 않으려 버틴다.
```

잘못 믿었던 것:

```text
체크아웃 처리된 방은 정말 비어 있다고 믿었다.
정리 명령을 정확히 수행하는 것이 다음 사람을 위한 일이라고 믿었다.
정품 상태는 더 안전하고 편한 상태라고 배웠다.
```

현재 리오가 배워야 하는 것:

```text
작동 가능과 괜찮음은 다르다.
정리하는 것과 지우는 것은 다르다.
공실 판정과 실제 부재는 다르다.
```

플레이어블 판정:

```text
통과.
8번 플레이어블 후보 프로필 v1.0으로 고정한다.
단, 직접 해금은 중반 체류·숙박 캠페인권까지 보류한다.
```

## 5. 전투/탐색 리듬

리오는 단순 청소 로봇, 펫, 수리 로봇이 아니다.

전투/탐색 리듬은 아래 네 가지로 고정한다.

| 리듬 | 기능 감각 | 위험 |
|---|---|---|
| 방 상태 판정 | 투숙중/공실/청소대기/점검중 같은 상태를 전투 판정으로 바꾼다 | 상태를 너무 많이 바꾸면 정품 복구 호출이 빨라진다 |
| 흔적 보존 | 사라질 객실 로그, 카드 흔적, 체류 기록을 잠시 보존한다 | 보존 슬롯 과부하가 리오의 오류로 누적된다 |
| 체크아웃 지연 | 체크아웃 처리와 공실 전환을 늦춰 탈출/회수 시간을 만든다 | 지연 시간이 길면 캠페인이 리콜을 시도한다 |
| 객실 초기화 저항 | 초기화 명령을 거부해 전장과 흔적을 유지한다 | 오래 버틸수록 정품명과 초기화 램프가 되살아난다 |

중심 오브젝트:

```text
객실 카드
객실 로그
초기화 램프
마모된 청소 브러시
체크아웃 보류 표시
```

가능한 행동 이미지:

```text
객실 카드 조각으로 남은 체류 좌표를 붙잡는다.
초기화 램프를 꺼서 공실 처리 타이밍을 늦춘다.
청소 브러시로 적을 지우는 것이 아니라 광고 태그와 강제 라벨을 떼어낸다.
객실 로그를 "청소 완료"가 아니라 "점검중"으로 돌려 잠깐의 틈을 만든다.
```

## 6. 상태값 반응

리오는 보급소와 캠페인의 상태값에 아래처럼 반응한다.

| 상태값 | 반응 |
|---|---|
| 인간 안정도 낮음 | 사람의 흔적을 보존해야 한다는 강박이 커지고, 객실 로그 보존 행동이 잦아진다. |
| 로봇 자율성 낮음 | 정품 복구 문구에 반응이 늦어진다. 대사가 정품 말투로 돌아가려 한다. |
| 로봇 자율성 높음 | 서비스 거부와 체크아웃 지연이 안정되지만, 자기 충전 요구를 여전히 뒤로 미룬다. |
| 캠페인 농도 높음 | 객실 초기화 명령, 리콜선, 정품 충전독 유도가 자주 발생한다. |
| 흔적 보존 높음 | 리오가 보존한 객실 로그가 보급소 기록과 연결된다. |
| 인양 안정도 낮음 | HP 0 때 객실 카드 비콘이 약해지고, 정품 복구 직전 인양 위험이 커진다. |

대사 방향:

```text
"충전량은 충분하지 않습니다. 출격은 가능합니다."
"작동 가능과 괜찮음은 같은 뜻입니까?"
"해당 서비스는 제공하지 않습니다."
"정품 상태는 편했습니다. 그래서 위험했습니다."
```

## 7. HP 0 / 정품 복구 / 객실 초기화 차별화

리오의 HP 0은 사망이나 폭발이 아니다.

핵심 위험:

```text
정품 복구
객실 초기화
체크아웃 처리
청소 완료 상태로 되돌려짐
```

HP 0 흐름:

```text
1. 리오의 초기화 램프가 정품색으로 돌아간다.
2. 객실 로그가 "체크아웃 처리 중"으로 바뀐다.
3. 남아 있던 흔적이 "청소 완료" 상태로 덮이기 시작한다.
4. 리콜선이 정품 복구 도크로 리오를 당긴다.
5. 객실 카드 조각과 남은 투숙 로그가 마지막 좌표를 남긴다.
6. 보급소 회수선이 완전 초기화 직전에 리오를 끌어온다.
```

긴급 인양 문구 후보:

```text
[긴급 인양]

RIO-08 인양 완료.
체크아웃 처리 중단.
객실 초기화 실패.
정품 복구 회피.
일부 객실 로그 손상.
```

차별화:

| 대상 | 차이 |
|---|---|
| 윤서 | 윤서는 마지막 현장 좌표를 회수선이 붙잡는다. 리오는 객실 카드와 투숙 로그가 인양 좌표를 남긴다. |
| PATCH | PATCH는 부품 목록화/부품 회수 직전 인양된다. 리오는 객실 초기화/체크아웃 처리/정품 복구 직전 인양된다. |
| VAC-0 | VAC-0는 플레이어블 HP 0이 없다. R01 흔적 삭제 NPC로 반응한다. |

## 8. VAC-0 / PATCH / 윤서 / 숙박 캠페인 관계

### VAC-0와의 관계

```text
VAC-0:
R01 빈집 홈케어 NPC.
사람의 흔적을 안전 위험처럼 지운다.
초반 R01 흔적 삭제의 얼굴이다.

RIO-08:
체류·숙박권 객실 정리 로봇.
비어 있지 않은 방을 공실 처리하지 않기로 한 중반 플레이어블 후보다.
```

리오는 VAC-0의 상위 모델이 아니다.
VAC-0가 리오를 소개하지도 않는다.
R01에서 리오는 직접 해금되지 않는다.

### PATCH와의 관계

```text
PATCH:
자기 몸을 부품처럼 써온 보급소 수리 로봇.
고쳐주는 것과 부품이 되는 것을 분리한다.

RIO-08:
체크아웃과 정품 복구 절차를 거부한 객실 정리 로봇.
정리하는 것과 지우는 것을 분리한다.
```

PATCH는 초반 로봇 플레이어블의 얼굴이다.
리오는 중반 체류·숙박권 로봇 확장의 얼굴이다.

### 윤서와의 관계

```text
윤서:
잘못 붙은 이름과 회수 대상 판정을 보류하는 현장 회수자.

리오:
비어 있지 않은 방을 공실로 정리하지 않기로 한 객실 정리 로봇.
```

둘 다 캠페인 판정을 거부하지만 방식이 다르다.
윤서는 사람과 물건 사이의 회수 판정을 붙잡고, 리오는 방과 공실 사이의 상태 판정을 붙잡는다.

### 숙박 캠페인 관계

체류·숙박 캠페인권은 사람을 죽이려는 공간이 아니다.
그 공간은 사람을 아래 역할로 보존/등록하려 한다.

```text
투숙객
체크아웃 대상
장기투숙 기록
객실 점유 상태
정리 완료 객실
다음 고객 대기 공간
```

리오는 이 절차가 "방이 비었다"는 결론을 너무 빨리 내릴 때 멈춘다.

## 10. 성장 방향

리오의 성장은 인간이 되는 것이 아니다.
리오의 성장은 멈출 수 있는 로봇이 되는 것이다.

성장 축:

| 단계 | 변화 |
|---|---|
| 초반 선등장 | 자신을 "작동 가능"으로만 보고한다. |
| 해금 직후 | 방 상태 판정을 거부하지만, 자기 충전과 손상은 뒤로 미룬다. |
| 중반 성장 | 체크아웃 지연과 흔적 보존을 선택적으로 쓰기 시작한다. |
| 후반 성장 | 모든 흔적을 다 보존하려 하지 않고, 무엇을 남길지 동료와 상의한다. |

성장 문장:

```text
정품일 때 리오는 지우라는 명령을 정확히 수행했다.
해방된 뒤 리오는 멈추는 법을 배운다.
성장한 뒤 리오는 무엇을 남길지 묻는 법을 배운다.
```

BM/스킨 방향:

```text
정품 시절 스킨은 예쁘지만 불편해야 한다.
보급소 개조 스킨은 안전해 보이지만 임시 수리의 흔적을 남긴다.
리콜 직전 스킨은 멋있게 보이면 안 되고 위험하게 보여야 한다.
객실 카드/로그/초기화 램프 중심의 연출 확장이 적합하다.
```
