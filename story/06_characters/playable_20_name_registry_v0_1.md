# 플레이어블 20인 이름 레지스트리 v0.1

## 문서 상태

```text
상태:
작성 완료 v0.1

작업 성격:
20명 상세 프로필 작성 아님.
플레이어블 20인 이름/코드명/유저 첫 표기 관리 문서.
중복 검사와 노출 규칙을 위한 레지스트리.

최신 반영:
1번 윤서 확정
2번 PATCH 확정
3번 OPEN-HOST / 서이겸 NPC 선등장 후 전환 후보
4번 강하람 / HOLD-04 프로필 v1.0 작성 완료
8번 리오 보류 잠금

다음 작업 포인터:
5번 한이루 / RETURN-05 / 반송 수취인 상세 결재
```

## 0. 판정 요약

판정:

```text
통과 / v0.1 이름 레지스트리로 사용
```

이 문서는 이름을 멋있게 붙이는 목록이 아니다.

목적은 세 가지다.

1. 이미 확정된 이름과 코드명을 흔들지 않는다.
2. 아직 상세 결재 전인 슬롯에는 중복 위험이 낮은 후보 이름을 붙인다.
3. 본명을 늦게 공개해야 하는 캐릭터는 첫 표기와 내부 본명을 분리한다.

중요:

```text
이름 레지스트리는 프로필 확정 문서가 아니다.
프로필 v1.0이 없는 슬롯의 이름은 상세 결재 때 바뀔 수 있다.
```

## 3. 중복 검사

### 3.1 기존 확정/준확정 이름

아래 이름은 이미 주요 문서에서 쓰인다.

```text
윤서
PATCH / 패치
OPEN-HOST / 서이겸
강하람
리오
미나
장도윤
팝시
세븐
복희
```

R01 핵심 NPC 이름:

```text
서이겸
강연우

문채율
권태오
노기준
류가람
정해원
채아린 / 아린
```

사람 이름 없음:

```text
MAIL-LOOP

```

### 3.2 현재 충돌 판정

| 항목 | 판정 | 메모 |
|---|---|---|
| 완전 동일 이름 | 통과 | 20인 레지스트리 안에서 완전 동일 본명 없음 |
| 윤서와 직접 충돌 | 통과 | 서/윤 조합 반복을 피했고, 윤서와 같은 이름 없음 |
| PATCH와 충돌 | 통과 | PATCH는 로봇 코드명 유지. 사람 이름 부여 금지 |
| OPEN-HOST와 충돌 | 주의 | 서이겸은 R01 NPC 레지스트리와 공유하는 의도된 본명 |
| 강하람과 충돌 | 주의 | R01 NPC 류가람과 끝음이 가깝다. 플레이어블 본명은 강하람으로 유지 |
| 리오와 충돌 | 통과 | 리오와 비슷한 2음절 로봇 이름 추가 없음 |
| R01 NPC와 충돌 | 통과 | 동일 본명 없음. 성씨 중복은 허용 |
| 발음 과밀 | 조건부 통과 | 5~19번은 상세 결재 때 다시 소리 내어 점검 |

중요:

```text
강하람은 유지한다.
하람/가람 호명감은 주의하지만, 류가람은 R01 NPC이고 강하람은 4번 플레이어블이므로 현 단계에서는 충돌로 보지 않는다.
```

## 4. 국적/지역 잠금

| 구분 | 슬롯 | 판정 |

| 한국인/E01 인간으로 운용 | 1, 3, 4, 5, 6, 7 | 1 윤서, 3 서이겸, 4 강하람은 확정/준확정. 5~7은 E01 초반 후보라 한국어권 이름 후보 사용 |

| 지역 미확정 | 9~20 | 한국어권 이름 후보를 임시 배정했지만, 상세 결재에서 글로벌 권역이 확정되면 현지명으로 교체 가능 |

운영 원칙:

```text
초반 1~7번은 E01 한반도 서부권 감각을 유지한다.
9~20번은 이름보다 캠페인 등록 방식이 먼저다.
세계 확장 지역이 정해지면 9~20번 이름은 과감히 조정할 수 있다.
```

## 5. 노출 규칙

### 5.1 첫 노출

첫 노출은 유저가 바로 이해하는 표기다.

예:

```text
윤서 / 현장 회수자
PATCH / 보급소 수리 로봇
OPEN-HOST / 전직 오픈하우스 안내인
하람 / 퇴원 보류자
이루 / 반송 수취인
```

첫 노출에서 하지 말 것:

```text
20명 본명을 한꺼번에 공개.
코드명을 전부 사람 이름으로 교체.
역할명을 빼고 이름만 노출.
성씨를 앞세워 현실 인명록처럼 보이게 하기.
```

### 5.2 상세 결재 노출

상세 결재 때는 아래 순서로 확정한다.

```text
1. 유저 첫 표기
2. 코드명
3. 본명 또는 본명 보류 여부
4. 본명 공개 시점
5. 이름이 캠페인 등록 방식과 충돌하는 지점
```

### 5.3 본명 보류

본명을 보류해야 하는 경우:

- 로봇이 사람 이름을 필요로 하지 않을 때.
- 캠페인이 이름 대신 역할명으로 고정하는 감각이 중요할 때.
- 본명이 너무 빨리 나오면 정체가 약해질 때.
- 20번처럼 시즌 후반 메타성이 너무 강해질 때.

## 7. 금지 방향

```text
20명 전체 상세 프로필을 이 문서에서 쓰지 않는다.
이름만 멋있게 붙이고 캠페인 등록 방식을 흐리지 않는다.
모든 캐릭터를 한국인 이름으로 최종 잠금하지 않는다.
모든 캐릭터를 코드명으로만 남기지 않는다.
로봇에게 억지 사람 이름을 붙이지 않는다.
본명 공개를 곧 구원 완료로 처리하지 않는다.
윤서와 비슷한 이름, 하람과 비슷한 이름, 리오와 비슷한 짧은 로봇명을 남발하지 않는다.
```

## 8. 다음 작업 포인터

권장 다음 작업:

```text
5번 한이루 / RETURN-05 / 반송 수취인 상세 결재
```

5번 상세 결재에서 확인할 것:

1. 한이루 / RETURN-05 / 이루, 반송 수취인 이름이 역할과 맞는가.
2. 윤서의 반품/회수 축과 겹치지 않는가.
3. 주소/배송 방향, 수령 완료와 반송 완료 사이의 틈을 전투 리듬으로 만들 수 있는가.
4. R03을 너무 빨리 최종 반품 심사실까지 열지 않는가.
