## 0. 최종 결론

출시 초기 플레이어블 구조는 아래처럼 잠근다.

```text
출시 초기 U5:
윤서 단독 시작.

출시 초기 +1:
현재 확정하지 않는다.
한국인 기본 주인공 +1~2는 최대치일 뿐이며, 현재는 +1도 열지 않는다.

PATCH:
U1~U4 보류.
보급소 존재감, 짧은 동행, 로봇 자율성 씨앗은 허용하되 U5 확정은 금지한다.

OPEN-HOST:
R01 NPC 선등장 유지.
출시 초기 U5 전환 금지.

HOLD / RETURN / METER / QUIET:
프로필 산출물은 보존하되 출시 초기 해금 금지.

미개방 캠페인 후보:
상세 결재/프로필/출시 초기 당김 금지.
```

이 문서는 캐릭터 수를 늘리기 위한 문서가 아니다.
초기 보급소가 작고 취약하게 느껴지도록, 무엇을 열지 않을지 잠그는 문서다.

## 2. 출시 초기 U5 잠금

출시 초기 U5는 윤서 단독을 기본으로 한다.

윤서 단독 시작이 안전한 이유:

1. 첫 보급소와 첫 R01 해석의 얼굴이 흐려지지 않는다.
2. 첫 긴급 인양이 윤서의 경험으로 선명하게 남는다.
3. 유저가 R01을 "동료 소개장"이 아니라 "혼자 판단해야 하는 지역"으로 읽는다.
4. 보급소가 처음부터 안정된 팀처럼 보이지 않는다.
5. PATCH와 OPEN-HOST의 존재감은 U1~U4 또는 NPC 선등장만으로도 충분히 줄 수 있다.

금지:

```text
튜토리얼 다양성을 이유로 PATCH를 U5로 확정.
R01 내부 설명을 이유로 OPEN-HOST를 U5로 확정.
한국인 동료가 필요하다는 이유로 HOLD/RETURN/METER/QUIET를 출시 초기로 당김.
```

## 3. PATCH 잠금

PATCH는 폐기하지 않는다.
하지만 출시 초기 +1로 확정하지 않는다.

허용:

```text
U1 신호 해금:
보급소 수리 흔적, 빠진 나사, 끊어진 회수선.

U2 조우:
보급소에서 직접 대면.

U3 거점 협력:
충전 스탠드, 회수선 안정, 장비 코멘트.

U4 동행/임무:
짧은 동행 또는 게스트 조작 가능.
```

금지:

```text
U5 출시 초기 +1 확정.
PATCH를 윤서의 장비로 처리.
PATCH를 힐러/마스코트/자동 수리봇으로 처리.
R04 정품/충전 캠페인을 PATCH 때문에 조기 개방.
```

## 4. OPEN-HOST 잠금

OPEN-HOST는 R01 NPC 선등장을 유지한다.

허용:

```text
U1:
뒤집힌 안내판, 어긋난 화살표.

U2:
상담 부스 직접 대면.

U3:
보스 진입 동선 검증, 보급소 논쟁.

후속 U4/U5 전환:
R01 결절 처리와 다른 NPC 기능 보존 뒤 별도 검토.
```

금지:

```text
출시 초기 U5.
R01 안내 NPC 기능 침범.
R01 절차 설명 캐릭터화.
WELCOME-DESK, ROOM-12, DINING-4, FRAME-LEFT, VAC-0, MAIL-LOOP 기능 흡수.
윤서 단독 시작 약화.
```

OPEN-HOST는 안내하는 사람이 아니라, 안내했다는 사실을 검증받는 사람이다.

## 5. E01 인간 후보 잠금

HOLD, RETURN, METER, QUIET는 모두 보존한다.
하지만 출시 초기 해금은 금지한다.

이유:

1. 윤서가 이미 한국인 기본 주인공이다.
2. OPEN-HOST도 R01 한국어권 NPC 선등장 후보로 남아 있다.
3. PATCH까지 U1~U4로 존재하면 초반 캐릭터 신호는 충분하다.
4. HOLD/RETURN/METER/QUIET를 당기면 E01 한국인 인간 후보가 과밀해진다.
5. 각 후보의 진짜 힘은 R02/R03/R04/R08 캠페인 상태 반응이 깔린 뒤 나온다.

## 6. 미개방 캠페인 잠금

미개방 캠페인 후보는 출시 초기 플레이어블 구조에 넣지 않는다.

금지:

```text
GATE-09를 첫 외부 후보라는 이유로 출시 초기로 당김.
BROAD-10, CLAUSE-11, LEDGER-12를 이미 프로필이 있다는 이유로 당김.
GUARD-13, MODEL-15, LOCKOUT-17을 산출물이 있다는 이유로 고정.
NAME/TICKET/RATION/LINE/OBS 계열 상세 결재 시작.
```

원칙:

```text
캠페인이 먼저 열린다.
NPC, 오브젝트, 상태 반응, 작은 결말이 먼저 깔린다.
그 뒤에 플레이어블 상세 결재가 가능하다.
```
