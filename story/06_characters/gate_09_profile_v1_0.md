## 0. 캐릭터 핵심

미카는 빠른 이동 캐릭터가 아니다.
미카는 역무원도, 지하철 안내원도, 개찰구를 몰래 넘는 사람도 아니다.

미카는 E02_C07 지하철 환승 심사권에서 막차/환승/개찰 절차가 사람을 "플랫폼 잔류자"로 고정하기 직전의 짧은 틈을 붙잡는 사람이다.

핵심 문장:

```text
문이 닫히기 전까지는,
아직 잔류자가 아니다.
```

최종 변화 문장:

```text
문이 닫힌 뒤에도,
사람은 완료된 절차가 아니다.
```

잘못 믿는 것:

```text
마지막 환승만 통과하면 아직 잔류자가 아니라고 믿는다.
```

이 믿음은 일부 맞다. 환승 심사 로그가 살아 있고, 개찰 클립이 완전히 끊기지 않았고, 막차 안내 기록이 닫히지 않았다면 캠페인은 미카를 아직 "이동 중"으로도 읽는다.

하지만 이 믿음은 위험하다. E02_C07은 사람을 죽이려는 곳이 아니다. 혼란한 플랫폼의 사람을 환승자, 미탑승자, 통행 허가 대기자, 플랫폼 잔류자로 안전하게 분류하려 한다. 문제는 그 분류가 닫히는 순간, 아직 돌아갈 곳을 찾는 사람까지 "잔류 처리 완료"로 보존한다는 점이다.

플레이어블로서 미카가 다루는 질문:

```text
막차를 놓친 사람은 아직 이동 중인가,
이미 잔류자인가?
```

## 4. 플레이어블 기능

미카의 플레이어블 기능은 이동 속도 증가가 아니다.
미카는 캠페인 절차가 내리는 통과/환승/미탑승/잔류 판정을 직접 밀고 당긴다.

핵심 기능:

| 기능 | 의미 | 실패 위험 |
|---|---|---|
| 환승 중 판정 유지 | 통행 금지나 추적 확정이 닫히기 전 짧은 보류 상태를 만든다 | 보류가 길어지면 환승 심사 로그가 오염된다 |
| 개찰 미통과 좌표 | 완전 통과도 실패도 아닌 개찰 클립 좌표를 남긴다 | 과사용하면 통행태그가 훼손된다 |
| 미탑승 확정 지연 | 막차 탑승 실패가 플랫폼 잔류 처리로 굳기 전 틈을 만든다 | 시간표가 압축되고 다음 안내가 거칠어진다 |
| 저주파 안내 반송 | 공식 안내 반복을 일부 늦추고 보급소 비콘과 충돌시킨다 | 수신태그 부담과 인양 불안이 커진다 |
| 잔류 처리 보류 | 적/함정/구역의 닫힘 판정을 잠시 플랫폼 대기열로 미룬다 | 군중 제어 마법처럼 커지면 실패다 |

플레이어블이어야 하는 이유:

```text
NPC 미카는 막차 안내를 말할 수 있다.
플레이어블 미카는 막차 안내가 닫히는 순간을 직접 비틀게 한다.
```

기존 후보와의 분리:

- 윤서는 현장 회수와 잘못 붙은 이름을 보류한다. 미카는 통행/미통과/잔류 판정을 보류한다.
- PATCH는 자기 몸을 부품처럼 써온 보급소 수리 로봇이다. 미카는 개찰 클립과 환승 로그에 묶인 인간 후보다.
- 여울은 낮은 수신으로 덜 들키지만 덜 구해진다. 미카는 공식 안내가 너무 선명하게 반복되어 잔류자로 굳는다.
- 리오는 방의 체크아웃과 객실 초기화를 거부한다. 미카는 플랫폼의 잔류 처리와 환승 로그 폐쇄를 거부한다.

## 5. 전투 리듬

기본 리듬:

```text
통과 / 환승 / 미탑승 / 잔류 판정 조작
```

고정 동사:

```text
보류한다
늦춘다
환승 중으로 둔다
미탑승 확정을 지연한다
개찰 클립을 남긴다
잔류 처리를 미룬다
닫히는 판정 사이를 통과한다
```

금지 동사:

```text
순간이동한다
질주한다
훔친다
역을 안내한다
철도망을 장악한다
문을 계속 연다
```

전투 흐름:

1. 닫히는 게이트나 추적 판정 앞에서 "환승 중" 상태를 만든다.
2. 개찰 클립을 남겨 통과 실패와 긴급 인양 좌표를 동시에 건다.
3. 막차 안내 기록을 태워 미탑승 확정을 한 박자 늦춘다.
4. 적의 호출이나 캠페인 닫힘 절차를 플랫폼 잔류 대기열로 우회시킨다.
5. 보류가 길어지면 환승 심사 로그가 오염되고, 다음 안내가 더 강한 공식 명령으로 돌아온다.

위험한 강점:

| 강점 | 플레이 감각 | 대가 |
|---|---|---|
| 짧은 통행 창 | 닫히는 문 사이의 한 박자를 만든다 | 통행태그 훼손 |
| 환승 중 상태 | 완전 통과 없이도 절차를 미룬다 | 환승 심사 로그 오염 |
| 막차 안내 지연 | 캠페인 호출을 한 박자 늦춘다 | 다음 호출 압축 |
| 개찰 클립 좌표 | HP 0 때 보급소가 잡을 단서를 남긴다 | 클립 절단 시 인양 불안 |
| 잔류 처리 보류 | 적과 함정의 닫힘 판정을 늦춘다 | 플랫폼 잔류표 누적 |

실패 시 후폭풍:

```text
시간표가 닫힌다.
개찰 클립이 잘린다.
환승 도장이 무효화된다.
안내 방송이 같은 문장을 반복한다.
미카는 이동 중인 사람이 아니라 플랫폼 잔류자로 읽힌다.
```

파티/동행 시 역할:

```text
팀 전체를 빠르게 이동시키지 않는다.
특정 캠페인 절차가 닫히는 순간을 늦추고,
위험한 통행 판정을 한 번 더 보류한다.
```

## 6. 태그 / 정산 / 자원

정산 루트:

| 항목 | 용도 | 금지선 |
|---|---|---|
| 통행태그 | 통과/미통과/환승 중 판정을 열고 닫는 주 태그 | 이동 속도 재화처럼 쓰지 않는다 |
| 수신태그 | 저주파 안내 반복, 보급소 비콘 충돌, 막차 방송 간섭 | 여울의 낮은 수신과 섞지 않는다. 미카는 공식 안내 과잉을 다룬다 |
| 환승 심사 로그 | 환승자/미탑승자/플랫폼 잔류자 판정 기록 | 단순 퀘스트 로그로 만들지 않는다 |
| 개찰 클립 | 미통과 좌표와 긴급 인양 단서 | 만능 열쇠나 도둑 도구로 쓰지 않는다 |
| 막차 안내 기록 | 시간표가 닫히는 압력과 HP 0 손실 | 관광 수집품으로 만들지 않는다 |
| 정기권 조각 | 제한적 재통과와 환승 중 상태 보조 | 무료 무제한 통행권으로 쓰지 않는다 |
| 플랫폼 잔류표 | 실패 정산과 잔류 처리 누적 흔적 | 영구 낙인이나 사망표로 쓰지 않는다 |

상태값 반응:

| 상태값 | 미카의 반응 |
|---|---|
| 캠페인 농도 높음 | 막차 안내와 개찰 심사가 더 빨리 닫히며, 잔류 처리 누적이 커진다 |
| 수신 안정도 낮음 | 보급소 비콘이 안내 반복과 충돌해 개찰 클립 좌표가 흔들린다 |
| 인양 안정도 낮음 | HP 0 때 환승 로그 불일치를 잡는 시간이 길어진다 |
| 보급소 노출도 높음 | 개찰 클립 좌표가 보급소 회수선을 역추적당할 위험이 커진다 |
| 통행태그 부족 | 환승 중 판정을 유지하지 못하고 미탑승 확정이 빨라진다 |

정산 감각:

```text
좋은 결과는 더 빨리 이동하는 것이 아니다.
잔류 처리 확정 전에 다음 경로와 인양 좌표가 아직 살아 있음을 남기는 것이다.
```

## 7. HP 0 / 긴급 인양

미카의 HP 0은 사망/부활이 아니다.

핵심 상태명:

```text
개찰 미통과 / 플랫폼 잔류 처리
```

단계별 흐름:

| 단계 | 상태 | 연출 |
|---|---|---|
| 1 | 개찰 미통과 | 개찰 클립이 잘리고 통행태그가 미확인으로 깜박인다 |
| 2 | 막차 탑승 실패 | 막차 안내판이 "출발 완료"와 "환승 대기"를 번갈아 표시한다 |
| 3 | 플랫폼 잔류 처리 | 캠페인이 미카를 이동 중인 사람이 아니라 플랫폼 잔류자로 보존하려 한다 |
| 4 | 저주파 안내 반복 | 같은 안내 문장이 반복되며 보급소 비콘과 충돌한다 |
| 5 | 환승 로그 폐쇄 직전 | 환승 심사 로그가 닫히고 개찰 클립 좌표가 끊기기 시작한다 |
| 6 | 긴급 인양 | 보급소가 개찰 클립 좌표와 환승 심사 로그 불일치를 잡아 되당긴다 |

보급소 인양 단서:

```text
개찰 클립 좌표
환승 심사 로그 조각
```

## 13. 성장 방향

초기:

```text
마지막 환승만 통과하면 아직 잔류자가 아니라고 믿는다.
```

미카는 막차를 놓치지 않으려 한다. "환승 중"이라는 판정이 남아 있으면 선택지도 남아 있다고 믿는다.

중반:

```text
환승 중 상태가 길어질수록 사람은 더 안전해지는 것이 아니라,
더 정확하게 플랫폼 잔류자로 보존된다는 것을 배운다.
```

미카는 보류가 구원은 아니라는 사실을 배운다. 판정을 늦추는 것은 필요하지만, 계속 늦추기만 하면 캠페인은 더 강한 문장으로 사람을 붙잡는다.

후반:

```text
닫힌 문을 부수는 것이 아니라,
닫힌 뒤에도 사람이 남아 있으면 절차가 완료되지 않았다고 말할 수 있게 된다.
```

최종 변화 문장:

```text
문이 닫힌 뒤에도,
사람은 완료된 절차가 아니다.
```

성장 방향의 핵심은 속도 향상이 아니다. 미카는 더 빨라지는 캐릭터가 아니라, 더 정확히 "아직 끝나지 않은 이동"을 구분하는 캐릭터가 되어야 한다.
