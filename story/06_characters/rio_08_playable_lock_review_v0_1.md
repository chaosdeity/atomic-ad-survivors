# 리오 8번 플레이어블 잠금 재검토 v0.1

## 문서 상태

```text
상태:
작성 완료 v0.1

작업 성격:
최종 프로필 신규 작성 아님.
기존 리오/RIO-08 후보의 8번 슬롯 잠금 재검토 문서.
VAC-0/PATCH와의 중복 위험, 해금 시점, HP 0 차별화 검토.

판정:
조건부 통과 / 8번 슬롯 유지 / 직접 해금은 중반 체류·숙박 캠페인권까지 보류

다음 작업 포인터:
9~12번 중반 플레이어블 후보군 선정
```

## 0. 판정 요약

| 항목 | 판정 | 메모 |

| 즉시 플레이어블 해금 | 보류 | R01/P0/P1에서 직접 열면 VAC-0, PATCH, R01의 질문이 흐려진다. |
| 최종 재검토 판정 | 조건부 통과 | 8번 번호는 유지하되, E08_C01 체류·숙박권이 열리기 전에는 직접 조작 캐릭터로 공개하지 않는다. |
| 후속 작업 | 프로필 v1.0 작성 | 조건부 해소 문서에서 VAC-0/PATCH 분리, 객실 초기화 HP 0, 해금 범위를 고정했다. |

```text
리오는 8번에 둔다.
단, 리오는 초반 R01 로봇도, PATCH의 후속 부품 로봇도 아니다.
리오는 체류/숙박 캠페인권에서 "비어 있지 않은 방을 비어 있다고 정리하지 않기로 한 로봇"이다.
```

## 1. 기존 로스터 기준

현재 1~8번 흐름은 아래처럼 유지한다.

| 슬롯 | 현재 잠금 | 역할 |
|---:|---|---|
| 1 | 윤서 확정 | R01 첫 출격, 현장 회수자, 첫 긴급 인양의 얼굴 |
| 2 | PATCH 확정 | 침묵 보급소 수리 로봇, 자기 몸을 부품처럼 써온 로봇 |
| 3 | OPEN-HOST 후보 예약 | R01 NPC 선등장 후 전환 후보 |
| 4 | 강하람 / HOLD-04 확정 | 퇴원 보류자, R02 의료/보험 절차 |
| 5 | 한이루 / RETURN-05 확정 | 반송 수취인, R03 물류/반품 절차 |
| 6 | 차유건 / METER-06 확정 | 폐충전소 계량자, R04 충전/계량 절차 |
| 7 | 백여울 / QUIET-07 확정 | 침묵권 피난자, R08 수신/인양 리스크 |
| 8 | 리오 / RIO-08 조건부 유지 | 체류·숙박 캠페인권의 탈정품 객실 정리 로봇 |

8번은 "첫 로봇" 슬롯이 아니다.

```text
2번 PATCH가 이미 초반 로봇 플레이어블이다.
8번 리오는 3~7번 인간 후보가 닫힌 뒤, 세계가 R01 밖 중반 체류·숙박권으로 넓어질 때 다시 여는 로봇 확장 슬롯이다.
```

## 2. VAC-0와의 차이

VAC-0는 R01 NPC다.

| 구분 | VAC-0 | 리오 |
|---|---|---|
| 위치 | R01 서부 스마일홈, 빈집/홈케어 장치 | E08_C01 체류·숙박 캠페인권, 객실 정리 로봇 |
| 역할 | 집을 안전하게 만들려고 흔적을 치우는 작은 청소기 | 객실을 비었다고 초기화하라는 명령을 거부한 로봇 |
| 핵심 오판 | "깨끗한 집이면 안전하다." | "정품 상태로 돌아가면 편해질 수 있다"는 유혹을 두려워한다. |
| 플레이어블 여부 | R01 핵심 보조 NPC 유지 | 8번 플레이어블 후보 |
| 다루는 흔적 | 먼지, 발자국, 낙서, 우편, 보급소 표식을 위험물처럼 청소 | 객실 로그, 카드키, 투숙 흔적, 아직 나가지 않은 사람의 체류 기록 |
| 기능 | R01의 흔적 보존/삭제 위험을 보여준다 | 체류·숙박권의 공실 처리와 객실 초기화를 비튼다 |

분리 기준:

```text
VAC-0는 "사람의 흔적을 위험물처럼 청소하는 R01 NPC"다.
리오는 "사람이 아직 있는 방을 공실로 처리하지 않기로 한 E08_C01 플레이어블 후보"다.
```

리오를 VAC-0의 업그레이드형, 후속 기체, R01 청소 로봇으로 만들면 탈락이다.

## 3. PATCH와의 차이

PATCH는 2번 플레이어블로 확정되어 있다.

| 구분 | PATCH | 리오 |
|---|---|---|
| 위치 | 침묵 보급소, C04 정품/수리 잔향 | E08_C01 체류·숙박 캠페인권 |
| 존재 방식 | 자기 몸을 부품처럼 써온 보급소 수리 로봇 | 객실 초기화 명령을 거부한 탈정품 객실 정리 로봇 |
| 핵심 문장 | 고쳐주는 것과, 부품이 되는 것은 다르다 | 비어 있지 않은 방을 비어 있다고 정리하지 않는다 |
| 전투 감각 | 패치, 리벳, 자석 수리팔, 임시 봉합, 자기부품화 | 객실 카드, 청소 브러시, 초기화 램프, 서비스 거부, 객실 로그 보존 |
| HP 0 위험 | 정품 복구/부품 회수/부품 목록화 직전 인양 | 정품 복구/객실 초기화/리콜 직전 인양 |
| 열리는 시점 | 윤서 이후 초반 합류 | 여러 캠페인권을 이해한 뒤 중반 8번 슬롯 |

분리 기준:

```text
PATCH는 보급소를 고치다가 자기 몸까지 자재로 써온 로봇이다.
리오는 방을 정리하다가 살아 있는 체류 흔적을 지우는 명령을 거부한 로봇이다.
```

리오가 PATCH처럼 수리하고, 부품을 떼어주고, 보급소 장비를 안정시키는 캐릭터가 되면 실패다.

## 4. R01 출신 금지와 체류·숙박권 고정

리오는 R01 출신이 아니다.

고정 출신:

```text
E08_C01 관광숙박 거주권
또는 같은 축의 체류/숙박 캠페인권
```

리오가 다루는 질문:

```text
방은 언제 비었다고 판정되는가?
체류 기록이 사라지면 사람도 나간 것인가?
정품 서비스가 편안하게 초기화해주는 것은 보호인가, 삭제인가?
```

R01에서 가능한 것은 암시뿐이다.

- 객실 카드 조각.
- "호환 가능한 홈케어 장치"라는 잘못된 분류 로그.
- VAC-0와 대비되는 청소 제외 목록의 작은 흔적.
- 리오 본인 직접 해금이 아니라, 체류·숙박권이 나중에 열린다는 예고.

R01에서 금지되는 것:

```text
리오를 R01 청소 NPC로 등장시키기.
리오를 VAC-0의 상위 모델로 설명하기.
리오를 PATCH의 보급소 로봇 친구로 조기 합류시키기.
R01 보스 전후에 리오를 플레이어블로 열기.
```

## 5. 8번 슬롯 적합성

판정:

```text
8번 슬롯은 적합하다.
다만 "초반 로봇 추가"가 아니라 "중반 외부 체류·숙박권 확장 로봇"으로 적합하다.
```

적합한 이유:

1. 3~7번은 한국인 인간 후보 중심으로 R02/R03/R04/R08을 넓힌다.
2. 8번에서 리오가 들어오면, 유저는 "R01만 문제가 아니고 다른 권역도 각자 사람을 등록한다"는 감각을 받는다.
3. PATCH와 달리 리오는 보급소 수리가 아니라 객실 초기화와 정품 서비스 거부를 다룬다.
4. VAC-0와 달리 리오는 NPC 씨앗이 아니라 조작 가능한 중반 관점이 될 수 있다.
5. 리오의 정품 복구 공포는 6번 차유건의 정품 충전권 위험을 장기적으로 이어받지만, 같은 R04 사건으로 앞당기지 않는다.

부적합해지는 경우:

```text
리오를 초반 3~7번 사이로 당긴다.
리오를 R01에서 직접 해금한다.
리오를 PATCH 이후 바로 열어 로봇 플레이어블을 과밀하게 만든다.
리오의 체류·숙박권이 열리기 전에 정품 복구 공포만 소비한다.
```

## 6. 해금 시점

해금 판정:

| 단계 | 처리 | 이유 |
|---|---|---|
| R01/P0 | 직접 해금 금지 | VAC-0, OPEN-HOST, PATCH의 기능과 첫 지역 감각을 보호한다. |
| R01/P1 | 암시 가능 | 객실 카드, 잘못된 호환 로그, 청소 제외 목록 정도만 남긴다. |
| R02/R03/R04/R08 | 직접 해금 금지 | 4~7번 인간 후보의 슬롯 리듬을 유지한다. |
| E08_C01-L01 | 선등장/흔적 추적 가능 | 체류·숙박권의 객실 로그와 공실 처리 규칙을 보여준다. |
| E08_C01-L02 이후 | 플레이어블 해금 검토 | 유저가 체류 기록, 객실 초기화, 정품 복구의 위험을 이해한 뒤 열 수 있다. |

권장 해금:

```text
E08_C01 체류·숙박 캠페인권이 실제 플레이 공간으로 열린 뒤,
리오가 비어 있지 않은 객실을 공실 처리하지 않았다는 사실이 드러나는 지점에서 해금한다.
```

## 7. 전투/탐색 리듬 후보

리오는 암살자나 은신 로봇이 아니다.

전투/탐색 중심:

| 요소 | 사용 방향 |
|---|---|
| 객실 카드 | 문을 여는 열쇠보다, 아직 끝나지 않은 체류 기록을 붙잡는 증거 |
| 마모된 청소 브러시 | 적을 쓸어내는 무기보다, 흔적을 지우지 않기로 멈춘 장비 |
| 초기화 램프 | 객실 공실 처리/정품 복구가 가까워졌음을 보여주는 위험 신호 |
| 객실 로그 | 투숙중/공실/청소대기/점검중 판정을 바꾸는 전투 리듬 |
| 서비스 거부 | 정품 명령을 끊고, 초기화/정리 명령을 지연시키는 행동 |

리듬 후보:

```text
1. 공실 처리 지연:
   방이나 적이 "비었다"고 판정되는 순간을 늦춘다.

2. 청소 제외 지정:
   지워질 흔적 일부를 보호 대상으로 잠시 바꾼다.

3. 객실 상태 전환:
   투숙중, 청소대기, 점검중 판정을 바꾸어 캠페인 호출과 이동 경로를 흔든다.

4. 서비스 거부:
   정품 장치 명령을 거절하는 대신 정품 복구 위험이 누적된다.
```

금지:

```text
넓은 범위 청소빔.
단순 기계 팔 공격.
PATCH처럼 수리/버프를 제공하는 역할.
VAC-0처럼 집 안 흔적을 자동 청소하는 NPC 기능 반복.
```

## 8. HP 0 / 정품 복구 / 객실 초기화 차별화

리오의 HP 0은 파괴가 아니다.

핵심 위험:

```text
정품 복구.
객실 초기화.
리콜.
공실 처리 완료.
```

차별화 표:

| 캐릭터 | HP 0 위험 | 긴급 인양 감각 |
|---|---|---|
| 윤서 | 마지막 좌표 상실, 현장 회수 실패 | 회수선이 마지막 좌표를 붙잡는다. |
| PATCH | 부품 목록화, 정품 복구, 부품 회수 | 보급소 수리선이 부품 회수 직전에 당긴다. |
| VAC-0 | 플레이어블 HP 0 없음 | NPC로서 청소 제외 목록/흔적 보존 반응을 담당한다. |
| 리오 | 객실 초기화와 정품 복구가 겹치는 리콜 직전 | 객실 카드 조각과 남은 투숙 로그가 비콘처럼 버티고, 보급소가 리콜선보다 늦게 잡아당긴다. |

리오 HP 0 연출 후보:

```text
1. 초기화 램프가 정품색으로 돌아간다.
2. 객실 로그가 "공실 / 청소 완료"로 덮이기 시작한다.
3. 객실 카드 조각이 마지막 투숙 기록을 붙잡는다.
4. 보급소 회수선은 PATCH의 수리선처럼 빠르지 않고, 리콜선과 경합한다.
5. 인양 후 일부 로그가 손상되어 다음 출격에서 정품 복구 위험이 남는다.
```

금지:

```text
HP 0을 단순 폭발/재조립으로 처리하지 않는다.
PATCH의 부품 회수 연출을 반복하지 않는다.
VAC-0의 청소통/청소 제외 목록을 플레이어블 HP 0로 가져오지 않는다.
리오가 죽었다 살아나는 식으로 쓰지 않는다.
```

## 9. 너무 빨리 열 때의 위험

리오를 너무 빨리 열면 아래 문제가 생긴다.

| 위험 | 손상되는 축 | 이유 |
|---|---|---|
| R01의 힘 약화 | VAC-0, OPEN-HOST, R01 NPC 12명 | R01의 주거/가족/흔적 질문이 숙박권의 객실 질문으로 덮인다. |
| PATCH 의미 약화 | 2번 플레이어블 | 초반 로봇 자율성의 얼굴이 PATCH에서 분산된다. |
| 로봇 과밀 | 2 PATCH, 8 리오 | 3~7번 인간 후보 사이의 생활 절차 감각이 약해진다. |
| 정품 복구 조기 소비 | 차유건, PATCH, 리오 | R04의 정품/충전 공포와 E08_C01의 정품 복구 공포가 한 덩어리로 뭉친다. |
| 세계 확장 과속 | E08_C01 | 체류·숙박권을 열기 전에 호텔/객실/장기투숙 질문을 설명해야 한다. |

따라서 리오는 번호를 유지하되, 공개 속도는 늦춘다.

## 10. 금지 방향

```text
리오를 R01 출신으로 만들지 않는다.
리오를 VAC-0의 상위 모델, 후속 모델, 업그레이드형으로 만들지 않는다.
```
