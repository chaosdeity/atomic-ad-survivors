## 0. 조건부 통과 사유 요약

기존 상세 결재 문서는 5번 반송 수취인을 조건부 통과로 판정했다.

조건부 판정의 이유:

1. R03 물류/반품 캠페인을 여는 기능은 강하지만, 윤서의 회수자/반품 담당자 정체성과 가까워 보일 위험이 크다.
2. 송장, 박스, 스캐너 이미지만 남으면 단순 택배원, 도적, 스캐너 캐릭터로 납작해질 수 있다.
3. R03-L04/L05까지 당기면 윤서 핵심 보스와 반품 담당자 과거가 너무 빨리 열린다.
4. R01 MAIL-LOOP와 연결하되, MAIL-LOOP의 주소 기능을 흡수하는 하위 캐릭터가 되면 안 된다.

보완의 목표:

```text
이 후보를 회수자가 아니라,
자기 자신이 반송/미수령/재배송 대상으로 읽힌 사람으로 잠근다.
```

## 2. 윤서와의 차이 고정

핵심 차이:

```text
윤서:
현장에서 위험물을 회수하고 반품하는 사람.

이루:
자기 자신이 반송/미수령/재배송 대상으로 읽힌 사람.
```

비교 잠금:

| 비교축 | 윤서 | 한이루 / RETURN-05 |
|---|---|---|
| 첫 역할 | 현장 회수자 | 반송 수취인 |
| 내부 정체성 | 반품 담당자에 가까운 사람 | 아직 도착하지 못한 수취인 |
| 캠페인이 붙이는 역할 | 반품품, 회수 대상, 잘못 붙은 이름 | 수취인, 반송품, 미수령자, 재배송 대상 |
| 전투 출발점 | 위험한 현장에서 라벨/이름/분류를 읽고 보류한다 | 자신과 적의 배송 상태를 수취/반송/보관/재배송 사이에서 비튼다 |
| 대표 이미지 | 반품 스캐너, 반품 도장, 회수선 릴 | 송장, 포장끈, 반송 스티커, 배송 박스 |
| 정산 감각 | 회수물, 흔적 보존, 보급태그 | 반송 로그, 배송 실패표, 미수령 보관 기록 |
| HP 0 감각 | 마지막 좌표에서 회수선이 끌어온다 | 수취인 확인 실패로 미수령 보관에 들어가기 직전 역배송 좌표가 잡힌다 |

금지:

```text
이루에게 반품 도장, 반품 스캐너, 현장 회수자 말투를 주지 않는다.
이루가 사람을 데려오는 역할을 맡지 않는다.
이루가 윤서의 R03 과거를 먼저 해설하지 않는다.
```

차이 문장:

```text
윤서는 사람을 물건처럼 회수당하지 않게 현장에서 버틴다.
이루는 사람을 배송 상태값으로 완료하지 못하게 목적지와 수취 판정을 비튼다.
```

## 3. 중심 이미지 고정

중심 이미지는 네 가지로 고정한다.

```text
송장
포장끈
반송 스티커
배송 박스
```

보조 이미지:

```text
바코드 스캐너
```

세부 판정:

| 이미지 | 채택 | 역할 | 이유 |
|---|---|---|---|
| 송장 | 중심 | 주소, 수취인, 반송 경로, 배송 실패 사유를 한 장에 담는다 | 이름/주소/목적지가 충돌하는 후보의 핵심 오브젝트 |
| 포장끈 | 중심 | 묶기, 풀기, 경로 늦추기, 벨트에서 이탈시키기 | 회수선이 아니라 배송 상태를 묶고 늦추는 물리 이미지 |
| 반송 스티커 | 중심 | 완료가 아니라 되돌림으로 처리하는 캠페인 판정을 보여준다 | "돌아감"이라는 말의 위험을 직접 드러냄 |
| 배송 박스 | 중심 | 임시 보관, 엄폐, 오배송 지대, 짧은 우회 경로 | 사람을 박스로 만들지 않고 상태값 공간으로 쓴다 |
| 바코드 스캐너 | 보조 | 배송 실패표와 송장 판독 보조 | 윤서의 반품 스캐너와 겹치므로 중심 금지 |

무기/행동 원칙:

```text
훔치지 않는다.
쏘지 않는다.
스캔해서 처리하지 않는다.
배송 상태를 바꾼다.
```

허용 행동:

- 송장의 목적지 줄을 뒤집어 적의 추적 방향을 어긋나게 한다.
- 포장끈으로 분류 벨트의 판정 타이밍을 늦춘다.
- 반송 스티커로 적의 "완료" 상태를 "재확인" 상태로 되돌린다.
- 배송 박스를 펼쳐 짧은 미수령 지대와 임시 우회로를 만든다.
- 바코드 스캐너는 이미 붙은 라벨을 확인하는 보조 행동으로만 사용한다.

## 4. 전투 리듬

핵심 판정:

```text
회수가 아니라 배송 상태를 바꾸는 리듬.
```

전투 리듬:

| 리듬 | 의미 | 플레이 감각 |
|---|---|---|
| 수취 보류 | 누가 받을지 확정하지 않는다 | 적의 타겟 확정과 등록 완료를 늦춘다 |
| 반송 전환 | 앞으로 가던 절차를 되돌림 상태로 바꾼다 | 돌진, 추적, 포획 동선을 역방향으로 꺾는다 |
| 미수령 보관 | 완료되지 않은 상태를 짧게 유지한다 | 위험한 오브젝트나 적을 임시 보관 구역에 묶는다 |
| 재배송 대기 | 실패가 끝나지 않고 다음 시도로 밀린다 | 한 번의 회피가 아니라 다음 판정을 준비하는 템포를 만든다 |
| 주소 정정 | 목적지를 임시로 다시 쓴다 | 통행태그/수신태그 조건을 짧게 바꿔 우회한다 |

전투 리듬 결론:

```text
이루는 회수하지 않는다.
이루는 배송 완료와 반송 완료 사이의 상태를 늘리고,
그 틈에서 적과 자신이 어디로 처리되는지를 바꾼다.
```

금지:

- 적을 스캔해 약점을 띄우는 캐릭터.
- 박스를 열어 보상을 훔치는 도적.
- 빠른 배달, 빠른 픽업, 빠른 배송을 콘셉트로 삼는 택배원.
- 윤서의 반품 도장처럼 판정을 찍어 보류하는 연출.

## 5. 정산 루트

정산 루트는 아래 네 가지로 고정한다.

| 정산 루트 | 역할 | 사용 방식 | 위험 |
|---|---|---|---|
| 반송 로그 | 이 후보의 핵심 기록 | 반송, 역배송, 보관 좌표, 실패 사유를 추적한다 | 윤서의 회수물 로그처럼 보이지 않게 목적지 중심으로 둔다 |
| 배송 실패표 | 실패한 배송을 완료로 닫지 않게 만드는 증거 | 수취 실패, 주소 불일치, 임시 보관 사유를 남긴다 | 단순 보상표가 되면 약하다 |
| 미수령 보관 기록 | HP 0과 해금의 핵심 기록 | 보관함, 적재 구역, 임시 수취 지점을 표시한다 | 박스 안에 갇히는 개그로 소비하면 안 된다 |
| 수신태그/통행태그 | 보조 접근권 | 주소 호출, 게이트 통과, 임시 우회에 쓴다 | 수신 캐릭터/OPEN-HOST와 겹치지 않게 배송 수신과 물류 게이트로 제한 |

정산 감각:

```text
이루의 정산은 무엇을 가져왔는가가 아니다.
그것이 어디로 가도록 적혀 있었는가,
그리고 왜 도착하지 못했는가를 남기는 일이다.
```

정산 금지:

- 회수품/반품품 기록을 주 정산으로 삼지 않는다.
- 보급태그를 택배 보상처럼 뿌리지 않는다.
- 박스 개봉 보상 루프를 중심으로 만들지 않는다.
- 반송 로그를 윤서의 회수물 정산표처럼 쓰지 않는다.

## 6. HP 0 / 긴급 인양 차별화

핵심 판정:

```text
HP 0의 중심은 수취인 확인 실패다.
시각적 고정은 미수령 보관이다.
후유증은 재배송 대기다.
```

단계별 잠금:

| 단계 | 핵심 상태 | 연출/의미 |
|---|---|---|
| HP 0 직전 | 수취인 확인 실패 | 이름보다 주소와 배송 상태가 먼저 뜬다 |
| HP 0 | 미수령 보관 시작 | 임시 보관함, 적재 구역, 접힌 박스 라인이 닫히려 한다 |
| 긴급 인양 성공 | 역배송 좌표 가로채기 | 보급소 회수선이 반송 로그를 역배송 좌표로 잡아당긴다 |
| 복귀 후유증 | 재배송 대기 | 배송 실패표와 반송 로그 일부가 오염되거나 갱신된다 |
| 인양 실패 위험 | 반송 처리 확정 | 원래 자리로 돌아가는 것이 아니라 캠페인이 정한 보관 좌표로 밀린다 |

윤서/PATCH/하람과의 차이:

| 캐릭터 | HP 0 위기 | 긴급 인양 감각 |
|---|---|---|
| 윤서 | 마지막 좌표에서 회수 대상처럼 끌려올 위기 | 회수선이 사람/물건 판정을 보류하며 당긴다 |
| PATCH | 정품 복구나 부품 회수로 돌아갈 위기 | 보급소 수리선이 정품화 직전 끊어 낸다 |
| 강하람 / HOLD-04 | 퇴원 취소와 병상 재배정으로 다시 환자화될 위기 | 팔찌 절단 신호와 들것 회수선이 재입원 직전 당긴다 |
| 한이루 / RETURN-05 | 수취인 확인 실패와 미수령 보관으로 배송 상태화될 위기 | 반송 로그의 역배송 좌표를 보급소가 가로챈다 |

한 줄 정의:

```text
수취인 확인 실패로 미수령 보관이 닫히기 직전,
보급소가 반송 로그의 역배송 좌표를 가로채 긴급 인양한다.
```

## 7. R03 노출 범위 제한

R03 전체 물류권을 열지 않는다.

허용 범위:

| 노드 | 사용 여부 | 사용 목적 |
|---|---|---|
| R01-L01 MAIL-LOOP 단서 | 사용 | 주소가 사람보다 오래 남는 감각을 선노출한다 |
| R01-L02 택배함/오배송 흔적 | 제한 사용 | R03 예고를 주되 R01 NPC 기능을 침범하지 않는다 |
| R03-L01 반품 접수 야드 | 사용 | 수취/반송/미수령 판정의 첫 직접 조우 |
| R03-L02 자동 분류장 | 사용 | 송장, 포장끈, 반송 스티커, 벨트 판정 기믹 |
| R03-L03 보증 심사 창고 | 보류 | PATCH/로봇 부품 거래와 엮여 초반 축이 과밀해질 수 있다 |
| R03-L04 파쇄 전 대기라인 | 금지 | 시간 제한 회수와 후반 압력을 앞당긴다 |
| R03-L05 최종 반품 심사실 | 금지 | 윤서 핵심 보스와 정체성 검산을 앞당긴다 |

해금 구조:

1. R01 MAIL-LOOP, 택배함, 반송 알림지로 "주소가 남아 있다"는 단서를 먼저 보여준다.
2. R03-L01에서 수취인 확인 실패와 미수령 보관 절차를 직접 본다.
3. R03-L02에서 송장/포장끈/반송 스티커가 전투 리듬으로 쓰일 수 있음을 확인한다.
4. R03-L03 이후는 다음 상세화나 윤서 핵심 서사까지 보류한다.

노출 제한 문장:

```text
5번 후보는 R03 전체를 여는 열쇠가 아니라,
R03이 사람을 "도착하지 못한 배송 상태"로 보존한다는 첫 단서다.
```

## 8. MAIL-LOOP와의 연결 방식

연결 가능:

```text
주소
반송 알림지
수신태그 조각
배송 실패표
반송 로그
```

연결 방식:

| 연결 | 허용 | 이유 |
|---|---|---|
| R01에서 주소/반송 알림지를 선노출 | 허용 | MAIL-LOOP의 주소 잔향을 R03 배송 판정으로 확장한다 |
| MAIL-LOOP가 이루의 이름을 직접 부름 | 제한 | 이름 해금이나 강한 개인 관계로 만들면 하위 캐릭터처럼 보인다 |
| MAIL-LOOP의 반송 로그가 R03-L01 진입 단서가 됨 | 허용 | R01에서 R03으로 이어지는 작은 단서로 적절하다 |
| MAIL-LOOP를 이루의 전용 NPC로 사용 | 금지 | R01 핵심 보조 NPC 기능을 뺏는다 |
| MAIL-LOOP가 이루의 과거를 설명 | 금지 | 프로필 잔향형 NPC가 플레이어블 과거 해설자가 되면 안 된다 |

관계 정의:

```text
MAIL-LOOP는 주소가 남아 사람을 부르는 R01 잔향이다.
이루는 그 주소가 R03에서 수취/반송/미수령 판정으로 변하는 순간을 몸으로 겪은 사람이다.
둘은 같은 질문을 다른 층에서 보여준다.
```

금지:

```text
이루는 MAIL-LOOP의 주인이 아니다.
이루는 MAIL-LOOP가 찾던 수령인으로 고정하지 않는다.
이루는 MAIL-LOOP의 인간판이 아니다.
```
