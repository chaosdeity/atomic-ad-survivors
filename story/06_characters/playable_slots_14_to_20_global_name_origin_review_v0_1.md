## 문서 상태

```text

14~20번 글로벌 이름/원점 재검토 문서.
상세 결재 문서 아님.
프로필 v1.0 작성 아님.
20인 로스터 전체 재작성 아님.

대상:
```

## 0. 판정 요약

14~20번은 한국어권 임시명으로 계속 밀지 않는다. 기존 `신이준`, `박다미`, `오진솔`, `남태건`, `최미결`, `나유찬`은 모두 `legacy placeholder`로 내리고, 외부 광역권의 현실권/언어권 후보를 우선한다.

이번 재검토의 결론:

1. 14~19 한국어권 임시명은 최종명으로 사용하지 않는다.
2. 20번 OBS-20은 메타적인 `시즌 관측자` 방향을 폐기하고, E03_C01 스마트단지 입주/관제 절차 후보로 재정의한다.
3. 14~20은 모두 상세 결재 전 후보로 보존하되, 직접 프로필 v1.0 작성은 금지한다.
4. 상세 결재 우선순위는 기존 번호가 아니라 실제 캠페인 오픈 가능성 기준으로 잡는다.
5. MODEL-15만 `통과 가능 / E10_C05 개방 후 후보`로 보존하고, LOCKOUT-17과 NAME-18은 캠페인 미개방으로 대기한다.

## 4. 슬롯별 이름 판정

### 4.1 TICKET-14 / Arjun Mehta

legacy `TRANSIT-14 / 신이준`은 폐기한다. 새 후보는 E05_C07 철도탑승 통행권의 `탑승권 보류자`다.

고정할 차이:

- GATE-09는 막차 환승 심사와 플랫폼 잔류다.
- TICKET-14는 초고밀도 철도권에서 탑승권은 있지만 탑승 확정이 보류되는 사람이다.
- 빠른 이동 캐릭터가 아니라, 검표/탑승/미탑승/화물칸 대기 판정을 흔드는 후보여야 한다.

### 4.2 MODEL-15 / Maya Chen

legacy `REVIEW-15 / 박다미`는 폐기한다. 새 후보는 E10_C05 알고리즘 추천권의 `추천 모델 이탈자`다.

고정할 차이:

- BROAD-10은 공식 안내를 너무 선명하게 수신하는 사람이다.
- MODEL-15는 추천 모델이 자신을 학습 완료 상태로 고정하기 전에 입력 데이터를 굶기는 사람이다.
- 리뷰어, 인플루언서, 방송인, 해커가 아니다.

### 4.3 RATION-16 / Sari Wijaya

legacy `오진솔`은 폐기한다. 새 후보는 E04_C08 고지대 보급권의 `배급 명단 밖 조리자`다.

고정할 차이:

- QUIET-07은 낮은 수신과 약한 인양 리스크다.
- RATION-16은 고지대 보급 명단에 없지만 조리/급수 순서를 바꿔 남는 몫을 만든다.
- 미나의 보급소 정산/조리 기능을 빼앗지 않는다.

### 4.4 LOCKOUT-17 / Omar Al-Hadid

legacy `남태건`은 폐기한다. 새 후보는 E06_C04 에너지 정품충전권의 `안전잠금 미승인자`다.

고정할 차이:

- PATCH는 자기 몸을 부품처럼 써온 보급소 수리 로봇이다.
- METER-06은 계량당하는 인간이다.
- LOCKOUT-17은 안전 승인/작업자 ID/비상 정지줄이 잠겨 노동과 충전 접근이 동시에 막힌 사람이다.

### 4.5 NAME-18 / Leila Haddad

legacy `최미결`은 폐기한다. 새 후보는 E08_C07 피난검역 통행권의 `피난명 보류자`다.

고정할 차이:

- 윤서/복희/R01은 이름을 되찾는 친밀한 축이다.
- NAME-18은 피난검역 통행권에서 이름이 맞아야 이동할 수 있다는 행정 축이다.
- 이름 보존 감동담이 아니라, 이름 대조/검역명/통행명 보류 판정을 다룬다.

### 4.6 LINE-19 / Mateo Rojas

legacy `나유찬`은 폐기한다. 새 후보는 E11_C07 버스터미널 통행권의 `장거리 회수선 조정자`다.

고정할 차이:

- 윤서는 현장 회수자다.
- LINE-19는 장거리 터미널과 검문소 사이에서 회수선이 사람을 구하는지, 화물을 당기는지 구분하지 못하는 리스크를 다룬다.
- 인양 시스템 NPC의 기능을 빼앗지 않는다.

### 4.7 OBS-20 / Lin Yue

기존 `본명 보류 / 시즌 관측자` 방향은 폐기한다. 너무 메타적이고, 후반 관측자 캐릭터로 굳으면 캠페인 관점을 사람보다 설정 장치로 만들 위험이 크다.

새 OBS-20은 E03_C01 스마트단지 입주권의 `관측 누락 입주자`다.

고정할 차이:

- MODEL-15는 추천 알고리즘이 학습하는 사람이다.
- OBS-20은 스마트단지 관제에는 계속 찍히지만 입주자/세대원으로 승인되지 않는 사람이다.
- 관측자 영웅이 아니라, 관측되고도 등록되지 못한 후보여야 한다.
