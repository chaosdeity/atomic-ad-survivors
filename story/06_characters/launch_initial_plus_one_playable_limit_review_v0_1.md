# 출시 초기 플레이어블 +1 후보 제한 검토 v0.1

## 문서 상태

```text
상태:
작성 완료 v0.1

작업 성격:
새 프로필 작성 아님.
새 상세 결재 시작 아님.
후보 확정 문서 아님.
출시 초기 플레이어블 추가 상한과 검토 조건 잠금 문서.

핵심 목적:
"누가 +1인가"를 확정하지 않는다.
"+1을 검토해도 되는 조건"만 잠근다.

최종 방향:
출시 초기 +1은 필수가 아니라 최대치다.
윤서 단독 U5 시작은 허용된다.
PATCH는 조건부 가능성만 남긴다.
OPEN-HOST는 NPC 선등장 후 전환으로 둔다.
E01 한국인 인간 후보는 출시 초기 +1에서 보류한다.
```

입력 문서:

```text
story/06_characters/playable_next_allowed_work_review_v0_1.md
story/06_characters/playable_unopened_campaign_hold_review_v0_1.md
story/06_characters/unopened_campaign_playable_hold_gate_v0_1.md
story/06_characters/playable_20_profile_number_code_migration_decision_v0_1.md
docs/world/CHARACTER_UNLOCK_STRUCTURE_V0_1.md
```

## 0. 결론

출시 초기 플레이어블 +1은 의무가 아니다.

잠금:

1. 출시 초기 정식 U5는 윤서 단독으로 시작해도 된다.
2. +1 후보가 필요하더라도 최대 1명만 검토한다.
3. +1 후보의 기본 우선권은 한국인 인간 후보가 아니라 PATCH의 조건부 가능성 검토다.
4. OPEN-HOST는 R01 NPC 선등장 후 전환 후보이며, 출시 초기 U5로 당기지 않는다.
5. HOLD-04, RETURN-05, METER-06, QUIET-07은 프로필 산출물을 보존하되 출시 초기 해금으로 읽지 않는다.
6. 미개방 캠페인 후보는 +1 후보로 당기지 않는다.
7. U1~U4 신호/조우/거점 협력/동행만으로도 출시 초기 캐릭터 존재감은 충분히 줄 수 있다.

따라서 이 문서의 핵심 문장은 아래와 같다.

```text
출시 초기 +1은 슬롯이 아니라 상한선이다.
정식 U5 추가가 필요하다고 확인되기 전까지, 윤서 단독 시작과 U1~U4 보조 운영을 기본값으로 둔다.
```

## 1. 해금 단계 기준

`CHARACTER_UNLOCK_STRUCTURE_V0_1.md`의 U0~U5 기준을 따른다.

| 단계 | 이름 | 출시 초기 운용 |
|---|---|---|
| U1 | 신호 해금 | 허용. 흔적, 무전, 회수물로 존재만 암시한다. |
| U2 | 조우 | 허용. 얼굴, 목소리, 기능을 짧게 만난다. |
| U3 | 거점 협력 | 허용. 보급소 기능, 패널 코멘트, 해석 보조를 한다. |
| U4 | 동행/임무 | 조건부 허용. 특정 노드에서만 임시 조작 또는 동행한다. |
| U5 | 플레이어블 | 제한. 출시 초기에는 윤서 기본 U5와 최대 +1만 검토한다. |

중요:

```text
U1~U4는 정식 플레이어블 +1이 아니다.
유저에게 캐릭터 존재감을 줄 수 있지만, 캐릭터 선택 화면의 U5 해금으로 처리하지 않는다.
```

## 4. PATCH 조건부 가능 기준

PATCH는 출시 초기 +1 후보로 `조건부 가능`하다.
하지만 이것은 곧바로 2번 정식 해금이라는 뜻이 아니다.

PATCH가 +1 후보 검토를 통과하려면 아래를 모두 만족해야 한다.

1. 윤서 단독 시작이 플레이 감각상 부족하다는 검증이 먼저 있어야 한다.
2. PATCH가 전투 회복 담당, 장비 메뉴, 귀여운 로봇 마스코트로 읽히면 안 된다.
3. 수리와 정품 인증을 해주는 로봇이 아니라, 자기 몸을 부품처럼 써온 로봇이어야 한다.
4. R04 충전/계량 축을 METER-06보다 먼저 과하게 설명하면 안 된다.
5. 리오의 객실 초기화/정품 복구 공포를 앞당기면 안 된다.
6. U1~U4 게스트/협력 운영으로 충분하다면 U5 +1로 올리지 않는다.

권장 기본값:

```text
PATCH는 출시 초기 정식 U5 확정보다 U3~U4 우선.
전투 다양성이 반드시 필요하다는 검증이 있을 때만 U5 +1을 재검토한다.
```

## 5. OPEN-HOST 전환 기준

OPEN-HOST는 출시 초기 +1 후보가 아니라 `NPC 선등장 후 전환` 후보로 둔다.

전환 검토 조건:

1. R01 안내/오픈하우스/NPC 기능이 먼저 충분히 작동해야 한다.
2. OPEN-HOST가 플레이어블로 전환되어도 R01 NPC 기능이 비면 안 된다.
3. 서이겸 본명이 한국인 인간 +1 슬롯처럼 소비되면 안 된다.
4. R01 내부 동선, 평면도, 가족 구성 중개 기능을 플레이어블이 독점하면 안 된다.
5. 전환은 U2~U3 선등장 뒤 U4 또는 후속 U5로 검토한다.

금지:

```text
출시 초기부터 OPEN-HOST를 캐릭터 선택 화면에 넣는다.
R01의 안내 NPC 역할을 OPEN-HOST 플레이어블 전환으로 비운다.
한국인 인간 후보가 필요하다는 이유로 OPEN-HOST를 +1로 당긴다.
```

## 6. E01 인간 후보 보류 기준

HOLD-04, RETURN-05, METER-06, QUIET-07은 폐기하지 않는다.
이미 작성된 프로필 산출물은 보존한다.

다만 출시 초기 +1 후보로 쓰지 않는다.

이유:

1. 윤서가 이미 한국인 기본 주인공이다.
2. OPEN-HOST도 R01 한국어권 NPC 선등장 후보로 남아 있다.
3. HOLD/RETURN/METER/QUIET까지 당기면 E01 한국어권 인간 후보가 과밀해진다.
4. 각 후보는 R02/R03/R04/R08 상태 반응과 NPC, 오브젝트 흔적이 먼저 깔린 뒤 강해진다.
5. 출시 초기의 보급소는 작고 취약해야 하며, 사람을 너무 빨리 많이 확보하면 세계 압력이 약해진다.

보류는 삭제가 아니다.
프로필 산출 번호와 최종 해금 순번을 분리하는 조치다.
