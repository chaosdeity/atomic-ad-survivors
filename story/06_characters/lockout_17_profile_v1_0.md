## 0. 캐릭터 핵심

오마르는 전기 마법사, 엔지니어, 수리공, 충전 버퍼, 안전요원, 탱커, 보디가드가 아니다.

오마르는 E06_C04 에너지 정품충전권에서 안전잠금과 작업자 ID가 자신을 `위험구역 잔류자`와 `잠금태그 책임자`로 고정하기 전, 작업 허가와 충전 접근 차단 판정을 늦추는 17번 플레이어블 후보다.

핵심 질문:

```text
안전을 위해 잠근 문은
누구를 보호하고,
누구를 작업자로 고정하는가?
```

핵심 문장:

```text
잠겼다면 안전한 줄 알았습니다.
그 안에 제 이름이 갇히기 전까지는요.
```

잘못 믿는 것:

```text
오마르는 작업자 ID가 발급되면,
그 ID가 자신을 보호한다고 믿는다.
```

이 믿음은 일부 맞다. 위험한 설비에서 안전 절차와 작업자 ID는 사람을 보호한다. 그러나 E06_C04는 그 ID를 보호 장치이자 소유 장치로 동시에 사용한다. ID가 닫히면 오마르는 아직 판단 중인 사람이 아니라 잠금태그 책임자와 위험구역 잔류자로 고정된다.

플레이어블로서 다루는 질문:

```text
안전절차가 사람을 책임 항목으로 닫기 전,
아직 미승인으로 남은 신호를 보급소가 읽게 할 수 있는가?
```

오마르의 조작은 안전절차를 부수는 방식이 아니다. 오마르의 조작은 잠금과 미승인 사이에 비상정지줄, 잠금태그의 접힌 부분, 작업 허가 보류표의 빈칸을 남기는 방식이다.

## 2. 유저용 첫 소개문

오마르, 산업 안전 미승인자.

오마르는 전기를 쏘는 사람이 아니다. 그는 잠긴 게이트와 작업 허가표 사이에서, 안전하다는 말이 언제 사람을 책임 항목으로 바꾸는지 배운 사람이다.

작업자 ID는 출입을 열어 주기도 했고, 위험구역 안에 이름을 묶어 두기도 했다. 잠금태그는 사고를 막기 위한 표식이었지만, 어느 순간부터 오마르가 아직 나가지 못했다는 증거가 되었다.

보급소가 잡은 것은 완전한 승인 기록이 아니라, 비상정지줄과 작업 허가 보류표가 서로 맞지 않는 작은 불일치다. 오마르가 합류하려면 안전절차를 무시하는 것이 아니라, 절차가 사람을 닫기 전 남은 미승인 신호를 지켜야 한다.

## 4. 플레이어블 기능

| 기능 | 의미 | 실패 위험 |
|---|---|---|
| 잠금태그 보류 | 장치/게이트의 활성 판정을 늦춘다 | 오마르가 잠금태그 책임자로 고정된다 |
| 비상정지줄 당김 | 특정 웨이브/장치의 행동을 끊는다 | 작업자 ID 호출이 빨라진다 |
| 작업 허가 미승인 | 위험구역 접근을 잠깐 열어 둔다 | 미승인 상태도 대기 중인 작업자로 누적된다 |
| 충전 접근 차단 역이용 | 정품 충전 절차를 적의 경로 차단으로 돌린다 | METER-06 계량/차단 축과 섞일 수 있다 |
| 에너지 게이트 지연 | 전장 이동과 장치 작동 순서를 한 박자 늦춘다 | E06 전체 스마트시티 안내자로 커질 수 있다 |
| 작업자 ID 조각 분리 | ID가 완성되기 전 이름과 책임 항목을 떼어 둔다 | 조각이 붙으면 작업자 ID가 더 강하게 닫힌다 |

오마르의 기능은 전력을 많이 얻는 것이 아니다. 안전절차가 사람을 작업자 ID로 닫는 순간을 늦춰 보급소가 아직 미승인 신호를 읽을 수 있게 하는 것이다.

파티/동행 시 역할:
PATCH가 고쳐주는 것과 부품이 되는 것을 분리하고, 유건이 충전받는 것과 사용량이 되는 것을 분리하며, 리오가 정리하는 것과 지우는 것을 분리한다면, 오마르는 안전하게 잠그는 것과 사람을 책임 항목으로 가두는 것을 분리한다.

## 7. HP 0 / 긴급 인양 차별화

핵심 상태명:

```text
작업자 ID 잠금 / 안전교육 미승인 확정 / 충전 접근 차단
```

HP 0 흐름:

| 단계 | 상태 | 처리 |
|---:|---|---|
| 1 | 안전교육 미승인 확정 | 미승인 상태가 대기 중이 아니라 고정 상태로 닫힌다 |
| 2 | 에너지 게이트 격리 | 게이트가 열리는 것이 아니라 오마르를 위험구역 안에 남긴다 |
| 3 | 충전 접근 차단 | 정품 충전 접근이 차단되고 접근권이 작업자 ID에 묶인다 |
| 4 | 작업자 ID 잠금 | 오마르의 이름 대신 작업자 ID와 잠금태그 번호가 표시된다 |
| 5 | 긴급 인양 | 보급소가 비상정지줄과 작업 허가 보류표의 불일치를 잡아 되당긴다 |

비교:

| 대상 | HP 0 감각 | LOCKOUT-17과 다른 점 |
|---|---|---|
| PATCH | 부품 목록화, 정품 부품 회수 | 오마르는 로봇 부품이 아니라 작업자 ID와 안전잠금으로 고정된다 |
| 유건 / METER-06 | 과방전, 계량 초과, 정품 도크 회수 직전 | 유건은 사용량으로 닫힌다. 오마르는 작업 허가와 잠금태그 책임으로 닫힌다 |

## 13. 성장 방향

초기:
오마르는 작업자 ID가 발급되면 자신을 보호한다고 믿는다. 승인 절차가 느리고 답답하더라도, 닫힌 문과 잠금태그가 위험을 줄여 준다고 배웠다.

중반:
오마르는 잠금이 사람을 지키기도 하지만 사람을 남겨 두기도 한다는 것을 배운다. 미승인 상태는 자유가 아니고, 승인 상태도 완전한 안전이 아니다. 둘 사이에 남은 빈칸만이 보급소가 읽을 수 있는 신호가 된다.

후반:
오마르는 안전절차를 부수거나 조롱하지 않는다. 대신 안전이라는 말이 언제 책임 전가와 소유로 바뀌는지 구분한다. 잠금태그는 사고를 막는 표식이면서, 아직 안에 사람이 남았다는 표식이 될 수 있다.

최종 변화 문장:

```text
잠그는 일은 사람을 지킬 수도,
사람을 남겨 둘 수도 있었습니다.
```

완료 요약:

4. 직접 해금은 E06_C04 개방 뒤 다시 판정하고 E06 전체 확장은 금지한다.
5. 다음은 미개방 캠페인 플레이어블 보류 게이트로 넘긴다.
