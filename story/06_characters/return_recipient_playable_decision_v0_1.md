## 0. 판정 요약

판정:

```text
조건부 통과
```

한이루 / RETURN-05 / 반송 수취인은 5번 슬롯에 둘 가치가 있다. R03 물류/반품 캠페인권을 "택배 지역"이 아니라 사람을 수취인, 미수령자, 재배송 대상, 반송품으로 읽는 절차로 보여줄 수 있다.

다만 바로 v1.0 프로필로 가지 않는다. 아래 조건이 먼저 잠겨야 한다.

1. 윤서의 반품 도장/반품 스캐너/현장 회수자 정체성을 반복하지 않는다.
2. 단순 택배원, 도적, 스캐너 캐릭터가 되지 않는다.
3. R03 전체를 초반에 열지 않는다. R03-L01~L02와 R01 MAIL-LOOP 단서까지만 쓴다.
4. 중심 전투 이미지는 바코드 스캐너가 아니라 송장, 포장끈, 반송 스티커, 배송 박스의 방향 변경이어야 한다.
5. HP 0은 윤서처럼 마지막 좌표에서 회수되는 것이 아니라 수취인 확인 실패와 미수령 보관으로 차별화한다.

핵심 질문:

```text
사람은 물건처럼 반송될 수 있는가?
반송된 사람은 어디로 돌아가는가?
```

## 1. 캐릭터 핵심

| 항목 | 결재안 |
|---|---|
| 후보 슬롯 | 5번 |
| 본명 | 한이루 |
| 코드명 | RETURN-05 |
| 유저 첫 표기 | 이루, 반송 수취인 |
| 출신 축 | R03 물류/반품/수취/배송 캠페인권 |
| 캠페인이 등록하려는 역할 | 수취인, 반송품, 미수령자, 재배송 대상 |
| 플레이어블 검토 목적 | R03을 윤서의 전용 과거 지역으로만 두지 않고, "주소와 수취 판정"의 절차로 먼저 이해시킨다 |
| 최종 프로필 여부 | 아직 아님. 말투, 외형, 세부 관계선은 확정하지 않는다 |

이 후보는 택배 기사나 물류센터 직원이 아니다. 더 정확히는 캠페인이 계속 "수취인 확인 실패"로 되돌리는 사람이다.

캠페인은 한이루를 죽이려 하지 않는다. 캠페인은 배송 완료를 원한다. 문제는 한이루가 사람으로 도착하는 것이 아니라, 배송 상태값으로 완료되어야 한다는 점이다.

이 후보의 생존 방식은 물건을 훔치거나 배송을 빠르게 처리하는 것이 아니다.

```text
배송 주소와 반송 라벨의 방향을 바꾸어,
캠페인이 사람을 한 목적지로 확정하는 순간을 빗나가게 한다.
```

## 2. 어떻게 붙잡혔는가

캠페인이 붙잡는 방식:

| 절차 | 붙잡는 방식 | 후보가 느끼는 압력 |
|---|---|---|
| 수취인 확인 | 이름보다 주소와 수령 가능 여부를 먼저 확인한다 | "내가 맞는 사람이 아니라, 이 주소에 맞는 물건인가" |
| 미수령 보관 | 사람이 응답하지 않으면 보관함이나 임시 적재장으로 보낸다 | "대답하지 못하면 보관되는가" |
| 반송 처리 | 배송 실패가 원래 자리로 돌아가는 일처럼 포장된다 | "돌아간다는 말은 누구의 원래 자리인가" |
| 재배송 대기 | 실패가 끝이 아니라 다음 배송 시도로 갱신된다 | "실패도 완료되지 않고 반복된다" |
| 주소 정정 | 사람의 기억보다 시스템 주소가 더 강해진다 | "내가 가야 할 곳을 내가 정할 수 있는가" |

아직 완전히 흡수되지 않은 이유:

- 수취인 확인이 끝나지 않았다.
- 반송 주소가 원래 주소인지, 임시 보관 주소인지, 캠페인이 새로 붙인 주소인지 충돌한다.
- R01의 우편함/주소 잔향과 R03의 물류 로그가 서로 다른 수령인을 부른다.
- 한이루는 이 충돌을 약점이 아니라 방향을 바꾸는 틈으로 쓰기 시작했다.

따라서 이 후보는 "배송을 잘 아는 사람"이 아니다.

```text
배송 완료와 반송 완료 사이에서,
도착하지 않은 상태를 전투 리듬으로 쓰는 사람이다.
```

## 3. 잘못 믿는 것

초안:

```text
반송되면 원래 자리로 돌아간다고 믿음.
```

결재안:

```text
주소가 남아 있으면 돌아갈 자리도 남아 있다고 믿는다.
```

이 문장이 더 낫다. "반송되면 원래 자리로 돌아간다"는 말은 물류 절차를 잘 보여주지만, 캐릭터의 상처를 조금 좁게 만든다. "주소가 남아 있다"는 믿음은 R01의 MAIL-LOOP, R03의 수취인 확인, 윤서의 이름/문패 감각과 모두 연결된다.

더 깊은 오류:

- 주소는 장소가 아니라 캠페인이 사람을 다시 배정하는 좌표일 수 있다는 사실을 늦게 받아들인다.
- 반송은 귀환이 아니라 다른 보관 상태로 밀리는 일일 수 있음을 인정하지 못한다.
- "미수령" 상태가 자신을 보존해 주었기 때문에, 완전한 도착을 두려워하면서도 주소를 버리지 못한다.

통과 조건:

```text
한이루는 돌아갈 곳을 찾는 캐릭터가 아니다.
돌아간다는 절차가 사람을 어디로 보내는지 끝까지 의심하게 만드는 캐릭터여야 한다.
```

## 4. 윤서와의 차이

| 비교축 | 윤서 | 한이루 / RETURN-05 |
|---|---|---|
| 기본 질문 | 사람을 반품품/회수품으로 확정하지 않을 수 있는가 | 반송된 사람은 정말 원래 자리로 돌아가는가 |
| 첫 역할 | 현장 회수자 | 반송 수취인 |
| 캠페인이 붙이는 역할 | 반품품, 회수 대상, 잘못 붙은 이름 | 수취인, 미수령자, 반송품, 재배송 대상 |
| 전투 리듬 | 라벨을 읽고 보류 표시를 찍어 회수한다 | 송장 방향과 포장끈을 바꿔 배송/반송 경로를 어긋나게 한다 |
| 대표 이미지 | 반품 스캐너, 반품 도장, 회수선 릴 | 송장, 포장끈, 반송 스티커, 배송 박스 |
| 정산 감각 | 회수물, 흔적 보존, 보급태그 | 수신태그, 통행태그, 반송 로그, 배송 실패표 |
| HP 0 감각 | 마지막 좌표에서 회수선이 끌어온다 | 수취인 확인 실패로 미수령 보관에 들어가기 직전 역배송 로그가 잡힌다 |

윤서는 밖에서 사람을 데려오고, 잘못 붙은 분류를 보류한다. 한이루는 사람을 어디로 보내는지 정하는 주소와 배송 방향을 비튼다.

윤서가 묻는다면:

```text
이 사람을 지금 데려오면 버틸 수 있어?
```

한이루가 묻는다면:

```text
돌아간다는 말이 어디로 가라는 뜻이야?
```

금지:

```text
한이루에게 반품 도장, 반품 스캐너, 현장 회수자 말투를 주지 않는다.
```

## 5. PATCH와의 차이

| 비교축 | PATCH | 한이루 / RETURN-05 |
|---|---|---|
| 핵심 질문 | 고쳐주는 것과 부품이 되는 것은 다르다 | 반송은 귀환인가, 다른 보관인가 |
| 존재 유형 | 보급소 수리 로봇 | R03 물류/반품 캠페인권 인간 후보 |
| 캠페인 압력 | 정품 부품, 리콜 대상, 수리 자재 | 수취인, 미수령자, 재배송 대상 |
| 행동 감각 | 결합부를 막고 임시 수리를 만든다 | 배송 방향과 수취 판정의 조건을 바꾼다 |
| 정산 | 충전태그, 로봇 자율성, 장비 안정 | 수신태그, 통행태그, 반송 로그 |
| HP 0 | 부품 목록화 직전 회수 | 미수령 보관/재배송 대기 직전 인양 |

PATCH가 부품으로 분류되는 위험을 다룬다면, 한이루는 사람의 목적지가 시스템에 의해 덮어쓰이는 위험을 다룬다.

중요:

```text
한이루는 PATCH의 물류 버전이 아니다.
수리나 부품 회수가 아니라, 도착/미도착/반송 판정의 방향을 다룬다.
```

## 6. 강하람 / HOLD-04와의 차이

| 비교축 | 강하람 / HOLD-04 | 한이루 / RETURN-05 |
|---|---|---|
| 출신 축 | R02 의료/보험/퇴원 심사 | R03 물류/반품/수취/배송 |
| 캠페인이 붙이는 역할 | 환자, 보험 심사 대기자, 퇴원 불가 대상 | 수취인, 미수령자, 반송품, 재배송 대상 |
| 잘못 믿는 것 | 퇴원 승인 없이 나가면 살아남은 것이 아니라고 믿는다 | 주소가 남아 있으면 돌아갈 자리도 남아 있다고 믿는다 |
| 전투 리듬 | 퇴원/재입원/심사 보류 | 수취/미수령/반송/재배송 대기 |
| 대표 이미지 | 퇴원 팔찌, 보험 도장, 병상 커튼 | 송장, 포장끈, 반송 스티커, 배송 박스 |
| HP 0 | 퇴원 취소, 병상 재배정, 보험 심사 재개 | 수취인 확인 실패, 미수령 보관, 재배송 대기 |

하람은 "나가도 된다는 승인"의 문제다. 이루는 "돌아가야 한다는 주소"의 문제다.

하람이 묻는다면:

```text
누가 나가도 된다고 승인해야 해?
```

이루가 묻는다면:

```text
누가 돌아갈 주소를 정했어?
```

## 7. OPEN-HOST와의 차이

| 비교축 | OPEN-HOST | 한이루 / RETURN-05 |
|---|---|---|
| 출신 감각 | R01 내부 안내, 주거/가족 절차 | R03 물류/반품/배송 절차 |
| 잘못 믿은 것 | 덜 나쁜 길을 안내하면 사람들이 덜 다친다고 믿었다 | 주소가 남아 있으면 돌아갈 자리도 남아 있다고 믿는다 |
| 플레이 리듬 | 문, 동선, 평면도, 안내 리본 | 송장, 포장끈, 라벨 방향, 박스 접힘 |
| 절차 비틀기 | 들어가는 길을 나가는 길로 바꾼다 | 보내는 길을 보류 경로와 역배송 경로로 바꾼다 |
| NPC 기능 위험 | R01 NPC 기능을 비울 수 있다 | R01 MAIL-LOOP의 주소 기능을 흡수할 수 있다 |

OPEN-HOST는 집 안의 동선과 안내를 다룬다. 한이루는 주소와 배송 방향을 다룬다.

둘은 "길"을 다룬다는 점에서 닮았지만, OPEN-HOST는 사람이 공간을 통과하는 절차이고 한이루는 사람이 배송 상태값으로 처리되는 절차다.

통과 조건:

```text
한이루는 길 안내인이 아니다.
배송 방향을 바꾸더라도 방/문/평면도 조작은 OPEN-HOST 영역으로 남긴다.
```

## 8. 리오와의 장기 대비

리오와의 대비는 장기적으로 중요하다. 둘 다 사람이 아닌 시스템의 완료 처리와 관련 있지만, 감각은 다르다.

| 비교축 | 한이루 / RETURN-05 | 리오 |
|---|---|---|
| 슬롯 | 5번 후보 | 8번 보류 잠금 |
| 존재 유형 | 인간 수취/반송 후보 | 탈정품 객실 정리 로봇 |
| 붙잡는 방식 | 배송 완료, 반송 처리, 재배송 대기 | 정품 복구, 객실 초기화, 청소 로그 |
| 두려운 상태 | 돌아갈 곳이 주소로만 남는 상태 | 비어 있지 않은 방을 비었다고 처리하는 상태 |
| 세계 확장 | R03 물류/반품 초반 단서 | E08 체류/숙박 중반 해석 |

공통점:

- 캠페인은 죽이려 하지 않는다.
- 캠페인은 완료 상태를 만들려 한다.
- 완료 상태가 당사자에게는 사라지는 일처럼 느껴진다.

차이:

```text
한이루는 아직 도착하지 못한 사람이다.
리오는 이미 정리된 방을 다시 비우라는 압력을 받는 로봇이다.
```

## 9. 전투/무기 이미지

중심 이미지:

```text
송장
포장끈
반송 스티커
배송 박스
```

보조 이미지:

```text
컨베이어 벨트
바코드 스캐너
```

판정:

| 이미지 | 판정 | 이유 |
|---|---|---|
| 송장 | 중심 | 주소, 수취인, 반송 경로가 한 장에 모이는 핵심 이미지 |
| 포장끈 | 중심 | 묶고, 늦추고, 방향을 바꾸는 물리적 행동이 가능 |
| 반송 스티커 | 중심 | 완료가 아니라 되돌림으로 보이는 위험을 직접 표현 |
| 배송 박스 | 중심 | 엄폐/밀기/임시 보관/오배송 경로를 만들 수 있음 |
| 컨베이어 벨트 | 보조 | 환경 기믹으로 강하지만 캐릭터 개인 무기로 삼으면 R03 전체가 너무 커짐 |
| 바코드 스캐너 | 보조/주의 | 윤서의 반품 스캐너와 겹치므로 중심 무기 금지 |

전투 리듬:

```text
치거나 훔치는 캐릭터가 아니다.
송장 방향을 바꾸고, 포장끈으로 분류 타이밍을 늦추고,
반송 스티커로 적/오브젝트의 목적지를 임시 보류하며,
배송 박스로 짧은 우회 경로와 미수령 지대를 만든다.
```

금지:

- 택배 기사 액션.
- 도적/루터 플레이.
- 스캐너로 적을 찍어 처리하는 플레이.
- 윤서의 도장/스캐너/회수선 리듬 반복.

## 10. 정산 루트

한이루의 정산은 물건을 많이 줍는 루트가 아니다.

정산 후보:

| 정산 루트 | 용도 | 위험 |
|---|---|---|
| 수신태그 | 주소/수취 확인, R01 MAIL-LOOP와의 약한 연결 | 방송/정보 캐릭터와 겹치지 않게 배송 수신으로 제한 |
| 통행태그 | 배송 경로, 게이트 통과, 임시 우회 | OPEN-HOST의 동선 변경과 겹치지 않게 물류 게이트 중심 |
| 반송 로그 | 반송/미수령/재배송 상태 확인 | 윤서의 회수물 로그와 혼동 주의 |
| 배송 실패표 | 실패한 배송을 완료 처리하지 않게 보류 | 단순 보상표가 되면 약함 |
| 회수품/반품품 기록 | R03 물류권의 잔여 물자와 사람 분류를 읽음 | 윤서의 회수물 정산과 직접 충돌 주의 |

핵심 정산 감각:

```text
이루는 무엇을 가져왔는가보다,
그것이 어디로 가도록 적혀 있었는가를 정산한다.
```

## 11. HP 0 / 긴급 인양 차이

한이루의 HP 0은 사망이나 부활이 아니다.

핵심 판정:

```text
수취인 확인 실패
```

연쇄 상태:

| 단계 | 화면/서사 감각 | 의미 |
|---|---|---|
| 1 | 수취인 확인 실패 | 캠페인이 한이루를 사람보다 배송 상태로 읽기 시작한다 |
| 2 | 미수령 보관 | 임시 보관함/적재 구역으로 이동될 위험 |
| 3 | 재배송 대기 | 실패가 끝나지 않고 다음 배송으로 갱신된다 |
| 4 | 반송 처리 | "돌아감"처럼 보이지만 실제로는 시스템이 정한 보관 좌표로 밀린다 |

긴급 인양 한 줄:

```text
수취인 확인 실패로 미수령 보관에 들어가기 직전,
반송 로그가 보급소 회수선에 역배송 좌표로 잡힌다.
```

윤서와의 차이:

| 윤서 | 한이루 |
|---|---|
| 마지막 현장 좌표에서 회수선이 당긴다 | 반송 로그와 미수령 좌표를 보급소가 가로챈다 |
| 회수물 일부 유실 | 배송 실패표/반송 로그 일부 오염 |
| 사람을 물건처럼 회수당할 위험 | 사람을 미수령/재배송 상태로 보관당할 위험 |

통과 조건:

```text
HP 0 연출은 "배송 박스에 담겨 돌아옴"으로 끝내지 않는다.
미수령 보관, 수취인 확인 실패, 재배송 대기 로그가 핵심이어야 한다.
```

## 12. 해금 조건

권장 해금:

```text
R01 MAIL-LOOP / 반송 알림지 / 주소 불안 단서 선노출
이후 R03-L01 반품 접수 야드 또는 R03-L02 자동 분류장에서 직접 조우
```

허용 범위:

| 범위 | 판정 | 이유 |
|---|---|---|
| R01-L01 MAIL-LOOP 단서 | 허용 | 주소가 사람보다 오래 남는 감각을 먼저 깔 수 있음 |
| R01-L02 택배함/오배송 단서 | 허용 | R01 기능을 침범하지 않는 선에서 R03 예고 가능 |
| R03-L01 반품 접수 야드 | 허용 | R03 첫 진입 노드. 윤서 정체성 진입과 연결 가능 |
| R03-L02 자동 분류장 | 조건부 허용 | 라벨/벨트/정산 기믹을 보여주되 보스 축은 열지 않음 |
| R03-L03 보증 심사 창고 | 보류 | PATCH/로봇 부품 거래와 얽혀 초반 축이 과밀해질 수 있음 |
| R03-L04 파쇄 전 대기라인 | 금지 | 너무 어둡고 R03 후반 압력을 앞당김 |
| R03-L05 최종 반품 심사실 | 금지 | 윤서 핵심 보스와 정체성 검산을 앞당김 |

MAIL-LOOP와의 연결:

```text
MAIL-LOOP는 주소가 남아 사람을 부르는 R01 잔향이다.
한이루는 그 주소가 R03에서 어떤 배송/반송 절차로 변하는지 보여주는 후보다.
둘은 연결되지만, 한이루는 MAIL-LOOP의 하위 캐릭터가 아니다.
```

## 14. 최종 판정과 통과 조건

최종 판정:

```text
조건부 통과
```

통과 이유:

1. 5번 슬롯에서 R03 물류/반품 캠페인을 여는 기능이 강하다.
2. 윤서와 닮은 축을 오히려 비교 대상으로 삼아 "회수"와 "반송"의 차이를 선명하게 만들 수 있다.
3. R01 MAIL-LOOP의 주소 불안을 R03의 수취/배송 절차로 확장할 수 있다.
4. PATCH, 강하람, OPEN-HOST, 리오와 전투 리듬과 정산 감각이 다르다.

조건:

```text
1. 중심 무기는 송장/포장끈/반송 스티커/배송 박스로 고정한다.
2. 바코드 스캐너는 보조 이미지로만 둔다.
3. 잘못 믿는 것은 "주소가 남아 있으면 돌아갈 자리도 남아 있다"로 고정한다.
4. HP 0 핵심은 수취인 확인 실패와 미수령 보관으로 둔다.
5. 해금 범위는 R01 MAIL-LOOP 단서, R03-L01, R03-L02로 제한한다.
6. 윤서의 반품 도장/반품 스캐너/현장 회수자 리듬을 쓰지 않는다.
```
