# 플레이어블 20인 캠페인 오픈 순번 재배치 v0.1

## 문서 상태

- 문서 성격: 긴급 교정 / 기존 해금 순번 무효화 / 재배치 기준 잠금
- 현재 판정: `기존 1~20 번호를 최종 플레이어블 해금 순번으로 사용 금지`
- 적용 범위: 플레이어블 20인 로스터, 이름 레지스트리, 후보 선정, 프로필 작성 프롬프트, 캐릭터/NPC 생성 프롬프트
- 재배치 결과: `story/06_characters/playable_20_campaign_open_order_rebuild_result_v0_1.md`

## 0. 교정 결론

기존 작업은 `프로필 작성 번호 = 플레이어블 해금 순번`처럼 굳어졌기 때문에 교정한다.

세계 구조는 단일 캠페인 진행이 아니라 `12개 광역권 x 8개 캠페인 = 96개 캠페인`의 장기 오픈 구조다. 따라서 플레이어블은 캐릭터 콘셉트가 먼저 좋다고 앞 번호에 배치하는 것이 아니라, 해당 캐릭터의 원점 캠페인이 어느 정도 열린 뒤에 해금 순번을 받는다.

RIO-08 사례가 기준이다. 리오는 로봇 계열이라 초반 후보처럼 보이더라도, 원점 캠페인이 늦게 열리기 때문에 2번이 아니라 뒤로 밀렸다. 이 원칙을 전체 로스터에 적용한다.

## 1. 즉시 중단선

- `14번 신이준 / TRANSIT-14` 이후로 번호를 계속 채우는 작업은 중단한다.
- 14~20번을 한국어 이름 중심으로 임시 확정하는 작업도 중단한다.
- 새 프로필 v1.0 작성은 순번 재배치 완료 전까지 중단한다.
- 기존 프로필 v1.0 문서는 삭제하지 않지만, 해당 번호는 최종 해금 순번이 아니라 `기존 프로필 산출 번호`로만 취급한다.
- 각 캐릭터의 코드명에 포함된 숫자는 재배치 완료 전까지 `구 번호 / 산출물 번호`이며, 최종 해금 번호로 재사용할지 별도 판정한다.

## 2. 재배치 원칙

1. 캠페인이 먼저 열린다.
2. 해당 캠페인의 지역 노드, NPC, 오브젝트, 상태 반응, 작은 결말이 충분히 깔린 뒤 플레이어블 해금 후보가 된다.
3. 같은 캠페인에서 나온 플레이어블은 캠페인이 어느 정도 진행된 다음에만 추가한다.
4. `E##` 코드는 지역/광역권 코드이지 출시 순번이 아니다.
5. U5 플레이어블은 U1~U4의 정보, 잔향, NPC, 사건 반응을 선행 조건으로 가진다.
6. 국가/언어권 분포는 세계 캠페인 분포를 따라야 하며, 초중반 슬롯이 특정 국가권에 몰리면 재배치한다.

## 5. 새 번호 부여 규칙

새 번호는 다음 조건을 모두 통과한 후보에게만 준다.

- 원점 캠페인 코드가 명확하다.
- 해당 캠페인이 출시/서사 진행상 어느 시점에 열리는지 명확하다.
- 같은 캠페인에서 이미 해금된 플레이어블과 기능이 겹치지 않는다.
- NPC가 맡아야 할 기능을 플레이어블이 가져가지 않는다.
- HP 0 판정이 전투 사망이 아니라 해당 캠페인의 제도적 판정으로 고정된다.
- 국적/언어권/문화권이 세계 분포를 망가뜨리지 않는다.

## 7. 금지

- `이전 포인터가 14번을 가리킨다`는 이유로 14번 프로필을 작성하지 않는다.
- 같은 캠페인이 아직 충분히 열리지 않았는데 같은 캠페인 플레이어블을 추가하지 않는다.
- 플레이어블 20인을 단일 지역 캠페인의 동료 목록처럼 만들지 않는다.
- 20인 전원을 확정 이름으로 채우지 않는다.
- 로봇/의사/방송/상인 같은 장르 직업으로 순서를 앞당기지 않는다.
