# Yunseo 0.2 Playable Spec

상태: 0.2 플레이어블 기준  
연결 문서: `yunseo.md`, `player_characters.md`, `character_affection_monetization.md`, `META_PROGRESSION_ARCHITECTURE.md`

## 목적

이 문서는 0.2에서 윤서를 실제 플레이어블 캐릭터로 구현할 때 필요한 전투, 성장, 대사, 애정 포인트를 고정한다.

기준은 하나다.

```text
유저가 윤서를 좋아할 것인가?
```

윤서는 기본 캐릭터다. 하지만 기본 캐릭터가 밋밋하면 게임 전체의 첫인상이 무너진다. 0.2에서 유저가 윤서를 좋아하지 않으면, 이후 캐릭터 수익 구조도 약해진다.

## 한 줄 정체성

```text
윤서는 캠페인이 끝내 상품 설명으로 완성하지 못한 인간이다.
```

유저에게 직접 설명할 필요는 없다. 유저가 봐야 하는 것은 다음이다.

- 반품 스캐너로 광고화된 적을 벗겨낸다.
- 반품 도장으로 전방을 시원하게 찍어낸다.
- 첫 회수 때 자기 이름이 불릴 뻔한다.
- 보급소로 돌아오면 사람들이 윤서의 이름을 확인한다.
- 짧은 대사가 건조하게 웃기다.

## 0.2 플레이 판타지

유저가 윤서를 선택했을 때 기대해야 하는 판타지:

```text
광고가 덮어씌운 세계에 반품 도장을 찍고,
자기 이름을 빼앗기기 전에 돌아오는 회수자.
```

전투 판타지:

- 가까운 적을 안정적으로 정리한다.
- 차징 순간에 전방을 확실히 밀어낸다.
- 광고 오브젝트와 보스 외피를 찢는 느낌이 있다.
- 죽는 것이 아니라 회수되는 감각이 있다.

감정 판타지:

- 거창한 영웅이 아니라, 끝까지 자기 이름을 붙잡는 사람.
- 웃긴 광고 문구에 짧게 받아치는 실무자.
- 위험하지만 유저가 데리고 돌아오고 싶은 기본 주인공.

## 외형 필수 포인트

0.2에서 완성 일러스트가 없더라도 아래 포인트는 필요하다.

| 요소 | 이유 | 우선순위 |
| --- | --- | --- |
| 손글씨 이름표 | 윤서의 정체성 핵심 | 필수 |
| 반품 스캐너 | 자동공격 실루엣 | 필수 |
| 도장/라벨기 형태 장비 | 차징 스킬 정체성 | 필수 |
| 회수선 릴 | 회수 루프 시각화 | 높음 |
| 바랜 작업복 | 영웅보다 실무자 느낌 | 높음 |
| 찢긴 고객 라벨 | 캠페인 등록 위험 | 중간 |

모바일 탑다운에서 우선 읽혀야 하는 것:

```text
머리/몸 실루엣
손에 든 스캐너
차징 시 찍는 동작
이름표 또는 회수선 색 포인트
```

## 기본 전투

### 자동공격: 반품 스캐너

역할:

- 초보자에게 안정적인 기본 공격.
- 광고 전단 떼 같은 잡몹을 꾸준히 정리.
- 타깃이 명확하고 읽기 쉬워야 함.

효과:

```text
가장 가까운 적에게 짧은 역송출 탄을 발사한다.
적중 시 적 표면의 광고 문구가 잠깐 뒤집힌다.
```

0.2 수치 방향:

| 항목 | 기준 |
| --- | --- |
| 피해 | 낮음~중간 |
| 사거리 | 중간 |
| 공격속도 | 중간 |
| 투사체 크기 | 모바일에서 읽히게 약간 큼 |
| 자동 조준 | 가장 가까운 적 우선 |

유저 선호 포인트:

- 계속 움직여도 공격이 안정적으로 나간다.
- 적중 이펙트가 광고를 벗겨내는 느낌을 준다.
- 초반 30초 안에 “기본캐인데 손맛 괜찮다”가 나와야 한다.

### 차징 스킬: 반품 도장

역할:

- 윤서의 핵심 손맛.
- 단일 버튼 조작의 재미.
- 보스/엘리트/위험 웨이브 대응.

효과:

```text
차징 윈도우에 버튼을 누르면 전방 부채꼴 범위에 강한 역송출 피해를 준다.
맞은 적은 짧게 둔화되거나 광고 외피가 약해진다.
```

0.2 기본 형태:

| 항목 | 기준 |
| --- | --- |
| 입력 | 단일 탭 |
| 범위 | 전방 부채꼴 |
| 사용 빈도 | 일정 주기 차징 윈도우 |
| 위험 | 빗나가면 손해지만 치명적이지 않음 |
| 보상 | 맞히면 웨이브가 확실히 정리됨 |

필수 피드백:

- 도장 찍는 짧은 사운드.
- 전방 광고 문구가 접히거나 찢기는 VFX.
- 맞은 적이 잠깐 “반품 처리”된 것처럼 흔들림.
- 엘리트에게 맞히면 외피가 깨지는 느낌.

차징 대사 후보:

```text
반품.
무효.
접수 끝.
네가 붙인 이름은 안 받아.
```

## 0.2 성장

윤서의 0.2 성장은 3개 계열만 체감시키면 된다.

### 1. 반품 스캐너 출력

유저 체감:

- 자동공격이 조금 더 안정적이다.
- 초반 잡몹 정리가 편해진다.

업그레이드 후보:

```text
스캐너 출력 +1
스캐너 사거리 +1
타깃 안정 보정
```

### 2. 반품 도장 개조

유저 체감:

- 차징을 맞히는 재미가 커진다.
- 엘리트/보스 대응이 좋아진다.

업그레이드 후보:

```text
도장 피해 +1
도장 범위 +1
도장 둔화 추가
신호 오브젝트 추가 피해
```

### 3. 회수선 안정화

유저 체감:

- 실패해도 정산이 더 납득된다.
- 첫 회수 경험이 장비 성장으로 이어진다.

업그레이드 후보:

```text
회수 직전 데이터 보존 +1
첫 치명타 방지 후보
신호 과부하 경고 시간 증가
```

0.2 금지:

- 너무 많은 패시브 트리.
- 복잡한 스킬 변형 3개 이상.
- 이름 고정/캠페인 활용을 UI 수치로 노출.
- 기본캐라서 약하게 설계하기.

## 첫 런 경험

### 0~30초

목표:

- 이동과 자동공격이 즉시 이해된다.
- 윤서가 안정적이라는 느낌을 준다.
- 광고 세계가 웃기고 이상하다는 첫인상을 준다.

문구 후보:

```text
캠페인: 오늘의 추천 고객이 입장했습니다.
윤서: 추천 끄는 버튼은 왜 항상 없어.
```

### 30~70초

목표:

- 차징 스킬을 처음 써본다.
- 반품 도장이 시원해야 한다.

문구 후보:

```text
윤서: 반품 처리.
```

### 70~100초

목표:

- 캠페인이 윤서를 이름으로 읽으려 한다.
- 강제 회수가 죽음이 아니라 시스템/서사 이벤트로 느껴진다.

문구 후보:

```text
캠페인: 고객명 확인 중...
윤서: 방금... 내 이름을 읽었어?
```

회수 직전:

```text
신호 과부하
침묵 보급소가 역송출선을 강제로 당깁니다
```

## 보급소 첫 반응

첫 회수 후 반드시 짧게 보여줄 반응:

```text
미나: 네 이름, 아직 멀쩡하지?
윤서: 아직은.
도윤: 장비가 버텼어. 다음엔 출력 제한부터 손보자.
```

유저가 느껴야 하는 것:

- 죽은 것이 아니라 돌아왔다.
- 윤서의 이름이 중요하다.
- 보급소 사람들이 윤서를 걱정한다.
- 첫 개조가 자연스럽다.

## 흔적 반응 0.2

윤서는 0.2에서 최소 2개, 권장 4개 흔적에 반응해야 한다.

| 흔적 | 윤서 반응 | 유저 감정 |
| --- | --- | --- |
| 지워지지 않은 문패 | `좋아. 이건 아직 주소가 아니라 집이야.` | 이름/집의 의미 |
| 반쯤 탄 가족사진 | `얼굴이 없어도, 손은 남아 있네.` | 조용한 슬픔 |
| 무음 라디오 | `아무 소리도 안 나는 게 이렇게 든든할 줄은 몰랐어.` | 침묵의 가치 |
| 이름 없는 영수증 | `제일 싫은 종류네. 산 건 있는데 산 사람이 없어.` | 상품화 공포 |

## 외곽 주택가 전용 반응

윤서가 외곽 주택가를 반복 방문하면 캠페인은 윤서를 더 정확히 부른다.

내부 조건:

```text
character_local_pair_count.pc_yunseo_001.r01_suburb >= 2
```

유저 표현:

```text
캠페인: 윤서 고객님께 어울리는 가족 구성을 준비했습니다.
윤서: 내 가족을 네가 정하지 마.
```

주의:

- 내부 조건은 UI에 보여주지 않는다.
- “윤서 성향” 같은 표현을 쓰지 않는다.
- 현상과 대사로만 전달한다.

## 보스전 기준

스마일 홈 시어머니 보스전에서 윤서가 가져야 할 역할:

- 보스 문구에 짧게 반응한다.
- 차징 스킬이 보스 외피 구간을 깎는 데 의미가 있다.
- 보스 후 선택에서 윤서의 관점이 드러난다.

보스 전 문구:

```text
윤서: 집을 팔려고 가족을 만든 거야?
```

결절 파괴:

```text
윤서: 이제 적어도 저 집들은 누굴 흉내 내진 않겠지.
```

기억 추출:

```text
윤서: 부서뜨리기 전에, 이름 하나라도 꺼내자.
```

## 유저 선호 점수 평가

0.2 윤서 기준 점수:

| 기준 | 점수 | 이유 |
| --- | ---: | --- |
| 첫눈 매력 | 7 | 작업복/이름표/스캐너가 좋지만 일러스트가 약하면 묻힐 수 있음 |
| 조작 손맛 | 8 | 자동공격 안정 + 반품 도장 한 방이 명확함 |
| 초반 감정 이입 | 8 | 이름을 빼앗길 뻔한 첫 회수가 강함 |
| 장기 애정 | 8 | 빈 고객란/이름표/반품 말투가 확장 가능 |
| 수익 확장성 | 7 | 스킨/대사 확장은 좋지만 첫 유료 후보는 아님 |
| 세계 해석력 | 9 | 세계관 입구로 매우 강함 |
| 리스크 관리 | 8 | 무료 기본캐라 과금 압박 위험 낮음 |

종합:

```text
8.0 / 10
```

판정:

윤서는 0.2 기본 캐릭터로 적합하다. 단, 손맛과 짧은 대사가 약하면 점수가 6점대로 떨어진다. 특히 `반품 도장`이 시원하지 않으면 윤서의 전투 매력이 크게 줄어든다.

## 10점 개선 경로

윤서의 점수는 0.2에서 고정되지 않는다. 기본 캐릭터이기 때문에 오히려 장기적으로 계속 좋아져야 한다.

개선 목표:

| 단계 | 목표 | 필요한 개선 |
| --- | ---: | --- |
| 0.2 | 8.0 | 기본 손맛, 첫 회수, 이름표 각인 |
| 0.3 | 8.5 | 반품 도장 VFX/사운드 강화, 흔적 반응 2개 추가 |
| 0.4 | 9.0 | 외곽 주택가 이후 로컬별 윤서 전용 반응 |
| 장기 | 10 후보 | 유저가 윤서를 기본캐가 아니라 게임의 얼굴로 기억 |

10점에 가까워지는 조건:

- 반품 도장을 누르는 순간이 윤서의 상징이 된다.
- 유저가 윤서의 이름표와 빈 고객란을 기억한다.
- 새 로컬이 추가될 때 윤서가 그 장소를 다르게 보게 한다.
- 윤서 스킨은 외형만 바꾸지 않고 회수/반품 감각을 조금 다르게 만든다.
- 유료 캐릭터가 늘어도 윤서를 다시 꺼낼 이유가 남는다.

금지:

- 윤서를 나중 캐릭터보다 일부러 약하게 두지 않는다.
- 윤서의 점수를 올린다는 이유로 긴 설명 대사를 추가하지 않는다.
- 기본 캐릭터 개선을 유료 스킨에만 넣지 않는다.

| 성능 약함 | “나중 캐릭터 전까지 버티는 캐릭” | 기본 전투 안정성 확보 |

## 0.2 합격 기준

윤서 0.2는 아래를 만족해야 한다.

- 첫 30초 안에 자동공격이 이해된다.
- 첫 차징 사용이 시원하다.
- 첫 강제 회수가 죽음이 아니라 회수로 느껴진다.
- 첫 회수 후 유저가 윤서의 이름을 기억한다.
- 보급소 사람들이 윤서를 걱정한다는 감각이 있다.
- 흔적 2개 이상에서 윤서다운 반응이 나온다.
- 보스전에서 반품 도장이 의미 있다.
- 내부 성향/철학 UI가 노출되지 않는다.
- 그냥 액션만 하는 유저도 윤서가 편하고 재밌다.
- 깊게 보는 유저는 윤서의 빈칸과 이름표에 관심을 갖는다.

## 구현 우선순위

1. 반품 스캐너 자동공격 손맛.
2. 반품 도장 차징 VFX/사운드.
3. 손글씨 이름표 시각 포인트.
4. 첫 강제 회수 문구.
5. 보급소 첫 반응 3줄.
6. 흔적 반응 최소 2개.
7. 보스 전/후 선택 반응.
8. 반복 방문 시 윤서 호명 문구.

## 금지 방향

- 윤서를 선택받은 초능력자로 만들지 않는다.
