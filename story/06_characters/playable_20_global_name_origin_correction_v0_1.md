# 플레이어블 20인 전역 이름/원점 교정 v0.1

## 문서 상태

- 문서 성격: 한국어권 임시명 교정 / 전역 캠페인 후보명 보정
- 교정 요약: `13~19 한국어권 임시명은 최종명으로 사용 금지`
- 상위 기준: `story/06_characters/playable_20_campaign_open_order_rebuild_result_v0_1.md`
- 후속 기준: 문서 마지막의 단일 다음 작업 포인터를 따른다.

## 0. 교정 결론

순번만 재배치하고 13~19번 이름을 한국어권 임시명으로 남겨 두면, 플레이어블 로스터가 다시 E01 중심으로 쏠린다.

따라서 13~19의 한국어권 임시명은 모두 `legacy placeholder`로 내리고, 원점 캠페인이 외부 광역권이면 현지권/다국적 이름 후보를 우선한다.

## 0.1 전역 플레이어블 교정 원칙

이번 교정의 핵심은 한국인 캐릭터를 계속 늘리는 것이 아니라, 96개 전역 캠페인이 순차적으로 열릴 때마다 해당 캠페인의 현실권/언어권/등록 절차에 맞는 플레이어블 후보를 만드는 것이다.

잠금 원칙:

1. 출시 초기는 한국인 기본 주인공과 추가 1~2명까지만 허용한다.
2. 이후 한국인 후보는 E01 추가 시즌 또는 전역 캠페인 분포가 충분히 열린 뒤 별도 승인 없이는 금지한다.
3. E01 한반도 권역 캐릭터를 계속 만드는 방식은 금지한다.
4. 플레이어블은 좋은 캐릭터 아이디어가 아니라 열린 캠페인을 다르게 읽는 관점이다.
5. 캠페인이 먼저 열리고, NPC/오브젝트/상태 반응/작은 결말이 깔린 뒤 플레이어블 후보가 나온다.
6. 기존 번호는 해금 순번이 아니다. `프로필 산출 번호`, `코드명 숫자`, `최종 해금 순번`을 반드시 분리한다.

출시 초기 한국인 제한:

```text
기본 주인공:
윤서 유지.

추가 한국인:
최대 1~2명까지만 허용.

이후 한국인 후보:
E01 추가 시즌 또는 전역 캠페인 분포가 충분히 열린 뒤 별도 승인 없이는 금지.

금지 사유:
"한국어 이름이 편하다."
"초기 문서에 한국어 임시명이 있다."
"이미 14~19에 한국어 이름이 있다."
```

전역 권역 원칙:


## 0.2 제작 절차 잠금

앞으로 플레이어블 후보를 만들 때는 캐릭터 아이디어가 먼저 오지 않는다. 열린 캠페인과 그 캠페인의 등록 절차가 먼저 온다.

작업 순서:

1. 먼저 어떤 캠페인이 실제로 열렸는지 확인한다.
2. 그 캠페인의 핵심 등록 방식이 무엇인지 확인한다.
3. `GLOBAL_ADMIN_REAL_LOCATION_MAP_V0_1.md`와 충돌하지 않는 현실권/언어권 이름 후보를 만든다.
4. 기존 플레이어블/NPC가 맡는 기능을 침범하지 않는지 확인한다.
5. HP 0을 전투 사망이 아니라 그 캠페인의 제도적 판정으로 만든다.
6. 후보를 바로 v1.0으로 쓰지 않고, 상세 결재 -> 조건부 해소 -> 프로필 순서로만 진행한다.

번호 분리:

| 구분 | 의미 | 잠금 규칙 |
|---|---|---|
| 프로필 산출 번호 | 문서가 작성된 기존 순서 | 기존 파일과 참조를 보존하기 위한 관리 번호다. |
| 코드명 숫자 | GATE-09, GUARD-13처럼 기존 산출물에 붙은 숫자 | 즉시 바꾸지 않는다. 마이그레이션 문서에서 alias 여부를 판정한다. |
| 최종 해금 순번 | 캠페인 오픈 구조에 따라 플레이어가 만나는 순서 | 기존 번호와 같다고 가정하지 않는다. |
| 유저 노출 순번 | 캐릭터 선택 UI나 도감에 보일 순서 | 최종 UX/엔진 단계에서 별도 결정한다. |

한국인 후보 처리:

| 구분 | 허용 | 금지 |
|---|---|---|
| 출시 기본 | 윤서 유지 | 윤서 외 한국인 다수를 기본 동료처럼 배치 |
| 출시 추가 | 최대 1~2명까지만 별도 승인 | 기존 3~7번을 모두 출시 초반 해금으로 처리 |
| E01 내부 후보 | 산출물과 후보성을 보존 | 캠페인 오픈 없이 한국어 이름이라는 이유만으로 다음 프로필 작성 |
| 외부 광역권 후보 | 해당 현실권/언어권 후보명 우선 | 14번 이후 한국어 임시명을 최종명처럼 유지 |

## 0.3 금지 방향 잠금

금지:

- 한국인 이름으로 빈 슬롯을 채우기.
- E01 캐릭터를 계속 늘리기.
- 모든 캠페인이 열리기도 전에 한국인 플레이어블을 추가하기.
- 14번 이후 기존 한국어 임시명을 최종명처럼 쓰기.
- 직업/국적/속성 조합으로 캐릭터 만들기.
- "로봇이라 초반", "의료라 중요", "교통이라 편리" 같은 이유로 캠페인 오픈 순서를 무시하기.
- NPC가 해야 할 기능을 플레이어블에게 넘기기.

특히 14번 이후는 `기존 번호 그대로 다음 프로필 작성`을 금지한다. 다음 작업은 기존 코드명과 해금 순번을 분리하는 마이그레이션이어야 한다.

교정 후 한국어권 인간 산출물은 `6명`으로 보존 관리한다. 단, 이것은 출시 초기 해금 허용 수가 아니다. 출시 초기 한국인 플레이어블은 기본 주인공과 추가 1~2명 한도 안에서 별도 재판정하며, 나머지는 E01 캠페인 오픈 깊이 또는 추가 시즌 승인 전까지 보류한다.

이 문서에서 `보류` 판정을 받은 E01 한국인 후보는 삭제 대상이 아니다. 다만 "프로필이 있으므로 초반 해금된다"는 해석을 금지한다.

## 4. 레지스트리 반영 규칙

- 13~19의 기존 한국어 이름은 삭제가 아니라 `legacy placeholder`로 남긴다.
- 새 후보명은 `교정 후보`로 레지스트리에 반영한다.
- 기존 파일명과 코드명 숫자는 마이그레이션 판정 전까지 바꾸지 않는다.
- 전태주가 들어간 기존 GUARD-13 프로필은 `legacy profile name`으로 보존하되, E12_C02 기준 최종명은 Amina Diop 우선으로 재검토한다.
