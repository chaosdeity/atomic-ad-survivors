## 0. 판정 요약

판정:

```text
조건부 통과
```

Mika Arakawa / GATE-09 / 미카, 막차 환승 심사자는 9번 슬롯에 둘 가치가 있다. 1~7번이 E01 한국인 인간 중심 후보이고, 8번 리오가 중반 체류/숙박 캠페인권 로봇 후보라면, 9번은 첫 외부 광역권에서 "다른 나라"가 아니라 "다른 생활 절차"를 플레이어블 관점으로 읽게 만드는 역할을 맡을 수 있다.

다만 바로 프로필 v1.0으로 가지 않는다. 아래 조건이 먼저 잠겨야 한다.

1. 단순 교통 캐릭터, 빠른 이동 캐릭터, 개찰구 도둑, 지하철 안내원으로 보이면 안 된다.
2. E02 전체를 열지 않는다. E02_C07-L01~L02 수준의 환승 통로와 막차 심사 플랫폼까지만 사용한다.
3. E01_C07 마지막 출항권, E06_C07 공항검역 이송권, E07_C07 통관철도 통행권과 역할을 분리해야 한다.
4. HP 0은 사망/부활이나 단순 회수선 실패가 아니라 개찰 미통과, 막차 탑승 실패, 플랫폼 잔류 처리로 차별화해야 한다.
5. 미카가 NPC로 남는 편이 더 강한지 검토해야 한다. 현 판정에서는 "절차를 플레이어가 직접 밀고 당겨야 강한 캐릭터"라 플레이어블 조건부 통과다.

핵심 질문:

```text
막차를 놓친 사람은 아직 이동 중인가,
이미 잔류자인가?
```

## 1. 한 줄 정의

```text
미카는 빠른 이동 캐릭터가 아니라,
막차/환승/개찰 심사가 사람을 잔류자로 고정하기 전의 짧은 틈을 전투화한 사람이다.
```

이 후보의 힘은 속도나 길 찾기가 아니다. 미카는 "통과했다", "환승 중이다", "탑승하지 못했다", "플랫폼에 남았다"라는 판정이 닫히기 직전의 간격을 읽는다.

따라서 전투의 중심 문장은 아래로 둔다.

```text
문이 닫히기 전까지는,
아직 잔류자가 아니다.
```

이 문장은 후보의 강점과 위험을 동시에 설명한다. 미카는 닫히는 문을 영원히 열어두는 사람이 아니다. 닫히기 전의 절차를 한 박자 늦출 뿐이고, 그 대가로 시간표와 안내 신호가 더 거칠게 미카를 따라붙는다.

## 3. 핵심 오판

초안:

```text
막차를 타면 돌아갈 수 있다고 믿음.
```

결재안:

```text
마지막 환승만 통과하면 아직 잔류자가 아니라고 믿는다.
```

이 믿음은 일부 맞다. 환승 로그가 살아 있고, 개찰 클립이 닫히지 않았고, 막차 안내가 완전히 끊기지 않았다면 캠페인은 미카를 "이동 중"으로도 읽는다.

하지만 이 믿음은 위험하다. E02_C07 지하철 환승 심사권은 사람을 죽이려는 곳이 아니다. 캠페인은 혼란한 플랫폼에서 사람을 안전하게 분류하려 한다. 문제는 막차를 놓친 사람, 환승 통로에 남은 사람, 정기권 심사가 끝나지 않은 사람을 "플랫폼 잔류자"로 보존하려 한다는 점이다.

더 깊은 오류:

- 시간표가 남아 있으면 선택지도 남아 있다고 믿는다.
- 환승 심사가 끝나지 않았으면 아직 누구에게도 붙잡히지 않았다고 믿는다.
- 개찰을 통과하지 못한 상태가 실패가 아니라 보류라고 믿고 싶어 한다.
- 하지만 보류가 길어지면 캠페인은 미카를 안전한 잔류자로 고정한다.

통과 조건:

```text
미카는 막차를 잡는 사람이 아니다.
막차를 놓친 사람을 플랫폼 잔류자로 확정하는 절차를,
플레이어가 직접 한 박자 늦춰 보게 하는 관점이어야 한다.
```

## 4. 플레이어블이어야 하는 이유

미카는 NPC로만 남겨도 강한 후보처럼 보인다. 역에서 만나 안내를 듣고, 막차 경고를 남기는 NPC는 쉽게 성립한다.

하지만 플레이어블로 검토할 가치가 있는 이유는 미카의 핵심이 "안내"가 아니라 "판정 조작"이기 때문이다. 플레이어가 직접 조작할 때 아래 감각이 생긴다.

1. 닫히는 게이트를 무작정 열어두는 것이 아니라, 어느 판정을 늦출지 선택한다.
2. 통행태그를 아껴야 할지, 환승 심사 로그를 태워야 할지 판단한다.
3. 시간표를 늦추면 적 호출도 늦지만, 다음 환승 경로도 꼬인다.
4. 플랫폼 잔류 처리를 피하려면 보급소 회수선이 잡을 좌표를 남겨야 한다.
5. "빠르게 도망가는 쾌감"이 아니라 "아직 이동 중이라고 우기는 긴장"이 플레이 감각이 된다.

NPC로 남을 때의 한계:

```text
NPC 미카는 막차 안내를 말할 수 있다.
플레이어블 미카는 막차 안내가 닫히는 순간을 직접 비틀게 한다.
```

따라서 현 판정은 조건부 통과다.

플레이어블로 통과 가능한 이유:

- E02 첫 외부 확장을 생활 절차 차이로 보여준다.
- 1~8번과 전투 리듬이 다르다.
- HP 0을 플랫폼 잔류 처리로 차별화할 수 있다.
- E02_C07을 E02 전체 확장 없이 작은 시즌 슬라이스로 열 수 있다.

보류될 수 있는 이유:

- 단순 속도형 캐릭터가 되면 실패다.
- E01_C07 마지막 출항권이나 E06_C07 공항검역 이송권과 섞이면 실패다.
- 관광/도시 이미지가 캠페인 절차보다 앞서면 실패다.

## 5. 전투 리듬

### 5.1 기본 리듬

기본 리듬:

```text
통과 / 환승 / 미탑승 / 잔류 판정을 밀고 당긴다.
```

미카는 적을 베거나 빠르게 돌파하는 캐릭터가 아니다. 미카의 조작감은 닫히는 게이트, 끊기는 안내, 시간표의 마지막 칸, 정기권 조각을 붙잡고 "아직 처리 완료가 아니다"라고 버티는 데서 나온다.

전투 흐름 예시:

1. 적이나 함정이 "진입 금지" 판정을 걸기 전에 환승 중 상태를 만든다.
2. 막차 안내판이 꺼지기 전 짧은 통행 가능 시간을 만든다.
3. 개찰 클립을 꽂아 적의 추적 경로를 플랫폼 잔류 대기열로 우회시킨다.
4. 환승 도장을 찍어 특정 구역의 통행 판정을 한 번만 보류한다.
5. 시간표를 늦추는 대신 다음 웨이브가 더 압축되어 들어온다.

### 5.2 위험한 강점

| 강점 | 효과 | 위험 |
|---|---|---|
| 환승 중 판정 | 통행 금지/추적/게이트 폐쇄를 짧게 보류한다 | 보류가 길어지면 잔류 처리 누적 |
| 막차 안내 지연 | 웨이브나 캠페인 호출을 한 박자 늦춘다 | 다음 안내가 더 강한 공식 명령으로 돌아온다 |
| 개찰 클립 좌표 | HP 0 때 보급소가 잡을 위치를 남긴다 | 클립이 과사용되면 통행태그 손실 |
| 정기권 조각 | 특정 경로를 제한적으로 다시 통과한다 | 자동 통과권처럼 보이면 실패 |
| 플랫폼 잔류 전환 | 적 일부를 플랫폼 대기열로 묶는다 | 군중 제어 마법처럼 쓰면 실패 |

### 5.3 실패 시 후폭풍

미카의 실패는 단순 쿨다운 실패가 아니다.

```text
시간표가 닫힌다.
개찰 클립이 잘린다.
환승 도장이 무효화된다.
플랫폼 안내가 같은 문장을 반복한다.
미카는 이동 중인 사람이 아니라 잔류자로 읽힌다.
```

실패 후폭풍:

- 통행태그 손실.
- 환승 심사 로그 오염.
- 저주파 안내 반복으로 수신태그 부담 증가.
- 보급소 회수선이 개찰 클립 좌표를 늦게 잡음.
- 다음 출격에서 일부 게이트가 더 빨리 닫힘.

### 5.4 플레이어 조작감

미카의 조작감은 고속 이동이 아니다.

```text
짧은 창을 열고,
그 창이 닫히기 전에 다른 판정을 걸어,
잔류 처리 확정을 늦추는 조작감.
```

추천 조작 감각:

- 짧은 타이밍 입력.
- 임시 통행 칸 만들기.
- 닫히는 문 사이에서 판정 선택.
- 과사용 시 시간표 꼬임.
- 회피보다 "아직 통과 중" 상태 유지.

### 5.5 파티/동행 시 역할

미카는 파티 이동 속도를 올리는 버퍼가 아니다.

파티 역할:

```text
팀 전체를 빠르게 움직이는 것이 아니라,
특정 캠페인 절차가 닫히는 순간을 늦추고,
위험한 통행 판정을 한 번 더 보류한다.
```

다른 캐릭터와의 조합:

- 윤서가 회수해야 할 물건을 "잔류물"로 고정하지 않도록 환승 중 상태를 만든다.
- 여울이 낮은 수신으로 놓칠 수 있는 안내를 공식 신호가 아니라 저주파 잔향으로만 남긴다.
- 유건이 계량 초과를 늦추듯, 미카는 통행 실패 확정을 늦춘다.
- 리오의 체크아웃 지연과 비슷해 보일 수 있으므로, 미카는 방이 아니라 통행 게이트와 시간표만 다룬다.

## 6. 태그 / 정산 / 자원

| 항목 | 결재안 |
|---|---|
| 관련 태그 | 통행태그, 수신태그 |
| 주 정산 자원 | 환승 심사 로그, 개찰 클립, 막차 안내 기록 |
| 보조 정산 자원 | 정기권 조각, 플랫폼 잔류표, 환승 도장 |
| 상태값 반응 | 캠페인 농도, 수신 안정도, 인양 안정도, 보급소 노출도 |
| 과사용 위험 | 잔류 처리 누적, 통행태그 훼손, 안내 반복 강화 |

정산 루트:

```text
주 루트:
통행태그
환승 심사 로그
개찰 클립
막차 안내 기록

보조 루트:
수신태그
정기권 조각
플랫폼 잔류표
환승 도장
```

정산 감각:

```text
좋은 결과는 더 빨리 이동하는 것이 아니라,
잔류 처리 확정 전에 다음 경로가 아직 살아 있음을 남기는 것이다.
```

금지:

```text
통행태그를 단순 이동 속도 재화로 만들지 않는다.
정기권 조각을 만능 통과권으로 만들지 않는다.
환승 도장을 도적의 열쇠처럼 쓰지 않는다.
막차 안내 기록을 관광 안내 수집품처럼 만들지 않는다.
```

## 7. HP 0 / 긴급 인양 차별화

미카의 HP 0은 사망/부활이 아니다.

핵심 상태명:

```text
개찰 미통과 / 플랫폼 잔류 처리
```

HP 0 흐름:

| 단계 | 상태 | 연출 |
|---|---|---|
| 1 | 개찰 미통과 | 개찰 클립이 잘리고, 통행태그가 미확인으로 깜박인다 |
| 2 | 막차 탑승 실패 | 막차 안내판이 "출발 완료"와 "환승 대기"를 번갈아 표시한다 |
| 3 | 플랫폼 잔류 처리 | 캠페인이 미카를 이동 중인 사람이 아니라 플랫폼 잔류자로 보존하려 한다 |
| 4 | 저주파 안내 반복 | 같은 안내 문장이 반복되며 보급소 비콘과 충돌한다 |
| 5 | 긴급 인양 | 보급소가 환승 심사 로그와 개찰 클립 좌표의 불일치를 잡아 되당긴다 |

## 9. 관계선

| 대상 | 관계선 |
|---|---|
| 윤서 | 윤서는 현장에서 사람과 물건의 마지막 좌표를 회수한다. 미카는 통과/미통과 판정이 좌표를 만들기 전의 순간을 붙잡는다. |
| PATCH | PATCH는 보급소 수리와 자기부품화의 로봇이다. 미카는 환승 게이트와 시간표에 묶인 인간 후보로, 수리/충전/부품을 다루지 않는다. |

| 한이루 / RETURN-05 | 이루는 자기 자신이 반송/미수령 대상으로 읽힌 사람이다. 미카는 물건처럼 반송되는 것이 아니라 플랫폼 잔류자로 고정될 위험을 다룬다. |
| 차유건 / METER-06 | 유건은 계량 초과와 충전 차단을 다룬다. 미카는 시간표 초과와 개찰 미통과를 다룬다. |
| 백여울 / QUIET-07 | 여울은 덜 들키지만 덜 구해지는 낮은 수신의 리스크다. 미카는 공식 안내가 너무 선명하게 반복되어 잔류자로 굳는 리스크다. |

## 13. 성장 방향

초기:

```text
마지막 환승만 통과하면 아직 잔류자가 아니라고 믿는다.
```

중반:

```text
환승 중 상태가 길어질수록 사람은 더 안전해지는 것이 아니라,
더 정확하게 플랫폼 잔류자로 보존된다는 것을 배운다.
```

후반:

```text
닫힌 문을 부수는 것이 아니라,
닫힌 뒤에도 사람이 남아 있으면 절차가 완료되지 않았다고 말할 수 있게 된다.
```

최종 변화 문장:

```text
문이 닫힌 뒤에도,
사람은 완료된 절차가 아니다.
```

성장 방향의 핵심은 속도 향상이 아니다. 미카는 더 빨라지는 캐릭터가 아니라, 더 정확히 "아직 끝나지 않은 이동"을 구분하는 캐릭터가 되어야 한다.

## 14. 프로필 v1.0 전환 조건
