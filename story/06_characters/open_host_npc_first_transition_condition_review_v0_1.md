## 0. 결론

OPEN-HOST는 출시 초기 플레이어블 +1이 아니다.

현재 판정:

1. OPEN-HOST는 R01 핵심 NPC로 먼저 등장한다.
2. R01 Vertical Slice에서는 직접 대면 NPC 선등장을 유지한다.
3. 출시 초기 캐릭터 선택 가능 U5로 올리지 않는다.
4. 플레이어블 전환은 R01 결절 처리와 NPC 기능 보존을 통과한 뒤에만 조건부 가능하다.
5. OPEN-HOST가 R01 절차를 설명하는 캐릭터가 되면 실패다.
6. OPEN-HOST가 WELCOME-DESK, ROOM-12, DINING-4, FRAME-LEFT 등 다른 R01 NPC의 기능을 흡수하면 전환 금지다.

잠금 문장:

```text
OPEN-HOST는 먼저 믿기 어려운 R01 내부자 NPC여야 한다.
플레이어블 전환은 그가 안내를 멈추고 선택지를 보여주는 법을 배운 뒤에만 의미가 있다.
```

## 1. 해금 단계 기준

| 단계 | OPEN-HOST 운용 | 이번 판정 |
|---|---|---|
| U1 신호 해금 | 뒤집힌 안내판, 어긋난 화살표, 가짜 귀환로 경고 흔적 | 허용 |
| U2 조우 | 상담 부스 뒤쪽 또는 모델하우스 접근 전 직접 대면 | 허용 |
| U3 거점 협력 | R01 내부 동선 데이터, 보스 진입 경로 검증, 보급소 논쟁 | 조건부 허용 |
| U4 동행/임무 | R01 후속 노드에서 제한 동행 또는 임무형 전환 | 후속 조건부 가능 |

중요:

```text
OPEN-HOST가 직접 대면 NPC로 등장해도 U5가 아니다.
OPEN-HOST가 R01 경로를 경고해도 U5가 아니다.
OPEN-HOST가 보급소에서 논쟁 대상이 되어도 U5가 아니다.
```

## 3. 다른 R01 NPC 기능 보존선

OPEN-HOST가 전환되더라도 아래 기능은 빼앗으면 안 된다.

| NPC/축 | 담당 기능 | OPEN-HOST가 빼앗으면 생기는 문제 |
|---|---|---|
| ROOM-12 | 가족대표 칸, 식량태그/침상 접근, 바로 구출할 수 없는 조건 | OPEN-HOST가 가족 칸 절차를 설명하면 ROOM-12의 도덕 압박이 약해진다. |
| PORCH-LINE | 현관 줄, 순번표, 질서와 폭력의 경계 | OPEN-HOST가 줄과 대기열 해결자가 되면 R01 생활감이 사라진다. |
| NAME-PAINTER | 문패, 이름 덧칠, 이름 흔적 보존 | OPEN-HOST가 이름 문제를 해설하면 윤서/복희/NAME-PAINTER 축이 흐려진다. |
| MAIL-LOOP | 주소 반복, 우편함 목소리, 반송/수신 잔향 | OPEN-HOST가 주소 힌트까지 맡으면 MAIL-LOOP가 단순 장치가 된다. |
| VAC-0 | 빈 집 내부 홈케어, 흔적 삭제, 로봇 자율성 씨앗 | OPEN-HOST가 집 내부 청소/흔적 보존을 해결하면 VAC-0의 불편함이 사라진다. |
| DINING-4 | 네 번째 식탁 자리, 모델하우스 가족 칸 압박 | OPEN-HOST가 식탁 절차를 설명하면 보스 전 감정 압박이 약해진다. |
| WELCOME-DESK | 상담 데스크, 가족 구성 질문, 거주태그 확인 보류 | OPEN-HOST가 상담 데스크 기능까지 흡수하면 WELCOME-DESK가 복제 캐릭터가 된다. |
| FRAME-LEFT | 보스 후 가족사진 프레임, 기억 추출, 흔적 보존 | OPEN-HOST가 보스 후 의미를 설명하면 FRAME-LEFT의 잔향 기능이 지워진다. |

보존 원칙:

```text
OPEN-HOST는 길을 안다.
하지만 R01의 모든 절차를 아는 해설자가 아니다.
길은 OPEN-HOST가 들고 오고,
자리, 줄, 이름, 주소, 청소, 식탁, 상담, 사진은 각 NPC가 들고 있어야 한다.
```

## 4. 플레이어블 전환 조건

OPEN-HOST의 전환은 아래 조건을 모두 통과할 때만 가능하다.

1. R01 오픈하우스 거리에서 OPEN-HOST의 안내 흔적을 2~3회 본다.
2. OPEN-HOST가 윤서를 한 번 돕지만, 동시에 위험한 동선으로 유도할 뻔한다.
3. R01 핵심 결절 또는 보스 전후 사건을 통해 OPEN-HOST의 안내 후폭풍을 본다.
4. 보급소가 OPEN-HOST 수용 여부를 두고 논쟁한다.
5. OPEN-HOST가 안내 배지를 깨뜨리거나 내려놓는다.
6. OPEN-HOST가 "안내하지 않고 선택지를 보여주는 법"을 배우려 한다.
7. 다른 R01 NPC의 역할이 전환 뒤에도 남아 있다.
8. 윤서와 긴장 관계가 유지된다.
9. OPEN-HOST가 착한 내부자, 세뇌 피해자, R01 설명 NPC로 단순화되지 않는다.

하나라도 실패하면 전환은 보류한다.

## 6. 출시 초기 운용 권장안

권장 기본값:

```text
윤서:
출시 초기 U5 단독 시작.

PATCH:
U1~U4 보류.

OPEN-HOST:
R01 직접 대면 NPC 선등장.
출시 초기 U5 금지.
후속 R01 진행 후 전환 조건부 가능.
```

Vertical Slice에서 OPEN-HOST는 직접 대면 NPC로 충분하다.
유저는 그를 먼저 믿어서는 안 된다.

OPEN-HOST의 첫 기능은 플레이어블 전투가 아니라 아래 세 가지다.

1. 친절한 길도 검증 대상이라는 감각을 만든다.
2. R01 내부 절차를 아는 사람이 왜 불편한지 보여준다.
3. 윤서가 직접 판단해야 하는 첫 지역 감각을 유지한다.
