# PATCH 2번 플레이어블 최종 판정 v0.1

## 문서 상태

```text
상태:
작성 완료 v0.1

작업 성격:
새 플레이어블 작성 아님.
PATCH의 2번 슬롯 적합성 최종 판정.

최종 판정:
통과

로스터 반영:
PATCH를 윤서 다음 2번째 플레이어블로 잠근다.

다음 작업 포인터:
3~7번 초반 플레이어블 후보 선정
```

---

## 0. 결론

PATCH는 윤서 다음 2번째 플레이어블로 통과한다.

이유는 단순히 "로봇 캐릭터가 하나 필요해서"가 아니다. 윤서가 첫 지역에서 사람을 물건처럼 처리하는 세계를 보여준다면, PATCH는 바로 다음 단계에서 로봇도 부품만은 아니라는 질문을 연다.

PATCH가 2번일 때 좋은 점은 아래 세 가지다.

```text
1. 보급소 생활감이 빨리 생긴다.
2. 충전태그, 부품 회수, 긴급 인양 장비 같은 시스템어가 감정과 붙는다.
3. 윤서가 "회수"의 기준을 사람에서 로봇까지 넓히는 계기가 된다.
```

따라서 이전 문서의 조건부는 해제한다. 단, 아래 항목은 구현 안전장치로 남긴다.

```text
마스코트화 금지.
단순 힐러화 금지.
윤서의 장비화 금지.
보급소를 너무 빨리 안정된 집처럼 보이게 만들지 않기.
리오의 정품 복구/객실 초기화 축과 겹치지 않기.
```

## 1. 판정 기준

| 기준 | 판정 | 메모 |
|---|---|---|
| 윤서 1번과 대비되는가 | 통과 | 윤서는 사람/태그/이름의 회수자이고, PATCH는 로봇/부품/충전의 회수자다 |
| 초반 시스템 이해에 도움이 되는가 | 통과 | 충전태그, 수리재, 부품 회수, 인양 장비 보강을 자연스럽게 연다 |
| 전투 이미지가 직업형 힐러를 피하는가 | 통과 | 리벳, 자석 수리팔, 패치 테이프는 "치유"보다 결합/분리/임시 봉합에 가깝다 |
| 보급소를 안정된 기지로 오해시키지 않는가 | 조건부 안전장치 필요 | PATCH가 있을수록 버티지만, PATCH 자신이 줄어든다는 비용을 반드시 보여준다 |
| OPEN-HOST를 밀어낼 만큼 초반성이 있는가 | 통과 | OPEN-HOST는 R01 내부자라 먼저 의심받아야 하고, PATCH는 첫 보급소 귀환 직후 열 수 있다 |
| 리오와 로봇 축이 분리되는가 | 통과 | PATCH는 자기부품화, 리오는 정품 복구와 흔적 삭제 공포다 |
| HP 0/긴급 인양이 다른가 | 통과 | 윤서는 좌표 회수, PATCH는 부품 목록화 직전 회수, 리오는 리콜 직전 회수다 |

판정:

```text
PATCH는 2번 슬롯에 충분하다.
보완 조건은 후보 자격 조건이 아니라 구현 금지선으로 관리한다.
```

## 2. PATCH가 2번 플레이어블로 적합한 이유

| 이유 | 설명 | 게임 내 효과 |
|---|---|---|
| 보급소의 두 번째 얼굴 | 윤서가 밖으로 나가는 사람이라면 PATCH는 보급소가 어떻게 버티는지 보여준다 | 첫 귀환 후 보급소가 메뉴가 아니라 생활 공간이 된다 |
| 로봇 자율성의 빠른 개방 | VAC-0는 R01 NPC 씨앗이고 리오는 중반 확장이다. PATCH는 초반에 로봇 자율성을 플레이어블로 열 수 있다 | 로봇을 적/장비/마스코트가 아니라 동료로 읽게 한다 |
| 수리와 분해의 양면성 | PATCH는 고쳐주는 캐릭터지만, 고치려면 무엇을 떼어낼지 결정해야 한다 | 전투, 정산, 성장에서 비용의 감각이 생긴다 |
| 충전태그의 감정화 | 충전태그가 단순 에너지 자원이 아니라 로봇이 남아 있을 조건이 된다 | 자원 UI가 세계관 압력으로 읽힌다 |
| 윤서 성장의 상대축 | 윤서가 사람을 물건처럼 보지 않으려 한다면 PATCH는 로봇도 부품만은 아니라는 기준을 요구한다 | 윤서의 회수 윤리가 초반부터 넓어진다 |

핵심 문장:

```text
고쳐주는 것과,
부품이 되는 것은 다르다.
```

이 문장이 전투, 정산, 긴급 인양, 대사에 모두 붙으면 PATCH는 2번으로 통과한다.

## 3. PATCH가 2번이면 생기는 위험

| 위험 | 실제 문제 | 처리 기준 |
|---|---|---|
| 윤서의 감정선을 너무 빨리 로봇 쪽으로 넘김 | R01의 사람/이름/주거 질문보다 로봇 자율성이 먼저 커질 수 있다 | PATCH는 R01 주인공이 아니라 보급소 두 번째 얼굴로 시작한다 |
| 보급소가 너무 빨리 안정되어 보임 | 수리 로봇이 있으니 보급소가 안전한 기지처럼 보일 수 있다 | PATCH가 고칠수록 PATCH의 부품 부담과 충전 순번 문제가 보인다 |
| 힐러/수리공으로 축소 | 성능 역할만 남고 자기부품화가 사라진다 | 회복량보다 임시 봉합, 결합부 약화, 자기 부품 부담을 강조한다 |
| 귀여운 마스코트화 | 작고 정중한 로봇의 상품성이 상처를 덮을 수 있다 | 귀여워도 되지만 상처를 개그로만 소비하지 않는다 |
| 리오 8번과 중복 | 로봇 둘이 모두 정품 복구가 싫은 캐릭터처럼 보일 수 있다 | PATCH는 부품화, 리오는 객실 초기화/정품 복구/흔적 삭제로 분리한다 |
| OPEN-HOST 배치 약화 | R01 내부자 관점이 밀리면 R01 플레이어블 전환 후보가 흐려질 수 있다 | OPEN-HOST는 2번 대체에서 내려오고 3~7번 또는 후속 전환 후보로 유지한다 |

위험 처리 결론:

```text
위험은 크지만 모두 관리 가능하다.
위험이 있다는 이유로 PATCH를 보류하기보다,
위험을 PATCH의 플레이 조건으로 쓰는 쪽이 더 강하다.
```

## 4. 윤서 1번과의 대비

| 항목 | 윤서 | PATCH |
|---|---|---|
| 첫 역할 | 현장 회수자 | 보급소 수리 로봇 |
| 세계를 읽는 방식 | 잘못 붙은 이름, 태그, 흔적을 보류한다 | 고장난 것과 아직 쓸 수 있는 것을 구분한다 |
| 캠페인이 붙이는 역할 | 반품품, 회수 대상, 잘못 붙은 이름 | 정품 부품, 리콜 대상, 비인가 수리 장치 |
| 전투 이미지 | 반품 스캐너, 반품 도장, 회수선 릴 | 자석 수리팔, 응급 패치 테이프, 리벳 발사기 |
| 정산 감각 | 무엇을 가져오고 무엇을 남길지 | 무엇을 고치고 무엇을 떼어내지 않을지 |
| HP 0 위험 | 현장 좌표에서 물건처럼 회수될 위험 | 이름이 지워지고 부품 목록으로 분류될 위험 |
| 핵심 질문 | 사람은 물건이 아닌데 왜 자꾸 반품되는가 | 로봇은 부품이 아닌데 왜 쓸모 있게 분해되는가 |

윤서와 PATCH는 서로를 반복하지 않는다. 윤서가 첫 질문을 열고 PATCH가 그 질문을 비인간 동료까지 확장한다.

## 5. OPEN-HOST 2번 대체 가능성과 비교

OPEN-HOST는 강한 후보다. 하지만 2번 슬롯에는 PATCH가 더 적합하다.

| 항목 | PATCH 2번 | OPEN-HOST 2번 |
|---|---|---|
| 초반 진입성 | 첫 귀환 직후 보급소에서 자연스럽게 열린다 | R01 내부 경험과 의심 과정이 먼저 필요하다 |
| 플레이어 감정 | "이 로봇도 남겨야 한다"가 빠르게 붙는다 | "이 사람을 믿어도 되는가"가 먼저 와야 한다 |
| R01 기능 보존 | R01 NPC 12명의 기능을 빼앗지 않는다 | 너무 빨리 플레이어블이 되면 안내/동선/NPC 긴장이 약해진다 |
| 시스템 연결 | 충전태그, 부품 회수, 수리 벤치, 긴급 인양 장비 | 통행태그, 거주태그, 상담표, 안내 동선 |
| 위험 | 힐러화, 마스코트화 | R01 설명 NPC화, 자기 책임 약화 |
| 최종 배치 | 2번 확정 | 3~7번 또는 후속 플레이어블 전환 후보 유지 |

OPEN-HOST를 깎아내리지 않는다. 오히려 OPEN-HOST는 너무 좋은 R01 내부자이기 때문에 2번으로 당기면 손해다. 그는 먼저 NPC로 의심받고, 도움을 주고, 불편함을 남긴 뒤 플레이어블 전환 후보가 되어야 한다.

판정:

```text
OPEN-HOST는 2번 대체 후보에서 내려온다.
PATCH 통과 이후 OPEN-HOST는 3~7번 후보 또는 R01 후속 전환 후보로 유지한다.
```

## 6. 리오 8번 보류와의 차이

리오는 8번 보류를 유지한다. PATCH가 2번으로 통과해도 리오의 자리는 약해지지 않는다.

| 항목 | PATCH | 리오 |
|---|---|---|
| 출신 | 침묵 보급소, 수리/정품 부품 잔향 | 체류/숙박 캠페인권 |
| 캠페인이 붙이는 역할 | 정품 부품, 리콜 대상, 수리 자재 | 정품 객실 초기화 장치, 체크아웃 후 정리 장치 |
| 핵심 공포 | 쓸모 있게 분해되는 것 | 정품 복구와 다시 흔적을 지우는 것 |
| 핵심 결함 | 자기 자신을 부품 창고처럼 계산한다 | 자기 충전과 자기 보존을 뒤로 미룬다 |
| 전투 감각 | 붙이고, 분리하고, 임시로 막는다 | 흡입하고, 청소하고, 서비스 제공을 거부한다 |
| 출시 위치 | 윤서 이후 초반 2번 | 여러 캠페인권 이해 후 중반 8번 |

차이의 핵심:

```text
PATCH는 "저는 아직 한 대입니다"를 배운다.
리오는 "이 방은 비어 있지 않았습니다"를 지킨다.
```

## 7. VAC-0 / PATCH / 리오 로봇 자율성 축 분리

| 구분 | VAC-0 | PATCH | 리오 |
|---|---|---|---|
| 위치 | R01 빈 집 내부 NPC | 침묵 보급소 플레이어블 2번 | 중반 체류/숙박 캠페인권 플레이어블 후보 |
| 첫 상태 | 흔적과 위험물을 구분하지 못한다 | 자기 몸을 부품 자원으로 본다 | 비어 있지 않은 방을 비웠다고 처리하지 않는다 |
| 자율성의 시작 | "이것도 치워야 합니까?" | "이 부품은 제 몸입니까?" | "해당 서비스는 제공하지 않습니다" |
| 주 상태값 | 로봇 자율성, 충전태그, 흔적 보존 | 충전태그, 부품 회수, 로봇 자율성 | 충전태그, 흔적 보존, 정품 복구 위험 |
| 금지 | 귀여운 청소 로봇화 | 힐러/마스코트화 | 인간이 되고 싶은 로봇화 |

정리:

```text
VAC-0는 질문이 생기는 장치다.
PATCH는 자기 몸을 남기는 법을 배우는 플레이어블이다.
리오는 자신이 지웠던 흔적과 정품 복구를 거부하는 중반 주인공이다.
```

## 8. 전투/무기 이미지

PATCH는 전투에서 사람을 치료하는 힐러가 아니다. PATCH는 캠페인 절차의 결합부를 임시로 풀고, 위험한 작동을 잠깐 막고, 쓸 수 있는 조각을 끌어와 보급소가 하루 더 버티게 한다.

| 전투 요소 | 기능 | 감정 비용 |
|---|---|---|
| 리벳 샷 | 결합부, 광고 태그, 장치 연결부를 흔든다 | 무언가를 부수기보다 임시 분리한다 |
| 자석 수리팔 | 부품, 태그 조각, 회수 가능한 물체를 끌어온다 | 무엇이 회수물이고 무엇이 남아야 할 것인지 묻는다 |
| 응급 패치 | 캠페인 장치 작동을 잠시 봉합한다 | 강하게 쓸수록 PATCH의 외장과 충전 부담이 쌓인다 |
| 자기 부품 부담 | 강한 수리/봉합/보강 사용 시 부품 유실 위험이 오른다 | 유저가 PATCH를 효율 도구로 쓰는 순간 불편함이 생긴다 |

전투 리듬:

```text
빠른 제압보다 임시 봉합.
강한 회복보다 위험 작동 보류.
무한 수리보다 자기 부품 부담 관리.
```

PATCH가 전투에서 고치는 것:

```text
끊어진 회수선
불안정한 충전 접점
과열된 보급 장치
캠페인 장치의 결합부
인양 장비의 임시 고정점
```

PATCH가 전투에서 소모하는 것:

```text
비정품 배터리 잔량
응급 패치 테이프
자기 외장 패널 안정도
작은 나사와 리벳
정품 추적 위험을 대신 받는 여유
```

## 9. 정산 루트

PATCH의 정산은 돈이나 전리품이 아니라 재사용 가능성과 분해 위험의 판단이다.

| 정산 항목 | 역할 | 위험 |
|---|---|---|
| 충전태그 | PATCH와 다른 로봇/장비가 남아 있을 전력 접근권 | 높을수록 정품 추적과 자동화 압력도 강해질 수 있다 |
| 부품 회수 | 고장난 장비를 다시 쓰기 위한 조각 | 누군가의 흔적이나 몸을 부품으로 읽을 수 있다 |
| 자기 부품 소모 | 강한 수리나 응급 패치의 대가 | PATCH가 한 대의 로봇이 아니라 부품 창고처럼 보이기 시작한다 |
| 자기 부품 복구 | 부품 유실 뒤 보급소 수리 벤치에서 복구 | 복구가 정품화로 흘러가지 않게 비정품 기준을 유지해야 한다 |
| 보급소 수리 벤치 | 수리, 충전, 인양 장비 보강의 허브 | PATCH를 편리한 장비 메뉴로만 만들면 실패한다 |

정산 문구 기준:

```text
사용 가능.
```

## 10. HP 0 / 긴급 인양 차이

PATCH가 HP 0이 되면 죽거나 폭발하지 않는다. 캠페인은 PATCH를 한 대의 로봇으로 회수하지 않고, 쓸 수 있는 부품 묶음으로 분류하려 한다.

| 캐릭터 | HP 0 위험 | 긴급 인양 차이 |
|---|---|---|
| 윤서 | 현장 회수자가 물건처럼 회수될 위험 | 회수선이 마지막 좌표에서 윤서를 당긴다 |
| PATCH | 이름이 지워지고 사용 가능한 부품 목록으로 분류될 위험 | 보급소 수리선이 중심 프레임을 당기고 일부 부품 유실/충전 손실이 남는다 |
| OPEN-HOST | R01 오픈하우스 호스트로 다시 고정될 위험 | 안내 배지 끈이 끊기며 보급소가 당긴다 |
| 리오 | 정품 충전독과 객실 초기화 절차로 돌아갈 위험 | 리콜 직전 보급소 케이블이 역으로 당긴다 |

PATCH 전용 결과 문구 기준:

```text
[긴급 인양]

PATCH 인양 완료.
부품 유실.
정품 추적 신호 잔류.
충전 손실.

PATCH:
"저는 아직 한 대입니다."
```

정품 복구/부품화 위험 위치:

```text
HP 0 직전 스캔 UI
긴급 인양 결과창
수리 벤치 복구 비용
충전태그 부족 상태
강한 응급 패치 반복 사용 후 경고
```

## 11. 최종 판정

```text
최종 판정:
통과

슬롯:
2번 플레이어블

상태:
잠금

이전 조건부:
해제

남는 조건:
후보 조건이 아니라 구현 금지선으로 관리한다.
```

판정 이유:

```text
PATCH는 윤서를 반복하지 않는다.
OPEN-HOST보다 초반 진입성이 높다.
리오보다 로컬하고 이해가 빠르다.
VAC-0와 달리 직접 조작 가능한 로봇 자율성 축을 연다.
충전태그, 부품 회수, 수리 벤치, 긴급 인양 장비를 감정적으로 연결한다.
```

점수:

```text
2번 슬롯 적합도:
9.2 / 10

구현 안전장치 준수 시:
9.6 이상 가능
```

## 12. 판정 이후 다음 작업

PATCH가 통과했으므로 다음 작업은 아래로 넘긴다.

```text
3~7번 초반 플레이어블 후보 선정
```

다음 작업에서 지켜야 할 기준:

```text
1. PATCH를 다시 검증하지 않는다.
2. 3~7번은 E01/R01/R02/R03/R04/R08 안에서 찾는다.
3. OPEN-HOST는 3~7번 후보 또는 R01 후속 전환 후보로 비교한다.
4. 리오는 8번 보류 잠금을 유지한다.
5. R01 NPC 기능을 플레이어블이 빼앗지 않게 한다.
6. 새 R01 NPC를 만들지 않는다.
```

## 13. 금지 방향

```text
PATCH를 귀여운 로봇 마스코트로만 쓰지 않는다.
PATCH를 힐러/수리공 직업 슬롯으로 축소하지 않는다.
PATCH를 윤서의 펫이나 장비로 만들지 않는다.
PATCH의 자기부품화 상처를 개그로만 소비하지 않는다.
PATCH가 있으면 보급소가 안전해진다는 착시를 만들지 않는다.
PATCH와 리오를 "정품이 싫은 로봇" 하나로 뭉개지 않는다.
OPEN-HOST를 PATCH에게 밀린 실패 후보처럼 쓰지 않는다.
```

최종 잠금 문장:

```text
윤서가 사람은 물건이 아니라고 버틴다면,
PATCH는 로봇도 부품만은 아니라고 버틴다.
```
