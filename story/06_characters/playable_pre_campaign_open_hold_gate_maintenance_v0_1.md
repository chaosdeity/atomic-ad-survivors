# 플레이어블 캠페인 개방 전 보류 게이트 유지 v0.1

## 1. 최종 판정

보류 게이트 유지.

`launch_initial_playable_pointer_residual_conflict_patch_review_v0_1.md`의 통과 판정을 기준으로, 플레이어블 트랙은 새 후보 제작으로 넘어가지 않는다. 출시 초기 구조는 윤서 단독 U5로 닫고, 기존 후보/프로필/상세 결재 산출물은 기능 산출물 또는 후보 메모로만 보존한다.

## 3. 출시 초기 플레이어블 구조 요약

| 항목 | 유지 판정 | 운영 의미 |
|---|---|---|
| 출시 초기 U5 | 윤서 단독 시작 | 새 +1 후보를 제작하지 않는다. |
| 현재 +1 | 미확정 | `한국인 기본 주인공 +1~2`는 최대치일 뿐 현재 +1 확정이 아니다. |
| PATCH-02 | legacy_code / 산출 번호 보존, U1~U4 보류 | 출시 초기 +1, 2번 해금 고정, 로봇 초반 합류로 읽지 않는다. |
| OPEN-HOST | NPC 선등장 유지 | 출시 초기 U5 전환 금지. R01 NPC 기능을 먼저 보존한다. |
| HOLD / RETURN / METER / QUIET | 출시 후 보류 | 기존 프로필 v1.0은 기능 산출물 보존이며 출시 초기 해금 확정이 아니다. |
| 미개방 캠페인 후보 | 캠페인 개방 전 상세 결재 금지 | MODEL / LOCKOUT / NAME / TICKET / RATION / LINE / OBS 등은 캠페인 개방 전 진행하지 않는다. |
| E02 이후 이름 원칙 | 한국어 이름 기본값 금지 | 외부 광역권 후보는 현실권/언어권 재검토 전 한국어 이름으로 빈 슬롯을 채우지 않는다. |

## 4. 신규 제작 금지 목록

| 금지 항목 | 금지 내용 |
|---|---|
| 새 플레이어블 후보 | 새 이름, 새 코드명, 새 원점 캠페인, 새 해금 순번을 만들지 않는다. |
| 새 상세 결재 | 후보 상세 결재, 조건부 해소, 최종 판정 문서를 새로 시작하지 않는다. |
| 새 프로필 v1.0 | 기존 산출물 보존 외에 새 프로필 v1.0을 작성하지 않는다. |
| 순번 재배치 | 기존 번호를 최종 해금 순번으로 재해석하거나 재배치하지 않는다. |
| PATCH +1 확정 | PATCH-02를 출시 초기 U5 또는 +1 플레이어블로 확정하지 않는다. |
| OPEN-HOST 전환 | OPEN-HOST를 출시 초기 플레이어블로 전환하지 않는다. |
| HOLD / RETURN / METER / QUIET 당김 | 출시 초기 한국인 추가 후보로 당기지 않는다. |
| 미개방 캠페인 후보 진행 | MODEL / LOCKOUT / NAME / TICKET / RATION / LINE / OBS 등을 캠페인 개방 전 상세화하지 않는다. |
| E02 이후 한국어 기본명 | 외부 광역권 후보를 한국어 이름 기본값으로 복구하지 않는다. |
| NPC 문서 변경 | 플레이어블 보류 게이트 유지를 이유로 NPC 문서를 수정하지 않는다. |

## 6. 캠페인 개방 전 재개 금지 조건

아래 조건 중 하나라도 남아 있으면 플레이어블 신규 제작을 재개하지 않는다.

| 조건 | 금지 판정 |
|---|---|
| 해당 원점 캠페인이 실제로 개방되지 않았다. | 재개 금지 |
| 스토리/레벨/NPC 구조에서 플레이어블 필요성이 확인되지 않았다. | 재개 금지 |
| 후보가 기존 NPC 기능, 윤서 기능축, PATCH 기능축, 캠페인 절차 기능을 침범한다. | 재개 금지 |
| 현실권/언어권/이름 원칙 재검토가 끝나지 않았다. | 재개 금지 |
| 기존 프로필 v1.0을 해금 확정으로 오독할 수 있다. | 재개 금지 |
| 후보 재검토 문서 없이 상세 결재 또는 프로필 작성을 시작하려 한다. | 재개 금지 |
| PATCH +1, OPEN-HOST U5 전환, HOLD/RETURN/METER/QUIET 출시 초기 당김을 전제로 한다. | 재개 금지 |

## 7. 재개 가능 조건

플레이어블 후보 재개는 아래 조건을 모두 만족할 때만 가능하다.

| 조건 | 필요 확인 |
|---|---|
| 해당 캠페인이 실제로 개방된다. | 원점 캠페인의 스토리/레벨/NPC 구조가 열린 상태여야 한다. |
| 캠페인별 플레이어블 필요성이 확인된다. | 플레이어블이 없으면 해결되지 않는 구조적 필요가 있어야 한다. |
| 기존 후보가 해당 캠페인의 현실권/언어권/기능축을 침범하지 않는다. | 이름, 출신권, NPC 기능, 월드 절차 기능을 다시 검토한다. |
| 별도 `후보 재검토` 문서를 먼저 작성한다. | 새 상세 결재 전에 보류 해제 사유와 금지선 준수 여부를 문서화한다. |
