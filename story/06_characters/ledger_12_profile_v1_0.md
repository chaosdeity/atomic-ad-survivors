## 0. 캐릭터 핵심

라파는 상인, 블랙마켓 보스, 도적, 밀수꾼 영웅, 환전상, 태그 교환 상점, 돈벌이 캐릭터가 아니다.

라파는 태그를 돈처럼 돌리던 비공식 장부를, 보급소 규칙과 미정산 책임으로 되접는 12번 플레이어블 후보다.

핵심 질문:

```text
태그를 돈처럼 팔던 사람이
보급소 규칙을 믿을 수 있는가?
```

핵심 문장:

```text
장부에 적었다고 해서,
사람이 정산된 것은 아니다.
```

최종 변화 문장:

```text
태그는 가격표가 아니라,
누가 버려졌는지 남기는 기록이다.
```

잘못 믿는 것:

```text
라파는 장부가 맞으면
거래도 공정해질 수 있다고 믿는다.
```

플레이어블로서 다루는 질문:

```text
비공식 장부가 사람을 거래 항목으로 닫기 전,
태그를 아직 미정산 책임으로 남길 수 있는가?
```

라파의 조작은 이익을 키우는 방식이 아니라, 이미 거래 완료처럼 닫힌 태그를 다시 "아직 누군가의 몫과 책임이 남아 있는 기록"으로 되돌리는 방식이다.

## 2. 유저용 첫 소개문

라파, 항만시장 태그 중개자.

라파는 물건을 싸게 파는 사람이 아니다. 그는 버려진 물건, 등록되지 않은 접근권, 공식 절차가 회수하지 못한 태그가 장부 위에서 잠깐 살아남는 방식을 배운 사람이다.

그 장부는 누군가에게는 구멍이고, 누군가에게는 숨 쉴 틈이었다. 그러나 장부가 닫히는 순간, 사람도 물건처럼 거래 완료 항목이 된다.

보급소가 잡은 것은 돈이 아니라 찢긴 장부 페이지와 가격이 지워진 태그 홀더의 빈칸이다. 라파가 합류하려면 태그를 더 잘 돌리는 사람이 되는 것이 아니라, 아직 정산되지 않은 책임을 남길 수 있어야 한다.

## 4. 플레이어블 기능

| 기능 | 의미 | 실패 위험 |
|---|---|---|
| 미정산 태그 보류 | 정산 완료로 닫히려는 태그를 아직 책임이 남은 상태로 둔다 | 보류가 길어지면 라파 이름 아래 미정산 태그가 묶인다 |
| 오염 태그 기록 분리 | 오염 기록을 지우지 않고, 누가 버려졌는지 읽을 수 있는 단서로 남긴다 | 오염 기록이 라파의 장부 보관 책임으로 고정된다 |
| 거래 장부 페이지 되접기 | 이미 닫힌 거래 항목을 다시 보급소가 읽을 수 있는 페이지로 접는다 | 페이지가 붙으면 거래 완료 항목으로 봉인된다 |
| 가격표가 된 태그 지우기 | 태그를 가격표가 아니라 접근 조건과 책임 기록으로 되돌린다 | 가격표가 다시 붙으면 태그가 아이템 가치로만 읽힌다 |
| 비공식 회수품 표식 보류 | 회수품이 거래 대상이 되기 전, 누가 잃었는지 남긴다 | 표식이 닫히면 회수품 유통자로 등록된다 |
| 장부의 빈칸 남기기 | 보급소가 인양할 수 있는 미정산 빈칸을 일부러 남긴다 | 빈칸이 채워지면 라파의 이름으로 책임이 고정된다 |

라파의 기능은 보급을 늘리는 것이 아니다. 태그가 돈처럼 닫히는 순간을 늦춰 보급소가 아직 사람과 책임을 읽을 수 있게 하는 것이다.

| 2 | 가격표가 된 태그를 지운다 | 태그가 회수품 가격 대신 접근 조건으로 흔들린다 |
| 3 | 장부 페이지를 되접어 보급소가 읽을 빈칸을 만든다 | 장부가 닫히며 라파 이름을 고정하려 한다 |

플레이어 조작감:
라파는 자원을 많이 얻는 캐릭터가 아니다. 이미 정산된 척 닫힌 기록에서 아직 회수선이 붙을 수 있는 빈칸을 찾고, 그 빈칸을 너무 오래 붙잡으면 자신이 장부 보관자로 고정되는 위험을 감수한다.

파티/동행 시 역할:
윤서가 현장에서 잘못 붙은 태그를 회수한다면, 라파는 이미 가격표가 붙은 태그를 미정산 책임으로 되돌린다. 미나는 보급소 안에서 정산하고, 라파는 보급소 바깥에서 정산된 척하던 기록을 다시 열어 둔다.

## 7. HP 0 / 긴급 인양 차별화

핵심 상태명:

```text
거래 장부 이름 고정 / 미정산 태그 폐쇄
```

HP 0 흐름:

| 단계 | 상태 | 처리 |
|---:|---|---|
| 1 | 장부 페이지 닫힘 | 라파가 남긴 빈칸이 거래 항목으로 채워진다 |
| 2 | 미정산 태그 폐쇄 | 아직 책임으로 남아야 할 태그가 정산 완료로 닫힌다 |
| 3 | 오염 태그 묶음 | 오염 기록이 라파 이름 아래 정리된다 |
| 4 | 거래 장부 이름 고정 | 라파가 항만시장 중개자/장부 보관자로 확정된다 |

비교:

| 대상 | HP 0 감각 | 라파와 다른 점 |
|---|---|---|

## 13. 성장 방향

초기:
라파는 장부가 맞으면 거래도 공정해질 수 있다고 믿는다. 공식 시스템이 버린 사람과 물건은 장부 위에서 잠깐 살아남았고, 라파는 그 방식을 너무 오래 믿어 왔다.

중반:
라파는 장부가 사람을 살려 두기도 하지만, 동시에 사람을 거래 완료 항목으로 닫을 수도 있다는 것을 배운다. 장부의 정확함은 책임을 남길 수도 있고, 책임을 지워 버릴 수도 있다.

후반:
라파는 태그를 없애거나 더 비싸게 돌리는 것이 아니라, 가격표로 닫히기 전의 미정산 책임을 남기는 법을 배운다. 보급소 규칙은 장부를 대체하는 법이 아니라, 장부가 닫아 버린 사람을 다시 읽는 법이 된다.

최종 변화 문장:

```text
태그는 가격표가 아니라,
누가 버려졌는지 남기는 기록이다.
```
