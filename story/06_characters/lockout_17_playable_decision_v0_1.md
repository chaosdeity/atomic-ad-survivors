## 0. 판정 요약

판정:

```text
캠페인 미개방으로 대기
```

Omar Al-Hadid / LOCKOUT-17 / 오마르, 산업 안전 미승인자는 E06_C04 에너지 정품충전권의 플레이어블 후보로 검토할 가치가 있다. 단, E06_C04는 현재 미개방 캠페인이므로 이 문서는 최종 상세 결재가 아니라 기능 초안/보류 산출물로만 취급한다.

이 후보는 전기 마법사, 엔지니어, 충전 버퍼, 산업 전사, 안전요원, 보디가드가 아니다.

핵심은 E06_C04가 사람을 `작업자`, `안전교육 대상`, `충전 승인 대기자`, `작업자 ID`, `정품 충전 접근 대상`으로 등록하려는 순간을, `안전잠금`과 `작업 허가 미승인`의 틈에서 늦추는 데 있다.

후보 보존 이유:

1. E06 걸프 스마트시티-에너지 벨트의 첫 플레이어블 관점으로, 1~15번의 한국/E02/E07/E09/E10/E11/E12 분포 뒤 전역 확장을 넓힌다.
2. PATCH의 자기부품화, METER-06의 계량당하는 인간, RIO-08의 정품 복구 공포와 다른 `안전잠금/작업자 ID/충전 접근 차단` 절차를 연다.
3. HP 0을 전투 사망이 아니라 `작업자 ID 잠금`, `안전교육 미승인 확정`, `충전 접근 차단`, `에너지 게이트 격리`로 차별화할 수 있다.
4. legacy 한국어권 임시명 남태건을 폐기하고 E06 현실권/언어권 후보로 전환한다.

보류 사유:

1. PATCH/METER-06/RIO-08과 정품/충전/수리 축이 겹치기 쉽다.
2. 오마르가 전기 마법사, 엔지니어, 충전 버퍼, 산업 노동자 영웅으로 새기 쉽다.
3. E06_C04를 열면 걸프 스마트시티/에너지/몰/VIP/공항권이 너무 빨리 커질 수 있다.
4. 이름과 언어권이 현실 문화권 장식으로 소비될 위험이 있다.
5. 14~20 글로벌 이름/원점 재검토에서 이미 `캠페인 미개방으로 대기`였으므로 프로필 v1.0 고정으로 읽으면 안 된다.

## 4. 캠페인이 등록하려는 역할

E06_C04 에너지 정품충전권이 오마르를 등록하려는 역할:

```text
작업자
안전교육 대상
작업자 ID
충전 승인 대기자
정품 충전 접근 대상
에너지 게이트 출입자
위험구역 잔류자
잠금태그 책임자
```

오마르가 비트는 절차:

```text
교육 대기 -> 작업자 ID 발급 -> 게이트 승인 -> 정품 충전 접근 -> 잠금태그 부착 -> 작업자 책임 고정
```

이 절차는 사람을 죽이려는 악의가 아니라, 위험한 설비 안에서 사람을 보호하고 사고를 막기 위해 출발한다. 문제는 안전 기준이 사람의 이동과 접근권을 모두 잠그는 순간이다.

## 5. 핵심 오판

초안:

```text
안전 승인만 받으면 공정하게 일하고 충전받을 수 있다고 믿음.
```

결재안:

```text
작업자 ID가 발급되면,
그 ID가 자신을 보호한다고 믿는다.
```

이 믿음은 일부 맞다. 위험한 에너지 설비에서 안전 절차와 작업자 ID는 사람을 보호한다. 그러나 E06_C04는 그 ID를 보호 장치이자 소유 장치로 동시에 사용한다. ID가 닫히면 오마르는 아직 판단 중인 사람이 아니라 `잠금태그 책임자`, `위험구역 잔류자`, `충전 접근 차단 대상`으로 고정된다.

더 깊은 오류:

- 잠긴 게 안전한 상태라고 믿는다.
- 승인 절차가 닫히면 책임도 공정하게 배분된다고 믿는다.
- 작업 허가가 없으면 아직 등록되지 않은 상태라고 믿는다.
- 하지만 미승인 상태도 캠페인은 `대기 중인 작업자`로 읽는다.

통과 조건:

```text
오마르는 안전 절차를 무시하거나 부수는 캐릭터가 아니다.
오마르는 안전잠금과 작업자 ID가 사람을 책임 항목으로 닫는 순간을 늦추고,
비상정지줄과 잠금태그의 불일치를 보급소가 읽을 수 있게 남기는 관점이어야 한다.
```

## 7. 플레이어블이어야 하는 이유

오마르는 NPC나 로그로도 작동할 수 있다. 에너지 게이트 옆의 작업 허가표, 잠금태그가 붙은 배터리 팜, 안전교육 미승인 방송으로도 E06_C04의 분위기는 설명할 수 있다.

하지만 플레이어블로 검토할 가치가 있는 이유는 오마르의 핵심이 정보 제공이 아니라 `잠금과 미승인 판정 조작`이기 때문이다.

플레이어블일 때 생기는 조작감:

1. 어떤 게이트를 승인 전 상태로 남길지 선택한다.
2. 잠금태그를 떼는 것이 아니라, 태그가 붙은 책임 항목을 아직 닫히지 않은 상태로 둔다.
3. 비상정지줄을 당기면 적 호출을 끊을 수 있지만, 오마르의 작업자 ID 고정 압력이 올라간다.
4. 유저가 "안전절차 무시"가 아니라 "안전절차가 사람을 닫지 못하게 보류"를 조작한다.

NPC로 남을 때의 한계:

```text
NPC 오마르는 안전잠금이 위험하다고 말할 수 있다.
플레이어블 오마르는 잠금과 승인 사이의 판정을 직접 늦추게 한다.
```

## 8. 전투 리듬

기본 리듬:

```text
안전잠금 / 작업 허가 / 게이트 승인 / 비상정지 / 충전 접근 차단 판정을 밀고 당긴다.
```

오마르는 엔지니어나 전기 마법사가 아니다. 에너지를 쏘거나 충전량을 나눠주는 캐릭터가 아니다. 오마르의 리듬은 에너지 설비가 사람을 작업자 ID와 잠금태그 책임으로 고정하기 전, 아직 보급소가 읽을 수 있는 비상정지 신호와 미승인 로그를 남기는 것이다.

전투 동사:

```text
잠근다
보류한다
끊는다
격리한다
정지시킨다
미승인으로 남긴다
잠금태그를 접어 둔다
작업자 ID 고정을 늦춘다
에너지 게이트 승인을 지연한다
충전 접근 차단을 역이용한다
```

금지 동사:

```text
발전한다
수리한다
충전한다
강화한다
해킹한다
폭파한다
감전시킨다
전기를 쏜다
장비를 제작한다
정품 충전을 공급한다
```

위험한 강점:

| 강점 | 효과 | 위험 |
|---|---|---|
| 잠금태그 보류 | 적 장치나 게이트의 활성 판정을 늦춘다 | 보류가 길면 오마르가 잠금태그 책임자로 고정된다 |
| 비상정지줄 당김 | 특정 웨이브/장치의 행동을 끊는다 | 오마르의 작업자 ID 호출이 빨라진다 |
| 작업 허가 미승인 | 위험구역 접근을 잠깐 열어 둔다 | 미승인 대기 상태도 캠페인 등록값으로 누적된다 |
| 충전 접근 차단 역이용 | 정품 충전 절차를 적의 경로 차단으로 돌린다 | METER-06의 계량/충전 차단 축과 섞일 위험이 있다 |
| 에너지 게이트 지연 | 전장 이동과 장치 작동 순서를 한 박자 늦춘다 | E06 전체 스마트시티 안내자로 커질 위험이 있다 |

## 10. HP 0 / 긴급 인양 차별화

오마르의 HP 0은 사망/부활이 아니다.

핵심 상태명:

```text
작업자 ID 잠금 / 안전교육 미승인 확정 / 충전 접근 차단
```

HP 0 흐름:

| 단계 | 상태 | 연출 |
|---|---|---|
| 1 | 안전교육 미승인 확정 | 미승인 상태가 대기 중이 아니라 고정 상태로 닫힌다 |
| 2 | 에너지 게이트 격리 | 게이트가 열리는 것이 아니라 오마르를 위험구역 안에 남긴다 |
| 3 | 충전 접근 차단 | 정품 충전 접근이 차단되고, 접근권이 작업자 ID에 묶인다 |
| 4 | 작업자 ID 잠금 | 오마르의 이름 대신 작업자 ID와 잠금태그 번호가 표시된다 |
| 5 | 긴급 인양 | 보급소가 비상정지줄과 작업 허가 보류표의 불일치를 잡아 되당긴다 |

긴급 인양 문구 후보:

```text
[긴급 인양]
LOCKOUT-17 인양 시도.
작업자 ID 잠금 직전.
비상정지줄 신호가 아직 끊기지 않았습니다.
```

차별점:

| 캐릭터 | HP 0 감각 | LOCKOUT-17과의 차이 |
|---|---|---|
| PATCH | 부품 목록화, 정품 부품 회수 | 오마르는 로봇 부품이 아니라 작업자 ID와 안전잠금으로 고정된다 |
| METER-06 | 과방전, 계량 초과, 정품 도크 회수 직전 | 오마르는 사용량이 아니라 작업 허가와 잠금태그 책임으로 고정된다 |
| RIO-08 | 정품 복구, 객실 초기화, 체크아웃 처리 | 오마르는 객실 상태가 아니라 에너지 게이트와 작업자 ID로 고정된다 |
| MODEL-15 | 모델 학습 완료, 광고 타깃 확정 | 오마르는 추천 프로필이 아니라 안전교육/작업 허가 상태로 닫힌다 |

```text
HP 0을 감전사, 폭발, 장비 고장, 엔지니어 실패로 처리하지 않는다.
```
