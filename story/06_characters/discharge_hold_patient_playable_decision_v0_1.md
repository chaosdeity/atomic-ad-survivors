## 0. 판정 요약

판정:

```text
조건부 통과
```

이 후보는 4번 슬롯에 둘 가치가 있다. 윤서/PATCH/OPEN-HOST와 전투 리듬이 뚜렷하게 다르고, R02 의료/보험 캠페인의 핵심 질문을 초반에 가장 선명하게 보여준다.

다만 바로 v1.0 프로필로 가지 않는다. 아래 조건이 먼저 잠겨야 한다.

1. 단순 힐러가 아니어야 한다.
2. 병약 미소녀, 비극 환자, 순한 피해자 클리셰로 가지 않는다.
3. R02 전체를 초반에 열지 않는다.
4. 의료 캠페인은 사람을 죽이려는 병원이 아니라, "아직 퇴원하면 안 되는 사람"으로 보존하려는 환경이어야 한다.
5. HP 0은 사망/부활이 아니라 재입원, 퇴원 취소, 보험 심사 재개로 처리한다.

핵심 질문:

```text
치료가 끝나지 않았다는 말은 보호인가, 붙잡기인가?
```

## 1. 캐릭터 핵심

| 항목 | 결재안 |
|---|---|
| 후보 슬롯 | 4번 |
| 임시명/역할명 | 퇴원 보류 환자 |
| 출신 축 | R02 의료/보험/퇴원 심사 캠페인권 |
| 캠페인이 등록하려는 역할 | 환자, 보호자 미확인자, 보험 심사 대기자, 퇴원 불가 대상 |
| 플레이어블 검토 목적 | R02를 "치료하는 지역"이 아니라 "살려 두는 방식으로 붙잡는 지역"으로 읽게 한다 |
| 최종 프로필 여부 | 아직 아님. 성별, 나이, 본명, 말투, 외형은 확정하지 않는다 |

이 후보는 병원에서 탈출한 환자가 아니다. 더 정확히는, 퇴원 승인과 입원 연장 사이에서 계속 "보류"되는 사람이다.

캠페인은 이 인물을 죽이려 하지 않는다. 캠페인은 계속 살려 두려 한다. 다만 그 생존 조건이 병상, 팔찌, 보호자 확인, 보험 심사, 약품 접근, 퇴원 승인으로 묶여 있다. 이 후보는 그 절차가 자신을 보호하는지, 아니면 자신을 환자 역할에 고정하는지 의심하기 시작한 사람이다.

## 2. 어떻게 붙잡혔는가

캠페인이 붙잡는 방식:

| 절차 | 붙잡는 방식 | 후보가 느끼는 압력 |
|---|---|---|
| 환자 등록 | 몸 상태보다 환자 번호가 먼저 열린다 | "나는 사람인가, 병상 번호인가" |
| 보호자 미확인 | 보호자가 확인되지 않으면 퇴원 결정이 미뤄진다 | "나를 대신 승인할 사람이 없으면 나는 나갈 수 없는가" |
| 보험 심사 대기 | 치료 완료보다 심사 완료가 더 중요해진다 | "완치보다 정산이 먼저인가" |
| 병상 점유 | 병상이 안전하다는 이유로 이동이 금지된다 | "안전한 자리가 곧 감금이 되는가" |
| 퇴원 보류 | 퇴원하면 위험하다는 판정이 계속 갱신된다 | "위험하다는 말은 누구를 위한 말인가" |

흡수되지 않은 이유:

- 퇴원 불가 판정이 확정되지 않고 계속 보류된다.
- 보호자 확인이 끝나지 않아 절차가 닫히지 않는다.
- 보험 심사 기록이 완결되지 않아 캠페인이 이 인물을 완전히 장기 환자로 묶지 못한다.
- 본인은 이 미완료 상태를 약점이 아니라 틈으로 쓰기 시작했다.

따라서 이 후보의 생존 방식은 "치료를 거부한다"가 아니다.

```text
완치도 입원도 확정되지 않은 상태를 이용해,
캠페인이 자신을 한 역할로 잠그는 순간을 늦춘다.
```

## 3. 잘못 믿는 것

초안:

```text
완치 판정이 나야 나갈 수 있다고 믿는다.
```

결재안:

```text
퇴원 승인 없이 나가면 살아남은 것이 아니라고 믿는다.
```

이 문장이 더 낫다. "완치"만 믿으면 몸이 약한 캐릭터나 치료 집착 캐릭터로 좁아진다. 반면 "퇴원 승인"을 믿으면 이 세계의 핵심인 절차, 태그, 정산, 승인, 보존의 문제와 바로 연결된다.

더 깊은 오류:

- 몸이 버틸 수 있는지보다 기록이 허락했는지를 더 믿는다.
- 안전한 병상과 자유로운 삶을 같은 것으로 착각한다.
- 보류 상태가 자신을 살려 두었다는 사실 때문에, 보류 상태를 완전히 버리면 위험해질 것이라고 생각한다.

통과 조건:

```text
이 후보는 치료를 싫어하는 사람이 아니다.
승인이 없으면 삶도 시작되지 않는다고 배운 사람이어야 한다.
```

## 4. 윤서와의 차이

| 비교축 | 윤서 | 퇴원 보류 환자 |
|---|---|---|
| 기본 질문 | 이 사람을 물건/반품품으로 확정하지 않을 수 있는가 | 이 사람을 환자/퇴원 불가 대상으로 확정하지 않을 수 있는가 |
| 전투 리듬 | 이름, 라벨, 회수선, 보류 표시 | 팔찌, 진료표, 보험 심사, 퇴원 승인 |
| 구하는 방식 | 현장에서 회수하고 이름/흔적을 보류한다 | 입원/퇴원 임계값을 흔들어 확정 판정을 늦춘다 |
| 정산 감각 | 회수물, 흔적 보존, 보급태그 | 진료태그, 심사 기록, 약품/병상 접근 |
| HP 0 감각 | 마지막 좌표에서 회수선이 끌어온다 | 재입원/퇴원 취소가 걸리기 전 팔찌 절단 신호가 잡힌다 |

윤서는 사람을 데리고 나오는 쪽에 가깝다. 퇴원 보류 환자는 "나가는 것이 승인될 때까지 기다려야 한다"는 절차를 내부에서 비튼다.

윤서가 묻는다면:

```text
나오면 버틸 수 있어?
```

퇴원 보류 환자가 묻는다면:

```text
누가 버틸 수 있다고 승인해야 나갈 수 있는 거야?
```

## 5. PATCH와의 차이

| 비교축 | PATCH | 퇴원 보류 환자 |
|---|---|---|
| 핵심 질문 | 고쳐주는 것과 부품이 되는 것은 다르다 | 치료가 끝나지 않았다는 말은 보호인가, 붙잡기인가 |
| 캠페인 압력 | 정품 부품, 리콜 대상, 수리 자재 | 환자, 보험 대상자, 퇴원 불가 대상 |
| 행동 감각 | 자기 몸을 임시 부품처럼 쓰며 결합부를 막는다 | 절차가 확정되는 순간을 진료표와 팔찌로 늦춘다 |
| 금지선 | 단순 힐러, 마스코트, 윤서 장비화 금지 | 단순 힐러, 병약 피해자, 의료 판타지 금지 |
| 정산 | 충전태그, 로봇 자율성, 장비 안정 | 진료태그, 보험 심사 기록, 병상/약품 접근 |

PATCH가 고장 난 것을 임시로 잇는다면, 퇴원 보류 환자는 "지금 이 상태를 확정해도 되는가"를 시스템에 되묻는다.

중요:

```text
퇴원 보류 환자는 PATCH의 인간판이 아니다.
고쳐 주는 캐릭터가 아니라, 치료/승인/퇴원 절차의 판정 타이밍을 흔드는 캐릭터다.
```

## 6. OPEN-HOST와의 차이

| 비교축 | OPEN-HOST | 퇴원 보류 환자 |
|---|---|---|
| 출신 감각 | R01 내부 안내, 주거/가족 절차 | R02 의료/보험/퇴원 심사 절차 |
| 잘못 믿은 것 | 덜 나쁜 길을 안내하면 사람들이 덜 다친다고 믿었다 | 퇴원 승인 없이는 살아남은 것이 아니라고 믿는다 |
| 플레이 리듬 | 문, 동선, 평면도, 안내 리본 | 병상, 팔찌, 진료표, 심사 기록 |
| NPC 기능 위험 | R01 NPC 기능을 비울 수 있다 | R02를 너무 빨리 크게 열 수 있다 |
| 차별점 | 어디로 들어가고 나갈지 다룬다 | 누가 나가도 된다고 판정하는지 다룬다 |

OPEN-HOST는 길과 방의 문제다. 퇴원 보류 환자는 승인과 상태값의 문제다.

OPEN-HOST가 "이 문으로 가세요"를 의심한다면, 퇴원 보류 환자는 "퇴원 가능"이라는 도장을 의심한다.

## 7. 리오와의 장기 대비

리오와의 대비는 장기적으로 중요하다. 둘 다 "안전한 장소에 남겨지는 것"과 관련 있지만, 감각은 다르다.

| 비교축 | 퇴원 보류 환자 | 리오 |
|---|---|---|
| 슬롯 | 4번 후보 | 8번 보류 잠금 |
| 존재 유형 | 인간 환자/심사 대기자 | 탈정품 객실 정리 로봇 |
| 붙잡는 방식 | 치료, 병상, 퇴원 승인, 보험 심사 | 정품 복구, 객실 초기화, 청소 로그 |
| 두려운 상태 | 살아 있지만 계속 환자인 상태 | 멀쩡하지만 다시 정품 장치가 되는 상태 |
| 세계 확장 | R02 의료/보험 초반 해석 | E08 체류/숙박 중반 해석 |

공통점:

- 캠페인은 죽이려 하지 않는다.
- 캠페인은 안전하고 정돈된 상태로 보존하려 한다.
- 그 보존이 당사자에게는 사라지는 일처럼 느껴진다.

차이:

```text
퇴원 보류 환자는 "아직 나가면 안 된다"에 묶인다.
리오는 "아직 방을 비우면 안 된다" 또는 "다시 정품 상태로 돌아가야 한다"에 묶인다.
```

이 차이가 유지되면 리오 8번 보류 잠금을 앞당기지 않아도 된다.

## 8. 전투/무기 이미지

이 후보는 힐러가 아니다. 치료량, 회복 주문, 성스러운 의료 도구 같은 방향은 금지한다.

우선 채택할 이미지:

| 이미지 | 사용 방향 | 금지 방향 |
|---|---|---|
| 퇴원 팔찌/환자 팔찌 절단기 | 등록 확정 직전 팔찌를 끊거나 임시 해제한다 | 멋있는 암살 무기처럼 쓰지 않는다 |
| 링거대 | 밀고 당기며 임시 지지대, 차단봉, 이동축으로 쓴다 | 회복 오라 발생기처럼 쓰지 않는다 |
| 진료표 | 상태를 확정하지 않고 재검사/관찰/보류로 돌린다 | 마법서처럼 쓰지 않는다 |
| 보험 도장 | 적의 절차를 재심사, 보류, 반려 상태로 찍는다 | 돈/골드 보너스 도장으로 쓰지 않는다 |
| 접이식 들것 | 긴급 인양과 위치 고정에 연결한다 | 탱커 방패나 탈것으로 쓰지 않는다 |
| 휠체어 브레이크 | 밀려나는 동선을 멈추고 임계값을 만든다 | 병약함의 상징으로 소비하지 않는다 |

전투 감정:

```text
상처를 치료하는 사람이 아니라,
절차가 상처보다 먼저 사람을 확정하는 순간을 끊는 사람.
```

가능한 리듬:

- 적의 등록/확정 동작을 "재검사"로 되돌린다.
- 위험 구역을 "관찰 병상"처럼 잠시 고정해 이동을 늦춘다.
- 퇴원 팔찌 절단으로 자신 또는 아군의 부정적 고정 상태를 풀어낸다.
- 진료표의 상태값을 바꿔 정산 루트와 전투 위험을 동시에 흔든다.

금지:

```text
체력 회복을 주 업무로 삼지 않는다.
병원 침대를 탱크처럼 굴리지 않는다.
몸이 약해서 보호받는 캐릭터로 만들지 않는다.
```

## 9. 정산 루트

| 정산 루트 | 의미 | 플레이어블로 열 수 있는 감각 |
|---|---|---|
| 진료태그 | 치료, 약품, 병상, 검사 접근 조건 | 치료권이 돈이 아니라 절차라는 점을 보여준다 |
| 보험 심사 기록 | 정산 완료/미완료, 보류, 반려, 재심사 기록 | 완치와 정산 완료가 다를 수 있음을 보여준다 |
| 약품 접근 | 약을 얻는 조건과 대가 | 약품은 회복 아이템이 아니라 접근권과 부작용을 가진다 |
| 병상 접근 | 누가 누울 수 있고 누가 밀려나는가 | 안전한 자리가 다른 사람의 이동을 막을 수 있다 |
| 퇴원 팔찌 | 환자 번호, 병상 번호, 퇴원 승인 상태 | 팔찌를 끊는 것이 자유와 위험을 동시에 부른다 |

정산 위험:

- 진료태그를 많이 쓰면 R02 신호가 선명해진다.
- 보험 심사를 자주 흔들면 재심사 기록이 쌓인다.
- 병상 접근을 얻으면 다른 생존자의 병상 점유 압력이 커질 수 있다.
- 퇴원 팔찌를 끊으면 당장은 풀리지만, 긴급 인양 실패 시 재입원 절차가 빠르게 붙는다.

정산 문장:

```text
진료태그는 돈이 아니라,
아픈 사람이 무엇에 접근해도 된다고 판정받는 조건이다.
```

## 10. HP 0 / 긴급 인양 차이

HP 0은 사망이 아니다. 이 후보의 HP 0은 R02가 다시 환자 상태를 확정하려는 순간이다.

| 단계 | 연출/시스템 감각 |
|---|---|
| HP 0 직전 | 퇴원 팔찌가 다시 병상 번호를 찾기 시작한다 |
| HP 0 | 진료표가 "퇴원 취소" 또는 "재입원 심사 재개"로 넘어간다 |
| 긴급 인양 성공 | 접이식 들것 회수선과 팔찌 절단 신호가 침묵 보급소 좌표를 붙잡는다 |
| 보급소 복귀 | 미정산 진료태그 일부, 약품 접근권, 심사 보류 기록에 손실 또는 오염이 남는다 |
| 실패 위험 | 장기 입원 대상, 보호자 미확인자, 보험 심사 대기자로 더 강하게 고정된다 |

윤서와의 차이:

```text
윤서는 마지막 좌표에서 회수선이 끌어온다.
퇴원 보류 환자는 재입원 절차가 닫히기 전에 팔찌 절단 신호와 들것 회수선이 먼저 잡힌다.
```

PATCH와의 차이:

```text
PATCH는 정품 복구/부품 회수 직전에 끌려온다.
퇴원 보류 환자는 퇴원 취소/보험 심사 재개 직전에 끌려온다.
```

## 11. 해금 조건

권장 해금:

```text
R02 진입 전 암시 + R02 초반 직접 해금
```

운영안:

| 단계 | 위치 | 내용 | 판정 |
|---|---|---|---|
| 암시 | R01 후폭풍 또는 R01-L03 정산 흔적 | 환자 팔찌, 보호자 미확인 기록, 보험 심사 스탬프를 발견한다 | 사용 |
| 첫 조우 | R02-L01 응급 접수 홀 | 퇴원 보류 상태로 접수/퇴원 사이에 걸린 인물을 본다 | 사용 |
| 검증 | R02-L02 장기 입원동 또는 R02-L03 보험 심사 약국 | 이 인물이 치료를 거부하는 것이 아니라 절차 확정을 늦춘다는 것을 확인한다 | 사용 |
| 해금 | R02-L01~L03 사이의 개인 결절 | 퇴원 취소 직전 긴급 인양 조건을 만든다 | 조건부 사용 |
| 금지 | R02-L05 퇴원 불가 병동 보스 이후 | R02 후반 보스까지 당겨야 해금되는 구조 | 금지 |

4번 슬롯 적합성:

- R02는 P1에서 R01 다음 외부 권역 후보로 읽힌다.
- R02-L01, R02-L02, R02-L03은 초반 해석 가능 범위다.
- R02-L04, R02-L05까지 당기면 세계가 너무 빨리 커진다.

따라서 4번 슬롯에 넣는 것은 적절하다. 단, 해금은 R02 전체 클리어가 아니라 R02 초반 접수/심사 절차 안에서 끝나야 한다.

## 12. 위험

| 위험 | 왜 위험한가 | 차단선 |
|---|---|---|
| 단순 힐러화 | PATCH의 수리 축과 겹치고, 의료 캠페인의 절차 질문이 사라진다 | 회복량 중심 스킬 금지. 상태 보류, 재심사, 팔찌 절단 중심으로 설계 |
| 병약 캐릭터 클리셰 | 캐릭터가 캠페인을 비트는 주체가 아니라 보호받는 대상으로 납작해진다 | 성별/나이/외형을 먼저 확정하지 않는다. 생존 방식과 절차 이해를 먼저 쓴다 |
| 비극 환자 소비 | R02가 눈물 장치가 되고 플레이어블 욕망이 약해진다 | 동정 유도보다 절차를 읽는 능력과 위험 감수로 매력을 만든다 |
| R02 과확장 | 의료 캠페인이 R01 후폭풍을 덮고 초반 세계가 급히 넓어진다 | R02-L01~L03만 사용. R02-L04/L05는 후속 |
| 병원이 악당화됨 | 세계 기준의 "보존하려는 환경"이 무너진다 | 의료 캠페인은 살려 두려 하지만 삶을 환자 역할로 잠근다 |
| 보호자 등록 거부자와 중복 | 13번 후속 슬롯의 기능을 빼앗는다 | 보호자 절차는 이 후보의 압력으로만 쓰고, 보호자/보증인 자체는 13번에 남긴다 |

## 13. 최종 판정과 통과 조건

최종 판정:

```text
조건부 통과
```

4번 슬롯에 남긴다. 단, 프로필 v1.0 작성 전 아래 조건을 문서로 한 번 더 잠근다.

통과 조건:

1. 전투 콘셉트는 치료량이 아니라 입원/퇴원 판정, 재심사, 상태 보류, 팔찌 절단이어야 한다.
2. 이 후보는 환자이지만 피해자 전시물이 아니다. 절차의 빈틈을 읽고 스스로 이용하는 사람이어야 한다.
3. 잘못 믿는 것은 "퇴원 승인 없이 나가면 살아남은 것이 아니다"로 둔다.
4. R02 해금 범위는 R02-L01~L03 안에 묶는다.
5. HP 0은 재입원, 퇴원 취소, 보험 심사 재개로 차별화한다.
6. PATCH의 수리/봉합, 윤서의 회수, OPEN-HOST의 동선, 리오의 정품 복구와 겹치지 않는다.
7. 보호자 등록 거부자 13번 후속 슬롯의 기능을 빼앗지 않는다.

보류로 돌릴 조건:

- 전투 설계가 회복량/버프 중심 힐러로 흐를 때.
- 캐릭터 매력이 병약함, 불쌍함, 순한 희생성에 기대게 될 때.
- R02 후반 보스나 퇴원 불가 병동을 해금 조건으로 끌어올 때.
