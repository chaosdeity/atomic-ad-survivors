## 0. 조건부 통과 사유 요약

리오는 8번 슬롯에 둘 힘이 충분하다.
다만 잠금 재검토 단계에서는 아래 이유로 조건부 통과였다.

| 조건 | 위험 | 이번 문서의 처리 |
|---|---|---|
| VAC-0와 중복 | 둘 다 청소 로봇처럼 보일 수 있음 | VAC-0는 R01 빈집 홈케어 NPC, 리오는 체류·숙박권 객실 정리 플레이어블 후보로 분리 |
| PATCH와 중복 | 둘 다 정품 복구/로봇 자율성 축을 건드림 | PATCH는 수리/부품/보급소, 리오는 객실 초기화/체크아웃/정품 복구 거부로 분리 |
| 해금 시점 | 초반에 열면 R01/VAC-0/PATCH의 의미가 흐려짐 | 8번 슬롯은 유지하되 직접 해금은 중반 체류·숙박권까지 보류 |
| 핵심 문장 | 청소 로봇 기믹으로 납작해질 수 있음 | "그 방은 비어 있지 않았습니다."로 고정 |
| HP 0 | PATCH의 정품 복구와 겹칠 수 있음 | 객실 초기화, 체크아웃 처리, 청소 완료 상태로 되돌려짐을 핵심 위험으로 고정 |
| 능력 방향 | 청소/펫/수리 캐릭터화 위험 | 방 상태 판정, 흔적 보존, 체크아웃 지연, 객실 초기화 저항 중심으로 고정 |

## 1. 최종 보완 판정

판정:

```text
통과로 승격 가능
```

이유:

1. VAC-0와 리오의 공간, 기능, 서사 질문이 분리되었다.
2. PATCH와 리오의 로봇 서사와 전투 리듬이 분리되었다.
3. 리오는 E01 초반 합류가 아니라 중반 체류·숙박권 해금 후보로 고정되었다.
4. 핵심 문장이 "청소"가 아니라 "공실 처리 거부"를 향한다.
5. HP 0은 정품 복구만이 아니라 객실 초기화/체크아웃/청소 완료 상태로 되돌려짐까지 묶였다.
6. 플레이어블 능력은 수리나 펫 행동이 아니라 방 상태 판정과 흔적 보존의 절차 비틀기로 읽힌다.

남는 조건:

```text
캐릭터 자체의 조건은 해소되었다.
운영 조건만 남는다.
리오는 8번 슬롯으로 유지하되, 직접 해금은 반드시 중반 체류·숙박 캠페인권에서 진행한다.
```

## 2. VAC-0와의 차이 고정

VAC-0와 리오는 같은 청소 로봇 계열처럼 보이면 안 된다.
둘의 질문은 아래처럼 갈라진다.

| 구분 | VAC-0 | 리오 / RIO-08 |
|---|---|---|
| 위치 | R01 빈 집 내부 | E08_C01 체류·숙박 캠페인권 |
| 존재 방식 | 홈케어 장치 / 초반 NPC | 객실 정리 로봇 / 8번 플레이어블 후보 |
| 핵심 절차 | 흔적 삭제 | 비어 있지 않은 방을 공실로 처리하지 않기 |
| 공간 감각 | 분양 주택, 빈집, 현관, 먼지, 우편, 발자국 | 객실, 카드키, 투숙 로그, 체크아웃, 객실 초기화 |
| 캠페인이 붙이는 역할 | 안전한 집을 유지하는 청소 장치 | 정품 객실 정리 장치, 객실 초기화 장치, 리콜 대상 |
| 플레이어 기능 | 없음. R01 NPC 반응과 흔적 보존 위험 담당 | 방 상태 판정, 흔적 보존, 체크아웃 지연, 초기화 저항 |
| 금지 연결 | 플레이어블 승격 금지 | VAC-0 후속 모델/업그레이드형 금지 |

고정 문장:

```text
VAC-0는 R01 빈 집 내부에서 사람의 흔적을 안전 위험처럼 지우는 홈케어 장치다.
리오는 체류·숙박 캠페인권에서 비어 있지 않은 방을 비어 있다고 정리하지 않기로 한 객실 정리 로봇이다.
```

R01에서 가능한 리오 암시:

- 낯선 객실 카드 조각.
- R01 청소 로그 안에 섞인 "객실 장치 호환 불가" 흔적.
- VAC-0가 지우지 못한 호텔식 체크아웃 문구.
- 리오 본인 등장 없이, 체류·숙박권이 나중에 열린다는 작은 단서.

R01에서 금지:

```text
리오 직접 합류.
VAC-0가 리오를 호출하거나 소개하는 구조.
VAC-0의 기능을 리오가 이어받는 구조.
리오가 R01 빈집 청소 문제를 해결하는 구조.
```

## 3. PATCH와의 차이 고정

PATCH와 리오는 모두 로봇이고 정품 복구 위험을 가진다.
그래서 중심 절차를 반드시 다르게 둔다.

| 구분 | PATCH | 리오 / RIO-08 |
|---|---|---|
| 위치 | 침묵 보급소, C04 정품/수리 잔향 | E08_C01 체류·숙박 캠페인권 |
| 핵심 | 자기 몸을 부품처럼 써온 보급소 수리 로봇 | 객실 초기화/정품 복구/체크아웃 절차를 거부하는 객실 정리 로봇 |
| 잘못 익힌 생존법 | 망가지면 자기 부품을 내어주면 된다고 배움 | 방을 비우라는 명령을 따르면 흔적이 사라진다는 것을 배움 |
| 전투 이미지 | 자석 수리팔, 패치 테이프, 리벳, 임시 봉합 | 객실 카드, 객실 로그, 초기화 램프, 청소 브러시, 체크아웃 보류 |
| 능력 방향 | 장비 안정, 임시 수리, 자기부품화 위험 | 방 상태 판정, 흔적 보존, 체크아웃 지연, 객실 초기화 저항 |
| HP 0 | 부품 목록화/정품 복구/부품 회수 직전 인양 | 정품 복구/객실 초기화/체크아웃 처리/청소 완료 상태 직전 인양 |

고정 문장:

```text
PATCH는 고치기 위해 자기 몸을 부품처럼 써온 로봇이다.
리오는 정리하기 위해 사람의 체류 흔적을 지우라는 명령을 거부한 로봇이다.
```

금지:

```text
리오가 PATCH처럼 수리한다.
리오가 보급소 장비를 안정시킨다.
리오가 자기 몸을 부품으로 내어준다.
PATCH의 정품 복구 공포를 리오가 초반에 다시 설명한다.
```

이 문장은 리오의 모든 설계 기준이다.

|---|---|

| 전투 리듬 요약 | 방 상태 판정을 늦추고, 남은 흔적이 공실 처리되지 않게 버틴다. |

```text

```

## 5. 해금 시점 고정

8번 슬롯은 유지한다.
하지만 직접 해금은 중반 체류·숙박권까지 보류한다.

| 시점 | 처리 | 이유 |
|---|---|---|
| E01/P0 | 직접 등장 금지 | 윤서, PATCH, R01 NPC 선등장 축을 보호한다. |
| R01 | 암시만 가능 | VAC-0와 R01 흔적 삭제 질문이 주인공이어야 한다. |
| PATCH 초반 합류 직후 | 금지 | 로봇 플레이어블의 첫 얼굴은 PATCH로 남겨야 한다. |
| 3~7번 후보 구간 | 금지 | 강하람, 한이루, 차유건, 백여울의 인간 절차 축을 보호한다. |
| E08_C01-L01 | 선등장/로그 추적 가능 | 체류·숙박 캠페인권의 객실 상태 규칙을 먼저 보여준다. |
| E08_C01-L02 이후 | 직접 해금 가능 | 유저가 공실 처리, 체크아웃, 객실 초기화 위험을 이해한 뒤 조작한다. |

고정 원칙:

```text
리오는 E01 초반 합류 금지.
리오는 R01/VAC-0/PATCH의 의미를 흐리면 안 된다.
리오는 중반 체류·숙박 캠페인권이 실제 플레이 공간으로 열린 뒤 해금한다.
```

## 6. 플레이어블 능력 방향

리오는 청소 딜러, 펫, 수리 로봇이 아니다.

능력 중심:

| 축 | 설계 방향 | 금지 방향 |
|---|---|---|
| 방 상태 판정 | 투숙중/공실/청소대기/점검중 같은 상태를 전투 판정으로 바꾼다 | 적을 방 청소하듯 쓸어버리는 광역 청소기 |
| 흔적 보존 | 사라질 로그, 카드 흔적, 체류 기록을 잠시 보존한다 | VAC-0처럼 흔적을 자동 청소한다 |
| 체크아웃 지연 | 체크아웃 처리와 공실 전환을 늦춘다 | 단순 시간 정지 |
| 객실 초기화 저항 | 초기화 명령에 저항하되 정품 복구 위험을 누적한다 | 무조건 안전한 방어막 |
| 서비스 거부 | 정품 서비스 호출을 거절하고 캠페인 호출을 어긋나게 한다 | 귀여운 반항 대사만 있는 마스코트 |

전투/탐색 리듬:

```text
1. 방 상태 판정:
   적이나 지형이 어떤 "객실 상태"로 읽히는지 바꾼다.

2. 흔적 보존:
   사라질 보상/로그/단서를 잠시 보존하지만, 보존 슬롯이 과부하되면 정품 복구 위험이 오른다.

3. 체크아웃 지연:
   체크아웃 처리, 퇴거 처리, 공실 전환을 늦춰 탈출/회수 시간을 만든다.

4. 객실 초기화 저항:
   초기화 명령을 거부해 전장을 유지하지만, 오래 버틸수록 리오가 정품 상태로 끌려간다.
```

## 7. HP 0 차별화

리오의 HP 0은 사망이나 폭발이 아니다.

핵심 위험:

```text
정품 복구
객실 초기화
체크아웃 처리
청소 완료 상태로 되돌려짐
```

HP 0 흐름:

```text
1. 리오의 초기화 램프가 정품색으로 돌아간다.
2. 객실 로그가 "체크아웃 처리 중"으로 바뀐다.
3. 남아 있던 흔적이 "청소 완료" 상태로 덮인다.
4. 리콜선이 정품 복구 도크로 리오를 당긴다.
5. 객실 카드 조각과 남은 투숙 로그가 보급소 회수선에 마지막 좌표를 남긴다.
6. 긴급 인양은 리콜보다 늦게 도착하지만, 완전 초기화 직전에 리오를 끌어온다.
```

차별화:

| 대상 | HP 0/인양 차이 |
|---|---|
| 윤서 | 마지막 현장 좌표를 회수선이 붙잡는다. |
| PATCH | 부품 목록화와 부품 회수 직전에 보급소 수리선이 당긴다. |
| VAC-0 | 플레이어블 HP 0이 없다. NPC 상태 반응과 청소 제외 목록으로만 작동한다. |
| 리오 | 체크아웃 처리와 객실 초기화가 완료되기 직전, 객실 카드와 투숙 로그가 인양 좌표를 남긴다. |

금지:

```text
리오가 폭발하고 재조립된다.
리오가 PATCH처럼 부품 창고로 끌려간다.
리오가 VAC-0처럼 청소통을 비우는 연출로 끝난다.
리오가 HP 0 때 안전하게 정품 복구되어 버프를 받는다.
```

## 8. R01/VAC-0/PATCH 의미 보호 규칙

리오는 R01의 질문을 빼앗지 않는다.

보호 규칙:

1. R01에서 청소와 흔적 삭제의 얼굴은 VAC-0다.
2. 초반 로봇 자율성과 보급소 생활감의 얼굴은 PATCH다.
3. 리오는 중반 체류·숙박권의 객실 상태 판정을 담당한다.
4. 리오는 R01 보스 전후 선택을 해결해주지 않는다.
5. 리오는 PATCH의 수리/부품/보급소 정비 루트를 대신하지 않는다.
6. 리오 암시는 객실 카드, 체크아웃 로그, 잘못 분류된 호환 기록 정도로 제한한다.

## 10. 통과 조건 체크리스트

| 조건 | 상태 |
