# Outpost Prototype Directive 0.2

상태: 0.2 침묵 보급소 구현 작업 지시서  
목적: `outpost_growth_minimum_0_2.md`를 Godot 프로토타입 작업 단위로 변환한다.  
기준: 보급소는 인양 후 메뉴가 아니라, 유저가 돌아오고 싶어지는 장소여야 한다.

## 핵심 목표

0.2 보급소는 작은 편의점형 피난처다. 건설 게임처럼 키우지 않는다. 대신 첫 회수, 첫 개조, 첫 흔적 보관, 첫 송출 기록 해석, 첫 보스 성과가 보급소에 남아야 한다.

유저가 느껴야 할 감정:

```text
이번 런이 헛되지 않았다.
윤서와 보급소가 조금 더 버틴다.
다음 출격은 내가 준비해서 나간다.
```

## 포함 범위

### P0 필수

| 항목 | 구현 요구 |
| --- | --- |
| 정산 카운터 | 런 결과, 보상, 획득 단서, NPC 한 줄 반응 표시 |
| 정비대 | 윤서 기본 개조 3종 중 1개 이상 구매 가능 |
| 이름 보관함 | P0 흔적 3개 보관 상태 표시 |
| 출격 게시판 | 외곽 주택가 상태와 송출 기록 진행 표시 |
| 차징 조율대 | 반품 도장/회수선 계열 개조 2종 이상 표시 |
| 첫 강제 회수 연결 | `signal_overload` 후 정산과 첫 개조로 자연스럽게 연결 |

### P1 권장

| 항목 | 구현 요구 |
| --- | --- |
| 보급소 오브젝트 변화 | 개조/흔적/송출 기록/코어 파편 조건별 작은 시각 변화 |
| NPC 조건 대사 | 상태 변화마다 1줄씩 출력 |
| 종이 카탈로그 벽 | 카드/광고 전단 분위기와 확장 슬롯 암시 |
| 보스 성과 반응 | 코어 파편 유리병, 세븐 반응, 게시판 변화 |

### P2 보류

| 항목 | 보류 이유 |
| --- | --- |
| 피난민 인구 시뮬레이션 | 0.2 범위 초과 |
| 보급소 레벨 시스템 | 메뉴형 성장으로 보일 위험 |
| NPC 호감도 UI | 내부 성향을 유저에게 노출할 위험 |
| 복잡한 건설/배치 | 모바일 첫 버전에 과함 |
| 유료 캐릭터 전용 시설 | 0.2 신뢰 형성 전에는 위험 |

## 필요 상태값

### OutpostState

```text
OutpostState
  stage
  first_recovery_seen
  unlocked_facilities
  facility_upgrade_ids
  stored_trace_ids
  broadcast_records_decoded
  transmitter_access_state
  last_run_result_type
  last_boss_outcome
  visual_flags
  npc_line_flags
```

주의:

- `stage`는 내부 정렬용이다. 유저에게 "보급소 3레벨"처럼 보여주지 않는다.
- `visual_flags`는 작은 오브젝트 변화에만 쓴다.
- `npc_line_flags`는 같은 대사가 반복 피로를 만들지 않도록 사용한다.

### FacilityData

```text
FacilityData
  id
  display_name
  unlock_condition
  upgrade_ids
  visible_when
  short_description
```

0.2 시설 ID:

```text
settlement_counter
maintenance_bench
name_archive
sortie_board
charge_calibration_bench
paper_catalog_wall
```

## 첫 10분 흐름

```text
1. 윤서가 외곽 주택가로 첫 출격한다.
2. 100초 전후 `signal_overload`가 발생한다.
3. HP 0은 사망이 아니라 비콘 기반 긴급 인양/보급소 강제 회수로 처리한다.
4. 정산 카운터에서 성과 기반 보상을 보여준다.
5. 광고 전단 조각으로 첫 개조 1개를 바로 살 수 있다.
6. 출격 게시판에 "고객명 확인 중" 상태가 뜬다.
7. 유저는 재출격 버튼을 누를 이유를 얻는다.
```

합격 감정:

```text
졌는데 억울한 게 아니라, 한 번 더 가고 싶다.
```

## 시설별 작업 지시

### 정산 카운터

필수 표시:

- 생존 시간.
- 처치 수.
- 엘리트 처치 여부.
- 광고 전단 조각 획득량.
- 캠페인 코어 파편 획득량.
- 송출 기록 획득 여부.
- 흔적 획득 여부.
- 추천 개조 1개.
- NPC 반응 1개.

금지 표시:

```text
인양 정산
trace_preservation +10
human_stability +5
local_memory_profile = haunted
```

첫 회수 예시:

```text
회수 완료
생존 01:42 / 처치 128
광고 전단 조각 +3
미나: 죽은 게 아니야. 저쪽이 네 이름을 다 읽기 전에 끊어낸 거야.
```

### 정비대

0.2 필수 개조:

| 개조 ID | 표시명 | 비용 | 효과 방향 |
| --- | --- | ---: | --- |
| scanner_output_01 | 스캐너 출력 보정 | 전단 2 | 자동공격 피해/안정성 증가 |
| scanner_return_line_01 | 회수선 절연 보강 | 전단 2 | 과부하 경고 여유 증가 |
| scanner_filter_01 | 보급소 필터 교체 | 전단 3 | 송출 기록 해석 문구 확장 |

첫 회수 후 최소 1개는 반드시 구매 가능해야 한다.

### 이름 보관함

0.2 P0 흔적:

```text
trace_nameplate_001
trace_burnt_family_photo_001
trace_nameless_receipt_001
```

필수 기능:

- 보관한 흔적 이름 표시.
- 각 흔적 짧은 문구 1개 표시.
- 보관 후 보급소 시각 변화 1개 발생.
- 흔적을 단순 재료처럼 즉시 소모시키지 않는다.

예시:

```text
문패 조각
가격표가 붙기 전의 이름이 남아 있다.
```

### 출격 게시판

필수 상태 문구 우선순위:

| 조건 | 게시판 문구 |
| --- | --- |
| 기본 | 영원히 팔리는 행복한 동네 |
| 첫 회수 후 | 고객명 확인 중 |
| 송출 기록 3개 | 송출관 접근 절차 확정 |
| 결절 파괴 | 조용한 오픈하우스 |
| 기억 추출 | 보정된 가족사진 |
| 3회 연속 방문 | 단골 고객 주간 |

우선순위:

```text
보스 결과 > 송출관 접근 절차 > 반복 방문 > 첫 회수 > 기본
```

유저에게는 내부 상태값이 아니라 게시판 문구만 보여준다.

### 차징 조율대

0.2 필수 개조:

| 개조 ID | 표시명 | 비용 | 효과 방향 |
| --- | --- | ---: | --- |
| return_stamp_pressure_01 | 반품 도장 압력 보정 | 전단 2 | 차징 피해 증가 |
| return_stamp_coolant_01 | 도장 냉각 절연 | 전단 3 | 차징 안정성 증가 |

주의:

- 차징은 윤서의 전투 매력 핵심이다.
- 첫 개조 후보에 차징 관련 선택지가 반드시 있어야 한다.

## NPC 대사 최소 세트

첫 회수:

```text
미나: 죽은 게 아니야. 저쪽이 네 이름을 다 읽기 전에 끊어낸 거야.
도윤: 장비가 버텼어. 그러면 고칠 수 있어.
윤서: 다음엔 내가 먼저 찍어. 반품이라고.
```

첫 개조:

```text
도윤: 됐다. 이제 적어도 네가 찍은 건 더 확실히 찢길 거야.
```

첫 흔적 보관:

```text
유리: 이건 팔 물건이 아니야. 그러니까 여기 둬도 돼.
```

송출 기록 3개:

```text
미나: 길이 맞춰졌어. 문제는, 저쪽도 우리가 온다는 걸 알 거라는 거야.
```

코어 파편 첫 회수:

```text
세븐: 이것은 돈이 아닙니다. 더 나쁜 것입니다. 쓸모가 있습니다.
```

보스 처리 후:

```text
한별: 오늘은 라디오가 덜 웃어. 그게 좋은 건지는 모르겠어.
```

## 보급소 시각 변화

0.2 필수 변화:

| 조건 | 변화 |
| --- | --- |
| 첫 개조 완료 | 정비대 조명 1개 점등 |
| 흔적 1개 보관 | 이름 보관함 라벨 추가 |
| 송출 기록 2개 | 출격 게시판에 찢긴 지도선 표시 |
| 보스 HP 25% 이하 | 코어 파편 유리병 등장 |
| 결절 파괴 | 보급소 밖 광고 소음 약화 |
| 기억 추출 | 가족사진 주변 잔향 문구 추가 |

작게 보여줘도 된다. 단, 같은 장소에 돌아왔다는 감각은 반드시 있어야 한다.

## 보상 연결

첫 회수 권장 지급:

```text
광고 전단 조각 3
첫 현장 데이터 1
송출 기록 없음 또는 튜토리얼 잔향 1
```

주의:

- 보상 원인은 생존 시간, 처치 수, 이벤트 반응이다.
- 회수 자체가 보상 원인으로 보이면 안 된다.
- 첫 회수 후 아무것도 살 수 없으면 실패다.

## 유저 선호 점수

예상 점수:

| 기준 | 점수 | 이유 |
| --- | ---: | --- |
| 첫 귀환 감정 | 8 | 죽음이 아니라 회수로 보이면 재도전 욕구가 생김 |
| 성장 체감 | 8 | 첫 회수 후 즉시 개조 가능 |
| 캐릭터 애정 | 8 | 윤서/도윤/미나 반응이 기능과 감정을 동시에 줌 |
| 반복 방문 매력 | 8 | 게시판과 보관함 변화가 누적됨 |
| 구현 현실성 | 8 | 시설은 5개지만 깊은 UI는 요구하지 않음 |
| 리스크 | 7 | 화면이 메뉴처럼 보이면 매력 급락 |

종합:

```text
8.0 / 10
```

10점으로 올리기 위한 다음 개선:

- 윤서 전용 보급소 모션/짧은 반응 추가.
- 보관한 흔적이 런 시작 문구에 다시 등장.
- 팝시-제로 같은 출시 캐릭터가 같은 시설을 다르게 해석.
- 보스 선택 후 시설 하나가 확실히 달라짐.

## 검증 항목

구현 후 반드시 관찰한다.

| 관찰 | 합격 기준 | 실패 시 수정 |
| --- | --- | --- |
| 첫 회수 후 감정 | 억지 패배보다 강제 회수로 이해 | 회수 경고/대사 강화 |
| 첫 개조 구매율 | 80% 이상 | 비용 인하 또는 추천 개조 강조 |
| 재출격률 | 첫 회수 후 70% 이상 | 게시판 목표/윤서 대사 강화 |
| 보급소 인식 | 메뉴가 아니라 장소로 기억 | 오브젝트 변화와 NPC 반응 추가 |
| 흔적 보관 이해 | 재료가 아니라 의미 있는 물건으로 인식 | 보관함 문구와 시각 변화 강화 |

## 금지 방향

- 보급소를 레벨업 메뉴로만 만들지 않는다.
- 내부 성향/점수/철학을 UI에 표시하지 않는다.
- 첫 회수 후 아무 개조도 못 하게 하지 않는다.
- 광고 전단 조각을 평범한 골드처럼 보이게 하지 않는다.
- 흔적을 단순 강화 재료로만 소비시키지 않는다.
- 유료 요소가 보급소 기본 기능을 막지 않는다.
