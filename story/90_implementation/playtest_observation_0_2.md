# Playtest Observation 0.2

상태: 0.2 플레이테스트 관찰표  
연결 문서: `story_cross_validation_0_2.md`, `godot_scope_0_2_from_story.md`, `reward_growth_loop_0_2.md`, `r01_suburb_content_lock_0_2.md`, `character_score_upgrade_backlog.md`

## 목적

이 문서는 0.2를 만든 뒤 무엇을 관찰해야 성공/실패를 판단할 수 있는지 정리한다.

기준:

```text
유저가 좋아할 것인가?
```

문서상 구조가 좋아도, 실제 플레이에서 아래가 실패하면 고쳐야 한다.

- 윤서가 기본캐처럼 밋밋하다.
- 첫 회수가 억지 패배처럼 보인다.
- 첫 개조가 약해서 다시 나가고 싶지 않다.
- 외곽 주택가가 재탕 맵처럼 보인다.
- 보스 선택 후 변화가 안 느껴진다.
- 스토리 문구가 길고 전투 흐름을 막는다.

## 테스트 원칙

- 유저에게 세계관 설명을 먼저 하지 않는다.
- 유저가 플레이 중 혼자 이해하는 것을 본다.
- 플레이 후 질문은 짧게 한다.
- “무엇이 좋았나”보다 “다시 하고 싶은 순간이 있었나”를 본다.
- 캐릭터, 전투, 보상, 거점, 지역 반응을 분리해서 평가한다.

## 테스트 세션

### 세션 A: 첫 10분

목적:

```text
첫 조작, 첫 차징, 첫 강제 회수, 첫 개조가 먹히는지 확인한다.
```

관찰:

- 30초 안에 이동/자동공격을 이해하는가.
- 첫 차징 버튼을 눌렀을 때 반응이 좋은가.
- 100초 전후 강제 회수를 죽음으로 오해하는가.
- 회수 후 보급소 문구를 읽는가, 바로 넘기는가.
- 첫 개조를 찾는 데 막히는가.
- 첫 개조 후 재출격 의지가 생기는가.

통과 기준:

```text
테스터 70% 이상이 첫 회수 후 "한 번 더"를 선택한다.
```

위험 신호:

- “왜 죽었지?”
- “보상이 왜 들어왔지?”
- “뭘 눌러야 강해져?”
- “기본 캐릭터라 그런지 평범하다.”

즉시 수정:

- 강제 회수 문구를 더 명확히 한다.
- 첫 개조 버튼을 더 앞에 둔다.
- 반품 도장 VFX/사운드를 강화한다.

### 세션 B: 첫 30분

목적:

```text
반복 출격, 송출 기록, 흔적, 보급소 반응이 지루하지 않은지 확인한다.
```

관찰:

- 유저가 120초 신호 이벤트를 목표로 인식하는가.
- 송출 기록을 재화가 아니라 단서로 이해하는가.
- 흔적 아이템을 기억하는가.
- 보급소 NPC 반응이 짧지만 남는가.
- 광고 전단 조각을 돈처럼만 느끼는가, 재료처럼 느끼는가.
- 외곽 주택가가 밝고 이상한 장소로 기억되는가.

통과 기준:

```text
테스터 60% 이상이 송출 기록이나 흔적 중 하나를 자기 말로 설명할 수 있다.
```

위험 신호:

- “송출 기록이 그냥 열쇠인가요?”
- “전단 조각은 그냥 골드죠?”
- “흔적은 왜 보관해야 해요?”
- “대사가 너무 많다.”

즉시 수정:

- 송출 기록 획득 구조를 목표형으로 더 선명하게 만든다.
- 흔적 보관 후 보급소 오브젝트 변화를 추가한다.
- 정산 화면 텍스트를 줄인다.

### 세션 C: 첫 1시간

목적:

```text
보스 접근, 보스 실패 보상, 보스 선택, 재방문 변화가 작동하는지 확인한다.
```

관찰:

- 보스에게 졌을 때 헛수고라고 느끼는가.
- HP 65%/25% 보상 기준이 납득되는가.
- 캠페인 코어 파편이 보스 성과 재화로 이해되는가.
- 결절 파괴/기억 추출 선택이 보상 차이 이상으로 느껴지는가.
- 선택 후 게시판/대사/지역 변화가 보이는가.
- 송출관과 스마일 홈 시어머니를 혼동하는가.

통과 기준:

```text
테스터 50% 이상이 보스 실패 후 재도전 의사를 보인다.
테스터 60% 이상이 보스 선택 후 지역이 바뀌었다고 느낀다.
```

위험 신호:

- “보스 못 잡으면 손해가 너무 크다.”
- “보스 선택이 그냥 보상 선택 같다.”
- “송출관이 이 보스 아닌가요?”
- “같은 맵 또 가는 이유를 모르겠다.”

즉시 수정:

- 보스 실패 정산에 패턴 기록/코어 파편을 명확히 표시한다.
- 선택 후 첫 재방문 변화를 더 크게 만든다.
- 송출관 문구를 `상위 결절` 또는 `접근 절차` 쪽으로 정리한다.

## 핵심 관찰 항목

### 윤서

질문:

```text
유저가 윤서를 좋아하는가?
```

관찰 항목:

| 항목 | 좋은 신호 | 나쁜 신호 |
| --- | --- | --- |
| 첫눈 | 이름표/장비를 기억함 | 외형 기억 못 함 |
| 자동공격 | 안정적이라고 느낌 | 밋밋하다고 느낌 |
| 차징 | 누르는 맛이 있다고 말함 | 버튼을 아낌/잊음 |
| 대사 | 짧은 문장을 기억함 | 대사를 넘김 |
| 감정 | 첫 회수 후 걱정함 | 그냥 사망 처리로 봄 |

점수 판정:

| 점수 | 기준 |
| ---: | --- |
| 9~10 | 기본캐가 아니라 게임의 얼굴로 느껴짐 |
| 8 | 계속 써도 괜찮다고 느낌 |
| 7 | 기능은 좋지만 애정은 약함 |
| 6 이하 | 튜토리얼 캐릭터로만 소비됨 |

0.2 목표:

```text
8점 이상
```

### 반품 도장

질문:

```text
단일 버튼 차징 스킬이 게임의 핵심 손맛이 되는가?
```

관찰:

- 차징 가능 상태를 알아보는가.
- 누르는 타이밍을 기다리는가.
- 맞췄을 때 웃거나 반응하는가.
- 빗나갔을 때 불공정하다고 느끼는가.
- 보스/엘리트에게 의미 있다고 느끼는가.

통과 기준:

```text
첫 10분 안에 테스터 70% 이상이 차징을 의식적으로 사용한다.
```

### 첫 강제 회수

질문:

```text
첫 실패가 억지 패배가 아니라 세계관 이벤트로 느껴지는가?
```

관찰:

- HP 0 긴급 인양으로 오해하는가.
- 보급소가 끌어당겼다는 것을 이해하는가.
- 잘한 유저가 더 많은 성과를 남겼다고 느끼는가.
- 다시 나가고 싶은가.

통과 기준:

```text
테스터 70% 이상이 "죽은 게 아니라 회수됐다"를 이해한다.
```

### 보상/성장

질문:

```text
보상이 성과에서 나온다고 느끼는가?
```

관찰:

- 광고 전단 조각이 너무 적거나 많다고 느끼는가.
- 첫 회수 후 개조 가능 여부를 이해하는가.
- 보스 조우만으로 코어를 기대하는가.
- 보스 HP 구간 보상을 납득하는가.

통과 기준:

```text
첫 회수 후 80% 이상이 개조를 한다.
보스 실패 후 50% 이상이 재도전을 선택한다.
```

### 외곽 주택가

질문:

```text
첫 지역이 튜토리얼 맵이 아니라 로컬 생태권처럼 느껴지는가?
```

관찰:

- 쿠폰 전단 떼, 미소 우편함, 세일 풍선을 기억하는가.
- 같은 집 반복을 의도된 기괴함으로 받아들이는가.
- 재방문 3회 문구가 재미있게 찝찝한가.
- 보스 선택 후 다음 런 변화가 보이는가.

통과 기준:

```text
테스터 60% 이상이 외곽 주택가를 "광고 동네", "가족 판매 동네"처럼 자기 말로 설명한다.
```

### 보급소

질문:

```text
보급소가 메뉴가 아니라 돌아오는 장소인가?
```

관찰:

- 정산 후 보급소를 빨리 닫고 싶은가.
- NPC 한 줄을 읽는가.
- 개조와 출격 게시판을 자연스럽게 찾는가.
- 흔적 보관 후 변화가 보이는가.

통과 기준:

```text
테스터 60% 이상이 보급소 NPC 또는 시설 하나를 기억한다.
```

## 플레이어 유형별 관찰

### 액션 유저

보고 싶은 것:

- 전투가 바로 재미있는가.
- 차징이 시원한가.
- 보스 실패 후 다시 도전하는가.

싫어할 수 있는 것:

- 대사가 길다.
- 정산이 길다.
- 흔적 선택이 느리다.

수정 기준:

- 대사량을 줄인다.
- 정산 화면을 압축한다.
- 전투 보상을 더 즉시 보여준다.

### 성장 유저

보고 싶은 것:

- 첫 개조가 빠른가.
- 전단 조각 수급이 납득되는가.
- 다음 런이 강해졌다고 느끼는가.

싫어할 수 있는 것:

- 보상이 짜다.
- 업그레이드 차이가 약하다.
- 코어 파편 조건이 불명확하다.

수정 기준:

- 첫 3런 보상을 넉넉하게 한다.
- 반품 도장/스캐너 첫 업그레이드 체감을 키운다.
- 보스 HP 구간 보상을 명확히 표시한다.

### 캐릭터 애정 유저

보고 싶은 것:

- 윤서가 기억에 남는가.
- 첫 회수 후 걱정되는가.
- 흔적 반응이 좋은가.

싫어할 수 있는 것:

- 윤서가 너무 무난하다.
- 대사가 너무 적거나 길다.
- 감정선이 너무 어둡다.

수정 기준:

- 윤서 짧은 대사 2~3개를 강화한다.
- 이름표/반품 도장 시각 신호를 강화한다.
- 우울함보다 건조한 유머를 섞는다.

### 세계관 탐색 유저

보고 싶은 것:

- 송출 기록의 의미.
- 송출관과 첫 보스의 차이.
- 흔적 선택의 찜찜함.
- 같은 지역 재방문 변화.

싫어할 수 있는 것:

- 너무 노골적인 설명.
- 선택 결과가 얕음.
- 미스터리가 전혀 보이지 않음.

수정 기준:

- 설명 대신 오브젝트/게시판/짧은 대사로 보여준다.
- 보스 선택 후 변화 폭을 키운다.
- 조작된 구조 신호 같은 씨앗을 한 줄 남긴다.

| 자동 계측 | 의미 |
| --- | --- |
| `charge_uses`, `charge_hits`, `charge_hit_rate` | 반품 도장 사용/명중/손맛 진입 여부 |
| `first_card_time`, `level_up_choices` | 카드 선택 도달 속도와 2장 조합 가능성 |

런 결과 화면에는 아래 형식으로 요약한다.

```text
플레이테스트 계측: 핵심 5/7 | 차징 67%(4/6) | 카드 38s | 후보 91s | 위험 27s
```

자동 QA:

- `tools/playtest_metrics_probe.gd`는 계측 연결을 확인하는 스크립트 런이다.
- `tools/qa_check.ps1`에서 Godot headless로 실행된다.
- 현재 기준은 `first_5_score >= 4`, 결과 화면 계측 라인 표시, 차징/카드/보급태그 후보 기록 존재다.
- `tools/ui_compaction_probe.gd`는 정산/보급소 긴 문구가 모바일 UI를 넘치게 만들지 않는지 확인하는 스크립트 런이다.
- 현재 기준은 정산 패널 12줄 이하, 보급소 요약 9줄 이하, 보급소 로그/힌트/액션 버튼 1줄 유지다.
- `tools/contamination_readability_probe.gd`는 오염 표식이 전투 위험 장판처럼 읽히지 않도록 시각 강도를 확인하는 스크립트 런이다.
- 현재 기준은 오염 표식 최대 4개, 반경 28.5 이하, 낮은 알파, 보스전 숨김, 적 밀집 시 라벨 숨김이다.

주의:

이 값은 사람 플레이 점수가 아니다. 실제 플레이테스트에서는 같은 지표를 보되, 조작 손맛과 위험 구역 욕심은 직접 관찰해야 한다.

3. 반품 도장을 누르는 맛이 있었나?
4. 광고 전단 조각으로 뭘 하는지 이해했나?
5. 송출 기록은 재화처럼 보였나, 단서처럼 보였나?
6. 외곽 주택가를 다시 가고 싶은 이유가 있었나?
7. 보스 선택 후 지역이 바뀐 것처럼 느꼈나?

선택 질문:

1. 기억나는 적이 있었나?
2. 기억나는 NPC 대사가 있었나?
3. 흔적 아이템 중 기억나는 것이 있었나?
4. 너무 길거나 방해되는 문구가 있었나?
5. 다음에 추가된다면 어떤 캐릭터를 써보고 싶은가?

## 실패별 수정표

| 실패 | 원인 후보 | 수정 |
| --- | --- | --- |
| 첫 회수가 죽음처럼 보임 | HUD/문구 약함 | `signal_overload`, 회수선 연출 강화 |
| 첫 개조를 못 찾음 | 보급소 동선 약함 | 정비대 강조, 추천 개조 표시 |
| 윤서가 밋밋함 | 차징/외형/대사 약함 | 반품 도장, 이름표, 짧은 대사 강화 |
| 보상이 짬 | 전단 지급량 낮음 | 첫 3런 전단 보정 |
| 보상이 너무 퍼짐 | 개조 속도 과함 | 3런 이후 비용 조정 |
| 송출 기록이 재화 같음 | UI 표현 문제 | `접근 절차` 표현 강화 |
| 보스 실패가 허무함 | HP 구간 보상 약함 | 코어 파편/패턴 기록 표시 |
| 재방문이 재탕 같음 | 변화가 안 보임 | 게시판/적/대사 변화 확대 |
| 대사가 길다 | 설명 과다 | 한 줄 반응으로 축소 |
| 세계관이 안 보임 | 문구/오브젝트 약함 | 광고 물리 법칙을 적 패턴에 더 연결 |

## 0.2 점수 판정

플레이테스트 후 아래 기준으로 10점 만점 재평가한다.

| 항목 | 목표 |
| --- | ---: |
| 윤서 | 8.0 이상 |
| 반품 도장 | 8.0 이상 |
| 첫 회수 | 8.0 이상 |
| 보상/성장 | 7.5 이상 |
| 외곽 주택가 | 8.0 이상 |
| 보급소 | 7.5 이상 |
| 보스 선택 | 7.5 이상 |
| 전체 재출격 욕구 | 8.0 이상 |

총평 기준:

```text
8.5 이상: 0.2 핵심 루프 강함. 확장 준비 가능.
8.0~8.4: 좋음. 손맛/보상/문구 일부 보정 후 진행.
7.0~7.9: 구조는 맞지만 재미 체감 부족. 핵심 루프 수정 필요.
6.9 이하: 문서 방향보다 실제 플레이가 약함. 윤서/회수/보상부터 재작업.
```

## 금지

- 플레이테스트 전에 세계관을 길게 설명하지 않는다.
- 유저에게 내부 점수표를 보여주지 않는다.
- 유저가 철학을 이해했는지 묻지 않는다.
- “좋았나요?”만 묻고 끝내지 않는다.
- 보스까지 못 갔다고 테스트 실패로만 보지 않는다.
- 액션 유저와 세계관 유저의 반응을 한 지표로 섞지 않는다.
