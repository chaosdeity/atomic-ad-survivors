# Story Cross Validation 0.2

상태: 0.2 스토리 문서 교차 검증 결과  
검증 기준일: 2026-05-16  
범위: `story/` 전체, `META_PROGRESSION_ARCHITECTURE.md`, `RPG_ARCHITECTURE_PLAN.md`

## 목적

이 문서는 0.2 스토리 문서들이 서로 충돌하지 않는지 검증하고, 구현 전에 반드시 고정해야 할 결정을 정리한다.

기준:

```text
문서는 많아져도, 0.2 구현 기준은 하나여야 한다.
```

## 최종 고정 사항

### 0.2 플레이어블

```text
윤서만 실제 플레이어블
```

보류:

- 팝시-제로.
- 나리.
- 고위험 역송출형.

이유:

- 0.2는 캐릭터 수보다 윤서의 손맛, 첫 회수, 보급소 귀환감 검증이 중요하다.
- 기본 캐릭터가 약하면 이후 캐릭터 수익 구조도 약해진다.

### 0.2 로컬

```text
r01_suburb / 외곽 주택가
```

보류:

- 폐쇼핑몰.
- 로봇 서비스센터.
- 방송국.
- 위성 송신지.

이유:

- 외곽 주택가 하나로 지역 생태권, 반복 방문, 흔적, 보스 선택을 증명한다.

### 0.2 보스

```text
스마일 홈 시어머니
```

역할:

- 첫 보스.
- 외곽 주택가의 중간 결절.
- 가족 상품화 법칙의 얼굴.

주의:

```text
스마일 홈 시어머니 != 캠페인 송출관
```

캠페인 송출관은 상위 결절이며 0.2에서는 직접 보스전으로 구현하지 않는다.

### 0.2 보스 선택

확정:

```text
결절 파괴
기억 추출
```

보류:

```text
코어 해방
신호 우회
```

이유:

- 코어 해방은 팝시-제로와 로봇 감정선이 쌓인 뒤 더 강하다.
- 신호 우회는 위험 카드와 캠페인 활용 루트가 보일 때 더 유혹적이다.
- 0.2 첫 보스에서 선택지를 4개로 열면 제작 범위와 설명량이 커진다.

### 0.2 흔적

P0 필수:

```text
trace_nameplate_001
trace_burnt_family_photo_001
trace_nameless_receipt_001
```

P1 권장:

```text
trace_mascot_badge_001
trace_lead_battery_001
trace_silent_radio_001
```

결정 이유:

- P0 3개만으로도 이름, 가족 기억, 윤서의 빈 고객란이 작동한다.
- P1 3개는 팝시-제로 씨앗, 보급소 개조, 조작된 구조 신호를 강화한다.
- 문서상 6개는 유지하되, 구현 필수 기준은 3개로 낮춘다.

### 0.2 재화와 단서

반복 성장 재화:

```text
광고 전단 조각
캠페인 코어 파편
```

스토리 진행 단서:

```text
송출 기록
송출 기록 해석
송출관 접근 절차 확정
```

금지:

```text
좌표 조각 재화
송출 기록 소비 재화
인양 정산
보스 조우 보상 코어 파편
```

## 수정한 충돌

### 충돌 1: 흔적 6개가 필수처럼 보임

문제:

- 일부 문서에서 `흔적 6개`가 0.2 필수처럼 읽혔다.
- 최신 구현 범위 문서에서는 P0 3개만 필수로 잡혀 있었다.

수정:

- `implementation_0_2_story_directive.md`를 `흔적 P0 3개, P1 권장 3개`로 변경.
- `story_integration_notes.md`도 같은 기준으로 변경.
- `r01_suburb_content_lock_0_2.md`와 `godot_scope_0_2_from_story.md`에 고장 난 마스코트 배지를 P1로 추가.

판정:

```text
해결됨
```

### 충돌 2: 0.2 보스 선택이 문서마다 다르게 읽힘

문제:

- `region_evolution_model.md`에 `결절 파괴`와 `코어 해방 또는 기억 추출`로 남은 표현이 있었다.
- 최신 0.2 범위는 `결절 파괴 / 기억 추출`이다.

수정:

- `region_evolution_model.md`를 `결절 파괴 / 기억 추출`로 고정.
- `코어 해방`은 0.3 이후, `신호 우회`는 후반 지역 후보로 명시.

판정:

```text
해결됨
```

### 충돌 3: 송출관이 첫 보스처럼 보일 위험

문제:

- 송출 기록 3개가 모이면 `송출관 접근 절차 확정`이 되므로, 유저나 개발자가 송출관을 바로 첫 보스로 오해할 수 있다.

현재 문서 상태:

- `r01_boss.md`, `r01_suburb.md`, `implementation_0_2_story_directive.md`, `godot_scope_0_2_from_story.md`에서 송출관은 상위 결절로 분리되어 있다.

판정:

```text
주의 필요, 현재 충돌 없음
```

운영 기준:

- 0.2에서 송출관은 직접 전투가 아니라 접근 목표/상위 결절 전조다.
- 첫 실제 보스는 스마일 홈 시어머니다.

### 충돌 4: 내부 점수/철학 UI 노출 위험

문제:

- 캐릭터 점수와 지역 성향 상태가 문서에 많아져서 UI로 노출될 위험이 있다.

현재 문서 상태:

- `player_experience_philosophy.md`, `reward_growth_loop_0_2.md`, `character_score_upgrade_backlog.md`, `godot_scope_0_2_from_story.md`가 모두 비노출 원칙을 명시한다.

판정:

```text
충돌 없음
```

금지:

- 유저에게 `trace_preservation`, `campaign_exploitation`, `sync_risk`, `local_memory_profile`, 캐릭터 점수를 보여주지 않는다.

| --- | ---: | --- |
| 세계관 명확성 | 8 | 캠페인/로봇/인간 구조가 분리됨 |
| 구현 현실성 | 8 | 한 캐릭터, 한 지역, 한 보스로 제한 |
| 반복 재미 | 8 | 재방문/보스 선택/송출 기록이 있음 |


## 남은 주의점

- `broadcast_records_and_recovery.md`에는 `좌표 확정`이라는 표현이 남아 있다. 의미는 재화가 아니라 접근 절차이므로 허용 가능하지만, UI에서는 `송출관 접근 절차 확정`을 우선한다.
- `run_cards.md`와 `trace_items.md`에는 코어 해방/신호 우회 계열 카드와 흔적이 남아 있다. 이는 장기 후보이므로 유지한다.
- `r01_boss.md`에는 전체 선택지 4개가 남아 있다. 단, 0.2 권장 선택이 `결절 파괴 / 기억 추출`로 명시되어 있으므로 유지한다.
- `region_progression_analysis_prompt.md`는 분석 프롬프트이므로 장기 선택지를 넓게 유지해도 된다.
