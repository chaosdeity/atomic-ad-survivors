# Character Score Upgrade Backlog

상태: 캐릭터 10점 개선 백로그  
연결 문서: `story/06_characters/character_affection_monetization.md`, `story/06_characters/player_characters.md`, `story/06_characters/yunseo_0_2_playable_spec.md`, `story/06_characters/character_long_term_model.md`

## 목적

이 문서는 기본 캐릭터와 출시 캐릭터를 업데이트하면서 유저 선호 점수를 계속 올리기 위한 작업 목록이다.

기준:

```text
캐릭터 점수는 출시 시점의 판정이 아니라, 운영하면서 계속 올려야 하는 지표다.
```

목표는 모든 캐릭터를 모두에게 10점으로 만드는 것이 아니다. 각 캐릭터를 좋아할 유저층에게 “이 캐릭터는 내 계정에 있어야 한다”고 느끼게 만드는 것이다.

## 10점의 의미

10점 캐릭터:

- 첫눈에 기억된다.
- 10초 안에 전투 차이가 느껴진다.
- 차징 스킬이 캐릭터의 상징이다.
- 짧은 대사가 유저에게 남는다.
- 같은 지역을 다른 눈으로 보게 한다.
- 스킨/대사/이펙트 확장이 자연스럽다.
- 성능 필수 캐릭터가 아니어도 갖고 싶다.

10점이 아닌 것:

- 단순 최고 딜러.
- 과금 안 하면 손해나는 캐릭터.
- 설정 설명이 긴 캐릭터.
- 예쁘거나 귀엽기만 한 캐릭터.
- 세계관 핵심 정보를 독점하는 캐릭터.

## 점수 축

| 축 | 질문 | 10점 기준 |
| --- | --- | --- |
| 첫눈 매력 | 보자마자 기억되는가 | 실루엣과 핵심 소품이 즉시 읽힘 |
| 전투 손맛 | 직접 쓰면 즐거운가 | 차징 1회로 캐릭터성이 느껴짐 |
| 장기 애정 | 오래 쓸수록 정이 드는가 | 반복 대사와 흔적 반응이 남음 |
| 세계 해석 | 기존 로컬을 다르게 보게 하는가 | 같은 맵이 새 의미를 얻음 |
| 수익 확장 | 스킨/대사/이펙트가 사고 싶은가 | 성능 압박 없이 소장 욕구 발생 |
| 리스크 관리 | 싫어할 이유가 적은가 | 필수화, 과설명, 톤 붕괴가 없음 |

## 운영 규칙

- 기본 캐릭터도 계속 업데이트한다.
- 출시 캐릭터는 출시 후 첫 업데이트에서 반드시 손맛 또는 반응을 보강한다.
- 성능 상향만으로 점수를 올렸다고 보지 않는다.
- 캐릭터 개선은 전투, 대사, 로컬 반응, 스킨 가능성 중 최소 2개를 함께 다룬다.
- 유료 캐릭터가 핵심 세계관 이해를 독점하면 안 된다.
- 무료 유저도 좋은 캐릭터 경험을 가져야 한다.

## 윤서

현재 점수:

```text
8.0 / 10
```

목표:

```text
0.3: 8.5
0.4: 9.0
장기: 10 후보
```

### 강점

- 기본 캐릭터인데 세계관 입구로 강하다.
- 반품 스캐너/반품 도장이 광고 디스토피아와 잘 맞는다.
- 이름표, 빈 고객란, 첫 회수의 감정 훅이 좋다.
- 무료 캐릭터라 유저 신뢰를 만든다.

### 약점

- 일러스트/실루엣이 약하면 평범한 기본캐가 된다.
- 반품 도장이 시원하지 않으면 전투 매력이 무너진다.
- 대사가 너무 적으면 애정이 쌓이지 않는다.
- 대사가 길면 철학 강의처럼 보인다.

### 0.2 필수 개선

| 작업 | 목적 | 유저 효과 |
| --- | --- | --- |
| 반품 도장 VFX/사운드 기준 고정 | 손맛 강화 | 버튼 누르는 맛 |
| 이름표 시각 포인트 고정 | 첫눈 매력 | 윤서를 기억 |
| 첫 회수 대사 3줄 고정 | 감정 이입 | 죽음이 아니라 회수 |
| 흔적 반응 최소 2개 | 장기 애정 | 윤서다운 해석 |
| 외곽 주택가 반복 방문 대사 | 지역 해석 | 같은 맵이 윤서를 기억 |

### 0.3 개선

| 작업 | 목적 |
| --- | --- |
| 반품 도장 적중 시 광고 문구 찢김 강화 | 차징 상징화 |
| 흔적 반응 2개 추가 | 애정 누적 |
| 보스 선택 후 윤서 전용 후속 대사 | 선택 체감 |
| 윤서 기본 스킨 후보 1종 기획 | 수익 확장 씨앗 |

### 0.4 개선

| 작업 | 목적 |
| --- | --- |
| 두 번째 로컬 전용 윤서 반응 | 세계 해석 확장 |
| 반품 스캐너 변형 카드 1개 | 전투 빌드 확장 |
| 보급소 이름 확인 루틴 확장 | 장기 애정 |

### 10점 후보 조건

```text
유저가 윤서를 "기본캐"가 아니라 "Atomic Ad Survivors의 얼굴"로 부른다.
```

## 팝시-제로

현재 점수:

```text
8.3 / 10
```

목표:

```text
출시 전: 8.6
출시: 8.8
출시 후 첫 업데이트: 9.2
장기: 10 후보
```

### 강점

- 귀여운 마스코트 표면이 강하다.
- 로봇 자아/리콜/반품 불안이 감정적으로 좋다.
- 스킨과 대사팩 확장성이 매우 높다.
- 첫 유료/확장 캐릭터 후보로 적합하다.

### 약점

- 귀여움만 밀면 깊이가 사라진다.
- 리롤/드롭 변환이 과하면 필수 캐릭터가 된다.
- 로봇이 정답 진영처럼 보이면 3자 구조가 약해진다.

### 출시 전 필수 작업

| 작업 | 목적 | 유저 효과 |
| --- | --- | --- |
| 전투 판타지 한 줄 고정 | 즉시 이해 | 어떤 캐릭터인지 바로 앎 |
| 차징 스킬 `상쾌한 오작동` 프로토타입 기준 | 손맛 | 윤서와 다른 조작감 |
| 리콜 불안 대사 5개 | 장기 애정 | 귀여움 아래 상처 |
| 마스코트 배지 반응 | 0.2 씨앗 회수 | 이미 본 흔적과 연결 |
| 성능 필수화 방지 규칙 | 리스크 관리 | 과금 압박 방지 |

### 출시 후 첫 업데이트

| 작업 | 목적 |
| --- | --- |
| 로봇 적 전용 반응 3개 | 세계 해석 |
| 같은 모델 다른 기체 스킨 후보 | 소장성 |
| 캠페인 문구를 고쳐 말하는 대사팩 | 캐릭터 말투 강화 |
| 리롤/드롭 변환 수치 점검 | 필수화 방지 |

### 10점 후보 조건

```text
유저가 귀여워서 뽑고, 자아 불안 때문에 계속 쓴다.
```

## 나리

현재 점수:

```text
7.2 / 10
```

목표:

```text
출시 전: 8.0
출시: 8.5
장기: 9+
```

### 강점

- 흔적/이름/보존 시스템과 잘 맞는다.
- 깊게 파는 유저에게 강하다.
- 조용한 감정 이입이 가능하다.

### 약점

- 첫눈 매력과 전투 손맛이 약할 수 있다.
- 방어형으로만 보이면 지루하다.
- 너무 선한 보존 캐릭터가 되면 선택 긴장이 죽는다.

### 출시 전 필수 작업

| 작업 | 목적 | 유저 효과 |
| --- | --- | --- |
| 위험한 회수 전투 판타지 확정 | 손맛 강화 | 안전캐가 아니라 도전캐 |
| 차징 `이름 붙잡기`를 공격/회수 혼합으로 조정 | 전투 재미 | 버튼을 누를 이유 |
| 이름표 핀 시각 모티프 강화 | 첫눈 매력 | 작은 화면 식별 |
| 가짜 감성 광고에 속을 위험 대사 | 깊이 | 보존이 정답만은 아님 |

### 10점에 가까워지는 조건

```text
흔적을 보관하는 유저가 나리를 자기 플레이 성향의 대표 캐릭터로 느낀다.
```

## 고위험 역송출형

현재 점수:

```text
7.5 / 10
```

목표:

```text
출시 전: 8.2
출시: 8.7
장기: 9+
```

### 강점

- 강한 딜러 판타지가 즉시 이해된다.
- 보스 HP 구간 돌파와 잘 맞는다.
- 위험 보상 플레이어에게 매력적이다.

### 약점

- 고과금 딜러처럼 오해받기 쉽다.
- 성능 필수화 위험이 크다.
- 감정 훅이 약하면 그냥 강한 캐릭터가 된다.

### 출시 전 필수 작업

| 작업 | 목적 | 유저 효과 |
| --- | --- | --- |
| 감정 상처 확정 | 장기 애정 | 단순 딜러 방지 |
| 위험 보상 수치 제한 | 리스크 관리 | 필수 캐릭터 방지 |
| 도윤과의 관계선 강화 | 거점 연결 | 보급소 서사 |
| 캠페인 주시 대사 | 세계 해석 | 강할수록 관측됨 |

### 10점에 가까워지는 조건

```text
유저가 강해서 쓰기 시작하지만, 캠페인에게 기억되는 공포 때문에 계속 신경 쓴다.
```

## 캐릭터별 업데이트 우선순위

0.2:

```text
윤서만 플레이어블.
윤서 8.0 달성 여부를 검증한다.
팝시-제로/나리/고위험 역송출형은 흔적과 NPC 대사로 씨앗만 둔다.
```

0.3:

```text
윤서 8.5 개선.
팝시-제로 출시 준비 또는 출시.
팝시-제로 성능 필수화 테스트.
```

0.4:

```text
윤서 9.0 개선.
팝시-제로 출시 후 반응 보강.
나리 전투 손맛 프로토타입.
```

장기:

```text
각 로컬 추가마다 기존 캐릭터 반응 1~2개 추가.
기본 캐릭터 윤서가 절대 낡지 않게 한다.
신규 캐릭터는 성능보다 렌즈와 애정으로 판다.
```

## 점수 검증 질문

캐릭터 업데이트 전:

- 이 업데이트가 점수 축 중 무엇을 올리는가?
- 전투 손맛과 애정 중 최소 하나를 올리는가?
- 기존 로컬을 다르게 보게 하는가?
- 유료 요소가 없어도 캐릭터가 좋아졌는가?

캐릭터 업데이트 후:

- 유저가 이 캐릭터를 다시 써볼 이유가 생겼는가?
- 이 캐릭터를 좋아하던 유저가 더 좋아할 이유가 생겼는가?
- 싫어하던 유저가 최소한 왜 인기 있는지 이해할 수 있는가?
- 성능 때문에 억지로 써야 한다는 불만이 생기지 않는가?

```text
캐릭터 방향은 좋다.

0.2의 최우선 과제는 윤서를 "괜찮은 기본캐"가 아니라 "계속 데리고 가고 싶은 얼굴"로 만드는 것이다.
```

## 금지 방향

- 캐릭터 점수를 성능 상향으로만 올리지 않는다.
- 기본 캐릭터를 유료 캐릭터보다 낡게 두지 않는다.
- 출시 캐릭터를 첫 달 이후 방치하지 않는다.
- 유료 캐릭터에게 핵심 세계관 정보를 독점시키지 않는다.
- 귀여움, 강함, 비극 중 하나만으로 캐릭터를 팔지 않는다.
- 장문 캐릭터 스토리로 애정을 강요하지 않는다.
- 내부 점수표를 유저 UI로 보여주지 않는다.
