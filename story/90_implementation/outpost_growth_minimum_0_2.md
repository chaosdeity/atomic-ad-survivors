# Outpost Growth Minimum 0.2

상태: 0.2 보급소 성장 최소안  
연결 문서: `story/02_hub/silence_outpost.md`, `story/02_hub/npcs.md`, `story/90_implementation/reward_growth_loop_0_2.md`, `META_PROGRESSION_ARCHITECTURE.md`

## 목적

0.2의 침묵 보급소는 큰 마을이 아니다. 하지만 유저가 돌아올 때마다 “조금 나아졌다”, “내가 가져온 것이 여기 남았다”, “다음 출격이 쉬워졌다”를 느껴야 한다.

기준:

```text
보급소는 메뉴가 아니라 돌아오는 장소여야 한다.
```

0.2에서 보급소 성장은 복잡한 건설 시스템이 아니다. 작은 시설 5개, 짧은 NPC 반응, 즉시 체감되는 개조만 있으면 된다.

## 역할

보급소가 유저에게 주는 감정:

- 죽은 것이 아니라 돌아왔다.
- 얻은 것이 헛되지 않았다.
- 윤서와 NPC들이 조금씩 버틴다.
- 다음 런을 더 잘할 수 있다.
- 세계는 무섭지만 여기만큼은 조용하다.

보급소가 시스템에서 하는 일:

- 광고 전단 조각 소비.
- 캠페인 코어 파편 소비.
- 송출 기록 해석.
- 흔적 보관/소모/캠페인 활용 선택.
- 윤서 기본 성장.
- 다음 출격 목표 제시.

## 0.2 시설

| 시설 | 기능 | 0.2 우선순위 |
| --- | --- | --- |
| 정산 카운터 | 런 결과, 재화, 송출 기록 표시 | 필수 |
| 정비대 | 윤서 장비 개조 | 필수 |
| 이름 보관함 | 흔적 보관, 이름 관련 반응 | 필수 |
| 출격 게시판 | 다음 목표, 송출 기록 진행도 | 필수 |
| 차징 조율대 | 반품 도장/회수선 계열 개조 | 높음 |
| 종이 카탈로그 벽 | 카드/광고 전단 분위기, 확장 슬롯 | 선택 |

0.2에서는 시설을 전부 별도 복잡 UI로 만들 필요는 없다. 정산 화면과 보급소 화면 안에서 버튼 또는 간단한 패널로 시작해도 된다.

## 성장 단계

### 단계 0: 첫 출격 전

상태:

- 불빛이 낮다.
- 정비대가 임시 배선으로 묶여 있다.
- 이름 보관함은 비어 있다.
- 출격 게시판에는 외곽 주택가 하나만 있다.

문구:

```text
미나: 여기서는 물건에 가격표를 붙이지 않아. 이름부터 확인해.
```

### 단계 1: 첫 강제 회수 후

해금:

- 정산 카운터.
- 정비대 1단계.
- 윤서 개조 3종 중 1개 선택.

필요:

```text
첫 현장 데이터
광고 전단 조각 3개 이상
```

문구:

```text
도윤: 장비가 버텼어. 그러면 고칠 수 있어.
```

유저 감정:

```text
방금 실패한 게 아니라, 다음 시도를 준비하게 됐다.
```

### 단계 2: 송출 기록 1개 확보

해금:

- 출격 게시판에 송출 기록 진행도 표시.
- 미나의 해석 문구.
- 다음 신호 이벤트 힌트.

문구:

```text
미나: 광고 문구 사이에 길이 끼어 있어. 이상하지? 광고가 길을 알려주는 척한다는 게.
```

### 단계 3: 흔적 1개 보관

해금:

- 이름 보관함 첫 반응.
- 보급소 내 작은 오브젝트 변화.
- 흔적 관련 NPC 대사 1개.

문구:

```text
유리: 이건 팔 물건이 아니야. 그러니까 여기 둬도 돼.
```

유저 감정:

```text
내가 주운 물건이 장소를 바꿨다.
```

### 단계 4: 보스 HP 65% 이하 도달

해금:

- 캠페인 코어 파편 사용.
- 보스 대응 개조 1개.
- 세븐의 패턴 기록 문구.

문구:

```text
세븐: 표면 손상이 아닙니다. 이번에는 결절 안쪽이 반응했습니다.
```

### 단계 5: 첫 보스 처리

해금:

- 외곽 주택가 후속 상태.
- 보급소 대화 2개.
- 다음 송출관 접근 절차 안내.
- 종이 카탈로그 벽에 보스 관련 찢긴 광고 1장 추가.

문구:

```text
미나: 끝난 건 아니야. 그래도 오늘은 저쪽 집들이 조금 덜 웃고 있어.
```

## 0.2 개조 목록

| 개조 | 비용 | 시설 | 효과 |
| --- | ---: | --- | --- |
| 스캐너 출력 보정 | 전단 2 | 정비대 | 윤서 자동공격 피해/안정성 증가 |
| 반품 도장 압력 보정 | 전단 2 | 차징 조율대 | 차징 피해 증가 |
| 회수선 절연 보강 | 전단 2 | 차징 조율대 | 신호 과부하 경고 여유 증가 |
| 보급소 필터 교체 | 전단 3 | 정산 카운터 | 송출 기록 해석 문구 확장 |
| 결절 파편 차폐 | 코어 1 | 정비대 | 보스 패턴 대응 보정 |
| 기록판 보강 | 전단 2 | 출격 게시판 | 다음 신호 이벤트 힌트 표시 |

0.2 권장:

- 첫 회수 후 1개는 반드시 살 수 있어야 한다.
- 3런 안에 2~3개를 살 수 있어야 한다.
- 코어 파편 개조는 보스에게 의미 있는 피해를 준 뒤에만 열린다.

## 보급소 변화 표현

보급소 성장은 숫자보다 작은 변화로 보여준다.

예시:

| 조건 | 변화 |
| --- | --- |
| 첫 개조 완료 | 정비대 조명이 하나 켜짐 |
| 흔적 보관 | 이름 보관함 위에 작은 라벨 추가 |
| 송출 기록 2개 | 출격 게시판에 찢긴 지도선 표시 |
| 보스 HP 25% 이하 | 코어 파편 보관 유리병 등장 |
| 결절 파괴 | 보급소 밖 광고 소음이 약해짐 |
| 기억 추출 | 보관함 주변에 가족사진 반응 대사 추가 |

주의:

- 변화는 과장된 건설물이 아니라 생활감 있는 물건이어야 한다.
- “레벨 3 보급소” 같은 UI 명칭을 쓰지 않는다.
- 유저가 못 봐도 괜찮지만, 본 유저는 기뻐해야 한다.

## NPC 반응

0.2에서는 상태 하나당 NPC 대사 1개면 충분하다.

첫 개조:

```text
도윤: 됐다. 이제 적어도 네가 찍은 건 더 확실히 찢길 거야.
```

첫 흔적 보관:

```text
유리: 물건이 조용해지는 건 오랜만에 봐.
```

송출 기록 3개:

```text
미나: 길이 맞춰졌어. 문제는, 저쪽도 우리가 온다는 걸 알 거라는 거야.
```

코어 파편 첫 회수:

```text
세븐: 이것은 돈이 아닙니다. 더 나쁜 것입니다. 쓸모가 있습니다.
```

첫 보스 처리:

```text
한별: 오늘은 라디오가 덜 웃어. 그게 좋은 건지는 모르겠어.
```

## 확장성

지금은 작은 보급소지만, 나중에 마을이 될 수 있다. 그래서 0.2 문서는 “마을 금지”가 아니라 “0.2에서 과욕 금지”다.

장기 확장 후보:

- 피난민 임시 침상.
- 해방 로봇 충전 구역.
- 이름 보관소 확장.
- 흔적 박물대.
- 위험 개조실.
- 침묵권 확장 장치.

확장 조건:

```text
로컬 2개 이상
NPC 관계 축 2개 이상
인구 압력 시스템 도입
보급소 변화가 전투/정산/대사에 실제 영향을 줄 때
```

## 유저 선호 점수

0.2 보급소 성장 예상:

| 기준 | 점수 | 이유 |
| --- | ---: | --- |
| 즉시 이해 | 8 | 정산/개조/출격 구조가 단순함 |
| 성장 체감 | 8 | 첫 회수 후 바로 개조 가능 |
| 감정 귀환감 | 8 | NPC 짧은 반응과 흔적 보관이 좋음 |
| 장기 확장성 | 9 | 마을/인구/로봇 구역으로 확장 가능 |
| 구현 안정성 | 8 | 0.2 시설 수가 제한됨 |
| 리스크 | 7 | 메뉴처럼 보이면 매력이 크게 떨어짐 |

종합:

```text
8.0 / 10
```

판정:

0.2 보급소는 좋은 출발점이다. 다만 시각 변화와 NPC 반응이 부족하면 그냥 업그레이드 메뉴로 느껴질 수 있다. 유저가 좋아하게 만들려면 “개조를 눌렀다”보다 “내가 돌아온 곳이 조금 살아났다”가 먼저 와야 한다.

## 0.2 합격 기준

- 첫 회수 후 보급소에서 개조 1개를 바로 할 수 있다.
- 보급소가 인양 후 메뉴가 아니라 귀환 장소로 느껴진다.
- NPC 대사는 짧고 기능과 감정을 동시에 준다.
- 흔적 1개 보관 시 보급소 변화가 보인다.
- 송출 기록 진행도가 출격 게시판과 연결된다.
- 보스 성과가 코어 파편과 보급소 반응으로 이어진다.
- 보급소 성장은 숫자 레벨이 아니라 오브젝트/대사/기능으로 보인다.

## 금지 방향

- 0.2부터 마을 건설 게임처럼 만들지 않는다.
- 보급소 레벨 UI를 앞세우지 않는다.
- NPC 대사를 긴 설명으로 쓰지 않는다.
- 첫 회수 후 아무것도 못 하게 하지 않는다.
- 흔적을 단순 재료로만 쓰지 않는다.
- 유료 캐릭터나 스킨이 보급소 핵심 기능을 잠그지 않는다.
