# Trace Reward Directive 0.2

상태: 0.2 흔적 아이템/보상 적용 작업 지시서  
목적: 흔적을 단순 재료가 아니라 전투, 보급소, 캐릭터 애정, 지역 반응에 연결한다.  
기준: 유저가 흔적을 보고 "이걸 그냥 써도 되나?" 하고 잠깐 망설여야 한다.

## 핵심 목표

0.2 흔적은 수집품 도감이 아니다. 런에서 회수하고, 보급소에서 보관하거나 소모하거나 활용하며, 다음 출격의 문구와 상태를 바꾸는 물건이다.

유저가 느껴야 할 것:

```text
이 물건은 효과만 있는 재료가 아니다.
내가 고른 처리 방식이 보급소와 지역에 남는다.
윤서가 이 물건을 그냥 지나치지 않는다.
```

## 포함 범위

### P0 필수 흔적 3개

| ID | 이름 | 핵심 역할 |
| --- | --- | --- |
| `trace_nameplate_001` | 지워지지 않은 문패 | 이름/집/인간 안정 |
| `trace_burnt_family_photo_001` | 반쯤 탄 가족사진 | 기억 추출/감정 보존 |
| `trace_nameless_receipt_001` | 이름 없는 영수증 | 윤서의 빈 고객란/캠페인 활용 위험 |

### P1 권장 흔적 3개

| ID | 이름 | 핵심 역할 |
| --- | --- | --- |
| `trace_mascot_badge_001` | 고장 난 마스코트 배지 | 팝시-제로/로봇 씨앗 |
| `trace_lead_battery_001` | 납 차폐 배터리 | 차징 안정/침묵권 |
| `trace_silent_radio_001` | 무음 라디오 | 조작된 구조 신호/침묵 반응 |

0.2 일정이 빠듯하면 P0 3개만 넣는다. 단, P0 3개는 반드시 윤서 반응, 보급소 반응, 지역 반응 중 2개 이상과 연결해야 한다.

## 데이터 구조

권장 필드:

```text
TraceItem
  id
  name
  short_description
  source_region
  source_type
  rarity
  related_npcs
  related_character_lines
  combat_effect_id
  hub_story_effect_id
  preserve_effect
  consume_effect
  campaign_use_effect
  unlocks
  ending_axis
  priority
```

필드 원칙:

- `short_description`은 1문장으로 끝낸다.
- `combat_effect_id`와 `hub_story_effect_id`를 분리한다.
- `preserve_effect`, `consume_effect`, `campaign_use_effect`는 내부 상태와 visible reaction을 함께 가진다.
- 내부 점수명은 유저에게 보여주지 않는다.

## 선택 처리

흔적 회수 후 기본 선택:

```text
보관
소모
캠페인 활용
```

0.2에서 모든 흔적에 세 선택을 전부 열 필요는 없다. 하지만 데이터 구조는 세 선택을 받을 수 있어야 한다.

### 보관

의미:

- 이름 보관함에 남긴다.
- NPC와 윤서 반응이 열린다.
- 지역/엔딩 축에 기억이 남는다.

유저 표현:

```text
이름 보관함에 보관했습니다.
```

### 소모

의미:

- 시설, 장비, 차징 계열 개조에 사용한다.
- 즉시 성장은 좋지만 해당 흔적은 보관함에 남지 않는다.

유저 표현:

```text
정비 재료로 사용했습니다.
```

### 캠페인 활용

의미:

- 위험 보상, 카드, 신호 우회 계열 해금에 사용한다.
- `campaign_pressure` 또는 `sync_risk`가 오른다.

유저 표현:

```text
캠페인 신호에 맞춰 변조했습니다.
```

주의:

- 캠페인 활용은 악한 선택이 아니라 강한 유혹이어야 한다.
- 단, 다음 런에서 지역이 유저를 더 정확히 부르는 식의 반응이 있어야 한다.

## P0 흔적 상세

### trace_nameplate_001

```text
name = "지워지지 않은 문패"
short_description = "페인트 아래에서 아직 사람 이름이 보입니다."
source_region = "r01_suburb"
source_type = ["object", "elite_drop", "boss_destroy_node_reward"]
priority = "P0"
```

전투 효과:

```text
최대 체력이 소폭 증가합니다.
```

보관 효과:

```text
이름 보관함에 문패 라벨이 추가됩니다.
외곽 주택가의 일부 문패가 광고판으로 변하는 속도가 느려집니다.
```

소모 효과:

```text
보급소 침상/회수선 안정 개조 재료로 사용합니다.
```

윤서 반응:

```text
윤서: 좋아. 이건 아직 주소가 아니라 집이야.
```

NPC 반응:

```text
미나: 문패는 작아도 버티는 힘이 있어.
```

### trace_burnt_family_photo_001

```text
name = "반쯤 탄 가족사진"
short_description = "얼굴은 흐리지만 손을 잡은 자세는 남아 있습니다."
source_region = "r01_suburb"
source_type = ["boss_extract_memory_reward", "memory_afterimage_event"]
priority = "P0"
```

전투 효과:

```text
체력이 낮을 때 1회성 작은 방어막을 생성합니다.
```

보관 효과:

```text
이름 보관함에 사진 잔향이 추가됩니다.
기억 추출 후 재방문 문구가 강화됩니다.
```

소모 효과:

```text
일회성 보호 계열 장비 개조 재료로 사용합니다.
```

윤서 반응:

```text
윤서: 얼굴이 없어도, 손은 남아 있네.
```

NPC 반응:

```text
복희: 얼굴보다 손이 오래 기억하는 법이 있단다.
```

### trace_nameless_receipt_001

```text
name = "이름 없는 영수증"
short_description = "산 물건은 많은데 구매자 이름만 비어 있습니다."
source_region = "r01_suburb"
source_type = ["mailbox_drop", "boss_extract_memory_reward", "signal_event"]
priority = "P0"
```

전투 효과:

```text
획득 재화 또는 경험치가 증가하지만 캠페인 농도도 상승합니다.
```

보관 효과:

```text
이름 보관함에 빈 구매자 칸 경고가 추가됩니다.
윤서의 빈 고객란 시나리오 씨앗이 강화됩니다.
```

소모 효과:

```text
고위험 카드 후보 또는 정산 효율 보정에 사용합니다.
```

캠페인 활용 효과:

```text
위험 카드 출현 확률이 오르고 외곽 주택가가 윤서를 더 직접적으로 호명합니다.
```

윤서 반응:

```text
윤서: 제일 싫은 종류네. 산 건 있는데 산 사람이 없어.
```

NPC 반응:

```text
복희: 빈칸을 그냥 두렴. 가끔은 그게 경고란다.
```

캠페인 활용 문구:

```text
캠페인: 구매자 정보가 업데이트되었습니다.
```

## P1 흔적 상세

### trace_mascot_badge_001

핵심:

- 팝시-제로 0.3 출시 씨앗.
- 로봇이 상품이었고, 아직 이름을 찾고 있다는 감정 연결.

대표 문구:

```text
웃는 얼굴 반쪽이 손바닥에 남았습니다.
팝시: 이 얼굴, 저랑 닮았나요? 아니면 제가 닮은 건가요?
```

해금 후보:

```text
card_mascot_handshake_001
```

### trace_lead_battery_001

핵심:

- 차징 안정성.
- 침묵권/보급소 장비 개조.

대표 문구:

```text
묵직한 침묵이 손에 들어왔습니다.
도윤: 이건 기록해둘 가치가 있어. 진짜 침묵은 희귀하거든.
```

해금 후보:

```text
return_stamp_coolant_01
card_quiet_warranty_001 candidate
```

### trace_silent_radio_001

핵심:

- 침묵이 보상인 세계관을 보여준다.
- 조작된 구조 신호 씨앗을 너무 직접적이지 않게 심는다.

대표 문구:

```text
아무 방송도 잡히지 않습니다. 그래서 귀합니다.
세븐: 신호 없음. 이상합니다. 안정적입니다.
```

해금 후보:

```text
card_premium_silence_001
malice_thread.faked_rescue_signal = "rumor"
```

## 보급소 연결

필수 연결:

| 흔적 상태 | 보급소 변화 |
| --- | --- |
| 문패 보관 | 이름 보관함에 문패 라벨 추가 |
| 가족사진 보관 | 보관함 주변에 사진 잔향 문구 추가 |
| 영수증 보관 | 정산 카운터에 빈 구매자 칸 경고 추가 |
| 마스코트 배지 보관 | 종이 카탈로그 벽에 찢긴 마스코트 얼굴 추가 |
| 납 차폐 배터리 소모 | 차징 조율대 불빛 안정화 |
| 무음 라디오 보관 | 출격 게시판 주변 광고 소음 감소 |

주의:

- 변화는 작아도 된다.
- 유저가 직접 알아차리면 기뻐야 한다.
- 모든 변화를 설명 팝업으로 띄우지 않는다.

## 정산 화면 연결

흔적 회수 시 정산 문구 예:

```text
흔적 회수: 지워지지 않은 문패
페인트 아래에서 아직 사람 이름이 보입니다.
```

선택 화면 문구:

```text
보관: 이름 보관함에 남깁니다.
소모: 보급소 개조 재료로 사용합니다.
활용: 캠페인 신호에 맞춰 변조합니다.
```

주의:

- 정산 화면에서 장문 사연을 쓰지 않는다.
- 선택 결과는 다음 런/보급소/NPC 반응으로 이어진다.

## 지역 반응

P0 흔적에 따른 외곽 주택가 반응:

| 조건 | 다음 런 반응 |
| --- | --- |
| 문패 보관 | 같은 골목의 문패 광고화가 조금 늦어진다 |
| 가족사진 보관 | 가족사진 잔상이 더 선명해진다 |
| 영수증 캠페인 활용 | 캠페인이 윤서를 고객명에 가깝게 부른다 |
| 영수증 보관 | 빈 고객란 관련 배너가 드물게 뜬다 |

예시:

```text
캠페인: 윤서 고객님께 어울리는 가족 구성을 준비했습니다.
윤서: 내 가족을 네가 정하지 마.
```

## 유저 선호 점수

예상 점수:

| 기준 | 점수 | 이유 |
| --- | ---: | --- |
| 감정 무게 | 8 | 문패/사진/영수증이 강함 |
| 선택 고민 | 8 | 보관과 소모가 모두 의미 있음 |
| 전투 연결 | 7 | 효과는 있지만 수치 조정 필요 |
| 보급소 귀환감 | 8 | 보관함 변화와 NPC 반응이 좋음 |
| 장기 확장성 | 9 | 캐릭터/로컬/엔딩 축 연결이 좋음 |
| 구현 현실성 | 8 | P0 3개면 작게 시작 가능 |

종합:

```text
8.0 / 10
```

10점으로 올리기 위한 다음 개선:

- 흔적 보관 후 윤서 전용 런 시작 대사 추가.
- 팝시-제로 출시 시 마스코트 배지 반응 강화.
- 흔적을 캐릭터 스킨/대사팩과 연결하되 필수 정보는 무료 유지.
- 폐쇼핑몰에서 쿠폰/반품/마스코트 흔적군 확장.

## 검증 항목

| 관찰 | 합격 기준 | 실패 시 수정 |
| --- | --- | --- |
| 흔적 이해 | 유저가 흔적을 재화와 다르게 인식 | 설명/보관함 연출 강화 |
| 선택 고민 | 보관/소모 중 한쪽만 정답처럼 보이지 않음 | 보상 수치 조정 |
| 윤서 애정 | 흔적 반응 후 윤서가 더 선명해짐 | 윤서 대사 재작성 |
| 보급소 변화 | 물건이 남았다는 감각이 있음 | 오브젝트 변화 추가 |
| 반복 런 연결 | 다음 출격에서 작은 변화가 보임 | 게시판/배너/적 이름 연결 |

## 금지 방향

- 흔적을 단순 강화 재료로만 만들지 않는다.
- 모든 흔적에 긴 회상 컷신을 붙이지 않는다.
- 보관 선택만 도덕적 정답으로 만들지 않는다.
- 소모 선택을 악한 선택으로 만들지 않는다.
- 캠페인 활용을 숨은 함정으로만 만들지 않는다. 강하고 찝찝한 선택이어야 한다.
- 내부 성향 수치를 UI에 노출하지 않는다.
