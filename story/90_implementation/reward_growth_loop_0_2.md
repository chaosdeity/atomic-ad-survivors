# Reward Growth Loop 0.2

상태: 0.2 보상/성장 루프 기준  
연결 문서: `META_PROGRESSION_ARCHITECTURE.md`, `implementation_0_2_story_directive.md`, `story/05_progression/broadcast_records_and_recovery.md`, `story/06_characters/yunseo_0_2_playable_spec.md`

## 목적

이 문서는 0.2에서 유저가 왜 한 번 더 출격하고 싶은지, 무엇을 얻었기 때문에 강해지는지, 보급소와 지역이 어떻게 반응해야 하는지 정의한다.

기준은 하나다.

```text
유저가 좋아할 것인가?
```

0.2 보상 루프는 철학을 설명하는 장치가 아니다. 유저에게 먼저 보여야 하는 것은 시원한 전투, 납득되는 정산, 즉시 체감되는 개조, 조금씩 이상해지는 지역, 좋아지고 싶은 기본 캐릭터다.

## 핵심 루프

```text
출격
-> 자동공격/회피/차징으로 전투 성과 획득
-> 흔적, 송출 기록, 광고 전단 조각, 코어 파편 조건 달성
-> 회수/인양/보스 처리로 보급소 귀환
-> 정산
-> 윤서/장비/보급소/지역 상태 변화
-> 다음 출격
```

중요한 원칙:

- 인양이나 회수는 보상 원인이 아니다.
- 보상 원인은 런 안에서 달성한 생존 시간, 처치, 엘리트 처치, 신호 이벤트, 보스 피해, 흔적 회수다.
- 회수는 정산 지점이다.
- 첫 전투 강제 회수도 보상 지급 장치가 아니라, 첫 현장 데이터가 보급소로 돌아오는 사건이다.

## 0.2 보상 종류

| 종류 | 예시 | 역할 | UI 노출 |
| --- | --- | --- | --- |
| 런 중 성장 | 카드, 임시 스탯 | 해당 런의 재미 | 노출 |
| 반복 재화 | 광고 전단 조각 | 기본 개조 재료 | 노출 |
| 보스 재화 | 캠페인 코어 파편 | 보스 대응/상위 개조 | 노출 |
| 스토리 단서 | 송출 기록 | 송출관 접근 진행 | 진행도만 노출 |
| 흔적 | 문패, 가족사진, 무음 라디오 | 선택/대사/지역 반응 | 노출 |
| 내부 상태 | 지역 기억, 성향, 동기화 위험 | 세계 반응 제어 | 비노출 |

유저에게 보여줄 문장:

```text
회수 완료
현장 성과가 정산되었습니다
```

유저에게 보여주지 말 문장:

```text
인양 정산 획득
local_memory_profile +1
campaign_exploitation_score 증가
```

## 첫 1시간 목표

### 0~10분

유저가 알아야 할 것:

- 움직이면 자동공격이 나간다.
- 차징 버튼은 시원하다.
- 첫 회수는 죽음이 아니라 보급소가 끌어당긴 사건이다.
- 첫 개조를 바로 할 수 있다.

성공 감정:

```text
아, 한 번 더 나가면 바로 더 강해지겠네.
```

### 10~30분

유저가 알아야 할 것:

- 오래 버티면 신호 이벤트가 나온다.
- 흔적을 주우면 보급소 반응이 바뀐다.
- 송출 기록은 보스 접근과 연결된다.
- 윤서의 반품 도장은 업그레이드할수록 확실히 좋아진다.

성공 감정:

```text
방금 뭔가 중요한 걸 찾았다.
```

### 30~60분

유저가 알아야 할 것:

- 송출 기록 3개가 모이면 송출관 접근 절차가 열린다.
- 보스에게 유의미한 피해를 주면 코어 파편을 얻는다.
- 보스를 못 깨도 성과가 있으면 다음 시도가 쉬워진다.
- 보스를 처리하면 지역이 끝나는 것이 아니라 다르게 변한다.

성공 감정:

```text
이번엔 못 깼지만 다음엔 깰 수 있겠다.
```

## 광고 전단 조각

역할:

- 0.2 기본 성장 재화.
- 윤서 장비, 보급소 장비, 역송출 안정화에 사용.
- 캠페인 물질을 보급소가 분해해 재사용하는 재료다.

세계관 정의:

```text
광고 전단 조각은 캠페인 환경에서 떨어져 나온 판촉 물질이다.
겉으로는 종이 조각처럼 보이지만, 안에는 신호 잔여물과 방사성 접착층이 얇게 남아 있다.
침묵 보급소는 이것을 정화해 역송출 장비의 절연재, 필터, 출력 보정판으로 쓴다.
```

획득 기준 초안:

| 조건 | 지급 | 메모 |
| --- | ---: | --- |
| 45초 미만 회수 | 0 | 너무 이른 실패 보상 방지 |
| 60초 생존 | +1 | 첫 성과 |
| 90초 생존 | +1 | 초반 안정 보상 |
| 120초 신호 이벤트 성공 | +1~2 | 목표 보상 |
| 180초 신호 이벤트 성공 | +2 | 중반 성과 |
| 240초 신호 이벤트 성공 | +2~3 | 보스 접근 전 성과 |
| 엘리트 처치 | +1 | 위험 대응 보상 |
| 처치 수 보너스 | +1, 최대 3 | 60초 이후부터 적용 |

첫 강제 회수 보정:

```text
첫 런은 signal_overload로 반드시 회수된다.
이때 기본 지급은 "첫 현장 데이터 정산"으로 처리한다.
권장: 광고 전단 조각 3개 + 달성 성과 0~3개.
```

문구 후보:

```text
판촉 물질 회수
보급소 필터로 정화 가능
```

정산 문구:

```text
광고 전단 조각 +3
도윤: 종이처럼 보여도 안쪽은 신호 찌꺼기야. 버리지 마.
```

## 캠페인 코어 파편

역할:

- 보스전 성과 재화.
- 보스 대응 개조, 상위 성장, 송출관 연구에 사용.
- 보스 얼굴만 봤다고 주지 않는다.

세계관 정의:

```text
캠페인 코어 파편은 결절 내부의 반복 송출 핵이 깨질 때 떨어지는 고밀도 신호 결정이다.
잡몹이나 일반 광고 물질에는 없다.
보스 외피를 실제로 깨고 내부 송출 구조를 흔들어야만 회수할 수 있다.
```

획득 기준 초안:

| 조건 | 지급 | 메모 |
| --- | ---: | --- |
| 보스 조우만 함 | 0 | 구경 보상 금지 |
| 보스 HP 65% 이하 첫 도달 | +1 | 의미 있는 피해 |
| 보스 HP 25% 이하 첫 도달 | +1 | 깊은 피해 |
| 보스 처치/해방/기억 추출 성공 | +2 | 처리 보상 |
| 같은 런 반복 지급 | 제한 | 파밍 폭주 방지 |

문구 후보:

```text
코어 파편 회수
송출 구조 일부가 떨어져 나왔습니다
```

정산 문구:

```text
캠페인 코어 파편 +1
세븐: 표면이 아닙니다. 이번에는 안쪽을 건드렸습니다.
```

## 송출 기록

역할:

- 재화가 아니라 스토리 진행 단서.
- 120초, 180초, 240초 전후의 신호 이벤트와 연결한다.
- 시간만 버티는 보상이 아니라, 이벤트 목표를 달성해야 확보된다.

권장 구조:

```text
시간 도달
-> 신호 이벤트 출현
-> 목표 달성/생존/처치
-> 송출 기록 확보
-> 보급소 해석
-> 송출관 접근 절차 진척
```

UI 표현:

```text
송출 기록 1/3 확보
```

금지 표현:

```text
좌표 조각 +1
스토리 재화 획득
```

보급소 문구:

```text
미나: 이건 광고가 아니라 길 안내 같아. 다만, 길이 우리한테 친절하진 않네.
```

## 0.2 소비 루프

0.2의 영구 성장은 작고 즉각적이어야 한다.

권장 첫 개조:

| 개조 | 비용 | 유저 체감 |
| --- | ---: | --- |
| 반품 스캐너 출력 +1 | 광고 전단 조각 2 | 초반 처치 안정 |
| 반품 도장 피해 +1 | 광고 전단 조각 2 | 차징 손맛 강화 |
| 회수선 안정화 +1 | 광고 전단 조각 2 | 회수 납득/경고 여유 |
| 보급소 필터 보강 | 광고 전단 조각 3 | 정산/해석 문구 확장 |
| 코어 공진 억제 | 코어 파편 1 | 보스 재도전 보정 |

첫 회수 후 유저는 반드시 하나 이상 개조할 수 있어야 한다.

권장 흐름:

```text
첫 회수 -> 개조 1개
2~3런 -> 개조 2~3개
첫 보스 실패 -> 코어 파편 또는 보스 대응 힌트
첫 보스 처리 -> 지역 반응 + 상위 개조 후보
```

## 정산 화면

정산 화면은 길면 안 된다. 유저가 바로 다음 선택으로 넘어가야 한다.

필수 표시:

- 생존 시간.
- 처치 수.
- 광고 전단 조각.
- 캠페인 코어 파편.
- 송출 기록 또는 흔적.
- NPC 짧은 반응 1개.
- 추천 개조 1개.

문구 예시:

```text
회수 완료
생존 02:14 / 처치 184
광고 전단 조각 +5
송출 기록: 잡음 속 주소 확보
```

NPC 반응 예시:

```text
도윤: 다음엔 도장 출력부터 손보자. 방금 그건 맞췄는데도 덜 찢겼어.
```

숨길 것:

- 내부 성향 점수.
- 로컬 기억 프로필.
- 캠페인 활용 수치.
- 철학적 판정명.

## 흔적 루프

흔적은 재화가 아니다. 유저가 세계에 감정을 붙이는 물건이다.

0.2 흔적 처리:

```text
회수
-> 보관/소모/캠페인 활용 중 하나 선택
-> NPC 짧은 반응
-> 지역 또는 대사 조건 변화
```

유저가 느껴야 하는 것:

- 물건 하나가 다음 런의 공기를 바꾼다.
- 선택이 거창한 분기표가 아니라 작은 찜찜함과 작은 보람을 남긴다.
- 같은 지역이 나를 조금 기억한다.

문구 예시:

```text
지워지지 않은 문패 보관
다음 출격부터 같은 골목의 문패가 더 늦게 광고판으로 변합니다.
```

## 보스 실패와 재도전

보스전은 실패해도 성과가 남아야 한다. 단, 보스에게 의미 있는 피해를 줬을 때만 남는다.

실패 정산 예시:

```text
보스 외피 손상: 65%
캠페인 코어 파편 +1
보스 패턴 기록: 가족 추천장
```

NPC 반응:

```text
세븐: 다음 송출 주기에는 두 번째 추천장이 0.8초 빨리 옵니다. 그것만 기억하십시오.
```

유저 감정:

```text
졌지만 헛수고가 아니었다.
```

금지:

- 보스 입장만 해도 코어 파편 지급.
- 아무 성과 없는 인양 정산.
- 실패 정산을 너무 길게 설명.

## 지역 반응

0.2 외곽 주택가는 클리어 후 끝나는 맵이 아니다.

보스 처리 후 기본 반응:

| 선택 | 지역 반응 | 유저 체감 |
| --- | --- | --- |
| 결절 파괴 | 일부 집이 조용해짐 | 안전 구역처럼 보이는 틈 |
| 기억 추출 | 가족 흔적이 더 선명해짐 | 미스터리 깊어짐 |

반복 방문 반응:

```text
캠페인: 지난 방문 기록을 반영한 가족 혜택을 준비했습니다.
윤서: 내 기록으로 네 가족 만들지 마.
```

주의:

- 지역 상태 UI를 복잡하게 만들지 않는다.
- 보이는 것은 현상, 문구, 적 배치, NPC 반응이다.
- 내부 상태는 설계자가 관리하고 유저에게 표로 보여주지 않는다.

## 캐릭터 점수 개선 루프

출시 캐릭터와 기본 캐릭터의 점수는 고정값이 아니다. 모든 캐릭터는 업데이트를 통해 10점에 가까워질 수 있어야 한다.

10점의 의미:

```text
모든 유저가 좋아한다는 뜻이 아니다.
그 캐릭터를 좋아할 유저층에게는 대체 불가능하다는 뜻이다.
```

개선 축:

| 축 | 올리는 방법 | 위험 |
| --- | --- | --- |
| 첫눈 매력 | 실루엣, 초상, 전투 자세, 이름표 | 과장된 장식 |
| 전투 손맛 | VFX, 사운드, 타격감, 차징 리듬 | 성능 인플레 |
| 장기 애정 | 짧은 대사, 반복 방문 반응, 흔적 반응 | 장문 설명 |
| 세계 해석 | 로컬별 관점 차이 | 유료 캐릭터 정보 독점 |
| 수익 확장 | 스킨, 대사팩, 이펙트 변형 | 필수 과금 압박 |
| 리스크 관리 | 밸런스, 무료 진행 보장 | 밋밋함 |

윤서 개선 목표:

| 단계 | 목표 점수 | 필요한 개선 |
| --- | ---: | --- |
| 0.2 | 8.0 | 기본 손맛, 첫 회수, 이름표 각인 |
| 0.3 | 8.5 | 반품 도장 VFX/사운드 강화, 흔적 반응 확대 |
| 0.4 | 9.0 | 외곽 주택가 이후 로컬별 전용 대사, 스킨 1종 |
| 장기 | 10 후보 | 유저가 윤서를 기본캐가 아니라 게임의 얼굴로 기억 |

팝시-제로 개선 목표:

| 단계 | 목표 점수 | 필요한 개선 |
| --- | ---: | --- |
| 출시 전 | 8.3 | 귀여운 표면과 로봇 불안 고정 |
| 출시 | 8.8 | 리롤/드롭 변환 손맛 검증 |
| 출시 후 | 9.2 | 자아 오류 대사, 마스코트 스킨, 로봇 로컬 반응 |
| 장기 | 10 후보 | 귀여움 때문에 샀다가 상처 때문에 남는 캐릭터 |

나리 개선 목표:

| 단계 | 목표 점수 | 필요한 개선 |

## 0.2 합격 기준

- 첫 강제 회수 후 유저가 최소 1개 개조를 할 수 있다.
- 유저가 죽어서 보상받았다고 느끼지 않는다.
- 120초 이후 신호 이벤트가 “시간 보상”이 아니라 “목표 보상”으로 느껴진다.
- 보스 실패 후에도 다음 시도를 하고 싶다.
- 광고 전단 조각과 캠페인 코어 파편의 차이가 명확하다.
- 송출 기록은 재화가 아니라 단서로 느껴진다.
- 윤서의 반품 도장이 보상 루프의 중심 손맛이 된다.
- 보급소 반응이 짧고 인간적이다.
- 지역 반응이 표로 설명되지 않고 현상으로 보인다.
- 캐릭터 점수 개선 항목이 다음 업데이트 백로그로 이어진다.

- 인양 정산처럼 보이게 만들지 않는다.
- 0.2부터 재화를 3개 이상 늘리지 않는다.

- 내부 성향 점수를 UI로 보여주지 않는다.
- 정산 화면을 설명문으로 채우지 않는다.
- 기본 캐릭터를 일부러 약하게 만들지 않는다.
- 유료 캐릭터가 핵심 세계관 이해를 독점하지 않는다.
- 보스 조우만으로 코어 파편을 지급하지 않는다.
- 지역 변화가 유저에게 전혀 보이지 않게 하지 않는다.
