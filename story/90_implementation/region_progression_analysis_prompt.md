## 핵심 전제

게임 한 줄 정의:

> 작은 침묵 보급소를 중심으로, 캠페인 환경에 잠식된 지역에 출격해 싸우고, 회수한 흔적과 선택으로 인간/해방 로봇/세계의 상태를 바꿔가는 런 기반 액션 RPG.

세계 핵심:

> 세상은 멸망하지 않았다. 다만 모든 것이 광고가 되었다.

캠페인은 악당 AI, 기업 하나, 로봇 군대가 아니다. 라이프라인 잔여 절차, 광고 신호, 자동화 인프라, 회색 오염, 소비 욕망, 기계 학습 모델이 결합한 환경층이다. 지역은 던전이라기보다 캠페인 생태계의 서로 다른 기후대에 가깝다.

## 반드시 고려할 문제

### 1. 전체 지역을 먼저 정해야 하는 이유

첫 지역만 확정하면 뒤 지역과 테마가 겹치거나, 서사 상승 곡선이 약해질 수 있다. 전체 지역 후보를 먼저 정하고, 각 지역이 어떤 캠페인 법칙을 보여주는지 구분해야 한다.

분석할 것:

- 전체 지역 수 초안
- 지역별 광고/소비/자동화 테마
- 지역별 감정 톤
- 지역별 인간/로봇/캠페인 갈등
- 지역별 보스가 상징하는 결절
- 지역 해금 순서
- 첫 지역 `외곽 주택가`가 전체 흐름에서 담당해야 할 역할

### 2. 지역은 클리어 후 끝나면 안 된다

이 게임은 런 기반 액션 RPG다. 지역은 한 번 클리어하고 버리는 스테이지가 아니라, 캠페인 농도, 회수 목표, 진영 상태, 보스 처리 결과에 따라 계속 재방문할 이유가 있어야 한다.

분석할 것:

- 보스 처치 후에도 지역을 다시 가야 하는 이유
- 지역별 반복 런 목표
- 지역 상태 변화 단계
- 캠페인 농도가 낮아진 지역의 새로운 위험
- 캠페인 농도가 높아진 지역의 보상 유혹
- 인간 우선/로봇 우선/공존/캠페인 활용 선택이 지역 재방문에 주는 변화

### 3. 지역 클리어의 의미를 다시 정의해야 한다

클리어는 지역을 완전히 정화한다는 뜻이 아닐 수 있다. 캠페인은 환경이므로, 한 번 보스를 잡았다고 지역이 끝나지 않는다.

가능한 정의:

- 결절 파괴: 지역 압력이 낮아지지만 일부 기억 손실
- 코어 해방: 로봇 자율성 상승, 지역 위험 일부 유지
- 기억 추출: 흔적 보존, 지역 변형 유지
- 신호 우회: 강한 보상, 캠페인 농도 장기 상승 위험

분석할 것:

- 각 처리 방식이 지역 상태에 주는 효과
- 보스 이후 지역이 어떻게 달라지는지
- 같은 지역을 다시 방문할 때 다른 미션이 열리는지
- 지역별 "완전 해결 불가능한 환경성"을 어떻게 유지할지

### 4. 지역 재사용은 단순 반복이면 안 된다

재방문은 숫자 파밍만이면 약하다. 플레이어가 "다시 가야 하는 이유"를 서사와 시스템 양쪽에서 느껴야 한다.

재방문 이유 후보:

- 새 흔적 회수
- NPC 부탁
- 특정 진영 안정화
- 캠페인 농도 조절
- 보스 선택 후 후유증 처리
- 해방 로봇 구조
- 침묵권 네트워크 확장
- 위험 카드나 변이 보상 획득
- 지역별 작은 결절 정화
- 다른 캐릭터 관점으로 재해석

분석할 것:

- 반복 런 목표 테이블
- 지역별 재방문 변형 이벤트
- 지역 상태에 따른 적/보상/대사 변화
- 반복 방문이 지루해지지 않게 하는 최소 장치

### 5. 전체 지역 후보를 제안하라

현재 후보:

- 외곽 주택가
- 폐쇼핑몰
- 방송국
- 놀이공원
- 로봇 서비스센터
- 위성 송신지
- 미래관/달 주택 홍보관

이 후보를 그대로 써도 되고, 더 나은 후보가 있으면 조정해도 된다. 단, 인디/모바일 규모에서 구현 가능해야 한다.

각 지역은 다음을 가져야 한다.

- 지역명
- 한 줄 테마
- 캠페인 법칙
- 주요 적 실루엣
- 보스 결절
- 주요 흔적
- 반복 방문 이유
- 인간 관점 의미
- 로봇 관점 의미
- 캠페인 관점 의미
- 구현 난이도

### 6. 첫 지역의 역할을 재검토하라

현재 첫 지역 초안은 `외곽 주택가`다. 첫 지역이 이대로 적절한지 검토하라.

검토 기준:

- 튜토리얼 지역으로 이해하기 쉬운가
- 캠페인 세계관을 가장 빠르게 보여주는가
- 인간 주인공의 출발점으로 적절한가
- 첫 보스와 잘 연결되는가
- 이후 지역과 테마가 겹치지 않는가
- 재방문 루프가 충분한가

필요하면 수정 제안을 하라. 다만 완전히 갈아엎기보다, 전체 지역 구조 안에서 역할을 더 선명하게 만드는 방향을 우선한다.

## 출력 요구

Markdown으로 작성하라.

다음 섹션을 반드시 포함하라.

1. 전체 판단
   - 현재 방향의 강점
   - 가장 큰 위험
   - 먼저 결정해야 할 것

2. 권장 지역 로드맵
   - 5~8개 지역
   - 해금 순서
   - 각 지역의 핵심 캠페인 법칙

3. 지역별 설계 표
   - 지역명
   - 테마
   - 주요 적
   - 보스
   - 흔적
   - 재방문 이유
   - 구현 난이도

4. 지역 상태 모델
   - 미방문
   - 결절 활성
   - 결절 처리 후
   - 안정화
   - 재침식
   - 특수 이벤트

5. 재방문 루프
   - 보스 후에도 지역을 다시 가야 하는 구조
   - 반복 런 목표 종류
   - NPC/진영/흔적과의 연결

6. 첫 지역 재검토
   - 외곽 주택가 유지/수정/교체 판단
   - 수정한다면 무엇을 바꿀지
   - 첫 보스와의 연결 방식

7. 개발 규모 조언
   - 0.2에서 할 것
   - 0.3에서 할 것
   - 확장팩까지 미룰 것

8. 금지 방향
   - 지역이 일회성 던전이 되는 것
   - 모든 지역이 같은 웨이브 스킨만 바뀌는 것
   - 지역 클리어가 캠페인을 완전히 제거하는 것
   - 거대한 AAA식 맵 구조를 전제로 하는 것
