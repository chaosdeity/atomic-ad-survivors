# RPG Growth Architecture v1

상태: 0.2 구현 전 기준서
작성 목적: 논의가 끝난 0.2 RPG 성장 구조를 구현 가능한 기준으로 고정한다.
주의: 이 문서는 수익화 상세 문서가 아니다. 수익화는 별도 문서에서 다루며, 이 문서는 성장 구조가 수익화와 충돌하지 않도록 연결 원칙만 남긴다.

## 0. Locked Decisions

이 문서는 아래 결정을 더 이상 논의 중인 가설로 다루지 않는다.

| 항목 | 결정 |
| --- | --- |
| 0.2 플레이어블 | 윤서 1명 |
| 윤서 자동공격 | 반품 스캐너 |
| 윤서 차징 | 반품 도장 |
| 첫 거점 | 침묵 보급소 |
| 첫 지역 | `r01_suburb / 외곽 주택가` |
| 첫 보스 | 스마일 홈 시어머니 |
| 보스 선택 | 결절 파괴, 기억 추출 |
| 반복 재화 | 광고 전단 조각 |
| 보스 재화 | 캠페인 코어 파편 |
| 스토리 단서 | 송출 기록 3개, 송출관 접근 절차 확정 |
| P0 흔적 | 지워지지 않은 문패, 반쯤 탄 가족사진, 이름 없는 영수증 |
| P1 흔적 | 고장 난 마스코트 배지, 납 차폐 배터리, 무음 라디오 |
| 첫 전투 종료 | HP 0 사망이 아닌 `signal_overload` 강제 회수 |
| 0.2 과금 | 없음 |

금지:

- `좌표 조각` 재화 추가
- `송출 기록` 소비 재화화
- 사망 자체 보상
- 보스 조우만으로 캠페인 코어 파편 지급
- 내부 상태 수치 UI 직접 노출
- 0.2에서 캠페인 송출관 직접 보스전 구현
- 0.2에서 코어 해방/신호 우회 선택 구현

핵심 문장:

```text
런은 매번 새롭고,
세계는 매번 누적된다.
```

다만 이 문장을 유저에게 직접 설명하지 않는다. 유저는 먼저 싸우고, 성장하고, 보급소로 돌아오고, 같은 지역이 조금 달라졌다는 사실을 느껴야 한다.

## 1. Scope And Boundaries

### 이 문서가 다루는 것

- 0.2 RPG 성장 루프
- 런 성과 정산 기준
- 광고 전단 조각과 캠페인 코어 파편의 역할
- 송출 기록과 흔적의 성장 연결
- 윤서 0.2 성장 계열
- 런 중 카드와 영구 성장의 분리
- 침묵 보급소 성장 최소 구조
- 외곽 주택가 로컬 생태권 성장
- 내부 상태 비노출 원칙
- 구현 작업 분해와 테스트 기준

### 이 문서가 다루지 않는 것

- 장비 세트 가챠
- 오퍼레이터 가챠
- 영구형 캠페인 카탈로그
- 월정액/패키지/코스튬 도감
- 확률/천장/상점/결제/법적 고지

수익화 상세는 별도 `MONETIZATION_ARCHITECTURE.md`에서 설계한다. 단, RPG 성장 구조는 추후 오퍼레이터/장비 세트 경제를 받을 수 있게 데이터 경계를 열어둔다.

## 2. Player Experience Goals

0.2는 시스템이 많은 게임처럼 보이면 안 된다. 유저가 첫 1시간 안에 느껴야 하는 것은 아래다.

1. 윤서는 기본 캐릭터지만 약하거나 밋밋하지 않다.
2. 반품 도장은 단일 버튼인데도 손맛이 있다.
3. 첫 회수는 죽음이 아니라 보급소가 끌어낸 사건이다.
4. 광고 전단 조각으로 바로 개조할 수 있다.
5. 오래 버티면 송출 기록과 흔적 같은 중요한 것을 얻는다.
6. 흔적을 보관/소모/캠페인 활용할 때 잠깐 고민한다.
7. 보스를 못 잡아도 HP 65%/25% 성과가 다음 시도에 의미를 만든다.
8. 보스를 처리해도 외곽 주택가는 끝나는 맵이 아니라 다르게 변한다.

유저에게 보여주면 안 되는 것:

```text
trace_preservation +10
human_stability +5
campaign_exploitation score
성향: 인간 우선
local_memory_profile
```

유저에게 보여줘야 하는 것:

```text
이름 보관함에 문패가 붙었다.
출격 게시판 문구가 바뀌었다.
광고 방송이 윤서를 더 정확히 부른다.
보스 외피 손상 기록이 남았다.
보급소 장비가 한 단계 더 안정됐다.
```

## 3. Growth Layers

RPG 성장은 네 층으로 나눈다.

```text
Run Layer
  이번 런의 카드, XP, 빌드, 전투 성과

Recovery Layer
  런 종료 정산, 회수물 확인, 성과별 보상 계산

Outpost Layer
  윤서/장비/보급소/흔적 선택 성장

World Response Layer
  외곽 주택가, 보스 선택, 지역 문구, 캠페인 적응 변화
```

핵심 원칙:

- 런 중 카드는 이번 런의 플레이 스타일을 바꾼다.
- 영구 성장은 다음 런의 바닥과 세계 반응을 바꾼다.
- 보급소는 상점이 아니라 해석과 정비의 장소다.
- 지역은 클리어 후 버리는 맵이 아니라 플레이어 행동을 먹고 변하는 로컬 생태권이다.

## 4. 0.2 Core Loop

```text
출격
-> 런 중 카드 선택
-> 자동공격/차징/회피로 성과 획득
-> 광고 전단 조각, 흔적, 송출 기록, 코어 파편 조건 달성
-> 회수/사망/보스 처리
-> 보급소 정산
-> 윤서/장비/보급소/지역 상태 변화
-> 재출격
```

회수/사망은 보상 원인이 아니다. 보상 원인은 생존 시간, 처치, 엘리트 처치, 신호 이벤트 성공, 흔적 회수, 보스 HP 성과, 보스 처리다.

### First 10 Minutes

목표:

- 이동/자동공격을 즉시 이해한다.
- 반품 도장을 처음 써보고 시원하다고 느낀다.
- 첫 전투는 약 100초 전후 `signal_overload`로 반드시 회수된다.
- 첫 회수 후 광고 전단 조각으로 개조 1개를 산다.

권장 흐름:

```text
0:00~0:30  쿠폰 전단 떼, 자동공격 학습
0:30~1:10  차징 윈도우와 반품 도장 학습
1:10~1:40  캠페인이 윤서 이름을 읽으려 함
~1:40      신호 과부하 강제 회수
보급소     첫 정산, 첫 개조
재출격     같은 지역이지만 더 잘 버팀
```

성공 감정:

```text
졌는데 끝난 게 아니라, 다음 출격이 바로 기대된다.
```

### First 30 Minutes

목표:

- 120초 신호 이벤트를 본다.
- P0 흔적 1개 이상을 회수한다.
- 보급소에서 흔적 선택과 시설 반응을 본다.
- 광고 전단 조각으로 2~3개 개조를 산다.

성공 감정:

```text
방금 중요한 걸 찾았고, 보급소가 그걸 알아봤다.
```

### First 60 Minutes

목표:

- 120/180/240초 신호 이벤트로 송출 기록 3개를 확보한다.
- 보급소에서 송출 기록을 해석한다.
- 스마일 홈 시어머니 조우 조건을 연다.
- 보스 HP 65%/25% 또는 처리 성과로 코어 파편을 얻는다.
- 보스 선택 후 외곽 주택가 재방문 문구가 달라진다.

성공 감정:

```text
이번엔 못 깼지만 다음엔 가능하다.
깼어도 이 지역은 아직 살아 있다.
```

## 5. Currency And Evidence Model

### Summary

| 종류 | 역할 | UI 노출 | 판매 원칙 |
| --- | --- | --- | --- |
| 광고 전단 조각 | 일반 성장 재료 | 노출 | 0.2 판매 없음. 장기적으로 대량 직접 판매 금지 |
| 캠페인 코어 파편 | 보스/고농도 결절 성과 재료 | 노출 | 대량 직접 판매 금지 |
| 송출 기록 | 스토리 단서 | 진행도 노출 | 판매/소비 금지 |
| 흔적 | 선택형 회수물 | 노출 | 재화가 아님 |
| 내부 상태 | 세계 반응 계산 | 비노출 | 판매 대상 아님 |

### 광고 전단 조각

정의:

광고 전단 조각은 캠페인 환경에서 떨어져 나온 판촉 물질이다. 침묵 보급소는 이를 정화해 역송출 장비의 절연재, 필터, 출력 보정판으로 쓴다.

획득처:

- 생존 시간
- 처치 수
- 엘리트 처치
- 신호 이벤트 성공
- 첫 강제 회수의 현장 데이터
- 특정 흔적 소모/캠페인 활용 보상

사용처:

- 윤서 반품 스캐너 개조
- 반품 도장 개조
- 회수선 안정화
- 정산 카운터/출격 게시판/이름 보관함 최소 성장

### 캠페인 코어 파편

정의:

캠페인 코어 파편은 보스급 결절의 반복 송출 핵이 깨질 때 떨어지는 고밀도 신호 잔해다. 표면 전단이 아니라 내부 부품이다.

획득처:

- 스마일 홈 시어머니 HP 65% 이하
- HP 25% 이하
- 보스 처리 성공
- 이후 고농도 결절 성과

사용처:

- 보스 대응 개조
- 상위 송출관 연구
- 일부 차징/외피 대응 개조

금지:

- 보스 조우만으로 지급
- 일반 DPS 대폭 상승 구매
- 전단처럼 반복 소비되는 소프트 재화화

### 송출 기록

송출 기록은 재화가 아니다. 캠페인 송출관 접근 절차를 여는 스토리 단서다.

구조:

```text
시간 도달
-> 신호 이벤트 출현
-> 목표 달성/생존/처치
-> 송출 기록 확보
-> 보급소 해석
-> 송출관 접근 절차 확정
```

0.2 송출 기록:

| 단계 | 이름 | 타이밍 | 의미 |
| --- | --- | --- | --- |
| 1 | 첫 잡음 영수증 | 120초 전후 | 캠페인 신호가 추적 가능한 구조임을 알림 |
| 2 | 길 잃은 카탈로그 | 180초 전후 | 신호가 방향과 경로를 가짐 |
| 3 | 개장 전 점검표 | 240초 전후 | 송출관 접근 절차 확정 |

## 6. Run Reward Formula v1

### 광고 전단 조각 지급 초안

| 조건 | 지급량 | 비고 |
| --- | ---: | --- |
| 45초 미만 | 0 | 빠른 죽음 파밍 방지 |
| 45~59초 | 0~1 | 첫 전투 예외 외에는 낮게 |
| 60초 생존 | +1 | 첫 의미 있는 생존 |

| 180초 신호 이벤트 성공 | +2 | 중반 성과 |
| 240초 신호 이벤트 성공 | +2~3 | 보스 접근 전 성과 |
| 엘리트 처치 | +1 | 60초 이후부터 유효 |
| 처치 수 보너스 | +1씩, 런당 최대 +3 | 60초 이후부터 적용 |

첫 강제 회수:

```text
기본 지급: 광고 전단 조각 3
성과 추가: 0~3
기대 지급: 3~5
```

첫 회수 후 유저는 반드시 개조 1개를 살 수 있어야 한다.

| 조건 | 지급량 | 반복 제한 |

| HP 25% 이하 첫 도달 | +1 | 보스별 최초 확정 |
| HP 25% 이하 반복 | 누적 2회당 +1 | 숙련 반복 보상 |
| 보스 처리 성공 | +2 | 선택 결과와 함께 지급 |
| 동일 보스 반복 처치 | +1 또는 전단/흔적 중심 | 코어 인플레 방지 |

## 7. Yunseo Growth Tree v1

0.2 윤서 성장은 3계열만 사용한다.

```text
반품 스캐너 출력
반품 도장 개조
회수선 안정화
```

UI는 거대한 스킬트리처럼 보이면 안 된다. 침묵 보급소의 정비대/차징 조율대에서 작은 개조를 고르는 느낌이 우선이다. 내부 데이터는 추후 오퍼레이터 성장 구조로 확장 가능해야 한다.

### 반품 스캐너 출력

| 개조 | 비용 | 효과 | 체감 |
| --- | ---: | --- | --- |
| 스캐너 출력 보정 I | 전단 2 | 자동공격 피해 +1 또는 안정성 증가 | 첫 잡몹 정리 개선 |
| 스캔선 초점 조정 | 전단 3 | 사거리 소폭 증가 | 이동 중 안정 |
| 가까운 반품 우선 | 전단 3 | 타깃 전환 낭비 감소 | 답답함 감소 |
| 판촉 외피 긁기 | 전단 4 | 엘리트 외피 누적 보정 소폭 | 보스 전 단계 대비 |

### 반품 도장 개조

| 개조 | 비용 | 효과 | 체감 |
| --- | ---: | --- | --- |
| 도장 압력 보정 I | 전단 2 | 차징 피해 +2 | 첫 손맛 강화 |
| 넓은 접수면 | 전단 3 | 전방 부채꼴 범위 소폭 증가 | 빗나감 스트레스 완화 |
| 반품 사유: 과장광고 | 전단 4 | 신호 오브젝트/엘리트 외피 추가 피해 | 목표 대응력 |
| 접수 후 둔화 | 전단 4 | 명중 적 짧은 둔화 | 컨트롤 보상 |
| 외피 균열 기록 | 코어 1 | 보스 외피 명중 피드백/효율 소폭 증가 | 보스전 차징 의미 강화 |

### 회수선 안정화

| 개조 | 비용 | 효과 | 체감 |
| --- | ---: | --- | --- |
| 회수선 절연 보강 I | 전단 2 | 신호 과부하 경고 시간 증가 또는 회수 직전 데이터 보존 | 실패 납득 |
| 릴 장력 조절 | 전단 3 | 치명 피해 후 짧은 무적/경고, 런당 1회 후보 | 초보자 안정 |
| 데이터 묶음 보존 | 전단 4 | 회수 시 일부 목표 진행 정보 보존 | 긴 런 실패 완화 |
| 침묵권 닻 | 전단 4 + 송출 기록 조건 | 신호 이벤트 목표 표시 안정화 | 180/240초 도전 보조 |

주의:

- 윤서는 기본 캐릭터지만 약하면 안 된다.
- 반품 도장 손맛은 0.2 최우선 polish 항목이다.
- 회수선 안정화는 사망 보상이 아니라 성과 보존 장치다.

## 9. Trace Growth

흔적은 재화가 아니다. 전투 효과와 거점/스토리 효과를 동시에 가진 감정적 아이템이다.

선택:

```text
보관
소모
캠페인 활용
```

| 선택 | 전투/기능 보상 | 거점/스토리 보상 | 감정 |
| --- | --- | --- | --- |
| 보관 | 즉시 전투 이득은 작음 | 이름 보관함, NPC 대사, 지역 반응 | 남겼다 |
| 소모 | 장비/시설 개조에 직접 도움 | 해당 흔적은 보관되지 않음 | 살기 위해 썼다 |
| 캠페인 활용 | 강한 보상/위험 카드/정산 효율 | 캠페인이 유저를 더 정확히 부름 | 유혹을 탔다 |

### P0 흔적

| 흔적 | 보관 | 소모 | 캠페인 활용 |
| --- | --- | --- | --- |
| 지워지지 않은 문패 | 이름 보관함 라벨, 일부 문패 변화 둔화 | 회수선/침상 안정 재료 | 0.2에서는 잠금 또는 비권장 |
| 반쯤 탄 가족사진 | 사진 잔향, 복희 대사, 기억 추출 반응 | 1회 보호/방어막 계열 재료 | 0.3 후보 |
| 이름 없는 영수증 | 빈 고객란 경고, 윤서 서사 씨앗 | 정산 효율/고위험 카드 재료 | XP/전단 보너스, 캠페인 농도 상승 |

보관이 손해처럼 보이지 않게 하려면 보관 즉시 오브젝트/대사/게시판 중 하나가 바뀌어야 한다.

## 12. Boss And Recovery

### First Forced Recovery

첫 전투는 아무리 잘해도 클리어 불가능하다. 이유는 플레이어가 약해서가 아니라 캠페인이 윤서를 `고효율 반응 고객`으로 등록하려는 순간 침묵 보급소가 회수선을 강제로 당기기 때문이다.

```text
forced_recovery_reason = "signal_overload"
```

첫 회수 후:

- 보급소 진입 보장
- 광고 전단 조각 3개 이상 지급
- 개조 1개 보장
- "죽음 보상"이 아니라 "첫 현장 데이터 정산"으로 표현

### First Boss

0.2 첫 보스는 `스마일 홈 시어머니`다. 캠페인 송출관은 상위 접근 목표이며 0.2 직접 보스가 아니다.

보스 성과:

| 조건 | 보상 |
| --- | --- |
| 조우만 함 | 코어 파편 0, 패턴 힌트 가능 |
| HP 65% 이하 | 캠페인 코어 파편 +1 |
| HP 25% 이하 | 캠페인 코어 파편 +1 |
| 처리 성공 | 캠페인 코어 파편 +2, 보스 선택 |

보스 선택:

| 선택 | 결과 |
| --- | --- |
| 결절 파괴 | 캠페인 농도 일부 감소, 일부 안정 반응 |
| 기억 추출 | 미스터리/가족사진/송출관 연구 강화 |

## 13. Hidden State Principle

내부 상태:

```text
human_stability
robot_autonomy
trace_preservation
campaign_exploitation
sync_risk
local_memory_profile
```

이 값들은 UI에 수치로 노출하지 않는다.

| 내부 변화 | 유저 표현 |
| --- | --- |
| `human_stability` 상승 | 침상 쪽 조명이 따뜻해짐 |
| `robot_autonomy` 상승 | 충전 스탠드 잡음 감소 |
| `trace_preservation` 상승 | 이름 보관함 오브젝트 증가 |
| `campaign_exploitation` 상승 | 원치 않는 쿠폰, 광고 문구 직접 호명 |
| `sync_risk` 상승 | 출격 게시판/차징 조율대 노이즈 |
| `local_memory_profile` 변화 | 외곽 주택가 배너/오브젝트/적 조합 변화 |

원칙:

```text
시스템은 계산한다.
세계는 현상으로 반응한다.
유저는 해석한다.
```

## 14. Monetization Linkage Boundary

0.2는 과금 없음이다. 그러나 RPG 성장 구조는 장기 수익화와 충돌하지 않게 설계해야 한다.

연계 원칙:

- 지역/스토리는 무료 무대가 된다.
- 런 안 재미는 무료 유저도 완성 경험을 해야 한다.
- 광고 전단 조각과 캠페인 코어 파편은 플레이 성과 재화로 보호한다.
- 송출 기록은 스토리 단서로 보호한다.
- 장기적으로 오퍼레이터/장비 세트/카탈로그/월정액이 들어와도 0.2 성장 루프를 무너뜨리면 안 된다.

수익화 문서로 넘길 것:

- 오퍼레이터 성장
- 장비 슬롯/세트 효과
- 코스튬 도감 스탯
- 영구형 캠페인 카탈로그
- 월정액/패키지
- 유료 재화와 마일리지

이 문서에서 수익화 상품의 확률, 가격, 천장, 상점 구조를 정의하지 않는다.
