# R01 Suburb Local Response Design

목적: `결절 파괴 / 기억 추출` 보스 outcome이 외곽 주택가 재방문, 침묵 보급소 문구, 출격 게시판 목표 문구에 어떻게 반영될지 최소 데이터 경계를 정한다.

범위: 설계 문서만 작성한다. 맵 구현, 지역 반응 코드 구현, 보스 선택 UI 구현, 장비/오퍼레이터/가챠/카탈로그 구현은 포함하지 않는다.

기준:

- 0.2 첫 보스는 `스마일 홈 시어머니`다.
- 보스 outcome 내부 값은 `destroy_node`, `extract_memory`만 사용한다.
- `캠페인 송출관`은 0.2 직접 보스가 아니라 상위 목표 / 후속 신호다.
- `송출 기록`은 재화가 아니라 스토리 단서다.
- `좌표 조각` 추가는 금지한다.

## 1. 목적

외곽 주택가는 클리어 후 버리는 맵이 아니라, 유저 행동을 기억하는 첫 로컬 생태권이어야 한다. 0.2에서는 큰 시뮬레이션 대신 짧은 문구, 오브젝트 표현, 출격 목표 문장만으로 "같은 곳인데 반응이 달라졌다"는 감각을 만든다.

이 설계의 목표는 세 가지다.

1. 보스 outcome이 다음 외곽 주택가 방문의 분위기를 바꾼다.
2. 보급소와 출격 게시판이 같은 상태를 서로 다른 말투로 반영한다.
3. 흔적 선택이 단순 보상 선택이 아니라 지역이 윤서를 기억하는 방식으로 이어진다.

유저에게는 내부 상태값을 직접 보여주지 않는다. 예를 들어 `r01_campaign_pressure = 2` 같은 표기는 UI에 나오지 않고, "광고음이 윤서의 이름을 더 또렷하게 끊어 읽는다" 같은 표현으로만 드러난다.

## 2. 지역 상태값 초안

0.2에서 확정할 최소 상태값:

| 상태값 | 타입 초안 | 역할 | UI 노출 |
| --- | --- | --- | --- |
| `r01_revisit_count` | int | 외곽 주택가 진입 횟수. 재방문 문구 분기용 | 수치 비노출 |
| `r01_signal_records_found` | int 0~3 | 송출 기록 확보 단계. 출격 게시판 목표 분기용 | "기록 1/3"처럼 진행 문구만 가능 |
| `r01_boss_outcome` | enum/nullable | `none`, `destroy_node`, `extract_memory` | 선택 결과 문장만 노출 |
| `r01_campaign_pressure` | int band | 광고 호명/압박 강도. 캠페인 활용과 outcome의 합산 결과 | 수치 비노출 |
| `r01_trace_preserved_count` | int | 보관한 P0 흔적 수. 보급소/지역 오브젝트 반응 | 수치 비노출 |
| `r01_trace_consumed_count` | int | 기능 성장에 쓴 P0 흔적 수. 잔향 약화/시설 안정 반응 | 수치 비노출 |
| `r01_campaign_used_count` | int | 캠페인 활용을 택한 P0 흔적 수. 강한 보상 후 광고 호명 강화 | 수치 비노출 |

보류할 상태값:

| 후보 | 판단 |
| --- | --- |
| 개별 집/문패별 기억 state | 0.2 과잉. 문패/사진/영수증 P0 흔적 단위로 충분하다. |
| 호감도/성향/윤서 심리 수치 | 금지. 지역 반응이 캐릭터 관계 시스템으로 번지면 0.2 범위를 넘는다. |
| 적응 타입별 세부 카운터 | 보스/지역 반응 P1 이후 후보. 0.2에서는 문구 분기만 둔다. |

설계 원칙:

- 상태는 계산용이며, UI는 결과 문구만 보여준다.
- `r01_campaign_pressure`는 위험을 느끼게 하는 숨은 band다. "압력 3"처럼 표시하지 않는다.
- `r01_trace_*_count`는 흔적 선택의 총량 반응을 위한 최소값이다. 개별 보상 지급 로직은 다음 작업에서 분리한다.
- `r01_signal_records_found`는 진행 단서다. 차감되거나 교환되는 자원이 아니다.

## 3. 보스 Outcome별 지역 반응

### `destroy_node`: 결절 파괴

요지: 중심 결절 일부를 침묵시켜 외곽 주택가의 광고 밀도가 낮아지고, 방어적 안정 반응이 열린다.

지역 반응:

- 캠페인 농도 일부 감소.
- 주택가 광고음 일부 감소.
- 일부 문패/집 오브젝트가 더 또렷해짐.
- 스마일 홈 관련 배너의 반복 주기가 느려짐.
- 모델하우스 앞 조명이 완전히 꺼지지는 않지만, 깜빡이는 시간이 길어짐.

보상/성장 방향:

- 안정 보상, 회수선 안정, 방어적 반응 쪽.
- 보급소 정비대/차징 조율대 문구가 "노이즈 감소", "필터 안정" 쪽으로 이동.
- 재방문 시 적/광고가 윤서를 덜 직접적으로 부르지만, 침묵의 빈자리가 불편하게 남는다.

문구 톤:

- 조용해졌지만 끝난 것은 아니다.
- 가족/기억의 비밀보다 "광고 구조가 한 겹 벗겨졌다"는 쪽이 우선이다.

### `extract_memory`: 기억 추출

요지: 결절을 완전히 죽이지 않고 안쪽 기억을 끌어내, 가족사진/송출관 연구/후속 신호가 강해진다.

지역 반응:

- 가족사진, 문패, 이름 보관함 관련 문구 강화.
- 캠페인 송출관 접근 절차와 후속 신호가 더 선명해짐.
- 일부 적/광고 문구가 윤서를 더 정확히 부름.
- 모델하우스 내부에서 "가족 구성"을 맞추려는 문장이 늘어남.
- 반쯤 탄 사진 냄새, 오래된 거실 먼지, 빈 고객란 같은 감각 표현 증가.

보상/성장 방향:

- 기록, 흔적, 후속 목표 쪽.
- 송출관 연구 문구와 출격 게시판 목표가 더 구체화됨.
- `r01_campaign_pressure`가 낮아지지 않거나 일부 상승할 수 있다. 다만 UI에는 수치가 아니라 호명/노이즈로 표현한다.

문구 톤:

- 더 위험하지만 더 많이 알게 된다.
- "가족"이라는 단어가 보상처럼 보이면서도 불편해야 한다.

## 4. 보급소 문구 반응

보급소 문구는 귀환 직후 정산 카운터, 이름 보관함, 출격 게시판 주변 짧은 문장으로 쓴다. 한 번에 여러 줄을 모두 띄우기보다 상태에 맞는 1~2줄을 후보에서 고른다.

### 결절 파괴 후

- 스마일 홈의 광고음이 한 겹 낮아졌다.
- 정비대 위 전단들이 덜 떨린다.
- 이름 보관함의 문패 글자가 오늘은 덜 번진다.
- 차징 조율대 필터가 늦게나마 숨을 고른다.
- 정산 카운터 아래로 작은 광고 재가 떨어졌다. 더는 노래하지 않는다.
- 출격 게시판의 모델하우스 사진 한 귀퉁이가 비어 있다.

### 기억 추출 후

- 이름 보관함 뒤편에서 낡은 가족사진 냄새가 난다.
- 출격 게시판에 송출관 접근 절차가 더 선명해졌다.
- 정산 카운터가 결과보다 이름을 먼저 읽으려 한다.
- 반쯤 탄 사진의 뒷면에 모르는 식탁 자리가 남아 있다.
- 보급소 조명이 잠깐 거실등처럼 따뜻해졌다가 꺼진다.
- 전단 더미 안쪽에서 윤서의 이름이 접힌 채 발견됐다.

### outcome 없음

- 외곽 주택가 신호는 아직 가족 광고의 외피를 유지 중이다.
- 출격 게시판이 모델하우스 방향을 제대로 붙잡지 못한다.
- 정비대 필터가 스마일 홈 노래를 아주 작게 흘린다.

## 5. 출격 게시판 반응

출격 게시판은 "다음에 무엇을 하면 되는지"를 암시한다. 내부 조건은 여러 개여도 UI 문장은 하나의 목표처럼 보이게 한다.

| 상태 | 목표 문구 후보 |
| --- | --- |
| 보스 outcome 없음 | 외곽 주택가의 신호 이벤트를 추적한다. |
| 보스 outcome 없음 | 모델하우스 방향 광고가 반복되는 지점을 확인한다. |
| 결절 파괴 | 스마일 홈의 낮아진 광고음 사이로 남은 결절 잔향을 확인한다. |
| 결절 파괴 | 또렷해진 문패들을 따라 모델하우스 잔해를 재조사한다. |
| 기억 추출 | 가족사진 뒤편의 송출관 접근 절차를 대조한다. |
| 기억 추출 | 윤서의 이름을 부르는 광고 문구의 출처를 확인한다. |
| 송출 기록 1/3 | 첫 잡음 영수증을 보급소 기록판에 고정했다. 다음 신호는 더 긴 광고 뒤에 숨어 있다. |
| 송출 기록 1/3 | 영수증 잡음이 모델하우스 방향을 한 번 가리켰다. |
| 송출 기록 2/3 | 길 잃은 카탈로그가 주택가 루프의 방향을 맞추기 시작했다. |
| 송출 기록 2/3 | 같은 집들이 같은 순서로 반복되지 않는다. 모델하우스 접근이 가까워졌다. |
| 송출 기록 3/3 | 개장 전 점검표가 송출관 접근 절차를 확정했다. 중심 결절을 노출시킨다. |
| 송출 기록 3/3 | 외곽 주택가 중앙의 모델하우스 결절에 접근할 수 있다. |
| 보스 처리 후 후속 목표 | 스마일 홈 결절 처리 기록을 바탕으로 캠페인 송출관의 후속 신호를 추적한다. |
| 보스 처리 후 후속 목표 | 외곽 주택가는 조용해졌지만, 송출관 쪽 신호는 아직 끊기지 않았다. |

문구 우선순위 초안:

1. `r01_boss_outcome`이 있으면 보스 처리 후 후속 목표를 우선한다.
2. 보스 처리 전에는 `r01_signal_records_found` 단계 문구를 우선한다.
3. 기록이 없으면 재방문/초기 신호 추적 문구를 쓴다.

## 6. 재방문 문구

재방문 문구는 외곽 주택가 진입 직후 짧게 보여준다. 수치 카운터를 보여주는 대신, 반복감을 문장으로만 전달한다.

| 조건 | 문구 후보 |
| --- | --- |
| 1회 | 같은 집들이 다시 줄을 맞췄다. |
| 1회 | 분명 지나온 울타리가 처음 보는 색으로 칠해져 있다. |
| 2회 | 환영 배너가 윤서를 모르는 척 흔들린다. |
| 2회 | 우편함들이 같은 시간에 입을 다문다. |
| 3회 | 단골 고객 배너가 먼저 켜졌다. |
| 3회 | 모델하우스 안내음이 길을 묻기도 전에 방향을 제시한다. |
| 4회 이상 | 주택가는 윤서의 걸음 간격을 외운 것처럼 불을 켠다. |
| 4회 이상 | 같은 집들이 조금씩 다른 가족사진을 걸어 둔다. |
| 보스 처리 후 공통 | 스마일 홈의 문구가 이전보다 작게 들린다. |
| 결절 파괴 후 | 광고 노래가 비는 자리마다 실제 문패가 보인다. |
| 결절 파괴 후 | 모델하우스 잔향이 한 박자 늦게 따라온다. |
| 기억 추출 후 | 가족사진 속 빈자리가 윤서 쪽으로 고개를 돌린다. |
| 기억 추출 후 | 주택가 방송이 윤서의 이름을 틀리지 않는다. |

## 7. 흔적과 지역 반응 연결

P0 흔적:

- 지워지지 않은 문패
- 반쯤 탄 가족사진
- 이름 없는 영수증

흔적 선택은 보관, 소모, 캠페인 활용 세 갈래다. 이번 작업에서는 구현하지 않고, 다음 단계가 참조할 문구/반응 후보만 둔다.

| 흔적 | 보관: 지역/보급소 오브젝트 변화 | 소모: 전투/시설 성장 반영 | 캠페인 활용: 강한 보상 + 호명 강화 |
| --- | --- | --- | --- |
| 지워지지 않은 문패 | 이름 보관함에 문패 라벨이 붙고, 외곽 주택가 일부 문패가 덜 번진다. | 회수선 안정/침상 보강 문구. 재방문 시 해당 집의 잔향은 약해진다. | 광고가 "윤서 고객님, 주소 확인 완료"처럼 더 정확히 부른다. |
| 반쯤 탄 가족사진 | 이름 보관함 뒤편에 사진 모서리가 남고, 기억 추출 후 문구 후보가 늘어난다. | 1회 보호/방어막 계열 성장의 재료로 해석. 사진 냄새 문구는 줄어든다. | 가족 세트/홈케어 광고가 윤서를 가족 구성원처럼 끼워 넣으려 한다. |
| 이름 없는 영수증 | 정산 카운터 옆 빈 고객란 경고가 생기고, 출격 게시판에 잡음 영수증 메모가 붙는다. | 정산 효율/초기 보급 안정 문구. 영수증 잡음은 보급소 필터에 흡수된다. | 전단/XP/위험 카드 쪽 보상 후보. 광고가 윤서의 구매 이력을 지어낸다. |

선택별 총량 반응:

| 조건 | 반응 후보 |
| --- | --- |
| 보관 수 증가 | 보급소 이름 보관함과 외곽 주택가 오브젝트가 더 인간적인 디테일을 되찾는다. |
| 소모 수 증가 | 보급소 시설 문구가 안정되지만, 외곽 주택가의 기억 잔향은 덜 남는다. |
| 캠페인 활용 수 증가 | 보상이 강해질 수 있으나 광고가 윤서를 더 정확히 부르고, 출격 게시판 노이즈가 늘어난다. |

주의:

- 흔적은 재화가 아니다.
- 보관이 손해처럼 보이지 않도록 보관 즉시 보급소/지역/게시판 중 하나가 반응해야 한다.
- 캠페인 활용은 매력적인 위험 선택이지, 필수 성장 루트가 아니다.

## 8. 구현 경계

다음 구현 때 필요한 최소 파일/모듈 후보:

| 후보 | 책임 |
| --- | --- |
| `scripts/local_response_state.gd` | R01 로컬 반응 상태 보관, outcome/재방문/흔적 선택 aggregate 계산 |
| `scripts/route_progression.gd` | 송출 기록 단계, 보스 접근 가능 여부, 출격 게시판 목표 우선순위 정리 |
| `scripts/meta_progression.gd` 일부 확장 | 세션/영구 진행 state와 기존 보급소 성장 데이터 연결 |
| `scripts/main.gd` 일부 연결 | 지역 진입/귀환 시 문구 요청 hook만 연결 |
| `scripts/hud_controller.gd` 또는 보급소 UI 계층 | 수치가 아닌 문구 표시만 담당 |

이번 문서 기준 다음 구현의 최소 흐름:

```text
run result / boss result / trace choice
-> meta progression update
-> local response state update
-> outpost phrase resolver / sortie board phrase resolver
-> UI phrase display
```

구현 시 경계:

- `local_response_state`는 보상 지급을 직접 하지 않는다.
- 출격 게시판은 state를 설명하지 않고 다음 목표 문구를 고른다.
- 보급소는 outcome과 흔적 선택의 감정적 반응을 보여준다.
- 맵은 나중에 state를 받아 오브젝트/문구를 바꾸는 소비자 역할만 한다.
