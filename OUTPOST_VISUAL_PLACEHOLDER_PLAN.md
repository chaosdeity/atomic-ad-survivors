# Outpost Visual Placeholder Plan

작성일: 2026-05-18
목표: 침묵 보급소가 단순 메뉴가 아니라 "돌아오는 장소"로 보이게 하는 visual placeholder 계획.
주의: 구현/에셋 생성 전 계획이다.

## 1. 핵심 방향

침묵 보급소는 회수 후 숫자를 사는 상점이 아니다. R01에서 들고 온 전단, 문패, 가족사진, 코어 조각이 잠시 조용해지는 곳이어야 한다.

화면 원칙:

- 한 화면 안에 시설 5개가 구분된다.
- UI 메뉴는 장소 위에 뜬다.
- 업그레이드 선택은 내부 수치보다 "정비/보관/출격 준비" 행위로 읽힌다.
- 보스 처리 결과가 시설 상태로 남는다.

## 2. 시설별 placeholder

| 시설 | 임시 visual 방향 | 필요한 아이콘/패널 | 상태 변화 후보 | 0.2 최소 표현 |
| --- | --- | --- | --- | --- |
| 정산 카운터 | 낡은 편의점/보급소 카운터, 영수증 프린터, 전단 정리함 | receipt, stamp, flyer stack | 첫 회수 후 영수증 증가, 보스 후 코어 조각 tray | 결과 화면에서 카운터 배경 + 정산 panel |
| 정비대 | 반품 스캐너/도장 수리대, 작은 조명, 공구 | scanner, stamp, wrench | 업글 구매 후 장비 부품/라벨 추가 | 업글 UI가 정비대 위에 뜸 |
| 이름 보관함 | 문패/사진/영수증 슬롯이 있는 금속 보관함 | nameplate, photo, locked slot | trace 획득 후 슬롯 점등, destroy/extract에 따라 색 변화 | 보유 흔적 summary와 연결 |
| 출격 게시판 | 다음 목표/송출 기록/모델하우스 방향을 붙이는 게시판 | route pin, radio, open-house sign | 송출 기록 1/2/3에 따라 문구/핀 이동 | 다음 목표 표시의 물리 장소 |
| 차징 조율대 | 반품 도장 타이밍/회수선 조율 장치 | metronome, stamp gauge, cable | charge 관련 업글 후 gauge 추가 | 차징 업글/도장 상태를 설명하는 작은 bench |

## 3. UI와 장소 연결

| UI | 장소 anchor | 연결 방식 |
| --- | --- | --- |
| 결과 정산 | 정산 카운터 | 결과 panel이 카운터 위 receipt처럼 출력 |
| 업글 구매 | 정비대 / 차징 조율대 | 선택지는 카탈로그 카드가 아니라 정비 항목처럼 표시 |
| 흔적 보유 | 이름 보관함 | trace count를 숫자만 보여주지 않고 슬롯/라벨로 표현 |
| 다음 목표 | 출격 게시판 | `다음 출격:` 문구를 게시판에 붙임 |
| 보스 처리 결과 | 이름 보관함 + 출격 게시판 | 결절 파괴/기억 추출 후 시설 한 곳 이상에 흔적 남김 |

## 4. 상태 변화 후보

### 첫 회수 후

- 정산 카운터에 찢긴 전단과 첫 영수증.
- 출격 게시판에 "첫 신호 이벤트 추적" pinned note.
- 정비대 조명 켜짐.

### 송출 기록 진행

- 출격 게시판의 open-house 방향 핀이 더 선명해짐.
- 이름 보관함에 잡음 영수증/문패 슬롯 예고.
- 정산 카운터의 기록 카드가 1장씩 늘어남.

### 보스 회수/처리 후

결절 파괴:

- 보관함 쪽이 조용해지고 일부 문패가 읽힘.
- 출격 게시판의 fake route 표시가 끊어진 선으로 바뀜.

기억 추출:

- 사진/영수증 슬롯이 더 밝지만 불안하게 점등.
- 게시판에 가족사진 잔향/후속 신호 hook.

## 5. 0.2 최소 구현 기준

P0 최소:

- 한 화면 outpost background placeholder.
- 시설 5개가 silhouette로 구분.
- 결과/업글/다음 목표 UI가 시설 중 하나에 시각적으로 anchor됨.
- 보스 처리 결과가 한 문장 이상 보급소 문구에 반영.

P1:

- 시설별 상태 frame 0/1.
- trace slot visual.
- 출격 게시판 pinned note 변화.
- 정산 카운터 receipt animation.

P2:

- NPC/동료 배치.
- 시설 확장/수리 단계.
- 고급 애니메이션과 컷인.

## 6. 금지/주의

- 보급소를 상점/가챠/카탈로그 화면으로 구현하지 않는다.
- 과금 UI처럼 보이면 안 된다.
- 내부 수치를 시설 위에 직접 노출하지 않는다.
- R01과 같은 과한 광고 색을 쓰지 말고, "침묵"과 "임시 안전"을 우선한다.
- 새 성장 시스템 구현은 이번 범위가 아니다.

## 7. PMO 판단 필요

- 보급소를 480x270 one-screen으로 먼저 만들지, 별도 hub scene으로 확장할지.
- 0.2에서 trace visual slot을 실제 구현할지, 문구/아이콘 placeholder로 둘지.
- 정비대와 차징 조율대를 UI상 같은 upgrade panel로 묶을지, 시설은 분리할지.
